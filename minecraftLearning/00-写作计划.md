# Minecraft 技术原理系列写作计划

> 本系列从**引擎与数据模型**视角拆解 Minecraft(以 Java 版为主,1.18+ 为基线)的底层原理:体素世界怎么组织、地形怎么靠种子和噪声生成、区块怎么渲染、世界怎么按 20 TPS 跑起来、数据怎么存盘,以及方块、方块实体、实体、流体、物品、生物群系、光照、注册表这一整套对象模型如何分层咬合。
> 它不是"生存攻略"也不是"红石教程",而是面向工程师的"Minecraft 是怎么被造出来的"。

---

## 核心定位

很多人玩了几百小时 Minecraft,却从没想过这些问题:

- "无限世界"为什么不会把硬盘撑爆?
- 同一块地里的麦子为什么有的先熟有的后熟?
- 为什么离开后远处的农场和红石机器会停摆?
- 床、箱子这种看起来普通的方块,凭什么是"方块实体"?
- 一个坐标上到底叠着多少层数据?

本系列只回答一类问题:

> **把 Minecraft 当成一个软件工程作品,讲清楚它的数据模型、存储格式、运行循环和渲染策略,以及这些设计为什么是这样取舍的。**

因此本系列默认选择:

| 维度 | 选择 | 原因 |
| --- | --- | --- |
| 版本基线 | Java 版,1.18+ | Anvil 存储、分段调色板、世界高度扩展后的现代形态 |
| 视角 | 引擎 / 数据结构 | 不讲玩法技巧,讲"底层怎么实现的" |
| 深度 | 机制 + 字节级 + 取舍 | 关键处下沉到字节布局与算法,并解释为什么这样设计 |
| 边界 | 不逐行读源码 | 讲清结构与原理,源码类名作为指针给出 |

---

## 三条主线

```
世界怎么"长出来"并被画出来   (体素 → 噪声地形 → 渲染)
   → 世界怎么"跑起来"并存下来  (Tick 系统 → 区块生命 → 存储格式)
   → 世界由"什么"构成          (方块/实体/流体/物品/… → 注册表 → 总图)
```

核心节奏:**前 8 篇打通"生成—运行—存储"的纵向链路,后 4 篇横向铺开整套对象模型并收束成一张架构总图。**

## 篇目设计 (21 篇)

### 第一层:世界的生成与呈现 (01-03)

- **01 体素世界、区块与无限地形的存法** (block/voxel 网格 / Chunk 与 16³ Section / 为什么不能逐方块渲染 / "无限世界"靠确定性生成而非全量存储的核心思路)
- **02 地形生成:种子、噪声与确定性世界** (Perlin/Simplex/OpenSimplex / 种子+坐标→确定输出 / 多层噪声叠加映射高度温湿度 / 生物群系与矿脉洞穴 / 分阶段生成流水线与 Status)
- **03 区块渲染:只画看得见的面** (face culling / 相邻实心面剔除 / 一个 chunk 合并成一个 mesh、一次 draw call / 改一格只重建该 chunk / 内存里的 PalettedContainer 与渲染的对偶)

### 第二层:世界怎么跑起来、又怎么存下来 (04-08)

- **04 Tick 系统:20 TPS 主循环与三类更新** (时间累加器与"追帧"/掉 TPS / scheduled tick 队列与水流红石延迟 / random tick 抽样与作物参差成熟 / 邻居通知级联与红石 / tile tick 上限)
- **05 区块的生命:加载、ticket 等级与模拟距离** (chunk 不是二元加载 / ticket level 距离场 / entity-ticking vs block-ticking vs border / simulation distance / 为什么远处机器停摆、区块加载器原理)
- **06 实体系统:脱离网格的自由对象** (Entity vs 方块的本质差异 / 浮点坐标与 AABB 碰撞逐轴裁剪 / 实体层级 / A\* 寻路与 Goal/Brain AI / 1.17 起实体单独存储 / 实体为何是 lag 头号来源)
- **07 存储(上):Region/Anvil 文件格式逐字节** (为什么打包成 region / 8KB 头:位置表+时间戳表 / 4KB 扇区对齐 / 长度+压缩类型字节 / zlib/LZ4 / 超大 chunk 外置 .mcc / 扇区分配与碎片)
- **08 存储(下):NBT 与 Palette 位压缩** (NBT 标签布局与类型表 / chunk NBT 现代结构 / 局部调色板 + bit-packed Long_Array / bitsPerEntry 规则 / 1.16 前后跨界与不跨界 / DataVersion 与版本迁移)

### 第三层:世界由什么构成 (09-12)

- **09 方块的三层:Block、BlockState 与方块实体** (Block 单例=类型/行为 / BlockState=属性组合与全局调色板 / 为何方块无逐实例数据 / BlockEntity:存数据 vs 特殊渲染 / 床为何是方块实体 / 箱子的物品+开盖动画+大箱子双方块配对)
- **10 世界里还有什么:流体、物品、生物群系、光照、粒子** (FluidState 与方块叠加、含水机制、独立 tick / ItemStack 寄生在容器 / Biome 4×4×4 粒度 / 光照双通道与洪水填充传播 / 粒子纯客户端)
- **11 注册表、标签、维度与修饰/信号/会话层** (Registry 是一切类型的总账 / 命名空间ID+数字ID / 冻结与进服同步 / 静态 vs 动态注册表 / Tag 分组 / 维度与存档 / 属性·效果·附魔修饰层 / 运行时信号 / 菜单会话层)
- **12 分层架构总图与一个方块的生命周期** (类型 vs 实例的正交轴 / 一格上叠多层数据 / "世界里的对象"有限集 vs "引擎系统"开放集 / 一个 16³ section 从生成→运行→存盘→加载的完整时序)

### 第四层:到达玩家与玩法系统 (13-21)

> 前 12 篇讲的是"世界是什么、怎么存、怎么跑"(几乎全是单机/服务端数据模型)。本层补三块盲区:世界怎么**到达玩家**(网络、渲染管线),建在数据模型之上的**玩法级系统**(红石、刷怪、世界生成进阶、指令),以及一条**横向对照**(Bedrock vs Java)。

- **13 网络与多人同步:集成服务端、预测与插值** (单机也跑集成服务端 / 服务端权威 / 协议分阶段与注册表同步 / chunk 包 / 实体插值与客户端预测、橡皮筋回正)
- **14 红石:信号传播、更新顺序与图灵完备** (0-15 衰减 / 中继器比较器 / 邻居级联与硬弱供能 / quasi-connectivity / 更新顺序与 0-tick/BUD/observer / 逻辑门到红石计算机 / Java vs Bedrock 差异根源)
- **15 刷怪机制:mobcap、光照条件与 ticking 耦合** (刷怪循环 / 分类 mobcap 与区块数成比例 / 位置/光照/距离条件 / despawn 与 persistent / 与 ticking 范围耦合 / 刷怪塔原理)
- **16 世界生成进阶:密度函数 router、多噪声生物群系与拼图结构** (1.18 密度函数 router / multi-noise 六气候参数 / 生物群系与地形解耦 / spline 高度 / aquifer 与噪声洞 / jigsaw 结构生成)
- **17 数据组件与 DataFixerUpper:物品新模型与版本迁移** (1.20.5 物品=类型化组件映射 / 默认组件与差异覆盖 / 为何取代 NBT tag / DFU 按 DataVersion 重写升级链 / 走向强类型可迁移)

- **19 性能诊断:lag 来源、profiler 与优化生态** (TPS vs FPS vs MSPT / 掉 TPS 来源拆解 / spark/tick/F3 / Lithium·Starlight·C2ME·Sodium / 先测后优的方法论)

## 贯穿视角

全系列围绕一个反复出现的核心洞察展开,而不是堆砌孤立知识点:

> **Minecraft 的省与快,几乎都来自同一招——"能算出来的就不存,能共享的就不复制"。** 地形靠种子+噪声确定性生成(不存)、方块靠 BlockState 共享+调色板下标(不复制)、内存结构与磁盘结构同构(不转换)。

| 篇目 | 这一招的体现 |
| --- | --- |
| 01 / 02 | 无限地形不存,靠种子+噪声现算 |
| 03 / 08 | 内存 PalettedContainer 与磁盘 palette 同构,加载几乎零转换 |
| 07 / 08 | 只存改过的 chunk;局部调色板把均质地形压到近乎为零 |
| 09 | 方块无逐实例数据,千万个石头共享一个 BlendState 引用 |

## 单篇写作模板

每篇尽量遵循固定结构,避免变成散点知识:

```text
1. 问题定位
   - 这个机制要解决体素世界里的什么核心难题
   - 朴素做法为什么扛不住(性能 / 存储 / 一致性)

2. 设计与结构
   - 核心数据结构、对象或文件布局
   - 关键算法或运行流程

3. 下沉细节
   - 字节级布局 / 位运算 / 调度顺序等关键处展开
   - 版本演进带来的差异

4. 取舍与边界
   - 为什么这样设计、牺牲了什么换了什么
   - 常见误解与反直觉点

5. 串联
   - 与前后篇、与"能算不存、能享不复制"主线的呼应
```
