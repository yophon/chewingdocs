# 世界生成进阶:密度函数 router、多噪声生物群系与拼图结构

第 02 篇讲地形生成时,给的是一套"直觉版"心智:多层噪声叠出高度,坐标加种子确定性出图。那套心智没错,但它停留在 2017 年的 Minecraft。1.18(Caves & Cliffs Part II)把整个世界生成**推倒重写**了一遍,今天真实跑的管线跟第 02 篇的直觉差得很远。

这一篇就讲重写之后的真实架构。核心就一句话:

> **1.18 之后,世界生成不再是"几个噪声函数硬编码出高度",而是一棵数据驱动的密度函数树 + 一个把气候参数映射到生物群系的多维查表。地形、生物群系、洞穴被彻底解耦,各算各的,最后叠在一起。**

如果你只记住一件事:**生物群系不再决定地形高度了**。这是 1.18 最反直觉、也最关键的变化。

---

## 一、从"硬编码噪声"到"密度函数 router"

老版本(1.17 及以前)的地形生成大致是:每个生物群系自带一组高度参数(基准高度、起伏度),噪声采样后按生物群系的参数缩放,拼出地表。问题很明显:

- 生物群系边界处高度参数突变,需要大量插值去抹平接缝;
- 想加一种"悬崖"或"超深洞穴",得改硬编码逻辑;
- 山脉只能跟着某个生物群系走,做不出"一座山横跨好几个生物群系"。

1.18 的答案是 **density function(密度函数)**。

### 密度是什么

把世界想成一个 3D 标量场:空间里每一点 `(x, y, z)` 都有一个**密度值**。规则极简单:

```
density(x, y, z) > 0   →  这一点是实心(放方块,默认石头)
density(x, y, z) <= 0  →  这一点是空气(或在海平面下放水)
```

地形长什么样,完全由"密度场在哪里翻正负号"决定。地表就是密度场的零等值面(iso-surface)。

### router:把密度场拆成可组合的节点

density function 不是一个巨大公式,而是一棵**可组合的运算节点树**,这套配置叫 **noise router**。节点种类大致是:

```
噪声源:    noise / shifted_noise / blend_density ...
运算:      add / mul / min / max / abs / clamp / 常数 ...
样条:      spline(把一个输入按控制点映射成另一条曲线)
特殊:      y_clamped_gradient(随高度线性变化)/ interpolated / cache_* ...
```

`cache_2d`、`cache_once`、`flat_cache` 这类节点是**性能要害**:同一列、同一区块内反复用到的子表达式被缓存下来,不重复采样噪声。世界生成是 CPU 热点,这些缓存节点决定了它能不能跑得动。

最关键的一点:**整套 router 是数据驱动的**。它写在 `worldgen/noise_settings/*.json` 里,是第 11 篇讲的**动态注册表**条目——数据包可以替换、客户端进服时同步。换句话说,**不写一行 Java、改 JSON 就能造一个完全不同地形规则的世界**。1.17 想都不敢想。

> 反直觉点:`overworld` 的 router 是一棵相当大的 JSON 树(几百行)。你以为"地形生成"是一段算法,其实它现在更像一份**配置**。算法只是那个"求值 density function 树"的解释器。

## 二、高度从哪来:continentalness / erosion / weirdness 与样条

密度场的主干靠三个低频噪声塑形,它们就是地形的"骨架参数":

| 参数 | 直觉含义 | 影响 |
| --- | --- | --- |
| **continentalness** | 大陆性:离海洋有多"内陆" | 低 → 海洋;高 → 内陆高地 |
| **erosion** | 侵蚀度:地势被磨平的程度 | 低 → 山脉/崎岖;高 → 平原/平坦 |
| **weirdness** | 怪异度(经过 PV 折叠) | 制造山峰山谷的反复、河流 |

这三个值不是直接当高度用的,而是喂进一组 **spline(样条)**:样条就是"给一串控制点,把输入插值成输出曲线"。比如 continentalness 经过样条后,在"海洋段"输出很低的基准高度,过了海岸线急剧抬升,内陆段缓慢爬高。erosion 的样条再在此基础上决定这块地是揉成尖山还是压成平原。

```
continentalness ─┐
erosion ─────────┼─► 多级 spline ─► 基准地表高度 + 起伏
weirdness ───────┘                    │
                                       ▼
                         喂回 density function,决定密度场零点在哪一层
```

样条嵌套样条,是 1.18 地形"既有大平原又有尖锐山脊"的来源。调地形,本质就是调这些控制点。

## 三、生物群系:6 维气候参数的最近邻查表

这是 1.18 跟老版本心智差别最大的地方。

老版本:**生物群系决定地形**(沙漠是平的、山地是高的)。
1.18:**地形和生物群系都由噪声独立算出,生物群系只决定"长什么样"(方块、植被、刷怪),不决定高度。**

生物群系由 **multi-noise(多噪声)** 选取。每个点采样 **6 个气候参数**:

```
temperature      温度
humidity         湿度
continentalness  大陆性   ← 和地形共用同一个噪声
erosion          侵蚀度   ← 和地形共用同一个噪声
depth            深度(离地表多深,让同一地表下不同高度是不同群系)
weirdness        怪异度   ← 和地形共用同一个噪声
```

这 6 个值构成一个 **6 维空间里的坐标**。每个生物群系在这个空间里占一个**矩形区域**(对每个参数给一个区间)。选群系的算法就是:**拿当前点的 6 维坐标,在所有生物群系里找"距离最近"的那个**(实现上是一棵 n 维搜索树)。

这套设计的威力:

- 地形(山/谷)和气候(冷/热/干/湿)**正交**,于是"一座雪山的阳面是针叶林、背面是冻原"这种过去做不到的组合自然出现;
- depth 参与选群系,意味着**同一个 x/z 列,地表是草原、洞里是溶洞群系**——洞穴生物群系(繁茂/滴水石)就是这么来的;
- 加一个新生物群系 = 在 6 维空间里划一块没被占的区域,不用碰地形代码。

> 关键澄清:很多人以为"1.18 山更高"是因为山地生物群系改了。不是。是 continentalness/erosion 的样条把基准高度拉高了,山地群系只是恰好被铺在高的地方。**地形不知道自己脚下是什么生物群系。**

## 四、洞穴与含水层:独立的密度函数

老洞穴是"地表生成完,再用一套蠕虫算法(carver)挖隧道"。1.18 的奶酪洞/面条洞是**直接长在密度场里**的——router 里有专门的噪声节点,在地下把密度压成负值,凿出大空腔(奶酪洞)和细长隧道(面条洞)。它们不是"挖出来的",而是"密度本来就没填上"。

**含水层(aquifer)** 是配套机制:地下空腔不再默认是空气,而是由一套独立噪声决定**局部水位**——所以你会在洞里遇到孤立的小水潭、小岩浆湖,高度还各不相同。没有含水层的话,深洞要么全空要么全淹,1.18 的"地下湿润感"就是含水层给的。

```
最终方块 = 主地形密度  ⊕  洞穴密度(把密度往负压)  ⊕  含水层(决定空腔填水/岩浆/空气)
```

三套噪声各算各的,最后合成。解耦的代价是 router 更复杂,收益是每一块都能单独调、单独被数据包替换。

## 五、结构生成:jigsaw 拼图

地形是连续噪声,**结构是离散的、要"拼"出来的**——村庄、前哨站、远古城市、要塞,走的是完全不同的一套系统:**jigsaw(拼图)**。

思路像乐高:

1. 结构由一堆**模板块**组成,每块是一个 `.nbt` 结构文件(一段预先保存的方块 + 方块实体快照,见第 08 篇 NBT)。
2. 模板块边缘埋着 **jigsaw 方块**,每个 jigsaw 声明自己的名字和"我能接什么名字"。
3. 从一个起始块开始,按 jigsaw 接口**递归往外拼**:村庄的"路"接"房子",房子接"装饰",直到达到深度上限。每次拼接带随机选择,于是每个村庄都不一样。

外层还有几层数据驱动配置(都在动态注册表里):

| 概念 | 作用 |
| --- | --- |
| **structure**(`worldgen/structure/`) | 这个结构是什么类型、用哪套 jigsaw 模板池、能出现在哪些生物群系 |
| **structure_set**(`worldgen/structure_set/`) | 一组结构的**分布规则**:间距、随机散布、salt(避免不同结构生成位重合) |
| **template_pool**(`worldgen/template_pool/`) | jigsaw 拼接时每个名字能抽到哪些 `.nbt` 模板、权重多少 |

**结构在什么时候定下来**:回到第 02/05 篇讲的 chunk **Status 流水线**。结构不是地形长完才放,而是在很早的 `structure_starts` 阶段就**确定好包围盒和拼图布局**(此时邻居 chunk 可能还没生成),后续 chunk 生成到对应阶段时再把模板块"盖章"进去。这就是为什么村庄能跨好几个 chunk 而不断裂——它的"图纸"在任何一块地表长出来之前就画好了。

> 反直觉点:结构生成必须**先于**它所占的地表细节,否则跨 chunk 的结构没法保证一致。代价是 chunk 生成有严格的阶段依赖(邻居要先到某阶段),这也是远处区块加载慢、区块加载器要钉住一圈邻居的原因之一(第 05 篇)。

## 六、和第 02 篇怎么对上

别把这篇当成推翻第 02 篇,而是**同一件事的两层**:

| 维度 | 第 02 篇(直觉) | 本篇(真实管线) |
| --- | --- | --- |
| 地形 | 多层噪声叠出高度 | density function 树求值,密度零点即地表 |
| 高度由谁定 | 生物群系参数 | continentalness/erosion/weirdness 经样条,**与生物群系无关** |
| 生物群系 | 温湿度二维查表 | 6 维气候参数最近邻 |
| 洞穴 | "挖出来" | 独立密度函数直接长出来 + 含水层 |
| 结构 | (没细讲) | jigsaw 拼图,structure_starts 阶段定图纸 |
| 可改性 | 改代码 | 改 JSON(动态注册表 + 数据包) |

第 02 篇回答"为什么无限世界不用存、走到哪算到哪",那套**确定性 = 种子 + 坐标 → 固定输出**的本质没变;本篇只是把"算"这一步,从"几个硬编码噪声"换成了"一棵数据驱动的密度函数树 + 一套解耦的气候/结构系统"。省与快的主线(能算不存)依旧成立,只是算的方式工程化了一大截。

---

## 小结

- 地形 = density function 树求值,密度 > 0 为实心;整棵树是数据驱动的 noise router(JSON,动态注册表)。
- 高度由 continentalness/erosion/weirdness 经样条决定,**和生物群系彻底解耦**——1.18 最反直觉的点。
- 生物群系 = 6 维气候参数在多维空间里找最近邻,只管"长什么样",不管高度;depth 维度让地下能有独立洞穴群系。
- 洞穴和含水层是独立密度函数,"长出来"而非"挖出来"。
- 结构走 jigsaw 拼图:`.nbt` 模板 + jigsaw 接口递归拼接,structure_set 管分布;在 chunk 的 structure_starts 阶段就定好图纸,所以能跨 chunk 不断裂。
