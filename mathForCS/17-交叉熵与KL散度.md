# 17 · 交叉熵与 KL 散度:用错码本要多花多少 bit

接着 16 篇。**熵 H 是"用最优码本"的下界,但你常常没有最优码本——你只有一个估计的分布 q,真实分布是 p**。这时候平均码长会是多少?**多出来的部分**就是 KL 散度,**总长度**就是交叉熵。

听起来像信息论玩具,但**今天 95% 的分类模型损失是交叉熵,RLHF 的策略约束是 KL,知识蒸馏的目标是 KL,VAE 的正则项是 KL,DPO 的核心是 KL**——这一篇不读懂,这堆模型的训练你都只能抄不能调。

> 一句话先记住:**H(p,q) = H(p) + KL(p||q)**。**交叉熵 = 真分布的熵 + 你用错码本多花的 bit**。损失下不去时,要么 p 本身熵高(数据噪声大),要么 q 离 p 远(模型拟合差)。

---

## 一、为什么必须懂这两个

| 场景 | 不懂时 | 懂之后 |
| --- | --- | --- |
| 分类训练 loss 长这样:`-y log p` | 抄就完了 | 知道这是 H(p,q) 在最小化 |
| 多分类用 CE 不用 MSE | "默认这么写" | MSE 在 softmax 后梯度太小,CE 没这问题 |
| RLHF 里有个 `β · KL(π‖π_ref)` | 调 β 玄学 | 知道 β 越大策略越保守 |
| 蒸馏要不要用 KL 而非 MSE | 看心情 | KL 保留概率结构,MSE 抹掉了 |
| VAE 损失里那个 KL 项 | 跳过 | 那是 reverse KL,会让 q 收缩到一个 mode |
| 数值不稳出 NaN | 加 `1e-8` 续命 | log-sum-exp 一行解决 |

> **经验法则**:**任何"两个分布对齐"的问题,默认第一反应是交叉熵或 KL**。其他度量(MSE / Wasserstein / JS)都有更窄的适用范围,熟练之后再考虑替换。

---

## 二、交叉熵:用 q 的码本编 p 的事件

回到 16 篇的"码本"视角:

```
H(p)   = -Σ p(x) log p(x)
         用 p 自己最优的码本编 p 的事件,平均码长(下界)

H(p,q) = -Σ p(x) log q(x)
         用 q 设计的码本去编 p 的事件,平均码长
         (因为概率小的 x 在 q 看来"长码",一旦 p 让它常出现就吃亏)
```

举个具体的:

```
真分布 p:  (0.5,  0.5)        → H(p) = 1 bit
估计 q:    (0.9,  0.1)        → 给 x1 短码、给 x2 长码

H(p,q) = -(0.5·log 0.9 + 0.5·log 0.1)
       = -(0.5·(-0.152) + 0.5·(-3.32))
       = 1.736 bit

→ 多花了 0.736 bit/事件,这就是 KL
```

> **核心直觉**:**q 错估了哪些事件常出现,就在那些事件上付编码长度的代价**。p 高、q 低的地方代价最大——所以 KL 对"q 在重要区域的低估"特别敏感。

---

## 三、KL 散度:多花的 bit

定义直接拼出来:

$$
\text{KL}(p \| q) = H(p,q) - H(p) = \sum_x p(x) \log \frac{p(x)}{q(x)}
$$

| 性质 | 数学 | 工程含义 |
| --- | --- | --- |
| 非负 | KL ≥ 0 | 用错码本只会变贵,不会变便宜 |
| 为 0 当且仅当 p=q | KL = 0 ↔ p=q a.s. | 完全拟合时无浪费 |
| **不对称** | KL(p‖q) ≠ KL(q‖p) | **方向选错训练目标就反了** |
| 不满足三角 | 不是真距离 | "KL 距离"是错叫法 |
| q 在 p > 0 处必须 > 0 | 否则 KL = ∞ | 这就是 forward KL "覆盖"行为的来源 |

> **避坑**:**它叫"散度"不叫"距离"是有原因的**。听人讲"两个分布的 KL 距离"立刻警觉——他大概率没分清方向。

---

## 四、非对称的工程含义:forward vs reverse

这一节是这篇文章的核心。**KL(p‖q) 和 KL(q‖p) 在训练里的行为完全不同**,光记公式没用,必须看图。

```
真分布 p 是双峰高斯混合:

p(x)
  │       __                      __
  │      /  \                    /  \
  │     /    \                  /    \
  │____/      \________________/      \____
  └─────────────────────────────────────── x


用单峰高斯 q 去拟合,有两种方向:

【Forward KL: minimize KL(p || q),又叫 mode-covering / 0-avoiding】
  在 p>0 的地方 q 必须>0 (否则 log(p/q)=∞),
  → q 被迫"摊开"覆盖两个峰

  q
  │      _____________________
  │     /                     \
  │ ___/                       \___
  └─────────────────────────────────  q 拉得很宽


【Reverse KL: minimize KL(q || p) ,又叫 mode-seeking / 0-forcing】
  在 q>0 的地方 p 也得>0(否则 log(q/p)=∞),
  → q 龟缩到 p 的一个峰里,稳赚不赔

  q
  │       __
  │      /  \
  │     /    \
  │____/      \_______________________
  └─────────────────────────────────── q 钉在一个峰
```

**口诀**:

> **核心直觉**:**Forward KL 让 q 把所有 mode 都罩住(怕漏),Reverse KL 让 q 缩进一个 mode 里(怕错)**。前者保守(过度估计),后者激进(漏估)。

代码看一下:

```python
import numpy as np
import scipy.stats as st

# 双峰真分布 p (混合高斯)
xs = np.linspace(-6, 6, 600)
p = 0.5*st.norm(-2, 0.6).pdf(xs) + 0.5*st.norm(2, 0.6).pdf(xs)
p = p / p.sum()

def fit_gaussian(direction='forward'):
    best, best_kl = None, np.inf
    for mu in np.linspace(-3, 3, 31):
        for sigma in np.linspace(0.4, 3.0, 27):
            q = st.norm(mu, sigma).pdf(xs); q = q/q.sum()
            if direction == 'forward':   # KL(p || q) :对 q 的"漏"惩罚大
                kl = (p * np.log((p+1e-12)/(q+1e-12))).sum()
            else:                         # KL(q || p) :对 q 的"妄"惩罚大
                kl = (q * np.log((q+1e-12)/(p+1e-12))).sum()
            if kl < best_kl: best_kl, best = kl, (mu, sigma)
    return best, best_kl

print(fit_gaussian('forward'))   # mu≈0, sigma≈大,把两个峰都罩住
print(fit_gaussian('reverse'))   # mu≈-2 或 2,sigma小,缩进一个峰
```

| 算法 / 模型 | 用的方向 | 为什么 |
| --- | --- | --- |
| **EM 算法** | forward KL | 真实是混合分布时,要求 q 覆盖所有 mode |
| **变分推断 / VAE** | reverse KL | 算 KL(q‖p) 才能用蒙特卡洛估计(p 是后验,只能采样) |
| **MLE 训练 LM** | forward KL(等价) | 最大化 log-likelihood = 最小化 KL(p_data‖p_model) |
| **RLHF / PPO** | reverse KL(对 SFT) | 限制新策略不要离参考策略太远 |
| **GAN** | 实际接近 reverse | 这就是 GAN 出 mode collapse 的根源 |

> **经验法则**:**生成模型容易 mode collapse 的本质是优化目标里有 reverse KL**。看到一个新生成模型只产单一风格,先去看它的 loss 是不是逆向 KL 主导的。

## 五、分类损失为什么是交叉熵

K 类分类的真分布 p 在 one-hot 下:**只有真标签那一位是 1,其他都是 0**。

```
y = 真标签是第 c 类       → p = [0,...,1,...,0]  (第 c 位为 1)
ŷ = 模型预测概率向量      → q = softmax(logits)

H(p, q) = -Σ p_i log q_i
        = -log q_c        ← 只剩真标签那一项

这就是 PyTorch 的 nn.CrossEntropyLoss 内部做的事
```

整个数据集上平均:

$$
\mathcal{L} = -\frac{1}{N}\sum_{n=1}^{N}\log q_{c_n}
$$

这同时也是 MLE(最大似然):**最大化 log P(y|x) = 最小化 -log P(y|x) = 最小化 CE**。两边接通了。

为什么不用 MSE?

```
softmax + MSE 的梯度:    ∂L/∂z_i = (q_i - p_i) · q_i (1 - q_i)
                                              ↑
                                  训练后期 q_i ≈ 0 或 1,梯度趋 0,学不动

softmax + CE   的梯度:    ∂L/∂z_i = q_i - p_i
                                  
                                  线性差,永远不会消失,梯度健康
```

> **核心直觉**:**softmax + cross_entropy 是天作之合,梯度形式干净到惊人**。这也是 PyTorch 把它们合并成 `cross_entropy` 一个 op 的原因——既数值稳定,又少一次 softmax 计算。

## 六、log-sum-exp:不写好这一步,训练直接 NaN

直接算 softmax + log:

```python
log_softmax = torch.log(torch.softmax(logits, dim=-1))
# logits 含很大的正数(比如 1000)→ exp(1000) → inf → log(inf) = inf
# logits 含很大的负数(比如 -1000)→ exp 接近 0 → log(0) = -inf
```

修法:**减最大值,数学上等价,数值上稳定**:

```python
m = logits.max(dim=-1, keepdim=True).values
shifted = logits - m
log_sum_exp = m + torch.log(torch.exp(shifted).sum(dim=-1, keepdim=True))
log_softmax = logits - log_sum_exp     # 安全
```

这个技巧叫 **log-sum-exp**,PyTorch 里 `F.log_softmax` / `F.cross_entropy` / `torch.logsumexp` 都默认这么做。

> **避坑**:**手写损失时只要中间出现 `log(exp(...))`,就一定要换成 logsumexp**。这一招省下来的 NaN 排查时间是数学回报最高的工程技巧之一。

## 七、RLHF 的 KL 罚项:让 RL 后的策略别野

PPO/RLHF 的目标函数(简化):

```
L(π) = E[ r(x,y) ]  -  β · KL( π(·|x) ‖ π_ref(·|x) )
       └─ 让 reward 大     └─ 让新策略别离 SFT 太远
```

| 调 β | 行为 |
| --- | --- |
| β = 0 | 只追 reward,容易 reward hacking、风格崩坏、答非所问 |
| β 适中(常见 0.01-0.1) | 在原模型周围搜更高 reward 的解 |
| β 极大 | KL 接近 0,策略几乎不动,RL 没效果 |

> **核心直觉**:**β 是"探索-保守"的旋钮**。Reward model 不可靠时调大,reward 信号清晰时调小。

注意方向:这里通常是 **reverse KL `KL(π‖π_ref)`**,意味着模型不能在 `π_ref` 概率为 0 的地方有概率(即"不会冒出参考模型从来不说的话")——这正是想要的"保持原模型风格"。

```python
# trl 里大致这样(伪代码)
logp_new = policy_model.log_prob(y, x)
logp_ref = ref_model.log_prob(y, x).detach()
kl = (logp_new - logp_ref).mean()        # 蒙特卡洛估计 KL(π||π_ref)
loss = -reward.mean() + beta * kl
```

`trl` 里实际还会用一个**自适应 KL 控制器**,让训练时的 KL 维持在目标范围,避免崩飞。

## 八、DPO:把 KL 直接吸进 loss

DPO(Direct Preference Optimization)2023 年红了一把,**它绕开了 RL,直接用偏好对训练**。核心是把 RLHF 的"reward + KL 罚项"那个最优解直接代入,得到一个看起来像分类的 loss:

```
L_DPO = -E[ log σ( β · ( log π(y_w|x)/π_ref(y_w|x)
                       - log π(y_l|x)/π_ref(y_l|x) ) ) ]

    y_w 是人偏好的回答, y_l 是不偏好的
    σ 是 sigmoid
```

KL 在哪儿?**藏在 β 里**。β 大 → 整体行为接近 RLHF 的"严限制";β 小 → 接近"放飞",和 RLHF 调 β 直觉一致。

> **经验法则**:**RLHF / DPO 调 β 是同一回事**——都是控制新策略和参考策略的 KL。换工具不换数学直觉。

## 九、知识蒸馏:把老师的概率分布灌给学生

蒸馏(Hinton 2015)的目标:**让学生模型的输出分布逼近老师模型的输出分布**,而不是只学硬标签。

```
硬标签:  [0, 1, 0, 0, 0]              ← 信息少
老师概率: [0.05, 0.7, 0.1, 0.05, 0.1]   ← 信息多(谁像谁,差多少)

蒸馏 loss = α · KL(p_teacher^T ‖ p_student^T)  +  (1-α) · CE(y, p_student)
                                ↑ T 是 temperature,让分布更平,梯度更丰富
```

为什么用 KL 不用 MSE?

| 视角 | KL | MSE |
| --- | --- | --- |
| 概率向量 | 自然处理 | 把它当成普通向量,丢掉概率结构 |
| 数量级 | 对小概率敏感 | 大概率主导 |
| 梯度 | softmax + KL = 干净的差 | 同 softmax + MSE,梯度饱和 |

> **避坑**:**温度 T 升高时,蒸馏 loss 要乘 T²**,因为 logits 被除了 T,梯度规模缩小。这是 Hinton 原论文里反复强调的实现细节。

```python
import torch.nn.functional as F

T = 4.0
soft_t = F.log_softmax(teacher_logits / T, dim=-1)
soft_s = F.log_softmax(student_logits / T, dim=-1)
kl = F.kl_div(soft_s, soft_t, reduction='batchmean', log_target=True) * (T*T)
loss = alpha * kl + (1-alpha) * F.cross_entropy(student_logits, y)
```

## 十、VAE 里的 reverse KL

VAE 损失 = 重建 + KL:

```
L_VAE = - E_q[ log p(x|z) ]  +  KL( q(z|x) ‖ p(z) )
        └── 重建误差         └── 让后验逼近先验(标准正态)
```

第二项是 reverse KL。**结果就是 q(z|x) 倾向于"塌"到先验内部一个区域,而不是覆盖先验的所有形状**——这正是 VAE 经常生成"模糊均匀风格"图像的根因之一(以及为什么后来的 diffusion 完全换了路线)。

## 十一、工程对应清单

```python
import torch
import torch.nn.functional as F

# ── 分类:CE ──
loss = F.cross_entropy(logits, target)            # 内置 log-softmax + NLL
loss = F.nll_loss(F.log_softmax(logits, -1), target)   # 等价拆开

# ── KL 散度 ──
# 注意 PyTorch 的 kl_div(input, target) 算的是 KL(target ‖ input)
# 而且 input 必须是 log 概率,target 是概率(或加 log_target=True)
kl = F.kl_div(F.log_softmax(s_logits, -1),
              F.softmax(t_logits, -1),
              reduction='batchmean')

# ── log-sum-exp ──
lse = torch.logsumexp(logits, dim=-1)             # 数值稳定

# ── label smoothing(把 one-hot 变成软标签,等价于改 p) ──
loss = F.cross_entropy(logits, target, label_smoothing=0.1)

# ── 二分类 ──
loss = F.binary_cross_entropy_with_logits(logits, target)
```

| 库 / 框架 | API | 注意 |
| --- | --- | --- |
| PyTorch | `F.cross_entropy` | input 是 logits 不是概率 |
| PyTorch | `F.kl_div` | 参数顺序反直觉,input 必须是 log_prob |
| TensorFlow | `tf.keras.losses.CategoricalCrossentropy(from_logits=True)` | 别忘 `from_logits` |
| HuggingFace TRL | `PPOTrainer` 的 `kl_penalty='kl'`、`init_kl_coef`、`adap_kl_ctrl` | 自适应 KL 控制器 |
| HuggingFace TRL | `DPOTrainer` 的 `beta` | DPO 的 β |
| OpenAI baselines | `klcoef` | RLHF 古早实现里 KL 系数也叫这个 |

## 十二、局限与什么时候不要用

- **KL 在 q=0 处爆炸**:做 RL 时如果策略某些动作概率精确为 0,KL 算不动。**实践中用一个最小概率裁剪(`logits.clamp_min`)或者改用 JSD / TV**
- **KL 不是真距离**:不能放进任何要"对称 / 三角不等式"的算法(比如 K-means)。要对称的话用 **JSD = 0.5·KL(p‖m) + 0.5·KL(q‖m)**,m=(p+q)/2
- **离散 KL 用错对齐会无穷**:p 在某个 token 有概率,q 没有,直接 ∞。蒸馏时学生词表必须包含老师词表
- **小样本估计 KL 噪声大**:RLHF 一个 batch 估的 KL 是带方差的,trl 的自适应控制器就是为了平滑它
