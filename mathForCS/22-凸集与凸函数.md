# 凸集与凸函数

这一篇要回答一个工程师在调参时反复撞上的问题:**为什么 SVM / Lasso 训练完了拿到的就是最优解,而神经网络跑十次能拿到十个不同结果?** 答案就一个词:**凸性**。凸优化是一个"理论上保证你拿到全局最优解"的小封闭世界,神经网络一脚踩出去就再也回不来。

> 一句话先记住:**凸函数任何局部最小 = 全局最小,梯度下降跑一遍就到家;非凸函数没有这个保证,深度学习实际跑通靠的是"高维下鞍点远多于局部最小 + SGD 噪声 + over-parameterization 让 loss landscape 变平"这套土工程**。这一篇分清两边,**该用 CVXPY 的时候别拿 SGD 硬撑,该上 SGD 的时候别幻想"找到全局最优"**。

---

## 一、为什么必须懂凸性

不懂凸性的工程师会犯两类错:

| 错法 | 真实场景 | 代价 |
| --- | --- | --- |
| 凸问题用 SGD 硬调 | Lasso / Logistic 当成神经网络训,跑一万步还在抖 | 慢、解不精确、超参玄学 |
| 非凸问题幻想全局最优 | 网络跑了十遍 loss 不一样,以为代码 bug | 一周白干、改无关超参 |
| 用错求解器 | 投资组合优化用 SGD 不加约束 | 权重和不为 1、有负仓位、监管不过 |
| 忽略问题结构 | LP / QP 直接喂 PyTorch | 比 CVXPY 慢 100 倍且不收敛 |

**这一篇的目标**:让你看到一个优化问题,能在 30 秒内判断"这是凸的还是非凸的、该上 CVXPY 还是 SGD"。

## 二、凸集:任意两点连线在集合内

### 2.1 定义(用图记)

集合 C 是凸的,当且仅当:**任取两点 x, y ∈ C,它们的连线段 [x, y] 整段都在 C 里**。

```
凸:                            非凸:
   _____                          ___       ___
  /     \                        /   \_____/   \
 |       |  ←─ 任意两点连线     |   月牙形    |  ←─ 两点连线穿出去了
  \_____/      都在内部           \___________/
                                    ↑
                                    这一段不在集合里
```

**最直观的例子**:

| 集合 | 凸? | 备注 |
| --- | --- | --- |
| 实数轴上的区间 [a, b] | 是 | 一维最简单的凸集 |
| 圆 / 椭圆(填实) | 是 | 二维 |
| 任意凸多边形(三角形、矩形、五角星填实?不,五角星不是) | 是/否 | 五角星有凹角,非凸 |
| 月牙、甜甜圈 | 否 | 中间挖空了 |
| 任意半空间 `{x : a^T x ≤ b}` | 是 | LP 的可行域基本由这些拼出来 |
| 半正定矩阵集合 | 是 | SDP 的可行域 |

### 2.2 工程里见到的凸集

90% 的凸优化可行域都是这几类的交集:

```
1. 半空间:    a^T x ≤ b           ←─ 一个线性不等式约束
2. 超平面:    a^T x = b           ←─ 一个线性等式约束
3. 范数球:    ||x||_p ≤ r         ←─ Lasso 的 L1 约束、岭回归的 L2 约束
4. 单纯形:    Σx_i = 1, x_i ≥ 0   ←─ 投资组合权重 / 概率分布
5. 半正定锥: X ⪰ 0                ←─ SDP / 协方差矩阵约束
```

**关键性质**:**凸集的交集还是凸集**。所以你把 LP 约束、Lasso 约束、单纯形约束堆一起,可行域还是凸的——CVXPY 拼约束就是利用这个。

> **核心直觉**:**凸集 = 没有"坑"也没有"洞"的形状**。任何方向走直线,要么一直在集合里,要么一出去就再也不回来。这个性质让"沿梯度走"变得安全——不会一脚踏空。

## 三、凸函数:图像上方区域是凸集

### 3.1 三个等价定义(挑一个记)

定义 1(几何):**任意两点的割线在函数图像上方**。

```
        f(x)
         │       ┌─── 割线
         │     ╱
         │   ╱
         │ ╱      ←─ 函数图像在割线下方
         │╱
         └─────── x
        x1      x2

f(λx1 + (1-λ)x2) ≤ λf(x1) + (1-λ)f(x2),   λ ∈ [0, 1]
```

定义 2(集合):**函数图像上方的区域(epigraph)是一个凸集**。

```
  f(x)
   │  ░░░░░░░░░░░░░░░░░░    ←─ epigraph(图像上方区域),是凸集
   │  ░░░░░░░░░░░░░░░░░░
   │  ░░░╲░░░░░░░░╱░░░░░
   │  ░░░░╲░░░░░░╱░░░░░░
   │  ░░░░░╲____╱░░░░░░░    ←─ f 的图像
   └──────────────── x
```

定义 3(导数):
- 一阶条件:**任意切线都在函数下方**——`f(y) ≥ f(x) + ∇f(x)^T (y - x)`
- 二阶条件:**Hessian 半正定**——`∇²f(x) ⪰ 0`(单变量就是 `f''(x) ≥ 0`)

> **经验法则**:**判断一个函数是不是凸,优先看二阶条件**。一元情况下就是 `f''(x) ≥ 0` 处处成立——`x²` 的二阶导是 `2 > 0`,所以 `x²` 是凸的;`-x²` 的二阶导是 `-2 < 0`,所以是凹的;`x³` 在 x<0 时凹、x>0 时凸,**整体非凸**。

### 3.2 哪些函数是凸的(背下来)

| 函数 | 凸? | 在哪儿用 |
| --- | --- | --- |
| 线性函数 `a^T x + b` | 凸 + 凹(同时) | LP 目标 |
| 二次型 `x^T P x`(P 半正定) | 凸 | 岭回归、QP |
| `x²`、`|x|`、范数 `||x||_p`(p ≥ 1) | 凸 | L1/L2 正则 |
| `e^x`、`-log(x)`(x>0) | 凸 | MLE 推导出的损失函数 |
| `log(1 + e^x)` (softplus) | 凸 | logistic 回归损失 |
| `log(Σ e^{x_i})` (log-sum-exp) | 凸 | softmax 的 log 版本 |
| 负 log-likelihood(对线性 logits) | 凸 | logistic / Poisson 回归 |
| `max(f1, f2, ..., fn)`(每个 fi 凸) | 凸 | hinge loss 的来源 |
| `sin(x)`、`x³`、神经网络 | 非凸 | 深度学习全场 |

**一些保凸操作**(组合凸函数得到新的凸函数):

```
1. 非负加权和:  α₁f₁ + α₂f₂ + ... + αₙfₙ     (αᵢ ≥ 0)
2. 仿射变换:    f(Ax + b)                     (f 凸 → 这玩意凸)
3. 取最大:      max{f₁, f₂, ..., fₙ}          (各 fᵢ 凸)
4. 与凸递增的复合: g(f(x))                    (f 凸,g 凸且不减)
```

> **避坑**:`sigmoid(x) = 1/(1+e^{-x})` **不是凸的**(在 x>0 时凹,x<0 时凸,整体 S 形)。所以"sigmoid 输出 + MSE 损失"这种古早写法**不凸**——这就是为什么二分类要用 logistic loss(它是 log + sigmoid 的复合,**结果是凸的**)而不是 MSE。

### 3.3 凸优化为什么好解

凸函数有一条所有工程师都该刻进 DNA 的性质:

> **核心直觉**:**凸函数的任何局部最小 = 全局最小**。

```
凸函数:                  非凸函数:
                              ╱╲      ╱╲    ←─ 多个局部最小
   ╲    ╱                    ╱  ╲    ╱  ╲
    ╲  ╱                    ╱    ╲__╱    ╲
     ╲╱   ←─ 唯一最小        ╲___╱        ╲___
                                                 ↑
   局部最小 = 全局最小       局部最小 ≠ 全局最小
```

这意味着:

- **梯度下降只要能走到 ∇f = 0 的点,就一定是全局最优**——不用担心被卡在局部坑里
- **拿到一个解,就敢说"这是最好的"**——这在金融、医疗、合规场景下是必须的
- **算法收敛性可证**——内点法、ADMM、加速梯度法都有明确的收敛速率

### 3.4 严格凸 vs 强凸

工程上经常区分这三个级别:

| 级别 | 定义 | 收敛速率 | 例子 |
| --- | --- | --- | --- |
| 凸 | `f''(x) ≥ 0` | 次线性 `O(1/k)` | `|x|`、线性函数 |
| 严格凸 | `f''(x) > 0`(处处) | 仍然 `O(1/k)` | `x²`、`e^x` |
| 强凸 | `f''(x) ≥ m > 0`(有下界) | 线性 `O((1-mc)^k)` | 岭回归 `||y-Xβ||² + λ||β||²` |

> **经验法则**:**强凸 = 函数底部"足够弯",梯度下降每一步都能砍掉固定比例的距离**。所以加 L2 正则的最大好处不是"防过拟合",而是**让原本只是凸的问题(如最小二乘)变成强凸,迭代次数从几千降到几十**。这是 Ridge 比 Lasso 更"好训"的根本原因。

## 四、神经网络非凸的代价

### 4.1 多个局部最小不是最大问题

非凸函数最直观的代价是"多个局部最小",但**深度学习真正难的不是这个**。

```
低维想象(错误):                   高维实际(正确):
                                                    ╱╲
   ╱╲      ╱╲                        ─ ─ 大部分    ╱  ╲
  ╱  ╲    ╱  ╲   ←─ 局部最小         ─ 是鞍点 ─   ╱    ╲
 ╱    ╲__╱    ╲                       ─ ─ ─ ─    ╱      ╲
                                                ╱        ╲

低维:局部最小是大敌                高维:鞍点才是大敌
```

**为什么高维下鞍点远多于局部最小?** 一个直觉的论证:

```
n 维空间里 ∇f = 0 的临界点,要是局部最小,
需要 Hessian 的所有 n 个特征值都 ≥ 0。

如果每个特征值正负的概率各 50%,
随机点是局部最小的概率 ≈ (1/2)^n

n = 1000 时,(1/2)^1000 ≈ 0
也就是说:几乎所有 ∇f = 0 的点都是鞍点
```

> **核心直觉**:**深度学习不是在"避开局部最小坑",而是在"快速逃出鞍点"**。鞍点的危险在于梯度趋于零,SGD 走得越来越慢——但只要有噪声,迟早能逃出去(某个方向是下坡的)。

### 4.2 鞍点为什么比局部最小更难逃

```
局部最小:                          鞍点:
   ╲    ╱        ←─ 四面八方       ──────╲          ╱──── ←─ 一个方向是
    ╲  ╱            都是上坡              ╲────────╱         下坡,但
     ╲╱             SGD 真出不去          ╱        ╲          梯度很小
                                         ╱          ╲
   出不去:除非有                       难发现:梯度接近 0,
   极大的随机扰动                       SGD 半天找不到下坡方向
```

**实际经验**:在大模型训练里,鞍点表现为"loss 长时间几乎不动",过一阵突然又开始下降——就是 SGD 找到了下坡方向,逃出去了。

### 4.3 深度学习实际跑通靠什么

非凸保证不了全局最优,但深度学习每天还在跑——靠的不是数学,是工程经验:

```
1. 随机初始化              ← 不同初始化 → 不同局部最小,但都"够用"
2. SGD 噪声                ← mini-batch 的随机性让你能逃出鞍点
3. Over-parameterization   ← 参数量远超数据,loss landscape 变平,
                              "局部最小"密集且全部接近全局最小
4. 隐式正则                ← SGD 偏好"宽"的最小,这种最小泛化更好
5. Skip connection / BN    ← 把 loss landscape 直接抹平
6. Warmup + 学习率调度     ← 早期不要走太快,后期稳定收敛
```

> **避坑**:**别拿"凸优化教科书的收敛性"去推神经网络的训练**。凸优化告诉你"梯度下降一定收敛到全局最优",这在神经网络里**完全不成立**——能拿到一个还可以的局部最小就谢天谢地了。但实际跑出来的模型质量,比凸优化时代的 SVM / 逻辑回归好得多——这就是 over-parameterization 的魔力。

### 4.4 凸函数 vs 非凸函数的可视化

```
凸 loss(SVM / Lasso):       非凸 loss(神经网络):

      ─╲              ╱─      ▁▂▆▇█▇▆▂▁▁▂▃▆▇█▇▆▃▂▁
       ─╲            ╱─       ▆▇█▇▆▃▂▂▃▆▇█▇▆▆▆▇▆▃
        ─╲          ╱─        █▇▆▆▃▂▁▁▂▃▆▇▆▆▆▆▆▆▃
         ─╲        ╱─         ▆▆▆▆▃▃▂▁▂▃▆█▇▆▆▆▆▆▆
          ─╲______╱─          ▆▇▆▆▆▆▆▆▆▆▆▆▆▆▆▆▆▆▆
              ↑                       ↑
        "碗状"全局最优                 "高原 + 鞍点 + 局部最小"
```

## 五、代码:看一个凸问题怎么解

### 5.1 凸问题的标准形式

```
min   f(x)         ←─ 目标函数(凸)
s.t.  g_i(x) ≤ 0   ←─ 不等式约束(凸函数 ≤ 0)
      h_j(x) = 0   ←─ 等式约束(线性)
```

只要 f 凸、g_i 凸、h_j 线性,这就是个凸优化问题——理论上有最优解,实践中 CVXPY 一行搞定。

### 5.2 CVXPY 解 Lasso(凸,有解析意义)

```python
import cvxpy as cp
import numpy as np

# 数据:y = Xβ + 噪声
np.random.seed(0)
n, p = 100, 50
X = np.random.randn(n, p)
beta_true = np.zeros(p)
beta_true[:5] = [1.5, -2.0, 3.0, -1.0, 0.5]   # 只有前 5 个非零
y = X @ beta_true + 0.1 * np.random.randn(n)

# Lasso:min ||y - Xβ||² + λ||β||₁
beta = cp.Variable(p)
lam = 0.1
loss = cp.sum_squares(y - X @ beta) + lam * cp.norm(beta, 1)
prob = cp.Problem(cp.Minimize(loss))
prob.value = prob.solve()        # 一行求解

print("最优 loss:", prob.value)
print("β 中非零的位置:", np.where(np.abs(beta.value) > 1e-3)[0])
# 输出:[0, 1, 2, 3, 4]  ←─ Lasso 自动稀疏选择
```

**这一段值得品味的几点**:

- `cp.sum_squares` 和 `cp.norm(., 1)` 都是凸的,加权和(权重非负)还是凸的——CVXPY 自动验证这点
- 不需要写梯度、不需要调学习率、不需要选优化器——**凸 + 写得出标准形式 → CVXPY 自己挑求解器**
- 解是**精确的**(到求解器精度,通常 1e-8),不是"训了一万步差不多"

### 5.3 同样问题用 SGD 跑一遍(对比)

```python
import torch

X_t = torch.tensor(X, dtype=torch.float32)
y_t = torch.tensor(y, dtype=torch.float32)
beta_sgd = torch.zeros(p, requires_grad=True)
opt = torch.optim.SGD([beta_sgd], lr=0.001)

for step in range(5000):
    opt.zero_grad()
    pred = X_t @ beta_sgd
    loss = ((y_t - pred) ** 2).sum() + 0.1 * beta_sgd.abs().sum()
    loss.backward()
    opt.step()

print("SGD loss:", loss.item())
# 跟 CVXPY 比:慢、不精确、超参靠猜、还得调 lr
```

> **经验法则**:**凸 + 中小规模(变量 < 10⁵) + 需要精确解 → CVXPY / cvxopt / Mosek**;**非凸 / 大规模 / 不要求精确 → SGD 系**。Lasso 用 CVXPY 三行,用 SGD 调一天——凸问题你不用专门求解器就是浪费。

### 5.4 验证 Hessian 半正定(用 SymPy 验凸)

```python
import sympy as sp

x, y = sp.symbols('x y', real=True)
f = x**2 + 2*y**2 + x*y          # 待验证函数
H = sp.hessian(f, (x, y))
print("Hessian:")
print(H)
# Matrix([[2, 1], [1, 4]])

print("特征值:", H.eigenvals())
# {3 - sqrt(2): 1, 3 + sqrt(2): 1}
# 都正 → 半正定 → f 凸

# 反例
g = x**2 - y**2
H2 = sp.hessian(g, (x, y))
print("g 的 Hessian:", H2.eigenvals())
# {2: 1, -2: 1}  ←─ 一正一负 → 非半正定 → g 非凸(实际上是鞍形)
```

## 六、凸优化在工程里的成功案例

凸优化不是一个"学院派"的话题——它在工业界有很多每天都在跑的成功案例:

| 系统 / 算法 | 凸吗 | 求解器 | 备注 |
| --- | --- | --- | --- |
| SVM(线性 / 核) | 凸 QP | libsvm / liblinear | 经典分类器,2000s 主力 |
| Lasso / 弹性网 | 凸 | scikit-learn / glmnet | 特征选择 + 稀疏回归 |
| Logistic 回归 | 凸 | scikit-learn | 仍是工业 CTR 模型基石 |
| Poisson / Gamma 回归 | 凸 | statsmodels | 计数 / 时长建模 |
| 线性规划(LP) | 凸 | Gurobi / CPLEX / GLPK | 排班、运输、网络流 |
| 整数规划(MIP) | 非凸(整数约束) | Gurobi(分支定界 + LP 松弛) | 排班、生产计划 |
| 投资组合优化 | 凸 QP | CVXPY / CVXOPT | Markowitz mean-variance |
| 鲁棒控制 / MPC | 凸(部分) | OSQP / CVXPY | 自动驾驶、机器人 |
| Compressed sensing | 凸 | 各种 ℓ1 求解器 | MRI、信号恢复 |
| Power system OPF | 凸松弛 | Mosek / CVXPY | 电网潮流优化 |

**这些问题之所以能"几十年不动还在跑"**,就是因为凸性给了你"求解器一跑就是全局最优"的保证。换成神经网络,你得每天处理"为什么这次结果跟上次不一样"。

### 6.1 投资组合优化(经典凸 QP)

```python
import cvxpy as cp
import numpy as np

n = 10                                # 10 只股票
mu = np.random.randn(n) * 0.1         # 期望收益
Sigma = np.random.randn(n, n)
Sigma = Sigma @ Sigma.T               # 协方差矩阵(半正定)

w = cp.Variable(n)
gamma = 1.0                           # 风险厌恶系数

# Markowitz: max μ^T w - γ w^T Σ w
# 约束:权重和=1,不允许做空
prob = cp.Problem(
    cp.Maximize(mu @ w - gamma * cp.quad_form(w, Sigma)),
    [cp.sum(w) == 1, w >= 0]
)
prob.solve()
print("最优权重:", w.value.round(3))
print("最优组合收益:", (mu @ w.value).round(4))
```

**这是凸优化在金融里最常见的应用**——目标凸(协方差是半正定)、约束凸(线性 = 和不等式)、CVXPY 几行解决。所有量化基金的组合构造都建在这套上面。

## 七、工程对应与局限

### 7.1 求解器选型清单

| 问题类型 | 推荐工具 | 备注 |
| --- | --- | --- |
| 一般凸优化(原型 / 研究) | **CVXPY**(Python)/ JuMP(Julia) | 写约束最舒服,自动调求解器 |
| LP(线性规划) | Gurobi / CPLEX / GLPK / HiGHS | Gurobi 工业首选(收费),HiGHS 开源不错 |
| QP(二次规划) | **OSQP** / Gurobi / Mosek | OSQP 开源、超快 |
| SOCP / SDP | **Mosek** / SCS / ECOS | Mosek 工业,SCS / ECOS 开源 |
| MILP(混合整数 LP) | Gurobi / CPLEX | 注意:整数约束**不凸** |
| 大规模(变量 > 10⁶) | 一阶法 / SGD / ADMM | CVXPY 撑不住,自己写 |
| 神经网络(非凸) | **PyTorch / JAX** | SGD / Adam,放弃全局最优 |

### 7.2 何时用凸求解器,何时用 SGD

```
                        变量数
              < 10^4       10^4 - 10^6      > 10^6
            ┌─────────────────────────────────────────┐
        凸  │  CVXPY        OSQP / Mosek    自定义一阶法  │
            │  (闭眼用)      (商业级)        (LBFGS / ADMM)│
            ├─────────────────────────────────────────┤
       非凸 │  小心局部最小   SGD / Adam      SGD + 调度    │
            │  (multi-start) (常规)          (大模型套路)  │
            └─────────────────────────────────────────┘
```

> **避坑**:**不要拿 PyTorch 训练 LP / Lasso / SVM**——它们是凸的,有比 SGD 快 100 倍且精确的求解器,SGD 只是因为你不知道。同时**不要拿 CVXPY 训练神经网络**——CVXPY 是声明式的,不能反传,不能 mini-batch,跑 1000 个变量就慢得让你想砸键盘。

### 7.3 凸问题转化技巧(看一眼有印象)

很多看起来非凸的问题,稍微改写一下就是凸的——这是凸优化老手的看家本领:

| 原形式 | 等价凸形式 | 例子 |
| --- | --- | --- |
| `min ||x||₀`(L0,稀疏) | `min ||x||₁`(L1) | Lasso 替代真稀疏 |
| `min max_i f_i(x)` | 引入 `t`,`min t s.t. f_i(x) ≤ t` | minimax 问题 |
| `f(x)/g(x)` 分式 | 变量替换 + 齐次化 | 比率优化 |
| `||Ax-b||/||x||` | SDP 松弛 | 最坏情况扰动 |

**这套技巧在 Boyd《Convex Optimization》里讲了 700 页**,工程师不用全会,但要知道"很多问题写得对就是凸的,写得不对就是非凸的"。

### 7.4 局限

凸优化也不是万能的:

1. **大规模**:CVXPY 一旦变量数 > 10⁵ 就开始吃力,大模型场景下连凸问题也得一阶法
2. **离散变量**:整数约束直接破坏凸性,LP 松弛 + 分支定界是工程办法,但 NP-hard 是命
3. **黑盒目标**:目标函数没有解析形式(模拟器、神经网络评估),凸求解器没法用
4. **强凸性条件不易满足**:很多正则化系数不够大,导致条件数差、收敛慢

> **核心直觉**:**凸优化是"理论保证 + 工业级求解器"的小封闭世界**——你能进就尽量进。但 2010s 以后的 ML 有一半问题(深度网络、组合优化、强化学习)进不来,这才是非凸方法(SGD / Adam)的舞台。**两边的工具不要混用,选错就是数量级的浪费**。
