# 期望、方差与中心极限

「均值 ± σ 落在 95% 区间内」——这是工程师对统计的全部印象,**也是误用最广的一句话**。监控、A/B 测试、压测报表、机器学习的 loss,十个人里九个把这句话当默认假设,但**真实世界里 80% 的指标分布不是高斯**:重尾延迟、点击率、营收、错误率,直接套这套公式做决策就是在埋雷。这一篇把期望、方差、CLT 拆开,告诉你**什么时候能套、什么时候必须换更稳的工具**(Bootstrap、分位数、稳健统计)。

> 一句话先记住:**期望是"长期均值"、方差是"平方惩罚"、CLT 是"样本均值近似高斯"**。三件事在工程里串成一句话:**A/B 比的不是单条数据,而是均值;均值的均值越多越像高斯;所以才能用 z/t 检验**。**前提是均值有意义(分布有限方差)**——这一句忘了就会被长尾分布反复打。

---

## 一、为什么必须懂这几个

不少人学过期望/方差,只记得"E[X] = Σ x·p(x)"和"Var(X) = E[(X-μ)²]"两条公式,出了课本就**完全不知道这玩意儿在工程里干啥**。直到撞上下面这些场景才发现公式不够用:

| 场景 | 不懂统计的结论 | 懂统计的结论 |
| --- | --- | --- |
| 监控 P50 = 80ms,均值 = 250ms | "怎么差这么多,系统有问题?" | 长尾分布,均值被几条慢请求拉高 |
| A/B 测试 B 比 A 高 1.2% | "上线!有提升!" | 样本量不够,这 1.2% 在噪声里 |
| 压测报告写 "200ms ± 50ms" | "稳" | 假设了高斯,P99 可能是 800ms |
| 推荐系统点击率 0 vs 0.001 | "差 1000 倍" | 都在置信区间里,无差异 |
| 蒙特卡洛模拟跑 10000 次还在抖 | "代码有 bug" | 重尾分布,样本均值收敛极慢 |
| 营收 = 用户数 × 客单价 | "两边各加 10% 就 +21%" | 协方差非零时这条公式不成立 |

**所有 SGD 训练、所有 A/B 实验、所有压测报告、所有蒙特卡洛模拟,本质上都在做一件事**:用样本估计总体。这一篇就是讲**这个估计什么时候靠谱、什么时候不靠谱**。

> **核心直觉**:工程里碰到的"统计错误",90% 不是数学算错了,是**把不该用高斯近似的场景用了高斯近似**——长尾、相关样本、小样本、有截断。算式没错,前提错了。

## 二、期望:长期均值,但长尾下要小心

### 2.1 期望的定义与工程意义

```
离散:E[X] = Σ x · P(X=x)
连续:E[X] = ∫ x · f(x) dx
```

不要被积分号吓住。**直观就一句话**:把每个可能的值,按概率加权平均。**长期跑 N 次实验,均值会收敛到 E[X]**——这是大数定律的承诺。

| 工程场景 | E[X] 是什么 |
| --- | --- |
| 抛硬币赔率 | 长期下注的期望收益 |
| 服务器请求延迟 | 跑 1 亿次请求的平均延迟 |
| ML 训练 loss | 整个数据集 loss 的平均(SGD 用一个 batch 估计它) |
| 推荐 CTR | 千次曝光的平均点击率 |
| A/B 实验指标 | 总体均值,样本均值是它的估计 |

### 2.2 期望的线性性:工程里最有用的一条性质

```
E[aX + bY] = a·E[X] + b·E[Y]
```

**不需要 X 和 Y 独立**。这是**统计里被低估的最有用一条性质**:不管 X、Y 怎么相关,期望都能拆开算。

工程例子:

```python
# 一次请求总耗时 = 网络 + DB + 业务逻辑
# 这三段强相关(慢机器啥都慢)
# 但 E[总耗时] = E[网络] + E[DB] + E[业务] 永远成立
```

### 2.3 长尾分布:期望可能不是好的代表

下面这张图把工程里常见的两种分布画出来。**报"均值"在第一种里靠谱,在第二种里就是骗自己**。

```
高斯分布(身高、考试成绩、噪声):
                    ▁▂▄▆██▇▅▃▂▁
                  ─────────────────
                       ↑
                    均值在这

长尾/重尾分布(请求延迟、营收、网络流量):
        ▆▅▃▂▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁ ······
        ─────────────────────────────────────────────
         ↑          ↑
        P50       均值(被尾巴拉到这)            P99 在更右边
```

> **避坑**:监控、压测、营收这些指标**都不是高斯的**。报"均值 200ms"会让你以为大多数请求 200ms,但实际可能 80% 在 100ms 以下,20% 拖到 800ms。**分位数(P50/P95/P99)永远比均值更能描述用户体感**。

更狠的:有些重尾分布**期望不存在**——理论上的均值是无穷大,你跑再多样本,均值也不收敛。Cauchy 分布、α<1 的 Pareto 都是这样。**真实世界里营收、文件大小、流量峰值经常接近 Pareto α≈1.5**——这时均值理论上存在,但**收敛极慢**:你跑 10 万次实验,均值还在跳。

```python
# 重尾分布下样本均值的"收敛"
import numpy as np
rng = np.random.default_rng(42)
samples = rng.pareto(1.2, 100_000) + 1   # α=1.2,接近不收敛
running_mean = np.cumsum(samples) / np.arange(1, len(samples)+1)
# 画 running_mean 你会看到它一直在跳,从来不稳下来
```

**记住**:看到一个分布就报均值,先问一句"这分布有方差吗?有期望吗?"——大多数业务指标都得用分位数。

## 三、方差:为什么要平方

### 3.1 方差的定义

```
Var(X) = E[(X - μ)²] = E[X²] - E[X]²

标准差 σ = √Var(X)   # 回到原单位
```

直观就是**"距离均值的平均距离的平方"**。

### 3.2 为什么是平方,不是绝对值

| 备选方案 | 问题 |
| --- | --- |
| `E[X - μ]` | 正负抵消,永远等于 0 |
| `E[\|X - μ\|]` | 绝对值不可导,数学上不好处理 |
| `E[(X - μ)²]` | **正负都被惩罚 + 处处可导 + 性质极好**(L2、可分解) |
| `E[(X - μ)⁴]` | 也行,但对极端值过度敏感(峰度才用) |

**平方有三个工程上的甜点**:

1. **可导**——梯度下降的命根。L2 损失就是方差,处处可导。
2. **可分解**——独立变量方差可加(`Var(X+Y) = Var(X) + Var(Y)`,前提是独立)。L1 / 绝对值损失没这性质。
3. **正负偏离都计入**——绝对值也行,但平方对大偏离的惩罚更狠(50 的偏差 = 2500 的方差,5 的偏差 = 25 的方差,差 100 倍而非 10 倍)。

> **核心直觉**:**方差和 L2 损失是同一回事**。机器学习里你最熟的 MSE = (1/N) Σ (y - ŷ)²,本质就是预测误差的方差(假设误差均值 = 0)。**回归任务用 MSE,数学上等价于"假设噪声是高斯"**——下一篇讲 MLE 时会证明这一点。

### 3.3 标准差为什么更常报

`Var(X)` 单位是**原单位的平方**——延迟方差是 ms²,这玩意儿没有物理意义。`σ = √Var` 回到 ms,可以直接读"波动大约 ±50ms"。

**报数据的惯例**:`mean ± σ`(不是 `mean ± Var`)。但前面说了,**这种写法只在分布近似高斯时才有意义**——长尾分布下 σ 也被尾巴拉得很大,失去解释力。

### 3.4 方差的可加性(独立才行)

```
独立时:    Var(X + Y) = Var(X) + Var(Y)
不独立时:  Var(X + Y) = Var(X) + Var(Y) + 2·Cov(X, Y)
```

工程例子:

```python
# 一个 batch 内 N 个样本独立采样
# Var(sum)  = N · Var(单样本)
# Var(mean) = Var(sum)/N² = Var(单样本) / N
# 所以 σ(mean) = σ(单样本) / √N
# 这就是著名的 "1/√N" 收敛律
```

**这条 1/√N 律是 SGD、A/B 测试、蒙特卡洛、Bootstrap 的共同底层**:你想把误差减半,样本量得乘 4;想减到十分之一,样本量得乘 100。**收敛慢,但保底能收敛**。

## 四、协方差与相关系数:线性关系强度,不是因果

### 4.1 定义

```
协方差:Cov(X, Y) = E[(X - μ_X)(Y - μ_Y)]
                 > 0:同涨同跌
                 = 0:无线性关系
                 < 0:此涨彼跌

皮尔逊相关:ρ = Cov(X, Y) / (σ_X · σ_Y)   # 归一化到 [-1, 1]
```

**协方差量纲是 X·Y 的乘积**(没法对比),所以归一化得到 ρ ∈ [-1, 1]——这才是工程里通用的相关系数。

### 4.2 三个关键反直觉

**反直觉 1:相关 ≠ 因果**

经典段子:夏天溺水人数和冰淇淋销量正相关 ρ=0.9。**冰淇淋不会让你溺水**,两者共同被"夏天热"驱动。**ρ 高只说明一件事——两个变量一起动**,谁是因果、有没有第三方变量,统计本身回答不了。

**反直觉 2:独立 → 相关为 0,反过来不一定**

```
X ~ Uniform(-1, 1)
Y = X²
```

X 和 Y 显然有关系(Y 完全由 X 决定),但 `Cov(X, Y) = 0`——因为是**非线性**关系,皮尔逊相关检测不到。**ρ=0 只能说"没有线性关系",不能说"独立"**。要查非线性相关得用互信息(本系列 18 讲)、距离相关、Spearman 秩相关。

**反直觉 3:Anscombe's quartet——四组完全不同的数据,均值/方差/相关都一样**

```
组 1:漂亮的线性关系
   ●  ●●●● ●●● ●

组 2:明显的二次曲线
       ●●●●●●●
     ●           ●
   ●               ●

组 3:线性 + 一个离群点
   ●●●●●●●●           ●

组 4:几乎所有点 x 相同 + 一个离群点
                       ●
   ●(8 个点垂直堆在一处) ●
```

四组数据 mean、var、ρ 完全一样。**只看汇总统计你完全分不清**——所以散点图比 ρ 更重要。

> **避坑**:看到 ρ=0.95 不要立刻下结论"两变量强相关"。先画散点图——可能是几个离群点撑起的高 ρ,可能是非线性,可能是 Simpson 悖论(分组后符号反转)。**ρ 是个粗糙工具,不能替代可视化**。

## 五、大数定律(LLN):频率派的基石

```
样本均值 X̄_N = (X_1 + X_2 + ... + X_N) / N

当 N → ∞:X̄_N → E[X]   (依概率)
```

**人话**:扔越多次硬币,正面比例越接近 0.5。这就是"频率 ≈ 概率"的依据。

### 5.1 工程意义:多大样本算"够"?

教科书会告诉你"N 越大越好",工程上要回答的是**"多大够用"**。前面推过 σ(均值) = σ(单样本) / √N,反过来:

```
想让均值的标准误差 ≤ ε,需要样本量:
N ≥ (σ / ε)²

例子:
  σ = 100ms,想估计均值精度到 ±5ms
  → N ≥ (100/5)² = 400 个样本
```

> **经验法则**:**样本量翻倍只能让误差缩小 √2 倍(约 30%)**,翻 4 倍误差减半。所以 100 → 400 提升明显,400 → 1600 提升边际,1600 → 6400 几乎看不出来。**A/B 实验决策应该在 1600 这个量级附近做,不是无脑等到 100 万**。

### 5.2 LLN 的三个隐藏假设(违反就翻车)

1. **样本独立同分布 (i.i.d.)**——时间序列(用户连续行为)、有缓存的请求、爬虫被限速,**都不 i.i.d.**,LLN 收敛慢甚至不收敛。
2. **方差有限**——前面说的 Pareto α≤2 的重尾分布,方差无穷大,LLN 失效。
3. **分布稳定**——A/B 跑到第 5 天用户行为变了,前 4 天数据不能简单平均。

## 六、中心极限定理(CLT):工程里最值钱的统计定理

### 6.1 一句话陈述

```
不管 X 原来是什么分布(只要均值 μ、方差 σ² 有限),
当 N 足够大时:

         X̄_N - μ
   Z = ──────── ~ N(0, 1)
        σ / √N

说人话:大量独立样本的均值,接近高斯分布。
```

注意:**不是单个样本接近高斯**——是**样本的均值**接近高斯。

### 6.2 直觉:为什么"均值都是高斯"

下图是从 Uniform(0, 1)(完全平的均匀分布)采样,看不同 N 下样本均值的分布:

```
N=1(就是原分布)        N=2                  N=10                 N=30
─────────────────       ─────────────────    ─────────────────    ─────────────────
████████████████        ▁▃▅▇█▇▅▃▁           ▁▂▄▆█▇▄▂▁           ▁▁▂▄▆█▆▄▂▁▁
均匀,完全不像高斯       已经像三角形         已经接近高斯          完美高斯
```

**N≈30 之后,基本上长得像高斯**(对很多常见分布而言)。**这就是工程里 t 检验、z 检验、置信区间、A/B 测试敢用高斯近似的依据**——你比的是均值,不是单条数据,均值就是近似高斯的。

### 6.3 CLT 是哪些工程的根基

| 工程系统 | 用 CLT 哪里 |
| --- | --- |
| A/B 测试(t 检验) | 假设两组均值差 ≈ 高斯,算 z 分数 |
| 监控告警(均值 ± 3σ) | 把高斯当假设(**很多场景错的**) |
| 蒙特卡洛模拟 | 用样本均值估计期望,误差 ≈ 1/√N · 高斯 |
| Bootstrap 置信区间 | 重采样均值近似高斯 |
| ML 模型方差估计 | 训练多次取均值,均值的波动是高斯 |

### 6.4 CLT 的边界:啥时候不能用

- **样本量不够**(N < 30,看分布的偏度,严重偏的可能要 N > 100)
- **方差不存在**(Pareto α≤2、Cauchy 分布——CLT 直接失效)
- **样本不独立**(时间序列、自相关)
- **想看尾部概率**——CLT 只对均值附近的近似好,**P99 / P99.9 用 CLT 算误差很大**(尾部要靠极值理论 EVT 或者 Bootstrap)

> **避坑**:**CLT 是"均值接近高斯",不是"数据接近高斯"**。这是统计课最常见的错记。监控里报"95% 请求落在 mean ± 2σ" 用了一个**错误的 CLT 应用**——你描述的是单条数据,不是均值。

## 七、Bootstrap:不知道分布也能算置信区间

### 7.1 思路

CLT 告诉你均值近似高斯,但**前提是 N 够大、方差有限**。如果手头只有 50 条样本、还是奇怪分布,怎么办?

**Bootstrap 的回答**:**手头那 50 条样本就当"总体"用,从里面有放回地重采样 50 个,算一次均值;重复 1 万次,得到 1 万个均值;这堆均值的分布,就是均值的抽样分布**。直接取它的 2.5% / 97.5% 分位数 = 95% 置信区间。

**这玩意儿听起来像作弊,但数学上证明它对很多统计量都收敛**(Efron 1979)。**关键的工程价值**:**不需要假设分布,不需要推公式,不需要 CLT 成立**——一段代码搞定。

### 7.2 一段最小代码

```python
import numpy as np

def bootstrap_ci(data, statistic=np.mean, n_boot=10_000, ci=0.95):
    """非参数自助法置信区间"""
    rng = np.random.default_rng(42)
    n = len(data)
    boot_stats = np.empty(n_boot)
    for i in range(n_boot):
        sample = rng.choice(data, size=n, replace=True)   # 关键:有放回
        boot_stats[i] = statistic(sample)
    alpha = (1 - ci) / 2
    return np.quantile(boot_stats, [alpha, 1 - alpha])

# 例子:50 条延迟数据,有几条很慢
latencies = np.array([42, 50, 55, 60, 48, 51, 49, 53, ..., 1200, 980])
low, high = bootstrap_ci(latencies, np.mean)
print(f"均值 95% CI: [{low:.1f}, {high:.1f}] ms")

# 也能给中位数、P95 算 CI:
low_p95, high_p95 = bootstrap_ci(latencies, lambda x: np.quantile(x, 0.95))
```

**对比 t 检验的 CI**:t 检验只能给均值算 CI 且要假设近高斯;**Bootstrap 能给任何统计量(中位数、分位数、相关系数、AUC)算 CI,且不假设分布**。

### 7.3 Bootstrap 的局限

- **样本本身得有代表性**——如果原始 50 条样本就有偏(比如全是周末数据),Bootstrap 也救不了
- **极端分位数(P99.9 之类)算不准**——你的 50 条样本里压根没出现真正的尾部,重采样也变不出来
- **不独立的数据要用 Block Bootstrap**——时间序列直接 Bootstrap 会破坏自相关结构
- **n_boot 至少 1000,常用 10000**——太少结果抖

> **经验法则**:**不知道用什么方法估计就用 Bootstrap**。它慢一点(10000 次重采样),但**结论比手推 t 检验稳得多**,而且代码 5 行写完。**小样本 + 不知道分布 + 想算个 CI**——直接 Bootstrap。

## 八、代码:把上面所有概念跑一遍

### 8.1 验证 CLT(均匀分布的均值接近高斯)

```python
import numpy as np
import matplotlib.pyplot as plt

rng = np.random.default_rng(0)

fig, axes = plt.subplots(1, 4, figsize=(16, 3))
for ax, N in zip(axes, [1, 2, 10, 30]):
    means = [rng.uniform(0, 1, N).mean() for _ in range(10_000)]
    ax.hist(means, bins=50)
    ax.set_title(f"N={N}, mean of mean={np.mean(means):.3f}")
# N=30 已经是漂亮的钟形
```

### 8.2 验证 1/√N 律

```python
sigma_x = 1.0
for N in [10, 100, 1000, 10000]:
    means = [rng.normal(0, sigma_x, N).mean() for _ in range(2000)]
    se = np.std(means)
    theory = sigma_x / np.sqrt(N)
    print(f"N={N:5d}  实测σ(mean)={se:.4f}  理论 σ/√N={theory:.4f}")
# 实测和理论几乎重合
```

### 8.3 长尾分布让均值不收敛

```python
# Pareto α=1.2 的重尾分布
samples = rng.pareto(1.2, 1_000_000) + 1
running_mean = np.cumsum(samples) / np.arange(1, len(samples)+1)
print("到 100 万次样本均值仍在跳:")
print(running_mean[[100, 1000, 10_000, 100_000, 999_999]])
# 输出像 [3.2, 5.1, 4.3, 7.8, 6.1] —— 完全没收敛
```

### 8.4 Bootstrap vs t 检验置信区间

```python
from scipy import stats

# 50 条带几条极端值的数据
data = np.concatenate([rng.normal(50, 5, 48), [200, 250]])

# 经典 t 检验 CI(假设高斯)
mean, se = data.mean(), data.std(ddof=1)/np.sqrt(len(data))
t_low, t_high = stats.t.interval(0.95, len(data)-1, loc=mean, scale=se)

# Bootstrap CI(不假设分布)
boot_means = np.array([rng.choice(data, len(data), replace=True).mean()
                       for _ in range(10_000)])
b_low, b_high = np.quantile(boot_means, [0.025, 0.975])

print(f"t 检验 CI:    [{t_low:.1f}, {t_high:.1f}]")
print(f"Bootstrap CI:[{b_low:.1f}, {b_high:.1f}]")
# 两个 CI 都能给出区间,Bootstrap 通常会更宽更诚实
```

### 8.5 协方差矩阵(numpy 一行)

```python
X = rng.normal(0, 1, (1000, 3))   # 1000 个样本,3 个特征
X[:, 1] = X[:, 0] * 0.8 + X[:, 1] * 0.2   # feature 1 和 0 强相关
cov = np.cov(X.T)              # (3,3) 协方差矩阵
corr = np.corrcoef(X.T)        # (3,3) 相关矩阵
print(corr)
# 对角线全是 1,corr[0,1] ≈ 0.97
```

## 九、工程对应与局限

### 9.1 库与 API 对照

| 任务 | 库 / API |
| --- | --- |
| 描述统计(mean/var/std) | `numpy.mean / .var(ddof=1) / .std(ddof=1)` |
| 协方差 / 相关矩阵 | `np.cov / np.corrcoef / pd.DataFrame.corr` |
| 拟合分布 | `scipy.stats.norm.fit / .lognorm.fit / .pareto.fit` |
| t 检验 / z 检验 / KS 检验 | `scipy.stats.ttest_ind / .ks_2samp` |
| Bootstrap | `scipy.stats.bootstrap`(SciPy ≥1.7) |
| 分位数 | `np.quantile / np.percentile`(注意 q 和 p 都用) |
| 稳健统计(中位数 / MAD) | `np.median / scipy.stats.median_abs_deviation` |
| 置信区间 | `scipy.stats.t.interval / .norm.interval / .bootstrap` |

`ddof=1` 是工程师最常忘的事:**numpy 默认 `ddof=0` 算的是有偏方差,统计推断要 `ddof=1`(无偏)**。两者差 N/(N-1),N 大时无所谓,N=20 时差 5%。

### 9.2 监控为什么用分位数而不是 mean ± σ

线上指标(延迟、错误率、营收)**几乎从不是高斯**,直接套 mean ± σ 会犯下面三种错:

| 错法 | 真实情况 |
| --- | --- |
| "99% 请求落在 mean ± 3σ"(误用经验法则) | 长尾分布下 P99 远大于 mean + 3σ |
| 报告写 "200ms ± 50ms" | 看着稳,实际 P99 = 800ms,用户体感很差 |
| 用 "均值大涨 50%" 触发告警 | 一个 5s 慢请求能把均值翻 10 倍,导致误告警 |

**正确姿势**:**报 P50 / P95 / P99,辅以最大值;告警阈值设在分位数上(P99 > 500ms)而不是均值上**。Prometheus 的 `histogram_quantile`、Grafana 的 percentile 面板,都是为这个设计的。

### 9.3 SGD 与 batch size 的统计含义

SGD 的核心是"用一个 mini-batch 估计整个数据集的梯度均值"。前面的 1/√N 律告诉你:

```
batch_size = 32  →  梯度估计标准误差 ∝ 1/√32  ≈ 0.177
batch_size = 128 →  梯度估计标准误差 ∝ 1/√128 ≈ 0.088   (减半)
batch_size = 512 →  梯度估计标准误差 ∝ 1/√512 ≈ 0.044   (再减半)
```

**batch size 翻 4 倍,梯度噪声减半**——这就是为什么 batch size 不是越大越好:**适当的噪声是正则化**,batch size 太大反而泛化变差(参考 LeCun 的"Don't decay the learning rate, increase the batch size"系列讨论)。

### 9.4 什么时候不能用这套工具

| 场景 | 该换什么 |
| --- | --- |
| 极端长尾(Pareto α<2) | 不用均值,用中位数 / 分位数 |
| 时间序列(自相关) | Block Bootstrap、HAC 标准误差(Newey-West) |
| 极端分位数(P99.99) | 极值理论(GEV、GPD)而不是经验分位数 |
| 因果推断 | 协方差/相关只能描述,因果要 RCT 或 do-calculus |
| 多峰分布 | mean / σ 都没意义,要先聚类或 mixture model |
| 离群点多 | 用稳健统计(中位数、MAD、trimmed mean) |

> **核心避坑清单**:
> 1. 报均值前问一句"这分布有方差吗、是不是长尾"——长尾就改报分位数
> 2. 看到 ρ=0.9 先画散点图——可能是离群点撑起的
> 3. CLT 是"均值接近高斯",不是"数据接近高斯"
> 4. 想算置信区间但分布不熟、样本小,直接 Bootstrap
> 5. 监控告警阈值不要设在均值上,设在分位数上
> 6. SGD batch size 不是越大越好,1/√N 律告诉你边际越来越低
