马尔可夫链与随机过程 ​

如果说 27 篇是用矩阵语言把"图"看穿,这一篇就是用矩阵语言把"时间序列上的随机过程"看穿。HMM 给了语音识别和生信序列分析二十年的底子;MCMC 是贝叶斯统计走通的引擎;MDP 是强化学习从理论到 AlphaGo / PPO / RLHF 的数学骨架;Diffusion 模型的前向加噪也是一条马尔可夫链——这一坨东西底下是同一套:转移矩阵 + 平稳分布 + 一个特征值控制收敛速率。

一句话先记住:马尔可夫链 = "下一步只看现在"的随机过程。它的全部行为被一个矩阵 P 装下,P^n 告诉你 n 步后的分布,第二大特征值 λ₂ 控制你要等多久才收敛到平稳分布 π(spectral gap 越大收敛越快——和 27 篇 PageRank 是同一个数学)。HMM 是状态藏起来的马尔可夫链,MDP 是状态上能选动作的马尔可夫链,MCMC 是反过来"造一条马尔可夫链让它的平稳分布等于我要采样的目标分布"。理解到这一层,RL / 贝叶斯 / 序列建模的论文你都能直接读到底。

一、为什么要这层数学 ​

场景不懂马尔可夫时懂一点之后
看 RL 论文 `s_{t+1} ~ P(·s_t, a_t)`"这堆下标在干啥"
PyMC 跑 MCMC 不收敛加更多 sample看 trace plot,调步长,看 R̂
写一个聊天 session 的状态机当作普通有限状态机知道哪些假设违反"马尔可夫性"
Diffusion 模型代码 x_t = √α·x_{t-1} + √(1-α)·ε不知道这是马尔可夫链前向是 markov chain,逆向是学出来的近似
推荐系统的"用户序列建模"全 RNN/Transformer知道用 HMM 也能做、各有什么假设

经验法则:"下一步只依赖现在"——这个假设是强力但不无害的简化。能用马尔可夫链建模就用,因为数学和算法成熟;不能用就别硬上(用户购买的"今天买啥还看上周买啥"明显违反马尔可夫性,这种就该用 RNN / Transformer 上更长上下文)。

二、马尔可夫性:下一步只看现在 ​

2.1 定义 ​

马尔可夫性(一阶):

P(X_{t+1} = j | X_t = i, X_{t-1}, ..., X_0) = P(X_{t+1} = j | X_t = i)

给定现在,过去和未来独立。所有历史信息被"现在的状态"完全压缩了。

2.2 这个假设何时成立、何时不成立 ​

成立(或近似成立):                       不成立(或勉强):
─────────────────────                ─────────────────────
天气模型(简化版,今天预测明天)         用户在电商的下次点击(依赖整周浏览历史)
棋盘游戏当前局面 → 下一局面             人的对话上下文(依赖前面几十轮)
分子动力学的下一时刻位置和速度          股票收益(微结构 + 宏观经济多层依赖)
PageRank 的随机游走                    自然语言文本(长程依赖 = LLM 出现的根本原因)
RL 里"明确建模为状态"的环境             遗忘曲线 / 个性化记忆系统

避坑:马尔可夫性的"破"和"补"是建模的常见手段:

  • 状态扩展:不满足 1 阶马尔可夫的过程,把过去 k 步打包进状态可能就满足了(k 阶马尔可夫,转 1 阶时状态空间膨胀 k 倍)
  • 隐变量:观测不马尔可夫,但背后存在一个状态过程是马尔可夫的——这就是 HMM
  • 彻底放弃:换 RNN / Transformer / 状态空间模型(Mamba 一类),用神经网络去学转移

三、转移矩阵 P 与 n 步转移 ​

3.1 P 是什么 ​

有 N 个状态的马尔可夫链:

P[i, j] = 从状态 i 一步转到状态 j 的概率

性质:
  - P[i, j] ≥ 0
  - 每行之和 = 1     (从 i 出去的概率加起来是 1)

每个状态分布是一个行向量 π_t,一步演化:

π_{t+1} = π_t · P            (左乘 P,行向量约定)

3.2 一个三状态例子(天气) ​

状态:晴(S)、阴(C)、雨(R)
转移图:

      0.7         0.2
   ┌─────┐     ┌─────┐
   ▼     │     ▼     │
  晴 ────┼──> 阴 ────┼──> 雨
   │ 0.2 │ 0.5 │ 0.3 │ 0.4
   │     ▼     │     ▼
   │    0.1    │    0.3
   └──────────────────┘

P = ┌ 0.7  0.2  0.1 ┐    ← 行 = "今天晴"时,明天的分布
    │ 0.3  0.5  0.2 │    ← 行 = "今天阴"
    └ 0.2  0.4  0.4 ┘    ← 行 = "今天雨"

3.3 n 步转移 = P^n ​

π_{t+n} = π_t · P^n

P^n[i, j] = 从 i 开始走 n 步到 j 的概率。

python
import numpy as np

P = np.array([[0.7, 0.2, 0.1],
              [0.3, 0.5, 0.2],
              [0.2, 0.4, 0.4]])

# 从"今天晴"出发,3 天后的分布
pi_0 = np.array([1.0, 0.0, 0.0])
pi_3 = pi_0 @ np.linalg.matrix_power(P, 3)
print(np.round(pi_3, 3))    # [0.527 0.295 0.178]

# 长期分布(随便从哪开始,P^n 趋同到同一行)
pi_long = pi_0 @ np.linalg.matrix_power(P, 100)
print(np.round(pi_long, 3))  # [0.467 0.311 0.222] ← 平稳分布

工程意义:P^n 在 RL 里就是"基于 model 的规划";在 PageRank 里就是"幂迭代";在 Diffusion 模型里就是"前向加噪 t 步后的边际分布"。整章的"加法"全在这一行矩阵幂里。

四、平稳分布 π:链的"长期归宿" ​

4.1 定义 ​

平稳分布 π 满足:

π · P = π            (行向量,π 是 P 的左特征向量,特征值 = 1)
Σ π[i] = 1            (概率分布)

直觉:一旦链进入 π 这个分布,再多走多少步,分布不变了。

4.2 何时存在唯一 ​

不可约(Irreducible):    任意两个状态互相可达(链是"强连通"的)
非周期(Aperiodic):      存在某个状态 i,使得"回到 i 的步数"的 gcd 是 1
                          (不会卡在"必须 2 步一回"这种节奏里)

两条都满足 ⇒ 平稳分布唯一存在 ⇒ 任意初始分布都收敛到 π

Perron-Frobenius 定理保证这个唯一性——和 27 篇 PageRank 是同一个数学根。

4.3 怎么算 π ​

两种工程做法:

python
# 方法 1:幂迭代(适合稀疏大矩阵)
pi = np.ones(3) / 3
for _ in range(1000):
    pi = pi @ P
print(np.round(pi, 3))      # [0.467 0.311 0.222]

# 方法 2:解特征方程(适合稠密小矩阵)
eigvals, eigvecs = np.linalg.eig(P.T)        # P 的左特征向量 = P.T 的右特征向量
idx = np.argmax(np.abs(eigvals - 1.0) < 1e-9)
pi = np.real(eigvecs[:, idx])
pi = pi / pi.sum()
print(np.round(pi, 3))      # [0.467 0.311 0.222]

4.4 周期性反例 ​

有向环:     0 → 1 → 2 → 0     P 是纯循环置换
                                每 3 步精确回到原位
                                链有周期 3,不收敛(分布 3 步循环)

这种链有平稳分布但不收敛;给它加一点点"原地停留"概率,马上变非周期——这就是 PageRank 的阻尼项 (1-α)/N 在干的事(顺便解决悬挂节点)。

五、收敛速率:第二大特征值控制一切 ​

5.1 核心结论 ​

P 的特征值按绝对值排:1 = |λ₁| > |λ₂| ≥ |λ₃| ≥ ... ≥ |λ_N|(不可约非周期时第一个严格大于其他)。

任意初始 π_0,演化 n 步后到 π_∞ 的距离:

  ||π_n - π||_TV  ≤  C · |λ₂|^n

其中 || · ||_TV 是总变差距离(分布间差异的度量)。

核心直觉:|λ₂| 越小(spectral gap = 1 - |λ₂| 越大),链收敛越快。这是 27 篇谱图里"代数连通度"在时间维度的版本——同一种"主结构 + 第二结构"的二分,在静态图上叫连通度,在动态过程上叫混合时间。

5.2 一个工程估算 ​

天气链 P 的特征值:
  λ₁ = 1.0     (平稳)
  λ₂ ≈ 0.45    (次主)
  λ₃ ≈ 0.15

要达到误差 < 0.01 的收敛:
  0.45^n < 0.01  ⇒  n > log(0.01)/log(0.45) ≈ 5.8

→ 大概 6 步就收敛了
python
eigvals = np.sort(np.abs(np.linalg.eigvals(P)))[::-1]
print(eigvals)                     # [1.0, 0.452, 0.148]
mixing_time_bound = np.log(0.01) / np.log(eigvals[1])
print("近似 mixing time:", mixing_time_bound)

避坑:MCMC 实战里你看不到 λ₂(链太大根本算不动),只能"经验估计"——看 trace plot 是否稳定、看自相关函数衰减、看 R̂(下一节)。这些都是 |λ₂| 的代理。

5.3 在不同系统里 λ₂ 是什么 ​

系统λ₂ 含义
PageRank第二大特征值 ≤ α(阻尼);α 越小收敛越快
MCMCmixing time;链能不能跳出 mode,体现在 λ₂ 大不大
谱聚类图的"代数连通度";λ₂ 接近 0 说明图能切两半
GNN邻接矩阵第二大特征值,与"过平滑"有关

这些场景里同一个 λ₂ 在做同一件事:衡量"第一主结构和第二主结构的差距"。这是整章离散数学最值钱的串联——27 + 29 两篇看完会形成一个"谱视角"的小宇宙。

六、HMM:隐藏的马尔可夫链 + 观测 ​

6.1 模型 ​

隐状态:   Z_1 → Z_2 → Z_3 → ...     (一个马尔可夫链,你看不见)
            │     │     │
            ▼     ▼     ▼
观测:      X_1   X_2   X_3           (你能观察到的输出)

参数:

  • 初始分布 π
  • 转移矩阵 A,A[i, j] = P(Z_{t+1}=j | Z_t=i)
  • 发射(emission)概率 B,B[i, x] = P(X_t = x | Z_t = i)

6.2 三个经典问题 ​

问题输入输出算法
评估(Evaluation)模型 + 观测序列 X给定模型下,这条序列的概率 P(Xθ)
解码(Decoding)模型 + 观测序列 X最可能的隐状态序列 Z*Viterbi 算法
学习(Learning)观测序列 X(无标注)模型参数 θ = (π, A, B)Baum-Welch(EM 在 HMM 上的特化)

6.3 各算法的直觉(不展开) ​

前向算法:  α_t(i) = "看到 X_1..X_t 且 Z_t = i 的联合概率"
           递推:α_{t+1}(j) = (Σ_i α_t(i) · A[i,j]) · B[j, X_{t+1}]
           总概率 = Σ_i α_T(i)
           复杂度 O(T·N²)

Viterbi:   把 α 里的"加和" 换成"取 max",并记录最优前驱
           回溯时沿最优前驱链拿到最可能路径
           复杂度 O(T·N²)

Baum-Welch:E 步用前向后向算 P(Z_t = i | X)
           M 步用这些后验更新 A、B
           EM 算法的标准节奏,保证似然单调上升(不保证全局最优)

核心直觉:前向算法和 Viterbi 是同一个递推骨架,差别只在"加 vs max"。CTC、Beam Search、CRF 都是这套递推的近亲——序列建模里这个 DP 框架值钱得离谱。

6.4 工程现状 ​

场景HMM 仍在用吗
语音识别已大面积被 CTC / Transformer 取代,但 GMM-HMM 是几代语音系统的根
词性标注 / NER主流是 BiLSTM-CRF / Transformer,CRF 是 HMM 的判别式表亲
生信(基因序列、蛋白结构)HMM 仍是工具箱核心(HMMER、Pfam、隐马尔可夫剖面)
简单状态推断(用户登录状态、设备状态机)小问题上 HMM 比深度学习更可控
时间序列异常检测部分场景仍用,但被状态空间模型(Kalman / Particle Filter)挤压

七、马尔可夫决策过程(MDP):RL 的数学骨架 ​

7.1 五元组 ​

M = (S, A, P, R, γ)

S:状态集合
A:动作集合
P(s' | s, a):转移函数(执行动作 a 后状态怎么变,可能随机)
R(s, a) 或 R(s, a, s'):奖励函数
γ ∈ [0, 1):折扣因子(未来奖励的衰减)

HMM 是"看不见状态的马尔可夫链";MDP 是"状态上能选动作的马尔可夫链"——增加了 agent 的决策权。

7.2 策略与价值函数 ​

策略 π:S → A    (或 S → 分布(A),随机策略)
价值函数:
  V^π(s)  = E[ Σ_{t≥0} γ^t · R_t | s_0=s, 跟随 π ]
  Q^π(s,a) = E[ Σ_{t≥0} γ^t · R_t | s_0=s, a_0=a, 此后跟随 π ]

V 是从 s 出发的长期累积奖励期望;Q 是先做 a 再跟 π 的版本。

7.3 Bellman 方程 ​

自一致性——价值函数等于"立即奖励 + γ · 下一步的价值":

V^π(s) = Σ_a π(a|s) · [R(s,a) + γ · Σ_{s'} P(s'|s,a) · V^π(s')]

Q^π(s,a) = R(s,a) + γ · Σ_{s'} P(s'|s,a) · V^π(s')

最优 Bellman 方程(把 π 换成最大化):

V*(s)   = max_a [R(s,a) + γ · Σ_{s'} P(s'|s,a) · V*(s')]
Q*(s,a) = R(s,a) + γ · Σ_{s'} P(s'|s,a) · max_{a'} Q*(s', a')

7.4 两种经典求解(模型已知) ​

价值迭代(Value Iteration):
  反复对 V(s) 用最优 Bellman 方程更新,直到收敛
  收敛速率: γ^n (γ < 1 保证压缩映射)

策略迭代(Policy Iteration):
  策略评估:在当前 π 下解线性方程算 V^π
  策略改进:对每个 s 选 argmax_a Q^π(s,a) → 新 π
  反复直到 π 不变

核心直觉:价值迭代和策略迭代都是不动点迭代——前者直接求 V*,后者交替求 V^π 和改进 π。γ 是收缩因子,γ < 1 保证存在唯一不动点(和马尔可夫链有平稳分布的逻辑同源)。

7.5 Q-learning(模型未知) ​

RL 在工程里几乎不是 MDP 模型已知——P 和 R 都得从交互中学。Q-learning 直接学 Q*:

Q(s, a) ← Q(s, a) + α · [r + γ · max_{a'} Q(s', a') - Q(s, a)]
                          └────────── TD target ──────────┘

不展开 DQN / PPO 的细节,这是 aiLearning 系列 RL 篇的活。这里只点:RL 的整个数学骨架就是 MDP + Bellman + 各种近似 Bellman 算子的方法。

7.6 工程对应 ​

库适合
Stable Baselines3入门 + 标准算法(PPO、SAC、A2C),封装好
RLlib(Ray)大规模分布式 RL,Anthropic / OpenAI / Meta 都用过类似栈
CleanRL单文件实现,看清算法细节
Gymnasium(原 OpenAI Gym)环境标准化接口
PettingZoo多智能体 RL 环境

避坑:RL 调参极其不稳定——同一份代码、同一组超参,换个 seed 收敛/发散/掉到鞍点的概率不低。论文报"平均 10 个 seed 的曲线"才看,单条曲线没意义。Henderson 等人 Deep RL That Matters 这篇 2018 文章今天仍然成立。

八、MCMC:反向用马尔可夫链 ​

8.1 核心想法 ​

正向:有一个马尔可夫链,我去算它的平稳分布 π。 MCMC:有一个目标分布 p(x),我去构造一条马尔可夫链使它的平稳分布恰好是 p(x),然后跑这条链,链上的样本就是来自 p(x) 的样本。

为什么要这么干?
  贝叶斯后验 p(θ|D) ∝ p(D|θ) · p(θ)
  分母 p(D) = ∫p(D|θ)p(θ)dθ 通常算不出来(高维积分)
  但!分子可以逐点算 → 用 MCMC 采样,不需要归一化常数

8.2 Metropolis-Hastings:最经典的 MCMC ​

构造转移过程:

1. 当前样本 x,提议一个新样本 x' ~ q(x' | x)        ← 任意"提议分布"
2. 算接受概率:
      α = min(1,  [p(x') · q(x | x')] / [p(x) · q(x' | x)] )
3. 以概率 α 接受(x ← x'),否则保持(x ← x)
ASCII 接受/拒绝示意:

     提议 x'    
       │
       ▼
    算 α = min(1, p(x')q(x|x') / p(x)q(x'|x))
       │
   ┌───┴───┐
   │       │
α≥1 → 收!  α<1
                │
              掷骰子 u ~ U(0,1)
                │
            ┌───┴───┐
            │       │
          u < α    u ≥ α
            │       │
          x ← x'   x 不变(把当前 x 再记一次)

关键不变量:这条链的平稳分布恰好是 p(x)——这是 MH 的"细致平衡"(detailed balance)给的保证,不需要 p 的归一化常数(分子分母都消掉了)。

python
import numpy as np

def metropolis(p_unnorm, x0, n_iter=10000, step=0.5):
    """从未归一化目标 p_unnorm 采样,高斯随机游走提议"""
    x = x0
    samples = []
    for _ in range(n_iter):
        x_new = x + np.random.randn() * step
        ratio = p_unnorm(x_new) / p_unnorm(x)
        if np.random.rand() < min(1, ratio):
            x = x_new
        samples.append(x)
    return np.array(samples)

# 用 MH 采"双峰高斯混合"
def target(x):
    return np.exp(-(x-2)**2/2) + np.exp(-(x+2)**2/2)

samples = metropolis(target, x0=0.0, n_iter=20000, step=1.0)
print("均值:", samples[2000:].mean())   # ≈ 0
print("标准差:", samples[2000:].std())  # ≈ 2.3

8.3 Gibbs 采样:MH 的特例 ​

多维分布 p(x_1, ..., x_d):每次只更新一个维度,其他维度保持不变,用该维度的条件分布采样:

循环:
  x_1 ← 从 p(x_1 | x_2, ..., x_d) 采样
  x_2 ← 从 p(x_2 | x_1, x_3, ..., x_d) 采样
  ...
  x_d ← 从 p(x_d | x_1, ..., x_{d-1}) 采样

核心直觉:Gibbs 是 MH 在"提议分布 = 条件分布"时的退化——接受概率永远等于 1。前提是"每个维度的条件分布能解析采样"——共轭先验 / LDA / 玻尔兹曼机这些场景成立。

8.4 更现代的:HMC 与 NUTS ​

  • Hamiltonian Monte Carlo(HMC):利用梯度信息让提议"沿着概率高的方向走",比随机游走 MH 在高维下高效几个数量级。
  • NUTS(No-U-Turn Sampler):HMC 的自适应版本,自动决定走多远——Stan / PyMC / NumPyro 的默认采样器。

8.6 收敛诊断:实战里看什么 ​

MCMC 不像 SGD 有明确的 loss 曲线,收敛要靠"诊断"判断:

诊断干啥用经验阈值
trace plot看每个参数的链是不是"在平稳分布里晃"视觉,毛毛虫状最佳;有 trend 说明没收敛
burn-in扔掉前 k 步(链还没进入 π)通常前 1000-5000 步丢掉,N 越大越保险
thin / 自相关函数链相邻样本相关性高,需要降采样自相关衰减到 0.1 的 lag 即采样间隔
有效样本数 ESS算"等价于多少独立样本"ESS > 400 / 参数(PyMC 建议)
R̂(Gelman-Rubin)多条独立链(通常 4 条)的"链间方差 vs 链内方差"R̂ < 1.01 = 收敛;> 1.1 一定没收敛

避坑:MCMC 不收敛时,先看 trace plot,而不是加 sample。trace 是平的还是漂的,直接告诉你问题——漂说明 step size 太大或目标分布多峰跳不过去;贴边说明先验/约束错了。"加更多 sample 就行"是 MCMC 实战最常见的错误经验。

九、Diffusion 模型里的马尔可夫链 ​

(只点一下,展开是 aiLearning 的活。)

前向过程:一条马尔可夫链,逐步向数据加噪:

x_0   →   x_1   →   x_2   →   ...   →   x_T  ≈ 纯噪声
        加点         加点                  加点
        噪声         噪声                  噪声

   q(x_t | x_{t-1}) = N(√(1-β_t) · x_{t-1},  β_t · I)

逆向过程:学一个神经网络去近似真正的逆向条件分布 p_θ(x_{t-1} | x_t),从纯噪声一步步还原成数据。

核心直觉:Diffusion 的"前向加噪"是闭式马尔可夫链,不学;"反向去噪"是神经网络近似的非马尔可夫(理论上是 Gaussian 但参数从 x_t 学出来)。VAE / Score-based / DDPM / DDIM 的差异在"反向那条链怎么参数化和采样",前向都是同一条加噪链。

10.2 局限与什么时候不该用 ​

  • 马尔可夫性是强假设——很多真实序列(对话、长文本、用户行为)显然不满足,硬上 HMM 会比 Transformer 差几个数量级。
  • MDP 的状态空间在工程里通常是观测,不是真实状态——**部分可观测 MDP(POMDP)**比 MDP 难得多,实战里大多用"窗口状态 + 神经网络"近似糊弄过去。
  • MCMC 在高维下慢——参数 > 100 时 MH 几乎不收敛,得用 HMC/NUTS 或干脆换变分推断(VI)。
  • MCMC 不能保证你看到了所有 mode——多峰分布上链可能卡在一个峰里,R̂ 看着还行,但其实漏了一半。多条独立链 + 多个起点 + 模拟退火是缓解手段。
  • Bellman 方程在连续状态/动作下没有闭式解——这是为什么 deep RL 需要函数逼近(神经网络当 V 和 Q),也是为什么 deep RL 这么难调。
  • HMM 对长依赖无能为力——一阶马尔可夫天然只看一步;状态扩展会让状态空间爆炸。这是 RNN / Transformer 出现的根本原因之一。

核心直觉:这一章所有算法的成功都建立在"马尔可夫链能收敛 + 收敛得够快"这两个前提上。理论收敛是干净的,工程收敛要靠诊断和调参——理论给你方向,trace plot 给你结论。
