    特征值与特征向量

    线性代数课本花一整章讲特征值,绝大部分人学完没记住"它解决什么工程问题"。然后看到 PageRank 公式、PCA 推导、RNN 梯度爆炸的分析,每个地方都出现 λ 和 v,以为是不同的东西——其实底下是同一件事:找矩阵作用下"不变方向"和"放缩倍数"。这一篇把这个概念串到 PageRank、PCA、协方差、Hessian、RNN 上,让你看到 Av=λv 时,知道它在工程里是哪个开关。

    一句话先记住:特征向量 = 矩阵作用下"方向不变"的方向,特征值 = 在这个方向上的"放缩倍数"。PageRank 的"重要网页"是邻接矩阵的最大特征向量;PCA 的"主成分"是协方差矩阵的特征向量;RNN 梯度爆炸/消失看权重矩阵的最大特征值 > 1 还是 < 1。全是同一个数学结构,套不同的工程壳子。

    一、为什么这一篇必须存在

    场景 不懂特征值 懂一点特征值
    看 PageRank 论文 "随机游走怎么就和矩阵特征向量等价了?" 一眼看出是幂迭代求最大特征向量
    RNN 训练 loss=NaN 乱试 gradient clip 阈值 看权重矩阵最大特征值 > 1 没
    PCA 取几个主成分 凭感觉 看特征值衰减谱,陡的地方截
    Hessian 训练动力学 "曲率"这个词云里雾里 Hessian 最大特征值 = 最陡方向的"二阶斜率"
    看 GNN 论文 "图拉普拉斯"是啥 图的连接结构 = 拉普拉斯矩阵的特征结构

    特征值是"看穿矩阵内部结构"的工具——一个 1000×1000 的矩阵直接看就是一百万个数,看特征值就只要看 1000 个数,而且这 1000 个数告诉你矩阵的"行为本质"。

    二、直觉:一个矩阵就是一种"扭曲"

    2.1 矩阵 = 线性变换

    把一个矩阵 A 想成一个"作用":给我一个向量 x,我吐出一个新向量 Ax。这个作用扭曲了空间——把圆变椭圆,把方块变菱形,把整个平面拉伸/旋转。

    普通向量被 A 作用后,方向会变:

   y                                       y
   ▲    x = (1, 1)                         ▲
   │       ↗                               │      Ax = (3, 1) ← 方向变了
   │     ↗                                 │           ↗
   │   ↗                  ─── A 作用 ──→  │        ↗
   │ ↗                                     │     ↗
   ┼───────────► x                         ┼─────────────► x

    但是某些特殊方向,A 作用后只是被拉长/压短,方向不变——这就是特征向量。

    2.2 Av = λv:不变方向 + 放缩倍数

    特征向量 v,被 A 作用后:

   y                                       y
   ▲                                       ▲
   │                                       │
   │     v = (1, 0.5)                      │      Av = (2, 1) ← 方向相同
   │      ↗                                │            ↗
   │    ↗                                  │          ↗
   │  ↗                   ─── A 作用 ──→  │        ↗  ← 长度变 2 倍
   │↗                                      │      ↗      所以 λ = 2
   ┼─────────────► x                       ┼─────────────► x

    Av = λv 的几何意义: - 向量 v 在 A 作用下,方向不变(在同一条直线上) - 只是被"放缩"了 λ 倍

    λ 是特征值,可以是任意实数(甚至复数): - λ > 1:被拉长 - 0 < λ < 1:被压短 - λ < 0:方向反转 + 拉长/压短 - λ = 0:压成零(v 落在 A 的零空间)

    2.3 一个矩阵有几个特征向量

    n×n 矩阵最多 n 个线性无关的特征向量(实数情况下可能更少,复数情况下保证 n 个)。

    A = [3  0]
    [0  2]

A · (1, 0) = (3, 0) = 3 · (1, 0)    ← v1=(1,0), λ1=3
A · (0, 1) = (0, 2) = 2 · (0, 1)    ← v2=(0,1), λ2=2

这是个对角矩阵,特征向量就是坐标轴

    非对角矩阵的特征向量不在坐标轴上,但作用本质一样——先找特征向量这组基,然后在每个方向上独立缩放。

    三、对角化:把扭曲拆成"独立缩放"

    3.1 为什么对角化重要

    普通矩阵作用很复杂(同时旋转 + 拉伸 + 剪切),对角矩阵作用很简单(每个坐标轴独立缩放)。

    对角矩阵 D 的作用:
  D = diag(λ1, λ2, ..., λn)
  D · x = (λ1·x1, λ2·x2, ..., λn·xn)   ← 每个分量独立缩放

    对角化的核心想法:换一组基(用特征向量当基),让原本复杂的矩阵在新基下变成对角矩阵。

    3.2 对角化的代数形式

    A = P D P^(-1)

其中:
  P 的列 = A 的特征向量 v1, v2, ..., vn
  D = diag(λ1, λ2, ..., λn) ← 特征值在对角线

直觉:
  P^(-1) ── 把 x 转换到"特征向量基"下
  D      ── 在每个特征方向独立缩放
  P      ── 转换回原来的基

            x
        │
        ▼ P^(-1):换到特征向量基下
     y = P^(-1) x
        │
        ▼ D:每个方向独立缩放
     Dy
        │
        ▼ P:换回原来的基
       Ax

    核心直觉:对角化 = 找一组"好基",让矩阵的复杂作用简化为各方向独立伸缩。这是线性代数里所有"分解"的祖师爷(SVD 的近亲就在这)。

    3.3 矩阵的幂:对角化的杀手锏

    为什么对角化重要? 因为算矩阵的幂 A^k 变得超级简单:

    A^k = (P D P^(-1))^k
    = P D P^(-1) · P D P^(-1) · ... · P D P^(-1)
    = P D^k P^(-1)

而 D^k = diag(λ1^k, λ2^k, ..., λn^k)  ← 标量的幂,O(n)

    O(n³ log k) 暴力 vs O(n²) 对角化后——这就是 PageRank、Markov chain、RNN 分析都能用特征值的根本原因:它们都涉及"矩阵反复作用很多次"。

    四、谱分解:对称矩阵的"福利套餐"

    对称矩阵(A^T = A)有一个超棒的性质——所有特征值都是实数,而且特征向量可以选成两两正交。

    对称矩阵的谱分解:
  A = Q Λ Q^T

其中:
  Q 的列是正交单位特征向量 (Q^T Q = I,Q^T = Q^(-1))
  Λ = diag(λ1, ..., λn),全是实数
    矩阵类型 特征值 特征向量 实例
    一般矩阵 复数 可能不存在完整一组 RNN 权重 W
    对称矩阵 实数 正交一组 协方差矩阵、Hessian
    正定矩阵 实数 > 0 正交一组 协方差矩阵、Gram 矩阵
    正交矩阵 λ =1
    随机矩阵(行和=1) 1 是最大特征值 平稳分布 PageRank 转移矩阵

    为什么协方差矩阵这么受欢迎? 因为它对称 + 半正定——拥有所有特征分解的好性质。PCA 整套基建立在协方差矩阵的谱分解上(下一节展开)。

    经验法则:只要看到"对称矩阵 + 求特征值",NumPy 用 eigh 不用 eig——快好几倍、数值更稳。

    五、PCA:协方差矩阵的特征向量 = 数据的主方向

    5.1 数据的"形状"

    把数据画成散点图,你会看到它在某些方向上"散得开",某些方向上"挤在一起"。协方差矩阵就是描述这件事的数学对象:

    散点图(2D 数据):

        y
        ▲
        │      ⋅ ⋅
        │   ⋅ ⋅ ⋅ ⋅       ← 主方向(数据散得最开)
        │  ⋅ ⋅ ⋅ ⋅ ⋅
        │ ⋅ ⋅ ⋅ ⋅ ⋅ ⋅
        │⋅ ⋅ ⋅ ⋅ ⋅
        │ ⋅ ⋅ ⋅ ⋅                 ↗
        │  ⋅ ⋅ ⋅                ↗
        │     ⋅              ↗
        │                ↗
        ┼─────────────────────────► x
                        ↓
                  次方向(散得最少)

    协方差矩阵 C 的特征向量 = 数据的"主方向"。对应的特征值 = 数据在这个方向上的方差。

    5.2 PCA 的核心一步

    PCA 算法:

1. 数据中心化:X = X - mean(X)
2. 算协方差矩阵:C = X^T X / (n-1)    ← 对称、半正定
3. 谱分解:C = Q Λ Q^T
4. 把 Λ 的特征值从大到小排
5. 取前 k 个特征向量当"新坐标轴"
6. 投影:Z = X · Q[:, :k]    ← 降维结果

直觉:在方差最大的几个方向上保留信息,其他方向丢掉

    为什么"方差大 = 信息多"? 因为方差小的方向上,所有点都挤在一起,这个维度没区分度——比如 1000 个用户每人都填了 "1990 年生",这个维度可以扔掉。

    核心直觉:PCA 不是"保留前 k 个特征",是"保留前 k 个特征方向"——这个方向是原始 n 个特征的线性组合,不一定对应任何"业务可解释"的特征。这是 PCA 和特征选择的本质区别。

    5.3 PCA 的工程使用

    场景 用 PCA 干什么 维度
    数据可视化 高维数据降到 2D 看一眼 k=2
    特征工程 把 1000 维降到 50 维喂模型 k=几十
    去噪 丢掉小特征值方向(噪声) k=多数
    Embedding 可视化 768 维 BERT embedding 降到 2D k=2,通常配 t-SNE/UMAP

    六、PageRank:互联网图的最大特征向量

    6.1 PageRank 的核心想法

    "如果很多重要网页都链接到你,那你也是重要的" ——这句话是"自指"的(重要 → 重要),数学上的对应物就是"特征向量"。

    设网页 i 的 PageRank 是 r_i,定义为:
  r_i = Σ_j (r_j / out_j) · [j 链接到 i]

其中 out_j 是网页 j 的出链数

写成矩阵形式:
  r = M r

其中 M 是"转移矩阵":M_ij = 1/out_j  如果 j → i
                              0       否则

这就是 r 是 M 的特征向量,对应特征值 1

    所以"重要网页"= M 的(对应特征值 λ=1 的)特征向量。这就是 Google 1998 年的核心想法,Brin 和 Page 拿了 Stanford PhD,顺手开了家公司。

    6.2 为什么 λ=1 一定存在

    M 是个"列随机矩阵"(每列加起来等于 1),数学上保证: - 最大特征值是 1 - 对应的特征向量元素全是非负 - 加点修正(damping factor)后,这个特征向量唯一

    PageRank 的完整公式:

    r = α M r + (1 - α) e / n

其中 α ≈ 0.85,e = (1, 1, ..., 1)^T

直觉:
  α M r        ← 跟着链接走
  (1-α) e/n    ← 1-α 概率随机跳到任意网页(防止 dead-end / spider trap)

    6.3 幂迭代:实际怎么算

    网页有数十亿个,直接谱分解不可能。幸运的是,我们只要最大特征值对应的特征向量,幂迭代够用:

    def power_iteration(M, num_iter=100):
    n = M.shape[0]
    r = np.ones(n) / n            # 初始均匀分布
    for _ in range(num_iter):
        r = M @ r                  # 反复作用
        r = r / np.linalg.norm(r)  # 归一化防止数值爆炸
    return r

    为什么会收敛?

    任意向量 x 可以写成特征向量的线性组合:
  x = c1 v1 + c2 v2 + ... + cn vn

M^k x = c1 λ1^k v1 + c2 λ2^k v2 + ... + cn λn^k vn

把 λ1 提出来:
  = λ1^k (c1 v1 + c2 (λ2/λ1)^k v2 + ...)

当 |λ2/λ1| < 1 时,后面项指数衰减
→ 只剩下 c1 v1,就是最大特征向量

    核心直觉:幂迭代的收敛速度由"前两个特征值之比" λ2/λ1 决定。这个比越小,收敛越快——这叫 spectral gap(谱隙)。PageRank 在真实互联网上 ~50 步收敛,因为 spectral gap 很大。

    6.4 PageRank 之外:同款数学的其他应用

    应用 矩阵是什么 最大特征向量代表什么
    PageRank 网页链接转移矩阵 网页"重要性"
    推荐系统 ItemRank 物品共现转移矩阵 物品"流行度"
    社交网络 EigenCentrality 社交图邻接矩阵 节点"影响力"
    Markov chain 平稳分布 状态转移矩阵 系统长期占据每个状态的概率
    Spectral clustering 图拉普拉斯矩阵 聚类结构(用前 k 个)

    全是同一套数学——找最大特征向量(或前 k 个)。

    七、谱半径:RNN 梯度爆炸/消失的根本原因

    7.1 RNN 反向传播里的矩阵幂

    RNN 的隐状态更新:h_t = tanh(W h_{t-1} + U x_t)

    反向传播到 t 步,梯度链式法则会出现 W^t(忽略 tanh 的话):

    ∂L / ∂h_0 ≈ ∂L / ∂h_t · W^t · (其他项)

W^t 的行为取决于 W 的特征值:
  谱半径 ρ(W) = max |λ_i|

  ρ(W) > 1 时,W^t 元素 → ∞    ← 梯度爆炸
  ρ(W) < 1 时,W^t 元素 → 0    ← 梯度消失
  ρ(W) = 1 时,W^t 保持不变(理想情况)

    核心直觉:RNN 训练崩溃的根源不是"序列太长",是"权重矩阵的特征值不在 1 附近"。LSTM 和 GRU 的门机制本质上是"自适应调整有效谱半径",让信息能传得更远。

    7.2 工程上看到的现象

    谱半径 > 1: 梯度爆炸
  ──────────────────────
  loss 突然变 NaN
  权重快速发散
  应对:gradient clipping(clip grad norm)

谱半径 < 1: 梯度消失
  ──────────────────────
  长序列依赖学不到
  早期 token 对最终输出几乎没影响
  应对:LSTM/GRU 门机制 / Transformer 用 self-attention 绕开

  ⚠ Transformer 之所以替代 RNN,部分原因就是它把"序列依赖"
    从矩阵幂(W^t)换成了一次性的 attention,不再有谱半径问题

    7.3 数值实验:看一眼谱半径的影响

    八、Hessian 的特征值:训练动力学

    Hessian 是 loss 对参数的"二阶导数矩阵"(H_ij = ∂²L / ∂θ_i ∂θ_j)——描述 loss 曲面在某点的"曲率"。Hessian 是对称矩阵,所以可以谱分解。

    8.1 Hessian 特征值告诉你什么

    某一点的 Hessian = Q Λ Q^T

Λ 的特征值 = loss 曲面在各个方向上的"曲率"

  λ 大     → 这个方向陡(曲率高)
  λ 小     → 这个方向平(曲率低)
  λ 为负   → 这个方向是下凸(非凸局部)

    Loss 曲面截面(2D 简化版):

  λ_max 方向(陡):
              loss
                │      ⋀
                │     ⋀ ⋀
                │    ⋀   ⋀
                │  ⋀       ⋀
                ┼─────────────► θ_max

  λ_min 方向(平):
              loss
                │ ─ ─ ─ ─ ─ ─
                │
                │
                ┼─────────────► θ_min

    8.2 训练里的实际后果

    特征值情况 影响
    λ_max / λ_min 很大(condition number) loss 曲面"窄又长",SGD 来回震荡,收敛慢
    全是正特征值 局部凸,SGD 朝下走
    有负特征值 鞍点(saddle point),loss 在某些方向还能下降
    接近零的特征值多 flat minima,泛化更好(经验观察)

    Adam / RMSProp 这类自适应优化器,本质是"按每个方向的曲率自动调步长"——曲率大的方向小步,曲率小的方向大步,等价于近似计算 Hessian 的对角线信息。

    核心直觉:学习率不是"全局选一个",是"每个参数选一个"——Adam 用梯度二阶矩近似 Hessian 的对角,这是它比 SGD 调参少的根本原因。

    九、数值求解:实际怎么算特征值

    理论上特征值是 det(A - λI) = 0 的根——但 n=1000 时这个多项式根本没法解。实际算法都是迭代的:

    9.1 算法谱

    算法 适合场景 复杂度 实际用
    幂迭代 只要最大那个 O(n²) per iter PageRank、谱聚类的最大特征向量
    QR 算法 全部特征值 O(n³) 总 numpy.linalg.eig / eigh 底层
    Lanczos 大稀疏对称矩阵的前 k 个 O(nnz · k) 图算法、量子化学
    Arnoldi 大稀疏一般矩阵的前 k 个 O(nnz · k) scipy.sparse.linalg.eigs
    Jacobi-Davidson 求特定区间的特征值 取决于谱 物理仿真

    9.2 幂迭代直觉

    反复作用 + 归一化:

    v ← random
repeat:
  v ← A @ v
  v ← v / ||v||     ← 防止溢出
λ ← v^T A v          ← Rayleigh quotient,估计特征值

    只能拿到最大那个——但前面 PageRank 已经看到,很多时候你只需要最大那个。

    9.3 QR 算法(只点一下)

    QR 算法是工业级"求全部特征值"的标准:

    A_0 = A
repeat:
  Q_k, R_k = QR(A_k)        ← QR 分解
  A_{k+1} = R_k Q_k         ← 反过来乘

最终 A_k 收敛到对角矩阵,对角元就是特征值

    为什么会收敛? 因为每次迭代"等价于一次幂迭代 + 正交化",直觉是"幂迭代同时跑 n 个,并且互相正交"。实际实现用 Householder 变换 + shift 加速,LAPACK 的 geev、NumPy 的 eig 都是这个。

    9.4 Lanczos:大稀疏矩阵的救星

    n = 10⁹(网页规模)时,O(n³) 的 QR 算法跑不动。Lanczos 算法用矩阵-向量乘把对称矩阵投影到一个 k 维 Krylov 子空间,在子空间内求特征值——只要矩阵向量乘快(稀疏矩阵就快),复杂度就和 nnz 成正比。

    应用场景:

    • 谱聚类(图拉普拉斯的前 k 个特征向量)
    • 物理仿真(超大对称矩阵)
    • 量子化学(哈密顿量的低能态)

def power_iter(A, num_iter=100, tol=1e-6):
    n = A.shape[0]
    v = np.random.randn(n)
    v = v / np.linalg.norm(v)
    for _ in range(num_iter):
        v_new = A @ v
        v_new = v_new / np.linalg.norm(v_new)
        if np.allclose(v, v_new, atol=tol):

        v = v_new

    PCA 完整一段

    def pca(X, k):
    # X: (n_samples, n_features)
    X_centered = X - X.mean(axis=0)
    C = X_centered.T @ X_centered / (X.shape[0] - 1)   # 协方差矩阵
    eigvals, eigvecs = np.linalg.eigh(C)
    # eigh 返回升序,反过来取前 k 个
    top_k = eigvecs[:, ::-1][:, :k]
    Z = X_centered @ top_k                              # 投影
    explained_ratio = eigvals[::-1][:k] / eigvals.sum()
    return Z, top_k, explained_ratio

X = np.random.randn(1000, 50)
Z, components, ratio = pca(X, k=10)
print(f"前 10 个主成分解释了 {ratio.sum()*100:.1f}% 的方差")

    坑 表现 避坑
    一般矩阵用 eig,期望实数特征值 拿到复数,后续报错 对称矩阵用 eigh;不确定时取实部 + 检查虚部
    大矩阵全部特征值 OOM / 太慢 只要前 k 个用 Lanczos / eigsh
    重复特征值 特征向量不唯一,数值上随便给一个 不要假设特征向量稳定;靠特征值排序
    PageRank 初值选错 收敛慢 用均匀分布 + 几十次迭代足够
    协方差矩阵特征值 ~0 数值上是负小数 → sqrt(λ) 出 NaN 加 epsilon 或 clip 到正数

    11.3 收敛速率的实际意义

    幂迭代收敛速度 ∝ (λ2/λ1)^k——前两个特征值越接近,收敛越慢:

    λ1 = 1.0, λ2 = 0.9 时:每步衰减 10%,需要 ~70 步降到 0.001
λ1 = 1.0, λ2 = 0.5 时:每步衰减 50%,需要 ~10 步降到 0.001
λ1 = 1.0, λ2 = 0.99 时:每步衰减 1%,需要 ~700 步降到 0.001

    真实工程里这意味着:互联网这种"明显有少数巨头节点"的图,PageRank 收敛快;均匀的"圈状图",收敛慢。这就是 spectral gap 的工程含义。

    11.4 局限:特征分解的硬伤

    问题 影响 替代
    不是所有矩阵可对角化 缺特征向量 Jordan 标准型 / Schur 分解 / 直接用 SVD
    一般矩阵特征值是复数 工程不友好 SVD(总是实数奇异值)
    数值稳定差(condition number 大时) 误差大 SVD 通常更稳
    不能处理非方阵 长矩阵直接死 SVD 适用任意矩阵

    核心直觉:任何"一般矩阵的特征分解能做的事",SVD 大概率能做得更好——这就是下一篇 SVD 篇的开篇语。特征分解是"漂亮但脆弱",SVD 是"实用主义"。
