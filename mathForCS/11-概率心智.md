# 概率心智

概率统计是工程师最容易「学过又不会用」的一层。课本里你算过 Monty Hall、扔过硬币、画过文氏图;但等到真实场景里——「这个 A/B 测试到底显著吗?」「报警系统的假阳率怎么调?」「为什么医院的化验报告说阳性、医生还要让我复查?」——大部分人会下意识用「频率」「直觉」蒙一个答案,**而这个答案经常是错的**。

> 一句话先记住:**「条件」是概率论里最重要的一个字**。脱离了「在什么条件下」,任何概率数都没意义。「99% 准确率」「假阳率 5%」「P 值 0.04」——这些数本身不是结论,**关键是「在什么前提下」**。下面 80% 的反直觉案例,根本原因都是「读者偷偷换了条件」。

---

## 一、为什么必须把概率心智先建好

来三个真实场景,先感受一下不懂的代价:

**场景 1**:你做了个分类器,准确率 99%。某天 PM 拿这个去筛查一个发生率 1% 的事件(广告点击、欺诈、罕见疾病……),问你「**预测为正的样本里,真的有多少是正的**」。直觉:99%。实际:**约 50%**。差了一倍。

**场景 2**:你给运营做 A/B 测试,组 A 转化率 5.0%、组 B 转化率 5.2%,P 值 0.04。运营兴奋地全量上线。三周后,转化率没动。**P 值 0.04 不等于「B 比 A 好的概率 96%」**。

**场景 3**:你看公司报告,「新功能让转化率提升 5%」,数据是真的。你按子群看:**手机用户转化率降了、PC 用户转化率降了、平板用户转化率也降了——全量却涨了 5%**。这不是 bug,这是辛普森悖论。

| 场景 | 凭直觉的结论 | 实际正确的结论 |
| --- | --- | --- |
| 99% 准确率筛查 1% 患病率 | 阳性大概率真患病 | 阳性中真患病只占约 50% |
| P=0.04 显著 | B 比 A 好的概率 96% | 「假设 A=B 的世界里出现这种差异的概率 4%」,完全不是一回事 |
| 全量涨,分组都跌 | 数据造假 | 辛普森悖论,数据真实 |

> **经验法则**:**看到一个概率 / 准确率 / P 值,先问「这是在什么条件下算的」**。换条件,数字就变;不同条件下的同一个数,意义完全不同。

## 二、三种概率观:频率派 / 贝叶斯派 / 主观置信

「概率」这个词本身有歧义——**不同流派对「P(A)=0.7 是什么意思」的解释完全不同**。

### 2.1 三种解释

```
频率派(Frequentist)
  P(A) = 长期重复实验中 A 发生的频率极限
  例:扔硬币 P(正面)=0.5,意思是扔无穷多次,正面比例趋于 0.5

贝叶斯派(Bayesian)
  P(A) = 我对 A 发生的置信度,可以随证据更新
  例:P(明天下雨)=0.7,意思是「基于当前所知,我认为下雨可能性 0.7」

主观置信(Subjective/Betting)
  P(A) = 我愿意以多少赔率为 A 押注
  例:P(A)=0.7 ⇔ 我愿意 7:3 押 A 发生(同等期望)
```

后两个其实是「同一个意思的两种表达」——主观置信是贝叶斯的「操作性」定义(用打赌行为校准)。

### 2.2 工程里何时该用哪个

| 场景 | 推荐流派 | 原因 |
| --- | --- | --- |
| 估算系统 QPS 的 95 分位 | 频率派 | 有大量历史数据,直接看频率 |
| 估算「明天某个用户会买这个商品的概率」 | 贝叶斯 | 单次事件、没法重复实验,只能基于证据更新置信 |
| A/B 测试 | 都行,但**结论解读差异极大** | 频率派给出 P 值、置信区间;贝叶斯给出后验分布 |
| 给推荐系统的物品打分 | 贝叶斯 | 新物品冷启动,直接频率会被噪音淹没,需要先验 |
| 监控里的「P99 延迟」 | 频率派 | 大量请求,直接看分位数 |
| 罕见事件的预测(地震、黑天鹅) | 贝叶斯 | 数据稀缺,只能靠先验加证据 |
| 异常检测 / 反作弊 | 贝叶斯 | 把「正常 vs 异常」当先验,根据新事件更新 |

> **核心直觉**:**有大量重复样本时频率派够用;面对单次事件、稀有事件、需要把领域知识塞进模型时,贝叶斯派更合适**。机器学习领域大部分时候都是贝叶斯思维——你做的所有「先训一个 prior、用新数据 finetune」、「dropout 近似贝叶斯模型平均」其实都在用贝叶斯。

### 2.3 一个工程例子

**问题**:一个新上架商品,头三天有 2 个点击、1 个转化。转化率是多少?

```
频率派答案:1/2 = 50%
贝叶斯答案:加一个 Beta(α=2, β=10) 的先验(整体转化率约 1/6),
            后验是 Beta(2+1, 10+1) = Beta(3, 11),期望 = 3/14 ≈ 21%
```

频率派被噪音放大了:**只有 2 个点击就说 50% 转化率,显然不可信**;贝叶斯先验告诉你「整体平均是 17%,你这个 50% 太离谱,得往整体均值拉一拉」。

**所以**:大部分推荐系统、广告系统、A/B 测试平台里的「平滑」「贝叶斯先验」其实都是这个意思——**小样本要往先验靠,大样本才信新数据**。

## 三、条件概率:概率论最值钱的一个工具

### 3.1 定义

```
P(A | B) = 在已知 B 发生的条件下,A 发生的概率

       = P(A ∩ B) / P(B)
             ↑
            A 和 B 同时发生的概率,除以 B 发生的概率
```

**几何直觉**:文氏图里,把 B 当作一个新「全集」,看 A 在这个新全集里占多大比例。

```
全集 Ω
┌───────────────────────────────┐
│                               │
│      ┌─────────┐              │
│      │   B     │              │
│      │  ┌──────┼──┐           │
│      │  │A∩B   │A │           │
│      │  └──────┼──┘           │
│      └─────────┘              │
│                               │
└───────────────────────────────┘

P(A | B) = (A∩B 的面积) / (B 的面积)
         = 把 B 当新全集后,A 占的比例
```

### 3.2 联合 vs 边缘 vs 条件

三个概念新人极容易混。

| 类型 | 写法 | 含义 |
| --- | --- | --- |
| 联合(joint) | `P(A, B)` 或 `P(A ∩ B)` | A 和 B 都发生的概率 |
| 边缘(marginal) | `P(A)` | A 发生的概率,不管 B |
| 条件(conditional) | `P(A | B)` | **已知 B 发生**,A 发生的概率 |

**关键等式**(乘法法则):

```
P(A, B) = P(A | B) · P(B)
        = P(B | A) · P(A)        ← 对称形式

P(A) = Σ_B P(A, B) = Σ_B P(A | B) · P(B)    ← 全概率公式
```

最后一行叫「全概率公式」,意思是:**把所有可能的 B 加起来,就能从条件回到边缘**。这是贝叶斯定理的分母来源。

### 3.3 条件概率的反直觉

**直觉陷阱**:`P(A | B)` 和 `P(B | A)` 是不同的。

```
P(下雨 | 多云)        ≠   P(多云 | 下雨)
P(国籍中国 | 姓张)    ≠   P(姓张 | 国籍中国)
P(癌症 | 化验阳性)    ≠   P(化验阳性 | 癌症)
```

**最后一个例子最致命**——化验报告告诉你的是「在你患癌的条件下、化验阳性的概率」(灵敏度,通常 95%+),你真正想知道的是「**化验阳性的条件下、你患癌的概率**」。这两个数差异极大,贝叶斯定理就是用来翻译这两个方向的。

## 四、贝叶斯定理:从已知反推未知

### 4.1 公式与术语

```
                P(B | A) · P(A)
P(A | B) = ─────────────────────────
                    P(B)

         = P(B | A) · P(A) / [ P(B|A)·P(A) + P(B|¬A)·P(¬A) ]
```

每个部分都有名字,**记住名字比记住公式重要**:

| 部分 | 名字 | 含义 |
| --- | --- | --- |
| `P(A)` | **先验**(prior) | 没看到 B 之前,你对 A 的置信度 |
| `P(B | A)` | **似然**(likelihood) | 假设 A 真的发生,看到 B 的概率 |
| `P(A | B)` | **后验**(posterior) | 看到 B 之后,你对 A 的更新置信度 |
| `P(B)` | **证据**(evidence) | B 整体发生的概率,起归一化作用 |

**贝叶斯的核心**:

```
后验 ∝ 似然 × 先验
```

> **核心直觉**:**贝叶斯定理就是「数据更新信念」的数学化**。先验是你的初始猜测、似然是「假如猜对了,观测到当前数据的可能性」、后验是把两者结合后的新猜测。所有 RAG、reranking、推荐、风控、垃圾过滤的核心运转方式都是这套。

### 4.2 基础率谬误:99% 准确率的真相

回到开头的例子。某医学化验:
- 灵敏度(真患病 → 阳性): 99%
- 特异度(没患病 → 阴性): 99%
- 人群患病率: 1%

**问题**:你化验阳性,真正患病的概率是多少?

直觉:99%。**正确答案:50%**。让贝叶斯算清楚:

```
A = 真患病
B = 化验阳性

P(A)       = 0.01           ← 先验,人群患病率
P(¬A)      = 0.99
P(B | A)   = 0.99           ← 灵敏度
P(B | ¬A)  = 0.01           ← 1 - 特异度,假阳率

P(B) = P(B|A)·P(A) + P(B|¬A)·P(¬A)
     = 0.99 · 0.01 + 0.01 · 0.99
     = 0.0099 + 0.0099
     = 0.0198

P(A | B) = 0.99 · 0.01 / 0.0198 = 0.5
```

**直觉错在哪**:99% 准确率说的是「**条件**」概率——**已知患病**时阳性的概率;读者偷偷把它当成「**阳性时患病**的概率」,**条件搞反了**。

NumPy 验证:

```python
import numpy as np

N = 1_000_000
sick     = np.random.rand(N) < 0.01   # 1% 患病
positive = np.where(
    sick,
    np.random.rand(N) < 0.99,         # 患病者 99% 阳性
    np.random.rand(N) < 0.01,         # 健康者 1% 阳性(假阳)
)

# 阳性中真患病的比例
print((sick & positive).sum() / positive.sum())
# ≈ 0.5
```

> **避坑**:**任何「分类器准确率 99%」类陈述,先问基础率(base rate)**。基础率越低,「阳性的真阳率」越接近 0。**这就是为什么医学筛查、欺诈检测、垃圾过滤都不能只看准确率,必须看 precision/recall/PR 曲线**。

### 4.3 工程对应:贝叶斯垃圾邮件过滤

经典的 Paul Graham 垃圾邮件过滤就是贝叶斯定理:

```
P(垃圾 | 邮件含"viagra"等关键词) 
  = P(含关键词 | 垃圾) · P(垃圾) / P(含关键词)
```

每来一封邮件,用历史样本统计 `P(每个词 | 垃圾)` 和 `P(每个词 | 非垃圾)`,**假设词之间独立**(naive Bayes,naive 就 naive 在这里),把所有词乘起来:

```
P(垃圾 | 整封邮件) ∝ P(垃圾) · ∏ P(词 i | 垃圾)
```

简单粗暴,但**在过滤任务上长期碾压复杂模型**——因为先验非常稳定、特征独立性「假错也错得对」、训练样本充足。

## 五、独立、不相关、互斥:三个常被混的概念

| 概念 | 数学定义 | 直觉 |
| --- | --- | --- |
| **独立** | `P(A, B) = P(A) · P(B)` 即 `P(A|B) = P(A)` | B 发生与否,对 A 的概率毫无影响 |
| **不相关** | `Cov(X, Y) = 0` 即 `E[XY] = E[X]E[Y]` | X 和 Y 的「线性关系」为零 |
| **互斥** | `A ∩ B = ∅` 即 `P(A, B) = 0` | A 和 B 不可能同时发生 |

**容易混的地方**:

```
独立 ⇒ 不相关 (反过来不成立)
独立 ⇒ 不互斥 (互斥意味着「B 发生了 A 一定不发生」,非常强的依赖)
互斥 ⇒ 不独立 (除非至少一个的概率是 0)
```

### 5.1 独立 ≠ 不相关:经典反例

```
X 服从 [-1, 1] 均匀分布
Y = X²

E[X] = 0
E[XY] = E[X³] = 0   (奇函数)
Cov(X, Y) = E[XY] - E[X]E[Y] = 0

所以 X 和 Y 不相关。但 Y 完全由 X 决定,**绝对不独立**。
```

**「相关性」只能捕捉线性关系**——这就是为什么 ML 里要用「互信息」(mutual information)来度量非线性依赖。

### 5.2 互斥 ⇏ 独立

```
抛一次硬币:
A = 正面
B = 反面

A 和 B 互斥(P(A,B)=0)
但 P(A|B) = 0 ≠ P(A) = 0.5
所以**不独立**——B 发生强烈影响了 A 的概率
```

> **经验法则**:**「独立」是最强的条件,「不相关」是中等强度,「互斥」其实是一种特殊的「极强依赖」**。论文里「assume features are independent」是非常硬的假设,naive Bayes 之所以「naive」就是因为它假设特征独立(往往不成立,但够用)。

## 六、Monty Hall 三门问题:条件概率最经典案例

### 6.1 题目

三扇门,一扇后面有车,两扇后面是羊。你选了 1 号门;**主持人(知道车在哪)从剩下两扇里打开一扇有羊的门**(比如 3 号);**问题:你要不要换到 2 号门?**

凭直觉:剩两扇,各 1/2 概率,换不换都行。

**正确答案:换!换的话中车概率 2/3,不换只有 1/3**。

### 6.2 用条件概率算清楚

```
事件定义:
  C_i:车在第 i 号门后面     (i=1,2,3)
  H_i:主持人打开第 i 号门
  你最初选的是 1 号门。

先验:P(C_1) = P(C_2) = P(C_3) = 1/3

观察到的条件:主持人打开了 3 号门(H_3)。

求:P(C_2 | H_3) vs P(C_1 | H_3)

主持人行为规则(关键):
  若 C_1(车在 1):主持人随机从 2、3 选一扇,所以 P(H_3 | C_1) = 1/2
  若 C_2(车在 2):主持人只能开 3(因为不能开 2,也不能开你选的 1),所以 P(H_3 | C_2) = 1
  若 C_3(车在 3):主持人不能开 3,所以 P(H_3 | C_3) = 0

贝叶斯:
  P(H_3) = P(H_3|C_1)·P(C_1) + P(H_3|C_2)·P(C_2) + P(H_3|C_3)·P(C_3)
         = 1/2 · 1/3 + 1 · 1/3 + 0
         = 1/2

  P(C_1 | H_3) = P(H_3|C_1)·P(C_1) / P(H_3) = (1/2 · 1/3) / (1/2) = 1/3
  P(C_2 | H_3) = P(H_3|C_2)·P(C_2) / P(H_3) = (1 · 1/3) / (1/2)   = 2/3
```

**所以换比不换好,概率从 1/3 涨到 2/3**。

### 6.3 直觉解释

> **核心直觉**:你最初选 1 号门时,有 1/3 概率选中、2/3 概率选错。**主持人开门是带信息的**——他知道车在哪,他必然开一扇有羊的门、不开车那扇。所以「主持人开门」的动作把 2/3 的「不在 1 号」概率**全部集中**到了「2 号」上(因为他帮你排除了 3 号)。

**关键**:**主持人的选择不是随机的——他知道答案**。如果换成「主持人随机开一扇,凑巧开到羊」,概率就真是 1/2 了。**条件不同,概率不同**。

### 6.4 Monte Carlo 验证

```python
import numpy as np

def play(switch):
    car = np.random.randint(3)
    pick = 0   # 不失一般性,总是先选 0 号
    # 主持人开一扇:不是车、不是你选的
    options = [d for d in range(3) if d != pick and d != car]
    revealed = np.random.choice(options)
    if switch:
        # 换到剩下那扇
        new_pick = [d for d in range(3) if d != pick and d != revealed][0]
        return new_pick == car
    return pick == car

trials = 100_000
print("不换胜率:", sum(play(False) for _ in range(trials)) / trials)  # ≈ 0.333
print("换胜率:",   sum(play(True)  for _ in range(trials)) / trials)  # ≈ 0.667
```

## 七、辛普森悖论:分组 vs 整体方向相反

### 7.1 经典案例

加州伯克利 1973 年研究生录取数据:
- 整体看:**男性录取率 44%,女性录取率 35%** → 看似存在性别歧视
- 按系分组看:**绝大多数系女性录取率 ≥ 男性** → 没有歧视

怎么回事?

```
                男申请数  男录取率   女申请数  女录取率
系 A (好进)      825      62%        108     82%       ← 女更高
系 B (好进)      560      63%         25     68%       ← 女更高
系 C (难进)      325      37%        593     34%       ← 男略高
系 D (难进)      417      33%        375     35%       ← 女更高
系 E (难进)      191      28%        393     24%       ← 男略高
系 F (极难)      373       6%        341      7%       ← 女更高
─────────────────────────────────────────
合计:           2691     45%       1835     30%       ← 男反而高
```

**真相**:**女性更倾向申请竞争激烈的系**(系 C-F 录取率低);男性更倾向申请好进的系(系 A-B)。**「申请的系」是个混杂变量(confounder)**,把它平均掉后,反而被它主导了整体趋势。

### 7.2 用条件概率写

```
P(录取 | 男) > P(录取 | 女)             ← 整体看到的
P(录取 | 男, 系=A) < P(录取 | 女, 系=A)  ← 但每个系内是反的
P(录取 | 男, 系=B) < P(录取 | 女, 系=B)
...

为什么:P(申请系=A | 男) ≠ P(申请系=A | 女)
       即「男 / 女申请什么系」分布不同,系是混杂变量
```

### 7.3 工程对应:数据分析里的陷阱

| 真实案例 | 整体看 | 分组看 |
| --- | --- | --- |
| 肾结石手术(经典) | 治疗 A 整体好 | 治疗 B 在小结石、大结石两组都更好 |
| A/B 测试上线 | 新版本整体涨 5% | 分平台看(iOS / Android / Web)都跌 |
| 推荐系统 CTR | 新模型整体 CTR 涨 | 按用户群看(老用户/新用户)都跌 |

> **避坑**:**A/B 测试上线前一定要看「分群指标」**——按平台 / 渠道 / 用户群 / 时段切片看。**只看整体涨,可能是流量构成变了导致的辛普森悖论**——这种「涨」是不稳定的,过几天流量构成又变了,涨幅消失甚至反转。

### 7.4 怎么发现混杂

```python
import pandas as pd

# 模拟 A/B 测试数据
df = pd.DataFrame({
    'group': ...,         # A or B
    'platform': ...,      # iOS / Android
    'converted': ...,     # 0 or 1
})

# 整体看
print(df.groupby('group')['converted'].mean())

# 分群看
print(df.groupby(['platform', 'group'])['converted'].mean().unstack())

# 关键:确认两组在「平台」分布上是否平衡
print(df.groupby('group')['platform'].value_counts(normalize=True))
```

**如果两组的混杂变量分布不平衡,整体数字就不可信**。这就是为什么 A/B 测试平台要做「SRM 检测」(Sample Ratio Mismatch)——流量分配比例对不上时直接报警。

## 八、选择偏差:数据「能被看到」就已经偏了

### 8.1 二战飞机弹孔的故事

二战时美军统计:返航的飞机上,**机翼弹孔多、机身次之、引擎几乎没有**。直觉建议:加强机翼装甲。

数学家 Abraham Wald 说:**反了——加强引擎**。理由:**你看到的是返航的飞机**——引擎中弹的根本回不来,所以「返航飞机里引擎没弹孔」恰恰证明引擎是致命部位。

### 8.2 这是个条件概率问题

```
你想知道:P(致命部位 = X)
你看到的:P(部位 = X 中弹 | 飞机返航)

这两个完全不同——你的样本被「能返航」这个条件筛过了
```

### 8.3 工程场景

| 场景 | 选择偏差表现 |
| --- | --- |
| **训练数据采集** | 只有「使用过功能」的用户行为被记录 → 流失用户不在数据里 |
| **推荐系统反馈** | 只看到「推荐了的物品的点击」 → 不知道「没推的物品会不会被点」 |
| **客户评价 / NPS** | 极满意 / 极不满意的人才填,中间的不填 → 评分双峰、平均值无意义 |
| **错误日志** | 只看到没把进程杀掉的错误 → OOM 杀进程后没日志的错误是隐形的 |
| **股票 / 基金的历史回测** | 「现存的基金」回测全是赢家 → 幸存者偏差,实际收益被高估 |

> **核心直觉**:**任何「能被你看到的样本」都已经被某个条件筛过**——这个筛子就是选择偏差的来源。**你以为在估 P(X),实际上在估 P(X | 通过了筛子)**。

### 8.4 缓解

- **采样要尽量覆盖「看不到的部分」**:埋点埋全(包括失败/退出),A/B 测试随机分流,推荐系统要 explore(给非推荐的物品分一点流量)
- **用 inverse propensity weighting**:把「被采样到的概率」算出来,给样本反向加权,纠正偏差
- **意识到边界**:**报告里写清楚「这是哪部分用户的数据」**,不要默默扩大结论范围

## 九、工程对应

### 9.1 A/B 测试里的条件概率

```
H0:A = B (新版本没效果)
P 值:在 H0 为真的世界里,看到「至少这么大差异」的概率

注意:P 值 ≠ 「B 比 A 好的概率」
正确解读:「假设 B 真的没好,出现你看到这种差异的概率是多少」
```

工程实现核对清单:

| 检查项 | 数学含义 |
| --- | --- |
| SRM 检测(实际流量 vs 期望流量) | 验证条件「随机分流」没坏 |
| 分群指标(细分维度看是否同方向涨) | 防辛普森悖论 |
| 检验功效计算(MDE 最小可检测效应) | P(发现真效应 | 真有效应),也是条件概率 |
| 多重比较矫正(Bonferroni) | 多次检验时整体 false positive 率 |

### 9.2 风控里的假阳率

风控系统通常要求**很低的假阳率**(false positive rate),因为正样本(欺诈)极稀疏。从基础率谬误的视角:

```
欺诈率:0.1%
假阳率:1%
真阳率:99%

P(欺诈 | 报警) = 0.99 · 0.001 / [0.99·0.001 + 0.01·0.999]
              = 0.00099 / 0.01098
              ≈ 9%
```

**91% 的报警是误报**——这就是为什么风控系统的 precision 永远是大头,不是 recall。

### 9.3 推荐系统里的贝叶斯平滑

新物品冷启动,只有几次曝光怎么估 CTR?

```
原始估计:点击数 / 曝光数              ← 小样本被噪音淹没
贝叶斯平滑:(点击数 + α) / (曝光数 + α + β)

α, β 是先验参数,可以从全局 CTR 设
  例:全局 CTR = 5%,选 α=5, β=95(表示「先验等价于 100 次曝光、5 次点击」)
```

**几乎所有推荐 / 广告 / 排序系统都有这层「贝叶斯平滑」**——名字可能叫「平滑」「shrinkage」「Bayesian smoothing」「Empirical Bayes」,数学本质都是一样的。

### 9.4 Naive Bayes 在垃圾过滤里的细节

```python
# 训练:统计每个词在 ham / spam 类下的频率
P_word_spam = {w: count_in_spam[w] / total_spam_words for w in vocab}
P_word_ham  = {w: count_in_ham[w]  / total_ham_words  for w in vocab}

# 预测:对一封新邮件
def classify(email_words):
    log_p_spam = math.log(P_spam_prior)
    log_p_ham  = math.log(1 - P_spam_prior)
    for w in email_words:
        log_p_spam += math.log(P_word_spam.get(w, 1e-6))   # 平滑避免 log(0)
        log_p_ham  += math.log(P_word_ham.get(w,  1e-6))
    return 'spam' if log_p_spam > log_p_ham else 'ham'
```

**为什么用 log**:乘很多 < 1 的概率会下溢成 0。**取 log 后变加法,数值稳定**。这个 trick 在所有概率连乘的场景都用——LLM 的 log probabilities、HMM 的 Viterbi、CRF 的 score 等等。

> **经验法则**:**只要看到「概率连乘」,立刻想到换成「log 概率求和」**——既数值稳定,又方便加正则项 / 先验。
