# 导数、偏导与梯度

这一篇是第五层「微积分与凸优化」的第一站。**整套深度学习只用到微积分里两个东西:导数和链式法则**。导数告诉你"参数动一点,loss 动多少";链式法则告诉你"loss 对深处某个参数的导数怎么算"。**这一篇讲第一件事**,21 篇讲第二件。

> 一句话先记住:**梯度 ∇f 就是一个向量,它指向 f 上升最快的方向、长度等于那个方向上的瞬时变化率**。深度学习训练就是不断算 `∇loss(θ)`、然后让 `θ ← θ - lr × ∇loss(θ)`——**全部数学就这一行,后面所有花哨的优化器(Momentum / Adam / AdamW)都是这一行的精修版**。

---

## 一、为什么这篇必须存在

> "Adam 收敛慢的时候是该调学习率,还是该换 Newton?Hessian 那么贵为什么没人用?方向导数和梯度是不是同一个东西?"

aiLearning 06 篇讲过反向传播是"loss 对参数求梯度",但**没说梯度是什么**——很多人停留在"它是一个数组"的层面。**这一篇就是把那个数组背后的几何讲清楚**:它为什么指向上升最快的方向、为什么和等高线垂直、为什么 Hessian 描述曲率、为什么 ML 只用一阶方法。

| 撞过的坑 | 不懂时 | 懂了之后 |
| --- | --- | --- |
| GD 走 zigzag | 只会调小学习率 | 知道是病态 Hessian → 用 Adam / 加预条件 |
| 训练卡平台 | 怀疑是数据 / 调参 | 想到鞍点 / Hessian 不定 |
| Newton 比 Adam 强吗 | 不知道为什么没人用 | 知道 Hessian 是 N×N(N=参数数) |
| 二阶方法的近似 | 没听说过 | 认得 L-BFGS / K-FAC / Shampoo 是干啥 |

## 二、单变量导数:斜率的工程译法

### 2.1 定义直觉

```
f'(x) = lim    [f(x + h) - f(x)] / h
        h → 0
```

**别管这个极限,记住它的两个译法**:

1. **几何**:f 在 x 这点切线的斜率
2. **工程**:**x 微动 dx,f 跟着动 f'(x) · dx**(这是更有用的那个)

```
y = x²        在 x = 3 处
f'(3) = 2·3 = 6

x: 3.00 → 3.01     变化 +0.01
y: 9.00 → 9.0601   变化 +0.0601 ≈ 6 × 0.01 ✓
```

### 2.2 几何直觉(三种情况)

```
                 ↑ y                            
                 |                              
   f'(x) > 0     |        /  ← 上升中,函数在涨
                 |       /                      
                 |      /                       
   ──────────────┼─────●──────→ x  ← 极值候选
                 |    / \                       
                 |   /   \                      
   f'(x) < 0     |  /     \  ← 下降中,函数在跌
                 | /       \                    
                 |/         \                   
                                                
   f'(x) = 0 这点是: 极小 / 极大 / 鞍点 / 平台?
        取决于二阶导(下面 Hessian 那节会讲)
```

> **核心直觉**:**导数 = 局部线性近似的斜率**。"局部"两字很关键——离 x 很远时,这个线性近似就完全失效了。这就是 GD 必须走小步(学习率不能太大)的根本原因。

### 2.3 工程译法清单

| 场景 | 数学说法 | 工程说法 |
| --- | --- | --- |
| `f'(x) = 0` | 临界点 | 这是个候选最小值 / 鞍点,得验证 |
| `f'(x) > 0` | 上升 | 沿 +x 方向 loss 在涨,该往 -x 走 |
| `f'(x) < 0` | 下降 | 沿 +x 方向 loss 在降,该往 +x 走 |
| `|f'(x)|` 很大 | 高梯度 | 这个参数对 loss 影响大,小心更新 |
| `|f'(x)|` 接近 0 | 平稳 | 这个参数没在学,或者已经收敛 |

## 三、偏导:多变量时只看一个

### 3.1 定义

输入是 `(x, y, z, ...)` 多个变量,每次**只让一个变,其他全部固定**——求出来的就是偏导。

```
f(x, y) = x² + 3xy + 2y²

∂f/∂x = 2x + 3y           # 把 y 当常数
∂f/∂y = 3x + 4y           # 把 x 当常数
```

### 3.2 偏导 ≠ 全导

**这是最常见的混淆点**。

```
单变量:           f(x)
                  d f / d x  = 全部变化率(只有它一个)

多变量:           f(x, y)
                  ∂f / ∂x    = "只让 x 变" 的变化率(没考虑 y 的连带)
                  d f / d x  = "x 变,y 也跟着 x 变" 的总变化率
```

**做 ML 时几乎只用偏导**——因为参数之间是独立维度,我们只关心"动这一个参数,loss 怎么变"。

> **避坑**:**写论文时 d 和 ∂ 别混**——`∂L/∂W` 是"L 对 W 的偏导(假设其他参数固定)",写 `dL/dW` 在多变量场景下意思完全不同。**别人代码注释里这两符号也是死规矩,不要乱用**。

### 3.3 工程意义

```python
# 神经网络里,"loss 对某层权重 W 的偏导"
# = 假设其他所有参数都不变,只动 W,loss 会怎么变
# = 这就是 W.grad

import torch
W = torch.randn(3, 4, requires_grad=True)
b = torch.randn(4, requires_grad=True)
x = torch.randn(2, 3)
loss = ((x @ W + b) ** 2).sum()
loss.backward()

print(W.grad.shape)   # (3, 4) - 和 W 同 shape
print(b.grad.shape)   # (4,)   - 和 b 同 shape
```

**`W.grad[i,j]` 就是 ∂loss/∂W[i,j]**——当其他所有参数(包括 W 的其他元素和 b)都不动时,W[i,j] 微动 dx,loss 增加 `W.grad[i,j] · dx`。

## 四、方向导数与梯度:核心一步

### 4.1 方向导数

**沿任意方向 v**(单位向量)的瞬时变化率,叫方向导数。

```
∂f/∂v = lim    [f(x + h·v) - f(x)] / h
         h → 0
```

**它和偏导的关系**:**偏导是方向导数的特例**——`∂f/∂x` 就是沿坐标轴 `(1, 0, ..., 0)` 方向的方向导数。

### 4.2 梯度的定义

把所有偏导拼成一个向量:

```
∇f(x, y, z) = [∂f/∂x, ∂f/∂y, ∂f/∂z]
```

### 4.3 关键定理(不证,只用)

**方向导数 = 梯度和方向的内积**:

```
∂f/∂v = ∇f · v       (v 是单位向量)
       = |∇f| × |v| × cos(θ)
       = |∇f| × cos(θ)
```

**这个公式直接给出三个工程结论**:

```
cos(θ) = +1   v 和 ∇f 同向  → 沿这个方向 f 涨最快
cos(θ) = -1   v 和 ∇f 反向  → 沿这个方向 f 跌最快  ← GD 用这个!
cos(θ) =  0   v 和 ∇f 垂直  → 沿这个方向 f 不变  ← 等高线方向
```

> **核心直觉(梯度的两条性质)**:
> 1. **梯度方向 = 函数上升最快的方向**(所以 GD 沿 -∇f 走)
> 2. **梯度垂直于等高线**(所以 GD 永远跨过等高线、不会沿着等高线滑)

### 4.4 等高线 + 梯度的 ASCII 图

二维 `f(x, y)` 的等高线 + 梯度向量(`∇f` 在每个点都垂直于过该点的等高线、指向"高处"):

```
           y
           ↑
           │     等高线 f = 4
           │   ╭───────────╮          
           │  ╱   f = 3     ╲         
           │ ╱  ╭───────╮    ╲        
           │╱  ╱  f = 2  ╲    ╲       
           ─┤  ╱  ╭───╮   ╲    ╲      ↑ ∇f 在这一点
           │  │  │ f=1 │   │    │      指向这个方向
           │  │  │  ●  │   │    │     (上升最快)
           │  │  │ min │   │    │      
           │  │  ╰───╯   │  │    │      ← -∇f 是 GD 走的方向
           │   ╲  f=2   ╱   │    │     
           │    ╲──────╱    │    │     
           │      f=3       │    │     
           │       ╰────────╯    │     
           │           f=4       │     
           └─────────────────────────→ x

  规则:
   - 在任意一点,∇f 指向最近的"高处等高线"
   - ∇f 永远垂直于过该点的等高线
   - GD 走 -∇f 方向,所以会"跨过等高线",不会"沿着等高线滑"
   - 等高线越密集,|∇f| 越大,梯度越陡
```

**为什么 GD 会走 zigzag(在窄峡谷里来回弹)**?**因为窄峡谷的等高线是细长椭圆,梯度方向垂直于它**——就指向"短轴方向",**而不是指向最低点(长轴方向)**。所以每一步都横跨过最低点,下一步又横跨回来,曲折前进。

```
   病态 (条件数大) 的等高线 + GD 轨迹

       ╭────────────────────────────╮
      ╱                              ╲
     │   ↘                            │
     │     ↗                          │
     │       ↘                        │
     │         ↗     ←  GD 一路 zigzag  │
     │           ↘                    │
     │             ↗                  │
     │      ●  min                    │
     │                                │
      ╲                              ╱
       ╰────────────────────────────╯

   长轴方向梯度小、短轴方向梯度大
   → GD 沿短轴反复横跳,长轴方向收敛极慢
   → 这就是为什么需要 Adam(它给每个维度一个自适应步长)
```

## 五、雅可比 J:多输入多输出的"梯度推广"

当 `f: R^n → R^m` 是多输入多输出时,梯度这个概念升级成**雅可比矩阵**。

### 5.1 定义

```
f(x) = (f₁(x), f₂(x), ..., fₘ(x))      其中 x ∈ R^n

J = [ ∂f₁/∂x₁  ∂f₁/∂x₂  ...  ∂f₁/∂xₙ ]      shape: (m, n)
    [ ∂f₂/∂x₁  ∂f₂/∂x₂  ...  ∂f₂/∂xₙ ]
    [   ...      ...      ...    ...   ]
    [ ∂fₘ/∂x₁  ∂fₘ/∂x₂  ...  ∂fₘ/∂xₙ ]
```

**形状口诀**:`(输出维 m, 输入维 n)`——记不住就推一遍 shape 验证。

### 5.2 几个常见情况

| f 的类型 | 输入 / 输出 | "梯度" 叫什么 | 形状 |
| --- | --- | --- | --- |
| 标量函数 | n → 1 | 梯度 ∇f | (n,) |
| 向量函数 | n → m | 雅可比 J | (m, n) |
| 矩阵函数 | (n×k) → 1 | 也叫梯度,记号 ∂f/∂W | (n, k) |
| 张量函数 | 一般 shape | 形状和输入一致 | 同 input |

> **PyTorch 约定**:**`.grad` 总是和参数 shape 一致**——loss 是标量,所以 `W.grad` 和 `W` 同 shape;真要算 Jacobian 用 `torch.autograd.functional.jacobian`,但日常训练用不上。

### 5.3 雅可比和链式法则

**复合函数**:`y = f(g(x))`,其中 `g: R^n → R^k`、`f: R^k → R^m`。

```
J_total(x) = J_f(g(x)) · J_g(x)
            (m × k)     (k × n)
            = (m × n)
```

**链式法则在多变量下,就是雅可比矩阵相乘**——这就是反向传播的全部数学(下一篇展开)。

## 六、Hessian H:二阶导描述曲率

### 6.1 定义

**Hessian 是梯度的雅可比**——即所有二阶偏导组成的方阵。

```
对 f: R^n → R(标量函数):

H = [ ∂²f/∂x₁²    ∂²f/∂x₁∂x₂  ...  ∂²f/∂x₁∂xₙ ]
    [ ∂²f/∂x₂∂x₁ ∂²f/∂x₂²    ...  ∂²f/∂x₂∂xₙ ]    shape: (n, n)
    [   ...        ...        ...    ...     ]
    [ ∂²f/∂xₙ∂x₁ ∂²f/∂xₙ∂x₂  ...  ∂²f/∂xₙ²   ]
```

**对称的(在二阶可微的常规函数下)**:`H[i,j] = H[j,i]`。

### 6.2 几何意义:曲率

| 单变量 | 多变量 |
| --- | --- |
| `f''(x) > 0` 凹向上(碗) | H 正定 → 局部最小(碗) |
| `f''(x) < 0` 凹向下(山顶) | H 负定 → 局部最大(山) |
| `f''(x) = 0` 拐点 | H 不定(有正有负特征值) → **鞍点** |

```
        H 正定                H 负定                H 不定 (鞍点)
                                                                
        ╲      ╱             ╱──────╲                ╲           ╱
         ╲    ╱             ╱        ╲                ╲   ●   ╱  
          ╲  ╱             ╱  顶部    ╲                ╲ 鞍点 ╱   
           ╲╱              │           │                ╲   ╱    
           ●               │           │                 ╲ ╱     
        碗底(min)         ╲          ╱                 ╳        
                            ╲   ●    ╱                ╱   ╲     
                             ╲ max  ╱                ╱     ╲    
                              ╲    ╱                ╱       ╲   
```

### 6.3 鞍点为什么在高维很普遍

```
单维:  f''(x) = 0 时,要么是极值要么是拐点(2 种可能)
n 维:  H 有 n 个特征值,鞍点 = 既有正又有负特征值
       → 当 n 很大,"全部正"或"全部负"的概率指数级小
       → "有正有负" 的概率压倒性大
       → 高维的临界点 ≈ 几乎全是鞍点!
```

**Dauphin et al. 2014 的结论**:**深度网络优化的真正困难不是"陷入局部极小",而是"卡在鞍点"**——因为高维下鞍点远比极小点多。**这是 momentum / Adam 比纯 GD 好的根本原因之一**(它们能"冲过"鞍点)。

> **核心直觉**:**高维优化里,你从来不会"陷入坏的局部最小"——你只会"在鞍点上磨蹭"**。这改变了你对"训练 loss 不降"的诊断:不是模型容量不够,**多半是优化卡了某个鞍点 / 平台**。换优化器 / 加 noise / 调 lr schedule 比加参数有效。

## 七、一阶 vs 二阶方法

### 7.1 GD(梯度下降):只用一阶

```
θ_{t+1} = θ_t - lr × ∇L(θ_t)
```

**只用 ∇L**——对 H 一无所知。

**优点**:每步代价 O(N)(N 是参数量),便宜
**缺点**:在病态(condition number 大)的曲面上 zigzag,收敛慢

### 7.2 Newton 法:用二阶

```
θ_{t+1} = θ_t - H⁻¹(θ_t) × ∇L(θ_t)
```

**它在做什么**:**在当前点用二次函数(用 H 描述)拟合 loss 曲面,直接跳到拟合曲面的最低点**——所以收敛速度是"二次收敛"(每步精度翻倍),远快于 GD 的"线性收敛"。

**优点**:对病态曲面无感(自动用 H 矫正方向),收敛极快
**缺点**:**H 是 N×N 矩阵,N=参数数**

### 7.3 为什么 ML 几乎只用一阶

| 模型规模 | 参数 N | Hessian 大小 | Hessian 求逆代价 |
| --- | --- | --- | --- |
| LeNet | 60K | 60K × 60K = 3.6B | 60K^3 ~ 2×10^14 |
| ResNet-50 | 25M | 25M × 25M = 6×10^14 | 完全不可行 |
| GPT-2 | 1.5B | 1.5B × 1.5B = 2×10^18 | 完全不可行 |
| GPT-4 (1.8T 估) | 1.8T | (1.8T)² | 整个宇宙的算力 |

**结论一句话**:**你训练一个 7B 模型,Hessian 占的内存比模型参数本身还多 7B 倍**。**没法存,也没法求逆,只能放弃 Newton 法**。

### 7.4 折中:近似二阶方法

**ML 里实际能用的"二阶味道"方法**:

| 方法 | 原理 | 在哪用 | 备注 |
| --- | --- | --- | --- |
| **Adam** | 用梯度的二阶矩(对角近似)代替 H | LLM 训练标配 | 内存 = 2× 参数 |
| **L-BFGS** | 用历史 (∇L, θ) 隐式估 H⁻¹ | 凸优化、传统 ML | DL 里几乎不用 |
| **K-FAC** | 块对角 Fisher 近似 | 极少数研究 | 实现复杂、收益有限 |
| **Shampoo / SOAP** | 用 Kronecker 因子近似 H | 2024-2025 LLM 训练新热点 | Google / Meta 试用中 |

> **经验法则**:**99% 的 DL 训练用 AdamW(β₁=0.9, β₂=0.999, weight_decay=0.01-0.1),其他都是科研玩具或特殊场景**。Newton 法在 ML 里只活在凸优化教科书里——真要用,模型参数得在万级以下。

## 八、代码:把这些概念跑一遍

### 8.1 单变量导数 + autograd

```python
import torch

# y = x² + 2x + 1
x = torch.tensor(3.0, requires_grad=True)
y = x ** 2 + 2 * x + 1
y.backward()
print(f"f'(3) = {x.grad.item()}")  # 2x + 2 = 8 ✓
```

### 8.2 偏导 + 梯度 + 雅可比

```python
import torch

# 标量函数 f(x, y) = x² + 3xy + 2y²
x = torch.tensor([2.0, 3.0], requires_grad=True)
f = x[0] ** 2 + 3 * x[0] * x[1] + 2 * x[1] ** 2
f.backward()
print(f"∇f = {x.grad}")
# ∂f/∂x = 2·2 + 3·3 = 13
# ∂f/∂y = 3·2 + 4·3 = 18
# → tensor([13., 18.])  ✓

# 向量函数的雅可比 - f: R² → R²
def f_vec(x):
    return torch.stack([
        x[0] ** 2 + x[1],     # f₁ = x² + y
        x[0] * x[1] ** 2,     # f₂ = xy²
    ])

x = torch.tensor([1.0, 2.0])
J = torch.autograd.functional.jacobian(f_vec, x)
print(f"Jacobian shape: {J.shape}")   # (2, 2)
print(J)
# [[ ∂f₁/∂x  ∂f₁/∂y ]    [[ 2x   1  ]    [[ 2.   1. ]
#  [ ∂f₂/∂x  ∂f₂/∂y ]] =  [ y²   2xy]] =  [ 4.   4. ]]
```

### 8.3 用 GD 找最小值(可视化 zigzag)

```python
import torch

# 病态二次函数:f(x, y) = 0.5·x² + 5·y²(短轴 y 方向曲率大 10 倍)
def f(p):
    return 0.5 * p[0] ** 2 + 5 * p[1] ** 2

p = torch.tensor([10.0, 1.0], requires_grad=True)
lr = 0.18
trajectory = [p.detach().clone()]

for step in range(20):
    if p.grad is not None:
        p.grad.zero_()
    loss = f(p)
    loss.backward()
    with torch.no_grad():
        p -= lr * p.grad
    trajectory.append(p.detach().clone())

# 看 y 维度的振荡幅度
ys = [t[1].item() for t in trajectory]
print("y 轨迹:", [round(y, 3) for y in ys[:8]])
# y 轨迹: [1.0, -0.8, 0.64, -0.512, 0.41, -0.328, 0.262, -0.21]
#         ^ 在 y 方向 zigzag 振荡,长轴 x 方向收敛极慢
```

**这就是病态 Hessian + 一阶方法的标志性现象**——**短维度疯狂横跳,长维度龟速前进**。Adam 通过对每个维度用历史梯度的二阶矩做归一化,**自动给短维度小步长、长维度大步长**,这就是 Adam 在病态曲面上比 GD 强的本质。

### 8.4 numpy 数值梯度(没 autograd 时怎么算)

```python
import numpy as np

def numerical_gradient(f, x, h=1e-5):
    """中心差分:O(h²) 精度,比单边差分准"""
    grad = np.zeros_like(x)
    for i in range(len(x)):
        x_plus = x.copy(); x_plus[i] += h
        x_minus = x.copy(); x_minus[i] -= h
        grad[i] = (f(x_plus) - f(x_minus)) / (2 * h)
    return grad

f = lambda x: x[0] ** 2 + 3 * x[0] * x[1] + 2 * x[1] ** 2
print(numerical_gradient(f, np.array([2.0, 3.0])))
# [13.000... 18.000...]  ← 和 autograd 结果一致
```

> **避坑**:**别用数值梯度做训练**——O(N) 代价 × N 个参数 = O(N²),完全不可行。**它只用来 debug autograd**(`torch.autograd.gradcheck` 干的就是这事)。**真要算梯度,只用 autograd**。

## 九、工程对应与局限

### 9.1 API 速查

| 任务 | 工具 | 一行代码 |
| --- | --- | --- |
| 标量 loss 反向求梯度 | `torch.autograd` | `loss.backward()` → `param.grad` |
| 求向量函数的雅可比 | `torch.autograd.functional` | `jacobian(f, x)` |
| 求 Hessian | `torch.autograd.functional` | `hessian(f, x)`(只对小问题) |
| 数值梯度 debug | `torch.autograd` | `gradcheck(f, x)` |
| numpy 数值梯度 | `numpy` | `np.gradient(arr)`(只对均匀网格数据) |
| JAX 梯度 | `jax` | `jax.grad(f)(x)` |
| JAX 雅可比 | `jax` | `jax.jacfwd(f)(x)` / `jax.jacrev(f)(x)` |
| JAX Hessian | `jax` | `jax.hessian(f)(x)` |

**JAX 的 `jacfwd` vs `jacrev`**:`jacfwd`(forward-mode)适合**输入维 < 输出维**;`jacrev`(reverse-mode,就是 BP)适合**输入维 > 输出维**——这就是为什么 ML 用 reverse-mode(参数维 >> loss 维 = 1)。

### 9.2 调参时的"梯度直觉"清单

| 现象 | 数学解释 | 该调什么 |
| --- | --- | --- |
| loss NaN / Inf | 梯度爆炸 (\|∇L\| 失控) | 梯度裁剪 / 降 lr / 检查数值稳定 |
| loss 不动 | 梯度消失或卡鞍点 | 检查激活、加 LayerNorm、换优化器 |
| loss 振荡 | lr 大了 / Hessian 病态 | 降 lr / 用 Adam 自适应 / lr warmup |
| 训练慢但稳 | lr 偏小 | 试着抬 2× lr,看 loss 还能不能稳 |
| eval loss 反弹 | 优化方向偏 | 加 weight decay / 早停 / 改 lr schedule |
