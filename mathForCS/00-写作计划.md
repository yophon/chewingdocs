# 程序员的数学系列写作计划

> 来源:`未来系列规划.md` 第三类「纵深主题」第十三项(M. mathForCS),⭐⭐⭐⭐。
> aiLearning 第 02 篇只起了个头(向量 / 矩阵 / 概率 / 求导),写到 RAG 评测、量化、推理优化、分布式训练、RLHF 时全部撞墙——**算法实现可以照抄,但选型、调参、排错、读论文都绕不开数学**。securityLearning 的密码学一层、algorithmLearning 的复杂度分析、distributedLearning 的时钟与共识,底下也都是同一套数学。
> **看完这层,你看 LLM / 推荐 / 搜索 / 密码学 / 控制论的论文都能直接读到底**——不是「会证」,而是「看到一行公式知道它在工程里是哪个开关、调它会动哪儿」。

---

## 设计思路

六层结构,从「数与近似」到「离散与图」,**每个抽象都立刻落到一个工程场景**:

```
心智 + 基础(为什么工程师要懂数学 / 数的表示 / 误差与近似 / 公式到代码的翻译)
   → 线性代数(向量空间 / 矩阵变换 / 特征 / SVD / 张量与 einsum)
   → 概率与统计(分布 / 期望方差 / MLE/MAP / A/B 测试)
   → 信息论(熵 / 交叉熵与 KL / 互信息 / LLM 里的应用)
   → 微积分与凸优化(梯度 / 反向传播 / 凸性 / SGD/Adam / 拉格朗日对偶)
   → 离散数学(组合 / 图论 / 数论与密码学 / 马尔可夫 / 工具链)
```

**这个系列不是大学《高等数学》《线性代数》《概率论》合订本**——那些课本讲究"完备 + 证明",本系列只讲究**"工程师拿来即用 + 看懂论文 + 不被术语唬住"**。每章只回答四个问题:**这个东西想解决什么?它的几何/代数直觉是什么?在哪个工程系统里出现?调它会动什么?**

不证明——**所有定理直接给结论 + 一张图 + 一段最小代码**;不算题——**所有"算式"都对应到 NumPy/PyTorch 一行代码或一个真实超参**。读者画像是**写过 ML / 后端 / 安全代码,数学课本翻不下去,但读 LoRA / vLLM / RSA / Raft 论文时被公式劝退的工程师**。

## 写作模板

每篇遵守"为什么 → 几何直觉 → 代码 → 工程对应"的四段式:

```
1. 为什么需要这个东西
   - 工程里在哪儿撞到的(具体场景:训练、推荐、密码学、A/B…)
   - 不懂它会犯什么错(真实事故 / 论文里的反例)

2. 直觉(一张图就讲明白)
   - 几何图 / 状态机 / 数据流,任选其一
   - 关键概念用最少的符号讲清楚
   - 把课本里的 ε-δ 翻译成"差多少够用"

3. 代码(10-30 行 NumPy / PyTorch / SymPy)
   - 真能跑、真能改超参看动效
   - 标注「这一行换成 X 会发生什么」
   - 数值稳定的关键技巧(log-sum-exp / 减最大值 / 归一化)

4. 工程对应与局限
   - 这个数学落到哪个真实系统(PyTorch op / Faiss / RSA / PageRank)
   - 调它会动什么(超参映射到数学变量)
   - 何时不该用 / 何时该换更复杂的工具
```

不堆证明——**只贴关键不变量、几何图和最小代码**,让读者「看到这一行公式知道它在工程里是哪个旋钮」就够了。

每篇遇到几何对象、数据流、状态变化,**一定画图**——线性代数 / 概率分布 / 计算图 / 图论不画图永远讲不清楚。

## 节奏

```
01-04  心智:看完知道"为什么 FP16 训练会 NaN、什么时候该担心数值稳定"
05-10  线代:**最值钱的一层**,看完读 Attention / LoRA / Embedding 论文不卡公式
11-15  概率统计:看完会写 A/B 测试、不会被「p<0.05 显著」这种说法骗
16-19  信息论:看完知道交叉熵为什么是分类损失、RLHF 里的 KL 在罚什么
20-24  优化:**最贴日常调参**,Adam 的 β1 β2 不再是"抄默认值"
25-30  离散:**给 security / algorithm / RL 兜底**,RSA / PageRank / MDP 都能看到底
```

**一个硬指标**:看完 05-10 + 16-19 + 20-23 这 14 篇,你应该能在白板前讲清楚:

- "为什么 Embedding 用 cosine 而不是欧氏距离,什么时候反过来"
- "LoRA 的 rank=8 为什么够用,这个 rank 是 SVD 里的哪个 r"
- "交叉熵损失里的 log 如果直接算会出什么问题,log-sum-exp 在解决什么"
- "RLHF 里的 KL 罚项往哪个方向罚,β 调大调小分别会怎样"
- "Adam 的 β1 / β2 / ε 各自在数学上对应什么,为什么默认是 0.9 / 0.999 / 1e-8"

能讲清楚一题这个系列就值了。

再加上 25-30 这 6 篇,你应该能看懂:

- "RSA 的安全性为什么落在大整数分解,椭圆曲线为什么换了一个更难的问题"
- "PageRank 的迭代为什么会收敛,这个收敛在线代里是什么定理"
- "RL 的策略迭代 / 价值迭代,数学上就是马尔可夫链 + 不动点"

## 反对的写法

这个系列**不写**这些东西,见到了就是写错了:

1. **不写《高等数学》课本节选 / 不抄维基百科公式**——课本和 Wiki 都在,本系列只补它们没串起来的"为什么 + 工程对应"
2. 「先讲个数学史 / 牛顿莱布尼茨之争 / 希尔伯特问题」——直接进重点,故事只在脚注引用
3. **证明题、习题集**——本系列零习题、零证明,看不懂证明完全可以学
4. 「这个公式很优美」式的审美安利——读者关心"调它会动什么",不关心你的情怀
5. **不画图就讲向量空间 / 概率分布 / 计算图 / 图论 / 马尔可夫链**——这些**不画图永远讲不清楚**
6. 抄 PyTorch / NumPy 文档——文档在那儿,这里只补"为什么 API 长这样"
7. 把 ML 算法的实现细节再讲一遍——那是 aiLearning 的活,本系列只往下挖到数学
8. 把密码学协议讲完整——那是 securityLearning 的活,本系列只到「底下的数论」
9. 「先复习一下集合论」式的过度铺垫——只在用到时一句话带过

## 立场声明

本系列**只服务于"读懂论文 + 调对超参 + 排查数值/统计错误"**——不是教你重证一遍中心极限定理,而是教你看到一段公式时,知道它对应代码里的哪一行、调它会动哪个指标。

所有数学描述限制在直觉、几何图与最小代码片段,**不提供严格证明**;所有工程映射都指向公开论文、开源系统(PyTorch / NumPy / Faiss / OpenSSL / NetworkX)或公开课程,不引用任何未公开的内部资料。

**面向 2026:**

- 默认你已经会用 Python + NumPy / PyTorch,系列里不重复教
- 默认你已经在用 Claude / Cursor / Copilot 协助理解公式,系列里直接给「问 LLM 时该怎么提问」的最小关键词
- 默认你看的论文是 2020 年之后的(Transformer / LoRA / RLHF / Diffusion / Mamba),老论文只在脚注引用
