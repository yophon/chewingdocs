# 向量与向量空间

这一篇是线性代数层的入口。**绝大多数人学完大学《线代》,记得"向量是有方向的箭头",但记不住"为什么 RAG 要用 cosine、L1 和 L2 在正则化里到底有什么不同、什么时候该 normalize、什么时候不该"**。这些不是"应用题",是同一个数学结构(向量空间 + 度量)在不同工程系统里的不同投影。一旦把"向量、内积、范数"这三件事的几何图压进直觉,**Embedding 检索、词向量、对抗扰动、权重衰减**就全是同一回事。

> 一句话先记住:**向量是"一组数 + 一个度量"——光有数还不是向量,你得告诉我"什么叫近、什么叫远"**。换一种度量(L1 / L2 / L∞ / cosine),"近"的几何形状就变了——L2 是圆、L1 是菱形、L∞ 是方形。**Embedding 用 cosine 不是品味,是因为 normalize 后的向量只剩"方向",方向就是语义**。这一篇就讲清这个。

---

## 一、为什么这一篇必须存在

### 1.1 工程里这层数学撞到哪儿

| 场景 | 不懂向量空间 | 懂一点向量空间 |
| --- | --- | --- |
| RAG 召回率低 | 换更大模型、加更多文档 | 先检查 query 和 doc 是不是同一个 embedding 空间、有没有 normalize |
| LoRA 权重为什么衰减用 L2 不用 L1 | "默认就这样" | L2 让权重平滑、L1 让权重稀疏(很多直接为 0) |
| 对抗样本扰动量级 | "ε 是多少?" | ε 在 L∞ 球里是"每个像素动多少",在 L2 球里是"整张图动多少" |
| Faiss 选 `IndexFlatIP` 还是 `IndexFlatL2` | 抄示例 | 看 embedding 有没有 normalize,normalize 了用 IP |
| 词向量算 king - man + woman | 知道大概等于 queen,但不知道为什么 | 加减在向量空间里是合法操作,语义关系可以表示成方向 |
| 推荐系统用 L2 召回 vs 用 cosine | "都一样吧" | normalize 后 L2 和 cosine 等价,**否则相差很远** |
| Embedding 维度选 768 还是 1536 | 抄 OpenAI 文档 | 维度 = 向量空间的"自由度",维度低召回粗,维度高存储贵 |

### 1.2 这一篇的目标

读完你应该能回答:

1. 为什么 RAG 召回用 cosine,什么时候反过来该用欧氏距离
2. L1 正则给出稀疏权重、L2 给出平滑权重,几何上为什么
3. 看到一个 `(B, n, d)` 的张量,知道哪个维度是"向量"
4. Faiss 的 `IndexFlatIP` 和 `IndexFlatL2` 什么时候选哪个
5. 对抗样本论文里 `ε-ball in L∞` 是什么形状的扰动空间

## 二、向量的三种视角

**同一个东西换三种说法,论文 / 代码 / 黑板各用一种,看不懂就是因为没把三种视角对上**。

### 2.1 几何视角:一个箭头

```
        y
        ▲
      4 │       ↗ v = (3, 4)
      3 │     ↗
      2 │   ↗
      1 │ ↗
        ┼─────────────► x
        0  1  2  3
```

**关键属性**:**有方向、有长度,起点在原点**。这是黑板上画的样子,直观但只能画到二维或三维——**实际工程里的向量都是几百几千维,画不出来,只能脑补**。

### 2.2 代码视角:一组数

```python
import numpy as np

v = np.array([3, 4])          # 2 维
v = np.array([1, 0, -2, 5])   # 4 维
v = np.random.randn(768)      # 768 维(BERT embedding 长这样)

print(v.shape)                # (768,)
```

**关键操作**:**索引、加减、乘标量、切片**——和 Python list 长得像,但**逐元素并行运算**。

```python
a = np.array([1, 2, 3])
b = np.array([4, 5, 6])

a + b                         # [5 7 9]    向量加
3 * a                         # [3 6 9]    标量乘
a - b                         # [-3 -3 -3]
```

### 2.3 数学视角:一组基的系数

```
v = (3, 4)  其实是  3·e₁ + 4·e₂
                   └────┘   └────┘
              x 方向单位向量 y 方向单位向量
```

**这个视角的价值**:**向量本身不变,但你换一组基,坐标就变**。**PCA / SVD 干的事就是"找一组更好的基,让 embedding 维度更紧凑"**。

> **核心直觉**:**三种视角只是同一件事的三种表达——几何画图、代码运算、数学讨论时换着用**。卡住一个视角讲不通的概念,换另一个视角往往就通了。

## 三、向量空间:加法封闭 + 标量乘封闭

**向量空间不是"装了向量的容器"——是"向量之间的运算规则"**。

### 3.1 一个空间要满足什么

```
V 是一个向量空间,要求:

1. 加法封闭:u, v ∈ V → u + v ∈ V
2. 标量乘封闭:u ∈ V, c ∈ ℝ → c·u ∈ V
3. 有零元:存在 0 ∈ V,使 u + 0 = u
4. 有逆元:每个 u 有一个 -u,使 u + (-u) = 0
5. 加法结合 / 交换、标量乘分配……(技术细节)
```

**工程上你只用记前两条**:**两个向量加起来还是向量,向量乘个数还是向量**——这就是"向量空间"的实际操作含义。

### 3.2 为什么"封闭"是个有用的概念

```
反例:单位长度向量的集合不是向量空间
  v1 = (1, 0),  v2 = (0, 1)   都是单位长度
  v1 + v2 = (1, 1)              长度 √2,不再是单位长度

  → 单位向量集合对加法不封闭,所以**不是子空间**
  → 这就是为什么 embedding 做完加减后必须重新 normalize
```

**词向量算术 `king - man + woman ≈ queen`,做完减法和加法后,结果不再是单位长度**——所以 retrieval 系统拿这个结果去查时,要先 normalize 才能保证 cosine 比对的对称性。

### 3.3 线性组合、基与维度

```
线性组合:   c₁·v₁ + c₂·v₂ + ... + cₙ·vₙ
基:        一组"独立"的向量,任意向量都能写成它们的线性组合
维度:       基的元素个数 = 向量空间的"自由度"
```

**直觉**:**维度 = 你需要多少个独立方向才能描述这个空间的任意点**。

| 空间 | 维度 | 例子 |
| --- | --- | --- |
| 一条直线 | 1 | 时间轴 |
| 平面 | 2 | 屏幕坐标 |
| 三维空间 | 3 | 物理世界 |
| 词向量(BERT-base) | 768 | 每个词一个 768 维向量 |
| GPT-3 embedding | 12288 | 极高维 |
| 图像特征(ResNet 倒数第二层) | 2048 | — |

> **避坑**:**"维度越高表达越好"是错的**——维度高了,**采样难度指数上升**(curse of dimensionality),**最近邻区分度变差**(高维空间任意两点距离都差不多)。**所以 OpenAI 默认 1536 维 embedding 而不是 100 万维,不是它做不到,是再高就废了**。

## 四、内积 / 点积:量化"同向程度"

**这是这一篇最重要的一节**——所有 retrieval、attention、similarity 都从这里来。

### 4.1 两种定义

```
代数定义:  a · b = Σ a_i · b_i = a₁b₁ + a₂b₂ + ... + aₙbₙ
                     └─ 逐位相乘再加和

几何定义:  a · b = |a| · |b| · cos(θ)
                     └─ 长度 · 长度 · 夹角余弦
```

**两个定义恒等**——这是线代里最有用的一个事实。**代数定义易算(一行 numpy),几何定义易理解(就是"同向程度")**。

### 4.2 代码

```python
a = np.array([1, 2, 3])
b = np.array([4, 5, 6])

# 三种等价写法
np.dot(a, b)             # 32
a @ b                    # 32
(a * b).sum()            # 32

# PyTorch
torch.dot(a_tensor, b_tensor)

# 高维:在最后一维做点积
A = np.random.randn(100, 768)            # 100 个 query
B = np.random.randn(1000, 768)           # 1000 个 doc
similarities = A @ B.T                   # (100, 1000)  每对的点积
```

### 4.3 为什么 cos(θ) 衡量"同向程度"

```
两个向量越同向,点积越大:

   ─────►  ─────►       同向         cos(0°) = 1     最大
   ─────►   ────►       近似同向     cos(30°) ≈ 0.87
   ─────►  │            垂直          cos(90°) = 0    没关联
   ─────►  ◄─────       反向          cos(180°) = -1  最小
```

**所以 dot product 大 = 方向接近**——前提是两个向量的长度都"差不多"。**如果长度差很多,长向量自带优势**,这就是 cosine 出场的地方。

### 4.4 cosine similarity:把长度除掉

```
                    a · b
cos(a, b) = ─────────────────────
              |a| · |b|
```

**几何含义**:**只比方向,不比长度**。

```python
def cosine(a, b):
    return (a @ b) / (np.linalg.norm(a) * np.linalg.norm(b))

# 或者先 normalize 再 dot
def cosine_v2(a, b):
    a_n = a / np.linalg.norm(a)
    b_n = b / np.linalg.norm(b)
    return a_n @ b_n

# PyTorch 内置
F.cosine_similarity(a.unsqueeze(0), b.unsqueeze(0))  # 要求 batch 维
```

> **核心直觉**:**dot product 把"方向 + 长度"打包成一个数,cosine 把"长度"剥掉只剩方向**。**当 embedding 模型不保证输出向量长度一致时**(很多原始模型都不保证),**dot product 会给"长向量"虚高分数,只有 cosine 公平**。

## 五、范数:量化"长度 / 距离"

**范数是"广义长度"**——同一个向量,在不同范数下"长度"不同。**做 ML 必须看懂的有四种**:L0、L1、L2、L∞。

### 5.1 一表速查

```
向量 x = (x₁, x₂, ..., xₙ)

‖x‖₀  = 非零元素的个数(严格说不是范数,是"伪范数")
‖x‖₁  = Σ |xᵢ|              (曼哈顿距离 / 出租车距离)
‖x‖₂  = √(Σ xᵢ²)            (欧氏距离 / 直线距离)
‖x‖∞ = max |xᵢ|             (最大坐标绝对值)
```

```python
x = np.array([3., -4., 0., 2.])

(x != 0).sum()                            # 3        L0
np.linalg.norm(x, ord=1)                  # 9.0      L1 = 3+4+0+2
np.linalg.norm(x, ord=2)                  # 5.385    L2 = √(9+16+0+4)
np.linalg.norm(x, ord=np.inf)             # 4.0      L∞ = max(|x_i|)
```

### 5.2 单位球:四个范数的形状

**"单位球"= 范数等于 1 的所有点**。**画出来才能看清四个范数的区别**:

```
L1 单位球(菱形):                  L2 单位球(圆):
     y                                 y
     ▲                                 ▲
   1 │  ╱╲                           1 │ ╭───╮
     │ ╱  ╲                            │╱     ╲
     ┼─────┼──► x                      ┼───────► x
     │ ╲  ╱                           │╲     ╱
  -1 │  ╲╱                          -1 │ ╰───╯
                                  
   ‖x‖₁ = |x|+|y| = 1               ‖x‖₂ = √(x²+y²) = 1
   |x| + |y| ≤ 1 的内部              x² + y² ≤ 1 的内部


L∞ 单位球(方形):                  L0 没有"球"——离散的
     y
     ▲
   1 │┌─────┐
     ││     │
     ┼┼─────┼──► x
     ││     │
  -1 │└─────┘

   max(|x|, |y|) = 1
```

**几何意义**:

| 范数 | 单位球形状 | 直觉 |
| --- | --- | --- |
| L0 | 坐标轴的点 | "有几个维度不为 0" |
| L1 | 菱形 | "总变化量"(各维度变化之和) |
| L2 | 圆 / 球 | "直线距离" |
| L∞ | 方形 | "最坏单维变化" |

### 5.3 哪个范数用在哪:工程实战

#### L1 → 稀疏正则(Lasso)

```python
# L1 正则:鼓励权重很多为 0(特征选择)
loss = mse(y_pred, y_true) + lambda_ * torch.abs(weights).sum()
```

**几何直觉**:**L1 单位球的顶点在坐标轴上**——优化轨迹"撞"到顶点时,部分维度直接归零。**Lasso 选特征、压缩感知、稀疏字典学习全靠这个**。

#### L2 → 权重衰减 / 平滑正则(Ridge / weight decay)

```python
# L2 正则:鼓励权重小但不为 0
loss = mse(y_pred, y_true) + lambda_ * (weights ** 2).sum()

# AdamW 内置 weight_decay 就是 L2(数学上有一点点差异,工程上一样)
optimizer = torch.optim.AdamW(model.parameters(), lr=1e-3, weight_decay=0.01)
```

**几何直觉**:**L2 单位球是光滑的圆**——优化轨迹"贴"在圆上滑,没有特殊方向,所有维度同步缩小。**深度学习几乎全用 L2**。

#### L∞ → 对抗扰动 / 鲁棒性

```python
# 对抗样本:在 L∞ ε-球里找最坏扰动
delta = torch.zeros_like(x, requires_grad=True)
for _ in range(steps):
    loss = -loss_fn(model(x + delta), y)
    grad = torch.autograd.grad(loss, delta)[0]
    delta = delta - lr * grad.sign()
    delta = torch.clamp(delta, -epsilon, epsilon)   # L∞ 投影
```

**几何直觉**:**L∞ 单位球是方形**——"每个像素最多动 ε"。**这就是 PGD / FGSM 攻击里 `--eps 8/255` 的含义**——每个像素最多动 8/255。

#### L2 → 几何距离 / Faiss 默认

```python
# 欧氏距离召回
dist = np.linalg.norm(query - doc, axis=-1)         # L2
nearest = np.argsort(dist)[:k]                       # 最近 k 个
```

> **经验法则**:**做 ML 见到正则一律先想 L2(平滑 / 训练稳)、做特征选择换 L1(稀疏)、做安全研究用 L∞(逐像素扰动)**。这三个选择 90% 的工程场景都覆盖了。

## 六、词向量为什么是向量:从语义到几何

**为什么"king - man + woman ≈ queen" 这种神奇现象能成立?**——因为词向量被训练成"语义关系 ↔ 几何关系"的映射。

### 6.1 向量空间的"线性结构"承载了什么

```
king ─ man + woman ≈ queen

意味着:
  king - man      ≈ queen - woman
  └─────────┘      └─────────────┘
  "国王 - 男性"     "女王 - 女性"
  
  这两个差向量代表同一个概念:"王室身份"
```

**几何上**:**向量空间里,"加减"能合法操作**——所以"概念关系"如果训出来是个差向量,就能在向量空间里平移、复用。

```
language: king ────────► man
              ╲           ╲
            "王室"        "王室"
                ╲           ╲
              queen ──────► woman
```

### 6.2 语义相似度 → 几何接近度

| 任务 | 几何对应 |
| --- | --- |
| 找近义词 | 距离最近的 k 个点 |
| 分类 | 落在哪个簇 |
| 类比(A:B :: C:?) | C + (B - A) 附近的点 |
| 检测异常 | 远离所有簇的点 |

**"语义"被压缩成了"向量空间里的方向 + 位置"**——所以 retrieval / clustering / classification 全可以用同一套向量数学解决。

### 6.3 cos vs dot vs L2:在召回里怎么选

**最重要的实战决策**:**用哪个度量做 retrieval**。**只看一件事:embedding 有没有 normalize**。

```python
# 假设 embedding 模型输出 query 和 docs
query = encode(text)               # shape (768,)
docs = encode(corpus)              # shape (N, 768)

# 路线 1:不 normalize,用 dot product
scores = docs @ query              # (N,)  长向量自带优势

# 路线 2:不 normalize,用 cosine
scores = (docs @ query) / (np.linalg.norm(docs, axis=1) * np.linalg.norm(query))

# 路线 3:不 normalize,用负 L2 距离
scores = -np.linalg.norm(docs - query, axis=1)

# 路线 4(推荐):先 normalize,再用 dot
docs_n = docs / np.linalg.norm(docs, axis=1, keepdims=True)
query_n = query / np.linalg.norm(query)
scores = docs_n @ query_n          # 此时 = cosine,且最快
```

**对应关系**:

```
                 不 normalize           已 normalize
 ─────────────────────────────────────────────────────
 dot product     长向量有优势           = cosine,正确
 cosine          公平,但每次要除范数    = dot,但多算一次范数
 L2 距离         "近"= 既同向又长度近    = √(2 - 2·cos),和 cos 单调对应
```

> **核心直觉**:**所有 embedding 都先 normalize、然后用 dot product 召回——这是 2026 年的工程标配**。原因:**计算最快(就一个矩阵乘)、数学上等价于 cosine、Faiss 的 IP 索引可以直接复用**。

### 6.4 normalize 的代码

```python
# NumPy
emb = emb / np.linalg.norm(emb, axis=-1, keepdims=True)

# PyTorch
emb = F.normalize(emb, p=2, dim=-1)

# 一行式 — sentence-transformers 等库默认都做这个
from sentence_transformers import SentenceTransformer
model = SentenceTransformer('all-MiniLM-L6-v2')
emb = model.encode(texts, normalize_embeddings=True)   # ← 这个参数
```

> **避坑**:**OpenAI 的 `text-embedding-3-*` 默认输出已 normalize,但 BERT 原始 CLS 输出没有 normalize**——你不知道这个差异,RAG 召回率可以差 20%+。**第一件事永远是检查 `np.linalg.norm(emb, axis=-1)` 是不是约等于 1**。

## 七、normalize 后 cosine = dot:为什么这件事这么重要

```
如果 |a| = |b| = 1(都 normalize 过):

  a · b = |a| · |b| · cos(θ) = 1 · 1 · cos(θ) = cos(θ)

  → dot product 直接等于 cosine,不需要再除范数
```

**这个等价在工程上的价值**:

| 价值 | 解释 |
| --- | --- |
| 计算最快 | 一个矩阵乘就出结果,没有除法 |
| 索引最简 | Faiss / Milvus / Pinecone 直接用 inner product 索引 |
| GPU 友好 | matmul 在 Tensor Core 上吃满,除法慢 |
| 数值稳定 | 没有 `/0` 问题(零范数向量) |

**这就是为什么所有现代 RAG / 推荐 / 召回流水线的第一行都是 `normalize_embeddings=True`**。

### 7.1 normalize 后的 L2 距离

**有趣的事实**:**normalize 后,L2 距离和 cosine 是单调一一对应的**——选谁都一样,排序结果相同。

```
|a - b|² = |a|² + |b|² - 2·a·b
         = 1 + 1 - 2·cos(θ)            (因为 normalize 后 |a|=|b|=1)
         = 2(1 - cos(θ))

→ L2 距离越小 ⇔ cos 越大
```

**这就是为什么 Faiss 的 IndexFlatL2 和 IndexFlatIP 在 normalized 向量上等价**——选哪个看心情(常用 IP 因为不开方更快)。

## 八、工程对应:Faiss / Milvus / vLLM 的 retrieval

### 8.1 Faiss 索引选型

```python
import faiss
import numpy as np

dim = 768
docs = np.random.randn(1_000_000, dim).astype('float32')
faiss.normalize_L2(docs)             # 必须先 normalize

# 方案 1:暴力精确召回(N 小用)
index = faiss.IndexFlatIP(dim)       # 内积 = 已 normalize 时的 cosine
index.add(docs)
D, I = index.search(query, k=10)

# 方案 2:聚类近似(N 大,百万级以上)
nlist = 100                          # 簇个数
quantizer = faiss.IndexFlatIP(dim)
index = faiss.IndexIVFFlat(quantizer, dim, nlist, faiss.METRIC_INNER_PRODUCT)
index.train(docs)
index.add(docs)
index.nprobe = 10                    # 查时探查多少簇,精度 vs 速度
D, I = index.search(query, k=10)

# 方案 3:HNSW 图(平衡延迟和召回,百万到亿级)
index = faiss.IndexHNSWFlat(dim, M=32, faiss.METRIC_INNER_PRODUCT)
index.add(docs)
D, I = index.search(query, k=10)
```

| 索引类型 | 适用规模 | 延迟 | 召回率 | 复杂度 |
| --- | --- | --- | --- | --- |
| `IndexFlatIP` / `IndexFlatL2` | < 10 万 | 慢(O(N)) | 100% | 简单 |
| `IndexIVFFlat` | 10 万 - 1 千万 | 中(O(N/nlist)) | 95-99% | 要 train |
| `IndexHNSWFlat` | 1 千万 - 10 亿 | 快(O(log N)) | 95-99% | 内存大 |
| `IndexIVFPQ` | 极大,且显存敏感 | 快 | 90-95% | 有损,需调参 |

**HNSW / IVF 不是这一篇的主题**(详见 algorithmLearning / aiLearning 的向量数据库篇)——这里只点一下:**底层度量都是 inner product 或 L2,前提是 embedding normalize 过**。

### 8.2 PyTorch 里的向量操作速查

```python
# 计算 batch 之间的余弦相似度矩阵
a = F.normalize(a, dim=-1)                    # (B1, D)
b = F.normalize(b, dim=-1)                    # (B2, D)
sim = a @ b.T                                 # (B1, B2)

# pairwise L2 距离
dists = torch.cdist(a, b, p=2)                # (B1, B2)

# top-k 召回
sim, idx = (a @ b.T).topk(k=10, dim=-1)       # 每个 a 找最相似的 10 个 b
```

### 8.3 这一层的局限

**向量空间表达力的边界**:

- **加法和标量乘必须封闭**——所以词向量做完 king-man+woman 后是合法的,但 retrieval 时不能直接拿这个结果(范数变了)。
- **几何上的"近"不一定对应"语义上相关"**——cosine 高的 doc 可能是同主题但答非所问。**这就是为什么 RAG 后面要加 rerank**(rerank 模型是 cross-encoder,直接判语义相关性,不局限于线性几何)。
- **维度越高,所有距离趋近**——这是高维灾难。**1536 维以上的 embedding,top-1 和 top-100 的余弦相似度可能只差 0.02**,排序信号变弱。
- **embedding 空间不一定有"中心"**——你不能假设原点是"无意义"。一些训练目标会让向量都聚在某个角落,这时 cosine 区分度会下降。

> **避坑**:**Embedding 模型换了一个,所有索引都要重新建——因为不同模型的向量空间不同,old query embedding 和 new doc embedding 不能比**。生产里换 embedding 模型 = 重灌整个索引,这是大决策。

```python
# ❌ 报警 1:embedding 没 normalize 就直接 dot

# ✓ 修复
scores = F.normalize(query, dim=-1) @ F.normalize(docs, dim=-1).T

# ❌ 报警 2:cosine 公式手写,且没处理零向量
cos = a @ b / (norm(a) * norm(b))             # |a|=0 时除零
# ✓ 修复
cos = F.cosine_similarity(a, b, dim=-1, eps=1e-8)

# ❌ 报警 3:L1 正则的"绝对值"忘记 .sum()
loss = lambda_ * torch.abs(W)                 # 这是 tensor,不是标量
# ✓ 修复
loss = lambda_ * torch.abs(W).sum()

# ❌ 报警 4:换 embedding 模型后没重建索引
new_query_emb = new_model.encode(query)

# ✓ 修复:重新 encode 所有 doc + 重建索引

# ❌ 报警 5:在 normalize 之后又做加减,然后又 dot
delta = a_norm - b_norm + c_norm              # 此时 |delta| != 1
score = delta @ d_norm                         # 不再是合法 cosine

# ❌ 报警 6:对抗扰动用 L2 算 ε,但 clip 时按 L∞ clip
delta = clip_l_inf(delta, eps)                # eps 含义错位

```
