拉格朗日对偶与约束优化 ​

这一篇要回答的是一类被工程师集体回避的问题:有约束的优化怎么办? SVM 推导里突然冒出来的"对偶"是干嘛的?投资组合"权重和=1"的约束怎么写进 loss?PPO 的 clip 又跟"约束"有什么关系?全部答案藏在拉格朗日乘子和 KKT 条件里——这玩意学院派讲得吓人,工程上其实就是"给每个约束配一个价格"。

一句话先记住:拉格朗日对偶把"min f s.t. g≤0" 改写成 "min f + λg",每个约束配一个乘子 λ,λ 是这个约束的"影子价格"——它告诉你"放松这个约束一单位能给目标省多少"。SVM 把对偶推到极致,自然推出核技巧。深度学习时代大部分场景(SGD 训神经网络)用不到这套——直接拉格朗日松弛(把约束加到 loss 里)就够。这一篇分清"什么时候上拉格朗日对偶 / 什么时候用罚函数"。

一、为什么要懂(即使你不证 SVM 对偶) ​

不懂约束优化的工程师会犯这些错:

场景不懂时的错法懂之后
投资组合"权重和=1, 不能做空"加到 loss 里加权,瞎调 λ用 CVXPY 写约束,自动求解
SVM 推导论文跳过对偶部分,看不懂"为什么有 α_i"看一眼乘子知道 active 约束在哪
看 RLHF 论文里的 KL 约束不知道为什么 PPO 要 clip这是约束优化的拉格朗日松弛形式
经济 / 资源调度不会用"影子价格"知道乘子是"约束的边际价值"
看 Karush-Kuhn-Tucker 条件看不懂论文里那一串等式一眼分辨哪些约束 active

这一篇的目标:让你看到一个带约束的优化问题,能 30 秒判断"用 CVXPY 直接写 / 拉格朗日松弛 / 完全可以忽略约束";看到 KKT 公式能直接读出哪些约束在边界上。

二、约束优化的标准形式 ​

2.1 写法 ​

min   f(x)               ←─ 目标函数
x

s.t.  g_i(x) ≤ 0,  i = 1, ..., m       ←─ m 个不等式约束
      h_j(x) = 0,  j = 1, ..., p       ←─ p 个等式约束

s.t. 是 "subject to"。约束把可行域从整个空间收缩到一个子集——你只能在这个子集里找最小。

无约束:              有约束:
                                
   *   ←─ 全局最小       │ * ╲  ←─ 真正最小被切掉
                         │    ╲
   ─────                 │ ╳   ╲
                         │可行域
                         │      ╲___
                         └────────────
                              约束边界

2.2 三种典型场景 ​

场景约束长什么样工程语境
资源 / 预算Σx_i ≤ B,x_i ≥ 0投放、调度、组合
物理 / 工艺`
概率 / 几何Σp_i = 1,p_i ≥ 0概率分布、稀疏正则
鲁棒性 / 安全f(x) ≤ τRL 的安全约束、KL 约束

三、拉格朗日函数:把约束放进目标 ​

3.1 定义 ​

给每个约束配一个乘子(multiplier),乘起来加到目标上:

L(x, λ, μ) = f(x) + Σ λ_i g_i(x) + Σ μ_j h_j(x)
                     ↑              ↑
              不等式约束乘子   等式约束乘子
              λ_i ≥ 0          μ_j 任意符号

关键不变量:

  • λ_i ≥ 0:不等式约束的乘子非负(后面解释为什么)
  • μ_j 可正可负:等式约束的乘子无符号要求

3.2 直觉:把约束转成"罚款" ​

L = f + λg,意思是:

  • 你在原目标 f 上,加一个跟约束违反程度成比例的"罚款"
  • 违反 g(x) > 0 时,λg(x) > 0,被罚
  • 满足 g(x) ≤ 0 时,λg(x) ≤ 0,不罚甚至给"奖励"

核心直觉:λ 是这个约束的"价格"——违反 1 单位约束,目标值要付出 λ 单位的代价。这正是经济学里"影子价格"的概念。后面会再展开。

3.3 几何理解 ​

约束 g(x) ≤ 0 限制可行域:                    
                                              
   ╱──╲          ←─ f 的等高线              
  │ *  │           (内圈是低值)             
   ╲__╱                                       
                                              
─────────  ←─ g(x) = 0 的边界               
                                              
   可行域(g ≤ 0 的一侧)                    
                                              
   最优解通常在边界上(切点)                 
   切点处 ∇f 平行于 ∇g                       
   即 ∇f = -λ ∇g, λ > 0

这就是拉格朗日条件的几何解释:约束最优 = 目标梯度跟约束梯度反向平行。

四、对偶函数:取下界 ​

4.1 定义 ​

拉格朗日对偶函数:对 x 取最小,只剩 λ 和 μ:

g(λ, μ) = inf_x  L(x, λ, μ)
        = inf_x  [f(x) + Σλ_i g_i(x) + Σμ_j h_j(x)]

性质:g(λ, μ) 是 λ ≥ 0, μ 的凹函数(无论原问题凸不凸)。

4.2 弱对偶:总是成立的下界 ​

g(λ, μ) ≤ p*    ∀ λ ≥ 0, μ      (p* 是原问题最优值)

直觉:对任何"价格" (λ, μ),你拿到的 L 的最小值都不会超过原问题的最优。

对偶问题:
  max   g(λ, μ)
  λ, μ
  s.t.  λ ≥ 0

对偶最优值 d* ≤ p*——这叫"弱对偶"。两者的差 p* - d* 叫"对偶间隙"。

4.3 强对偶:凸 + Slater → d* = p* ​

好消息:当原问题凸,且满足 Slater 条件(存在严格内点,即 g_i(x) < 0),则 d = p——对偶解原问题**。

凸问题 + Slater 条件:
  ┌────────────────────────────┐
  │  d* = p*                    │
  │                              │
  │  →  解对偶问题 ≡ 解原问题     │
  └────────────────────────────┘

核心直觉:强对偶让"原问题难解时换对偶解"成为可能——SVM 就是利用这个。原问题是"找最大间隔超平面",变量是高维 w;对偶问题是"调每个样本的权重 α",变量数 = 样本数。很多时候对偶问题变量更少、结构更好。

4.4 非凸时怎么办 ​

非凸时通常 d* < p*,对偶值给你原问题的下界(可以用来评估近似算法的差距)。

非凸场景的常见用法:
  - 对偶值作为 lower bound,衡量启发式算法离最优有多远
  - 部分 NP-hard 问题(MAXCUT)的 SDP 松弛 = 求对偶
  - 大规模整数规划的"拉格朗日松弛"

五、KKT 条件:约束最优的判定 ​

5.1 四条件 ​

KKT(Karush-Kuhn-Tucker)条件是约束优化最优解的"必要条件"(凸 + Slater 下也充分):

1. 平稳性 (Stationarity):
   ∇f(x*) + Σλ_i* ∇g_i(x*) + Σμ_j* ∇h_j(x*) = 0
   
   即 ∇_x L(x*, λ*, μ*) = 0

2. 原始可行性 (Primal feasibility):
   g_i(x*) ≤ 0,    h_j(x*) = 0

3. 对偶可行性 (Dual feasibility):
   λ_i* ≥ 0

4. 互补松弛 (Complementary slackness):
   λ_i* · g_i(x*) = 0    ←─ 对每个 i 都成立

5.2 互补松弛的几何意义 ​

第 4 条互补松弛最重要、最直观:

对每个约束 i,要么:
  - λ_i* = 0  (约束的乘子是 0)
  - g_i(x*) = 0  (约束在边界上,active)
  - 或者两者都为 0

但不会出现:λ_i > 0 且 g_i < 0(约束严格满足但还有非零价格)
图示:
                                         
   active 约束:            inactive 约束:  
                                         
   x* ●─────                    ●  x*    
       │     ←─ g_i = 0          │      
       │     ←─ λ_i > 0          │  g_i < 0
       │                         │  λ_i = 0
                                         
   "约束在边界上,            "约束没被碰到,
    且有非零价格"              价格为 0"

核心直觉:互补松弛 = active 约束 ↔ 乘子非零。对偶变量 λ_i 的大小,直接告诉你"约束 i 有多 active",λ_i = 0 就是"完全可以忽略这个约束"。这是对偶问题最有用的副产品——告诉你哪些约束在 binding(束缚)解。

5.3 互补松弛在 SVM 里就是支持向量 ​

在 SVM 里,样本 i 对应一个不等式约束 y_i(w·x_i + b) ≥ 1。

KKT 互补松弛说:

  • α_i > 0 ⟺ y_i(w·x_i + b) = 1(样本恰好在边界上)
  • 这些样本就是 支持向量(support vectors)
  • α_i = 0 的样本远离边界,对最终的 w 完全没贡献

核心直觉:SVM 的"只有支持向量决定分类面"是 KKT 互补松弛的直接推论。这就是为什么 SVM 可以稀疏地存模型——大部分样本的 α 是 0,丢掉就行。

六、对偶问题为什么有用 ​

6.1 变量数变了 ​

原问题(SVM):
  变量:w ∈ R^d  (维度可能上万)
  
对偶问题:
  变量:α ∈ R^N  (样本数,如 1万)

如果 d > N,对偶问题更小
如果 d < N,原问题更小

更重要的是:对偶问题的变量结构(每个样本对应一个 α)
让"核技巧"成为可能

6.2 SVM 对偶的两大产物 ​

不证明,直接看结论:

原问题:
  min  (1/2)||w||²
  s.t. y_i (w·x_i + b) ≥ 1

对偶问题:
  max  Σα_i - (1/2) ΣΣ α_i α_j y_i y_j (x_i · x_j)
  s.t. Σα_i y_i = 0,  α_i ≥ 0

关键观察:
  1. 对偶里只出现 x_i · x_j(内积),不出现单个 x_i
  2. → 把 x_i · x_j 换成 K(x_i, x_j) = φ(x_i)·φ(x_j) → 核技巧!
  3. 不需要显式算 φ(x)(可能是无穷维),只要会算核 K

核心直觉:核技巧不是 SVM 的发明,是对偶形式的副产品——你写出对偶问题,发现里头只有 x_i · x_j,自然就想到"把内积换成更花哨的东西"。这是数学结构推动算法发明的经典例子。

内积在对偶里出现 → 替换为核 K → 隐式映射到无穷维特征空间

K(x, y) 选:
  - 线性:    K = x·y                          (普通线性 SVM)
  - 多项式:  K = (x·y + c)^d                  
  - 高斯:    K = exp(-||x-y||²/2σ²)           (RBF, 最常用)
  - 字符串:  K = 编辑距离 / 公共子串数 / ...   (字符串/序列分类)

6.3 对偶给原问题下界 ​

原问题难解 → 对偶值 d* 给 p* 一个下界 → 评估算法差距

例子:
  你跑了一个启发式算法,得到 obj = 100
  对偶问题给出 d* = 95
  → 你的解距离最优至多 (100 - 95)/95 = 5%
  → 不需要继续优化了

这在组合优化、运筹学里非常常用——整数规划的 LP 松弛 + 对偶分析就是分支定界算法的核心。

6.4 影子价格的经济解释 ​

问题:    min f(x)  s.t. g(x) ≤ 0
最优:    p* = f(x*)
乘子:    λ*

性质:    ∂p*/∂(约束 RHS) = -λ*

意思:    "把约束放松一单位(g ≤ 0 改成 g ≤ 1),
         目标值能降 λ* 单位"
经济学例子:
  目标:    最大化收益
  约束:    "原料 A 不超过 100 单位"
  乘子 λ:  10
  
  解读:   "如果再给我 1 单位原料 A,
          我能多赚 10 块钱"
          → λ = 10 是 A 的"影子价格"
          → 如果市场上 A 的实际价格 < 10,买一单位赚
          → 如果实际价格 > 10,不买

这就是经济学里"边际成本 = 边际收益"的数学形式。所有商品定价、资源调度、物流优化都建在这个直觉上。

七、代码:CVXPY 写约束优化 ​

7.1 投资组合优化(经典 QP) ​

python
import cvxpy as cp
import numpy as np

# 10 只股票
n = 10
mu = np.array([0.10, 0.08, 0.12, 0.05, 0.15,
              0.07, 0.09, 0.11, 0.06, 0.13])
np.random.seed(42)
Sigma = np.random.randn(n, n)
Sigma = Sigma @ Sigma.T / 10        # 协方差矩阵

# 决策变量:权重
w = cp.Variable(n)
gamma = 1.0

# 目标:最大化 (μ^T w - γ w^T Σ w)
# 约束:权重和=1,不允许做空
problem = cp.Problem(
    cp.Maximize(mu @ w - gamma * cp.quad_form(w, Sigma)),
    [cp.sum(w) == 1, w >= 0]
)
problem.solve()

print("最优权重:", w.value.round(3))
print("约束乘子(影子价格):")
print("  权重和 = 1:", problem.constraints[0].dual_value.round(3))
print("  w_i ≥ 0:    ", problem.constraints[1].dual_value.round(3))

对偶值的实际含义:

  • 权重和=1 的乘子:如果允许总仓位 105% 而不是 100%(加一点杠杆),收益能提升多少
  • w_i ≥ 0 的乘子:第 i 只股票如果允许小幅做空(放松 0 下界),收益变化多少(>0 表示对应股票"想被做空"才赚得多)

7.2 直接看 SVM 对偶解出什么 ​

python
import cvxpy as cp
import numpy as np
from sklearn.datasets import make_blobs

# 生成线性可分的二分类数据
X, y = make_blobs(n_samples=20, centers=2, random_state=0)
y = 2 * y - 1           # 把 0/1 转成 -1/+1
N = len(y)

# 对偶 SVM
alpha = cp.Variable(N)
K = X @ X.T              # 内积核
Y = np.outer(y, y)
problem = cp.Problem(
    cp.Maximize(cp.sum(alpha) - 0.5 * cp.quad_form(alpha, cp.psd_wrap(Y * K))),
    [alpha >= 0, y @ alpha == 0]
)
problem.solve()

# 看哪些样本是支持向量
sv_idx = np.where(alpha.value > 1e-4)[0]
print(f"总样本数: {N}")
print(f"支持向量数: {len(sv_idx)}")
print(f"支持向量的 α: {alpha.value[sv_idx].round(3)}")
print(f"非支持向量的 α: {alpha.value[~np.isin(np.arange(N), sv_idx)].round(3)}")
# 输出会显示:大部分 α ≈ 0,只有 2-3 个 α > 0(支持向量)

这一段值得品味:互补松弛在代码里直接体现为"大部分 α 是 0"——不需要你去 prune,KKT 条件自己告诉你哪些样本被丢掉。

7.3 等式约束的乘子可正可负 ​

python
import cvxpy as cp

x = cp.Variable(2)
problem = cp.Problem(
    cp.Minimize(cp.sum_squares(x)),
    [x[0] + x[1] == 5]              # 等式约束
)
problem.solve()

print("x* =", x.value)              # ≈ [2.5, 2.5]
print("等式约束乘子 μ =", problem.constraints[0].dual_value)
# μ ≈ -5,符号无关,意思是"放松约束(改成 4)目标会变 5 单位"

八、拉格朗日松弛:工程上的"近似"做法 ​

8.1 罚函数法(Penalty Method) ​

真正的拉格朗日对偶在大规模 ML 里很少直接用——一般直接搞"罚函数"。

原约束问题:  min f(x)  s.t. g(x) ≤ 0
罚函数版本:  min f(x) + ρ · max(0, g(x))²

ρ:  罚系数,你手动调
ρ → ∞:  解越接近原约束问题
ρ 太大:  数值不稳定
python
# 罚函数法实现"约束"在 SGD 里
import torch

def loss_with_penalty(x, rho=10.0):
    f = (x - 3) ** 2                # 想最小化 (x-3)²
    g = x - 1                       # 约束:x ≤ 1
    penalty = rho * torch.relu(g) ** 2   # 罚违反部分
    return f + penalty

x = torch.tensor(3.0, requires_grad=True)
opt = torch.optim.SGD([x], lr=0.01)
for _ in range(1000):
    opt.zero_grad()
    loss = loss_with_penalty(x)
    loss.backward()
    opt.step()
print("x =", x.item())              # ≈ 1.05(接近 1,但不是精确)

避坑:罚函数得到的解通常"约束基本满足,但不精确"——x ≈ 1.05 而不是 1.0。这对 ML 训练通常无所谓,但对金融、控制等需要精确约束的场景可能不行——这种场景必须用 CVXPY / Mosek 这种支持精确约束的工具。

8.2 增广拉格朗日(Augmented Lagrangian) ​

罚函数法和拉格朗日法的杂交,精确约束 + 数值稳定:

L_ρ(x, λ) = f(x) + Σλ_i g_i(x) + (ρ/2) Σ max(0, g_i(x))²

迭代:
  1. 固定 λ,最小化 L_ρ 得到 x
  2. 更新 λ: λ_i ← max(0, λ_i + ρ g_i(x))
  3. 重复

这是 ADMM、ALM 这些大规模约束优化算法的基础——你在论文里看到 "augmented Lagrangian",就是这套。

8.3 PPO 的 clip 是约束的近似 ​

RL 里的 PPO:

原约束目标(TRPO):
  max  E[ratio · advantage]
  s.t. KL(π_new || π_old) ≤ δ        ←─ 信任域约束

PPO clip 近似:
  max  E[min(ratio · adv, clip(ratio, 1-ε, 1+ε) · adv)]
                              ↑
                       直接 clip ratio 在 [1-ε, 1+ε]
                       近似实现"不要偏离 π_old 太多"

核心直觉:PPO 的 clip 是"用截断代替严格约束"——不写拉格朗日,不写对偶,直接 clip。这种"工程化的拉格朗日松弛"在 RL 里特别普遍——TRPO 写得严谨但慢,PPO 用 clip 近似但快得多,实际效果差不多。这就是工程师对约束优化的态度:只要能 work,精确性次之。

8.4 RLHF 的 KL 罚项 ​

RLHF 标准目标:
  max  E[reward(x, y)] - β · KL(π_new || π_ref)
                          ↑
                  这是个拉格朗日松弛形式
                  原问题是"max reward s.t. KL ≤ τ"
                  转成 "max reward - β·KL"
                  β 越大,越被约束在 ref 附近

这就是 InstructGPT / ChatGPT / LLaMA-2 RLHF 训练的核心目标——一个标准的拉格朗日松弛。β 调小 → 模型可以远离 ref(reward 高但可能崩坏);β 调大 → 模型紧贴 ref(安全但学不到东西)。

九、什么时候不必管对偶 ​

实话:深度学习 95% 的训练场景,对偶完全用不到。

什么时候用拉格朗日松弛 / 罚函数就够:
  1. ML 训练里的"软约束"(正则化、KL 罚项)
  2. RL 里的信任域(直接 clip 就行)
  3. 工程系统里的近似约束(保证 99% 满足就行)
  4. 大规模问题(变量百万级,对偶求解器吃不消)

什么时候必须上 CVXPY / 对偶:
  1. 凸优化问题且需要精确解(投资组合、信号处理)
  2. SVM / Lasso 这些经典凸 ML 模型(传统库里就内嵌)
  3. 需要"影子价格"的经济 / 资源问题
  4. LP / QP / SDP 等结构化凸问题

经验法则:写 PyTorch 训神经网络,几乎用不上拉格朗日对偶——你需要的是"加正则项 + 调系数"。写 CVXPY 解凸优化问题,根本不用自己写对偶——CVXPY 自动处理。真正需要手推对偶的场景:你在写一个新的求解器、研究 SVM / LASSO 的稀疏性、做经济/物流的影子价格分析。

十、调对偶 / 罚系数的工程经验 ​

10.1 罚系数 / 拉格朗日乘子怎么调 ​

场景               典型超参         调参方向
─────────────────────────────────────────────────
L1 / Lasso 正则    λ ∈ [0.001, 1]   交叉验证
L2 / Ridge 正则    λ ∈ [0.001, 1]   交叉验证
RLHF KL 罚项 β     0.01 ~ 0.5       β 大 → 紧跟 ref;小 → 偏离 ref
PPO clip ε         0.1 ~ 0.3        ε 大 → 学得快但可能崩
TRPO 信任域 δ      1e-3 ~ 1e-2      δ 大 → 步长大;小 → 步长小
增广拉格朗日 ρ     1 ~ 100,逐步↑    数值上从小往大调

10.2 看乘子诊断问题 ​

如果 CVXPY 解出的某个约束乘子异常大 → 这个约束在严重 binding
                                      → 要么解决不了你的问题,要么放松它

如果某个约束乘子是 0 → 约束完全没起作用
                    → 可以从模型里删掉,简化问题

如果 RLHF 训练 KL 持续增大 → β 太小,模型在偏离 ref
                          → 调大 β

如果 PPO 训练 ratio 经常被 clip → 学习率或 ε 不匹配
                                → 调小 lr 或调小 ε

十一、工程对应清单 ​

工具 / 系统跟约束优化的关系
CVXPY constraints写凸约束的 DSL,直接 [w >= 0, sum(w) == 1, ...]
scikit-learn SVM内部用 libsvm / liblinear 解对偶问题

12.2 KKT 条件不满足的常见原因 ​

1. 解不在可行域内  → 某个 g_i(x*) > 0     (原始可行性破)
2. λ_i < 0         → 不等式约束乘子不能负 (对偶可行性破)
3. λ_i > 0 且 g_i < 0 → 互补松弛破        (要么 λ=0,要么 g=0)
4. ∇L ≠ 0          → 没找到驻点          (平稳性破)

CVXPY 出问题时,先 print 每个约束的 .dual_value 和 .residual,
能定位到具体哪条约束坏了

12.3 罚函数 vs 对偶的取舍 ​

                        罚函数(SGD 风)        对偶(CVXPY 风)
─────────────────────────────────────────────────────────────
精确约束?              不精确(99% 满足)       精确
代码复杂度?            简单                    DSL 写约束
能反传?                能(autograd)           不能
能 mini-batch?         能                      不能(全数据求解)
变量规模               > 10^6                  < 10^5
求解时间               毫秒(单步)             几秒到几分钟(整个问题)
适合场景               神经网络训练            凸优化、金融、控制
诊断能力               看不到乘子              .dual_value 给影子价格

核心直觉:这两种是不同尺度的工具,不是替代关系。神经网络训练里"加 L2 正则"是罚函数版的拉格朗日,你不需要写对偶——HuggingFace 一行 weight_decay=0.01 解决。但你做投资组合、做电网调度、做 SVM 推导,对偶是绕不开的——你想看影子价格、想稀疏地存模型、想用核技巧,都得通过对偶。
