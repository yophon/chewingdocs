## 二、NumPy:一切的底座

### 2.1 ndarray 与矢量化思维

NumPy 的核心不是 API,是**"把循环换成数组操作"的思维**:

```python
# 反例(Python 循环):慢 100x
result = []
for i in range(N):
    result.append(np.sin(x[i]) ** 2 + np.cos(x[i]) ** 2)

# 正解(矢量化):底层 C / SIMD,极快
result = np.sin(x) ** 2 + np.cos(x) ** 2
```

> **核心直觉**:**NumPy 快不是因为它"是 C 写的",是因为你写的每一行表达式都对应一次"批量数据 → BLAS / SIMD"的调用,绕开了 Python 解释器**。**写循环就是把这条快车道关掉,**这是 NumPy 新手最大的那个坑。

### 2.2 broadcasting 的红线

广播是 NumPy 最香也最容易出 bug 的地方。**红线规则**(从最右一维往前对齐):

```
A.shape = (5, 1, 3)
B.shape = (   4, 3)
─────────────────────
对齐:    (5, 1, 3)
         (1, 4, 3)         ← 自动补 1 在前面
─────────────────────
都满足:相等 / 1 / 不存在 → 结果 shape (5, 4, 3)
```

> **避坑**:**形状 (n,) 和 (n, 1) 不是同一个东西**——前者是"长度 n 的向量",后者是"n×1 的矩阵"。`a + a.T` 在前者是 1D + 1D = 标量加;在后者是 (n,1)+(1,n) → (n,n) 外积。**一行代码意思完全不一样**——见到 broadcast 行为不符合预期,先 `print(a.shape, b.shape)`。

### 2.3 何时该用 numba / Cython

矢量化解决"批量元素同样操作";**有些算法天然有依赖、不能矢量化**(比如某些 DP、状态机模拟、自定义递推)。这时候:

| 工具 | 何时用 |
| --- | --- |
| **numba `@jit`** | 数值代码,纯 NumPy 写不出来,加几行装饰器就快 50-100x |
| **Cython** | 需要和 C 库深度互操作,或编译期类型检查 |
| **C / Rust / pybind11** | 真正性能关键,愿意写另一种语言 |
| **JAX `@jit`** | 同时要 autograd + GPU,见后文 |

> **避坑**:**先矢量化,矢量化不行才上 numba**——见到 NumPy 代码里 for 循环就 numba,大概率你只是没想清楚怎么矢量化。Profiler 里看见瓶颈在 Python 层再上加速。

## 三、SciPy:NumPy 的"上层超市"

NumPy 是底座,SciPy 是建在上面的"应用层数学"。**模块清单值得记住**——用到时直接进对应子模块:

| 子模块 | 干什么 | 你日常用到的 |
| --- | --- | --- |
| **scipy.stats** | 概率分布、检验、相关系数 | `norm.cdf` / `t.ppf` / `ks_2samp` / `pearsonr` |
| **scipy.linalg** | BLAS/LAPACK 包装,比 `numpy.linalg` 全 | `svd` / `lu` / `qr` / `solve` / `eig`(含广义特征值) |
| **scipy.optimize** | 凸 + 非凸、约束、根求解、最小二乘 | `minimize` / `least_squares` / `fsolve` / `linprog` |
| **scipy.sparse** | 大稀疏矩阵 + 稀疏求解 | `csr_matrix` / `csc_matrix` / `csgraph` / `sparse.linalg.eigs` |
| **scipy.signal** | 信号处理、滤波器、卷积、小波 | `fft` / `convolve` / `welch` / `lfilter` |
| **scipy.spatial** | KD-tree、距离、凸包、Voronoi | `KDTree` / `distance.cdist` / `ConvexHull` |
| **scipy.special** | 特殊函数(gamma、beta、贝塞尔) | `softmax` 实现里隐藏的 logsumexp |
| **scipy.integrate** | 数值积分、ODE 求解 | `quad` / `solve_ivp` |

### 3.1 几个高频套路

```python
# 1. 凸优化:最小化任意可调用 f(x)
from scipy.optimize import minimize
res = minimize(lambda x: (x[0]-2)**2 + (x[1]+1)**2, x0=[0,0])
print(res.x)                 # [2, -1]

# 2. 检验两个样本来自同分布
from scipy.stats import ks_2samp
stat, p = ks_2samp(sample_a, sample_b)
# p < 0.05 → 拒绝"同分布"假设;但记住 15 篇说过的多重比较陷阱

# 3. 大稀疏矩阵的特征值(N=百万级图算 PageRank 时就这玩意)
from scipy.sparse.linalg import eigs
vals, vecs = eigs(sparse_matrix, k=6, which='LM')

# 4. 快速傅里叶变换
from scipy.fft import fft, ifft
```

> **经验法则**:**遇到一个数值问题先问"SciPy 有没有",再考虑造轮子**。SciPy 底层是几十年沉淀的 LAPACK / FFTW / SuperLU,**精度和速度都比你手写好很多个量级**。

## 四、SymPy:符号计算,数学手稿的对账员

NumPy/SciPy 是数值的;SymPy 是**符号**的——它不算 1.4142,它给你 `√2`。

### 4.1 它能干什么

```python
import sympy as sp

x, y = sp.symbols('x y')

# 求导
expr = sp.exp(-x**2)
print(sp.diff(expr, x))                  # -2*x*exp(-x**2)

# 求积
print(sp.integrate(sp.sin(x)*sp.exp(-x), (x, 0, sp.oo)))  # 1/2

# 化简
print(sp.simplify((sp.sin(x)**2 + sp.cos(x)**2)))         # 1

# 解方程
print(sp.solve(x**2 - 5*x + 6, x))                        # [2, 3]

# 矩阵符号运算
A = sp.Matrix([[1, x], [y, 2]])
print(A.det())                                            # 2 - x*y
print(A.inv())                                            # 解析逆矩阵

# 生成 LaTeX
print(sp.latex(sp.diff(sp.sin(x)/x, x)))
# \frac{\cos{\left(x \right)}}{x} - \frac{\sin{\left(x \right)}}{x^{2}}
```

### 4.2 SymPy 比 NumPy 香的场景

| 场景 | 为什么 SymPy 更合适 |
| --- | --- |
| **验证手推公式** | 数值噪声会掩盖真错;符号能精确告诉你"两个表达式恒等" |
| **生成 LaTeX 公式** | 写论文 / blog 时 `sp.latex(expr)` 直接出 |
| **看清楚链式法则展开后是什么** | autograd 跑得起来但你不知道在算什么时,SymPy 能给你"展开式" |
| **教学 / 学习** | 看见 `diff(x**2, x)` 直接给 `2*x`,直觉强化 |
| **求小规模解析解** | ODE、特征多项式、对偶问题等,SymPy 给闭式 |

> **核心直觉**:**SymPy 不是"另一个 NumPy",是数学家手稿的电子化对账员**。当你怀疑自己手推一个公式漏了系数,**SymPy `simplify(my_formula - reference)` 给 0 就是对的,不是 0 就漏了什么**。

### 4.3 SymPy 的红线

- **慢**——符号运算的复杂度天然比数值高几个数量级,不要拿 SymPy 跑大规模数值实验
- **化简能力有上限**——复杂表达式 `simplify` 可能跑半天给你一个等价但更长的式子;`trigsimp` / `radsimp` / `cancel` 这些专门化简器要分情况用
- **不要把它当工程库**——SymPy 的产出是"公式"或"代码生成"(`sp.lambdify` 把符号表达式转成 NumPy 函数,这是它进生产的正确姿势)

## 五、JAX:函数式 + autograd + jit + vmap + pmap

JAX 是 2018 年 Google Brain 推出的"NumPy 的可微版本"。**API 和 NumPy 几乎完全一样**(`jax.numpy as jnp`),但加了四个变换函数,**改变科研代码风格的事就在这四个函数上**:

| 函数 | 干什么 | 一句话直觉 |
| --- | --- | --- |
| `grad(f)` | 自动求导 | 给我函数,我给你梯度函数 |
| `jit(f)` | 即时编译 | XLA 编译,GPU/TPU 上跑 |
| `vmap(f)` | 自动 batch | 写单样本函数,自动支持 batch 维 |
| `pmap(f)` | 多设备并行 | 自动跨多 GPU/TPU 复制并行 |

### 5.1 一段示例

```python
import jax
import jax.numpy as jnp

# 一个标量函数
def loss(w, x, y):
    return jnp.mean((jnp.dot(x, w) - y) ** 2)

# 自动求 w 的梯度
grad_loss = jax.grad(loss, argnums=0)

# 编译加速
fast_grad = jax.jit(grad_loss)

# vmap:写单样本 loss,自动支持 batch
def single_loss(w, xi, yi):
    return (jnp.dot(xi, w) - yi) ** 2
batch_loss = jax.vmap(single_loss, in_axes=(None, 0, 0))
# in_axes=(None, 0, 0) 意思:w 不 batch,xi 和 yi 沿 axis 0 batch
```

### 5.2 函数式心智:为什么科研在迁移

```
PyTorch:       JAX:
─────────      ─────
nn.Module       纯函数 + 显式参数树(pytree)
state in self   state pass through
loss.backward() jax.grad(loss)
optim.step()    params = update_fn(params, grads)
```

> **核心直觉**:**JAX 强迫你写"无副作用的纯函数",参数显式传进出**。这种风格让 `vmap` / `pmap` / `jit` 这些变换"自动组合"——一个写好的纯函数自动支持 batch、自动支持多 GPU、自动 jit。**PyTorch 里同样的事要手动改 `nn.Module`,所以高阶魔法的成本高得多**。

### 5.3 PyTorch vs JAX:心智差异

| 维度 | PyTorch | JAX |
| --- | --- | --- |
| 默认 mode | eager,所见即所得 | 没 jit 时也是 eager,但**生产路径是 jit** |
| 状态 | 隐式藏在 `nn.Module` | 显式 pytree,函数无状态 |
| 调试 | print + pdb | jit 内部不能直接 print(用 `jax.debug.print`) |
| 生态 | **生产、推理、产业 + 巨量预训练模型** | **科研、Google 系、JAX-DeepMind 一系列库** |
| 工业部署 | TorchScript / ONNX / TensorRT | XLA / TPU |
| 大模型框架 | HuggingFace Transformers / Lightning / DeepSpeed | Flax / Haiku / EquinoX |

> **避坑**:**JAX 的"陷阱"是 jit 边界**——纯函数里不能有 Python `if`(条件依赖 tracer)、不能有 print 副作用、不能改全局变量。**第一次撞上 `ConcretizationTypeError` 时回去看官方"sharp bits"文档,半小时能避掉 90% 的坑**。

## 六、PyTorch 里的数学库

PyTorch 不只是"建神经网络的库",它的几个数学子模块用好了能省一吨自己的代码:

| 模块 | 干啥 | 例子 |
| --- | --- | --- |
| **`torch.linalg`** | LAPACK 风格的线性代数 | `svd` / `eigh` / `qr` / `pinv` / `solve` / `cholesky` |
| **`torch.distributions`** | 概率分布对象,可采样 + 求 log_prob + 自动求导 | `Normal` / `Categorical` / `Dirichlet` / KL / `rsample` 重参数化 |
| **`torch.special`** | 特殊函数 | `gammaln` / `i0`(贝塞尔)/ `expit`(sigmoid)/ `digamma` |
| **`torch.fft`** | 快速傅里叶 | `fft` / `ifft` / `rfft` / `fftn` |
| **`torch.optim`** | 优化器(23 篇细讲) | `AdamW` / `SGD` / `LBFGS` |
| **`torch.autograd.functional`** | 函数式 grad / Hessian / vjp / jvp | `jacobian` / `hessian`(小问题用,大问题慢) |

### 6.1 几段值钱的小代码

```python
import torch

# 1. SVD 或低秩近似
U, S, Vh = torch.linalg.svd(A, full_matrices=False)
A_lowrank = U[:, :r] @ torch.diag(S[:r]) @ Vh[:r, :]

# 2. 用 Distribution 写一个可微的"采样 + 损失"
dist = torch.distributions.Normal(mu, sigma)
z = dist.rsample()                         # 重参数化采样,梯度能回流到 mu/sigma
log_prob = dist.log_prob(target).sum()
kl = torch.distributions.kl_divergence(dist, prior_dist)

# 3. 数值梯度自检
from torch.autograd import gradcheck
gradcheck(my_op, (x.requires_grad_(),), eps=1e-6)
# 写自定义 op 时必跑;不通过你的 backward 实现错了
```

> **核心直觉**:**`torch.distributions` 是把概率论搬进 autograd 的杠杆**——VAE / RL 策略梯度 / Diffusion 反向都依赖它。**学到 `rsample` 这一招(重参数化让梯度穿过随机采样),你看 VAE 论文那一行 `z = mu + sigma * eps` 就秒懂**。

## 八、反对的姿势

这一节是给容易走弯路的同学的"红牌":

1. **不要从证明题开始**——你不是数学系学生,不需要证 ε-δ。**从直觉 + 工程对应入手,公式当工具用,推导能跟得上即可**。

3. **不要囤课不动手**——Coursera / Udemy / B 站收藏夹里的课不会自动让你变强。**一周写 3 行 SymPy 代码 > 一个月只看视频**。
4. **不要"一遍过完所有数学再看论文"**——**反过来**:**先看你想看的论文,卡公式时回头查对应这一篇 / 这本书的对应章节**,这样数学才有去处。
5. **不要觉得"自己实现一遍"是浪费时间**——**线性回归 / SVD / PageRank / Viterbi / RSA 这种,自己 30 行写一遍比看 30 篇博客都强**。这是 Karpathy nn-zero-to-hero 那么受欢迎的原因。
6. **不要用工具替代理解**——LLM 可以解释、SymPy 可以验证,但**最后理解必须发生在你脑子里**。看 LLM 给的解释能复述出来 + 自己跑通代码 = 真懂。

## 九、工程师的"30 分钟数学回血"路径

**当你工作中卡在一段公式上,以下是一条经过验证的可执行流程:**

```
─── 第 0 分钟 ───────────────────────────────────────
   把那段公式抄下来(或截图)
   写下:"我现在不懂这一行的什么"——是符号、是项的物理意义、还是整体结论?

─── 第 0-5 分钟 ──────────────────────────────────────
   把公式扔给 LLM,要求:
     1. 拆解每个符号的含义和量纲
     2. 用工程语言一句话讲整体在干啥
     3. 给一个最小数值例子(尽量是 2-3 维的可手算的)

─── 第 5-15 分钟 ─────────────────────────────────────
   打开 Jupyter / Python REPL:
     - 用 NumPy 把 LLM 给的最小例子实际跑一遍
     - 改几个参数看输出怎么变化
     - 如果是符号公式,用 SymPy `simplify(my_form - reference)` 验证恒等

─── 第 15-25 分钟 ────────────────────────────────────
   真正的理解时刻:
     - 写下"如果我把这个超参 / 这一项调大 / 调小,数学上会怎样、对应工程指标会怎样"
     - 找这一公式在哪个真实库 / 论文里出现,看一下它的实现

─── 第 25-30 分钟 ────────────────────────────────────
   写一句话总结发到自己的笔记里:
     "这一公式干的事是 X,工程上对应的旋钮是 Y,调它会动 Z"

   下次再遇到引用这条公式,直接秒懂。
```

> **核心直觉**:**"看公式 → 问 LLM 拆解 → NumPy/SymPy 跑 → 改超参看动效"——这个循环让数学从"教材里的死符号"变成"我可以实验的活变量"**。**所有这套系列教你的,本质上就是把这个循环跑顺**。
