# 矩阵作为线性变换

上一篇讲清楚"向量住在空间里",这一篇讲清楚"矩阵把空间扭一下"。**绝大多数人学完线代记住的是"矩阵是一张数表",这是个误导**——看到 `nn.Linear(768, 768)`,你觉得它是 768×768 个数;**懂矩阵的人看到的是"一个把 768 维空间扭曲成另一种 768 维空间的变换"**。这两种视角差很多——前者只能调超参,后者能看出"这层在压缩信息"还是"在旋转坐标"还是"在挑选子空间"。**Attention 的 W_Q W_K W_V、Conv 的 kernel、图像增强的仿射,底下都是这一件事**。

> 一句话先记住:**矩阵不是数表,矩阵是函数**——它把一个向量映射到另一个向量,`A·v` 读作"用 A 这个变换作用到 v 上"。**所有几何变换(旋转、缩放、剪切、投影、反射)都是 2×2 或 3×3 矩阵;所有神经网络的 Linear / Dense / FC 层,本质都是一次大矩阵的线性变换**。换这个视角后,你看 LoRA 的 BA 分解、Attention 的 QKV 投影、Conv 的局部滤波,**全是同一个数学结构,套不同的物理含义**。

## 一、为什么这一篇必须存在

| 场景 | "矩阵是数表"的视角 | "矩阵是变换"的视角 |
| --- | --- | --- |
| 看 `nn.Linear(768, 1024)` | 768×1024 个权重 | 把 768 维空间映射到 1024 维空间 |
| 看 LoRA 的 `B @ A`,A:(r,d), B:(d,r) | 两个数表相乘,r 是个超参 | 通过低维瓶颈的"投影 → 还原"变换 |
| 看 Conv 2×2 kernel | 4 个数滑窗 | 每个局部块的小线性变换 |
| 看 Attention 的 `W_Q` | 又一个权重矩阵 | 把"原始 token"扭到"提问空间" |
| `det(A) = 0` 报错"奇异矩阵" | 不知道为什么不能求逆 | 变换把空间压扁了,信息丢了 |
| Hessian 病态(condition number 大) | "训练不稳" | 损失曲面有"窄峡谷",优化器卡住 |
| 看 PCA / SVD | 一堆步骤 | 找一个好的旋转,让方差对齐到坐标轴 |
| 图像增强 `cv2.warpAffine(M)` | M 是 2×3 表 | M 描述一个旋转 + 缩放 + 平移的变换 |

**核心收益**:**懂"变换"后,你看模型架构变成"信息流",看反向传播变成"变换的逆推",看初始化变成"控制变换的强度"**——这就是为什么这一篇压根不能跳过。

### 1.1 这一篇的目标

读完你应该能:

1. **看到任何矩阵,问自己"它是什么变换"**——旋转 / 缩放 / 投影 / 一般线性
2. **知道 `Wx + b` 是仿射变换**,理解神经网络每一层在干什么
3. **看到 rank、null space、column space,有几何画面**
4. **看到 `det = 0` 知道变换把空间压扁,看到 condition number 大知道数值不稳**
5. **看到 LoRA / Conv / Attention 的矩阵,能讲出它各扭曲了什么**

## 二、视角切换:从数表到函数

### 2.1 朴素视角:矩阵是张表

```
A = [ 2  0 ]
    [ 0  3 ]
```

**这个视角下,A 就是 4 个数**——你能算行列式、能算逆、能转置,但看不出"它在干什么"。

### 2.2 函数视角:矩阵是 ℝⁿ → ℝᵐ 的映射

```
A: ℝ² → ℝ²
A(v) = A·v    把 v 映射到 A·v

  v = (1, 1)       A·v = (2, 3)
  v = (2, 0)       A·v = (4, 0)
  v = (0, 1)       A·v = (0, 3)

→ A 把 x 方向拉 2 倍,y 方向拉 3 倍
→ A 不是"4 个数",A 是"x 拉 2、y 拉 3"这件事
```

**A 是函数**,`v` 是输入,`A·v` 是输出。**你脑子里要画的是"整个空间被怎么扭"**,不是"两个数怎么乘"。

### 2.3 线性:这个变换的两个铁律

**所有矩阵代表的变换都满足两条规则**(这就是"线性"的定义):

```
1. A(u + v) = A·u + A·v       加法保持
2. A(c·v)   = c·A·v           标量乘保持
```

**几何含义**:**变换前的"加和"等于变换后的"加和";变换前的"拉伸"等于变换后的"拉伸"**——**直线还是直线、原点还是原点、平行线还是平行线**。**矩阵不能弯曲空间,只能"扭"成另一种平直的空间**。

> **核心直觉**:**线性变换 = "只能拉伸和旋转,不能弯折"**。把弹性的橡皮膜均匀拉伸成另一个形状——拉、压、斜切、翻转都行,但不能折叠或撕裂。

## 三、列向量解读:矩阵是"基向量的去向"

**这是看穿矩阵最重要的一个视角**——理解它,你就懂 Attention / LoRA / Conv 在做什么。

### 3.1 第 i 列 = 第 i 个基向量被变换后的位置

```
A = [ 2  1 ]
    [ 0  3 ]

A 的第一列 (2, 0) = A 把 e₁=(1,0) 变到哪
A 的第二列 (1, 3) = A 把 e₂=(0,1) 变到哪
```

**为什么**:

```
A · e₁ = A · (1,0)ᵀ = ( 2·1 + 1·0,
                       0·1 + 3·0 )ᵀ = (2, 0)ᵀ    ← 就是 A 的第一列

A · e₂ = A · (0,1)ᵀ = ( 2·0 + 1·1,
                       0·0 + 3·1 )ᵀ = (1, 3)ᵀ    ← 就是 A 的第二列
```

### 3.2 一切线性变换都被这个决定

**任意向量 v = (x, y) = x·e₁ + y·e₂**,变换后:

```
A·v = A·(x·e₁ + y·e₂) = x·(A·e₁) + y·(A·e₂)
                       = x·(第一列) + y·(第二列)
```

**所以**:**只要告诉我"基向量去哪儿了",整个变换就完全确定**。**A 的每一列就是一个基向量的目的地**——这是矩阵几何的核心 mnemonic。

```
A = [ ─── 第一列 ─── │ ─── 第二列 ─── │ ... ]
        ↓                  ↓
      e₁ 去这里          e₂ 去这里
```

### 3.3 行向量解读(次要,但用到时不能搞错)

**A 的第 i 行**:**告诉你输出的第 i 个分量,是输入各分量的什么线性组合**。

```
A = [ a  b ]              y₁ = a·x₁ + b·x₂   ← A 的第一行
    [ c  d ]              y₂ = c·x₁ + d·x₂   ← A 的第二行
```

| 看 | 含义 |
| --- | --- |
| **列向量视角** | "输入空间的基向量去哪了"——画几何图、看变换性质 |
| **行向量视角** | "输出的每个分量怎么从输入算出来"——读代码、debug shape |

> **经验法则**:**做几何 / 理解模型架构看列;写代码 / debug shape 看行**。两种视角同一矩阵,但在不同问题上一个会比另一个清晰得多。

## 四、2×2 矩阵 NumPy 演示:5 种基本几何变换

**这一节用代码把抽象的"变换"画清楚**。每个变换都给"变换前 / 变换后"的坐标和文字描述,**不需要画图工具**——你脑子里就能跑。

### 4.1 缩放(Scaling)

```python
import numpy as np

S = np.array([[2, 0],
              [0, 3]])

# 把 x 方向拉 2 倍,y 方向拉 3 倍
points = np.array([[1, 0],   # e₁
                   [0, 1],   # e₂
                   [1, 1]]).T   # 对角点
S @ points
# [[2 0 2]
#  [0 3 3]]

# 输入:  e₁=(1,0) → (2,0)   x 方向拉伸 2 倍
#        e₂=(0,1) → (0,3)   y 方向拉伸 3 倍
#        (1,1)   → (2,3)   对角点跟着走
```

**几何效果**:

```
变换前:                            变换后:
   y                                  y
   ▲                                  ▲
   │                                3 │ ────────╮
 1 │ ────╮                            │         │
   │     │                            │         │
   ┼─────┼──► x                       ┼─────────┼──► x
   0     1                            0         2

   单位方格变成 2×3 的长方形
```

**对角矩阵 = 沿各坐标轴独立缩放**——这是最简单的变换,也是最有用的。**RMSNorm 的 `x * γ` 就是逐通道缩放,γ 是个对角矩阵**(代码里写成向量,数学上等价)。

### 4.2 旋转(Rotation)

```python
theta = np.pi / 4    # 45 度
R = np.array([[np.cos(theta), -np.sin(theta)],
              [np.sin(theta),  np.cos(theta)]])

R @ np.array([[1, 0],   # e₁
              [0, 1]]).T
# [[ 0.707 -0.707]
#  [ 0.707  0.707]]

# 输入:  e₁=(1,0) → (0.707, 0.707)    旋转到 45°
#        e₂=(0,1) → (-0.707, 0.707)   也旋转 45°
```

**几何效果**:**整个空间绕原点逆时针转 θ**——所有长度不变,所有夹角不变。

```
变换前:                          变换后:
   y                                y
   ▲                                ▲   ↗ e₁ 转到这
   │                                │  ╱
 1 │ ▲ e₂                           │ ╱   45°
   │ │                              │╱
   │ │──► e₁                      ◄─┼──────► x
   ┼─┴──────► x                     │
   0 1                              0   ↘ e₂ 转到这
```

**关键性质**:`det(R) = 1`,`R·Rᵀ = I`——**旋转矩阵保持长度、保持方向**。Transformer 里的 RoPE(旋转位置编码)就是给每个位置一个不同的 R。

### 4.3 剪切(Shear)

```python
H = np.array([[1, 1],
              [0, 1]])

H @ np.array([[1, 0, 1],
              [0, 1, 1]])
# [[1 1 2]
#  [0 1 1]]

# 输入:  e₁=(1,0) → (1, 0)     x 轴上的点不变
#        e₂=(0,1) → (1, 1)     y 轴方向被"扳"向斜
#        (1,1)   → (2, 1)
```

**几何效果**:**y 轴被"推"向 x 方向**——单位正方形变成平行四边形,**长方形不再是长方形**。

```
变换前(正方形):              变换后(平行四边形):
   y                              y
   ▲ ┌─┐                          ▲   ╱─────╲
 1 │ │ │                        1 │  ╱       ╲
   │ │ │                          │ ╱         ╲
   ┼─┴─┴─► x                      ┼─────────────► x
   0 1                            0 1         2
```

**工程含义**:**剪切是一种"耦合"——某些维度的输入污染了另一个维度**。**RNN 训练发散有时是因为权重矩阵在引入剪切,导致梯度方向倾斜**。

### 4.4 投影(Projection)

```python
P = np.array([[1, 0],
              [0, 0]])

P @ np.array([[3, 0, 5],
              [4, 2, 1]])
# [[3 0 5]
#  [0 0 0]]

# 输入:  e₁=(1,0) → (1, 0)    在 x 轴上的点不变
#        e₂=(0,1) → (0, 0)    在 y 轴上的点全压到 0
#        任意 (x, y) → (x, 0)  整个空间被压扁到 x 轴
```

**几何效果**:**二维空间被压扁成一维**——**信息丢失,不可逆**。

```
变换前:                       变换后:
   y                            y
   ▲                            ▲
   │                            │
   │  · · ·                     │
   │ · · · ·                    │
   ┼─────────► x                ┼·········► x
   0                            0   (所有点都压到 x 轴上)
```

**关键性质**:`P² = P`(投影两次等于一次)、`det(P) = 0`(压扁了)、`rank(P) < n`(秩亏)。

**工程含义**:**所有"低秩近似 / Dropout / 信息瓶颈"本质都是投影**——LoRA 的 `B @ A`(A 把 d 维压到 r 维,B 再升回去)就是一种近似投影,**只保留信息量最大的 r 个方向**。

### 4.5 反射(Reflection)

```python
F = np.array([[1, 0],
              [0, -1]])

F @ np.array([[1, 0, 2],
              [0, 1, 3]])
# [[1 0 2]
#  [0 -1 -3]]

# 输入:  e₁=(1,0) → (1, 0)    x 轴上点不变
#        e₂=(0,1) → (0,-1)    y 轴翻过来
#        任意 (x, y) → (x, -y)
```

**几何效果**:**沿 x 轴翻转**——**朝向反了**。

```
变换前:                       变换后:
   y                            y
   ▲ ↗                          ▲
   │↗  v=(2,3)                  │
   │                            │
   ┼─────► x                    ┼─────► x
   0                            │
                                │↘  Fv=(2,-3)
                                ▼
```

**关键性质**:`det(F) = -1`——**行列式负值,意味着方向反转**(右手系变左手系)。

### 4.6 五种变换的速查表

| 变换 | 矩阵 | det | 性质 | 工程对应 |
| --- | --- | --- | --- | --- |
| 缩放 | `diag(s₁,...,sₙ)` | s₁·s₂·...·sₙ | 沿各轴独立缩放 | RMSNorm、attention 的 1/√d、quantization scale |
| 旋转 | `[[cos,-sin],[sin,cos]]` | +1 | 保长度、保夹角 | RoPE 位置编码、坐标系变换 |
| 剪切 | `[[1,k],[0,1]]` | 1 | 一维"染色"另一维 | RNN 中权重的"耦合" |
| 投影 | `diag(1,...,1,0,...,0)` | 0 | 不可逆、压扁 | LoRA、PCA、低秩近似、Dropout |
| 反射 | `diag(1,...,-1)` | -1 | 方向反转 | 数据增强(镜像翻转) |

## 五、神经网络一层 = 仿射变换

**`y = Wx + b` 在工程里是 `nn.Linear`,在数学里是"仿射变换"**(线性 + 平移)。

### 5.1 仿射 = 线性 + 平移

```
线性变换:   y = W·x          (经过原点)
仿射变换:   y = W·x + b      (允许平移开原点)
```

**几何上**:**矩阵 W 扭一下空间,向量 b 再把整个东西平移**。

```
变换前:                变换后(只 W):           最后(W·x + b):
   y                        y                        y
   ▲ ┌─┐                    ▲   ╱─╲                  ▲       ╱─╲
   │ │ │                    │  ╱   ╲                 │      ╱   ╲
   ┼─┴─┴► x                 ┼─────────► x            ┼─────────────► x
                                                       └─ b 平移 ─┘
```

### 5.2 一个 `nn.Linear` 在干什么

```python
import torch.nn as nn

layer = nn.Linear(768, 1024)
# layer.weight: shape (1024, 768)   ← W
# layer.bias:   shape (1024,)        ← b
# 输入 x: (B, 768)
# 输出:   y = x @ W.T + b   shape (B, 1024)
```

**几何解读**:

- **W 把 768 维空间映射到 1024 维空间**——升维。
- **W 的每个列向量**(看 W.T 的列,= W 的行)是**输入空间的基向量在输出空间的"目的地"**。
- **W 的秩 ≤ min(768, 1024)**——理论上可以是 768,但训练后可能是任意小的有效秩(LoRA 论文里观察到很多 Linear 实际有效秩很低)。

**Transformer 里的 Linear 全是这套**:

```
W_Q: (d_model, d_k)         ← 把 token embedding 映射到 query 空间
W_K: (d_model, d_k)         ← 映射到 key 空间
W_V: (d_model, d_v)         ← 映射到 value 空间
W_O: (d_model, d_model)     ← 把 attention 输出投回原空间
FFN: (d_model, 4·d_model)   ← 升维(expansion)
     (4·d_model, d_model)   ← 降维回去
```

**每一个 W 都是一次几何变换**——把信息扭到对这一层有用的子空间里。**LoRA 的发现是**:**这些变换里有很多是低秩的**,可以用 `B @ A`(其中 A 是 r×d、B 是 d×r,r << d)近似。

### 5.3 激活函数:打破"只能拉伸"的限制

**纯仿射变换的局限**:**多层仿射 = 一层仿射**——`W₂(W₁x + b₁) + b₂ = (W₂W₁)x + (W₂b₁ + b₂)`,本质还是一次变换。**所以神经网络必须有非线性**(ReLU、GeLU、SiLU 等),**否则再深也只等价于一层 Linear**。

> **核心直觉**:**线性变换只能"均匀拉伸"空间——所有的"弯折"都来自激活函数**。**网络深度的价值,就是允许多次"弯折再拉伸"**——这就是为什么深层比浅层表达力强。

## 六、秩、零空间、列空间:几何直觉

这三个概念在课本里是定义,在工程里是**"信息流过这一层后剩下多少"**的描述。

### 6.1 秩(rank):变换后剩多少维

```
rank(A) = "A 把输入空间映射到的目标子空间的维度"
        = "A 列向量里独立的有几个"
```

**例**:

```
A = [ 1  2 ]               rank(A) = 1
    [ 2  4 ]               (第二列 = 2 倍第一列,只贡献一个独立方向)

A = [ 1  0 ]               rank(A) = 2
    [ 0  1 ]               (单位矩阵,满秩)

A = [ 1  0  0 ]            rank(A) ≤ 2
    [ 0  1  0 ]            (三维输入,但只能输出二维)
```

**直觉**:

| 矩阵形态 | rank | 几何 |
| --- | --- | --- |
| 满秩(full rank) | min(m, n) | 不丢信息,变换"满"(可能升维或保持) |
| 秩亏(rank deficient) | < min(m, n) | 信息丢失,空间被压扁 |
| 严重低秩(low rank) | r << min(m,n) | 只保留 r 个有效方向 |

### 6.2 零空间(null space):被压成 0 的方向

```
N(A) = { x : A·x = 0 }
     = "被 A 映射到原点的所有方向"
```

**例**:**投影矩阵 `P = diag(1, 0)`**:

```
P·(0, 1) = (0, 0)       ← (0, 1) 在 N(P) 里
P·(0, 5) = (0, 0)       ← (0, 5) 也在 N(P) 里
N(P) = y 轴
```

**几何含义**:**零空间就是"被这个变换抹掉的方向"**——这些方向上的信息进了变换就没了。**矩阵越"低秩",零空间越大,丢的信息越多**。

### 6.3 列空间(column space):变换的"值域"

```
C(A) = "A·x 能取到的所有值"
     = "A 的列向量张成的子空间"
```

**例**:

```
A = [ 1  2 ]
    [ 2  4 ]

A 的列向量:(1, 2), (2, 4)    都在 y = 2x 这条直线上
→ C(A) = y = 2x 这条直线(一维)
```

**几何含义**:**列空间就是"变换的输出能到达的子空间"**——A·x 永远落在 C(A) 里,**外面的点 A 怎么也变不出来**。

### 6.4 三个的关系:维度守恒(rank-nullity)

```
dim(N(A)) + dim(C(A)) = n   (输入空间的维度)

     │              │
     │              └─ 经过变换"存活"下来的维度 = rank(A)
     └─ 被变换"抹掉"的维度
```

**直觉**:**输入有 n 维信息,变换后一部分进 C(A)(被保留),另一部分进 0(被抹掉)——加起来还是 n**。

> **核心直觉**:**rank 是"信息保留量"的度量**。**LoRA 的发现**:大模型很多层的"信息"实际只占很低的有效秩——所以可以用 `r=8` 的瓶颈近似 `d=4096` 的全秩矩阵。**这是数学上为什么 LoRA work 的根本**。

## 七、行列式:体积变化倍数

### 7.1 几何意义

```
det(A) = "A 把单位立方体变成的体积"
       = "A 把任何体积放缩的倍数"
```

**对 2×2 矩阵**:**det 是单位正方形被 A 变换后的面积(带符号)**。

```
A = [ 2  0 ]                 单位正方形(面积 1)
    [ 0  3 ]                 → 2×3 长方形(面积 6)
                             det(A) = 6  ✓

A = [ cos θ  -sin θ ]        旋转
    [ sin θ   cos θ ]        det = cos²θ + sin²θ = 1  ✓ (不变面积)

A = [ 1  0 ]                 投影到 x 轴
    [ 0  0 ]                 单位正方形被压成线段(面积 0)
                             det = 0  ✓

A = [ 1   0 ]                沿 x 轴翻转
    [ 0  -1 ]                面积不变,但方向反转
                             det = -1  ✓
```

### 7.2 三个关键事实

| det 值 | 几何含义 | 代数含义 |
| --- | --- | --- |
| det > 0 | 保持方向,体积放缩 \|det\| 倍 | 可逆 |
| det < 0 | 反转方向(右手系变左手系) | 可逆 |
| det = 0 | 把空间压扁,体积变 0 | **不可逆**(singular) |
| \|det\| > 1 | 放大体积 | — |
| \|det\| < 1 | 缩小体积 | — |

### 7.3 为什么 det = 0 不可逆

```
det = 0 → 空间被压扁(rank 不满)
        → 多个不同的 x 被映射到同一个 y
        → 给我一个 y,我说不出"是哪个 x 来的"
        → 没有逆函数
```

**工程上的反映**:`np.linalg.inv(A)` 在 `det(A) = 0` 时会报 `singular matrix`。**实际工程里我们更怕的不是 det=0,是 det 很小**——下一节讲。

## 八、矩阵的逆与病态矩阵

### 8.1 解 Ax = b:什么时候有唯一解

```
A·x = b

有唯一解   ⇔   A 可逆(det ≠ 0,满秩)
无解       ⇔   b 不在 A 的列空间里
无穷多解   ⇔   A 不满秩,但 b 在列空间里
```

**实际工程里几乎只关心"唯一解",所以遇到 Ax=b 第一件事是看 A 满秩不**。

```python
import numpy as np

# 满秩,有唯一解
A = np.array([[1., 2.], [3., 4.]])
b = np.array([5., 11.])
x = np.linalg.solve(A, b)   # [1., 2.]

# 奇异(det=0),报错
A_bad = np.array([[1., 2.], [2., 4.]])
np.linalg.solve(A_bad, b)   # LinAlgError: Singular matrix
```

### 8.2 条件数(condition number):病态矩阵

**"病态"= A 不是奇异,但接近奇异**——`det` 不是 0,但很小。**这时数值上很难求逆**——一点小扰动会让结果差很多。

```
condition number κ(A) = σ_max(A) / σ_min(A)
                      = "最大奇异值 / 最小奇异值"
                      = "最被拉长的方向 vs 最被压扁的方向 的比"
```

**直觉**:

| κ(A) | 状态 | 数值行为 |
| --- | --- | --- |
| 1 | 完美正交 | 数值稳定 |
| 10-100 | 良好 | 小扰动小误差 |
| 1000-10⁶ | 略病态 | 注意误差累积 |
| 10⁶-10¹² | 病态 | 不要直接求逆 |
| > 10¹² | 数值上的奇异 | 当奇异处理 |

```python
A = np.array([[1, 0.999],
              [0.999, 1]])
np.linalg.cond(A)    # ≈ 2000   略病态

# 看一下扰动效应
b = np.array([1, 0.998])
x1 = np.linalg.solve(A, b)                      # [1, 0]
b_perturbed = b + 1e-5 * np.random.randn(2)
x2 = np.linalg.solve(A, b_perturbed)            # 可能差挺远
```

### 8.3 工程上不要直接 inv(A) 解方程

```python
# ❌ 报警:数值不稳,且慢
x = np.linalg.inv(A) @ b

# ✓ 正确:用 solve(),底下用 LU / Cholesky / QR 分解,更稳更快
x = np.linalg.solve(A, b)

# ✓ 对最小二乘:用 lstsq()
x, residuals, rank, sv = np.linalg.lstsq(A, b, rcond=None)
```

**这是数值线代铁律**——**有解方程的专门 API,永远不要先求逆再乘**。

### 8.4 Hessian 条件数与训练动力学

**这是个工程师最该知道的"condition number"应用场景**:

- **损失函数的 Hessian H = ∇²L**(二阶导数矩阵)描述了 loss 曲面的曲率。
- **Hessian 的最大特征值 / 最小特征值 = condition number**——大,意味着 loss 曲面像"窄长峡谷",一些方向陡、一些方向平。
- **梯度下降在病态 Hessian 下走 zig-zag**——这就是为什么需要 momentum / Adam(它们等价于对 Hessian 做某种近似预条件)。

> **核心直觉**:**Adam 在干的事就是"按每个维度的曲率,自适应调步长"——等价于把 Hessian 的 condition number 拉小**。**这是 23 篇会展开的事**,但矩阵层面的直觉在这里。

## 九、工程对应:Conv / Attention / 图像增强

### 9.1 Conv 是局部线性变换

**卷积核 = 一个小矩阵 + 在图像上滑动**——**每个位置都做一次同样的线性变换**(参数共享)。

```python
import torch.nn as nn

conv = nn.Conv2d(3, 64, kernel_size=3, stride=1, padding=1)
# 输入: (B, 3, H, W)
# 输出: (B, 64, H, W)
# 参数: 3×64×3×3 + 64 = 1792 个
```

**每个输出像素的计算**:

```
y[i, j] = Σ Σ Σ  W[c_out, c_in, k_h, k_w] · x[c_in, i + k_h, j + k_w]  +  b
        c_in k_h k_w

       = 一个 (3·3·3) → 64 的小 Linear 在每个位置 (i, j) 上跑一遍
```

**几何含义**:**Conv 是"输入图像每个 3×3 邻域被 64 个不同的 3×3 探针(每个对应一个输出通道)做内积,每次得到一个数"**。**这 64 个探针就是 64 个不同方向的线性"特征探测器"**——边缘探测、颜色变化、纹理。

**im2col 技巧**:**Conv 在底层被改写成大 GEMM**(见 07 篇)——把所有 3×3 邻域展开成列,然后 `output = W @ patches`,**Conv 就是一个特殊数据布局的大矩阵乘**。

### 9.2 Attention 是"先三种变换再做匹配"

```python
Q = X @ W_Q          # 把每个 token "变成 query 形态"
K = X @ W_K          # "变成 key 形态"
V = X @ W_V          # "变成 value 形态"
```

**每个矩阵的几何含义**:

| 矩阵 | 几何 |
| --- | --- |
| `W_Q` | 把原始 embedding **扭到"提问空间"**——每个 token "想问什么" |
| `W_K` | 扭到"被检索空间"——每个 token "能回答什么" |
| `W_V` | 扭到"内容空间"——每个 token "实际带的信息" |
| `Q @ K.T` | 在 query 空间和 key 空间做内积,**算"谁回答谁"** |
| `W_O` | 把 attention 结果**扭回原空间**,以便后续层用 |

**关键洞察**:**多头 Attention 的每个 head 学到不同的 `W_Q W_K W_V`,等于"多个不同视角的提问/检索"——每个 head 在不同子空间里找关系**。

### 9.3 仿射变换在图像增强里

```python
import cv2
import numpy as np

# 仿射变换矩阵(2×3):
# [a  b  tx]      x' = a·x + b·y + tx
# [c  d  ty]      y' = c·x + d·y + ty

# 旋转 30°,放大 1.5 倍,中心 (cx, cy)
M = cv2.getRotationMatrix2D(center=(cx, cy), angle=30, scale=1.5)
img_aug = cv2.warpAffine(img, M, (W, H))

# 平移
M = np.float32([[1, 0, 50],     # x 方向移 50 像素
                [0, 1, 30]])    # y 方向移 30 像素
img_aug = cv2.warpAffine(img, M, (W, H))

# 透视变换(更广义,3×3 矩阵,允许"非线性")
M = cv2.getPerspectiveTransform(src_4pts, dst_4pts)
img_aug = cv2.warpPerspective(img, M, (W, H))
```

**几何含义**:

- **2×3 仿射矩阵 = 线性 2×2(扭一下)+ 平移**(b 向量)
- **3×3 透视矩阵 = 仿射 + 远近变形**——直线还是直线,但平行线不再平行
- **数据增强(rotate / scale / shear / flip)全是仿射变换的特例**

## 十、一份"看到这种代码就报警"的清单

```python
# ❌ 报警 1:直接 inv 解方程
x = np.linalg.inv(A) @ b                # 慢、不稳
# ✓ 修复
x = np.linalg.solve(A, b)

# ❌ 报警 2:不检查 condition number 就批量求逆
A_inv = np.linalg.inv(huge_batch_of_matrices)   # 病态时静默出 NaN
# ✓ 修复:先看 cond
if np.linalg.cond(A) > 1e10:
    print("Warning: ill-conditioned")

# ❌ 报警 3:把 LoRA 的 rank 当无脑超参
lora = LoRA(d=4096, r=64)               # r 大,但 trainable 参数翻倍
# ✓ 思路:r=8 是因为"原始权重的有效秩接近 8",不是越大越好

# ❌ 报警 4:模型加深但没加非线性
net = nn.Sequential(nn.Linear(d, d), nn.Linear(d, d))  # 等价一层
# ✓ 修复
net = nn.Sequential(nn.Linear(d, d), nn.ReLU(), nn.Linear(d, d))

# ❌ 报警 5:行列式判可逆,float 误差导致判错
if np.linalg.det(A) == 0:               # 几乎永远不会精确等于 0
    return None
# ✓ 修复:看条件数或秩
if np.linalg.cond(A) > 1e12:
    return None

# ❌ 报警 6:把矩阵当数表传给 torch,数据排布反了
weights = torch.tensor(np.random.randn(out, in))   # 没问题
# 但底层调用方对 (in, out) vs (out, in) 不敏感时会静默错
# ✓ 检查:nn.Linear 的 weight.shape 是 (out, in),不是 (in, out)

# ❌ 报警 7:旋转矩阵手写没考虑数值漂移
R = R @ small_rotation                  # 迭代多次后 det 不再是 1
# ✓ 修复:周期性重新正交化(SVD 或 QR)
U, _, Vt = np.linalg.svd(R)
R = U @ Vt                              # 强制 R ∈ SO(n)
```

### 10.1 这一层的局限

**矩阵作为线性变换的边界**:

- **只能描述线性结构**——所有"弯折、阈值、决策"必须靠激活函数 / 非线性层。
- **维度过高时几何直觉失效**——2D / 3D 画图清晰,1000 维你脑子里画不出来,**只能靠 rank / 特征值 / SVD 抓住"结构"**(详见 08-09 篇)。
- **数值稳定性独立于代数性质**——`det ≠ 0` 不代表可逆"在数值上没问题",condition number 才是真指标。
- **稀疏矩阵 / 大矩阵的存储和计算另一套**——LLM 里 W 都是密集大矩阵,真要做稀疏(剪枝、MoE)是另一套 trick。

### 10.2 工程层 / 数学层 / API 速查

| 操作 | 数学 | NumPy | PyTorch |
| --- | --- | --- | --- |
| 矩阵乘 | A·B | `A @ B` | `A @ B` |
| 转置 | Aᵀ | `A.T` | `A.T` 或 `A.transpose(-1,-2)` |
| 逆 | A⁻¹ | `np.linalg.inv(A)` | `torch.linalg.inv(A)` |
| 解 Ax=b | x = A⁻¹b | `np.linalg.solve(A, b)` | `torch.linalg.solve(A, b)` |
| 最小二乘 | argmin ‖Ax-b‖² | `np.linalg.lstsq(A, b)` | `torch.linalg.lstsq(A, b)` |
| 行列式 | det(A) | `np.linalg.det(A)` | `torch.linalg.det(A)` |
| 秩 | rank(A) | `np.linalg.matrix_rank(A)` | `torch.linalg.matrix_rank(A)` |
| 条件数 | κ(A) | `np.linalg.cond(A)` | `torch.linalg.cond(A)` |
| 零空间 | N(A) | `scipy.linalg.null_space(A)` | 用 SVD 算 |
| 仿射变换 | y=Wx+b | `x @ W.T + b` | `nn.Linear(in, out)(x)` |
