# 链式法则与反向传播

上一篇讲了梯度是什么。**这一篇讲怎么算它**——具体到一个 50 层神经网络里、loss 对第 3 层某个权重的偏导,机器是怎么"机械地"算出来的。**这个机械过程就是反向传播(BP),就是 autograd 的全部内核**。

> 一句话先记住:**反向传播 = 链式法则在计算图上的拓扑序应用**。前向算数值并缓存激活、反向从 loss 出发逆拓扑序传梯度,**每个算子只需要知道自己怎么把"上游 grad"变成"下游 grad"——它不需要知道整张网络长什么样**。autograd 库的全部魔法,就是在运行时记下这张图、然后按顺序调每个算子的 `.backward()`。

---

## 一、为什么这篇必须存在

aiLearning 06 篇讲了 BP 是"loss 反向传梯度",但**没把那个"传"字拆开**——很多人能写 `loss.backward()` 但说不清:

- "为什么 LSTM 比 RNN 不容易梯度消失"
- "gradient checkpointing 为什么是时间换空间"
- "为什么 Transformer 必须有 residual connection"
- "TF 静态图和 PyTorch 动态图本质差在哪儿"

**这些问题的答案,全都在 BP 这套机制里**。这一篇把它讲透。

| 撞过的坑 | 不懂时 | 懂了之后 |
| --- | --- | --- |
| 梯度突然 NaN | 怀疑数据 / 学习率 | 知道是某层激活值过大 / 链式相乘失控 |
| OOM 挂在反向时 | 不理解为什么前向能跑、反向不行 | 知道反向要存所有中间激活 |
| `detach()` 和 `no_grad` 的区别 | 看文档也说不清 | 知道前者切断该节点回传、后者整段不建图 |
| 50 层网络梯度消失 | 试着加层数 | 知道是雅可比连乘的特征值问题、要 skip connection |

## 二、链式法则:剥洋葱

### 2.1 单变量

复合函数 `y = f(g(x))`,导数:

```
dy/dx = f'(g(x)) · g'(x)
```

**直觉**:外层导 × 内层导。**像剥洋葱,从外往里一层一层乘**。

```
y = sin(x²)
  外层 f(u) = sin(u),  f'(u) = cos(u)
  内层 g(x) = x²,      g'(x) = 2x

dy/dx = cos(x²) · 2x
```

### 2.2 多层链式(深度网络的关键)

更长的复合 `y = f₁(f₂(f₃(...fₙ(x))))`:

```
dy/dx = f₁'(·) · f₂'(·) · f₃'(·) · ... · fₙ'(x)
```

**每多一层,多乘一个导数**——这件事在多变量下变成"多乘一个雅可比矩阵"。

### 2.3 多变量链式 = 雅可比连乘

```
x ──→ h₁ ──→ h₂ ──→ h₃ ──→ y

       g₁     g₂     g₃     g₄
       
∂y/∂x = J_g₄ · J_g₃ · J_g₂ · J_g₁
        (从输出端开始,逆向乘)
```

**这就是反向传播在数学上的全部内容**。

> **核心直觉**:**深度网络的梯度 = 一长串雅可比矩阵的乘积**。这个连乘的"积分"性质,正是梯度爆炸 / 消失的根源——后面 9.3 节会展开。

## 三、计算图:把链式法则画出来

### 3.1 节点 = 操作,边 = 数据

把任何复杂表达式拆成基本算子,画成 DAG(有向无环图)。

### 3.2 一个最简单的 2 层 MLP 计算图

```
公式:
    z₁ = W₁ · x + b₁
    h₁ = ReLU(z₁)
    z₂ = W₂ · h₁ + b₂
    ŷ  = softmax(z₂)
    L  = CrossEntropy(ŷ, y)

计算图(前向):

   x ─┐
       \           
        ●  matmul ──→ z₁ ──→  ●  ReLU ──→ h₁ ─┐         
       /                                       \         
   W₁ ─┘                                        ●  matmul ──→ z₂ ──→  ●  softmax ──→ ŷ ─┐
                                               /                                          \
                                          W₂ ─┘                                            ●  CE ──→ L
                                                                                          /
                                                                                       y ─┘
   
   b₁(加在 z₁ 上,省略)        b₂(加在 z₂ 上,省略)


反向传播(从 L 逆向走,每个算子调自己的 backward):

   ← x.grad ←┐
              \  
               ●  matmul.backward ←── grad_z₁ ←── ●  ReLU.backward ←── grad_h₁ ←┐                  
              /                                                                   \                
   ← W₁.grad ─                                                                     ●  matmul.backward ←── grad_z₂ ←── ●  softmax+CE.backward ← grad_L = 1
                                                                                  /
                                                                          ← W₂.grad ─

   每个算子: 拿到上游 grad、用前向时缓存的输入、算出"对自己输入的 grad",传给下游
```

### 3.3 三类节点

| 类型 | 例子 | 反向时干什么 |
| --- | --- | --- |
| 叶子节点(参数) | W₁, W₂, b₁, b₂ | **接收梯度,存进 `.grad`,不再往下传** |
| 中间节点(运算) | matmul, ReLU, softmax | **算上游 grad → 下游 grad,继续往下传** |
| 输入节点(数据) | x, y | 通常不需要梯度(`requires_grad=False`) |

> **PyTorch 概念**:**只有叶子节点才存 `.grad`**——中间节点的梯度算完就丢(除非你 `retain_grad()`)。这就是为什么你 print `(W @ x).grad` 总是 None。

## 四、前向 vs 反向:数据流和缓存

### 4.1 前向(forward)

```
1. 按计算图拓扑序,从输入到输出,逐个算子计算数值
2. 每个算子算完输出后,缓存中间结果(为反向用)
3. 最后一个算子的输出 = 标量 loss
```

**缓存了什么**?**反向时用得着的中间量**——通常是:

| 算子 | 前向缓存 | 反向用它算什么 |
| --- | --- | --- |
| matmul `z = W·x` | `x`(因为 `dz/dW = x.T`) | grad_W |
| ReLU `y = max(0, x)` | mask `x > 0` | 把 grad 在负位置归零 |
| softmax `y = softmax(x)` | 输出 `y` 本身 | 用 `y(1-y)` 计算梯度 |
| sigmoid | 输出 `y` | `y(1-y)` |
| LayerNorm | 均值、方差、归一化结果 | 链式展开后用 |

### 4.2 反向(backward)

```
1. 从 loss 出发,grad_loss = 1.0(标量对自己求导是 1)
2. 按计算图的逆拓扑序遍历
3. 每个算子调用自己的 backward 函数:
     输入: 上游传来的 grad
     输出: "对自己输入的 grad",传给上游(图里的下游)
4. 走到叶子节点(参数),把累加的 grad 写进 .grad
```

**关键**:**每个算子只关心自己的局部导数**(雅可比),它**不知道整张网络是什么样**——这就是 autograd 能"自动"工作的原因。**整体的链式法则被分摊到了每个算子的局部 backward 里**。

```
对 matmul 算子 z = W · x:
  上游传来 grad_z  (shape 同 z)
  本算子的局部雅可比: ∂z/∂W = x.T,  ∂z/∂x = W.T
  → grad_W = grad_z · x.T   (链式法则)
  → grad_x = W.T · grad_z   (链式法则)
  把 grad_W 累加到 W.grad、grad_x 传给上游
```

## 五、矩阵微积分:5 个最常用公式

不证明,直接给——**前向时认得它们,反向时知道形状对不对**。

| 表达式 | 对哪个求导 | 结果 |
| --- | --- | --- |
| `y = W·x` | `∂y/∂x` | `W.T` |
| `y = W·x` | `∂y/∂W` | `x.T`(实际形状要看上下文) |
| `y = x.T · A · x` | `∂y/∂x` | `(A + A.T) · x`(A 对称时为 `2Ax`) |
| `y = ‖x‖²` | `∂y/∂x` | `2x` |
| `y = softmax(x)` | `∂y_i/∂x_j` | `y_i(δ_ij - y_j)`(实际反向有简化) |

**特殊**:**softmax + cross-entropy** 一起反向时**奇迹般地简化**——把它们当一个算子,反向输出就是 `softmax(x) - y_true`(one-hot 标签)。这就是为什么 PyTorch 的 `CrossEntropyLoss` 内部把 softmax 也包了进去:既数值稳定(log-sum-exp 技巧),又反向极简。

> **避坑**:**实战中不要靠记这些公式手推梯度**——`autograd` / `jax.grad` 自动算就好。**这些公式的用处是: 排错时心算 shape,以及读论文里的反向公式时认得它们**。**靠手推会写错,只有 autograd 不会**。

## 六、手写 2 层 MLP 的 BP(NumPy)

把上面所有概念跑一遍——**纯 NumPy,不依赖 autograd,30 行做完前向 + 反向,然后用 PyTorch 验证一致**。

```python
import numpy as np
import torch
import torch.nn.functional as F

np.random.seed(0)
torch.manual_seed(0)

# 数据(2 个样本、3 维输入、3 类)
N, D_in, H, D_out = 2, 3, 4, 3
x_np = np.random.randn(N, D_in).astype(np.float32)
y_np = np.array([0, 2])  # 类别标签

W1_np = np.random.randn(D_in, H).astype(np.float32) * 0.5
b1_np = np.zeros(H, dtype=np.float32)
W2_np = np.random.randn(H, D_out).astype(np.float32) * 0.5
b2_np = np.zeros(D_out, dtype=np.float32)

# ============== 前向(缓存中间量)==============
z1 = x_np @ W1_np + b1_np                # (N, H)
h1 = np.maximum(0, z1)                   # ReLU
z2 = h1 @ W2_np + b2_np                  # (N, D_out)
# softmax + cross-entropy(数值稳定)
z2_shift = z2 - z2.max(axis=1, keepdims=True)
exp_z2 = np.exp(z2_shift)
probs = exp_z2 / exp_z2.sum(axis=1, keepdims=True)
loss = -np.log(probs[np.arange(N), y_np]).mean()

# ============== 反向(逆拓扑序)==============
# softmax + CE 的合并梯度: probs - one_hot(y)
grad_z2 = probs.copy()
grad_z2[np.arange(N), y_np] -= 1
grad_z2 /= N                             # 因为前面取了 mean

# matmul z2 = h1 @ W2 + b2
grad_W2 = h1.T @ grad_z2                 # (H, D_out)
grad_b2 = grad_z2.sum(axis=0)            # (D_out,)
grad_h1 = grad_z2 @ W2_np.T              # (N, H)

# ReLU h1 = max(0, z1)
grad_z1 = grad_h1 * (z1 > 0)             # mask

# matmul z1 = x @ W1 + b1
grad_W1 = x_np.T @ grad_z1               # (D_in, H)
grad_b1 = grad_z1.sum(axis=0)

# ============== 用 PyTorch autograd 验证 ==============
W1 = torch.tensor(W1_np, requires_grad=True)
b1 = torch.tensor(b1_np, requires_grad=True)
W2 = torch.tensor(W2_np, requires_grad=True)
b2 = torch.tensor(b2_np, requires_grad=True)
x = torch.tensor(x_np)
y = torch.tensor(y_np)

z1_t = x @ W1 + b1
h1_t = F.relu(z1_t)
z2_t = h1_t @ W2 + b2
loss_t = F.cross_entropy(z2_t, y)
loss_t.backward()

# 对比
print("loss 对得上:    ", np.allclose(loss, loss_t.item(), atol=1e-5))
print("grad_W1 对得上: ", np.allclose(grad_W1, W1.grad.numpy(), atol=1e-5))
print("grad_W2 对得上: ", np.allclose(grad_W2, W2.grad.numpy(), atol=1e-5))
print("grad_b1 对得上: ", np.allclose(grad_b1, b1.grad.numpy(), atol=1e-5))
print("grad_b2 对得上: ", np.allclose(grad_b2, b2.grad.numpy(), atol=1e-5))

# 输出:
# loss 对得上:     True
# grad_W1 对得上:  True
# grad_W2 对得上:  True
# grad_b1 对得上:  True
# grad_b2 对得上:  True
```

**这 30 行就是 PyTorch / TensorFlow / JAX 内部对一个 2 层 MLP 做的事**。它们的 autograd 只是把这个流程**自动化、向量化、GPU 化、对任意计算图泛化**而已。

> **核心直觉**:**反向那段代码,是从 loss 出发、按"每个算子的 backward"逆向走的——这个顺序就是计算图的逆拓扑序**。每个算子的 backward 长什么样?**它就是该算子局部雅可比的形状变换**。把这个想清楚,你以后看任何 autograd 实现都不再神秘。

## 七、autograd 实现:动态图 vs 静态图

### 7.1 两种范式

| 范式 | 代表 | 怎么建图 |
| --- | --- | --- |
| **动态图(Define-by-Run)** | PyTorch / Chainer | **运行 Python 时实时记录算子,跑完就有图** |
| **静态图(Define-and-Run)** | TF 1.x / Theano | **先用 API 描述完整张图,再 `sess.run` 跑** |
| **混合(JIT/Trace)** | JAX / TF 2.x / `torch.compile` | **第一次跑时记图,后面复用** |

### 7.2 PyTorch autograd 是怎么工作的

```
1. 创建 tensor 时,如果 requires_grad=True,
   它有个 .grad_fn 字段(指向"创建它的算子"的 backward 实现)

2. 每次算 z = op(x, y) 时,
   PyTorch 后台:
     - 算出 z 的数值
     - 给 z 配一个 grad_fn,挂上对 x, y 的指针(图的边)
     - 缓存反向需要的中间量(在 grad_fn 的 ctx 里)

3. loss.backward() 时:
     - 从 loss 的 grad_fn 出发,递归调用每个 grad_fn 的 backward
     - 按图逆拓扑序传 grad
     - 走到叶子节点时,累加到 leaf.grad
```

**关键:整张图是 `loss.backward()` 跑完之后默认就被释放的**——下一次 forward 重新建。**这就是"动态图"**——每一次 iteration 的图可以完全不一样(支持条件分支、变长循环)。

```python
# 每个 iter 图都不一样,Python 的控制流直接生效
for x in data:
    if x.sum() > 0:
        y = x @ W1 @ W2          # 这一 iter 用 2 层
    else:
        y = x @ W1 @ W2 @ W3     # 这一 iter 用 3 层
    loss = ((y - target) ** 2).sum()
    loss.backward()              # 都能正常 BP
```

**TF 1.x 静态图做不到这个**——你得用 `tf.cond` / `tf.while_loop` 写专门的图算子。这就是 PyTorch 当年从研究界压过 TF 1.x 的根本原因。

### 7.3 trade-off

| 方面 | 动态图 (PyTorch) | 静态图 (TF 1.x) |
| --- | --- | --- |
| 写代码自由度 | 高(任意 Python 控制流) | 低 |
| 调试 | 直接 print / breakpoint | 难 |
| 优化空间 | 小(每次都重建) | 大(可全局优化) |
| 部署 | 需要导出 / TorchScript | 直接序列化图 |

**JAX / `torch.compile` / `torch.fx` 是中间路线**:**第一次跑做 trace 拿到图,后面复用 + 编译优化**——保留写法的自由,又能享受编译收益。**这是 2024-2025 年 LLM 训练的主流方案**(`torch.compile()` 一行代码加速 30-50%)。

## 八、内存代价与 gradient checkpointing

### 8.1 反向传播的"内存帐"

**反向需要前向所有中间激活**——这是 BP 的硬性代价。

```
单层 Transformer block 的激活内存(粗算):
  attention 输出: (B, S, D)
  FFN 中间:      (B, S, 4D)         ← 这层最大
  LayerNorm 中间: (B, S, D) × 2
  总计 ≈ (B, S, D) × 8 一层

GPT-3 175B(L=96 层, D=12288, S=2048, B=1):
  一层激活 ≈ 1 × 2048 × 12288 × 8 × 2 (fp16) ≈ 400 MB
  全部 96 层 ≈ 38 GB
  + 模型参数 (175B × 2 fp16) ≈ 350 GB
  + Adam 状态 (×2) ≈ 700 GB
  → 训练 GPT-3 不靠多种技术叠加根本放不下
```

**"激活占的内存随 batch × seq × depth 线性增长"——这是大模型训练显存墙的主因之一**。

### 8.2 Gradient Checkpointing(时间换空间)

**思路**:**前向时不缓存所有中间激活,只在每隔几层"打卡点"。反向时,用最近的 checkpoint 重新前向算一遍丢失的激活,再反向**。

```
普通 BP:
  forward:   x → h₁ → h₂ → h₃ → ... → h_n → loss     (全部缓存)
  backward:  ←  ←  ←  ←  ←  ←  ←  ←  ←  ←
  
  内存:O(n)  时间:1× forward + 1× backward

Gradient Checkpointing(每 √n 层一个 ckpt):
  forward:   x → [缓存 ckpt₁] → ... → [缓存 ckpt₂] → ... → loss
                  ╳ 中间不缓存       ╳ 中间不缓存
  backward:  到了某段,先从 ckpt 重做 forward 算回中间激活,再反向
  
  内存:O(√n)  时间:2× forward + 1× backward
```

**典型数据**:**显存降 5-10×、训练时间增加 30-40%**。LLM 训练几乎默认开。

```python
# PyTorch 原生支持
from torch.utils.checkpoint import checkpoint

class BlockCheckpointed(nn.Module):
    def __init__(self, block):
        super().__init__()
        self.block = block
    def forward(self, x):
        return checkpoint(self.block, x, use_reentrant=False)
```

> **经验法则**:**LLM 训练显存不够 → 先试 gradient checkpointing,基本能塞下大一截**。**HuggingFace Trainer / DeepSpeed / FSDP 都内置了开关**(`gradient_checkpointing=True`)。

### 8.3 为什么 Adam 不需要存 Hessian

上篇讲了 Hessian 是 N×N(N = 参数数),完全存不下。

**Adam 的精彩在于**:**它不存矩阵,只存两个和参数同 shape 的"对角线近似"**:

```
Adam 内存:
  m_t  (一阶矩,梯度的指数移动平均):  shape = (N,)
  v_t  (二阶矩,梯度平方的指数移动平均): shape = (N,)
  
更新:
  θ ← θ - lr · m_t / (√v_t + ε)
  
  v_t 起到"对角 Hessian 倒数"的近似作用 — 给每个维度自适应步长
  → 病态曲面 zigzag 大幅缓解
  → 内存只多了 2× 参数(Adam),AdamW 同
```

**这就是 Adam 一统江湖的核心**:**用 O(N) 的代价拿到接近二阶方法的好处**。详见 23 篇。

## 九、链式连乘的代价:梯度爆炸 / 消失

### 9.1 数学根源

深度网络的梯度 = 一长串雅可比矩阵的乘积。设每层雅可比谱半径(最大特征值绝对值)是 σ:

```
‖梯度‖ ≈ σ^L  (L = 层数)

σ > 1:  ‖梯度‖ → ∞    → 梯度爆炸 (NaN)
σ < 1:  ‖梯度‖ → 0    → 梯度消失 (学不动)
σ ≈ 1:  正常
```

**RNN 的灾难**:**它在序列长度 T 上展开成一张 T 层的网,每个时间步用同一组权重 W**——所以 σ = `‖W‖` 的连乘 = `σ^T`,**T 一大就爆 / 消**。

### 9.2 LSTM / Transformer 怎么缓解

| 机制 | 直觉 | 数学效果 |
| --- | --- | --- |
| **LSTM 的 cell state** | 一条"信息高速公路",梯度沿它直接走 | 跨时间步的梯度走加性路径而不是乘性 |
| **Residual connection (ResNet)** | `y = x + f(x)` 让梯度直接走"+x"那条捷径 | 梯度有一条"= 1"的旁路,不会被 f 的雅可比衰减 |
| **LayerNorm / BatchNorm** | 把激活分布拉回 mean 0 / var 1 | 让每层雅可比的谱保持在 σ ≈ 1 附近 |
| **梯度裁剪** | `if ‖grad‖ > C: grad *= C / ‖grad‖` | 直接断开爆炸,不解决病根但有用 |
| **优秀初始化(He / Xavier)** | 让初始时 σ ≈ 1 | 训练前期不爆不消 |

**Transformer 的稳定训练 = (Residual + LayerNorm + 优秀初始化 + 梯度裁剪)四件套缺一不可**。把哪一个去掉,深度过 50 层都基本训不起来——你试一下就知道。

```
Residual connection 的梯度账:

  普通深网:        x ──→ f₁ ──→ f₂ ──→ ... ──→ y
                  ∂y/∂x = J_fL · J_fL-1 · ... · J_f1   ← 容易爆/消

  Residual:        x ──→ (x + f₁) ──→ (· + f₂) ──→ ... ──→ y
                  ∂y/∂x = (I + J_fL) · ... · (I + J_f1)
                  
                  → 即使 J_f 全是 0,梯度仍然是 I · I · ... = I
                  → 即"什么都不学"也至少能让梯度无衰减地传到底
```

**这就是 ResNet 让"训 100 层网"成为可能的原因**——不是它学得更好,而是**它至少给了梯度一条不会衰减的路**。

### 9.3 真实场景里的诊断

| 现象 | 多半原因 | 处理 |
| --- | --- | --- |
| 训练几 step 后 loss = NaN | 梯度爆炸 | 加梯度裁剪 `clip_grad_norm_(params, 1.0)` |
| 训练几 epoch 都不动 | 梯度消失 / 死 ReLU | 加 LayerNorm / 换 GELU/SiLU / 调初始化 |
| 浅层学不动、深层在学 | 梯度消失到浅层 | 加 residual / 检查是否漏了 LayerNorm |
| LSTM 长序列学不到长程依赖 | 梯度仍在某些方向衰减 | 换 Transformer / 加 attention |

## 十、工程对应与局限

### 10.1 API 速查

| 任务 | PyTorch | JAX | TF |
| --- | --- | --- | --- |
| 标量 loss 反向 | `loss.backward()` | `jax.grad(loss_fn)(params)` | `with tf.GradientTape() as tape: ...` |
| 算梯度但不存 | `torch.autograd.grad(loss, params)` | `jax.grad` 默认就这样 | `tape.gradient(loss, vars)` |
| 整段 no_grad | `with torch.no_grad():` | 不需要(纯函数式) | 默认就是 |
| 切断某节点 | `x.detach()` | `jax.lax.stop_gradient(x)` | `tf.stop_gradient(x)` |
| Gradient Checkpointing | `torch.utils.checkpoint.checkpoint` | `jax.checkpoint` | `tf.recompute_grad` |
| 梯度裁剪 | `clip_grad_norm_(params, max_norm)` | `optax.clip_by_global_norm` | `tf.clip_by_global_norm` |
| JIT 编译加速 | `torch.compile(model)` | `@jax.jit` | `@tf.function` |
| 高阶导(对梯度再求导) | `create_graph=True` | `jax.grad(jax.grad(f))` | 嵌套 GradientTape |

### 10.2 三个常见混淆

| `detach()` | 切断**这一个 tensor** 与计算图的连接(后面对它的运算不再回传到它来源) |
| --- | --- |
| `with torch.no_grad():` | 整段代码块**不建图**(显存省一大半,推理时必用) |
| `requires_grad=False` | 这个 tensor **本身不参与梯度**(但通过它做的运算还是建图,只是它不接收 grad) |

```python
# 推理(整段不建图,显存最省)
with torch.no_grad():
    output = model(x)

# 推理 + 不需要梯度但要参与图(罕见,如冻结某层)
for p in frozen_layer.parameters():
    p.requires_grad = False

# 把某个中间量"冻结"(GAN / RL 里常用)
fake_score = D(G(z).detach())   # G 这边不要回传
```

### 10.3 这套机制不能解决什么

| autograd 答得好 | autograd 答不好 |
| --- | --- |
| 任意可微表达式的梯度 | 离散变量的梯度(要 Gumbel-Softmax / REINFORCE) |
| 任意深度的链式 | 随机过程 / 控制流里的梯度(要 RL / 策略梯度) |
| 高阶导(要 `create_graph=True`) | 不可微算子(argmax / 排序 / 采样)需手动近似 |
| 标量输出对参数求 grad | 多输出对参数求 grad(每个都要单独 backward) |

**RLHF / VQ-VAE / GAN 里的"棘手梯度"** 都是 autograd 答不好的场景——它们要么用代理目标(REINFORCE / straight-through estimator)、要么把不可微部分隔离开(detach + 单独训)。

### 10.4 记住这一根线

```
loss.backward()
  → 沿计算图逆拓扑序,每个算子调它的 backward
    → 每个算子的 backward = 它的局部雅可比 × 上游 grad
      → 走到叶子参数时,梯度累加到 .grad
        → 优化器读 .grad 更新参数
          → 一步训练完成
```

**这一行 = 整套深度学习训练**。所有花哨的内容(Adam、混合精度、checkpointing、distributed training)都是围绕这一行做工程优化。
