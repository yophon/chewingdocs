# 18 · 互信息与特征选择:决策树、CLIP、SimCLR 的同一把尺子

接着 16、17 篇。**H 量化"一个变量的不确定"、KL 量化"两个分布的差异",这一篇要讲的互信息 I(X;Y) 量化"两个变量共享了多少信息"**。

听起来抽象,但你天天在用:**sklearn 的特征选择、决策树的分裂准则、CLIP 的图文对齐、SimCLR/DINO 的对比学习——背后都是同一把尺子 I**。读不懂这一层,你就没法解释"为什么 contrastive loss 要拉近正样本、推远负样本"——你只能记结论。

> 一句话先记住:**I(X;Y) = "知道了 X 之后,Y 还剩多少不确定的减少量"**。皮尔逊相关只能抓线性,**MI 抓任意依赖**。所有"我想知道 X 对预测 Y 到底有多少帮助"的工程问题,默认第一反应就是 MI。

---

## 一、为什么必须懂这个

| 场景 | 不懂时 | 懂之后 |
| --- | --- | --- |
| 特征工程要排序特征重要性 | 乱用皮尔逊相关系数 | 知道用 `mutual_info_classif` 抓非线性 |
| 决策树为什么按某个特征分裂 | "信息增益最大" 这个词没感觉 | 信息增益就是 MI |
| 对比学习的 loss 长得很怪 | 抄 SimCLR | 知道这是 I(z, z') 的下界 |
| CLIP 图文为什么这么训 | "对齐图文特征" | 在最大化 I(图, 文) |
| 表示学习目标怎么设 | 跟着论文抄 | "z 与 y 的 MI 大、与噪声的 MI 小"——信息瓶颈 |
| MI 估计经常 0 或负数 | 怀疑 sklearn 有 bug | 是高维下 MI 估计本来就难 |

> **经验法则**:**一切"两个东西到底相关不相关、相关程度多大"的问题,先试皮尔逊看线性、不行就上 MI 看非线性**。看不到结构再考虑表示学习。

---

## 二、定义:不确定的减少量

最自然的引入是从条件熵开始:

```
H(Y)        = 不知道任何事时, Y 的不确定
H(Y | X)    = 知道 X 之后,   Y 还剩的不确定

I(X ; Y)    = H(Y) - H(Y | X)   ← 知道 X 后, Y 减少了多少不确定
            = H(X) - H(X | Y)   ← 对称, 反过来也成立
            = H(X) + H(Y) - H(X, Y)
```

韦恩图(信息论里非常经典的图,值得手画一遍记牢):

```
            H(X)                 H(Y)
       ┌──────────────────┬────────────────────┐
       │                  │                    │
       │   H(X|Y)         │       H(Y|X)       │
       │                  │                    │
       │     X 独有       │     Y 独有         │
       │                  │                    │
       │           ┌──────┴──────┐             │
       │           │             │             │
       │           │  I(X ; Y)   │             │
       │           │  共享部分   │             │
       │           │             │             │
       │           └─────────────┘             │
       └──────────────────┴────────────────────┘
                        H(X, Y)
                = H(X) + H(Y) - I(X;Y)
                = H(X|Y) + H(Y|X) + I(X;Y)
```

> **核心直觉**:**MI 就是两个变量在韦恩图里的"交集"**。X、Y 完全独立 → 没交集 → I=0;X 完全决定 Y → Y 整个被 X 包住 → I = H(Y)。

## 三、与 KL 的关系:为什么 I 是一种"距离"

把定义展开就得到:

$$
I(X;Y) = \sum_{x,y} p(x,y) \log \frac{p(x,y)}{p(x)\,p(y)} = \text{KL}\big(p(x,y) \,\|\, p(x)p(y)\big)
$$

**意思**:**联合分布 p(x,y) 和"假装独立"的乘积分布 p(x)p(y) 离得有多远**。

| X、Y 的关系 | p(x,y) vs p(x)p(y) | I |
| --- | --- | --- |
| 完全独立 | 完全相等 | 0 |
| 弱相关 | 略偏离 | 小正数 |
| 强相关 / 强依赖 | 相差很大 | 大 |

> **避坑**:**MI 一定 ≥ 0**(因为它是 KL)。代码里出来负数,只可能是估计器的偏差,不是真的负相关——MI 本身没有"负"这个概念,要正负看皮尔逊。

---

## 四、与皮尔逊相关系数对比:线性 vs 任意依赖

皮尔逊 `r ∈ [-1, 1]` 衡量**线性**相关。**只要关系不是直线,r 就抓不到**。

```
Y = sin(X), X ∈ [-π, π]:
  X 完全决定 Y → I(X;Y) 接近 H(Y) (很大)
  但 r = 0    (因为正负抵消, 没有线性)

  X
  │              ╭───╮               ╭───╮
  │             ╱     ╲             ╱     ╲
  │            ╱       ╲           ╱       ╲
  │           ╱         ╲         ╱         ╲
  │──────────╯           ╲───────╯           ╲──── X
  │                       ╲                   
  │                        ╲                  
  │                         ╲                 
  └────────────────────────────────────────── Y
```

代码验一下:

```python
import numpy as np
from sklearn.feature_selection import mutual_info_regression
from scipy.stats import pearsonr

x = np.random.uniform(-np.pi, np.pi, 5000)
y = np.sin(x)

print("Pearson r:", pearsonr(x, y))         # ≈ 0  线性看不到
print("MI:", mutual_info_regression(x.reshape(-1,1), y))   # 远大于 0
```

| 关系类型 | 皮尔逊 r | 互信息 I |
| --- | --- | --- |
| 完全线性 y=2x | ±1 | 大 |
| 二次 y=x² | 0 | 大 |
| sin / 周期 | ≈0 | 大 |
| 完全独立 | 0 | 0 |
| 单调非线性 y=x³ | 接近 1 | 大 |

> **经验法则**:**特征筛选别只看皮尔逊**。一个看似"无关"的特征可能是非线性强相关——用 MI 再过一轮,会捞到不少漏掉的好特征。

## 五、互信息估计的难处

定义干净,但**实际数据里 X、Y 通常是连续的,p(x,y) 你拿不到**——只能从样本估。这一估就麻烦了。

| 估计方法 | 思路 | 问题 |
| --- | --- | --- |
| 直方图 / binning | 切格子算频率 | bin 太大丢细节,太小方差爆;高维下格子指数爆 |
| KDE(核密度估计) | 平滑插值 | bandwidth 难选;高维同样不行 |
| **k-NN 估计 (KSG)** | 用最近邻距离反推局部密度 | sklearn 默认就这个,中等维度还行 |
| **MINE / InfoNCE** | 用神经网络估 KL 下界 | 需要大量样本和训练,不稳但能扩到高维 |

**MINE 的核心思路**(只点不展开):利用 Donsker-Varadhan 表示,把 KL(联合 ‖ 边缘乘积)写成一个变分下界,再用一个神经网络 T 去逼近这个下界。**InfoNCE 是 MINE 思路的对比学习版**——下一节会用到。

> **避坑**:**高维数据(几百维以上)直接算 MI 几乎都是不准的**。这就是为什么近代方法不直接最大化 MI,而是最大化它的某个下界(InfoNCE)或代理(对比损失)。

## 六、决策树的信息增益就是 MI

ID3 / C4.5 / sklearn 的 `DecisionTreeClassifier(criterion='entropy')` 用的"**信息增益**":

```
IG(D, A) = H(D)  -  Σ_v |D_v|/|D| · H(D_v)

H(D)               熵 of 标签分布在整个数据集
Σ ... H(D_v)        按特征 A 分裂后, 每个子集熵的加权和
                    = H(D | A)
```

也就是 **IG = H(D) - H(D|A) = I(A; D)**——**信息增益就是 "特征 A 与标签 D 的互信息"**。

| 准则 | 数学 | sklearn 名字 |
| --- | --- | --- |
| Information Gain | I(A;Y) | `criterion='entropy'` |
| Gain Ratio | I/H(A) | C4.5,sklearn 没原生 |
| Gini Impurity | 1 - Σp² | `criterion='gini'` 默认 |

> **核心直觉**:**决策树每次分裂都在贪心找"和标签互信息最大的特征"**。MI 越大,这次分裂减少的不确定就越多——树就长得越浅、越好解释。

## 七、特征选择:MI 和 chi-square 谁该用

sklearn 的 `feature_selection` 模块提供了一堆打分函数,选哪个有讲究:

| 函数 | 适用 | 抓什么 | 备注 |
| --- | --- | --- | --- |
| `mutual_info_classif` | 任意特征 → 离散标签 | **任意依赖** | k-NN 估计,稍慢但通用 |
| `mutual_info_regression` | 任意特征 → 连续标签 | 任意依赖 | 同上 |
| `chi2` | **非负**离散特征 → 离散标签 | 拟合优度差异 | 文本词频专用,要求非负 |
| `f_classif` | 连续特征 → 离散标签 | 类间均值差(线性) | ANOVA F 检验 |
| `f_regression` | 连续 → 连续 | 线性相关(等价 r²) | 只抓线性 |

```python
from sklearn.datasets import load_iris
from sklearn.feature_selection import mutual_info_classif, SelectKBest

X, y = load_iris(return_X_y=True)
print(mutual_info_classif(X, y))     # 4 个特征对标签的 MI
selector = SelectKBest(mutual_info_classif, k=2).fit(X, y)
X_sel = selector.transform(X)
```

> **经验法则**:
> - **想抓非线性 → MI**(`mutual_info_*`)
> - **文本 / one-hot / 计数特征 → chi-square**(快,假设满足)
> - **只在乎线性 + 速度 → f_regression / f_classif**
> - 三个都跑一遍,看哪些特征三家都打高分,这种最稳

## 八、表示学习的目标:信息瓶颈预告

监督学习的"理想编码 z"应该满足:

```
maximize   I(z ; y)        让 z 把标签 y 的信息全编进去
minimize   I(z ; x | y)    去掉 x 中和 y 无关的噪声(只是冗余的输入信息)

合起来叫 "信息瓶颈" (Information Bottleneck, IB):
  L_IB = -I(z; y) + β · I(z; x)
                      ↑ β 控制压缩与表达力的平衡
```

> **核心直觉**:**好的表示 z = "把标签信息保留尽,把输入噪声丢掉"**。这就是 PCA、autoencoder、self-supervised representation 共同想做的事——只是各自的 surrogate(代理目标)不一样。

19 篇会展开,本篇先种这个种子。

## 九、InfoNCE:对比损失就是 MI 的下界

2018 年的 CPC 论文给出了今天对比学习的核心目标:

```
给定一个"正样本对"(z, z+) 来自联合分布 p(z, z+)
   和 K 个"负样本"   z⁻_1, ..., z⁻_K 来自边缘 p(z+)

InfoNCE = -log [ exp(sim(z, z+) / τ) /
                 ( exp(sim(z, z+) / τ) + Σ_k exp(sim(z, z⁻_k) / τ) ) ]
```

形式上**就是一个 (K+1) 路 softmax 分类问题**——"在 K+1 个候选里,把正样本挑出来"。

**关键性质**(直觉,不展开推导):

$$
I(z; z^+) \geq \log K - \mathcal{L}_{\text{InfoNCE}}
$$

**最小化 InfoNCE = 最大化 I(z, z+) 的下界**。

> **核心直觉**:**对比学习的本质是"让模型学会区分'真的来自同一个东西'和'来自不同东西'"——区分能力越强, z 和 z+ 共享的信息就越多**。下界 `log K` 也告诉你:**负样本越多(K 越大),下界越紧,学到的表示越好**——这就是 SimCLR 用大 batch、MoCo 用 queue 的原因。

## 十、SimCLR / CLIP:几何上看 InfoNCE

几何上,InfoNCE 干两件事:

```
                         z+
                         ●
                        ╱
                  正样本对拉近
                      ╱
                    ●  z
                  ╱  ╲
                ╱      ╲
              ╱          ╲ 负样本推远
            ●              ●
          z⁻_1            z⁻_2
```

| 方法 | 正样本怎么造 | 负样本怎么取 |
| --- | --- | --- |
| **SimCLR** | 同一张图两次不同数据增强 | 同 batch 里其他样本(2N-2 个) |
| **MoCo** | 同上,加动量编码器 | 维护一个 queue 存历史负样本 |
| **CLIP** | (图, 该图的描述文)对 | batch 内其他文本(对图)和其他图(对文) |
| **DINO** | 教师-学生不同 view | 不显式取负样本,用中心化 + sharpening 防塌缩 |
| **BYOL** | 同上 | 不取负样本(靠 stop-gradient + 投影头) |

CLIP 的损失更精确叫"对称 InfoNCE":

```python
# image_features, text_features: (N, d), 已 L2 归一化
logits = image_features @ text_features.T / temperature   # (N, N)
labels = torch.arange(N, device=logits.device)
# 行 softmax: 给每张图找正确的文本
loss_i2t = F.cross_entropy(logits, labels)
# 列 softmax: 给每段文本找正确的图
loss_t2i = F.cross_entropy(logits.T, labels)
loss = (loss_i2t + loss_t2i) / 2
```

**对角线就是正样本对(同一索引的图文),非对角线是 N²-N 个负样本对**——一个 batch 一次性算了 2N 个 InfoNCE 项。

> **经验法则**:**对比学习里调 batch size 比调 lr 重要**。batch 翻倍,负样本数 K 翻倍,MI 下界变紧,效果常常显著提升。OpenAI 训 CLIP 用了 32k batch、Google 训 SimCLR 用了 8k batch,不是没道理。

## 十一、工程对应清单

```python
# ── 特征选择(sklearn) ──
from sklearn.feature_selection import (
    mutual_info_classif, mutual_info_regression,
    chi2, f_classif, f_regression,
    SelectKBest, SelectPercentile,
)
mi = mutual_info_classif(X, y, n_neighbors=3)   # k-NN 估计的 MI

# ── 决策树(信息增益准则) ──
from sklearn.tree import DecisionTreeClassifier
clf = DecisionTreeClassifier(criterion='entropy')   # 用 IG (= MI)
# 默认是 'gini',速度更快,精度通常差不多

# ── 离散两个变量直接算 MI ──
from sklearn.metrics import mutual_info_score
mi = mutual_info_score(labels_a, labels_b)        # 单位 nat

# ── InfoNCE / 对比损失 ──
import torch.nn.functional as F
def info_nce(z1, z2, tau=0.07):
    z1 = F.normalize(z1, dim=-1); z2 = F.normalize(z2, dim=-1)
    logits = z1 @ z2.T / tau
    labels = torch.arange(z1.size(0), device=z1.device)
    return F.cross_entropy(logits, labels)
```

| 库 / 系统 | API / 文件 | 备注 |
| --- | --- | --- |
| sklearn | `mutual_info_classif/_regression` | KSG (k-NN) 估计 |
| sklearn | `DecisionTreeClassifier(criterion='entropy')` | IG = MI |
| HuggingFace | `transformers` 里 CLIP 模型 | `CLIPModel.forward` 自带对比 loss |
| OpenAI | `clip` repo | 参考实现,看 `clip/model.py` |
| Lightly / Solo-learn | self-supervised 训练框架 | SimCLR / MoCo / DINO / BYOL 都有 |
| PyMINE | 第三方 MINE 实现 | 高维 MI 估计实验性 |

## 十二、局限与什么时候不要用

- **MI 估计在高维下极不靠谱**:几十维以上,KSG 严重低估;MINE 训不稳。**所以表示学习改用 InfoNCE 这种"对比代理"而不是直接估 MI**
- **MI 不是"距离"**:三角不等式不成立;两个完全相关但不同的变量 I=H,但它们不是"同一个东西"
- **MI 对小样本敏感**:估计值带很大方差,做特征排序时不要看绝对值,只看相对排序
- **MI 抓不到"因果"**:I(X;Y) = I(Y;X) 完全对称,但 X 是不是 Y 的原因它不管。要因果用 do-calculus / SCM,不是 MI
- **InfoNCE 的下界很松**:理论下界 log K 在 K 不大时离真值远;数值上别拿 InfoNCE 损失反推真实 MI
