# 数论与密码学

整本书最"小学生"的一层数学,撑起了**全人类目前在公网上信任彼此身份和秘密的全部基础**。RSA、Diffie-Hellman、ECC、SHA、TLS、签名、区块链——底下都是模算术、欧几里得、欧拉定理这几样东西。这一篇不教你写一个 RSA(那是 securityLearning 的活),只回答一个问题:**为什么"乘法容易、分解难"和"指数容易、求对数难"这两个不对称性,可以拿来加密**——理解到这一层,后面读 TLS 1.3 握手、Signal 协议、PQC 选型论文,就不会再被那一行公式劝退。

> 一句话先记住:**现代公钥密码学全靠"陷门函数"——从私钥推公钥容易,从公钥反推私钥需要解一个数论上的难问题**。RSA 的难问题是大整数分解,DH/ECC 的难问题是离散对数,Kyber/Dilithium 的难问题是格上的最短向量。**当某个难问题被证明能在多项式时间解出来(Shor 算法对量子机就是这么干 RSA/ECC 的),对应的算法就死了**——这就是为什么 NIST 在搞后量子标准化,而不是因为 RSA 本身有 bug。

## 一、为什么工程师要懂数论

| 场景 | 不懂数论时 | 懂一点之后 |
| --- | --- | --- |
| 看 TLS 1.3 握手抓包 | "X25519 是啥?" | 椭圆曲线 DH,ECDHE 的现代默认 |
| 选 RSA 还是 ECC | 抄默认 | 知道 ECC-256 ≈ RSA-3072,带宽/速度差 10x |
| 看到 `e=65537` | 不知道为什么 | 二进制 10000000000000001,快速幂只算 17 步 |
| MD5/SHA1 弃用 | "应该是不安全了吧" | 抗碰撞被攻破,签名场景全废 |
| Shor 算法新闻 | "量子要颠覆密码学了" | RSA/ECC 死,对称密钥和哈希只是减半位数 |
| 自己实现 AES | "看着挺简单" | side-channel,常数时间,timing attack,直接被打穿 |

> **经验法则**:**永远不要自己实现密码学**——这一篇看完你会更明白为什么。但你要懂"它在干什么、调它会动什么、什么时候这个算法死了"。

## 二、模运算:全部公钥密码学的底座

### 2.1 a mod n 的环

`a mod n` = a 除以 n 的余数。模 n 之后所有整数被压成 `{0, 1, 2, ..., n-1}`,**整数线被卷成一个 n 个位置的环**:

```
n = 7 的环:

         0
      6     1
    5         2
      4     3
         (从 0 开始,加 1 转一圈又回到 0)

  17 mod 7 = 3   (17 = 2×7 + 3)
   3 + 5 mod 7 = 1  (溢出绕回来)
   2 × 4 mod 7 = 1  (8 - 7 = 1)
   3^4 mod 7 = 81 mod 7 = 4
```

模 n 下加减乘都封闭——**结果永远落回环里**。除法不一定有,得看模逆元(下一节)。

> **核心直觉**:**所有公钥密码的"加密 / 解密 / 签名 / 验签"运算全在某个模 n 的环里转**。RSA 的环是 `Z/nZ`(n 是两个大素数的积);DH 的环是 `Z/pZ` 的乘法子群;ECC 的"环"换成了椭圆曲线上的点群。

### 2.2 模幂:加密的核心动作

RSA 的加密就是 `c = m^e mod n`,**这一行需要算 65537 次乘法吗?不需要**——快速幂(square-and-multiply)只要 `O(log e)` 步:

```
要算 m^13 mod n,把 13 写成二进制 1101:

m^13 = m^(8+4+1)
     = m^8 · m^4 · m^1

迭代:
  result = 1, base = m
  bit 0 = 1: result *= base = m         ;base = m^2
  bit 1 = 0:                            ;base = m^4
  bit 2 = 1: result *= base = m · m^4   ;base = m^8
  bit 3 = 1: result *= base = m · m^4 · m^8 = m^13

每步都 mod n,数字始终不爆炸。
```

```python
def modpow(base, exp, mod):
    result = 1
    base = base % mod
    while exp > 0:
        if exp & 1:
            result = (result * base) % mod
        base = (base * base) % mod
        exp >>= 1
    return result

# Python 内置 pow(base, exp, mod) 已经是这个,且用 C 实现
print(pow(7, 65537, 1000000007))
```

> **避坑**:**永远用 `pow(a, b, m)` 三参数版本,不要 `(a**b) % m`**——后者会先把 `a**b` 算成一个几千位的天文数字,然后才取模,内存炸 + 慢一万倍。这一行能区分会用 Python 和真懂数论的人。

### 2.3 费马小定理 / 欧拉定理:模幂的"周期律"

**费马小定理**(p 为素数,gcd(a, p) = 1):

```
a^(p-1) ≡ 1 (mod p)
```

**欧拉定理**(n 任意,gcd(a, n) = 1,φ(n) 是欧拉函数,即小于 n 且与 n 互素的整数个数):

```
a^φ(n) ≡ 1 (mod n)

当 n = pq(两素数积)时,φ(n) = (p-1)(q-1)
```

> **核心直觉**:**模 n 下,`a` 的幂在某个周期上循环;φ(n) 是这个周期的"上界"**。RSA 的解密能成立,就是因为 `m^(ed) = m^(1 + k·φ(n)) ≡ m (mod n)`——只要选 e 和 d 满足 `ed ≡ 1 (mod φ(n))`,加密之后用 d 再幂一次就能还原 m。这就是 RSA 的全部数学。

### 2.4 模逆元 / 扩展欧几里得

模运算下没有"除法",只有"乘逆元":

```
a 在模 n 下的逆元 a^(-1) 是满足 a · a^(-1) ≡ 1 (mod n) 的整数
```

**逆元存在 ⇔ gcd(a, n) = 1**(a 与 n 互素)。算逆元的工具是**扩展欧几里得算法**:

```
普通欧几里得求 gcd(a, b):反复 (a, b) → (b, a mod b),余数变 0 时停
扩展版顺便求出整数 x, y 使得  a·x + b·y = gcd(a, b)
当 gcd(a, n) = 1 时,a·x + n·y = 1,等式两边 mod n  ⇒  a·x ≡ 1  ⇒  x 就是 a^(-1)
```

不展开实现,**记住它能在 O(log n) 步内同时给出 gcd 和逆元**就够了。

```python
# Python 3.8+ 内置:pow(a, -1, n) 直接给模逆元
print(pow(7, -1, 11))   # 8,因为 7*8 = 56 = 5*11 + 1
```

> **核心直觉**:**RSA 密钥生成里"算 d 使得 ed ≡ 1 (mod φ(n))"——这一步就是扩展欧几里得求 e 在模 φ(n) 下的逆元**。一行 `d = pow(e, -1, phi_n)` 解决。

## 三、为什么"分解大整数"难

```
乘法:   p × q = n         (10⁻⁶ 秒内做完,n 是 2048 位也一样)
分解:   n = ? × ?         (目前最强经典算法 GNFS 在 2048 位上要 ~10²⁰ 年)
```

**这是一个不对称问题**。注意:**没人证明过"分解一定难"**——理论上可能哪天有人发现一个多项式时间分解算法,RSA 当晚就死。我们靠的是**没人能在合理时间内分解 RSA 大数 + 全世界几十年攻击都没成功**这个经验事实。

| 模数位数 | 状态 | 时间 |
| --- | --- | --- |
| RSA-512(155 位十进制) | 1999 年破 | 几个月集群 |
| RSA-768(232 位) | 2009 年破 | 2 年 ~ 千核 |
| RSA-829(250 位) | 2020 年破 | ~2700 核·年 |
| **RSA-1024(309 位)** | **未公开破解,但被认为接近"国家级机构能破"** | **不再用于新系统** |
| **RSA-2048** | **远超公开破解能力** | **NIST 认为可用到 2030 年左右** |
| RSA-4096 | 高边际安全 | 加密慢 8 倍,极少用 |

> **避坑**:**所有新系统都该用 RSA-2048 起步,RSA-3072 / RSA-4096 用于长寿命场景(根证书、长期签名)**。RSA-1024 仍在很多老 CA 链里,见到就该警觉。

经典最强分解算法是**通用数域筛(GNFS)**,亚指数复杂度 `exp((1.92 + o(1)) · (ln n)^(1/3) · (ln ln n)^(2/3))`——比指数好,比多项式差。**Shor 算法在量子计算机上是多项式的**,这是 PQC 整个领域存在的原因。

## 四、RSA:基于"分解难"的公钥密码

### 4.1 密钥生成

```
1. 随机选两个大素数 p, q (各约 1024 位,RSA-2048 时)
2. n = p · q                       ← 公钥的一部分
3. φ(n) = (p-1)(q-1)               ← 私钥推导用,生成完就该销毁
4. 选 e,通常 e = 65537             ← 公钥的另一部分
5. d = e^(-1) mod φ(n)             ← 私钥
6. 公布 (n, e),保密 d。p, q, φ(n) 也要严格销毁
```

### 4.2 加密 / 解密

```
加密:  c = m^e mod n               (任何人用公钥都能算)
解密:  m = c^d mod n               (只有持私钥 d 的人能算)
```

为什么解密能还原?因为 `c^d = m^(ed) ≡ m^(1 + k·φ(n)) ≡ m · 1^k ≡ m (mod n)`——欧拉定理。

### 4.3 签名 / 验签

签名是加解密的"反向"用法:

```
签名:  s = m^d mod n               (只有私钥持有者能产生)
验签:  m' = s^e mod n,验证 m' == m (任何人用公钥都能验)
```

实际中签名的 `m` 不是原文,而是 `Hash(原文)`——这就是为什么哈希函数的"抗碰撞"性质至关重要(下一节)。

### 4.4 为什么 e 常取 65537

`65537 = 2^16 + 1 = 0x10001`,二进制 `10000000000000001` 只有两个 1。**快速幂只要 17 次平方 + 1 次乘法**——加密极快。

为什么不取 e=3?**也可以**(更快),但有几个老攻击(Coppersmith / Håstad)要求加密前严格做随机填充才安全。65537 是"足够大避免低指数攻击 + 足够小让加密快"的甜点。

> **避坑**:**永远用 RSA-OAEP 填充,不要"裸 RSA"**(`m^e mod n` 直接加 m 是 textbook RSA,有可塑性、能被选择密文攻击打穿)。OpenSSL `RSA_PKCS1_OAEP_PADDING` 是默认选项。

### 4.5 为什么 d 不能太小:Wiener 攻击

如果有人偷懒选了一个小的 d(比如想加快解密),`d < n^(1/4) / 3` 时,**Wiener 用连分数算法在多项式时间能从公钥 (n, e) 反推出 d**。**所以 RSA 实现里 d 必须是接近 φ(n) 量级的随机数**——这一步是密钥生成自动给你处理的,但你得知道为什么不能"自己挑一个 d=3"。

### 4.6 一段最小 RSA 示例(教学用,不要上生产)

```python
from sympy import nextprime, mod_inverse
import random

def gen_rsa(bits=512):
    p = nextprime(random.getrandbits(bits))
    q = nextprime(random.getrandbits(bits))
    n = p * q
    phi = (p - 1) * (q - 1)
    e = 65537
    d = mod_inverse(e, phi)
    return (n, e), d  # 公钥,私钥

(n, e), d = gen_rsa(512)
m = 12345678
c = pow(m, e, n)
m_back = pow(c, d, n)
assert m == m_back
print("RSA 成功还原:", m_back)
```

> **生产用**:`cryptography.hazmat.primitives.asymmetric.rsa` 或更高层的 PyNaCl / age。**自己写的版本会漏掉填充 / 时序攻击 / 随机数熵**——一漏一个。

## 五、Diffie-Hellman:不传密钥的密钥交换

RSA 解决"我加密、你解密"。**DH 解决另一个问题:Alice 和 Bob 在公共信道上,不预先共享任何秘密,如何商量出一个只有他俩知道的对称密钥?**

### 5.1 基本协议

```
公开参数(全网公开): 大素数 p,生成元 g(g 是 Z/pZ* 的乘法生成元)

Alice                              Bob
──────                            ──────
私钥 a (随机)                      私钥 b (随机)
计算 A = g^a mod p     ──A──>      
                       <──B──     计算 B = g^b mod p
共享密钥 = B^a mod p              共享密钥 = A^b mod p
       = g^(ab) mod p                    = g^(ab) mod p
```

两人算出来的都是 `g^(ab) mod p`,**完全相同**。中间窃听者 Eve 只看到 `g, p, A=g^a, B=g^b`,**要算 `g^(ab)` 必须从 A 反推 a(或从 B 反推 b),这是离散对数问题**。

### 5.2 为什么"离散对数"难

```
正向:  给 g, a, p,算 A = g^a mod p           ← 快速幂,O(log a)
反向:  给 g, A, p,反求 a 使 g^a = A mod p    ← 没有亚指数以下的经典算法
```

**和大整数分解一样,这是一个广为相信但未证明的"难问题"**。最强经典算法(数域筛的离散对数变体)也是亚指数的;Shor 算法在量子机上同样多项式时间能解,**所以 DH 和 RSA 同生共死**。

### 5.3 工程现状

| 名字 | 形式 | 在哪 |
| --- | --- | --- |
| DH | 模 p 上的乘法群,p 通常 2048-3072 位 | TLS 1.2 旧 ciphersuite,逐渐淘汰 |
| ECDH | 椭圆曲线上的点加群,密钥 256 位 | TLS 1.3 主流 |
| **X25519** | Curve25519 上的 ECDH,**TLS 1.3 默认之一** | 几乎所有现代 TLS / SSH / Signal / WireGuard |

> **避坑**:**DH 必须用"短期密钥对"(ephemeral, 即 DHE / ECDHE),每次会话新生成,用完即焚**——这样即使长期私钥日后泄漏,过去录下来的会话也无法解密(**前向保密 forward secrecy**)。TLS 1.3 强制了所有握手都用 ECDHE,这是它比 1.2 重要的安全改进之一。

## 六、椭圆曲线密码学(ECC):换个更难的问题

### 6.1 椭圆曲线的"加法"是什么

椭圆曲线在密码学里通常长这样:

```
y² = x³ + ax + b   (mod p)
```

曲线上的"点"组成一个群——你可以定义两个点 P 和 Q 的"和":

```
                ●  Q
              ╱
            ╱
          ╱           过 P 和 Q 画一条直线,交曲线于第三点 R
        ╱             把 R 关于 x 轴翻折,得到 P + Q
      ●               
   P  ●                 ●
       ╲             ╱  R'     <── P + Q (R 翻折)
        ╲         ╱
          ●     ╱
            R ●
              ╲
                ╲ (这是椭圆曲线点加几何;在密码学里 mod p 之后没有"图像",只是代数运算)
```

**重点不是公式**,重点是:**这个"加法"满足结合律,有零元(无穷远点),每个点有逆元——所以是个群**。然后有"标量乘": `n·P = P + P + ... + P`(n 次)。

### 6.2 椭圆曲线离散对数问题(ECDLP)

```
正向:  给 P, n,算 Q = n·P                  ← 快(类似快速幂的 double-and-add,O(log n))
反向:  给 P, Q,反求 n 使 Q = n·P            ← 比模 p 离散对数还难
```

**关键:**"椭圆曲线离散对数"目前**没有亚指数算法**——最强经典算法是 Pollard's rho,纯指数 `O(√n)`。所以同等安全等级下,ECC 用的密钥比 RSA / 模 p DH 短得多:

| 安全等级(对称密钥) | RSA | DH | ECC |
| --- | --- | --- | --- |
| 80 位 | 1024 | 1024 | 160 |
| 112 位 | 2048 | 2048 | 224 |
| **128 位**(主流) | **3072** | **3072** | **256** |
| 192 位 | 7680 | 7680 | 384 |
| 256 位 | 15360 | 15360 | 521 |

> **核心直觉**:**256 位 ECC ≈ 3072 位 RSA,纯粹因为 ECDLP 没有亚指数算法,而分解和模 p 离散对数都有**。算法复杂度的差异直接换算成密钥位数的差异——这是密码学里"难问题选得好"的具体红利。

### 6.3 工程现状

| 曲线 | 用在哪 | 备注 |
| --- | --- | --- |
| **secp256r1 / P-256** | 大多数旧 TLS 证书 | NIST 标准,被怀疑可能有后门(无证据) |
| **Curve25519** | TLS 1.3 / SSH / Signal / WireGuard | DJB 设计,常数时间实现友好,**事实上的现代默认** |
| Ed25519 | 签名(SSH key、Git commit、JWT) | 基于 Curve25519 的签名版本(EdDSA) |
| Curve448 | 高安全等级场景 | 224 位安全 |

> **避坑**:**ECC 的曲线选择和参数选择都极其敏感**——一个不当的参数能让整个系统崩。**用 X25519 / Ed25519,不要自己挑曲线**。

## 七、哈希函数:密码学的"通用胶水"

哈希函数把任意长度输入映射成固定长度输出。在密码学里它需要满足三条性质:

| 性质 | 含义 | 失败的代价 |
| --- | --- | --- |
| **抗原像**(preimage) | 给 h,找 m 使 H(m)=h 难 | 密码哈希被反推 |
| **抗第二原像**(2nd preimage) | 给 m1,找 m2≠m1 使 H(m1)=H(m2) 难 | 文件签名能被替换 |
| **抗碰撞**(collision) | 找任意 m1≠m2 使 H(m1)=H(m2) 难 | 证书签名能伪造 |

**抗碰撞最强**——它蕴含抗第二原像。生日攻击让 n 位哈希的抗碰撞强度只有 `2^(n/2)`(这就是 25 篇组合数学讲过的生日悖论的工程后果)。

### 7.1 当代选择

| 算法 | 输出位数 | 状态 |
| --- | --- | --- |
| **MD5** | 128 | **死透了**——2004 年抗碰撞被破,现实中能秒生成碰撞 |
| **SHA-1** | 160 | **死了**——2017 年 Google SHAttered 公开了实际碰撞;Git/Mercurial 之类容器还在用,但签名场景全废 |
| **SHA-256 / SHA-384 / SHA-512** | 256/384/512 | **生产首选**,广泛硬件加速(AES-NI 之外的 SHA-NI) |
| SHA-3(Keccak) | 224-512 | 备份选择,设计哲学和 SHA-2 不同(海绵结构),工程上不如 SHA-256 普及 |
| **BLAKE3** | 任意 | 2020 年新出,**比 SHA-256 快几倍**,树形结构天然并行 |
| BLAKE2 | 任意 | BLAKE3 的前一代,仍广泛 |

> **避坑**:**对密码做哈希永远不要用"一次 SHA-256"**——要用 **bcrypt / scrypt / Argon2**,这些是"慢哈希"(参数控制成本),专门设计来抵抗 GPU/ASIC 暴力破解。SHA-256 太快,GPU 一秒能算几十亿次。

### 7.2 一段代码

```python
import hashlib

# SHA-256
print(hashlib.sha256(b"hello world").hexdigest())
# b94d27b9934d3e08a52e52d7da7dabfac484efe37a5380ee9088f7ace2efcde9

# BLAKE2b (BLAKE3 标准库还没进 hashlib)
print(hashlib.blake2b(b"hello world", digest_size=32).hexdigest())

# 密码哈希(不要用上面的!用 argon2)

```

## 八、后量子密码学(PQC):RSA / ECC 的退场倒计时

### 8.1 Shor 算法为什么是核弹

1994 年 Peter Shor 发现:**理想量子计算机能在多项式时间内分解大整数,也能在多项式时间内解模 p 离散对数和椭圆曲线离散对数**。

```
经典电脑:     RSA-2048 分解 ~ 10²⁰ 年
量子电脑(够大):RSA-2048 分解 ~ 几小时
```

**结论**:一旦"密码学相关量子计算机(CRQC)"造出来,**所有依赖大整数分解或离散对数的算法当晚就死**——RSA、DH、ECC、所有现行 TLS 长期身份证书全废。

**目前现状**(2025-2026):还没有 CRQC,但**进展速度让很多政府/大公司开始行动**——NSA 的 CNSA 2.0 路线图要求 2030 年前关键系统迁移;**Apple iMessage、Signal、Cloudflare、Google Chrome 已经上了"混合 PQC"**(传统 ECDH + Kyber 同时跑,任一安全就行)。

### 8.2 NIST 选定的算法(2024 标准化)

| 名字 | 类型 | 难问题 | 备注 |
| --- | --- | --- | --- |
| **ML-KEM(Kyber)** | KEM(密钥封装) | 模格学习(Module-LWE) | 替代 ECDH,密文 ~1KB |
| **ML-DSA(Dilithium)** | 签名 | 模格 + 短整数解 | 替代 ECDSA,签名 ~2.5KB |
| **SLH-DSA(SPHINCS+)** | 签名(无状态哈希签名) | 仅依赖哈希函数安全 | 慢但保守,签名 ~8KB |
| FALCON | 签名 | NTRU 格 | 签名小但实现难,高敏感场景 |

> **核心直觉**:**PQC 算法基于"格上的最短向量问题"或"哈希函数自身安全",这两类难问题没有已知的量子多项式算法**。代价是密钥/签名/密文都比 RSA/ECC 大一个量级,**带宽和存储要重新规划**——这就是为什么混合方案先上,纯 PQC 还在试。

### 8.3 哈希和对称密码会被量子打击吗

不会死,只是减半。**Grover 算法把 `2^n` 暴力搜索压成 `2^(n/2)`**——所以:

- AES-128 → 等价于 64 位安全(**不够,要换 AES-256**)
- SHA-256 抗原像 → 等价于 128 位安全(还行)
- SHA-256 抗碰撞 → 仍是 128 位(生日攻击已经是 √n,Grover 帮不了多少)

**结论**:**对称密钥和哈希直接把位数加倍/换 SHA-384 就够了**;真正要换的是公钥那一层。

### 9.2 TLS 1.3 握手里数论用在哪

```
ClientHello  →  支持的曲线列表 (X25519, P-256, ...) + Client 的 ECDHE 公钥
                                          ↓
                                   ServerHello + Server ECDHE 公钥 + Server 证书
                                          ↓
两侧各自算出共享密钥 g^(ab) (椭圆曲线 ECDH)
                  ↓
派生 traffic key (HKDF, 基于 SHA-256/384)
                  ↓
此后所有应用数据用 AES-GCM 或 ChaCha20-Poly1305 加密
```

**这一次握手里就用到了**:椭圆曲线(ECDH 部分)、签名(证书验证用 RSA 或 ECDSA)、哈希(HKDF + Transcript hash)、对称密码(AES-GCM)——**整本书的数论几乎全部在这一段路里**。

### 9.3 为什么"不要自己实现密码学"

公式正确不代表实现安全。常见三类灾难:

```
1. 时序攻击(timing attack):
   if password == stored:        ← 字符串比较是 short-circuit,前几位错就立刻返回
                                  → 攻击者从响应时间反推出正确前缀
   修复:用常数时间比较 hmac.compare_digest

2. side-channel:
   AES 的 S-box 查表 → CPU 缓存命中模式泄漏密钥 bit
   修复:用硬件 AES 指令(AES-NI)或 bitsliced 实现

3. 随机数熵不够:
   2008 Debian OpenSSL 把熵源 patch 错了 → 全网生成的 SSH/SSL key 只有 2^15 种可能
                                          → 攻击者枚举即可
   修复:用 OS 提供的 /dev/urandom 或 getrandom(),不要自己造 PRNG
```

> **核心直觉**:**密码学算法的安全性是"针对所有攻击模型的最弱环节"——不是"公式对了就行"**。timing / cache / 电压 / 电磁辐射 / 故障注入,任一通道泄漏 1 bit 都可能导致全部密钥被恢复。**所以用经过审计的 libsodium / OpenSSL,不要自己写**。

### 9.4 局限与开放问题

- **PQC 性能开销**:Kyber 密文 ~1KB,Dilithium 签名 ~2.5KB,**对带宽紧张的协议(QUIC、DNS-over-HTTPS)是真问题**。优化和混合方案还在快速演进。
- **量子时间表**:**没人知道 CRQC 何时出现**——保守派说 30 年,激进派说 10 年,**"先记录后解密"(harvest now, decrypt later)的攻击意味着今天加密的长寿数据可能在未来被解开**,所以高敏感场景今天就该上 PQC。
- **非对称密码总比对称密码慢 1000x 以上**——**实战中永远是"用非对称交换一个对称密钥,然后用对称密钥加密大数据"**,RSA/ECC 直接加密大文件不是设计意图。
- **侧信道攻击没有上限**——HSM、Secure Enclave、TEE 都能减轻不能消除。**最敏感的密钥要"隔离 + 多因子 + 物理保护"组合**。
- **量子安全的对称密码 / 哈希**已经没问题(只是位数加倍),所以**对称密码和哈希这一层的算法选择 10 年内不会大变**;**剧变在公钥层**。
