# 项目实战

这个页面不新增知识系列,只定义几个贯穿项目。它们的作用是把已有系列串起来,让知识从"读过"变成"用过"。

一句话:**项目实战不是再开一堆教程,而是给已有知识找真实落点。**

---

## 一、项目 A:AI 知识库 / Agent 平台

目标:做一个能摄取文档、检索问答、调用工具、评测质量、上线监控的 AI 应用。

涉及系列:

```text
AI 学习
数据工程
后端学习
AI Infra
DevOps / SRE
安全
Claude Code
```

核心模块:

| 模块 | 要解决的问题 |
| --- | --- |
| 文档摄取 | 文件解析、清洗、去重、版本管理 |
| 切分与索引 | chunk 策略、metadata、embedding、向量库 |
| RAG 问答 | 检索、重排、引用、拒答、上下文预算 |
| Agent 工具 | Tool schema、权限、沙箱、审计 |
| Eval | 测试集、回归、打分、人工复核 |
| 服务端 | API、鉴权、限流、日志、成本 |
| 推理部署 | API 模型或自部署 vLLM / SGLang |
| 生产化 | 监控、告警、灰度、事故复盘 |

建议里程碑:

```text
M1:本地文档 RAG 能跑通
M2:支持多知识库和引用
M3:加入工具调用和权限控制
M4:建立评测集和回归门禁
M5:部署上线并接入监控
```

- [Embedding与向量数据库](/aiLearning/23-Embedding与向量数据库)

## 二、项目 B:高并发业务系统

目标:做一个能承受高并发、库存竞争、缓存冲击、异步削峰和故障降级的业务系统。

候选题目:

```text
秒杀 / 库存系统
短链与访问统计系统
Feed / 通知系统
订单与支付状态机
```

涉及系列:

```text
后端学习
系统设计
分布式系统
DevOps / SRE
算法
安全
```

核心模块:

| 模块 | 要解决的问题 |
| --- | --- |
| API 层 | 鉴权、限流、幂等、错误处理 |
| 存储层 | MySQL / PostgreSQL、索引、事务 |
| 缓存层 | Redis、缓存一致性、热点 key |
| 异步层 | 消息队列、削峰、重试、死信 |
| 状态机 | 订单、支付、库存、超时关闭 |
| 可观测 | 指标、日志、链路追踪、告警 |
| 压测 | 容量估算、瓶颈定位、降级预案 |

建议里程碑:

```text
M1:单体版本跑通核心流程
M2:加入缓存和消息队列
M3:实现幂等、限流和降级
M4:做压测和容量估算
M5:接入监控并演练故障
```

优先阅读:

- [Redis实战应用](/backendLearning/20-Redis实战应用)
- [消息队列基础](/backendLearning/31-消息队列基础)

## 五、项目怎么写

每个项目后续如果展开,统一按这个结构写:

```text
01 项目目标与范围
02 架构图与边界
03 最小可运行版本
04 核心模块一
05 核心模块二
06 监控与测试
07 发布与回滚
08 事故演练与复盘
```

项目文档不要变成 API 罗列。每一篇都必须回答:

```text
这个模块解决什么真实问题?
什么时候会出事故?
怎么测试?
怎么上线?
怎么回滚?
```

## 六、当前优先级

建议优先顺序:

```text
AI 知识库 / Agent 平台
  -> 跨端产品闭环
  -> 小型视频会议 + 录制回放
  -> 高并发业务系统
```
