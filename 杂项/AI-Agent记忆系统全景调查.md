# AI Agent 记忆系统全景调查

> 调查当前业界 AI agent(智能体)的记忆是怎么做的。覆盖:记忆分类体系、核心技术手段(检索/更新/遗忘/反思)、主流开源框架(Letta、Mem0、Zep、A-MEM、MIRIX)、大厂产品(Claude / ChatGPT / Gemini)的记忆方案,以及学术前沿与开放问题。
>
> 调研方法:多角度网络检索 → 抓取 24 个源 → 抽取 115 条声明 → 对抗式验证(2/3 反驳才推翻)→ 综合。学术骨架为高置信(3-0 通过验证);产品与框架细节来自第三方拆解/厂商博客,证据强度标注为中。
>
> 时效:核心证据多为 2025-12 ~ 2026-03 的 arXiv 预印本,多数未经同行评审,半年内分类体系仍可能变动。

---

## 一句话结论

业界已从「照搬认知心理学四分法」(工作 / 情景 / 语义 / 程序性)走向**面向工程实现的多维分类**。落地上没有银弹——生产系统普遍是 **向量 + 图 + 短期情景缓冲** 的混合架构;而一个尴尬的方法论真相是:随着上下文窗口涨到 128k~1M,很多记忆 benchmark 出现「上下文饱和」,外部记忆「看上去不那么必要」了,这反而成了当前最大的研究争议点。

---

## 一、分类体系:从认知科学到工程轴线

### 1. 理论基线:CoALA 四分法(最被广泛引用)

**CoALA**(Cognitive Architectures for Language Agents, arXiv:2309.02427, Princeton)奠定了被反复引用的规范分类,源自认知科学家 Tulving 的类型学:

- **工作记忆(Working)**:当前决策周期的活跃符号变量(本质就是上下文窗口里的东西)
- **情景记忆(Episodic)**:过往经验、交互轨迹
- **语义记忆(Semantic)**:关于世界和自身的知识
- **程序性记忆(Procedural)**:LLM 权重里的隐式知识 + agent 代码里的显式技能

它还定义了三类**内部记忆动作**:`retrieval`(读)、`reasoning`(用 LLM 更新工作记忆)、`learning`(写),外加 `external grounding`(与环境交互)。这套「读—想—写」动作模型是后续几乎所有框架的概念底座。*(高置信,3-0 验证)*

### 2. 另一条主线:三层 LLM 记忆

很多综述用更贴近实现的三层划分:

- **参数记忆(Parametric)**:固化在模型权重里
- **工作记忆**:实时交互的上下文窗口
- **显式外部记忆(External)**:外部存储 + 检索

再叠加两个正交轴:**nature**(情景=交互轨迹 vs 语义=知识库)、**scope**(inside-trail 单轨迹内 vs cross-trail 跨会话持久)。*(高置信)*

### 3. 2025–2026 的转向:工程化多维分类

最新几篇大综述明确「不再只用心理学类别」:

| 综述 | 分类轴 | 子类 |
|---|---|---|
| **Memory in the Age of AI Agents**(2512.13564) | Forms / Functions / Dynamics | Forms: token级 / parametric / latent;Functions: factual / experiential / working;Dynamics: 形成 / 演化 / 检索 |
| **Memory for Autonomous LLM Agents**(2603.07670) | 时间范围 / 表征基质 / 控制策略 | — |
| **Anatomy of Agentic Memory**(2602.19320) | 纯系统架构视角 | 轻量语义记忆 / 实体中心记忆 / 情景与反思记忆 / 结构分层记忆 |

> ⚠️ 注意:这些综述说「抛弃传统类别」有自我营销成分——验证时一条相关声明只拿到 2-1 票,审查指出传统四类实际上是被**重组**进新轴(比如塞进时间范围轴),而非真的废弃。

### 4. 人脑启发路线

**Human-inspired Perspectives**(2411.00489)先讲人类长期记忆机制,再逐一映射到 AI,提出统一理论框架 **SALM**(自适应长期记忆的认知架构)。这条线代表「用神经科学指导工程」的努力。

## 二、核心技术手段

### 写—读—更新—遗忘 完整生命周期

Xi et al.(2309.07864)早就点明两大根本挑战:**① 历史超出 Transformer 序列长度需截断;② 海量历史里难抽取相关记忆**。对应三族增强方法:抬高长度上限(截断 / 分段 / 改注意力)、记忆摘要、用向量或数据结构压缩。

完整生命周期建模为**闭环**(extraction → updating → retrieval → utilization),并出现生物启发的**自适应遗忘**:SAGE 借用艾宾浩斯遗忘曲线,主动剔除低价值信息以降低认知负荷。2603.07670 把所有实现归为**五个机制族**:

1. 上下文驻留压缩
2. 检索增强存储(RAG 式)
3. 反思式自我改进(reflection)
4. 分层虚拟上下文(MemGPT 式)
5. 策略学习式管理

### 检索打分:Recency + Relevance + Importance(经典三件套)

源自斯坦福 **Generative Agents**(Park et al. 2023),至今被广泛复用(连 AWS Bedrock AgentCore 都用):

- **Recency**:指数衰减(0.995 / 小时)
- **Relevance**:查询嵌入余弦相似度
- **Importance**:LLM 打 1–10 分

三者各归一化到 [0,1] 加权求和,取高分记忆注入上下文。*(高置信,原始论文 + 综述双源)*

### 知识图谱记忆

代表是 **Zep / Graphiti**(arXiv:2501.13956):Graphiti 是**时序感知知识图谱**引擎,动态融合非结构化对话 + 结构化业务数据,**保留历史关系**(不是覆盖,而是给关系打时间戳),这样能回答「某事实在某时间点是否成立」。*(高置信)*

### Agentic 记忆与记忆演化

**A-MEM**(2502.12110,Zettelkasten 启发):每条记忆是一个含 LLM 生成属性(上下文描述、关键词、标签)的 **note**;系统自动分析历史记忆建立链接,形成互联知识网;关键是**记忆演化**——新记忆写入会触发对旧记忆属性的更新。*(高置信)*

**MIRIX**(2507.07957)把分类扩到**六型**,并用模块化多智能体实现(每类一个 Memory Manager + 一个负责路由的 Meta Manager):

| 六型 | 作用 |
|---|---|
| Core | 核心人格 / 持久事实 |
| Episodic | 带时间戳的事件 |
| Semantic | 抽象知识 |
| Procedural | 可执行技能 |
| **Resource Memory** | 资源 / 文档(新增) |
| **Knowledge Vault** | 安全凭据 / 敏感逐字信息库(新增) |

## 三、开源框架横向对比

> 以下框架对比来自抓取的单源声明(博客评测 + 论文),证据强度为**中**,性能数字尤其需谨慎看待。

| 框架 | 架构核心 | 记忆组织 | 备注 |
|---|---|---|---|
| **Letta / MemGPT** | OS 启发的**分层记忆**(core / recall / archival) | 虚拟上下文,LLM 自己决定何时「换页」 | 「记忆即操作系统」范式开创者 |
| **Mem0** | **混合三存储**:向量索引 + 实体图层 + KV 存储 | LLM 自动抽取事实 | 主打通用、易接入 |
| **Zep / Graphiti** | **图优先**:时序知识图谱 | 实体关系 + 时间有效性 | 评测中长期一致性占优 |
| **A-MEM** | Zettelkasten note 网络 | 自动链接 + 记忆演化 | 学术原型 |
| **LangMem** | LangChain 生态记忆 SDK | 提取 / 检索原语 | 偏 SDK 工具箱 |
| **Cognee** | 图 + 向量混合管线 | ECL(Extract-Cognify-Load) | 偏数据管线 |

**有一个被引用的基准数字**:Zep/Graphiti 在 **LongMemEval** 上得 **63.8%** vs Mem0 **49.0%**,约 15 分差距,归因于时序知识图谱架构。⚠️ 这是单一博客来源、厂商口径,**未经独立横评验证**,只能当线索。

**多份评测的共识**:没有单一系统全维度胜出,选型应做「决策矩阵」而非问「谁最好」;**生产系统几乎都是 向量 + 图 + 短期情景缓冲 的混合**。

## 四、大厂产品的记忆方案

> 这部分大厂多无技术论文,来自第三方拆解(如 Simon Willison)+ 官方文档,**证据强度中等**,但几家路线差异清晰且有意思:

| 产品 | 记忆实现路线 | 特点 |
|---|---|---|
| **Claude(消费版)** | **按需检索式**:两个显式函数工具 `conversation_search` + `recent_chats`,实时搜原始历史 | **不建持久画像、不做 AI 摘要**——只搜原始对话,每次干净起步,可解释性强 |
| **Claude(API memory tool)** | **文件式**:在 `/memories` 目录里增删改查记忆文件 | 跨会话持久、不占满上下文窗口;开发者可控 |
| **ChatGPT** | **画像式 + "dreaming"** | 显式 saved-memories 列表 + 后台 "dreaming" 进程跨多对话**合成 / 巩固**记忆状态 |
| **Gemini** | **两段式**:压缩的 `user_context` 画像(LLM 生成、周期刷新)+ 近期对话轮次 | 画像 + 近期窗口拼接 |

**一条清晰的产品分野**:Claude 走「**实时搜原始记录、零画像**」(隐私 / 可解释优先);ChatGPT 和 Gemini 走「**LLM 蒸馏出持久用户画像**」(个性化优先)。这是两种根本不同的哲学。

## 五、学术前沿与开放问题

**五大开放挑战**(2603.07670):

1. **持续巩固(consolidation)**——如何像睡眠一样把短期沉淀为长期
2. **因果对齐检索**——检索到的不只是相似,而是因果相关
3. **可信反思**——reflection 会不会自我强化错误
4. **学习式遗忘**——让遗忘成为可学习策略而非固定规则
5. **多模态具身记忆**

**最尖锐的方法论缺陷——「上下文饱和」(Context Saturation)**(2602.19320):

> 很多记忆 benchmark 的所有相关信息其实**能塞进单个 prompt**,这让外部记忆系统「看上去多余」。当窗口扩到 128k–1M,HotpotQA(~1k)、MemBench(~100k)全都落进窗口内。

为此新基准在补课:论文提出 **Context Saturation Gap** 指标;**Memora** 引入 **FAMA**(遗忘感知记忆准确率,惩罚依赖过时信息),用跨周 / 跨月、带显式记忆变更的对话来测真实长期记忆。这意味着:**「外部记忆相对长上下文基线到底有多大增量价值」目前尚无定论**,是该领域最需要解决的评测问题。

## 六、给落地的实践建议

1. **不要纠结「选哪个框架最好」**——按维度做决策矩阵;生产基本是混合架构。
2. **默认起步**:向量检索(语义召回)+ 短期情景缓冲(最近对话)。需要「实体随时间变化的一致性」再上知识图谱(Zep 式)。
3. **检索打分直接抄** Recency + Relevance + Importance 三件套,成熟可靠。
4. **写入要有遗忘 / 更新机制**,否则记忆库会越长越脏(A-MEM 的记忆演化、艾宾浩斯式衰减值得借鉴)。
5. **隐私 vs 个性化要先选哲学**:Claude 式「搜原始记录」可解释、好审计;ChatGPT/Gemini 式「蒸馏画像」个性化强但有隐私和「记忆漂移」风险。
6. **先确认你真的需要外部记忆**——如果场景信息量塞得进长上下文,可能长窗口 + 简单缓冲就够了,别过度工程。
