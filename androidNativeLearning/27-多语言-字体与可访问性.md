# 27-多语言、字体与可访问性

> 一句话导读:把 App 翻译成几种语言、换个好看的字体、加几行 `contentDescription`,看似是收尾装饰,但这三件事任何一件没做对,你就会在某个地区的应用商店收到一星差评,或者直接被某些可访问性认证拒之门外。

第 06 篇做完 edge-to-edge,你的 UI 适配了不同尺寸屏幕;第 08 篇做完表单,你能处理输入和键盘。但"全球可用"还差三层:**语言**(英语用户看英文,日语用户看日文,且用户能在系统设置直接切而不用重装 App)、**字体**(默认 Roboto 在中日韩看着别扭,需要 Downloadable Fonts 或 Variable Fonts)、**可访问性**(视障用户用 TalkBack 听得懂你的 UI,色弱用户能看清按钮)。这三层在 Android 13+ 有了显著的新工具:per-app language preferences 让用户在系统设置切语言,`LocaleConfig` 让系统知道你支持哪些语言;Downloadable Fonts 通过 Google Fonts Provider 把字体延迟到运行时下载,不再把巨大的字体文件打包进 APK;Variable Fonts 一份 ttf 顶多个 weight,包体更小。可访问性侧 Compose 提供 `contentDescription` 与 `semantics {}` 两套机制,前者是图标按钮的"屏幕阅读器读什么",后者是更高级的"语义节点"控制。

本篇按"多语言 → 字体 → 可访问性"递进展开,每段都给最小可运行代码,最后给一个能在发版前直接走一遍的 TalkBack 验收清单(以表格形式)。这一篇做完,你的 App 才真正具备"可上架到全球任何 Play Store 区域并通过基础合规审查"的能力。

## 1. 机制定位

国际化(i18n)、字体(typography)、可访问性(a11y)三件事在工程上是分离的,但失控的方式高度相似:**只在英语 + Roboto + 视力正常用户**的假设下开发,导致其他场景出大问题。

旧 Android 时代典型的失控写法有几个:

- **文案硬编码**:`Text("Login")` 满天飞,要翻译就要全文搜替换,翻译漏一个,日语包就有一个英文残留。
- **字体粗暴打包**:把 6 个 weight 的 `Roboto-{Light,Regular,Medium,Bold,Black,...}.ttf` 全塞进 `res/font/`,APK 多 3-5MB,实际只用 3 个 weight。
- **图标按钮无描述**:`IconButton(onClick = ...) { Icon(Icons.Default.Delete, contentDescription = null) }`,TalkBack 用户听到 "button" 但不知道是什么按钮。
- **依赖颜色传达状态**:"红色 = 错误,绿色 = 成功",色盲用户分辨不出。
- **手动塞 locale 切换**:在 App 内做一个"切语言"页面,自己 recreate Activity 强制刷新,跟系统 locale 不同步,跟 per-app language 设置打架。

Android 13(API 33)引入的 per-app language preferences 把"切语言"这事从 App 内迁到了系统设置——Settings → Apps → 你的 App → Language 直接选。这背后是一个统一的 `LocaleManager` API,App 自己仍可调用 `setApplicationLocales` 程序化切换,但用户在系统设置里改了的话,你的 App 会立刻收到 `onConfigurationChanged` 并切换。配合 `LocaleConfig`(一个 XML 资源,列出你支持的语言),系统才知道在 Settings 里给你的 App 显示哪些选项。

Downloadable Fonts 解决的是包体问题。Google Fonts Provider 是预装在所有有 Play Services 的设备上的一个 ContentProvider,字体不再打包进 APK,而是首次需要时由系统拉下来缓存。配合 Compose 的 `GoogleFont` 适配器,你可以用任意 Google Fonts 库里的字体而不增加 APK 一字节。Variable Fonts 是 OpenType 1.8+ 的特性,一份 ttf 文件内编码了字体的"weight 轴"、"width 轴"、"slant 轴"等连续维度,你声明 `weight = 350` 就能拿到介于 Regular(400)和 Light(300)之间的精确粗细。

可访问性侧,Compose 的核心机制是 **Semantics Tree**(语义树)——Compose 渲染时除了 UI 树,还会构建一棵独立的语义树,TalkBack 和无障碍服务读的就是这棵树。`Text("...")` 自动给一个 `Text` 语义节点;`Image(...)` 默认没有,需要你给 `contentDescription`;`Modifier.semantics { ... }` 可以手动 override 或追加语义信息(角色、状态、动作)。`semantics(mergeDescendants = true)` 把子节点的语义合并到父节点,适合"这一整块卡片对 TalkBack 是一个节点"的场景。

把这三层的机制定位拼起来:**多语言是数据层的隔离**(文案从代码抽到 strings.xml),**字体是渲染层的隔离**(字体从打包抽到运行时下载),**可访问性是语义层的注入**(把屏幕阅读器需要的信息显式声明)。三者都是"代码不直接写死,通过资源/语义系统间接表达",这是 Android 框架在国际化与可访问性上的统一设计哲学。

## 2. Android 心智

### 资源限定符与 locale

`res/values/strings.xml` 是默认(通常是英文 + 当 App default);`res/values-zh-rCN/strings.xml` 是简体中文;`res/values-ja/strings.xml` 是日文。资源系统在运行时根据当前 Locale 选最匹配的目录。`stringResource(R.string.login)` 自动走这套查找,你不直接选目录。

注意 `values-zh-rCN`(简体)与 `values-zh-rTW`(繁体)是两个独立目录,简繁不能用一份文件;`values-zh`(无区域)是兜底,如果你只有 `values-zh-rCN` 没有 `values-zh`,香港用户(zh-rHK)会回退到默认而不是简体。最稳的做法是给主目标地区准备完整资源,加一个 `values-zh` 当兜底。

### per-app language preferences (Android 13+)

| API/资源 | 作用 |
| --- | --- |
| `res/xml/locales_config.xml` | 列出 App 支持的所有 locale,系统设置据此显示语言选项 |
| `AndroidManifest` `android:localeConfig="@xml/locales_config"` | 把 `LocaleConfig` 挂到 Application 上 |
| `LocaleManager.setApplicationLocales(LocaleList)` | 程序化切换 |
| `AppCompatDelegate.setApplicationLocales(LocaleListCompat)` | AndroidX 的兼容封装,推荐用 |
| `getResources().configuration.locales` | 读当前生效 locale |

API < 33 没有 per-app 设置项,但 AndroidX `AppCompatDelegate` 提供向下兼容的实现(13+ 走系统 API,13- 走自己存 SharedPreferences + recreate)。

### Downloadable Fonts (Google Fonts Provider)

| 文件/类 | 作用 |
| --- | --- |
| `res/font/<name>.xml` | XML 描述字体来源(provider authority、查询参数、签名证书) |
| `androidx.compose.ui.text.googlefonts.GoogleFont` | Compose 的 Google Fonts 适配器 |
| `androidx.compose.ui.text.googlefonts.Font(googleFont, fontProvider, weight, style)` | 构建 `Font` |
| `FontFamily(...)` | 多个 weight 组合成一个 family |

字体首次使用会触发异步下载,系统给 Compose 一个 loading 状态,默认会用 fallback 字体过渡(避免文字"消失"等待)。

### Compose Semantics

| 概念 | API |
| --- | --- |
| 默认语义节点 | `Text`、`Button`、`Switch` 等组件自带 |
| 图像描述 | `Modifier.semantics { contentDescription = "..." }` 或组件参数 `contentDescription = ...` |
| 角色 | `semantics { role = Role.Button }` |
| 状态 | `semantics { stateDescription = "selected" }` |
| 自定义动作 | `semantics { onClick(label = "delete") { ... } }` |
| 合并子节点 | `semantics(mergeDescendants = true) { ... }` |
| 清空并重设 | `clearAndSetSemantics { ... }`(忽略子节点,完全自定义) |

`contentDescription` 是最常见的;`semantics { ... }` 是更强大的自定义入口。装饰性图像应显式 `contentDescription = null`,TalkBack 会跳过,不然会读 "image"。

### TalkBack 心智

TalkBack 是 Android 自带的屏幕阅读器(Settings → Accessibility → TalkBack)。开启后:
- 单指点击不直接触发,先聚焦节点并读出节点内容;
- 双击触发当前聚焦节点;
- 单指左右滑切换节点;
- 两指上下滑滚动列表。

测试时打开 TalkBack 走一遍 App,所有按钮都有意义的描述、所有动态状态(选中、禁用)能被听到、所有动作都能被触发,就算可访问性合格。

## 3. 工程实现

### 3.1 多语言:strings.xml 与 stringResource

`res/values/strings.xml`(英文默认):

```xml
<resources>
    <string name="app_name">NotedX</string>
    <string name="action_add">Add note</string>
    <string name="action_delete">Delete</string>
    <string name="hint_new_note">New note</string>
    <string name="msg_empty_list">No notes yet</string>
    <string name="note_count">%1$d notes</string>
    <plurals name="note_count_plural">
        <item quantity="one">%d note</item>
        <item quantity="other">%d notes</item>
    </plurals>
</resources>
```

`res/values-zh-rCN/strings.xml`:

```xml
<resources>
    <string name="app_name">NotedX</string>
    <string name="action_add">添加笔记</string>
    <string name="action_delete">删除</string>
    <string name="hint_new_note">新建笔记</string>
    <string name="msg_empty_list">暂无笔记</string>
    <string name="note_count">%1$d 条笔记</string>
    <plurals name="note_count_plural">
        <item quantity="other">%d 条笔记</item>
    </plurals>
</resources>
```

`res/values-ja/strings.xml`:

```xml
<resources>
    <string name="app_name">NotedX</string>
    <string name="action_add">メモを追加</string>
    <string name="action_delete">削除</string>
    <string name="hint_new_note">新しいメモ</string>
    <string name="msg_empty_list">メモはまだありません</string>
    <string name="note_count">%1$d 件のメモ</string>
    <plurals name="note_count_plural">
        <item quantity="other">%d 件のメモ</item>
    </plurals>
</resources>
```

Compose 里使用:

```kotlin
// app/src/main/java/com/notedx/ui/notes/NotesScreen.kt
import androidx.compose.ui.res.stringResource
import androidx.compose.ui.res.pluralStringResource

@Composable
fun NotesHeader(count: Int) {
    Text(
        text = pluralStringResource(R.plurals.note_count_plural, count, count),
        style = MaterialTheme.typography.titleMedium,
    )
}

@Composable
fun EmptyHint() {
    Text(text = stringResource(R.string.msg_empty_list))
}
```

`pluralStringResource` 是 Compose 1.5+ 的 API,自动按 locale 处理"1 note / 2 notes / 0 notes"的复数规则。中文、日文复数规则只有 `other`,英文有 `one + other`,俄语 / 阿拉伯语 / 波兰语规则更复杂——交给资源系统不要自己 if 判断。

### 3.2 per-app language preferences 与 LocaleConfig

`res/xml/locales_config.xml`:

```xml
<?xml version="1.0" encoding="utf-8"?>
<locale-config xmlns:android="http://schemas.android.com/apk/res/android">
    <locale android:name="en" />
    <locale android:name="zh-CN" />
    <locale android:name="ja" />
</locale-config>
```

`AndroidManifest.xml`:

```xml
<application
    android:name=".NotedXApplication"
    android:localeConfig="@xml/locales_config"
    android:label="@string/app_name">
    ...
</application>
```

挂上 `localeConfig` 后,Android 13+ 设备 Settings → Apps → NotedX → Language 会自动出现三个选项,用户切换后系统通过 `Configuration.locales` 推给 App。

如果你需要在 App 内提供"切换语言"页面(很多 App 仍然提供,因为不是所有用户知道系统设置里能切),用 `AppCompatDelegate.setApplicationLocales`:

```kotlin
// app/src/main/java/com/notedx/ui/settings/LanguagePicker.kt
package com.notedx.ui.settings

import androidx.appcompat.app.AppCompatDelegate
import androidx.core.os.LocaleListCompat
import androidx.compose.foundation.layout.*
import androidx.compose.material3.*
import androidx.compose.runtime.Composable
import androidx.compose.ui.res.stringResource

@Composable
fun LanguagePicker() {
    Column {
        ListItem(
            headlineContent = { Text("English") },
            modifier = androidx.compose.ui.Modifier.clickable {
                AppCompatDelegate.setApplicationLocales(LocaleListCompat.forLanguageTags("en"))
            },
        )
        ListItem(
            headlineContent = { Text("简体中文") },
            modifier = androidx.compose.ui.Modifier.clickable {
                AppCompatDelegate.setApplicationLocales(LocaleListCompat.forLanguageTags("zh-CN"))
            },
        )
        ListItem(
            headlineContent = { Text("日本語") },
            modifier = androidx.compose.ui.Modifier.clickable {
                AppCompatDelegate.setApplicationLocales(LocaleListCompat.forLanguageTags("ja"))
            },
        )
        ListItem(
            headlineContent = { Text(stringResource(R.string.follow_system)) },
            modifier = androidx.compose.ui.Modifier.clickable {
                AppCompatDelegate.setApplicationLocales(LocaleListCompat.getEmptyLocaleList())
            },
        )
    }
}
```

`LocaleListCompat.getEmptyLocaleList()` 等价于"跟随系统",清空 per-app 偏好。

### 3.3 Downloadable Fonts 与 Variable Fonts

声明 Google Fonts Provider。`res/values/font_certs.xml` 引用 androidx.core 提供的证书数组(`com_google_android_gms_fonts_certs`),不需要手写。

```kotlin
// app/src/main/java/com/notedx/ui/theme/Typography.kt
package com.notedx.ui.theme

import androidx.compose.material3.Typography
import androidx.compose.runtime.Composable
import androidx.compose.ui.text.TextStyle
import androidx.compose.ui.text.font.Font
import androidx.compose.ui.text.font.FontFamily
import androidx.compose.ui.text.font.FontStyle
import androidx.compose.ui.text.font.FontWeight
import androidx.compose.ui.text.googlefonts.GoogleFont
import androidx.compose.ui.unit.sp
import com.notedx.R

val GoogleFontsProvider = GoogleFont.Provider(
    providerAuthority = "com.google.android.gms.fonts",
    providerPackage = "com.google.android.gms",
    certificates = R.array.com_google_android_gms_fonts_certs,
)

val NotoSansSC = GoogleFont("Noto Sans SC")
val InterVariable = GoogleFont("Inter")

val AppFontFamily = FontFamily(
    androidx.compose.ui.text.googlefonts.Font(
        googleFont = InterVariable,
        fontProvider = GoogleFontsProvider,
        weight = FontWeight.Normal,
    ),
    androidx.compose.ui.text.googlefonts.Font(
        googleFont = InterVariable,
        fontProvider = GoogleFontsProvider,
        weight = FontWeight.Medium,
    ),
    androidx.compose.ui.text.googlefonts.Font(
        googleFont = InterVariable,
        fontProvider = GoogleFontsProvider,
        weight = FontWeight.Bold,
    ),
)

val CjkFontFamily = FontFamily(
    androidx.compose.ui.text.googlefonts.Font(
        googleFont = NotoSansSC,
        fontProvider = GoogleFontsProvider,
        weight = FontWeight.Normal,
    ),
    androidx.compose.ui.text.googlefonts.Font(
        googleFont = NotoSansSC,
        fontProvider = GoogleFontsProvider,
        weight = FontWeight.Medium,
    ),
)

val AppTypography = Typography(
    displayLarge = TextStyle(fontFamily = AppFontFamily, fontWeight = FontWeight.Bold, fontSize = 32.sp),
    titleMedium = TextStyle(fontFamily = AppFontFamily, fontWeight = FontWeight.Medium, fontSize = 16.sp),
    bodyLarge = TextStyle(fontFamily = AppFontFamily, fontWeight = FontWeight.Normal, fontSize = 16.sp),
    bodyMedium = TextStyle(fontFamily = AppFontFamily, fontWeight = FontWeight.Normal, fontSize = 14.sp),
)
```

主题里用 `AppTypography`:

```kotlin
@Composable
fun NotedXTheme(content: @Composable () -> Unit) {
    MaterialTheme(
        colorScheme = lightColorScheme(),
        typography = AppTypography,
        content = content,
    )
}
```

Variable Font 的体现:Inter 是一个 Variable 字体,`Font(googleFont = InterVariable, weight = FontWeight.Medium)` 实际上是从同一个 ttf 文件按 weight 轴取 500 的实例。系统不再为不同 weight 下载多份字体,缓存命中率高。

如果你需要更细粒度的 weight(比如 350、450),可以用 `FontWeight(350)`(自定义 weight 值)。这是 Variable Font 的真正好处:连续插值,而不是离散的 100/200/.../900。

CJK 字符自动 fallback 到 `CjkFontFamily`?Compose 目前不支持自动 fallback,需要根据 locale 在主题里切 Typography,或者直接给 CJK 文本显式指定 `fontFamily = CjkFontFamily`。生产做法是写两个 Typography(`AppTypographyLatin` / `AppTypographyCJK`),在 `NotedXTheme` 里根据 `LocalConfiguration.current.locales[0].language` 选。

### 3.4 contentDescription 与 semantics

#### 图标按钮的描述

```kotlin
// app/src/main/java/com/notedx/ui/notes/NoteRow.kt
package com.notedx.ui.notes

import androidx.compose.foundation.layout.*
import androidx.compose.material.icons.Icons
import androidx.compose.material.icons.filled.Delete
import androidx.compose.material.icons.filled.Star
import androidx.compose.material3.*
import androidx.compose.runtime.Composable
import androidx.compose.ui.Modifier
import androidx.compose.ui.res.stringResource
import androidx.compose.ui.semantics.semantics
import androidx.compose.ui.semantics.stateDescription
import com.notedx.R
import com.notedx.data.Note

@Composable
fun NoteRow(
    note: Note,
    onDelete: () -> Unit,
    onToggleStar: () -> Unit,
) {
    Row(verticalAlignment = Alignment.CenterVertically) {
        Text(text = note.title, modifier = Modifier.weight(1f))

        IconButton(
            onClick = onToggleStar,
            modifier = Modifier.semantics {
                stateDescription = if (note.starred) "Starred" else "Not starred"
            },
        ) {
            Icon(
                imageVector = Icons.Default.Star,
                contentDescription = stringResource(
                    if (note.starred) R.string.action_unstar else R.string.action_star
                ),
            )
        }

        IconButton(onClick = onDelete) {
            Icon(
                imageVector = Icons.Default.Delete,
                contentDescription = stringResource(R.string.action_delete),
            )
        }
    }
}
```

要点:
- 图标按钮必须给 `contentDescription`,否则 TalkBack 只能读 "button" 而不是 "delete";
- 切换态(收藏 / 未收藏)用 `stateDescription` 而不是把状态塞进 `contentDescription`——TalkBack 会按 "Star, button, not starred" 的顺序读,语义更清晰;
- 描述文案从 `strings.xml` 取,跟随多语言切换。

#### 装饰性图像跳过

```kotlin
@Composable
fun DecorativeBanner() {
    Image(
        painter = painterResource(R.drawable.banner_decoration),
        contentDescription = null,  // 显式 null,TalkBack 跳过
        modifier = Modifier.fillMaxWidth(),
    )
}
```

`contentDescription = null` 不是"我忘写了",是显式声明"这个图像没语义价值"。如果给空字符串 `""`,TalkBack 会停在节点上读一秒空内容,体验更差。

#### 整张卡片合并语义

```kotlin
@Composable
fun NoteCard(note: Note, onClick: () -> Unit) {
    Card(
        onClick = onClick,
        modifier = Modifier
            .fillMaxWidth()
            .semantics(mergeDescendants = true) {
                contentDescription = "Note: ${note.title}, created ${note.createdAtText}"
            },
    ) {
        Column {
            Text(note.title, style = MaterialTheme.typography.titleMedium)
            Text(note.createdAtText, style = MaterialTheme.typography.bodyMedium)
        }
    }
}
```

`mergeDescendants = true` 让 TalkBack 把整个 Card 当成一个节点,一次性读完。否则用户要分别滑过标题和时间两个节点,慢且割裂。

#### 自定义动作

```kotlin
import androidx.compose.ui.semantics.CustomAccessibilityAction
import androidx.compose.ui.semantics.customActions

@Composable
fun NoteCardWithActions(note: Note, onDelete: () -> Unit, onArchive: () -> Unit) {
    Card(
        modifier = Modifier.semantics(mergeDescendants = true) {
            customActions = listOf(
                CustomAccessibilityAction("Delete") { onDelete(); true },
                CustomAccessibilityAction("Archive") { onArchive(); true },
            )
        },
    ) {
        Text(note.title)
    }
}
```

TalkBack 用户在卡片上"双指上滑"会看到一个动作菜单,列出 "Delete" / "Archive",不需要滑出隐藏按钮。这是给"右滑显示删除"这类交互的可访问性兜底。

## 4. 调参与验收

**翻译完整度审计。** Android Studio 自带 Translations Editor(Tools → Editor → Translations Editor),能横向对比所有 locale 的 string 项,一眼看出哪个 locale 缺翻译。CI 上加一个简易脚本:遍历 `res/values-*/strings.xml`,对比 `values/strings.xml` 的 key 集合,缺一个就 fail。这比"翻译完了再人肉检查"靠谱多了。

**`%1$d` vs `%d` 占位符。** 单参数能用 `%d`;多参数必须用 `%1$d` `%2$s` 这样的位置占位符——不同语言里参数顺序可能反过来。错误写法 "found %d in %s" 在中文翻译时如果想说"在 %s 中找到 %d 个",位置占位符是唯一办法。

**字体下载失败的兜底。** Downloadable Fonts 第一次会有几百毫秒延迟,网络断开直接失败。Compose 默认在 Font 加载期间显示 fallback 字体(系统 sans-serif),完成后无缝替换。如果你想做更精细的 loading UI,监听 `FontLoadingStrategy.Async` 但生产 99% 不需要。

**字体许可与商用。** Google Fonts 都是 SIL OFL 或 Apache 2.0,商用免费。但部分中文字体(思源黑体例外)是商业字体,生产 App 绝对不能随便用 Mac/Win 自带的字体打包进 APK——这是常见的法务雷区。Noto Sans CJK / 思源 / 站酷字体是几个常被推荐的免费可商用 CJK 字体。

**TalkBack 验收清单。** 这是本篇的硬约束,以表格形式给出。每个项目都应在发版前打开 TalkBack 在真机上走一遍。

| # | 项目 | 检查方法 | 通过标准 |
| --- | --- | --- | --- |
| 1 | 所有图标按钮都有意义的 contentDescription | 打开 TalkBack,聚焦每个图标按钮 | 听到具体动作("Delete", "Add note")而不是 "button" |
| 2 | 装饰性图像不读 | 聚焦背景插画、品牌 banner | TalkBack 跳过该节点 |

**Accessibility Scanner 自动审计。** Google 出品的 Accessibility Scanner(Play Store 可装),扫一遍 UI 直接列出"contentDescription 缺失"、"对比度不足"、"触摸目标过小"等问题。CI 阶段可以集成 `androidx.test.espresso.accessibility.AccessibilityChecks`,把无障碍检查嵌进 instrumented test,断言失败即测试红。

**Compose Compiler Metrics 与字体重组。** 字体 family 是不可变对象,放到 `MaterialTheme.typography` 里 once-only。不要在 Composable 里重复 new FontFamily,会触发重组级联。

**字体下载与首屏性能。** Downloadable Fonts 第一次冷启动会增加 100-300ms(网络拉字体)。如果你的 App 强烈依赖首屏文字渲染速度,可以打包一个最小 fallback 字体(如 Inter Regular 一个 weight,~150KB)进 APK,首屏用打包字体,后续 weight 走下载。两者通过 `FontFamily` 优先级 fallback。

**Plurals 资源的覆盖度。** 不要只写 `one` 和 `other`,俄语需要 `one / few / many / other`,阿拉伯语需要 `zero / one / two / few / many / other`。Android Studio Translations Editor 会提示哪些 quantity 没填。

**验收完整路径。**

1. `res/values/strings.xml` 与所有 `res/values-*/strings.xml` 的 key 集合一致(脚本断言);
2. `res/xml/locales_config.xml` 列出的语言数量 = 实际有 `values-*/strings.xml` 的目录数;
3. 切换系统语言到中文/日文,App 全部 UI 刷新,无英文残留;
4. Settings → Apps → NotedX → Language 显示三个选项;在系统设置选日文,App 立刻切日文;
5. Inter 字体加载成功,文本渲染用 Inter 而不是 Roboto(打开 Layout Inspector → Attributes 看 fontFamily);
6. CJK 文本渲染用 Noto Sans SC(打开中文环境后看);
7. TalkBack 验收清单全部 15 项通过;
8. Accessibility Scanner 扫描无 Critical 警告。

## 5. 踩坑

**`values-zh-rCN` 是必须有的硬目录名,不要写成 `values-zh_CN` 或 `values-zh-CN`。** Android 资源系统用 BCP 47 的"r"前缀标记区域,这是历史包袱。`values-zh-rTW`(繁体台湾)、`values-zh-rHK`(繁体香港)同理。`values-pt-rBR`(巴西葡语)、`values-pt-rPT`(欧洲葡语)是另一个常见对照。

**`stringResource` 在非 `@Composable` 函数里调不到。** 比如你想在 ViewModel 里拼一个 toast 文案,不能 `stringResource(R.string.xxx)`。要么走 Context.getString(需要 ApplicationContext 注入,但 ViewModel 持有 Context 是反模式),要么把字符串 ID 暴露给 UI 层再 stringResource,后者是推荐做法。

**`per-app language` 在 Android 12 及以下不生效。** `localeConfig` 在 manifest 里是 forward-compatible,低版本忽略;但用户没有"系统设置切单 App 语言"入口。`AppCompatDelegate.setApplicationLocales` 在 12 及以下走兼容路径(自己存 SharedPreferences + 重建 Activity),功能上可用但跟系统设置不同步。

**`Configuration.locales[0]` 在 per-app 切换后,可能跟系统 locale 不一致。** 这是预期的——per-app 就是要 override 系统。如果你的代码做"根据系统 locale 决定行为"(比如时间格式),要用 `Configuration.locales[0]`(per-app 生效后的),不要用 `Locale.getDefault()`(可能滞后)。

**`LocaleConfig` 加完之后必须 clean build。** 否则系统设置可能还看不到 Language 入口。`./gradlew clean assembleDebug` 再装一次。

**Downloadable Fonts 在国内 Google Services 不可用。** 国内设备没有 `com.google.android.gms.fonts` Provider,Google Fonts 直接拉取会失败,fallback 到系统字体。生产 App 在国内市场要么打包字体进 APK,要么自建 ContentProvider(技术上可行但工程重)。这是出海项目和国内项目最大的字体方案分歧。

**Variable Fonts 的 weight 不是连续的,有最小步长。** 不同字体设计 weight 轴的步长不同,Inter 实际支持 100-900 整数 weight,你写 `FontWeight(347)` 会被舍入到最接近的有效值。不要假设你写多少就渲染多少。

**`semantics { contentDescription = "..." }` 与组件参数 `contentDescription` 同时给会冲突。** 后者最终会覆盖前者(或 merge),取决于 Compose 版本。最稳:只在组件参数处给(`Icon(contentDescription = ...)`),不要再在 Modifier 里加 semantics 写一遍。

**`Modifier.semantics(mergeDescendants = true)` 与 `clearAndSetSemantics` 不同。** 前者把子节点的语义合并到父(子节点的 contentDescription 会被拼接到父的描述里);后者把父节点变成单一节点,子节点的语义全部丢弃。合并卡片用前者,完全自定义某个组件的语义用后者。混用会让 TalkBack 行为不可预测。

**`TalkBack` 测试必须在真机上做。** 模拟器的 TalkBack 行为与真机有差异(尤其是手势识别)。Pixel / 三星 / 小米 各家都装一遍最稳。生产 App 至少在一台 Android 13+ 真机上完整跑一遍验收清单。

**`contentDescription` 不要写"图片"、"按钮"这种废话。** TalkBack 自己会读节点类型(Image, Button),你再写一遍变成 "图片 图片 删除按钮"。只写"内容是什么"、"动作是什么",节点类型让 TalkBack 自己读。

**触摸目标 48dp 不是 Compose 默认。** Material3 的 `Button` 是 40dp 高,不够 a11y 标准。要么手动 `Modifier.heightIn(min = 48.dp)`,要么用 `Modifier.minimumInteractiveComponentSize()`(Compose 1.5+ 提供,给可点击区域加最小 48dp 触摸区,不影响视觉大小)。

**`LiveRegion` 在 Compose 里要显式声明。** SnackBar 默认是 `LiveRegionMode.Polite`,会自动读;但你自己写的"实时更新文本"(比如倒计时)默认不是 LiveRegion,TalkBack 不会读变化。`Modifier.semantics { liveRegion = LiveRegionMode.Polite }` 显式声明。

**字号放大测试经常忽略 `LazyColumn` 横向溢出。** 字体放到 200% 后,长文本可能超出列宽。`Text` 默认 `maxLines = Int.MAX_VALUE`,会换行;但 `Text(maxLines = 1)` 的地方就会被截断或推到屏幕外。审计所有 maxLines 限制,确保字号放大后仍可读。

**Accessibility Scanner 与真实 TalkBack 不完全等价。** Scanner 检查的是"代码上有没有合规问题",TalkBack 检查的是"实际听起来怎么样"。Scanner 能过的 App 仍可能有"contentDescription 写了但写得没意义"的问题(比如 "按钮"、"图标")。两者都做,Scanner 防线 1,TalkBack 防线 2。

**Per-app 切换语言后,某些缓存的 string 不会刷新。** 你之前从 `context.getString(...)` 取出来 String 然后存到内存或 SharedPreferences 的,切语言后那些缓存仍是旧的。规则是:只在渲染时取 string,不缓存 String 实例。如果必须缓存,缓存的是 string resource ID(int),用时再 getString。

**`FontFamily` 加载状态默认是 `Async`,渲染会有"字体跳变"。** 第一次显示时是 fallback,加载完无缝换成真字体——视觉上会看到一瞬间的字形变化。如果对首帧极其敏感,可以预热:在 splash 阶段先 render 一个不可见的小 Text,触发字体下载并缓存。
