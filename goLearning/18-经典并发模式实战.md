# 经典并发模式实战

> **导读**：只会用 Goroutine 和 Channel 只是第一步，如何组合它们形成高可用的并发架构模式才是真正的功力。

## 一、Worker Pool (协程池)
控制最大并发数，防止协程爆炸导致 OOM 或压垮下游系统。

```go
func worker(id int, jobs <-chan int, results chan<- int) {
    for j := range jobs {
        fmt.Printf("Worker %d processing job %d\n", id, j)
        time.Sleep(time.Second) // 模拟工作
        results <- j * 2
    }
}

func main() {
    jobs := make(chan int, 100)
    results := make(chan int, 100)

    // 开启 3 个 worker（限制最大并发数为 3）
    for w := 1; w <= 3; w++ {
        go worker(w, jobs, results)
    }

    // 塞入 5 个任务
    for j := 1; j <= 5; j++ {
        jobs <- j
    }
    close(jobs) // 通知 worker 任务下发完毕

    // 收集结果
    for a := 1; a <= 5; a++ {
        <-results
    }
}
```

## 二、Fan-in (扇入) 与 Fan-out (扇出)
- **扇出**：一个源 Channel 把数据分发给多个 Goroutine 处理（类似 Worker Pool）。
- **扇入**：多个 Goroutine 把处理完的结果汇聚到一个目标 Channel 中。

## 三、Pipeline (流水线模式)
把大任务拆分成多个阶段，每个阶段用一个 Goroutine 运行，阶段之间用 Channel 连接。

```go
// 阶段 1：生成数字
func generator(nums ...int) <-chan int {
    out := make(chan int)
    go func() {
        for _, n := range nums { out <- n }
        close(out)
    }()
    return out
}

// 阶段 2：求平方
func squarer(in <-chan int) <-chan int {
    out := make(chan int)
    go func() {
        for n := range in { out <- n * n }
        close(out)
    }()
    return out
}
```
