# 指针与内存布局基础

> **导读**：Go 保留了 C 语言的指针以保证性能，但去掉了指针运算以保证安全。为什么到处都是 `&` 和 `*`？

## 一、指针基础
Go 中所有的参数传递本质上都是**值传递**。要把变量真正地传过去修改，必须传地址（指针）。

- `&`：取地址操作符
- `*`：解引用操作符（根据地址取值/修改值）

```go
func change(val int) {
    val = 20 // 毫无作用，只修改了内部的拷贝
}

func changePtr(ptr *int) {
    *ptr = 20 // 顺着指针修改了原来的内存
}

func main() {
    a := 10
    change(a)
    fmt.Println(a) // 10
    changePtr(&a)
    fmt.Println(a) // 20
}
```

## 二、`new` 与 `make` 的区别
面试高频题：两者都是用于内存分配的内置函数，但有本质区别。

- `new(T)`：分配类型 `T` 的零值内存，**返回指针 `*T`**。多用于值类型和结构体（如 `new(int)`, `new(User)`）。
- `make(T, args)`：仅用于给 `slice`、`map`、`channel` 分配内存并初始化，**返回类型 `T` 本身（引用）**。如果你用 new 去分配 map，拿到的是一个未初始化的 map 指针，向里面写数据会立刻 panic。

## 三、内存对齐 (Memory Alignment)
在定义结构体时，字段顺序会影响结构体占用的真实内存大小。编译器会自动做内存对齐填充（Padding）。
```go
// 错误示范（占用空间大）
type BadStruct struct {
    a int8  // 1 字节
    b int64 // 8 字节
    c int16 // 2 字节
} // 总计可能会占 24 字节

// 优雅示范（占用空间小）
type GoodStruct struct {
    b int64 // 8
    c int16 // 2
    a int8  // 1
} // 按照由大到小排序字段，可最大程度减少 Padding，总计 16 字节
```
