# 基础语法速通

> **导读**: Go 的基础语法不是为了炫技,而是为了让代码在团队里更容易读、改、审查和上线。

很多人第一次写 Go 会觉得它"语法少得离谱":没有三元表达式,没有 while,没有 try-catch,没有类,没有隐式类型转换,连未使用变量都会直接编译失败。这个设计不是偷懒,而是 Go 的核心取舍:把语言层面的选择减少到最低,让工程团队在同一种写法里协作。

这一篇不追求把 Go 语法字典背一遍,而是帮你建立基础心智模型:Go 的语法很小,但每个语法点都服务于工程可维护性。变量、常量、控制流、包、作用域、defer 和零值,这些东西看起来普通,实际上决定了你后面写服务、写并发、写库时会不会踩坑。

---

## 一、核心心智模型:少语法,强约束,显式优先

Go 基础语法可以用一句话概括:

> **Go 用很少的语法表达足够多的工程意图,并且尽量让错误在编译期暴露。**

这带来几个直接结果:

- 变量必须使用,否则编译失败
- import 必须使用,否则编译失败
- 不允许隐式类型转换
- 控制流只有 `if`、`for`、`switch`、`select`
- 错误通过返回值显式传递
- 资源释放通常用 `defer` 写在获取资源之后
- 每个类型都有零值,很多场景不需要构造函数

这些限制一开始会让人不适应,但它们会换来一个很重要的工程收益:读代码时不需要猜太多。

在 Go 里,你很少看到"这里到底有没有隐式转换"、"这个异常会不会从深处冒出来"、"这个变量是不是某种魔法属性"。Go 鼓励你把控制流摊开,把错误摊开,把资源生命周期摊开。

## 二、包、文件和入口函数

Go 源文件以 `package` 开头。一个目录通常就是一个包,同一个目录下的 `.go` 文件必须属于同一个包。

可执行程序必须有 `package main`,并且有一个 `main` 函数:

```go
package main

import "fmt"

func main() {
    fmt.Println("hello, go")
}
```

运行:

```bash
go run main.go
```

几个基本规则:

- `package main` 表示这个包可以编译成可执行文件
- `func main()` 是程序入口
- `import` 引入其他包
- 包名通常和目录名一致,但不是硬性要求
- 大写开头的标识符会被导出,小写开头只在包内可见

导出规则是 Go 里非常重要的可见性设计:

```go
package user

type User struct {
    Name string // 包外可访问
    age  int    // 包外不可访问
}

func NewUser(name string, age int) User { // 包外可调用
    return User{Name: name, age: age}
}
```

Go 没有 `public`、`private`、`protected`,只用首字母大小写表达可见性。这看起来粗糙,但非常直接。

## 三、变量声明:能少写,但不能乱写

Go 是静态强类型语言,但有类型推导。常见声明方式有三种:

```go
package main

import "fmt"

func main() {
    var a int = 10      // 显式类型
    var b = 20          // 类型推导
    c := 30             // 简短声明,只能在函数内部使用
    name := "gopher"

    fmt.Println(a, b, c, name)
}
```

工程里最常见的是:

- 包级变量用 `var`
- 函数内部局部变量用 `:=`
- 需要强调类型时写显式类型
- 零值有意义时直接 `var`

例如:

```go
var cache map[string]int // nil map,还不能直接写入

func main() {
    count := 0
    var enabled bool

    fmt.Println(count, enabled)
}
```

`:=` 不是"重新赋值",而是"至少声明一个新变量":

```go
package main

import "fmt"

func main() {
    x := 1
    x = 2 // 赋值

    x, y := 3, 4 // y 是新变量,所以允许; x 是重新赋值

    fmt.Println(x, y)
}
```

常见坑是误以为 `:=` 一定会修改外层变量。它可能在内层作用域创建新变量:

```go
package main

import "fmt"

func main() {
    err := error(nil)

    if true {
        err := doSomething() // 新的 err,只在 if 块里生效
        fmt.Println("inner:", err)
    }

    fmt.Println("outer:", err)
}

func doSomething() error {
    return fmt.Errorf("failed")
}
```

这类 shadowing 在真实项目里很容易制造 bug。团队里通常会用 linter 检查。

## 四、零值:Go 很多设计都围绕零值展开

Go 的每个类型都有零值:

| 类型 | 零值 |
| --- | --- |
| `int` / `float64` | `0` |
| `bool` | `false` |
| `string` | `""` |
| 指针 | `nil` |
| slice | `nil` |
| map | `nil` |
| channel | `nil` |
| struct | 每个字段都是零值 |

零值让很多类型可以直接使用:

```go
package main

import "fmt"

type Counter struct {
    n int
}

func (c *Counter) Inc() {
    c.n++
}

func main() {
    var c Counter // 不需要构造函数
    c.Inc()
    c.Inc()
    fmt.Println(c.n)
}
```

但不是所有零值都能直接写:

```go
package main

func main() {
    var m map[string]int
    // m["a"] = 1 // panic: assignment to entry in nil map

    m = make(map[string]int)
    m["a"] = 1
}
```

工程判断:

- struct 的零值最好设计成可用状态
- map 写入前必须初始化
- slice 的 nil 零值通常可以直接 append
- 指针、channel、function 的 nil 要谨慎调用

## 五、常量与 iota

常量用 `const` 声明,只能是编译期可确定的值:

```go
package main

import "fmt"

const AppName = "billing"
const TimeoutSeconds = 30

const (
    StatusPending = iota
    StatusPaid
    StatusCanceled
)

func main() {
    fmt.Println(AppName, TimeoutSeconds)
    fmt.Println(StatusPending, StatusPaid, StatusCanceled)
}
```

`iota` 在每个 `const` 块里从 0 开始递增,常用于枚举类常量。

更工程化的写法通常会定义专门类型:

```go
package main

import "fmt"

type OrderStatus int

const (
    OrderPending OrderStatus = iota
    OrderPaid
    OrderCanceled
)

func (s OrderStatus) String() string {
    switch s {
    case OrderPending:
        return "pending"
    case OrderPaid:
        return "paid"
    case OrderCanceled:
        return "canceled"
    default:
        return "unknown"
    }
}

func main() {
    fmt.Println(OrderPaid.String())
}
```

不要滥用裸 `int` 表示业务状态。定义新类型可以让函数签名更清楚。

## 六、if:支持前置语句,但作用域很短

Go 的 `if` 不需要括号,但大括号必须写:

```go
package main

import "fmt"

func main() {
    score := 86

    if score >= 90 {
        fmt.Println("A")
    } else if score >= 80 {
        fmt.Println("B")
    } else {
        fmt.Println("C")
    }
}
```

Go 的 `if` 可以带前置初始化语句:

```go
package main

import (
    "fmt"
    "strconv"
)

func main() {
    if n, err := strconv.Atoi("42"); err == nil {
        fmt.Println(n + 1)
    } else {
        fmt.Println("parse failed:", err)
    }
}
```

这里的 `n` 和 `err` 只在 `if-else` 内部可见。这种写法适合短生命周期变量,尤其是错误处理。

常见坑:

```go
if f, err := openFile(); err != nil {
    return err
} else {
    defer f.Close()
}
// 这里访问不到 f
```

如果后面还要继续用变量,就提前声明:

```go
f, err := openFile()
if err != nil {
    return err
}
defer f.Close()
```

## 七、for:一个循环关键字覆盖所有场景

Go 没有 `while`,只有 `for`。

经典计数循环:

```go
for i := 0; i < 3; i++ {
    fmt.Println(i)
}
```

模拟 while:

```go
n := 0
for n < 3 {
    fmt.Println(n)
    n++
}
```

无限循环:

```go
for {
    break
}
```

遍历集合用 `range`:

```go
package main

import "fmt"

func main() {
    nums := []int{10, 20, 30}

    for i, v := range nums {
        fmt.Println(i, v)
    }

    users := map[string]int{"alice": 18, "bob": 20}
    for name, age := range users {
        fmt.Println(name, age)
    }
}
```

如果不需要索引或值,用 `_` 忽略:

```go
for _, v := range nums {
    fmt.Println(v)
}
```

注意:遍历 map 的顺序是不稳定的,不要依赖 map 输出顺序。

## 八、switch:默认 break,适合表达分支表

Go 的 `switch` 默认不会继续执行下一个 case,所以不需要手写 `break`:

```go
package main

import "fmt"

func main() {
    day := "mon"

    switch day {
    case "sat", "sun":
        fmt.Println("weekend")
    case "mon":
        fmt.Println("start")
    default:
        fmt.Println("workday")
    }
}
```

也可以不带表达式,相当于更清晰的 if-else 链:

```go
package main

import "fmt"

func main() {
    score := 87

    switch {
    case score >= 90:
        fmt.Println("A")
    case score >= 80:
        fmt.Println("B")
    default:
        fmt.Println("C")
    }
}
```

如果确实要继续执行下一个 case,可以用 `fallthrough`,但工程里很少用。多数时候它会降低可读性。

## 九、defer:把清理动作写在资源获取旁边

`defer` 会把函数调用延迟到当前函数返回前执行。它最常用于关闭文件、释放锁、记录耗时、恢复 panic。

```go
package main

import (
    "fmt"
    "os"
)

func main() {
    if err := writeFile("demo.txt", "hello\n"); err != nil {
        fmt.Println("write failed:", err)
        return
    }
    fmt.Println("done")
}

func writeFile(path, content string) error {
    f, err := os.Create(path)
    if err != nil {
        return err
    }
    defer f.Close()

    _, err = f.WriteString(content)
    return err
}
```

多个 `defer` 按 LIFO 顺序执行,后注册的先执行:

```go
package main

import "fmt"

func main() {
    defer fmt.Println("first")
    defer fmt.Println("second")
    defer fmt.Println("third")
}
```

输出:

```text
third
second
first
```

`defer` 的参数会在注册时求值:

```go
package main

import "fmt"

func main() {
    x := 1
    defer fmt.Println(x)
    x = 2
}
```

输出是 `1`,不是 `2`。

## 十、可运行综合例子:解析输入并统计状态

下面这个例子把变量、常量、类型、if、for、switch、map、defer 放在一起:

```go
package main

import (
    "fmt"
    "strconv"
    "strings"
    "time"
)

type Level int

const (
    LevelLow Level = iota
    LevelMedium
    LevelHigh
)

func main() {
    start := time.Now()
    defer func() {
        fmt.Println("cost:", time.Since(start))
    }()

    input := "10,25,80,abc,35"
    levels := map[Level]int{}

    for _, part := range strings.Split(input, ",") {
        n, err := strconv.Atoi(part)
        if err != nil {
            fmt.Println("skip invalid value:", part)
            continue
        }

        level := classify(n)
        levels[level]++
    }

    fmt.Println("low:", levels[LevelLow])
    fmt.Println("medium:", levels[LevelMedium])
    fmt.Println("high:", levels[LevelHigh])
}

func classify(n int) Level {
    switch {
    case n >= 60:
        return LevelHigh
    case n >= 30:
        return LevelMedium
    default:
        return LevelLow
    }
}
```

这段代码没有任何第三方依赖,可以直接 `go run main.go`。

## 十一、常见坑

### 11.1 未使用变量会编译失败

```go
name := "alice"
// 没使用 name 会报错
```

这不是 Go 讨厌你,而是它在阻止死代码和半成品逻辑进入仓库。

### 11.2 类型不会隐式转换

```go
var a int = 10
var b int64 = 20
// fmt.Println(a + b) // 编译失败
fmt.Println(int64(a) + b)
```

显式转换让精度损失和业务假设暴露出来。

### 11.3 `:=` 可能制造变量遮蔽

尤其是 `err :=`。在嵌套作用域里要警惕新建了一个同名变量。

### 11.4 map 遍历无序

如果输出顺序有业务含义,先取 key,排序后再访问。

### 11.5 defer 在循环里可能拖延释放

```go
for _, path := range paths {
    f, err := os.Open(path)
    if err != nil {
        return err
    }
    defer f.Close() // 要等整个函数结束才关闭
}
```

如果循环很多文件,应把每次处理抽成函数,让 defer 在小函数返回时执行。

## 十二、工程判断

写 Go 基础语法时,重点不是"能不能少写几行",而是"下一位维护者能不能立刻看懂"。

几个实践建议:

- 局部变量优先用 `:=`,但不要为了短而牺牲类型意图
- 包级变量少用,共享状态会让测试和并发都变复杂
- 业务枚举不要裸奔,定义专门类型
- `if err != nil` 尽量就近处理,不要吞错
- `defer` 紧跟资源获取,让生命周期一眼可见
- 控制流宁可平铺,不要过度嵌套
- map 输出顺序不要参与业务逻辑
- 设计 struct 时让零值尽量可用

Go 的代码审查通常也围绕这些点:作用域是否过大,错误是否处理,资源是否释放,类型是否表达了业务含义。

## 十三、小结

Go 基础语法的核心不是"简单",而是"受约束的简单"。

你需要记住:

1. 一个目录通常是一个包,`package main` + `func main` 才是可执行程序入口。
2. `var`、`:=`、`const` 分别服务于不同场景,不要滥用。
3. 零值是 Go 的重要设计,但 nil map、nil pointer 不能乱用。
4. `if`、`for`、`switch` 都很少,但足够表达绝大多数业务控制流。
5. `defer` 是资源管理的核心习惯,但要理解它的执行时机和参数求值。
6. Go 更关心工程可读性,不是语法表现力。
