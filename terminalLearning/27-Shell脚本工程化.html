Shell и„ҡжң¬е·ҘзЁӢеҢ–:6 д»¶и®© shell и„ҡжң¬дёҚеҶҚ"еҚҠеқҸ"зҡ„е·ҘзЁӢеҠЁдҪң вҖӢ

е·ҘзЁӢеёҲеҜ№ shell и„ҡжң¬жңҖеӨ§зҡ„иҜҜи§Ј,жҳҜжҠҠе®ғеҪ“жҲҗгҖҢдёҖж¬ЎжҖ§ commit messageгҖҚвҖ”вҖ”йҡҸжүӢеҶҷгҖҒи·‘йҖҡе°ұеҘҪгҖҒдёҚеҶҷжөӢиҜ•гҖҒдёҚи·‘ lintгҖҒеҮәй”ҷи·ҹ git stash дёҖиө·иў«йҒ—еҝҳгҖӮдҪ жү“ејҖиҮӘе·ұ ~/scripts/ жҲ–еӣўйҳҹ ops/ зҡ„зӣ®еҪ•зңӢдёҖзңј,жҢ‘дёҖдёӘ 50 иЎҢд»ҘдёҠзҡ„ .sh еҮәжқҘ:д№қжҲҗд№қж»Ўи¶ідёӢйқўд»»ж„ҸдёҖжқЎ:ж–Үд»¶еӨҙжІЎ set -euo pipefailгҖҒrm -rf "$dir/foo" йҮҢ $dir жІЎеј•еҸ·гҖҒfor f in $(ls) ж’һеҲ°еёҰз©әж јж–Үд»¶еҗҚе°ұзӮёгҖҒдёҙж—¶ж–Үд»¶йқ "зЁӢеәҸе‘ҳи®°еҫ—еҲ "иҖҢдёҚжҳҜ trap жё…зҗҶгҖҒдҫқиө– which python иҝҷз§Қеәҹејғзҡ„жЈҖжөӢгҖҒecho -e еңЁ macOS е’Ң Linux дёҠиЎҢдёәдёҚдёҖж ·гҖӮиҝҷдәӣдёҚжҳҜ"д»Јз ҒйЈҺж ј",жҳҜйҡҗи—Ҹ bugвҖ”вҖ”жң¬ең°и·‘еҫ—еҘҪеҘҪзҡ„,зӣҙеҲ°жҹҗеӨ©иҝӣ CI / иЈ…еҲ°е®ўжҲ·жңәеҷЁ / е‘ҪдёӯжҹҗдёӘиҫ№з•ҢжқЎд»¶,еҮҢжҷЁзӮёдәҶгҖӮиҝҷз§Қ"еҚҠеқҸ"зҠ¶жҖҒеңЁ Python / Go йЎ№зӣ®йҮҢеҮ д№ҺдёҚеӯҳеңЁ(зұ»еһӢзі»з»ҹ + жөӢиҜ•жЎҶжһ¶жҢЎдҪҸдәҶ),дҪҶеңЁ shell йҮҢжҳҜй»ҳи®ӨзҠ¶жҖҒвҖ”вҖ”shell иҜӯжі•е…Ғи®ёеҶҷеҮәиҜӯд№үж··д№ұзҡ„д»Јз Ғ,й»ҳи®Өй…ҚзҪ®е…Ғи®ёе‘Ҫд»ӨеӨұиҙҘд№ӢеҗҺз»§з»ӯеҫҖдёӢи·‘,й»ҳи®ӨиЎҢдёәе…Ғи®ёжңӘе®ҡд№үеҸҳйҮҸеҪ“з©әеӯ—з¬ҰдёІз”ЁгҖӮдҪ еҶҷ shell и„ҡжң¬дёҚдё»еҠЁеҠ йҳІеҫЎ,е°ұжҳҜз»ҷз”ҹдә§жҢ–еқ‘гҖӮ

дёҖеҸҘиҜқе…Ҳи®°дҪҸ:shell и„ҡжң¬и¶…иҝҮ 100 иЎҢе°ұжҳҜй”ҷзҡ„йҖүжӢ©вҖ”вҖ”дёҚжҳҜ shell дёҚиЎҢ,жҳҜе®ғзҡ„й”ҷиҜҜеӨ„зҗҶе’ҢжөӢиҜ•еҹәзЎҖи®ҫж–ҪжҠөдёҚдёҠ Python 5 еҲҶй’ҹжҗӯиө·жқҘгҖӮ100 иЎҢжҳҜиҫ№з•ҢгҖӮ100 иЎҢд»ҘдёӢ,жҠҠе®ғе·ҘзЁӢеҢ–(set -euo pipefail + shellcheck + shfmt + bats + trap + еҸӢеҘҪй”ҷиҜҜдҝЎжҒҜ),е®ғиғҪи·ҹдҪ зҡ„д»Јз ҒдёҖж ·иҝӣ CIгҖҒиў« reviewгҖҒиў« maintain;100 иЎҢд»ҘдёҠ,иҪ¬ Python(subprocess + click + uv 5 еҲҶй’ҹиө·жӯҘ),дёҚиҰҒеңЁ shell йҮҢеҶҷ 200 иЎҢеҸӮж•°и§ЈжһҗгҖҒ200 иЎҢ JSON еӨ„зҗҶгҖҒе…іиҒ”ж•°з»„еөҢеҘ—вҖ”вҖ”shell еңЁиҝҷз§ҚеңәжҷҜиў«и®ҫи®ЎдёҠе°ұдёҚеӨҹгҖӮ

иҝҷдёҖзҜҮжӢҶ 6 дёӘе·ҘзЁӢеҢ–еҠЁдҪң,жҜҸдёҖжқЎй…Қзңҹе®һдәӢж•…еңәжҷҜ:

1. set -euo pipefail + IFS(й»ҳи®Ө shell дёҚе®үе…Ё,иҝҷдёүиЎҢжҳҜе…ҘеңәеҲё)
2. shellcheck(йқҷжҖҒеҲҶжһҗ,80% зҡ„еқ‘иҝҷдёҖе…іе°ұжӢҰдҪҸ)
3. shfmt(ж јејҸеҢ–,еӣўйҳҹи„ҡжң¬йЈҺж јз»ҹдёҖ)
4. trap жё…зҗҶ(и„ҡжң¬жӯ»д№ҹиҰҒе–„еҗҺ)
5. bats еҚ•е…ғжөӢиҜ•(shell и„ҡжң¬д№ҹиғҪжңү test)
6. дҪ•ж—¶иҜҘж”ҫејғ shell еҶҷ Python(зҹҘйҒ“иҫ№з•Ң,жүҚзҹҘйҒ“е“ӘеӨҙжҳҜ shell дё»еңә)

иҜ»е®ҢдҪ иғҪжӢҝиө°дёҖд»Ҫ 30 иЎҢе·ҘзЁӢеҢ–жЁЎжқҝ,д»ҘеҗҺжүҖжңү shell и„ҡжң¬д»Һиҝҷд»ҪжЁЎжқҝжҙҫз”ҹвҖ”вҖ”жҜ”д»Һз©әзҷҪж–Үд»¶ејҖе§Ӣ,иҙЁйҮҸзӣҙжҺҘи·ідёӨжЎЈгҖӮ

дёҖгҖҒдёәд»Җд№Ҳй»ҳи®Ө shell жҳҜдёҚе®үе…Ёзҡ„ вҖӢ

1.1 дёҖж®өзңӢиө·жқҘжІЎй—®йўҳзҡ„и„ҡжң¬ вҖӢ

bash

# deploy.sh - з®ҖеҚ•йғЁзҪІи„ҡжң¬

cd /app
git pull
npm install
npm run build
rm -rf dist/*
cp -r build/* dist/
systemctl restart myapp
echo "Deploy done"

иҝҷ 8 иЎҢйҮҢ,иҮіе°‘жңү 6 дёӘжҪңеңЁ bug:

1. cd /app еӨұиҙҘ(зӣ®еҪ•дёҚеӯҳеңЁ / жқғйҷҗдёҚеӨҹ)вҶ’ дёҚжҠҘй”ҷ,еҗҺз»ӯе‘Ҫд»ӨеңЁй”ҷзҡ„зӣ®еҪ•и·‘
2. git pull еӨұиҙҘ(зҪ‘ж–ӯ / еҶІзӘҒ)вҶ’ дёҚжҠҘй”ҷ,йғЁзҪІзҡ„жҳҜдёҠж¬Ўзҡ„ж—§д»Јз Ғ
3. npm install еӨұиҙҘ вҶ’ дёҚжҠҘй”ҷ,з”Ёж®Ӣзјәдҫқиө–жһ„е»ә
4. npm run build еӨұиҙҘ вҶ’ rm -rf dist/* иҝҳжҳҜжү§иЎҢ,ж—§ dist жІЎдәҶ
5. rm -rf dist/* вҖ”вҖ” еҒҮеҰӮ dist еҸҳйҮҸдёәз©әеұ•ејҖ,зӯүд»· rm -rf /*
6. systemctl restart еӨұиҙҘ(жқғйҷҗ / unit дёҚеӯҳеңЁ)вҶ’ "Deploy done" з…§ж ·жү“,
   дҪ д»ҘдёәжҲҗеҠҹдәҶ

й»ҳи®Ө bash зҡ„иҜӯд№ү:жҜҸдёҖиЎҢе‘Ҫд»ӨеӨұиҙҘ,й»ҳи®ӨдёӢдёҖиЎҢз»§з»ӯи·‘гҖӮиҝҷз§ҚиЎҢдёәеңЁ C йҮҢеҸ« undefined behavior,еңЁ bash йҮҢиў«и§„иҢғжҲҗ"feature"гҖӮз”Ё 1979 е№ҙ Bourne shell зҡ„й»ҳи®ӨиЎҢдёәеҶҷ 2026 е№ҙзҡ„ CI и„ҡжң¬,дәӢж•…жҳҜж—©жҷҡзҡ„гҖӮ

false | echo "hi"
echo "still here"        # д»Қз„¶жү§иЎҢвҖ”вҖ”иҝҷе°ұжҳҜдёәд»Җд№Ҳеҝ…йЎ»й…Қ pipefail
