# Flutter 路由与导航

Flutter 的路由从 `Navigator 1.0`(命令式)发展到 `Navigator 2.0`(声明式),再到社区方案 `go_router`。本篇按"够用就好"的顺序讲。

**结论先放这**:**新项目直接用 `go_router`**,2.0 原生 API 太繁琐。

---

## 一、Navigator 1.0:命令式(够用 80% 场景)

### 直接 push

```dart
Navigator.push(
  context,
  MaterialPageRoute(builder: (_) => const DetailPage()),
);

Navigator.pop(context);                       // 返回
Navigator.pop(context, 'result');             // 带返回值

// 等待返回值
final result = await Navigator.push<String>(
  context,
  MaterialPageRoute(builder: (_) => InputPage()),
);
print(result);
```

### 命名路由

```dart
MaterialApp(
  routes: {
    '/': (_) => HomePage(),
    '/detail': (_) => DetailPage(),
    '/settings': (_) => SettingsPage(),
  },
)

// 跳转
Navigator.pushNamed(context, '/detail');
Navigator.pushNamed(context, '/detail', arguments: {'id': 42});

// 接收参数
final args = ModalRoute.of(context)!.settings.arguments as Map;
```

### 常用方法

| 方法 | 作用 |
| --- | --- |
| `push` | 入栈一个页面 |
| `pop` | 出栈 |
| `pushReplacement` | 替换当前页(登录成功后用) |
| `pushAndRemoveUntil` | 清栈到某条件,再 push |
| `popUntil` | 弹栈到某条件 |
| `pushNamed` | 命名版 push |

```dart
// 登录成功:替换登录页 → 首页
Navigator.pushReplacementNamed(context, '/home');

// 登出:清空栈 → 登录页
Navigator.pushNamedAndRemoveUntil(
  context,
  '/login',
  (route) => false,
);
```

### 嵌套 Navigator

底部 Tab 切换时各 Tab 各自有路由栈:

```dart
Scaffold(
  body: IndexedStack(
    index: _index,
    children: [
      Navigator(
        key: _tab1NavKey,
        onGenerateRoute: (s) => MaterialPageRoute(builder: (_) => Tab1Page()),
      ),
      Navigator(
        key: _tab2NavKey,
        onGenerateRoute: (s) => MaterialPageRoute(builder: (_) => Tab2Page()),
      ),
    ],
  ),
  bottomNavigationBar: BottomNavigationBar(...),
)
```

每个 Tab 的页面栈独立,切回 Tab 还是原来的位置。

## 二、Navigator 1.0 的局限

它是**命令式**的:Push 一下,新页面被推上来。问题:

- **Web URL 同步差**:浏览器后退/前进、刷新都对不上
- **深度链接处理麻烦**:`/user/123/posts/456` 要自己拆 URL
- **状态恢复难**:App 杀进程后回来,之前的路径丢了

这就是 Navigator 2.0 出现的原因。

---

## 三、Navigator 2.0(原生声明式,很啰嗦)

核心思想:**用一个 list 描述当前路由栈**,栈变了页面跟着变。

```dart
Navigator(
  pages: [
    MaterialPage(child: HomePage()),
    if (selectedId != null) MaterialPage(child: DetailPage(selectedId!)),
  ],
  onPopPage: (route, result) {
    if (!route.didPop(result)) return false;
    setState(() => selectedId = null);
    return true;
  },
)
```

需要自己写 `RouterDelegate`、`RouteInformationParser`、`RouteInformationProvider` 三个类,**模板代码巨多**,所以社区直接用 `go_router` 这种封装。

## 四、go_router(推荐方案)

它是 Flutter 官方支持的、基于 Navigator 2.0 的封装,API 接近 Web Router。

```yaml
dependencies:
  go_router: ^14.2.0
```

### 最简配置

```dart
final router = GoRouter(
  routes: [
    GoRoute(path: '/', builder: (_, __) => HomePage()),
    GoRoute(path: '/detail', builder: (_, __) => DetailPage()),
  ],
);

void main() {
  runApp(MaterialApp.router(            // 注意是 .router
    routerConfig: router,
  ));
}
```

### 跳转

```dart
context.go('/detail');                  // 替换栈
context.push('/detail');                // 入栈
context.pop();                          // 出栈
context.replace('/detail');             // 替换当前
```

### 路径参数 + Query

```dart
GoRoute(
  path: '/user/:id',
  builder: (context, state) {
    final id = state.pathParameters['id']!;
    final from = state.uri.queryParameters['from'];
    return UserPage(id: id);
  },
)

// 跳转
context.go('/user/42?from=home');
```

### 嵌套路由

```dart
GoRoute(
  path: '/user/:id',
  builder: (_, state) => UserPage(id: state.pathParameters['id']!),
  routes: [
    GoRoute(
      path: 'posts',                    // 注意没有前导 /
      builder: (_, state) => PostsPage(userId: state.pathParameters['id']!),
    ),
  ],
)

// /user/42/posts
```

### ShellRoute:带底部导航的嵌套

```dart
ShellRoute(
  builder: (context, state, child) {
    return Scaffold(
      body: child,                       // 子页面在这里渲染
      bottomNavigationBar: BottomNavigationBar(
        currentIndex: _calcIndex(state.uri.toString()),
        onTap: (i) => context.go(['/home', '/cart', '/me'][i]),
        items: [...],
      ),
    );
  },
  routes: [
    GoRoute(path: '/home', builder: (_, __) => HomePage()),
    GoRoute(path: '/cart', builder: (_, __) => CartPage()),
    GoRoute(path: '/me', builder: (_, __) => MePage()),
  ],
)
```

切换 Tab 时只换 `child`,Scaffold 不动。

### StatefulShellRoute:每个 Tab 独立栈(更常用)

底部导航场景必备:每个 Tab 的导航栈独立,切回去还在原来的位置。

```dart
StatefulShellRoute.indexedStack(
  builder: (context, state, navigationShell) {
    return Scaffold(
      body: navigationShell,
      bottomNavigationBar: BottomNavigationBar(
        currentIndex: navigationShell.currentIndex,
        onTap: (i) => navigationShell.goBranch(i),
        items: [...],
      ),
    );
  },
  branches: [
    StatefulShellBranch(routes: [GoRoute(path: '/home', ...)]),
    StatefulShellBranch(routes: [GoRoute(path: '/cart', ...)]),
    StatefulShellBranch(routes: [GoRoute(path: '/me', ...)]),
  ],
)
```

### 路由守卫(Redirect)

未登录拦截到登录页:

```dart
GoRouter(
  redirect: (context, state) {
    final loggedIn = AuthService.instance.isLoggedIn;
    final goingToLogin = state.matchedLocation == '/login';

    if (!loggedIn && !goingToLogin) return '/login';
    if (loggedIn && goingToLogin) return '/home';
    return null;                         // 不重定向
  },
  routes: [...],
)
```

也可以放在单个 `GoRoute` 上,只拦该路由。

### 错误页

```dart
GoRouter(
  errorBuilder: (_, state) => Scaffold(
    body: Center(child: Text('404:${state.uri}')),
  ),
  routes: [...],
)
```

### 监听路由变化(配合状态管理)

```dart
class AuthService extends ChangeNotifier {
  bool isLoggedIn = false;
  void login() { isLoggedIn = true; notifyListeners(); }
}

GoRouter(
  refreshListenable: AuthService.instance,   // ← 它变了,重新跑 redirect
  redirect: ...,
)
```

登录后立刻跳走,登出后立刻拦截,完全自动。

## 五、深度链接 / 网页 URL 同步

### Web

go_router 自动同步浏览器 URL,刷新页面、前进/后退都正确。
默认带 `/#/` 前缀,去掉:

```dart
import 'package:flutter_web_plugins/url_strategy.dart';

void main() {
  usePathUrlStrategy();             // 用干净的 URL
  runApp(MyApp());
}
```

### iOS / Android 深度链接

需要在原生工程配置 URL Scheme / Universal Link / App Link,然后 `go_router` 会自动接收 URL。

`AndroidManifest.xml`:

```xml
<intent-filter>
  <action android:name="android.intent.action.VIEW" />
  <category android:name="android.intent.category.DEFAULT" />
  <category android:name="android.intent.category.BROWSABLE" />
  <data android:scheme="https" android:host="myapp.com" />
</intent-filter>
```

iOS Info.plist 配置 Associated Domain。

进入 App 后 URL 会自动喂给 go_router,**不需要手写解析代码**。

## 六、传参数的几种方式

| 方式 | 写法 | 适用 |
| --- | --- | --- |
| 路径参数 | `/user/:id` → `state.pathParameters['id']` | URL 友好的 ID |
| Query 参数 | `?from=home` → `state.uri.queryParameters['from']` | 可选筛选 |
| extra(任意对象) | `context.go('/edit', extra: user)` → `state.extra as User` | 整个对象传过去(不会出现在 URL) |

⚠️ `extra` 不能 web 刷新恢复(因为不在 URL 里),仅限 App 内部跳转。

## 七、go_router 完整示例

```dart
final router = GoRouter(
  initialLocation: '/home',
  refreshListenable: authService,
  redirect: (context, state) {
    final logged = authService.isLoggedIn;
    final atLogin = state.matchedLocation == '/login';
    if (!logged && !atLogin) return '/login';
    if (logged && atLogin) return '/home';
    return null;
  },
  routes: [
    GoRoute(path: '/login', builder: (_, __) => const LoginPage()),

    StatefulShellRoute.indexedStack(
      builder: (_, __, shell) => MainScaffold(shell: shell),
      branches: [
        StatefulShellBranch(
          routes: [
            GoRoute(
              path: '/home',
              builder: (_, __) => const HomePage(),
              routes: [
                GoRoute(
                  path: 'article/:id',
                  builder: (_, s) => ArticlePage(id: s.pathParameters['id']!),
                ),
              ],
            ),
          ],
        ),
        StatefulShellBranch(routes: [
          GoRoute(path: '/cart', builder: (_, __) => const CartPage()),
        ]),
        StatefulShellBranch(routes: [
          GoRoute(path: '/me', builder: (_, __) => const MePage()),
        ]),
      ],
    ),
  ],
  errorBuilder: (_, state) => NotFoundPage(uri: state.uri),
);
```

## 八、常见坑

### 1. context.go vs context.push

- `go`:**替换栈**,清掉当前所有,只剩目标
- `push`:**入栈**,可以 pop 回来

底部 Tab 切换用 `go`,详情页用 `push`。

### 2. 跨层调用 Navigator

```dart
Navigator.of(context).pop();         // ⚠️ 在嵌套 Navigator 里可能 pop 错栈
```

`Navigator.of(context, rootNavigator: true)` 可以指定到根 Navigator。
`go_router` 里直接用 `context.pop()` 通常是对的。

### 3. 异步后用 context

```dart
await api.save();
context.go('/home');                 // ⚠️ Widget 可能已销毁
```

加 `if (!context.mounted) return;` 守卫(回顾 04 那篇)。

### 4. 弹窗用 Navigator 还是 go_router?

弹窗(`showDialog`、`showModalBottomSheet`)走 `Navigator`,**不要走 go_router**——弹窗本质是 Overlay,不是路由。

```dart
showDialog(context: context, builder: (_) => AlertDialog(...));   // ✅
```

### 5. Web 刷新后 extra 丢失

```dart
context.go('/edit', extra: user);    // ⚠️ Web 刷新后 user 是 null
```

Web 想保留就放 query 或路径参数,extra 只在 App 内可靠。

## 九、什么时候用什么?

| 场景 | 推荐 |
| --- | --- |
| 简单 App、几个页面 | Navigator 1.0 + 命名路由 |
| 中型 App、底部 Tab、深度链接 | **go_router**(强烈) |
| 大型 App、Web、复杂 URL | go_router + 模块化 |
| Tab 内独立栈 | `StatefulShellRoute` |
| 已用 GetX | `Get.to / Get.toNamed`(GetX 自带路由) |

## 十、心智模型

```
Navigator 1.0  = 命令式:push、pop、改栈
Navigator 2.0  = 声明式:用 list 描述栈
go_router      = 声明式 + URL 同步 + 守卫,推荐默认选择
```

**核心选型公式**:**新项目用 go_router,老项目维持现状**。

---

## 十一、和已学知识的串联

- `Navigator.of(context)` 通过 context **向上**找最近的 Navigator(回顾 04)
- 路由跳转跟 BlocListener / ref.listen 配合最香:Cubit 出 Success 状态,listener 里 `context.go('/home')`
- 嵌套 Navigator 的"独立栈"本质是利用了 Element/State 的复用机制(回顾 05)

路由其实就是另一种"状态" —— **当前栈**。理解了 Flutter 的 Widget 树,路由就是一棵特殊的子树。
