# Flutter 手势系统

Flutter 处理触摸 / 鼠标输入分两层:

```
高层:GestureDetector / InkWell / Dismissible / Draggable
低层:Listener / RawGestureDetector / GestureRecognizer
```

90% 场景用高层,**只有冲突 / 自定义手势**才下到低层。

---

## 一、GestureDetector:最常用

```dart
GestureDetector(
  onTap: () => print('点击'),
  onDoubleTap: () => print('双击'),
  onLongPress: () => print('长按'),

  onTapDown: (d) => print('按下 ${d.globalPosition}'),
  onTapUp: (d) => print('抬起'),
  onTapCancel: () => print('取消'),

  // 滑动
  onPanStart: (d) => ...,
  onPanUpdate: (d) {
    print('移动 ${d.delta}');
  },
  onPanEnd: (d) => print('松手 速度 ${d.velocity}'),

  // 拖拽方向
  onHorizontalDragUpdate: (d) => ...,
  onVerticalDragUpdate: (d) => ...,

  // 缩放
  onScaleStart: (d) => ...,
  onScaleUpdate: (d) => print('${d.scale} ${d.rotation}'),
  onScaleEnd: (d) => ...,

  child: Container(width: 100, height: 100, color: Colors.blue),
)
```

注意:`GestureDetector` 没有视觉反馈,**点击没水波纹**。

---

## 二、InkWell:点击有水波纹

Material 风格,适合按钮、卡片等:

```dart
InkWell(
  onTap: () {},
  borderRadius: BorderRadius.circular(8),
  child: Container(
    padding: const EdgeInsets.all(16),
    child: const Text('点我'),
  ),
)
```

⚠️ **InkWell 必须在 Material 祖先内才显示水波纹**。Container + InkWell 时水波纹被遮挡是常见坑:

```dart
// ❌ 水波纹被 Container 颜色挡住
Container(
  color: Colors.white,
  child: InkWell(onTap: () {}, child: const Text('hi')),
)

// ✅ 用 Material + InkWell
Material(
  color: Colors.white,
  child: InkWell(onTap: () {}, child: const Text('hi')),
)

// 或 Card / ListTile 内部已经有 Material
Card(
  child: InkWell(onTap: () {}, child: const Text('hi')),
)
```

## 三、各种手势的细节

### onTap 与 onTapDown 的差异

```dart
GestureDetector(
  onTapDown: (d) => print('1. 手指按下'),
  onTapUp:   (d) => print('2. 手指抬起'),
  onTap:     ()  => print('3. 完成点击'),
  onTapCancel: () => print('被竞争走或者拖出范围,取消'),
)
```

`onTap` 是"完整的点击事件":手指按下 → 抬起 → 没移动太多 → 才算 tap。
`onTapDown` 立刻触发,**适合做按下的视觉反馈**。

### onPan vs onHorizontalDrag vs onVerticalDrag

| 手势 | 行为 |
| --- | --- |
| `onPan*` | 任意方向拖动 |
| `onHorizontalDrag*` | 仅水平,垂直会冲突让出 |
| `onVerticalDrag*` | 仅垂直 |

不能同时设 `onPan` 和 `onHorizontalDrag`,会报错。

## 四、手势冲突(竞争系统)

Flutter 用 **GestureArena(手势竞技场)** 决定谁赢。

经典冲突:

```dart
GestureDetector(                    // 父
  onTap: () => print('父 tap'),
  child: GestureDetector(           // 子
    onTap: () => print('子 tap'),
    child: Container(...),
  ),
)
```

**默认子级赢**——离手指最近的赢。

但如果父级 `onPan` 子级 `onTap`,**有歧义**:手指动了几像素属于 pan 还是 tap?Flutter 用"延迟+判定"机制,根据动作特征判断。

### 让子级强制赢

```dart
GestureDetector(
  behavior: HitTestBehavior.opaque,    // 即使透明区域也接收
  onTap: ...,
  child: ...,
)
```

`HitTestBehavior`:
- `deferToChild`(默认):有 child 时由 child 决定
- `opaque`:整个区域都响应,不管透明
- `translucent`:自己接收的同时,事件继续往下传

## 五、Listener:最低层指针事件

不参与手势竞争,直接拿原始指针事件:

```dart
Listener(
  onPointerDown: (e) => print('按下 ${e.position}'),
  onPointerMove: (e) => print('移动'),
  onPointerUp:   (e) => print('抬起'),
  onPointerCancel: (e) => print('取消'),
  child: Container(...),
)
```

适合**完全自定义手势**(画板、绘图、特殊交互)。日常不用。

---

## 六、AbsorbPointer / IgnorePointer:屏蔽手势

```dart
// 完全吃掉所有手势,自己也不响应
AbsorbPointer(
  absorbing: _disabled,
  child: ElevatedButton(onPressed: () {}, child: const Text('OK')),
)

// 自己和子级都不接收
IgnorePointer(
  ignoring: _disabled,
  child: ...,
)
```

| Widget | 子级响应? | 自己响应? |
| --- | --- | --- |
| `AbsorbPointer` | ❌ | ✅ |
| `IgnorePointer` | ❌ | ❌ |

例:loading 时不允许点击:

```dart
Stack(children: [
  myUI,
  if (loading) AbsorbPointer(child: Container(color: Colors.black54)),
])
```

## 九、Dismissible:滑动删除

```dart
Dismissible(
  key: ValueKey(item.id),
  direction: DismissDirection.endToStart,    // 向左滑
  background: Container(color: Colors.red),
  confirmDismiss: (direction) async {
    return await showDialog<bool>(
      context: context,
      builder: (_) => AlertDialog(
        content: const Text('确定删除?'),
        actions: [
          TextButton(child: const Text('取消'), onPressed: () => Navigator.pop(context, false)),
          TextButton(child: const Text('删除'), onPressed: () => Navigator.pop(context, true)),
        ],
      ),
    );
  },
  onDismissed: (direction) {
    setState(() => items.remove(item));
  },
  child: ListTile(...),
)
```

## 十、Draggable / DragTarget:拖放

```dart
// 可拖
Draggable<String>(
  data: 'apple',
  feedback: const Icon(Icons.apple, size: 50),    // 拖动中的样子
  childWhenDragging: const Opacity(opacity: 0.3, child: Icon(Icons.apple)),
  child: const Icon(Icons.apple),
)

// 接收
DragTarget<String>(
  onAcceptWithDetails: (details) {
    setState(() => _dropped.add(details.data));
  },
  builder: (_, candidate, rejected) {
    return Container(
      width: 200, height: 200,
      color: candidate.isNotEmpty ? Colors.green : Colors.grey,
      child: const Center(child: Text('拖到这里')),
    );
  },
)
```

适合:看板拖卡片、文件拖入区域、图片拼图。

## 十二、自定义手势识别:RawGestureDetector

完全自定义识别逻辑(复杂场景才用):

```dart
RawGestureDetector(
  gestures: {
    AllowMultipleGestureRecognizer:
      GestureRecognizerFactoryWithHandlers<AllowMultipleGestureRecognizer>(
        () => AllowMultipleGestureRecognizer(),
        (instance) => instance.onTap = () => print('tap'),
      ),
  },
  child: ...,
)

class AllowMultipleGestureRecognizer extends TapGestureRecognizer {
  @override
  void rejectGesture(int pointer) {
    acceptGesture(pointer);    // 强制赢
  }
}
```

**慎用**,理解手势竞技场之后再用。

## 十四、HitTestBehavior 深入

```dart
// 透明区域不响应
GestureDetector(
  behavior: HitTestBehavior.deferToChild,    // 默认
  child: SizedBox(width: 100, height: 100),  // 透明,无 child → 点击穿过
)

// 透明区域也响应
GestureDetector(
  behavior: HitTestBehavior.opaque,
  child: SizedBox(width: 100, height: 100),  // 即使透明,点击触发
)
```

加 `behavior: opaque` 是常用技巧,**让有 child 但 child 透明的区域也能点击**。

## 十五、点击区域过小

UI 设计可能要求"小图标 + 大点击区域"。**别加大图标,加大点击区**:

```dart
// 用 InkWell 自动撑满
SizedBox(
  width: 48, height: 48,
  child: InkWell(
    onTap: () {},
    child: const Icon(Icons.menu, size: 20),
  ),
)
```

或 `IconButton`(默认 48x48):

```dart
IconButton(
  iconSize: 20,         // 图标 20,点击区 48
  icon: const Icon(Icons.menu),
  onPressed: () {},
)
```

## 十六、ListTile 整行点击

```dart
ListTile(
  onTap: () {},          // 整行可点
  trailing: IconButton(  // 行内还能放可点的子组件
    icon: const Icon(Icons.delete),
    onPressed: () => _delete(),    // 不会冒泡到 onTap
  ),
)
```

子级 GestureDetector 会"吃掉"事件,父级不会再触发。这是默认行为,正常 UI 设计要的就是这个。

## 十七、滑动方向锁定

如果 ListView 内部子级有水平滑动需求,但 ListView 本身是垂直的,会冲突。

```dart
ListView(children: [
  // 一个 PageView 横向滚
  SizedBox(
    height: 200,
    child: PageView(...),     // ✅ 横向手势归 PageView
  ),
])
```

Flutter 自动判断手势方向归谁。但**纵向滚动的 ListView 里嵌纵向 ListView** 就要小心,通常用 `NeverScrollableScrollPhysics` 让内层不滚。

## 十八、常见坑

### 1. InkWell 没水波纹

→ 上面没 Material 包裹,或者被 Container 颜色挡了。

### 2. 手势透传不到下层

→ 加 `behavior: HitTestBehavior.translucent`。

### 3. 父 onTap 子 onTap 冲突

→ 默认子级赢,父级不触发。要双方都触发,父级用 `Listener` 拿原始事件。

### 4. 长按之后 onTap 也触发

→ Flutter 默认会判定:**长按触发后,onTap 不会再触发**。如果发现两个都触发,检查是不是嵌套了多个 GestureDetector。

### 5. 列表里点击没反应

→ 检查父级是不是有 `IgnorePointer` / `AbsorbPointer`。

### 6. iOS 长按掉事件

iOS 长按超过一定时间 Flutter 会让出给系统(放大镜效果),用 `LongPressGestureRecognizer` 自定义。

### 7. 拖动很卡

`onPanUpdate` 里做重活会卡。把数据更新放到 `setState` 外:

```dart
onPanUpdate: (d) {
  _offset += d.delta;       // 累加但不 setState
  _throttle.run(() => setState(() => _ui = _offset));  // 节流
}
```

或用 `ValueNotifier` + `AnimatedBuilder`(回顾 02 / 18)做局部刷新。

## 二十一、推荐选择

```
点击 / 长按           → InkWell(有水波纹)/ GestureDetector(无)
按钮                  → ElevatedButton 等(已包了点击 + 视觉反馈)
列表整行点击          → ListTile.onTap
左右滑删除            → Dismissible
拖排序                → ReorderableListView
拖放(看板)          → Draggable + DragTarget
hover                → MouseRegion(桌面)
快捷键                → Shortcuts + Actions
完全自定义手势        → RawGestureDetector / Listener
```

## 二十二、心智模型

```
事件流向:
  原始指针(Listener)
    ↓
  手势识别(GestureRecognizer 在竞技场竞争)
    ↓
  GestureDetector 回调

手势竞争原则:
  - 离手指最近的 GestureDetector 优先
  - 不同手势(tap / pan / scale)按特征区分
  - 模糊场景:延迟 + 速度 + 距离判定
```

**绝大多数手势用 InkWell / GestureDetector 就够**。理解了竞技场,你能解释为什么手势"不响应""触发错位置""父子冲突"——基本就告别 90% 手势 bug 了。
