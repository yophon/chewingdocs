## 一、共通基础架构

任何中大型 Flutter 项目的骨架基本一致:

```
lib/
├── main.dart                    # 入口,初始化
├── bootstrap.dart               # 全局初始化(Sentry/DI/i18n)
├── app.dart                     # MaterialApp + Router + Theme
├── core/                        # 业务无关
│   ├── network/                 # Dio interceptor / cache
│   ├── storage/                 # Drift / Hive 封装
│   ├── di/                      # GetIt / Riverpod providers
│   ├── error/                   # Failure 体系
│   ├── analytics/               # 埋点封装
│   └── utils/
├── features/                    # 按业务垂直拆分
│   ├── auth/
│   │   ├── data/                # repository + datasource
│   │   ├── domain/              # entity + usecase
│   │   └── presentation/        # page + state
│   ├── home/
│   ├── cart/
│   └── ...
├── shared/                      # 跨 feature 复用
│   ├── widgets/
│   └── models/
└── l10n/
```

## 二、电商 App

### 1. 业务地图

```
启动 → 引导 → 登录(可游客)
       ↓
首页(瀑布 / 推荐 / Banner)
   ├─ 分类 / 搜索
   ├─ 商品详情(图文 / 规格 / SKU / 评价)
   ├─ 加购物车
   ├─ 下单(地址 / 配送 / 优惠券 / 支付)
   ├─ 订单列表 / 物流
   ├─ 收藏 / 浏览历史
   └─ 我的(资产 / 设置 / 客服)
```

### 2. 技术选型

| 模块 | 选型 | 原因 |
| --- | --- | --- |
| 状态管理 | Riverpod / Bloc | 复杂,模块多 |
| 路由 | GoRouter | 深链接 / 分享 / 推送跳转 |
| 网络 | Dio + retrofit + freezed | 接口多,类型安全 |
| 缓存 | Hive(用户 / 设置) + 内存 | 列表数据走 SWR |
| 图片 | cached_network_image + photo_view | 详情页大图 |
| 支付 | 微信 / 支付宝 / Apple Pay 各 SDK | 国内必备 |
| IM(客服) | 接第三方(融云 / 网易) | 自研不值 |
| 统计 | 友盟 / GA / 自家埋点 | 漏斗分析 |
| 地图 | 高德 / Google Maps | 地址定位 |

### 4. 商品列表无限滚动 + SWR

```dart
class ProductsNotifier extends AsyncNotifier<PagedList<Product>> {
  @override
  Future<PagedList<Product>> build() => _load(page: 1);

  Future<void> loadMore() async {
    final cur = state.value!;
    final next = await _load(page: cur.page + 1);
    state = AsyncData(cur.copyWith(
      page: next.page,
      items: [...cur.items, ...next.items],
      hasMore: next.hasMore,
    ));
  }
}

// UI
final list = ref.watch(productsProvider);
list.when(
  data: (paged) => ListView.builder(
    itemCount: paged.items.length + 1,
    itemBuilder: (_, i) {
      if (i == paged.items.length) {
        ref.read(productsProvider.notifier).loadMore();
        return Loader();
      }
      return ProductCard(paged.items[i]);
    },
  ),
  ...
);
```

### 5. SKU / 多规格选择

经典模型:

```dart
class Product {
  final List<SkuAttr> attrs;       // [{颜色: [红,白,黑]}, {尺寸: [S,M,L]}]
  final List<Sku> skus;             // [{颜色:红, 尺寸:M, 价格,库存}, ...]

// 用户选择某属性 → 计算"剩下哪些 SKU 可选" → 灰掉不可选项
```

UI 组合用 Wrap + ChoiceChip,逻辑用纯函数,容易单测。

### 6. 下单流程

```
购物车选品 → 创建预订单(后端返回:商品快照、可用券、支付方式)
        → 用户确认地址、券、备注
        → 提交订单(后端创建订单 + 锁库存)
        → 跳支付(SDK)
        → 支付回调 / 轮询订单状态
        → 跳订单详情
```

支付**绝不能信任客户端结果**:回调以服务端通知 + 订单状态查询为准。

### 7. 性能要点

- 首页 + 详情页:首屏图片 LQIP / blurhash 占位
- 列表项 widget 严格 const + key
- 商品详情图懒加载,详情大图用 photo_view 单独路由
- 网络列表:dio cache 中间件 + 本地 SWR
- 包体积:cached_network_image + flutter_image_compress

### 8. 推送(回顾 26)

- 订单状态变化 → 服务端推
- 心愿单降价 → 定时任务 + 推送
- 个性化运营 → 标签推送

## 三、社交 App(图文 / 短视频)

### 1. 业务地图

```
启动 → 登录(手机/Apple/Google)
       ↓
推荐 Feed(无限滚动)
   ├─ 关注 Feed
   ├─ 发布(图文 / 视频 / 草稿)
   ├─ 个人主页(我 / 他人)
   ├─ 私信(IM)
   ├─ 通知(评论 / 点赞 / 关注)
   └─ 发现(话题 / 搜索)
```

### 2. 技术选型

| 模块 | 选型 |
| --- | --- |
| Feed 流 | Bloc / Riverpod + 自定义无限列表 |
| 视频播放 | better_player / video_player + ExoPlayer / AVPlayer |
| 视频拍摄 | camera + ffmpeg_kit_flutter(剪辑) |

| 富文本 | flutter_quill / flutter_markdown |
| 长连接 | WebSocket / Socket.IO(回顾 42) |
| Feed 缓存 | Drift(SQLite),离线可看 |
| AB 测试 | Firebase Remote Config |

视频控制器池(避免疯狂创建):

```dart
class VideoControllerPool {
  final _cache = LinkedHashMap<String, VideoPlayerController>(maximumSize: 5);

  VideoPlayerController get(String url) {
    if (_cache.containsKey(url)) return _cache[url]!;
    if (_cache.length >= 5) _cache.remove(_cache.keys.first)!.dispose();
    final c = VideoPlayerController.networkUrl(Uri.parse(url));
    _cache[url] = c;
    return c;
  }
}
```

### 4. 发布流程

```
拍摄 / 选择 → 编辑(裁剪 / 滤镜 / 文字 / 音乐)
          → 上传(本地保存草稿 + 后台上传)
          → 创建动态 / 推荐
```

**上传必须后台进行**:用户切走 / 关后台不影响。Android 用 `WorkManager`,iOS 用 `NSURLSession` 后台,Flutter 包 `background_uploader` / 自己写。

## 四、IM App

### 1. 业务地图

```
启动 → 登录
   ├─ 会话列表(最近联系人 + 未读)
   ├─ 单聊 / 群聊
   ├─ 通讯录 / 好友请求
   ├─ 消息(文字 / 图片 / 语音 / 视频 / 文件 / 位置 / 名片)
   ├─ 设置 / 隐私
   └─ 推送 / 离线消息
```

### 2. 技术选型

| 模块 | 选型 |
| --- | --- |
| 长连接 | WebSocket / Socket.IO / 自研 TCP |
| 协议 | Protobuf / JSON | 大量消息建议 Protobuf |
| 本地存储 | Drift(SQLite,索引) | 对消息查询 / 分页要求高 |
| 消息列表 | 倒序 ListView + 分页 + 反向预加载 |
| 推送 | FCM + 厂商通道(回顾 26) |
| 媒体 | 图片走 CDN,语音 / 视频走对象存储 |
| 第三方方案 | 融云 / 网易云信 / Tencent IM / Sendbird | 不想自研就用 |

### 3. 消息存储模型

```sql
CREATE TABLE messages (
  id TEXT PRIMARY KEY,
  conv_id TEXT NOT NULL,
  sender_id TEXT NOT NULL,
  type INTEGER,                -- 1=text 2=image 3=audio ...
  content TEXT,
  created_at INTEGER,
  status INTEGER,              -- 0=sending 1=sent 2=delivered 3=read 4=fail
  local_id TEXT                -- 客户端临时 id,服务端确认后替换
);

CREATE INDEX idx_conv_time ON messages(conv_id, created_at DESC);
```

### 4. 发送消息状态机

```
[本地写入 sending]
    ↓ socket.send
[发出 → 等 ack]
    ↓ ack
[sent]
    ↓ 对方设备拉取
[delivered]
    ↓ 对方读
[read]

若超时无 ack → status = fail,UI 显示重发按钮
```

```dart
Future<void> sendText(String convId, String text) async {
  final msg = Message.local(convId: convId, type: 1, content: text);
  await db.insertMessage(msg);                                 // 1
  ref.read(convProvider(convId).notifier).appendOptimistic(msg);

  try {
    final ack = await socket.send(msg).timeout(15.seconds);
    await db.updateStatus(msg.localId, status: 1, serverId: ack.id);
  } catch (e) {
    await db.updateStatus(msg.localId, status: 4);
  }
}
```

### 5. 长连接

```
App 在前台              → 长连
App 后台 / 锁屏(iOS)  → 走 APNs(系统级)
App 杀掉(Android)     → 走厂商通道
重连策略             → 指数退避 + jitter,网络变化重连
心跳                 → 30s 一次,3 次失败重连
```

回顾 42(WebRTC 与 IM)有更详细的 socket 客户端实现。

### 6. 离线消息

```
登录后调 syncMessages(lastSeq)
  ↓
服务端返 lastSeq 之后的消息(带 seq 单调递增)
  ↓
本地按 seq 写入,跳过重复
  ↓
更新 lastSeq
```

**永远以 server seq 为准**,客户端时间不可靠。

### 7. 会话列表

```dart
SELECT c.id, c.title,
       (SELECT m.content FROM messages m
        WHERE m.conv_id = c.id ORDER BY created_at DESC LIMIT 1) AS lastMsg,
       (SELECT COUNT(*) FROM messages m
        WHERE m.conv_id = c.id AND status < 3 AND m.sender_id != ?) AS unread
FROM conversations c
ORDER BY (SELECT MAX(created_at) FROM messages WHERE conv_id = c.id) DESC;
```

或维护 `conversations` 表反范式 (lastMsg, unread 字段),消息变化时同步更新。**消息几十万条后实时聚合查询会慢**,反范式必须做。

### 8. 富消息渲染

每种 type 对应一个 Widget Builder:

```dart
Widget buildMessage(Message m) {
  return switch (m.type) {
    1 => TextBubble(m.content),
    2 => ImageBubble(m.url),
    3 => AudioBubble(m.url, duration: m.meta['dur']),
    4 => VideoBubble(m.url, thumb: m.meta['thumb']),
    5 => FileBubble(m.meta['name'], size: m.meta['size']),
    6 => LocationBubble(m.meta['lat'], m.meta['lng']),
    _ => UnknownBubble(),
  };
}
```

### 9. 群聊

群消息 = 单聊 + `groupId`。性能关键:
- 服务端写一份(群消息),客户端按 group + seq 拉
- "已读"不要每个成员单独存,**只存"我读到的 seq"**
- 群头像九宫格用九张小图合成或服务端预生成

### 10. 端到端加密(可选高级)

Signal Protocol(双 Ratchet):每条消息一把 key,即便服务器被入侵也读不到。
Flutter 端有 `libsignal_protocol_dart`,但工程量大。**普通商业 IM 通常不需要 E2EE**,做 TLS + 服务端加密落库即可。

## 五、三类项目的难点对比

| 难点 | 电商 | 社交 | IM |
| --- | --- | --- | --- |
| 数据量 | 中 | 大(Feed 流) | 极大(消息) |
| 实时性 | 低(订单状态) | 中(点赞 / 评论) | 高(秒级到达) |
| 离线 | 浏览缓存即可 | 草稿 + 缓存 | 必须能离线看历史 |
| 多媒体 | 图为主 | 图 + 视频 | 图 + 音 + 视 + 文件 |
| 后端复杂度 | 高(库存 / 支付 / 物流) | 高(推荐 / 审核) | 极高(长连接 / 分布式) |
| 单端难点 | 多 SKU / 支付 | Feed 流性能 | 长连接稳定 + 本地索引 |

## 七、模块抽象示例(电商版)

```dart
// domain
abstract class CartRepository {
  Stream<Cart> watch();
  Future<void> add(Sku sku, int qty);
  Future<void> remove(String skuId);
  Future<void> clear();
}

// data
class CartRepositoryImpl implements CartRepository {
  CartRepositoryImpl(this._api, this._db);
  final CartApi _api;
  final CartDao _db;

  @override
  Stream<Cart> watch() => _db.watch();

  @override
  Future<void> add(Sku sku, int qty) async {
    await _db.upsert(sku.id, qty);            // 本地立即
    try { await _api.add(sku.id, qty); }      // 异步同步
    catch (e) { /* 排队重试 */ }
  }
}

// presentation
final cartProvider = StreamProvider<Cart>((ref) {
  return ref.read(cartRepoProvider).watch();
});
```

各 feature 都按这模式拆,新人能 5 分钟懂。

## 八、心智模型

```
任何商业 App = 数据 + 状态 + UI + 网络 + 持久化 + 监控

按业务划分 feature        →  按技术划分核心(core)
保持 feature 之间无引用   →  共享走 core 或 shared
统一异常处理 / 网络拦截   →  避免每页重复 try/catch

电商:从"商品到支付"的漏斗优化
社交:从"内容到互动"的留存优化
IM:从"长连到离线"的可靠性建设
```

**做完 1-39 的笔记后,再串 40 这一篇:你已经具备从 0 搭一个商业级 App 的能力。**剩下的就是堆业务、调性能、追体验。
