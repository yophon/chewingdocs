# Flutter 本地存储

按"存什么、存多少、查得复不复杂"三个维度选方案。常见方案对比:

| 方案 | 数据形式 | 适合 | 性能 | 复杂度 |
| --- | --- | --- | --- | --- |
| `shared_preferences` | KV(基本类型) | 配置、Token | 一般 | 极低 |
| `flutter_secure_storage` | KV(加密) | Token、密码 | 一般 | 低 |
| `hive` | KV / 对象 | 中量结构化 | 极快 | 低 |
| `isar` | 对象 + 索引查询 | 大量数据 + 查询 | 极快 | 中 |
| `sqflite` | 关系型(SQL) | 复杂关系数据 | 中 | 高 |
| `drift`(原 moor) | 关系型 + 类型安全 | SQL + 工程化 | 中 | 中高 |
| `path_provider` | 文件系统 | 图片、缓存、大文件 | 看磁盘 | 中 |

---

## 一、shared_preferences:简单 KV

```yaml
dependencies:
  shared_preferences: ^2.3.2
```

```dart
final prefs = await SharedPreferences.getInstance();

await prefs.setString('token', 'abc');
await prefs.setBool('darkMode', true);
await prefs.setInt('count', 42);
await prefs.setStringList('tags', ['a', 'b']);

prefs.getString('token');         // 同步读
prefs.remove('token');
await prefs.clear();
```

**只支持基本类型**:`String / int / double / bool / List<String>`。
存对象?自己 jsonEncode 成字符串。

**底层**:
- iOS:NSUserDefaults
- Android:SharedPreferences
- Web:localStorage

⚠️ **不要存敏感数据**(token、密码),会被反编译看到。

---

## 二、flutter_secure_storage:加密 KV

存 token、密码、密钥等敏感数据。

```yaml
dependencies:
  flutter_secure_storage: ^9.2.2
```

```dart
const storage = FlutterSecureStorage();

await storage.write(key: 'token', value: 'abc');
final token = await storage.read(key: 'token');
await storage.delete(key: 'token');
```

**底层**:
- iOS:Keychain
- Android:EncryptedSharedPreferences(API 23+)/ KeyStore
- Web / Desktop:平台对应方案

**注意**:速度比 SharedPreferences 慢一些,只用来存敏感的少量数据。

---

## 三、hive:快、零依赖、支持对象

Hive 是纯 Dart 写的 NoSQL 数据库,**不依赖原生**(Web、Desktop 全平台一致),启动极快。

```yaml
dependencies:
  hive: ^2.2.3
  hive_flutter: ^1.1.0
dev_dependencies:
  hive_generator: ^2.0.1
  build_runner: ^2.4.9
```

### 存基础类型

```dart
await Hive.initFlutter();
final box = await Hive.openBox('settings');

await box.put('name', '张三');
await box.put('count', 42);

box.get('name');                  // 同步读
box.delete('name');
```

`box.listenable()` 还能配合 ValueListenableBuilder 自动刷新 UI:

```dart
ValueListenableBuilder(
  valueListenable: box.listenable(),
  builder: (_, Box box, __) => Text('${box.get('count')}'),
)
```

### 存自定义对象

```dart
@HiveType(typeId: 0)
class User extends HiveObject {
  @HiveField(0) String name;
  @HiveField(1) int age;
  User(this.name, this.age);
}

// 注册 Adapter
Hive.registerAdapter(UserAdapter());

final users = await Hive.openBox<User>('users');
await users.add(User('张三', 20));     // 自动分配 key
await users.put('me', User('我', 30));
```

跑 build_runner 生成 `UserAdapter`。

### LazyBox(大量数据时用)

```dart
final box = await Hive.openLazyBox<User>('users');
final user = await box.get('id-1');   // 异步读,不会一次性载入全部
```

## 四、isar:Hive 作者的下一代,带索引查询

如果你需要 **大量数据 + 复杂查询**,isar 是 Flutter 生态最快的选择(声称比 sqlite 快 10 倍,实测差不多 3-5 倍)。

```yaml
dependencies:
  isar: ^3.1.0
  isar_flutter_libs: ^3.1.0
  path_provider: ^2.1.4
dev_dependencies:
  isar_generator: ^3.1.0
  build_runner: ^2.4.9
```

```dart
@collection
class User {
  Id id = Isar.autoIncrement;

  @Index(unique: true)
  late String email;

  @Index(type: IndexType.value)
  late String name;

  late int age;
}

// 初始化
final dir = await getApplicationDocumentsDirectory();
final isar = await Isar.open([UserSchema], directory: dir.path);

// 增
await isar.writeTxn(() async {
  await isar.users.put(User()..email = 'a@b.com'..name = '张三'..age = 30);
});

// 查询(链式 + 强类型)
final found = await isar.users
  .filter()
  .ageGreaterThan(18)
  .nameContains('张')
  .sortByAge()
  .limit(10)
  .findAll();

// 监听变化
isar.users.watchLazy().listen((_) => print('数据变了'));
```

**优点**:
- 强类型查询,IDE 补全
- 自动索引,大数据集查询飞快
- 支持事务

**缺点**:
- API 复杂度比 Hive 高
- 包体积大(各平台原生 lib)

> 注意:Isar v4 在重写中,**新项目可考虑 Drift,稳定且成熟**。

## 五、sqflite:经典 SQLite

最贴近"数据库"的方案,适合**懂 SQL** 的人。

```yaml
dependencies:
  sqflite: ^2.3.3+1
  path: ^1.9.0
```

```dart
final db = await openDatabase(
  join(await getDatabasesPath(), 'app.db'),
  version: 1,
  onCreate: (db, v) async {
    await db.execute('''
      CREATE TABLE users(id INTEGER PRIMARY KEY, name TEXT, age INTEGER)
    ''');
  },
);

// 增
await db.insert('users', {'name': '张三', 'age': 30});

// 查
final list = await db.query('users', where: 'age > ?', whereArgs: [18]);

// 改
await db.update('users', {'age': 31}, where: 'id = ?', whereArgs: [1]);

// 删
await db.delete('users', where: 'id = ?', whereArgs: [1]);

// 原生 SQL
final rows = await db.rawQuery('SELECT * FROM users WHERE age > ?', [18]);
```

**问题**:
- SQL 字符串容易写错,运行时才发现
- 字段加减要写迁移

所以诞生了 Drift。

## 六、drift:类型安全的 SQL ORM

把 SQL 字符串变成**强类型代码**,类似后端的 jOOQ / Diesel。

```yaml
dependencies:
  drift: ^2.20.0
  sqlite3_flutter_libs: ^0.5.24
  path_provider: ^2.1.4
  path: ^1.9.0
dev_dependencies:
  drift_dev: ^2.20.0
  build_runner: ^2.4.9
```

### 定义表

```dart
import 'package:drift/drift.dart';

class Users extends Table {
  IntColumn get id => integer().autoIncrement()();
  TextColumn get name => text().withLength(min: 1, max: 50)();
  IntColumn get age => integer()();
  DateTimeColumn get createdAt => dateTime().withDefault(currentDateAndTime)();
}
```

### 定义数据库

```dart
@DriftDatabase(tables: [Users])
class AppDatabase extends _$AppDatabase {
  AppDatabase() : super(_openConnection());

  @override
  int get schemaVersion => 1;

  // 增
  Future<int> addUser(UsersCompanion u) => into(users).insert(u);

  // 查
  Future<List<User>> getAdults() =>
      (select(users)..where((u) => u.age.isBiggerThanValue(18))).get();

  // 监听(返回 Stream)
  Stream<List<User>> watchAdults() =>
      (select(users)..where((u) => u.age.isBiggerThanValue(18))).watch();
}
```

跑 build_runner 后,直接用强类型 API:

```dart
final db = AppDatabase();

await db.addUser(UsersCompanion.insert(name: '张三', age: 30));

final users = await db.getAdults();
db.watchAdults().listen((users) => print(users));    // 自动响应数据变化
```

**对比 sqflite**:全部表名、字段名、类型都是编译期检查;query 错了直接红线。

## 七、path_provider:文件系统访问

存大文件、图片、缓存:

```yaml
dependencies:
  path_provider: ^2.1.4
```

```dart
final docDir = await getApplicationDocumentsDirectory();
// iOS: ~/Documents
// Android: /data/data/<pkg>/files

final cacheDir = await getTemporaryDirectory();
// 系统可清理的缓存目录

final supportDir = await getApplicationSupportDirectory();
// 不展示给用户的应用数据

final extDir = await getExternalStorageDirectory();   // 仅 Android
```

存文件:

```dart
final file = File('${docDir.path}/data.json');
await file.writeAsString(jsonEncode({'a': 1}));
final json = await file.readAsString();
```

存图片:

```dart
final bytes = await dio.get<List<int>>('https://...');
await File('${docDir.path}/avatar.jpg').writeAsBytes(bytes.data!);
```

> **iOS 注意**:`Documents` 会被 iCloud 备份。临时文件应该放 `getTemporaryDirectory()`,缓存放 `getApplicationCacheDirectory()`,避免占用用户备份空间。

## 八、缓存图片:cached_network_image

99% 的项目不需要自己写图片缓存:

```yaml
dependencies:
  cached_network_image: ^3.4.1
```

```dart
CachedNetworkImage(
  imageUrl: 'https://...',
  placeholder: (_, __) => CircularProgressIndicator(),

```

## 九、典型组合

### 小型 App

```
配置 → shared_preferences
Token → flutter_secure_storage
列表数据 → hive
图片 → cached_network_image
```

### 中型 App

```
配置 → shared_preferences
敏感 → flutter_secure_storage
业务数据 → drift / isar
图片缓存 → cached_network_image
临时文件 → path_provider + getTemporaryDirectory
```

### 离线优先 App

```
所有数据 → drift(本地)
后台同步 → Dio + 拦截器 → 写入 drift
UI 永远从 drift 的 stream 读 → 自动响应
```

## 十、Repository 模式封装

不要让 UI 直接调 Hive / Drift。包一层:

```dart
abstract class TodoStore {
  Future<List<Todo>> getAll();
  Future<void> add(Todo t);
  Stream<List<Todo>> watch();
}

class HiveTodoStore implements TodoStore {
  final Box<Todo> box;
  HiveTodoStore(this.box);

  @override
  Future<List<Todo>> getAll() async => box.values.toList();

  @override
  Future<void> add(Todo t) => box.add(t);

  @override
  Stream<List<Todo>> watch() => box.watch().map((_) => box.values.toList());
}
```

后面想从 Hive 换 Drift,只改 Store 的实现,UI 一行不动。这就是抽象的价值。

## 十一、加密 / 安全

### Hive 加密

```dart
final key = await Hive.generateSecureKey();
final box = await Hive.openBox(
  'secret',
  encryptionCipher: HiveAesCipher(key),
);
```

加密 key 本身记得存 `flutter_secure_storage`。

### Drift / sqflite 加密

用 `sqlcipher_flutter_libs` 替换默认 lib,API 完全一样,只是数据库文件加密。

## 十二、迁移(数据结构升级)

### Hive

加字段不用迁移(老字段是 `null`)。删字段需要重新 open box 时跑迁移:

```dart
Hive.registerAdapter(UserAdapter());
final box = await Hive.openBox<User>('users');
// 想删字段就改 Adapter,旧数据自动忽略未知字段
```

### Drift

```dart
@override
MigrationStrategy get migration => MigrationStrategy(
  onCreate: (m) => m.createAll(),
  onUpgrade: (m, from, to) async {
    if (from < 2) {
      await m.addColumn(users, users.email);
    }
  },
);
```

每次改表结构 → schemaVersion + 1 → 在 onUpgrade 写迁移逻辑。

## 十三、性能 tips

1. **大量写入用事务**(Isar/Drift)— 单次写比循环写快几十倍
2. **List 数据用 LazyBox / Stream**— 避免一次性载入
3. **不要在 main 同步初始化大数据库**— 阻塞启动,放 splash 后做
4. **用 stream/watch 代替轮询**— 数据变化驱动 UI,不要 timer 反复查
5. **TextEditingController 等带状态的对象不要存数据库**— 它们不是数据

## 十五、心智模型

```
存什么?
  少量配置                → shared_preferences
  少量敏感                → flutter_secure_storage
  中量 KV / 对象          → hive
  大量 + 索引查询         → isar / drift
  关系型 + 团队懂 SQL     → drift
  文件 / 图片             → path_provider + cached_network_image
```

**选型核心两问**:**数据有多复杂?查询有多复杂?**两个都简单 → KV;两个都复杂 → Drift。
