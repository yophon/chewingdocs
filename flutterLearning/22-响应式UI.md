# Flutter 响应式 UI:平板 / 折叠屏 / 桌面适配

Flutter 一份代码六端跑(iOS / Android / Web / Windows / macOS / Linux),但**默认布局只针对手机**。要在大屏 / 折叠屏上看着舒服,需要主动适配。

---

## 一、核心思路:Adaptive vs Responsive

| 概念 | 含义 |
| --- | --- |
| **Responsive(响应式)** | **同一布局**根据宽度伸缩(Web 网页常见) |
| **Adaptive(自适应)** | **不同布局**针对不同设备(手机底导航 vs 桌面侧边栏) |

实战通常**两者结合**:小变化用 Responsive,结构性变化用 Adaptive。

## 二、查询屏幕信息:MediaQuery

```dart
final media = MediaQuery.of(context);

media.size.width                  // 屏幕宽
media.size.height                 // 屏幕高
media.devicePixelRatio            // 像素密度
media.padding                     // 安全区(刘海、底部 home 条)
media.viewInsets                  // 键盘等遮挡
media.orientation                 // 横屏 / 竖屏
media.platformBrightness          // 系统亮 / 暗
```

### 常用便捷:

```dart
final width = MediaQuery.sizeOf(context).width;       // 仅订阅 size
final padding = MediaQuery.paddingOf(context);
```

`sizeOf` 比 `of` 更精细,只在 size 变化时重建(回顾 05 的细粒度订阅)。

## 三、LayoutBuilder:拿父级约束

`MediaQuery` 是屏幕级,`LayoutBuilder` 是**当前 Widget 父级给的最大尺寸**:

```dart
LayoutBuilder(builder: (_, constraints) {
  if (constraints.maxWidth > 600) return TabletLayout();
  return PhoneLayout();
})
```

适合"组件内部根据可用空间调整"。

## 四、Material 3 的 Window Size Class

Material 3 给了一套断点(breakpoint)规范:

| 类型 | 宽度 | 典型设备 |
| --- | --- | --- |
| Compact | < 600 | 手机竖屏 |
| Medium | 600~840 | 折叠屏展开 / 平板竖屏 |
| Expanded | 840~1200 | 平板横屏 / 小桌面 |
| Large | 1200~1600 | 桌面 |
| ExtraLarge | > 1600 | 大显示器 |

```dart
WindowSizeClass _windowSize(double w) {
  if (w < 600) return WindowSizeClass.compact;
  if (w < 840) return WindowSizeClass.medium;
  if (w < 1200) return WindowSizeClass.expanded;
  if (w < 1600) return WindowSizeClass.large;
  return WindowSizeClass.extraLarge;
}
```

或用包 `flutter_adaptive_scaffold`(官方实验包),帮你按规范布局。

## 五、典型自适应模式

### 1. 底导航 → 侧边栏 → 抽屉

```dart
class HomeScaffold extends StatelessWidget {
  final Widget body;
  const HomeScaffold({required this.body, super.key});

  @override
  Widget build(BuildContext context) {
    final width = MediaQuery.sizeOf(context).width;

    if (width < 600) {
      // 手机:底部导航
      return Scaffold(
        body: body,
        bottomNavigationBar: NavigationBar(...),
      );
    } else if (width < 1200) {
      // 平板:NavigationRail
      return Scaffold(
        body: Row(
          children: [
            NavigationRail(...),
            const VerticalDivider(width: 1),
            Expanded(child: body),
          ],
        ),
      );
    } else {
      // 桌面:常驻 Drawer
      return Scaffold(
        body: Row(
          children: [
            const SizedBox(width: 256, child: ExtendedDrawer()),
            Expanded(child: body),
          ],
        ),
      );
    }
  }
}
```

### 2. Master-Detail(列表 + 详情)

手机:点击列表跳详情页
平板 / 桌面:左右分栏

```dart
class MasterDetail extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    final wide = MediaQuery.sizeOf(context).width >= 720;

    if (wide) {
      return Row(
        children: [
          SizedBox(width: 320, child: TodoList()),
          const VerticalDivider(width: 1),
          Expanded(child: TodoDetail()),
        ],
      );
    }

    return TodoList(onTap: (id) => Navigator.push(
      context,
      MaterialPageRoute(builder: (_) => TodoDetail(id: id)),
    ));
  }
}
```

### 3. 内容最大宽度限制

桌面屏幕宽,文字横通屏看着累。包一个 `ConstrainedBox`:

```dart
Center(
  child: ConstrainedBox(
    constraints: const BoxConstraints(maxWidth: 720),
    child: Article(),
  ),
)
```

## 八、折叠屏适配

折叠屏(三星 Fold、Pixel Fold)有"折痕"和"展开/折叠"两种状态。

### DisplayFeature

```dart
final features = MediaQuery.of(context).displayFeatures;

for (final f in features) {
  if (f.type == DisplayFeatureType.hinge) {
    // 折痕在 f.bounds 区域,避开它
  }
}
```

### TwoPane

`dual_screen` 包(社区)简化双屏布局:

```yaml
dependencies:
  dual_screen: ^1.0.4
```

```dart
TwoPane(
  pane1: TodoList(),
  pane2: TodoDetail(),
  panePriority: TwoPanePriority.both,    // 折叠时只显 pane1
)
```

折叠时显示一个 pane,展开时双 pane,自动适配。

## 九、字体 / 间距适配

### 1. 设计稿映射:flutter_screenutil

设计稿 375x812,代码里直接写设计稿数值:

```yaml
dependencies:
  flutter_screenutil: ^5.9.3
```

```dart
ScreenUtilInit(
  designSize: const Size(375, 812),
  builder: (_, __) => MaterialApp(...),
)

// 用
Container(width: 100.w, height: 50.h);
Text('hi', style: TextStyle(fontSize: 14.sp));
```

但 `screenutil` 在大屏(平板 / 桌面)会**字体超大**,需要按 windowSize 设上限:

```dart
designSize: width < 600 ? const Size(375, 812) : const Size(600, 800),
```

或干脆只在手机用 screenutil,平板手写。

### 2. textScaleFactor

系统级"字体大小"用户可设。Flutter 自动应用,但要保证布局不爆:

```dart
MediaQuery(
  data: MediaQuery.of(context).copyWith(
    textScaler: TextScaler.linear(1.0),     // 无视系统设置
  ),
  child: SomeFixedLayout(),
)
```

通常**不要写死**,尊重用户设置;但表格、状态栏等空间紧张的可以限制。

## 十七、常见坑

### 1. 写死宽高

```dart
SizedBox(width: 375, ...)    // ❌ 平板上太挤
```

→ 用 `Expanded`、`Flexible`、约束(`ConstrainedBox`)。

### 2. 字体在大屏巨大

`screenutil.sp` 按比例放大,平板上 14sp 可能变成 30+。要么限制 designSize,要么平板用 fontSize 直接写。

### 3. 横屏键盘弹起内容被压

`Scaffold.resizeToAvoidBottomInset = false` 关掉自动调整,手动用 `SingleChildScrollView` 包裹表单。

### 4. 桌面没有手势,只有鼠标

```dart
GestureDetector(onLongPress: ...)   // ⚠️ 桌面没人长按
```

→ 桌面用 `onSecondaryTapDown`(右键)。

### 5. iPad 默认上下分屏不支持

需要在 `Info.plist` 加配置:

```xml
<key>UISupportsDocumentBrowser</key>
<true/>
<key>UIRequiresFullScreen</key>
<false/>
```

### 6. Scaffold 外用 BottomNavigationBar

→ 报错。**BottomNavigationBar 必须在 Scaffold 内**。

- Theme(17)的 `colorScheme` 跨设备一致

## 二十、心智模型

```
< 600    手机  → 底导航  + 单栏
600~840  小平板 → NavigationRail + 单栏 / 双栏
840+     大平板 → 双栏 Master-Detail
1200+    桌面  → 三栏 + 鼠标交互 + 快捷键
```

**核心**:**让 UI 跟内容,而不是设备**。检查"这块在 800px 看着挤吗?"比"这是不是平板?"更稳。

写出真正能在多设备跑的 Flutter UI 不复杂,但要从一开始就想着——后期重构成本高得多。
