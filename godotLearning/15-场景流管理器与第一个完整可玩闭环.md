# 场景流管理器与第一个完整可玩闭环

做到这里,你已经有角色、地图、敌人、HUD、存档。但这还不是游戏。游戏需要一个完整循环:开始、进入关卡、死亡、重开、胜利、结算、回主菜单。

> 一句话先记住:**场景切换权集中到 SceneManager,不要散在 Player、Goal、按钮脚本里。**

---

## 一、为什么需要 SceneManager

反例:

```gdscript
# Player.gd
func die() -> void:
    get_tree().change_scene_to_file("res://ui/main_menu.tscn")
```

```gdscript
# Goal.gd
func complete() -> void:
    get_tree().reload_current_scene()
```

```gdscript
# PauseMenu.gd
func quit() -> void:
    get_tree().change_scene_to_file("res://ui/main_menu.tscn")
```

切场景逻辑散在各处,后面想加淡入淡出、保存进度、统计击杀数,你要改很多文件。

更好的做法:

```text
Player 只发 died
Goal 只发 completed
PauseMenu 只发 quit_requested
SceneManager 决定切到哪里
```

---

## 二、状态先写清楚

```gdscript
enum State {
    BOOT,
    MAIN_MENU,
    LOADING,
    PLAYING,
    PAUSED,
    DEATH,
    VICTORY,
}
```

游戏当前处在哪个状态,不是靠“哪个场景在树上”猜,而是显式记录。

状态会影响:

- 是否允许玩家输入。
- HUD 是否显示。
- 暂停键是否生效。
- 是否写存档。
- 背景音乐播哪首。

## 三、SceneManager 最小实现

做成 Autoload:

```gdscript
# res://systems/scene_manager.gd
extends Node

enum State { BOOT, MAIN_MENU, LOADING, PLAYING, PAUSED, DEATH, VICTORY }

signal state_changed(from: State, to: State)
signal level_loaded(level_id: StringName)

const MAIN_MENU := "res://ui/main_menu.tscn"
const DEATH_SCREEN := "res://ui/death_screen.tscn"
const VICTORY_SCREEN := "res://ui/victory_screen.tscn"

var state: State = State.BOOT
var current_level_id: StringName = &""
var current_level_path: String = ""

func go_to_main_menu() -> void:
    await _change_scene(MAIN_MENU, State.MAIN_MENU)

func start_level(level_id: StringName, path: String) -> void:
    current_level_id = level_id
    current_level_path = path
    await _change_scene(path, State.PLAYING)
    level_loaded.emit(level_id)

func restart_level() -> void:
    if current_level_path.is_empty():
        await go_to_main_menu()
        return
    await _change_scene(current_level_path, State.PLAYING)
    level_loaded.emit(current_level_id)

func player_died() -> void:
    await _change_scene(DEATH_SCREEN, State.DEATH)

func level_completed(stats: Dictionary) -> void:
    SaveSystem.save_game(1, stats)
    await _change_scene(VICTORY_SCREEN, State.VICTORY)

func _set_state(next: State) -> void:
    if state == next:
        return
    var prev := state
    state = next
    state_changed.emit(prev, next)

func _change_scene(path: String, next_state: State) -> void:
    _set_state(State.LOADING)
    get_tree().paused = false

    var err := get_tree().change_scene_to_file(path)
    if err != OK:
        push_error("Cannot change scene to %s: %s" % [path, err])
        _set_state(State.MAIN_MENU)
        return

    await get_tree().scene_changed
    _set_state(next_state)
```

这里假设第 14 篇的 `SaveSystem` 已经是 Autoload。

## 四、关卡根节点负责接线

关卡根脚本:

```gdscript
# res://levels/level_01.gd
extends Node2D

@onready var _player: PlayerBody = %Player
@onready var _goal: Area2D = %Goal
@onready var _pause_menu: PauseMenu = %PauseMenu

func _ready() -> void:
    _player.died.connect(_on_player_died)
    _goal.body_entered.connect(_on_goal_body_entered)
    _pause_menu.restart_requested.connect(_on_restart_requested)
    _pause_menu.quit_requested.connect(_on_quit_requested)

func _on_player_died() -> void:
    SceneManager.player_died()

func _on_goal_body_entered(body: Node) -> void:
    if body == _player:
        SceneManager.level_completed(_collect_stats())

func _on_restart_requested() -> void:
    SceneManager.restart_level()

func _on_quit_requested() -> void:
    SceneManager.go_to_main_menu()

func _collect_stats() -> Dictionary:
    return {
        "level_id": "level_01",
        "checkpoint_id": "start",
        "gold": 0,
        "inventory": [],
    }
```

`Player` 不知道 `SceneManager`。`Goal` 也不知道。关卡根节点是胶水。

## 五、主菜单只发起请求

```gdscript
# res://ui/main_menu.gd
extends Control

func _on_start_button_pressed() -> void:
    SceneManager.start_level(&"level_01", "res://levels/level_01.tscn")

func _on_continue_button_pressed() -> void:
    var save := SaveSystem.load_game(1)
    if save.is_empty():
        return
    SceneManager.start_level(StringName(save["level_id"]), "res://levels/level_01.tscn")

func _on_quit_button_pressed() -> void:
    get_tree().quit()
```

按钮不直接碰关卡内部节点。

## 六、转场遮罩

裸切场景很突兀。`SceneManager` 可以自己带一个 `CanvasLayer`:

```text
SceneManager (autoload scene)
└── TransitionLayer (CanvasLayer)
    └── FadeRect (ColorRect, Full Rect, black)
```

封装:

```gdscript
@onready var _fade: ColorRect = %FadeRect

func _fade_to(alpha: float) -> void:
    var tween := create_tween()
    tween.set_pause_mode(Tween.TWEEN_PAUSE_PROCESS)
    tween.tween_property(_fade, "modulate:a", alpha, 0.2)
    await tween.finished
```

切场景流程:

```gdscript
await _fade_to(1.0)
change_scene
await scene_changed
await _fade_to(0.0)
```

先把逻辑跑通,再加这层。不要一开始就在转场动画里调试场景流。

## 七、死亡和胜利不要立刻清状态

死亡时常见流程:

```text
Player HP = 0
Player emit died
Level 通知 SceneManager
SceneManager 切到 DeathScreen
DeathScreen 的 Retry 按钮调用 restart_level()
```

胜利时:

```text
Goal 检测玩家进入
Level 收集统计
SceneManager 保存
SceneManager 切到 VictoryScreen
VictoryScreen 选择下一关或回主菜单
```

不要在 `Player.die()` 里直接 `queue_free()` 后切场景。先让流程管理器接管,避免信号还没发完对象就没了。

## 八、第一个完整闭环验收

从冷启动开始,应该能完整跑:

1. 打开主菜单。
2. 点开始进入 Level01。
3. 玩家能移动、战斗、看 HUD。
4. 按暂停能继续、重开、回主菜单。
5. 玩家死亡进入死亡画面。
6. 死亡画面点重试能干净重开。
7. 进终点进入胜利结算。
8. 胜利时写入存档。
9. 回主菜单后继续按钮能读到进度。

这才叫第一个原型闭环。

---

## 常见坑

**坑 1:Player 自己切场景。**

Player 只发 died。场景怎么处理死亡由外层决定。

**坑 2:重开只把玩家挪回出生点。**

敌人、拾取物、机关都没复位。原型期直接 reload 当前关卡最干净。

**坑 3:切场景后 autoload 状态没清。**

场景会重建,autoload 不会。SceneManager 要自己清当前状态。

**坑 4:暂停状态下转场卡住。**

切场景前先 `get_tree().paused = false`,转场 tween 设置 pause process。
