# 多平台导出、D3D12 与增量补丁

能在编辑器里跑,不等于能发给玩家。导出要处理平台、渲染后端、纹理压缩、签名、补丁和资源包。

> 一句话先记住:**导出配置是工程资产,`export_presets.cfg` 要进版本控制。**

---

## 一、先建 Export Preset

Godot 导出配置存在:

```text
export_presets.cfg
```

它记录:

```text
导出平台
包含/排除哪些资源
图标和版本信息
渲染后端
签名配置
纹理压缩
```

不要只在自己机器的 Export 窗口里点完就算。CI 和别人接手项目都需要这份文件。

---

## 二、先做三个目标

独立 2D 项目常见目标:

```text
Windows
macOS / Linux
Web 或 Android
```

不要一开始承诺全平台。每个平台都要测试输入、存档、字体、性能、窗口、音频。

---

## 三、Windows 的 D3D12

Godot 4.6 的 Windows 新项目默认倾向 D3D12。老项目升级时不要假设自动切了。

导出前至少测:

```text
D3D12
Vulkan
Compatibility
```

如果玩家机器驱动太旧,需要有 fallback 思路。2D 项目通常不依赖特别新的 GPU 功能,但 shader、光照、GDExtension 动态库都要实际跑一遍。

---

## 四、PCK 是资源包

Godot 导出通常会得到:

```text
game.exe
game.pck
```

`.pck` 放资源。可执行文件启动后加载它。

运行时也可以加载额外资源包:

```gdscript
ProjectSettings.load_resource_pack("user://mods/example.pck")
```

这条路可以做 DLC、mod、补丁。

---

## 五、增量补丁

补丁不要总是让玩家重新下载完整包。Godot 4.6 支持 delta patch PCK,补丁只包含差异。

实际策略:

```text
1.0.0 完整包
1.0.1 patch pck
1.0.2 patch pck
```

启动时按顺序加载补丁:

```gdscript
func load_patches() -> void:
    var patches := [
        "user://patches/patch_1_0_1.pck",
        "user://patches/patch_1_0_2.pck",
    ]

    for path in patches:
        if FileAccess.file_exists(path):
            ProjectSettings.load_resource_pack(path)
```

补丁机制很强,但也会增加发布复杂度。小项目早期直接发完整包也可以,等包体变大再上补丁。

---

## 六、Web 导出要单独测

Web 不是“小号 PC”。

重点检查:

```text
浏览器是否允许音频自动播放
存档是否真的写进 IndexedDB
字体是否全部打包
纹理格式是否支持
服务器 HTTP header 是否正确
加载首屏是否过大
```

Web 上 `user://` 是浏览器存储,不要按桌面文件系统思路设计。

---

## 七、移动端重点

Android / iOS 重点不是能不能打包,而是:

```text
触屏输入
安全区
返回键
暂停/恢复生命周期
纹理压缩
包名和签名
性能和发热
```

移动端先做最小导出测试,别等项目最后才发现 UI 全贴边、字体太小、触控按钮挡住游戏内容。

---

## 八、导出前检查

- `export_presets.cfg` 已提交。
- export template 版本和编辑器版本一致。
- Windows 至少测 D3D12 和 Vulkan。
- 存档路径在导出版可写。
- 字体在所有目标平台都有。
- GDExtension 动态库按平台打包。
- Web 版本真实部署到 HTTP 服务测试,不是只打开本地文件。
- 补丁策略清楚:完整包还是 patch PCK。

---

## 常见坑

**坑 1:只在编辑器里测试。**

导出版资源路径、日志、存档、性能都可能不同。

**坑 2:忘记 export template 版本。**

编辑器和模板版本不一致会出奇怪问题。

**坑 3:Web 本地打开 HTML。**

很多浏览器能力需要 HTTP 环境。用本地服务器测。

**坑 4:补丁覆盖顺序混乱。**

patch PCK 要有明确版本顺序。
