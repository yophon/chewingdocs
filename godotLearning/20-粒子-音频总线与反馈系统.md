# 粒子、音频总线与反馈系统

角色打中了敌人,如果没有粒子、音效、停顿、震动,玩家会怀疑自己到底有没有打中。反馈不是装饰,是操作结果的回执。

> 一句话先记住:**短时反馈不要挂在会马上销毁的对象下面。**

---

## 一、先列反馈通道

一次命中可以同时触发:

```text
hit stop
屏幕震动
命中粒子
受击闪白
命中音效
血量变化
击退
```

前面已经讲过 hit stop 和镜头震动。这一篇重点讲粒子和音频,并给一个统一入口。

---

## 二、GPUParticles2D 还是 CPUParticles2D

默认用:

```text
GPUParticles2D
```

适合:

- 命中火花
- 落地灰尘
- 爆炸烟雾
- 魔法碎片
- 冲刺拖尾

`CPUParticles2D` 只有在特殊平台或需要 CPU 侧精细控制时再考虑。

关键设置:

```text
one_shot = true
emitting = false
local_coords = false
amount = 8-80
lifetime = 0.2-1.0
```

`local_coords = false` 很重要。角色冲刺后,烟尘应该留在原地散开,不是跟着角色一起飞走。

---

## 三、粒子不要现场创建太多

反例:

```gdscript
var p := hit_particles.instantiate()
add_child(p)
p.global_position = pos
p.emitting = true
```

少量可以,大量命中时会频繁创建/销毁节点。

更好的做法是做一个反馈管理器,复用场景:

```gdscript
# res://systems/feedback_manager.gd
extends Node

@export var hit_particle_scene: PackedScene
@export var dust_particle_scene: PackedScene
@export var hit_sounds: Array[AudioStream] = []

func play_hit(position: Vector2, strength: float = 1.0) -> void:
    _spawn_particle(hit_particle_scene, position)
    _play_sfx(hit_sounds, position, strength)

func play_dust(position: Vector2) -> void:
    _spawn_particle(dust_particle_scene, position)

func _spawn_particle(scene: PackedScene, position: Vector2) -> void:
    if scene == null:
        return
    var node := scene.instantiate()
    get_tree().current_scene.add_child(node)
    node.global_position = position
    if node is GPUParticles2D:
        node.restart()
        node.finished.connect(node.queue_free)

func _play_sfx(streams: Array[AudioStream], position: Vector2, volume: float) -> void:
    if streams.is_empty():
        return

    var player := AudioStreamPlayer2D.new()
    player.stream = streams.pick_random()
    player.bus = &"SFX"
    player.volume_db = linear_to_db(clampf(volume, 0.0, 1.0))
    player.pitch_scale = randf_range(0.95, 1.05)
    get_tree().current_scene.add_child(player)
    player.global_position = position
    player.finished.connect(player.queue_free)
    player.play()
```

原型期现场创建能接受。后面如果反馈密度很高,再把 `_spawn_particle` 和 `_play_sfx` 改成对象池。

## 四、粒子参数从哪里调

`ParticleProcessMaterial` 里优先看:

```text
direction
spread
initial_velocity_min/max
gravity
damping
scale_min/max
color_ramp
```

几个起步例子:

```text
命中火花: lifetime 0.25, amount 12, speed 80-180, gravity 0
落地灰尘: lifetime 0.45, amount 10, speed 20-60, gravity -20
爆炸烟雾: lifetime 0.8, amount 40, speed 40-120, damping 高
```

粒子不是越多越好。方向、寿命、颜色衰减对手感更重要。

## 五、音频总线先建好

至少四条:

```text
Master
Music
SFX
UI
```

去 Audio 面板里建 bus。代码里明确指定:

```gdscript
player.bus = &"SFX"
```

不要所有音频都走 Master。否则暂停时想压低音乐、关闭 SFX、保留 UI 点击,会非常麻烦。

## 六、2D 音效

世界里的声音用:

```text
AudioStreamPlayer2D
```

UI 点击用:

```text
AudioStreamPlayer
```

世界音效根据距离衰减:

```gdscript
player.max_distance = 600.0
player.attenuation = 2.0
```

如果你的游戏镜头视野很小,`max_distance` 不要设太大。远处敌人的脚步声不应该和身边一样响。

## 七、统一触发

命中时:

```gdscript
func _on_hit_confirmed(position: Vector2, damage: int) -> void:
    FeedbackManager.play_hit(position, clampf(float(damage) / 50.0, 0.2, 1.0))
    GameCamera.add_trauma(0.25)
```

拾取道具时:

```gdscript
func _on_item_picked(item_id: StringName, _count: int) -> void:
    FeedbackManager.play_ui_confirm()
```

建议让 `FeedbackManager` 监听 `EventBus` 的关键事件,这样玩家、敌人、UI 不用各自管理音效和粒子。

---

## 八、验收

- 项目有 `Music / SFX / UI` 音频总线。
- 命中、落地、拾取至少有一种粒子或音效反馈。
- 世界音效用 `AudioStreamPlayer2D`,UI 音效用普通播放器。
- 短时反馈不会因为敌人 `queue_free()` 被截断。
- 同类音效有轻微 pitch 随机,不会重复得僵硬。
- 粒子 `local_coords` 设置符合效果目标。

---

## 常见坑

**坑 1:音效全走 Master。**

后面做音量设置会很痛。先分 bus。

**坑 2:粒子挂在敌人下面。**

敌人死亡释放,粒子也没了。短时反馈挂当前场景或反馈管理器。

**坑 3:粒子数量太多。**

先调速度、方向、颜色曲线,不要只加 amount。

**坑 4:同一个音效连续播放完全一样。**

给 `pitch_scale` 做 0.95-1.05 的小随机。
