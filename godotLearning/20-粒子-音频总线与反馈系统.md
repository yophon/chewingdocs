# 20-粒子、音频总线与反馈系统

> 一句话导读:粒子和音频不是"加点漂亮",它们是玩家与系统之间的反馈通道;打中怪物的粒子飞溅,和命中音效里那一帧响,共同回答了"我的操作生效了吗"。

任何一款有手感的 2D 游戏,把粒子和音效全部静音之后都会立刻"露怯"——人物动作还在,但每一次落地、攻击、受伤、捡道具的"质感"都没了。这个章节要说清两件事:

- **粒子**:Godot 4.6 里 `GPUParticles2D` 和 `CPUParticles2D` 怎么选?`ParticleProcessMaterial` 那几十个属性哪些是手感关键?Trail 在 4.x 怎么用?
- **音频**:为什么一个像样的游戏要有 `Master / Music / SFX / UI` 至少 4 条总线?动态音乐(战斗时音乐加 LowPass、Boss 战切轨)怎么用 bus 实现?`AudioStreamPlayer2D` 的衰减曲线为什么默认行为常常不对?

这一篇与 19 篇是同层(表现力提升),与 21 篇互补——21 篇的 shader 也是反馈,但它属于"贴图改色",而本篇的反馈在节点和总线层。一个稳健的反馈系统骨架,后续做闪白(21 篇)、屏幕震动(10 篇)、hit stop(7 篇)只是往同一个调度器上挂新的通道,不会需要重写。

## 1. 机制定位

### 反馈系统的物理层和心理层

反馈是玩家对操作的"回声"。从设计角度,反馈由四层组成:

1. **视觉脉冲**:角色受击时的闪白(21 篇)、屏幕震动(10 篇)、命中粒子(本篇);
2. **动效**:hit stop(7 篇里讲过的命中停顿)、镜头推拉、击退;
3. **音效**:命中音、受击音、UI 点击,通常 100ms 内必须给出;
4. **环境响应**:命中后场上别的对象的反应(树叶颤动、灯光闪烁、远处响动)。

本篇主要负责 1 的"粒子"部分和 3、4 的"音效与总线"部分。一个常见的错误是把这些反馈分散写在各处:角色脚本里 `instantiate` 粒子、`AudioStreamPlayer` 写死 `Master` 总线、命中音直接挂在敌人节点上随场景一起销毁。这样写 demo 没问题,做到 200 个反馈点之后维护成本会指数上涨。本篇要给一个"集中、可复用、可静音"的反馈系统骨架。

更深一层的设计观:粒子和音效本质都是"短时事件",它们的生命周期与触发它们的逻辑实体(玩家攻击、敌人受击、UI 点击)**不应该耦合**。逻辑实体可能瞬间死亡(角色被秒杀),但那一帧粒子的最后几个像素和那一声死亡音效需要继续播完。这是为什么本篇所有反馈节点都放在场景根的 Autoload 池 / 全局组里,而不是某个敌人/玩家的子节点——子节点会随父节点 `queue_free` 被强制释放,反馈被截断。这一原则在 16 篇的事件总线里会被进一步形式化。

### 新手常见写法为什么失控

粒子的常见坑有两类:

- **一帧上千个 `GPUParticles2D` 实例**:每个粒子节点都有自己的 `ParticleProcessMaterial`,一次性 `add_child` + `queue_free` 模式会把 GPU 上传 / 释放当家常便饭。正确做法是**预先放好节点池,只 reuse 节点改 `position` + `restart()`**。
- **滥用 `local_coords = true`**:这是新手最常踩的坑。打开 local_coords,粒子会跟随发射器一起移动,看上去像"挂在角色身上的烟",但角色冲刺时所有粒子会被瞬间拖拽,没法做"角色冲刺、烟雾留在原地散开"的效果。

音频的常见坑同样集中:

- **全部走 Master**:`AudioStreamPlayer.bus = "Master"` 是默认值,新手懒得改。一旦想加"暂停时把 SFX 静音但 Music 继续",或者"音乐渐弱,SFX 不变",就全得改一遍。
- **`AudioStreamPlayer2D.attenuation` 默认 = 1.0,衰减太弱**:玩家会发现远处怪物的音效几乎和近处一样大。`attenuation` 的合理范围是 1.5–3.0,具体看相机视野。
- **重复音节奏感丢失**:同一个脚步声 30 帧内连续触发,如果没有 `pitch_scale` 微抖动,会听成"机关枪"。

后面的章节把这些一并展开。

## 2. Godot 心智

### 粒子:CPU vs GPU 的真实差别

Godot 4 提供两种粒子节点:

- **`GPUParticles2D`**:粒子在 GPU 上的 compute shader 里更新。能撑到数千粒子不掉帧,但**它的所有逻辑都在 `ParticleProcessMaterial`(资源)里**,无法逐粒子在 GDScript 中介入。
- **`CPUParticles2D`**:粒子在 CPU 上每帧更新,所有属性直接在节点 Inspector 上。性能不如 GPU,但**与 GDScript 共享 CPU 内存,可以 hook 单个粒子(虽然 API 也很有限)**;在 GPU-bound 的场景(比如 Web 平台没有 compute shader)反而更稳。

官方明确说"无新功能会加到 `CPUParticles2D`,优先用 `GPUParticles2D`"。本系列默认全部用 GPU 版本。需要兼容 Web 出口时,通过工具栏的 `Convert to CPUParticles2D` 一键转换——但 trail 是 GPU only,转过去会丢。

判断哪个更适合,看两个问题:① 粒子数量上千?用 GPU。② 需要根据每个粒子的运行时状态做 GDScript 决策(比如"撞到墙就反弹")?用 CPU,或者忘掉这个需求改用普通 Node2D 池(不是粒子)。粒子系统的设计前提就是"大量相似的、不可寻址的小对象",一旦你想给"第 17 号粒子"做点事,它已经不该是粒子了。

### `ParticleProcessMaterial` 的关键字段

`ParticleProcessMaterial` 是 `GPUParticles2D.process_material` 字段挂载的 `Resource`。它有几十个属性,真正影响手感的只有这些:

| 字段 | 含义 | 调手感的方向 |
| --- | --- | --- |
| `emission_shape` | 发射形状(Point / Sphere / Box / Ring) | 命中粒子用 Sphere,小爆炸用 Box |
| `direction` | 发射方向向量 | 重力下落用 `(0, 1, 0)`,跟随击退方向时脚本动态写 |
| `spread` | 方向扇形半角(度) | 0 = 直线;45 = 锥形;180 = 全方向 |
| `initial_velocity_min/max` | 初速度区间 | 决定散得多快,血溅常 80–200 |
| `gravity` | 重力向量 | 2D 里通常 `(0, 600, 0)` 模拟下落 |
| `scale_min/max` | 大小区间 | 配合 `scale_curve` 做"由大变小"很关键 |
| `color` + `color_ramp` | 颜色基色 + 生命周期颜色 | `color_ramp` 末尾 alpha=0 实现淡出 |
| `damping_min/max` | 阻尼(空气阻力) | 大阻尼让粒子很快静止 |

`emission_shape` 决定粒子从哪里诞生——Point 是从单点(适合喷射 / 火花)、Sphere 是球内随机(适合爆炸,2D 里看起来像圆盘)、Box 是矩形内随机(适合"地面尘埃"这种贴地分布)、Ring 是环形(适合冲击波)、Points 是预先指定的点列表(适合"沿某条曲线发射")。**先选对 shape 再调其它参数**,常见错误是用 Point 加大 spread 模拟爆炸——粒子初始全在同一点,远看是"线"而非"球"。

`color_ramp` 是 `GradientTexture1D`,横坐标是粒子生命周期 0→1,纵坐标是 RGBA。所有命中、灰烬、烟雾粒子的"由暖到冷、由浓到淡"都靠它。`scale_curve` 同理,横坐标生命周期,纵坐标 scale 倍率——典型曲线是"0→快速涨到 1.2→缓慢降到 0",这样粒子有"喷出来、扩散、消散"的呼吸感。

### Trail(4.x)

`GPUParticles2D.trail_enabled = true` + `trail_lifetime` 启用拖尾。Godot 4 的 trail 用粒子轨迹生成一条 mesh ribbon,通过 `trail_sections` 和 `trail_section_subdivisions` 控制平滑度。注意:

- Trail 只对 `GPUParticles2D` 有效,`CPUParticles2D` 没有这个能力(粒子的 mesh ribbon 需要 GPU)。
- 拖尾的纹理来自 `GPUParticles2D.texture`,要专门画"沿轴方向延伸"的图(中央亮、两端透明的胶囊形),不能直接用普通圆形粒子图。
- `trail_lifetime = 0.3` 是合理起点,>1.0 会让画面非常糊。

### 音频:Bus 是路由不是音量调节

每个 `AudioStreamPlayer` / `AudioStreamPlayer2D` 都把声音送到一个**Audio Bus**(总线)。bus 是一条独立的音频通道,可以挂效果器(Reverb / LowPass / Compressor),最终汇入 `Master`。`AudioServer` 是 Godot 暴露的 bus 管理 API:

- `AudioServer.get_bus_index(name)` 查 bus 序号(运行时大量使用)
- `AudioServer.set_bus_volume_db(idx, db)` 改音量(支持 tween 渐变)
- `AudioServer.set_bus_mute(idx, bool)` 静音(整条总线)
- `AudioServer.add_bus_effect(idx, AudioEffectXxx.new())` 运行时加效果器
- `AudioServer.set_bus_effect_enabled(idx, effect_idx, bool)` 切换效果器

Godot 提供三个 player 节点,差别要分清:`AudioStreamPlayer` 没有空间属性,适合 BGM / UI;`AudioStreamPlayer2D` 有 `global_position`,可以做 2D 距离衰减和左右声道偏移;`AudioStreamPlayer3D` 是 3D 版本,本系列不用。**UI 用 2D 版本不会出错,但浪费 CPU**——3 个总是更优的:UI 走 `AudioStreamPlayer`、世界音效走 `AudioStreamPlayer2D`、音乐走 `AudioStreamPlayer`。

bus 布局默认存在 `res://default_bus_layout.tres`,可以在 `Editor → Audio Buses` 面板可视化编辑后保存。**bus 名字一旦用脚本引用,就成了"软合同",修改 bus 名时记得同步**——这是为什么本系列推荐用 `const` 集中管理 bus 名:

```gdscript
class_name AudioBuses
const MASTER  : StringName = &"Master"
const MUSIC   : StringName = &"Music"
const SFX     : StringName = &"SFX"
const UI      : StringName = &"UI"
const AMBIENT : StringName = &"Ambient"
```

`StringName`(带 `&` 前缀)在 Godot 4 里是字符串的不可变化版,bus 查找会用它做 hash,比普通 `String` 略快。

## 3. 工程实现

构建三个模块:命中粒子工厂(GPU 池)、音频管理 Autoload(bus + 全局音量)、空间音效辅助(2D 距离衰减)。

### 命中粒子:节点池而非动态生成

`demo/scenes/fx/hit_sparks.tscn` 是一个 `GPUParticles2D` 场景,挂上 `ParticleProcessMaterial`,关闭 `emitting`,`one_shot = true`、`amount = 24`、`lifetime = 0.45`、`explosiveness = 0.9`,粒子配置好"爆炸式"的喷射。

挂在 root 上的池脚本 `demo/scripts/fx/hit_spark_pool.gd`:

```gdscript
extends Node
class_name HitSparkPool

@export var spark_scene: PackedScene
@export var pool_size: int = 16

var _pool: Array[GPUParticles2D] = []
var _next: int = 0

func _ready() -> void:
    assert(spark_scene != null, "HitSparkPool.spark_scene must be set")
    for i in pool_size:
        var node: GPUParticles2D = spark_scene.instantiate()
        node.emitting = false
        node.one_shot = true
        node.visible = false
        add_child(node)
        _pool.append(node)

# 在被攻击的位置喷一组粒子;direction 控制喷射朝向
func spawn(world_pos: Vector2, direction: Vector2 = Vector2.UP,
        scale_mult: float = 1.0) -> void:
    var node: GPUParticles2D = _pool[_next]
    _next = (_next + 1) % pool_size
    node.global_position = world_pos
    node.visible = true
    # 改方向不应直接 set process_material(那会上传整张 material),
    # 用 set_instance_shader_parameter 走 per-instance uniform 最便宜
    var mat: ParticleProcessMaterial = node.process_material
    mat.direction = Vector3(direction.x, direction.y, 0.0)
    mat.scale_min = 0.5 * scale_mult
    mat.scale_max = 1.5 * scale_mult
    node.restart()                          # restart 比 emitting=true 更可靠
```

关键工程取舍:

- **池大小 16**:屏幕上同时可见的"命中粒子"很少超过 8 个(玩家+几个敌人),16 是 2 倍冗余,够覆盖最坏情况。如果你做弹幕,池要扩到 64-128。
- **`one_shot = true`**:让 GPU 在播放完后自动停止,不需要外部计时。
- **`restart()` 而不是 `emitting = true`**:在节点 still 处于"上一发还没演完"状态时,`emitting = true` 不会重新触发;`restart()` 强制重新开始。
- **直接改 `mat.direction` 是有代价的**:每改一次,`ParticleProcessMaterial` 整体 dirty 一次。如果池里所有粒子共用一份 material,改一个会影响其他正在播放的——这就是为什么 pool 里每个粒子节点要独立 `process_material`(在 `.tscn` 里 duplicate)。

### 音频管理 Autoload

`demo/autoload/audio.gd`,在 `Project Settings → Autoload` 注册为单例 `Audio`。这是音量管理 + 全局 mute 的入口。

```gdscript
extends Node

const MASTER  : StringName = &"Master"
const MUSIC   : StringName = &"Music"
const SFX     : StringName = &"SFX"
const UI      : StringName = &"UI"
const AMBIENT : StringName = &"Ambient"

func _ready() -> void:
    # 用户设置在 14 篇会接到存档系统,这里先用本地缓存
    load_volume_settings()

func set_volume(bus: StringName, linear: float) -> void:
    var idx := AudioServer.get_bus_index(bus)
    if idx < 0:
        push_warning("Audio bus '%s' not found" % bus)
        return
    # linear → dB,linear=0 时直接静音(避免 log(0) = -inf)
    if linear <= 0.0001:
        AudioServer.set_bus_mute(idx, true)
    else:
        AudioServer.set_bus_mute(idx, false)
        AudioServer.set_bus_volume_db(idx, linear_to_db(linear))

func tween_volume(bus: StringName, target_linear: float, duration: float) -> void:
    var idx := AudioServer.get_bus_index(bus)
    if idx < 0:
        return
    var current_db := AudioServer.get_bus_volume_db(idx)
    var target_db := linear_to_db(maxf(target_linear, 0.0001))
    var tween := create_tween()
    tween.tween_method(
        func(db: float) -> void: AudioServer.set_bus_volume_db(idx, db),
        current_db, target_db, duration)

func enter_combat() -> void:
    # 战斗时压低环境音 60%、加 LowPass 让音乐"闷"一下
    tween_volume(AMBIENT, 0.4, 0.5)
    _set_music_lowpass(true)

func leave_combat() -> void:
    tween_volume(AMBIENT, 1.0, 1.0)
    _set_music_lowpass(false)

func _set_music_lowpass(on: bool) -> void:
    var idx := AudioServer.get_bus_index(MUSIC)
    if idx < 0:
        return
    # 假设 Music bus 上第 0 号 effect 是 AudioEffectLowPassFilter
    AudioServer.set_bus_effect_enabled(idx, 0, on)

func load_volume_settings() -> void:
    # 14 篇会把这里接到 user:// 存档;此处给默认
    set_volume(MASTER, 1.0)
    set_volume(MUSIC, 0.8)
    set_volume(SFX, 1.0)
    set_volume(UI, 0.8)
    set_volume(AMBIENT, 0.6)
```

bus 布局需要在编辑器 `Audio` 面板里建好,持久化到 `res://default_bus_layout.tres`。最小推荐布局:

```
Master                      # 总输出,只挂 Limiter
├── Music     send→Master   # BGM 通道,挂 LowPass(默认 disabled)
├── SFX       send→Master   # 战斗音效,挂 Compressor
├── UI        send→Master   # 菜单点击,不挂效果器,延迟最低
└── Ambient   send→Master   # 环境音(风声、虫鸣),挂 Reverb
```

`send→Master` 是 bus 的路由,在 Inspector 里设置 `send` 字段。**Master 挂 Limiter** 几乎是必须的——它能把所有总线汇总的峰值压在 0dB 以下,防止爆音。

### 空间音效:`AudioStreamPlayer2D` 的衰减

`demo/scripts/audio/sfx_emitter.gd`,作为 `AudioStreamPlayer2D` 子类挂在场景里。它在 `_ready` 时强制设置合理的衰减参数,避免 Inspector 上忘记改。

```gdscript
extends AudioStreamPlayer2D
class_name SFXEmitter

@export_range(1.0, 10.0) var inverse_attenuation: float = 2.0
@export_range(100.0, 4000.0) var hearing_radius: float = 1200.0
@export var random_pitch: float = 0.08            # ±8% 音高随机
@export var sfx_bus: StringName = &"SFX"

var _rng := RandomNumberGenerator.new()

func _ready() -> void:
    _rng.randomize()
    bus = sfx_bus
    attenuation = inverse_attenuation
    max_distance = hearing_radius
    # 让远处声音 stereo 偏移更明显
    panning_strength = 1.0
    max_polyphony = 4

func play_random() -> void:
    pitch_scale = 1.0 + _rng.randf_range(-random_pitch, random_pitch)
    play()

# 击中位置发声:由系统调用,自动选最近一个空闲 emitter 池
static func emit_at(parent: Node, stream: AudioStream,
        world_pos: Vector2, pitch_jitter: float = 0.08) -> void:
    var node := SFXEmitter.new()
    node.stream = stream
    node.random_pitch = pitch_jitter
    parent.add_child(node)
    node.global_position = world_pos
    node.finished.connect(node.queue_free)
    node.play_random()
```

几条工程经验:

- **`attenuation = 1.0` 几乎听不出 2D 定位**。把它推到 2.0 以上,远处的脚步声、远处怪物的咆哮才有距离感。
- **`hearing_radius` 与相机视野相关**。如果相机视野是 1280×720,声音半径设 1200 左右就够;再大反而把听觉范围拉到屏幕之外,玩家听到看不见的东西,体验差。
- **`max_polyphony = 4`**:同一个 `AudioStreamPlayer2D` 同时最多 4 个重叠播放。脚步声、剑击声需要重叠;只是"打开背包"这种 UI 音不需要。
- **`pitch_scale` 随机化**:同一段 wav 连播 5 次,人耳会感到机械;加 ±8% 的随机音高就自然了。

调用方写法:

```gdscript
# 在被击中的敌人脚本里
func take_damage(amount: int, source_position: Vector2) -> void:
    health -= amount
    HitSparkPool.spawn(global_position,
        (global_position - source_position).normalized())
    SFXEmitter.emit_at(get_tree().current_scene, hit_sfx, global_position)
    if health <= 0:
        die()
```

`HitSparkPool` 是单例池(在 17 篇会扩展成更通用的"反馈调度器"),`SFXEmitter.emit_at` 是临时音效节点,播完自动 free。

### 动态音乐:平行轨道淡入淡出

"动态音乐"指音乐随游戏状态(探索 / 战斗 / Boss)切换层次。最稳健的实现是**多个 `AudioStreamPlayer` 同时播相同节奏的不同轨道,通过 bus 音量切换可听性**——切换时不会有"音乐重启"的违和感,因为所有轨道时间轴对齐,只是某些轨道音量为 0。

`demo/scripts/audio/music_manager.gd`,管理 3 个并行 BGM 轨道:

```gdscript
extends Node
class_name MusicManager

@export var explore_stream: AudioStream            # 探索层(钢琴 + 弦乐)
@export var combat_stream: AudioStream             # 战斗层(鼓 + 贝斯)
@export var boss_stream: AudioStream               # Boss 层(铜管 + 合唱)

@onready var _explore: AudioStreamPlayer = $Explore
@onready var _combat: AudioStreamPlayer = $Combat
@onready var _boss: AudioStreamPlayer = $Boss

enum State { EXPLORE, COMBAT, BOSS }

var _state: State = State.EXPLORE
var _tween: Tween

func _ready() -> void:
    for player in [_explore, _combat, _boss]:
        player.bus = &"Music"
        player.volume_db = -80.0          # 默认全部静音
        player.autoplay = false
    _explore.stream = explore_stream
    _combat.stream = combat_stream
    _boss.stream = boss_stream
    # 同时启动,保持时间轴对齐
    _explore.play()
    _combat.play()
    _boss.play()
    _set_audible(_explore, true, 0.0)

func transition(new_state: State, fade: float = 1.5) -> void:
    if new_state == _state:
        return
    _state = new_state
    var layers: Dictionary = {
        State.EXPLORE: _explore,
        State.COMBAT: _combat,
        State.BOSS: _boss,
    }
    for s in layers:
        _set_audible(layers[s], s == new_state, fade)

func _set_audible(player: AudioStreamPlayer, audible: bool, fade: float) -> void:
    var target_db: float = 0.0 if audible else -80.0
    if _tween and _tween.is_running():
        # 同一玩家上一次未完成的渐变不要打架,Godot tween 默认会替换
        pass
    var t := create_tween()
    t.tween_property(player, "volume_db", target_db, fade)
```

要点:

- 3 个 player 永远都在播放,时间戳一致,切换只调整音量。**`volume_db = -80` 不等于停止**,音频引擎仍在解码——这是平行轨道方案的代价,在低端机要权衡。
- `play()` 在 `_ready` 集中调用,确保所有轨道从 0 帧开始;如果分散在不同节点 `_ready`,可能有几毫秒错位,长段循环时人耳会感觉"鼓没踩稳"。
- 进 Boss 战时直接 `MusicManager.transition(MusicManager.State.BOSS, 0.5)`,0.5 秒淡入,玩家几乎感觉不到切换。


## 4. 调参和验收

### 粒子参数与手感

| 反馈类型 | `amount` | `lifetime` | `explosiveness` | `spread` | 备注 |
| --- | --- | --- | --- | --- | --- |
| 命中火花 | 16-32 | 0.3-0.5 | 0.9-1.0 | 30-60 | 越爆炸越短促 |
| 跳跃尘烟 | 6-12 | 0.4-0.6 | 0.6 | 90 | 半圆形,扁平 |
| 受击血溅 | 24-48 | 0.6-0.9 | 0.95 | 45-90 | 加重力下落 |
| 治疗光粒 | 12-20 | 1.2-1.8 | 0.0 | 360 | 持续上飘 |
| 走路落叶 | 4-8 | 2.0-3.0 | 0.0 | 180 | 轻 damping |

`explosiveness` 是粒子生成时间分布的"集中度":`0` 表示在 `lifetime` 内均匀生成,`1` 表示**所有粒子在第一帧全部生成**。命中和爆炸用 `1`,持续效果(雨、烟、雪)用 `0`。

### 音频参数与可读性

| 反馈类型 | bus | attenuation | max_distance | max_polyphony | pitch_jitter |
| --- | --- | --- | --- | --- | --- |
| 玩家攻击 | SFX | 1.0(玩家身边) | 1500 | 4 | 0.08 |
| 敌人攻击 | SFX | 2.5 | 1200 | 4 | 0.10 |
| 远处环境(风、瀑布) | Ambient | 1.5 | 3000 | 2 | 0 |
| UI 点击 | UI | (用 AudioStreamPlayer,不需要 2D) | - | 2 | 0 |
| BGM | Music | (用 AudioStreamPlayer) | - | 1 | 0 |

UI 与 BGM 用 `AudioStreamPlayer`(非 2D),它们没有空间属性,bus 直接路由。**UI 音效尤其要保证短延迟**,所以 UI bus 不挂效果器。

### 验收清单(运行时)

`HitSparkPool` 完成后,验证项是:

1. 让 60 个敌人同时被击中,粒子不出现"空帧"(池没满),也不出现"丢粒子"(`restart()` 被覆盖)。
2. 玩家冲刺时,留在身后的命中粒子**不跟随玩家移动**(`local_coords = false` 验证)。
3. 改 `mat.direction` 不影响"已经飞出的粒子",只影响"下一发"的初始方向。

音频系统的验收:

1. 在 `Audio.set_volume(SFX, 0.0)` 后,所有走 SFX 总线的音效消失,但 BGM(Music 总线)继续播放。
2. 触发 `Audio.enter_combat()` 后 0.5 秒内,音乐应明显变"闷"(LowPass 起效),环境音音量降低。
3. 远处的敌人脚步声(40-50% `max_distance` 处)比近处明显小,且能听出左右声道偏移(`panning_strength = 1.0` 验证)。
4. 连续触发 100 次玩家攻击音效(用 `Input.is_action_just_pressed` 在 `_process` 里疯狂调用),不出现卡顿或破音(`max_polyphony` 与 Limiter 配合验证)。

### Bus 监控

`Audio` 单例里加一个调试方法,把所有 bus 当前 dB 和峰值打到 console:

```gdscript
func debug_dump_buses() -> void:
    for i in AudioServer.bus_count:
        var name := AudioServer.get_bus_name(i)
        var db := AudioServer.get_bus_volume_db(i)
        var peak_l := AudioServer.get_bus_peak_volume_left_db(i, 0)
        var peak_r := AudioServer.get_bus_peak_volume_right_db(i, 0)
        push_warning("[bus %d] %s volume=%.1f dB peak L=%.1f R=%.1f"
            % [i, name, db, peak_l, peak_r])
```

把它绑在 F4 上,运行时观察 peak。如果 Master 的 peak 经常顶到 0 dB,说明 Limiter 在持续工作,要么 SFX 太响,要么音效本身过响。0 dB 顶峰 = 爆音边界。

## 5. 踩坑

### `GPUParticles2D` 的 `visibility_rect` 默认太小

`GPUParticles2D.visibility_rect` 默认是 `Rect2(-100, -100, 200, 200)`,以发射器为中心。这意味着粒子飞出这个 200×200 矩形后,即便还在屏幕里,Godot 也会判定整个 `GPUParticles2D` 节点"不可见",GPU 上的粒子模拟仍在继续,但**屏幕上消失了**——你会看到粒子飞着飞着突然消失。

修正方法两种:① 把 `visibility_rect` 改大(命中粒子设 `Rect2(-256, -256, 512, 512)` 足够);② 调用 `node.set_visibility_rect_from_aabb()`(4.x 新增,但只支持 3D)或在工具脚本里一次性生成合适大小。本系列推荐**每个粒子场景在 .tscn 里就把 `visibility_rect` 设成正确大小**,不要靠默认。

### `restart()` 与 `emitting` 的微妙差别

`emitting = true` 仅在节点当前 `emitting = false` 时触发新一轮发射;如果上一发还在播,设置成 true 不会重启。`restart()` 强制重置内部状态,从头开始。**池里复用粒子节点时必须用 `restart()`**,这是新手经常忽略的坑——表现就是"快速连击时部分粒子不出现"。

### `local_coords` 的两面性

`local_coords = true` 时,粒子坐标相对于发射器:跑步的角色身上挂个"光晕"粒子,光晕始终包裹角色。但所有粒子都"被锁住"——你想做"留在原地的尘烟"绝对不能开这个。**默认 `local_coords = false`,只有"挂在身上不脱离"的特效才开**。

### `ParticleProcessMaterial` 是 `Resource`,默认会被实例间共享

在 Inspector 里给 `GPUParticles2D.process_material` 拖一份 `.tres`,**所有引用同一份 .tres 的节点共享 material**。改一个就影响所有,这是粒子池里"我明明只改了一个粒子的方向,所有粒子的方向都变了"的原因。

修正方法:① 池里每个节点 `process_material = process_material.duplicate()` 显式 duplicate;② 在 .tscn 里把 material 内联保存(不另存为 .tres),Godot 会自动给每份 .tscn 实例独立的 material。本系列推荐做法是 **共享通用模板 .tres,使用前 duplicate**。

### `AudioStreamPlayer.bus` 没有运行时校验

文档写得很清楚:`bus` 字段在设置时**不校验名字是否存在**。原因是 bus 布局可能在场景加载后才载入。如果你写错了名字(`"sfx"` vs `"SFX"`),`AudioServer` 找不到就**回退到 Master**,声音照常播,但你以为它走的是 SFX 总线、改 SFX 音量没生效——查这种 bug 非常痛苦。

防御办法:① bus 名集中在 `const StringName` 里,引用而不是字面量;② 在 `Audio` 单例 `_ready` 时遍历断言所有 bus 都存在:

```gdscript
for bus in [MASTER, MUSIC, SFX, UI, AMBIENT]:
    assert(AudioServer.get_bus_index(bus) >= 0, "Missing bus: %s" % bus)
```

### 极小线性音量 → 极小 dB 不等价于静音

`linear_to_db(0.0001)` 大约是 `-80 dB`,人耳几乎听不见但**不是真正静音**——音频仍在被处理,效果器仍在 CPU 上算。如果想"暂停 BGM",应该用 `AudioServer.set_bus_mute(idx, true)`,或者把 `AudioStreamPlayer` 的 `stream_paused = true`。本篇代码里 `set_volume(linear = 0)` 时同步触发 mute,就是这个原因。

### Trail 需要专门的纹理

很多人开 `trail_enabled = true` 后看到拖尾是一串"圆形粒子排成线",觉得很丑。这是因为 `GPUParticles2D.texture` 用了普通圆形粒子图。Trail 的 ribbon 会沿轴拉伸贴图,需要画一张**两端透明、中间不透明、横向延伸**的胶囊形 PNG;或者用渐变白线。让 trail "看起来像光剑残影"需要的纹理设计与单粒子完全不同。

### `pitch_scale < 0` 不会倒放

直觉上 `AudioStreamPlayer.pitch_scale = -1.0` 会倒放音频,但 Godot 不支持这种用法,负值会被裁到接近 0,效果是"几乎不动"。要倒放需要预先生成倒放 wav 文件,或用 `AudioStreamGenerator` 自己写采样回调——后者超出本系列范围。

### Web 平台的音频策略限制

浏览器在用户没交互前不允许播放音频(Chrome/Firefox/Safari 都有这个安全限制)。这意味着主菜单 BGM 在 `_ready` 阶段播放是无效的——需要等到第一次点击或按键。修正:在主菜单第一个 `_unhandled_input` 里启动 BGM,或者把"按任意键继续"做成标题画面的一部分。Godot 不会替你处理这个,Web 出口时务必显式测试。

### `AudioStreamPlayer.finished` 信号不保证立即发出

`AudioStreamPlayer.finished` 在音频播完时发出,但**有几十毫秒延迟**——音频引擎的 buffer 还在播,信号才落到主线程。如果你写 `play() → await finished → play_next()`,几乎不会卡顿;但如果你的逻辑依赖"刚 `finished` 立刻同步另一个状态",会观察到非确定性。靠谱的做法是用 `Timer` 计时 + `stream.get_length()` 计算,关键节奏(比如音乐节拍触发)不要靠 `finished`。
