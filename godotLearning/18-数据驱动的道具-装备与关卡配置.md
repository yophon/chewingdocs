# 数据驱动的道具、装备与关卡配置

如果每新增一个道具都要新写一个脚本,内容很快会拖垮项目。数据驱动的目标不是炫技,而是让新增内容尽量不改代码。

> 一句话先记住:**数值、图标、文案、引用进 Resource;行为和流程留在代码。**

---

## 一、先分清数据和行为

适合进数据:

```text
道具 id
名字
描述
图标
价格
堆叠上限
恢复多少血
掉落权重
对应场景
```

不适合进数据:

```text
背包怎么排序
攻击流程怎么取消
敌人 AI 怎么决策
场景怎么切换
```

数据驱动不是把代码写进表里。表描述“是什么”,代码处理“怎么做”。

## 二、ItemResource

```gdscript
# res://items/item_resource.gd
class_name ItemResource
extends Resource

enum Kind { CONSUMABLE, EQUIPMENT, KEY, MATERIAL }

@export var id: StringName
@export var display_name: String
@export_multiline var description: String
@export var kind: Kind = Kind.CONSUMABLE
@export var icon: Texture2D
@export var max_stack: int = 99
@export var price: int = 0

@export_group("Effect")
@export var heal_amount: int = 0
@export var attack_bonus: int = 0
@export var defense_bonus: int = 0
```

然后创建:

```text
res://items/data/potion_small.tres
res://items/data/key_red.tres
res://items/data/iron_sword.tres
```

新增一瓶药水,只建 `.tres`,不写新脚本。

## 三、拾取物 = 场景 + 数据

地上的拾取物只有一个通用场景:

```text
PickupItem (Area2D)
├── Sprite2D
└── CollisionShape2D
```

脚本:

```gdscript
# res://items/pickup_item.gd
class_name PickupItem
extends Area2D

@export var item: ItemResource
@export var count: int = 1

@onready var _sprite: Sprite2D = %Sprite2D

func _ready() -> void:
    assert(item != null, "PickupItem.item must be assigned")
    _sprite.texture = item.icon
    body_entered.connect(_on_body_entered)

func _on_body_entered(body: Node) -> void:
    if body.has_method("add_item"):
        body.add_item(item, count)
        EventBus.item_picked.emit(item.id, count)
        queue_free()
```

掉一瓶小药水 = 实例化 `PickupItem.tscn`,把 `potion_small.tres` 塞进去。

## 四、掉落表

```gdscript
# res://items/drop_table_resource.gd
class_name DropTableResource
extends Resource

@export var entries: Array[DropEntry] = []

func roll(rng: RandomNumberGenerator) -> Array[ItemStack]:
    var result: Array[ItemStack] = []
    for entry in entries:
        if rng.randf() <= entry.chance:
            result.append(ItemStack.new(entry.item, rng.randi_range(entry.min_count, entry.max_count)))
    return result
```

配套条目:

```gdscript
# res://items/drop_entry.gd
class_name DropEntry
extends Resource

@export var item: ItemResource
@export_range(0.0, 1.0, 0.01) var chance: float = 1.0
@export var min_count: int = 1
@export var max_count: int = 1
```

简单栈对象:

```gdscript
# res://items/item_stack.gd
class_name ItemStack
extends RefCounted

var item: ItemResource
var count: int

func _init(p_item: ItemResource, p_count: int) -> void:
    item = p_item
    count = p_count
```

敌人死亡:

```gdscript
@export var drop_table: DropTableResource
@export var pickup_scene: PackedScene

func drop_items() -> void:
    if drop_table == null or pickup_scene == null:
        return

    var rng := RandomNumberGenerator.new()
    for stack in drop_table.roll(rng):
        var pickup: PickupItem = pickup_scene.instantiate()
        pickup.item = stack.item
        pickup.count = stack.count
        get_tree().current_scene.add_child(pickup)
        pickup.global_position = global_position
```

以后调掉落率,改 `.tres`。

## 五、道具数据库

业务代码常常只有 `item_id`,需要查资源。做一个索引:

```gdscript
# res://items/item_database.gd
extends Node

@export var items: Array[ItemResource] = []

var _by_id: Dictionary = {}

func _ready() -> void:
    for item in items:
        assert(not _by_id.has(item.id), "Duplicate item id: %s" % item.id)
        _by_id[item.id] = item

func get_item(id: StringName) -> ItemResource:
    return _by_id.get(id)
```

把它做成 Autoload 或挂在内容管理节点上。注意检查重复 id。

## 六、关卡配置也用 Resource

```gdscript
# res://levels/level_config.gd
class_name LevelConfig
extends Resource

@export var id: StringName
@export var scene: PackedScene
@export var display_name: String
@export var next_level_id: StringName
@export var music: AudioStream
@export var par_time_sec: float = 120.0
```

主菜单开始关卡时,拿 `LevelConfig`,不是硬编码路径。

---

## 验收

- 新增普通道具不需要新增脚本。
- 拾取物场景通用,具体道具由 `ItemResource` 决定。
- 掉落率写在 `DropTableResource`。
- 道具 id 有集中索引,并检查重复。
- 关卡路径和音乐放进 `LevelConfig`。

---

## 常见坑

**坑 1:Resource 里写复杂流程。**

字段可以描述效果,但具体怎么应用交给系统。

**坑 2:id 随手写字符串。**

用 `StringName`,并在数据库启动时检查重复。

**坑 3:每个道具一个场景。**

大多数道具共用拾取场景,差异用资源配置。

**坑 4:运行时直接改共享 Resource。**

Resource 是配置。背包里的数量、耐久是运行时状态,不要写回共享道具资源。
