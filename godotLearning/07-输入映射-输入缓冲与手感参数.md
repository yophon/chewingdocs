# 输入映射、输入缓冲与手感参数

第 06 篇的角色已经能跑能跳,但“能动”和“好按”是两回事。新手做出来的角色常见问题是:按跳晚半拍就不跳,按早一点也不跳,松开方向键还滑很远,打中敌人没有重量。

这些都不是“速度不够大”的问题,而是输入和物理之间缺了一层缓冲。

> 一句话先记住:**手感不是参数越大越爽,而是玩家的意图不要被游戏漏掉。**

---

## 一、先把 InputMap 定下来

游戏逻辑不要直接判断键盘按键:

```gdscript
if Input.is_key_pressed(KEY_SPACE):
    jump()
```

这会让改键、手柄、键盘布局全部变麻烦。

先在项目设置里建动作:

```text
move_left
move_right
move_up
move_down
jump
attack
dash
pause
```

代码只认动作:

```gdscript
var input_x := Input.get_axis(&"move_left", &"move_right")

if Input.is_action_just_pressed(&"jump"):
    jump()
```

键盘、手柄、触屏以后都可以映射到同一个动作。

---

## 二、为什么会“按了但没跳”

平台跳跃里有两个常见尴尬:

```text
玩家还没落地,提前按了跳
玩家刚离开平台边缘,才按下跳
```

严格代码会这么写:

```gdscript
if Input.is_action_just_pressed(&"jump") and is_on_floor():
    velocity.y = jump_velocity
```

这句太苛刻。它要求玩家必须在“按下跳跃”的那一帧同时“正在地面上”。现实里玩家不是机器,人会早一点或晚一点。

所以要加两层容错:

```text
jump buffer: 按早了,先记住这次跳跃意图
coyote time: 刚离地,还允许短时间补跳
```

## 三、输入缓冲

先写一个小节点,专门记“刚才按过什么”。

```gdscript
# res://player/input_buffer.gd
class_name InputBuffer
extends Node

@export var jump_buffer_sec: float = 0.12
@export var attack_buffer_sec: float = 0.10
@export var dash_buffer_sec: float = 0.08

var _jump_timer: float = 0.0
var _attack_timer: float = 0.0
var _dash_timer: float = 0.0

func _unhandled_input(event: InputEvent) -> void:
    if event.is_action_pressed(&"jump"):
        _jump_timer = jump_buffer_sec
    if event.is_action_pressed(&"attack"):
        _attack_timer = attack_buffer_sec
    if event.is_action_pressed(&"dash"):
        _dash_timer = dash_buffer_sec

func _physics_process(delta: float) -> void:
    _jump_timer = maxf(_jump_timer - delta, 0.0)
    _attack_timer = maxf(_attack_timer - delta, 0.0)
    _dash_timer = maxf(_dash_timer - delta, 0.0)

func consume_jump() -> bool:
    if _jump_timer <= 0.0:
        return false
    _jump_timer = 0.0
    return true

func consume_attack() -> bool:
    if _attack_timer <= 0.0:
        return false
    _attack_timer = 0.0
    return true

func consume_dash() -> bool:
    if _dash_timer <= 0.0:
        return false
    _dash_timer = 0.0
    return true
```

为什么用 `_unhandled_input`?

因为 UI 先处理输入。暂停菜单打开时,菜单消费掉按键,角色就不会在后台缓存一堆跳跃和攻击。

## 四、土狼时间

土狼时间就是“刚离开地面的一小段时间,仍然允许跳”。

在玩家脚本里加:

```gdscript
@export var coyote_time_sec: float = 0.10

var _coyote_timer: float = 0.0

func _physics_process(delta: float) -> void:
    if is_on_floor():
        _coyote_timer = coyote_time_sec
    else:
        _coyote_timer = maxf(_coyote_timer - delta, 0.0)
```

判断能否跳:

```gdscript
func _can_jump() -> bool:
    return is_on_floor() or _coyote_timer > 0.0
```

把它和输入缓冲合起来:

```gdscript
@onready var _buffer: InputBuffer = %InputBuffer

func _physics_process(delta: float) -> void:
    _update_coyote_time(delta)
    _apply_horizontal(delta)
    _apply_gravity(delta)

    if _buffer.consume_jump() and _can_jump():
        velocity.y = jump_velocity
        _coyote_timer = 0.0

    move_and_slide()
```

这样玩家按早一点、晚一点,都能得到符合直觉的结果。

## 五、加速度和摩擦不是玄学

角色横向手感主要由三个数决定:

```gdscript
@export var max_speed: float = 220.0
@export var acceleration: float = 1600.0
@export var friction: float = 1800.0
```

代码:

```gdscript
func _apply_horizontal(delta: float) -> void:
    var input_x := Input.get_axis(&"move_left", &"move_right")

    if absf(input_x) > 0.01:
        velocity.x = move_toward(velocity.x, input_x * max_speed, acceleration * delta)
    else:
        velocity.x = move_toward(velocity.x, 0.0, friction * delta)
```

调参时不要只看数值,看“多久到顶速”:

```text
到顶速时间 = max_speed / acceleration
```

`220 / 1600 = 0.1375 秒`,就是 137ms 左右到满速。动作游戏通常会比较短,解谜平台可以更长。

## 六、跳跃高度怎么调

跳跃不是只调 `jump_velocity`。

```gdscript
@export var jump_velocity: float = -420.0
@export var gravity_up: float = 980.0
@export var gravity_down: float = 1400.0
@export var jump_cut_multiplier: float = 0.45
```

上升时用较小重力,下落时用较大重力:

```gdscript
func _apply_gravity(delta: float) -> void:
    if is_on_floor() and velocity.y >= 0.0:
        return

    var g := gravity_down if velocity.y > 0.0 else gravity_up
    velocity.y += g * delta
```

松开跳跃键截断上升:

```gdscript
func _unhandled_input(event: InputEvent) -> void:
    if event.is_action_released(&"jump") and velocity.y < 0.0:
        velocity.y *= jump_cut_multiplier
```

这样能做出“轻按小跳,长按高跳”。

## 七、hit stop 是打击感的第一步

命中停顿不是让代码 `await` 一下,而是短时间冻结角色更新。

最简单做法:

```gdscript
@export var hit_stop_sec: float = 0.08

var _freeze_timer: float = 0.0

func _physics_process(delta: float) -> void:
    if _freeze_timer > 0.0:
        _freeze_timer = maxf(_freeze_timer - delta, 0.0)
        return

    _normal_physics_update(delta)

func apply_hit_stop(amount: float = hit_stop_sec) -> void:
    _freeze_timer = maxf(_freeze_timer, amount)
```

被击中、打中敌人、重攻击命中时都可以调用。不要一开始就全局 `Engine.time_scale = 0`,那会把 UI、镜头、计时器都一起拖进来。

## 八、推荐起步参数

不是标准答案,只是能跑起来的初始值:

```text
max_speed              200-260
acceleration           1400-2200
friction               1600-2600
jump_velocity          -360 到 -460
gravity_up             850-1100
gravity_down           1200-1700
jump_buffer_sec        0.08-0.14
coyote_time_sec        0.08-0.12
hit_stop_sec           0.04-0.10
```

调参顺序:

1. 先定角色大小和一格地砖大小。
2. 再定跳多高。
3. 再调水平速度。
4. 最后加输入缓冲和土狼时间。

别反过来。

## 验收

- 所有动作都来自 `InputMap`,不是硬编码键位。
- 跳跃按早 0.1 秒,落地后仍能跳。
- 离开平台边缘 0.1 秒内,仍能补跳。
- 松开跳跃键会降低高度。
- 横向移动有加速和减速,不是瞬间满速。
- hit stop 不会暂停整个 UI。

---

## 常见坑

**坑 1:缓冲时间太长。**

`jump_buffer_sec = 0.3` 会让角色像自动跳。一般 0.1 秒上下够了。

**坑 2:coyote time 太长。**

太长会让玩家明显感觉“离开平台很远还能跳”。横版动作先从 0.1 秒开始。

**坑 3:把输入和 UI 抢在一起。**

游戏角色用 `_unhandled_input`,让菜单先消费事件。

**坑 4:hit stop 用 await。**

`await create_timer()` 只会让当前函数等,不会真正冻结角色物理。
