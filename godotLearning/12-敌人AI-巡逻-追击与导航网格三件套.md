# 敌人 AI、巡逻、追击与导航网格三件套

有了地图以后,下一步是让敌人动起来。不要一开始就写“智能 AI”。先做一个能巡逻、能发现玩家、能追、能攻击、能被打断的敌人。

> 一句话先记住:**状态机决定敌人想做什么,NavigationAgent2D 只负责告诉它下一步往哪走。**

---

## 一、别把追击写成直线移动

最简单的追击:

```gdscript
global_position = global_position.move_toward(player.global_position, speed * delta)
```

开阔场地能用,一有墙就卡。敌人不知道哪里能走,只知道玩家在哪。

Godot 2D 导航用三件套:

```text
NavigationRegion2D    提供可走区域
NavigationAgent2D     每个敌人一个,负责路径和避让
NavigationObstacle2D  动态障碍,可选
```

`NavigationRegion2D` 是地图。`NavigationAgent2D` 是敌人的导航助手。敌人的移动仍然由你自己写。

---

## 二、敌人先有五个状态

最小敌人状态:

```text
IDLE      发呆或等待
PATROL    在几个点之间巡逻
CHASE     发现玩家后追击
ATTACK    距离够近时攻击
STAGGER   被打后硬直
```

不要把这些都塞进一个 `_physics_process` 巨型 if。先写枚举:

```gdscript
enum State { IDLE, PATROL, CHASE, ATTACK, STAGGER, DEAD }

var _state: State = State.PATROL
```

## 三、一个可用的敌人骨架

场景:

```text
Enemy (CharacterBody2D)
├── %NavigationAgent2D
├── %VisionArea (Area2D)
├── %AttackArea (Area2D)
└── %AnimationPlayer
```

脚本:

```gdscript
# res://enemy/enemy.gd
class_name Enemy
extends CharacterBody2D

enum State { IDLE, PATROL, CHASE, ATTACK, STAGGER, DEAD }

@export var speed: float = 120.0
@export var attack_distance: float = 28.0
@export var patrol_points: Array[Node2D] = []
@export var player: Node2D

@onready var _agent: NavigationAgent2D = %NavigationAgent2D
@onready var _anim: AnimationPlayer = %AnimationPlayer

var _state: State = State.PATROL
var _patrol_index: int = 0
var _stagger_timer: float = 0.0

func _ready() -> void:
    _agent.path_desired_distance = 8.0
    _agent.target_desired_distance = 12.0

func _physics_process(delta: float) -> void:
    match _state:
        State.IDLE:
            velocity = Vector2.ZERO
        State.PATROL:
            _tick_patrol()
        State.CHASE:
            _tick_chase()
        State.ATTACK:
            velocity = Vector2.ZERO
        State.STAGGER:
            _stagger_timer -= delta
            if _stagger_timer <= 0.0:
                _state = State.CHASE
        State.DEAD:
            velocity = Vector2.ZERO

    move_and_slide()

func _tick_patrol() -> void:
    if patrol_points.is_empty():
        velocity = Vector2.ZERO
        return

    var target := patrol_points[_patrol_index].global_position
    _agent.target_position = target
    _move_to_next_path_point()

    if global_position.distance_to(target) < 12.0:
        _patrol_index = (_patrol_index + 1) % patrol_points.size()

func _tick_chase() -> void:
    if player == null:
        _state = State.PATROL
        return

    if global_position.distance_to(player.global_position) <= attack_distance:
        _state = State.ATTACK
        _anim.play(&"attack")
        return

    _agent.target_position = player.global_position
    _move_to_next_path_point()

func _move_to_next_path_point() -> void:
    if _agent.is_navigation_finished():
        velocity = Vector2.ZERO
        return

    var next := _agent.get_next_path_position()
    var direction := global_position.direction_to(next)
    velocity = direction * speed

func see_player(target: Node2D) -> void:
    player = target
    if _state != State.DEAD:
        _state = State.CHASE

func take_hit(knockback: Vector2) -> void:
    if _state == State.DEAD:
        return
    velocity = knockback
    _stagger_timer = 0.18
    _state = State.STAGGER
    _anim.play(&"hurt")
```

这个敌人还不聪明,但结构是对的:状态机管行为,导航代理给路径点,移动仍然走 `CharacterBody2D`。

## 四、发现玩家

可以先用 `Area2D`:

```gdscript
func _on_vision_area_body_entered(body: Node) -> void:
    if body is PlayerBody:
        see_player(body)
```

如果要判断墙后看不见,再加 `RayCast2D`:

```gdscript
func has_line_of_sight(target: Node2D) -> bool:
    %VisionRay.target_position = to_local(target.global_position)
    %VisionRay.force_raycast_update()
    return not %VisionRay.is_colliding() or %VisionRay.get_collider() == target
```

先不要写复杂感知系统。能发现、能丢失、能追击,已经足够支撑原型。

## 五、攻击不要直接扣血

攻击动画打开 hitbox:

```gdscript
func enable_attack_hitbox() -> void:
    %AttackHitbox.monitoring = true

func disable_attack_hitbox() -> void:
    %AttackHitbox.monitoring = false
```

hitbox 碰到玩家 hurtbox 后再发伤害:

```gdscript
func _on_attack_hitbox_area_entered(area: Area2D) -> void:
    if area.has_method("receive_hit"):
        area.receive_hit(10, global_position.direction_to(area.global_position))
```

敌人脚本不需要知道玩家血量系统怎么写。它只制造一次命中。

## 六、导航常见设置

`NavigationAgent2D` 常用字段:

```text
path_desired_distance     离路径点多近算到达
target_desired_distance   离最终目标多近算完成
avoidance_enabled         是否开启避让
radius                    代理半径
```

原型期先不开复杂避让。敌人少时,直接让它们沿路径点走就够。

敌人多了再开启:

```gdscript
_agent.avoidance_enabled = true
_agent.velocity_computed.connect(_on_safe_velocity_computed)
```

不要一开始就把避让、队形、权重全写进去。

## 七、掉落物

死亡时不要在敌人脚本里硬编码掉什么。先导出一个场景:

```gdscript
@export var drop_scene: PackedScene

func die() -> void:
    _state = State.DEAD
    if drop_scene != null:
        var drop := drop_scene.instantiate()
        get_tree().current_scene.add_child(drop)
        drop.global_position = global_position
    queue_free()
```

第 18 篇会把它升级成掉落表资源。

## 验收

- 敌人至少有巡逻、追击、攻击、硬直四种状态。
- 追击不是直线穿墙,而是用 `NavigationAgent2D`。
- 攻击通过 hitbox/hurtbox,不是敌人直接改玩家 HP。
- 被打后有短暂硬直,不会立刻继续追。
- 掉落物通过 `PackedScene` 生成,不是硬编码节点。

## 常见坑

**坑 1:把 NavigationAgent2D 当自动移动器。**

它只给你下一个路径点。真正移动还是你写 `velocity`。

**坑 2:每帧都重新烘焙导航。**

静态地图烘焙一次。动态障碍用 obstacle 或简单避让。

**坑 3:敌人状态太少。**

只有 chase 和 attack 会导致被打也不停、攻击也能滑动。至少加硬直。

**坑 4:敌人直接引用 HUD 或存档。**

敌人只管行为和掉落,流程交给关卡或管理器。
