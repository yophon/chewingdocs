# 27-本地化、字体与输入设备适配

> 一句话导读:本地化、字体与输入设备适配看上去是三件不相关的事,但它们都在回答同一个问题——"我的游戏在 100 个不同口味的玩家手里,长什么样,操作怎么响应"。这一篇把这三条链路在 Godot 4.6 里的契约串到一起,并补上独立游戏最容易忽略的可访问性基础。

把本地化、字体、输入这三件事写在一篇里不是偷懒,而是它们共享一个心智:**游戏不该假定玩家用英文键盘 + 美式生活方式 + 完美视力。** 这三块只要漏一块,游戏在某个用户那里就会卡死——日韩玩家看到方块字符、欧洲玩家手柄按 X 跳跃但 UI 提示 A、色弱玩家分不清红绿子弹。每一块独立修都很简单,串起来一起设计能让你在原型阶段就避开 70% 的发布期返工。

## 1. 机制定位

新手在这三个点上常见的错误,各有典型形态。

**本地化方向**最常见的错觉是"先用中文字符串硬写,以后再改"。等到要加日语时才发现,所有 GDScript 文件、所有 Label 节点、所有 print 调试语句、所有报错消息都散落着中文字符串,迁移成本是从头开始重写文本层。Godot 提供的 `tr()` + `Translation` 资源 + CSV 工作流的设计正好回避这个陷阱:**从第一天起,你的代码里出现的就是 `START_GAME` 这样的 key,真正的字符串永远住在 CSV 表里。** 早期阶段你可以只填一种语言,但语义上已经预留了扩展位。

**字体方向**最常见的错觉是"我喜欢这个字体,全游戏用它就好"。但 Latin 字体不带 CJK 字符,日韩中字符渲染成方块(俗称 tofu);而 CJK 字体之间——日文、简中、繁中——同一个 Unicode 码点在不同字形下长得完全不一样。`fallback` 链可以解前一个问题,但解不了后一个,这部分要走 locale-aware 的字体切换。

**输入方向**最常见的错觉是"我先用键盘做,以后手柄玩家自己适配"。Godot 的 `InputMap` 让"按键 → 抽象动作名"这层映射是统一的(无论键盘、手柄、触屏),但 UI 上显示什么提示("按 A 跳跃" vs "按 Space 跳跃")需要根据当前活跃的输入设备动态切换。这是 UI 设计意识,不是技术问题,但有典型的 Godot 实现路径——监听 `InputEvent` 类型变化、维护一个"当前活跃设备"状态。

把这三块放一起的真正心智:**做一款"对玩家透明"的游戏,意味着同时在文本、字形、操作三层都建立可切换的契约**。每一层都有 Godot 给你的官方机制,本系列前面的篇目讲手感、UI 时已经用过其中的一部分,本篇把它们补齐。

还有一条经常被忽略的理由:**这三块都是发售前最容易拖延、最容易拉爆排期的地方。** 翻译需要外包协作、字体需要 license 谈判、手柄需要真机测试。如果在原型阶段就铺好基础设施(CSV key 化、字体 fallback 链、设备追踪),后期只是"填表"——CSV 多加一列日语,Theme 多挂一份字体,UI 提示多一组手柄图标。如果原型阶段是硬编码字符串、单一字体、键盘 only,发售前两周才补,通常会延期一个月。这是工程经验,不是危言耸听。

## 2. Godot 心智

### 2.1 `Translation` 资源与 `tr()` 调用链

Godot 的本地化模型有四个核心组件:

- **`Translation` 资源**:一种 Godot 资源类型,内部是 `key → string` 字典,序列化为 `.translation` 文件,绑定一个 locale 标签(`en`、`zh_CN`、`ja` 等)。CSV 文件被 `ResourceImporterCSVTranslation` 自动 import 成多个 `.translation` 资源,每个 column 一个 locale;
- **`TranslationServer`**:全局单例,维护当前 locale 和所有已注册的 `Translation` 资源;调用 `tr(key)` 时它在所有匹配 locale 的 translation 里查;
- **`tr()` 全局函数**:Object 方法,所有节点都能直接调,语义是"按当前 locale 查 key";
- **`Node.auto_translate_mode`**:Godot 4 把 auto-translate 属性从 Control 移到了 Node 层,默认 `AUTO_TRANSLATE_MODE_INHERIT`;`Label.text = "START_GAME"` 这种字符串在渲染时被 auto-translate 一遍,这就是 `tr("START_GAME")` 的等价路径,但走 UI 节点的 setter 不需要手动调 `tr`。

调用链:玩家在设置里选了"日本語" → 你调 `TranslationServer.set_locale("ja")` → 所有 UI 节点重绘时各自走一遍 auto-translate → 用 ja translation 重新填充文本。**自定义 GDScript 的字符串构造(如 `"%s 命中" % name`)不会被 auto-translate,要么手动 `tr()`,要么用 `tr("%s_HIT").format([name])` 这种 key 化形式。**

CSV 是 Godot 推荐的工作流,因为表格协作友好,翻译同事用 Excel / Google Sheets 就能改。PO(gettext)格式 Godot 也支持,适合需要复数形式(`tr_n`)的欧洲语言;CSV 不支持复数,日韩中没这个问题,大多数独立游戏 CSV 足够。

### 2.2 字体与 fallback 链

Godot 字体层有两类资源:

- **`FontFile`**:一份 TTF / OTF / WOFF / WOFF2 字体文件的封装,加载到内存后供 TextServer 渲染;
- **`FontVariation`**:基于一个 `Font`(`FontFile` 或另一个 `FontVariation`)做派生,可以改 OpenType 轴(weight、italic)、字距(spacing)、faux bold/italic、添加 fallback。

fallback 是一个有序列表:**渲染某个 Unicode 字符时,从主字体开始查,主字体没有这个字形就查 fallback[0],没有就查 fallback[1]……找完仍没有就显示 tofu 方块。** 你可以在 `FontFile` 上直接 `add_fallback()`,也可以在 `FontVariation` 里维护 fallback 列表。后者的好处是不污染原始 `FontFile` 资源,你的 emoji fallback 只挂在 UI 用的 `FontVariation` 上,游戏里别的地方继续用原字体。

CJK 字形重叠是 fallback 链解不了的语言学问题:Unicode 码点 U+6587 在简中是"文",日文是"文",字形细节(撇捺角度)不同,正确做法是**按 locale 切换主字体**而不是把简中日文都堆进 fallback 链。一个 `Theme` 资源可以挂在每个语言版本上,locale 切换时一并切 theme。

Godot 4.6 在 macOS / Windows / Linux 上有 system font fallback,即操作系统装了哪些字体可以被借用;Web 上**这条机制不可用**,所以 Web 导出必须 bundle 你需要的所有字体文件,否则 Hebrew、emoji、CJK 都会渲染成方块。

### 2.3 输入设备与 `InputEvent` 分支

`Input` 与 `InputMap` 的关系前面 07 篇展开过。本篇关注的是另一面:**判断玩家当前在用哪种设备,然后切换 UI 提示**。Godot 的 `InputEvent` 类树:

- `InputEventKey` — 键盘按键;
- `InputEventMouseButton` / `InputEventMouseMotion` — 鼠标;
- `InputEventJoypadButton` / `InputEventJoypadMotion` — 手柄按钮 / 摇杆;
- `InputEventScreenTouch` / `InputEventScreenDrag` — 触屏。

`_unhandled_input(event)` 接到的 event 可以做 `is_class("InputEventJoypadButton")` 之类的类型检查。维护一个 autoload 单例,把最后一个非空事件类型记下来,UI 提示按它切——这就是"动态按键提示"的标准实现。

要点是 `InputEvent` 这条继承链上每个具体子类都有一个 `device` 字段,表示触发它的设备 index。一个项目同时插两个手柄,`device` 就会是 0 和 1,两个玩家可以分别绑定。本地多人合作类游戏会重度依赖这个字段,单人游戏可以忽略它,但要记住:**"键盘 + 手柄"在 Godot 看来是两个独立 device,不要假设它们的事件统一调度。**

### 2.4 三层契约的串联

本篇的本地化、字体、输入三块,串起来在数据流上是这样的:

1. **启动期**:autoload `Localization` 读 user config → 调 `TranslationServer.set_locale(locale)` + 加载对应 Theme(主字体 + fallback 链);
2. **运行期**:UI 节点访问 `text` 时触发 auto-translate;`tr("KEY")` 显式调用走同一 `TranslationServer`;字体渲染走当前 Theme 的 fallback 链;
3. **输入响应**:`Input._input` 接到 event → autoload `InputDevices` 判断设备类型 → 广播 `device_changed` → UI 切换按键提示图标。

这三条都不依赖游戏主循环,但都在第一次场景加载前就要就绪。本系列的 autoload 顺序约定:`Localization` 最前,`InputDevices` 紧跟,然后是 EventBus、SaveSystem 等业务相关的全局对象。

## 3. 工程实现

### 3.1 CSV 翻译表

`res://localization/strings.csv`,UTF-8 with BOM,逗号分隔,第一列必须是 `keys`:

```text
keys,en,zh_CN,ja,ko
TITLE_MAIN,Cave Runner,洞穴跑者,ケイブランナー,동굴 러너
MENU_START,Start Game,开始游戏,ゲームを始める,게임 시작
MENU_OPTIONS,Options,设置,オプション,설정
MENU_QUIT,Quit,退出,終了,종료
HUD_SCORE,Score: {value},得分:{value},スコア:{value},점수: {value}
HUD_TIME,Time: {minutes}:{seconds},时间:{minutes}:{seconds},時間:{minutes}:{seconds},시간: {minutes}:{seconds}
DEATH_REASON,You died from {cause},你死于 {cause},{cause}で死亡,{cause}로 사망
COMBAT_HIT,{attacker} hit {target} for {damage} damage,{attacker} 对 {target} 造成 {damage} 点伤害,{attacker} が {target} に {damage} ダメージ,{attacker}이(가) {target}에게 {damage} 피해
```

存盘后回到 Godot 编辑器,选中 `strings.csv` → Import 面板会自动识别为 "CSV Translation",再点 Reimport,旁边生成 `strings.en.translation`、`strings.zh_CN.translation` 等。然后 Project Settings → Localization → Translations,把这些 `.translation` 文件全部 Add 进去。注意 Godot 4 没有这个动作就**不会**自动注册——CSV 只是生成资源,资源要被项目识别还要登记。

placeholder 用 `{name}` 格式,在 GDScript 端通过 `String.format()` 替换:

```gdscript
$ScoreLabel.text = tr("HUD_SCORE").format({"value": score})
$TimeLabel.text = tr("HUD_TIME").format({"minutes": m, "seconds": "%02d" % s})
```

### 3.2 Locale 切换与持久化

`res://autoload/localization.gd`,autoload 单例:

```gdscript
extends Node

const SUPPORTED: PackedStringArray = ["en", "zh_CN", "ja", "ko"]
const DEFAULT_LOCALE: String = "en"
const CONFIG_PATH: String = "user://settings.cfg"
const CONFIG_SECTION: String = "i18n"
const CONFIG_KEY: String = "locale"

signal locale_changed(new_locale: String)

func _ready() -> void:
	var saved: String = _load_saved_locale()
	if saved.is_empty():
		saved = _guess_system_locale()
	set_locale(saved)

func set_locale(locale: String) -> void:
	if not SUPPORTED.has(locale):
		locale = DEFAULT_LOCALE
	if TranslationServer.get_locale() == locale:
		return
	TranslationServer.set_locale(locale)
	_apply_locale_theme(locale)
	_save_locale(locale)
	locale_changed.emit(locale)

func current_locale() -> String:
	return TranslationServer.get_locale()

func _guess_system_locale() -> String:
	var os_locale: String = OS.get_locale()           # e.g. "zh_CN.UTF-8"
	var short: String = os_locale.split(".")[0]       # "zh_CN"
	if SUPPORTED.has(short):
		return short
	var lang: String = short.split("_")[0]            # "zh"
	for candidate in SUPPORTED:
		if candidate.begins_with(lang + "_") or candidate == lang:
			return candidate
	return DEFAULT_LOCALE

func _apply_locale_theme(locale: String) -> void:
	# CJK 字形不能靠 fallback 解决,按 locale 切换主 Theme。
	var theme_path: String = "res://ui/themes/theme_%s.tres" % locale
	if not ResourceLoader.exists(theme_path):
		theme_path = "res://ui/themes/theme_en.tres"
	var theme: Theme = load(theme_path) as Theme
	if theme:
		ThemeDB.fallback_font = theme.default_font
		ProjectSettings.set_setting("gui/theme/custom", theme_path)

func _save_locale(locale: String) -> void:
	var cfg := ConfigFile.new()
	cfg.load(CONFIG_PATH)
	cfg.set_value(CONFIG_SECTION, CONFIG_KEY, locale)
	cfg.save(CONFIG_PATH)

func _load_saved_locale() -> String:
	var cfg := ConfigFile.new()
	if cfg.load(CONFIG_PATH) != OK:
		return ""
	return cfg.get_value(CONFIG_SECTION, CONFIG_KEY, "")
```

几个细节:

- `OS.get_locale()` 返回宿主系统的 locale 字串(含编码后缀),要切到只保留 `语言_地区` 部分;
- 找不到精确匹配 locale 时回落到语言代码层,例如 `zh_TW` 找不到就用 `zh_CN` 作 fallback;
- locale 切换时**同时切 Theme**,而不是只调 `TranslationServer.set_locale`。Theme 切换保证 CJK 字形正确;
- 持久化用 `ConfigFile` 写到 `user://settings.cfg`,跨平台一致。

### 3.3 字体 fallback 链组装

`res://ui/themes/theme_zh_CN.tres` 对应一份简中主题。组装一个 `FontVariation` 资源 `res://ui/fonts/main_zh_CN.tres`:

```text
[gd_resource type="FontVariation" load_steps=4 format=3]

[ext_resource type="FontFile" path="res://ui/fonts/SourceHanSansCN-Regular.otf" id="1_main"]
[ext_resource type="FontFile" path="res://ui/fonts/NotoColorEmoji.ttf"        id="2_emoji"]
[ext_resource type="FontFile" path="res://ui/fonts/NotoSansSymbols2.ttf"      id="3_symbol"]

[resource]
base_font = ExtResource("1_main")
fallbacks = Array[Font]([ ExtResource("2_emoji"), ExtResource("3_symbol") ])
spacing_glyph = 0
spacing_space = 0
```

注意三点:

- **emoji 字体放在 fallback 链里**,而不是当主字体;主字体决定 Latin / CJK 字形,emoji 字体补足 U+1F300 区段;
- **不要把简中、日文、繁中堆在同一个 fallback 链里**;按 locale 切主 Theme;
- 字体文件**必须 bundle 进项目**,Web 导出尤其要这么做——system font fallback 在 Web 上不可用。

如果你担心字体文件体积过大(CJK 字体动辄 8-15 MB),用 `godot-font-baker` 之类的工具把字体烘焙成只含 cache 的 `.tres` 资源,license 与体积一举解决。

### 3.4 输入设备活跃状态追踪

`res://autoload/input_devices.gd`,autoload:

```gdscript
extends Node

enum DeviceKind { KEYBOARD_MOUSE, GAMEPAD, TOUCH }

signal device_changed(new_kind: DeviceKind)

var _current: DeviceKind = DeviceKind.KEYBOARD_MOUSE

func current() -> DeviceKind:
	return _current

func _ready() -> void:
	process_mode = Node.PROCESS_MODE_ALWAYS

func _input(event: InputEvent) -> void:
	var next: DeviceKind = _classify(event)
	if next == _current:
		return
	# 滤掉低强度事件:鼠标无位移微抖、手柄死区内摇杆。
	if event is InputEventMouseMotion and (event as InputEventMouseMotion).relative.length() < 1.5:
		return
	if event is InputEventJoypadMotion and absf((event as InputEventJoypadMotion).axis_value) < 0.25:
		return
	_current = next
	device_changed.emit(next)

func _classify(event: InputEvent) -> DeviceKind:
	if event is InputEventKey or event is InputEventMouseButton or event is InputEventMouseMotion:
		return DeviceKind.KEYBOARD_MOUSE
	if event is InputEventJoypadButton or event is InputEventJoypadMotion:
		return DeviceKind.GAMEPAD
	if event is InputEventScreenTouch or event is InputEventScreenDrag:
		return DeviceKind.TOUCH
	return _current
```

UI 节点上挂一个监听器,根据 `device_changed` 切换按键提示图标。这里有几个工程要点:

- **`_input` 而不是 `_unhandled_input`**,因为我们只想观察事件而不消费;
- **加死区过滤**,否则鼠标在手柄玩家的桌上颠一颠,提示就闪回键盘了;
- **`process_mode = PROCESS_MODE_ALWAYS`** 让暂停时也能继续追踪设备切换;
- **触屏作为独立分支**,移动端如果同时插了蓝牙手柄,优先级处理要明确(通常是"最近活跃的赢")。

UI 按键提示用一个 dictionary 映射不同设备:

```gdscript
const HINT_SPRITES: Dictionary = {
	"jump": {
		InputDevices.DeviceKind.KEYBOARD_MOUSE: preload("res://ui/hints/key_space.png"),
		InputDevices.DeviceKind.GAMEPAD:         preload("res://ui/hints/btn_a.png"),
		InputDevices.DeviceKind.TOUCH:           preload("res://ui/hints/touch_tap.png"),
	},
}
```

## 4. 调参和验收

### 4.1 Locale 检测与启动顺序

启动期顺序很重要:

1. autoload `Localization` 在 `_ready()` 里读 user config 拿到 saved locale;
2. 若没有,用 `OS.get_locale()` 猜系统语言;
3. 调 `TranslationServer.set_locale()`,**在场景初始化之前**;
4. 加载主菜单场景,UI 节点首次渲染时已经是正确 locale。

如果你在主菜单 `_ready()` 才设 locale,会出现 UI 闪一帧英文再切到目标语言。autoload 顺序在 Project Settings → Autoload 里调整,`Localization` 应该放最前面。

### 4.2 字体 fallback 链顺序

经验值的顺序:**主字体 → 数学符号字体 → emoji 字体 → 装饰符号字体**。emoji 放太靠前会让某些"看起来像字符的 emoji"被优先匹配,把正文渲染得很怪;放太靠后会让 emoji 序列被装饰字体抢走。

测试 fallback 链最快的方法是写一个文本 `"Hello 你好 こんにちは emoji 😀 math ∑ symbol ♻"`,放在 Label 上看每个字符是否正确显示。任何一个出现 tofu 都说明 fallback 链漏了一段。

### 4.3 输入设备死区

- 手柄摇杆死区:0.2-0.25,小于这个值不响应。Godot `InputMap` 的 deadzone 在 Action 配置里设;
- 手柄按钮的"瞬间按下识别"不需要死区;
- 鼠标位移阈值 1-2 像素,小于这个值视为漂移;
- 触屏 tap 与 drag 分界:5-10 像素位移开始算 drag。

这些参数影响"切换提示"的灵敏度,而不影响实际的角色控制。角色控制走 `InputMap`,本系列 07 篇细说。

### 4.4 可访问性的最低基线

独立游戏可访问性不需要全套 AAA 工作量,但有几个低成本高收益的点:

- **字号缩放**:设置菜单提供 80% / 100% / 130% 三档,通过 `Theme.default_font_size` 全局缩放;
- **色盲友好**:UI 关键状态(血量、敌我区分)**不依赖单一色相**。低血红条配震动图标,敌人红光配尖刺轮廓,而不是纯红 vs 纯蓝;
- **字幕**:对话与剧情语音必须有字幕开关,默认开;字幕字号也走全局缩放;
- **键位重映射**:`InputMap.action_erase_event` + `action_add_event` 在 GDScript 里完全可写,设置菜单允许玩家自定义任何 action 的绑定。

这些都不要算"以后再做",从第一天就把"色盲友好"作为 sprite 设计的硬约束、把字号缩放作为 UI 系统的接口设计,比发布后 retrofit 容易十倍。

更细一点的可访问性清单:**字幕背景半透明色块**(避免白底白字),**伤害闪烁可关闭**(光敏感玩家),**镜头震动幅度可调**(0% / 50% / 100% 三档),**自动重试关卡**选项(对手残友好)。这些选项不一定都做满,但每一项的实现都不超过 50 行 GDScript,放到 dev backlog 里慢慢补。第一次提交版本时,**至少把字号、震动开关、字幕开关这三项做掉**——大多数玩家不会用,但需要的玩家会因为有这个选项而买你的游戏。

### 4.5 验收清单

- 启动游戏 4 种 locale 各跑一遍,UI 上没有 tofu;
- 系统 locale 设为不支持的语言(如 fr),游戏回落到 en 而不是崩;
- locale 切换后所有 UI 立即重绘,不需要重启;
- 键盘玩、手柄玩、(移动)触屏玩,UI 提示图标各自对应;
- 同时插键盘和手柄,最后一次输入的设备决定 UI 提示。

### 4.6 文本扩展实验

落地阶段做一次"全 locale 文本扩展模拟"很省后期返工。具体动作:把 CSV 里某一列(比如德语 `de`)填成原文 + 30% 长度的占位符,例如英文 `Start Game` 对应德语填 `Start Game iiiiiiiii`。然后切到 de 跑游戏,看 UI 哪里溢出。这种"伪本地化"在国际化工程里叫 pseudo-localization,是发现布局问题最有效的方法,比真正等翻译稿来再修早三个月。

工具化做法:写一个一次性 GDScript `@tool` 脚本,扫描 CSV 第一列把 en 列的每个 value 加上前缀 `[`、后缀 `]` 和 30% padding 字符,保存为 `pseudo` 列再 reimport。dev build 默认开启 `pseudo` locale,QA 玩这个版本能立刻发现"按钮被裁掉一半"、"字数太长换行错乱"等问题。

## 5. 踩坑

### 5.1 字符串拼接绕过 `tr()`

`"Welcome back, " + player.name` 这种字符串拼接在 GDScript 里仍然合法,但**不会触发 auto-translate**。即便 player.name 本身是 user 数据不需要翻译,前缀部分也会停留在英文。

正确做法:

```gdscript
$Label.text = tr("WELCOME_BACK").format({"name": player.name})
```

CSV 里:`WELCOME_BACK,Welcome back, {name},欢迎回来,{name},お帰りなさい、{name}`。

这条规则机械:**任何展示给玩家的字符串,如果是从 GDScript 写出来的常量,必须经过 `tr()`。** code review 时一旦看到 `"..."` 直接赋给 `.text`,要警觉。

### 5.2 CJK 字形混用("和"字事件)

把简中和日文堆在 fallback 链里,简中"和"字在日文优先的字体下会被渲染成日文字形,撇捺角度细微不同,中文母语者一眼看出"这看起来像日文"。看似无害但显著降低游戏的"成品感"。

唯一可靠解法:**locale 切换时同时切 Theme,Theme 里的主字体按语言分别定制。** 不要试图用一个万能 fallback 链解决所有 CJK。

### 5.3 Web 上 system font fallback 不可用

在桌面端,Godot 可以让操作系统提供 emoji / CJK 兜底,这让项目体积可以做得很小;但 Web 端浏览器沙盒里没有 system font 访问权限,任何依赖 system fallback 的字体在 Web 导出后渲染成方块。

补救:**Web 项目必须把所有需要的字体显式 bundle**。这与第 26 篇讨论的 Web 包体大小预算冲突,所以一种实践是 Web 版本只支持英文 + 一种亚洲语言,把字体文件控制在 5 MB 以下;桌面 / 移动版本支持全部语言。

### 5.4 `OS.get_locale()` 不可信

`OS.get_locale()` 在不同操作系统返回格式略有差异——Windows 给 `zh_CN`,macOS 给 `zh-Hans-CN` 或 `zh_CN`,Linux 给 `zh_CN.UTF-8`。直接拿来做 `SUPPORTED.has()` 经常不命中。处理:先 split 掉编码后缀,再 split 掉子标签,最后回落到语言代码层级。3.2 节代码里的 `_guess_system_locale()` 就是这个 pattern。

### 5.5 手柄按键标签与实际按键不一致

Xbox 手柄、PlayStation 手柄、Nintendo Switch 手柄的物理按键标签**完全不同**:Xbox 的 A 在 PS 上是 ×、在 Switch 上是 B(位置交换)。Godot 把这些都归一化成 `JOY_BUTTON_A`、`JOY_BUTTON_B` 等抽象标签,但**这套抽象本身偏 Xbox**。

UI 提示需要按手柄品牌再分一层。检测手柄品牌:

```gdscript
var joypad_name: String = Input.get_joy_name(0)
if joypad_name.contains("PlayStation") or joypad_name.contains("PS"):
	# 显示 X / O / △ / □ 提示
elif joypad_name.contains("Nintendo") or joypad_name.contains("Switch"):
	# 显示 B / A / Y / X 提示(注意位置)
else:
	# Xbox 默认
```

这是工作量大但不复杂的事,通常用 Joypad Glyph 之类的资源包一次性把三套图标都准备好。

### 5.6 触屏 + 鼠标在某些 Android 设备上同时触发

某些 Android 平板把触屏事件**同时**作为 `InputEventScreenTouch` 和 `InputEventMouseButton` 派发(emulate mouse from touch),如果你按"最后一个 event 决定设备"逻辑,会看到提示在 KEYBOARD_MOUSE 和 TOUCH 间反复跳。处理:

- 项目设置 `input_devices/pointing/emulate_mouse_from_touch = false` 强制只发触屏事件;
- 或者代码里加上对 `InputEventMouseButton.device == -1` 的特判(emulated mouse 通常 device 为 -1)。

### 5.7 Auto-translate 在 RichTextLabel 上的边界

`RichTextLabel` 的 BBCode 内嵌字符串 **不会**走 auto-translate 整体,因为 Godot 不知道哪一段是要翻译的 key、哪一段是 BBCode 标签。处理方式:把整段文本作为一个 key 存到 CSV,翻译时也把 BBCode 翻进去:

```text
keys,en,zh_CN
TUTORIAL_JUMP,Press [color=yellow]Space[/color] to jump.,按 [color=yellow]空格[/color] 跳跃。
```

或者更细粒度,用 BBCode 模板拼接:`tr("TUTORIAL_JUMP_TEMPLATE").format({"key": tr("KEY_SPACE")})`。

### 5.8 文字宽度差异打爆 UI 布局

德语单词平均比英文长 30%,日韩字符块平均比英文宽,阿拉伯文 RTL 排版方向相反。一个按钮在英文下正好填满,德语下会溢出,日文下会显得空。

处理:**UI 容器用 `Container` 而不是固定宽度**(`HBoxContainer`、`MarginContainer`、`AspectRatioContainer`),让节点根据内容自适应。绝对定位的 Label 在多语言项目里几乎一定会翻车,只在保证"该位置永远是数字 / 图标 / 单字符"时才用。

### 5.9 翻译 key 的命名一致性

CSV 的第一列 key 命名混乱是后期翻译协作的灾难。常见 anti-pattern:`button_start`、`MenuStart`、`MAIN_START_BTN`、`UI_HOME_START` 五种风格出现在同一项目里,翻译同事 grep 不到、找不到对应位置。约定一套命名规范并写进项目 README:

- **全大写下划线**:`MENU_START`、`HUD_SCORE`、`DEATH_REASON`;
- **第一段表分类**:`MENU_*`、`HUD_*`、`DIALOGUE_*`、`ITEM_*`、`SKILL_*`、`ERROR_*`;
- **placeholder 用 `{snake_case}`**:`{player_name}`、`{damage_value}`,绝不要 `{1}`、`{a}` 这种位置式;
- **专有名词不走 tr**:游戏名、品牌、人名地名(除非这类本身就要本地化为不同书写体系),直接硬编码。

key 拥有这种规范后,翻译漏洞可以靠脚本检测:扫描所有 `.gd` 文件里的 `tr("...")` 字符串,与 CSV 第一列做差集,缺失部分立刻能列出来。本系列 30 篇会把这类检查纳入"发售前清单"。

### 5.10 Translation 资源与 PackedScene 的资源切换

一些场景下你想做的不只是文本翻译,而是**整张图、整段音频也按 locale 切换**(例如教程截图、配音)。Godot 提供 Project Settings → Localization → Remaps 机制:绑定一个原始资源路径,为每个 locale 指定替换路径。运行时 `load("res://tutorial/page1.png")` 会自动按当前 locale 替换成 `res://tutorial/page1.zh_CN.png`。

陷阱在于:**remap 在编辑器内可能不能即时切换**,locale 设置后第一次 load 走 remap、但 ResourceLoader 缓存第二次 load 就拿到缓存版本。可靠做法是 locale 切换时显式 `ResourceLoader.has_cached(...)` 检查并清掉关键资源缓存,或者干脆切 locale 时 reload 受影响的场景。这条坑写得不够好,但在引擎层修复前只能这么绕。
