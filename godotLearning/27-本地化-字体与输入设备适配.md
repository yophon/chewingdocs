# 本地化、字体与输入设备适配

发售前最容易返工的三件事:硬编码文本、字体缺字、只支持键盘。它们看起来不相关,但本质都是“不该假设玩家和你一样”。

> 一句话先记住:**文本用 key,字体有 fallback,输入显示跟着设备变。**

---

## 一、不要硬编码显示文本

反例:

```gdscript
start_button.text = "开始游戏"
```

推荐:

```gdscript
start_button.text = tr("START_GAME")
```

CSV:

```csv
keys,en,zh_CN,ja
START_GAME,Start Game,开始游戏,ゲーム開始
OPTIONS,Options,设置,設定
QUIT,Quit,退出,終了
```

把 CSV 导入后,Godot 会生成 Translation 资源。

---

## 二、切换语言

```gdscript
# res://systems/localization.gd
extends Node

signal locale_changed(locale: String)

func set_locale(locale: String) -> void:
    TranslationServer.set_locale(locale)
    locale_changed.emit(locale)
```

UI 收到后刷新文本:

```gdscript
func _ready() -> void:
    Localization.locale_changed.connect(_refresh_text)
    _refresh_text(TranslationServer.get_locale())

func _refresh_text(_locale: String) -> void:
    %StartButton.text = tr("START_GAME")
    %OptionsButton.text = tr("OPTIONS")
```

自动翻译能帮一些 Control 文本,但动态拼接文本最好手动刷新。

---

## 三、字体 fallback

英文字体通常不含中文、日文、韩文。结果就是方块。

字体策略:

```text
主字体: 游戏风格字体
fallback: Noto Sans CJK / emoji / 符号字体
```

不同语言可能需要不同主字体。简中、繁中、日文不要全靠 fallback 混过去,字形细节可能不对。

Web 导出尤其要把字体打包进项目。不要指望浏览器能用玩家系统字体。

---

## 四、文本长度会变

英文、德文、日文、中文长度差很多。UI 不要为中文长度写死。

做法:

```text
按钮设置 custom_minimum_size
文本放 Container
长描述用 RichTextLabel 或 Label autowrap
关键 UI 留 30%-50% 冗余宽度
```

不要把按钮做成刚好装下“开始游戏”的宽度。

---

## 五、输入动作不等于显示提示

逻辑用 InputMap:

```text
jump
attack
dash
pause
```

UI 提示要根据设备显示:

```text
键盘: Space
Xbox: A
PlayStation: Cross
Switch: B/A 位置还要小心
```

做一个设备追踪:

```gdscript
# res://systems/input_device_tracker.gd
extends Node

signal device_changed(kind: StringName)

var current_kind: StringName = &"keyboard"

func _input(event: InputEvent) -> void:
    var next := current_kind
    if event is InputEventJoypadButton or event is InputEventJoypadMotion:
        next = &"gamepad"
    elif event is InputEventKey or event is InputEventMouse:
        next = &"keyboard"

    if next != current_kind:
        current_kind = next
        device_changed.emit(current_kind)
```

提示 UI 监听它,切换图标或文字。

## 六、按键重映射

至少提供:

```text
移动
跳跃
攻击
冲刺
暂停
确认
返回
```

保存到配置文件,不要写死在项目设置里就结束。第 14 篇的 `ConfigFile` 可以复用。

---

## 七、基础可访问性

发售前至少检查:

```text
字幕开关
音量分 bus 调节
屏幕震动强度
闪光强度
字体大小或 UI 缩放
色盲友好颜色
手柄和键盘都能完整操作菜单
```

这些不是“高级功能”。很多玩家没有它们就玩不了。

---

## 验收

- UI 文本使用 key,不是硬编码。
- 至少有一份 CSV 翻译表。
- 字体 fallback 能显示 CJK 和常用符号。
- UI 能承受更长的翻译文本。
- 输入提示会跟键盘/手柄切换。
- 设置里能调音量、震动、语言。

---

## 常见坑

**坑 1:代码里拼中文句子。**

不同语言语序不同。整句做 key,不要把词硬拼起来。

**坑 2:字体只在自己电脑正常。**

导出版和 Web 要带字体文件。

**坑 3:手柄能玩关卡但不能操作菜单。**

菜单 focus 和确认/返回动作也要测。

**坑 4:只翻译 UI,不翻译道具和教程。**

所有玩家可见文本都要进本地化表。
