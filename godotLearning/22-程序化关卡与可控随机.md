# 程序化关卡与可控随机

程序化生成的重点不是“随机”,而是“能复现”。玩家说某个种子坏了,你必须能在本地生成同一张图。

> 一句话先记住:**生成器只吃 seed 和参数,输出确定结果。**

---

## 一、别用全局 rand 做关卡

不要在生成器里直接用:

```gdscript
randi()
randf()
```

这些是全局随机流。UI、特效、AI 只要也调用了随机,关卡结果就会变。

用独立 RNG:

```gdscript
var rng := RandomNumberGenerator.new()
rng.seed = level_seed
var x := rng.randi_range(0, 100)
```

地形、敌人、宝箱最好各用自己的子种子。

---

## 二、参数进 Resource

```gdscript
# res://pcg/generation_config.gd
class_name GenerationConfig
extends Resource

@export var width: int = 80
@export var height: int = 45
@export var fill_chance: float = 0.45
@export var smooth_steps: int = 4
@export var min_room_size: int = 5
@export var max_room_size: int = 12
@export var room_count: int = 10
```

这样调关卡不是改代码,而是改 `.tres`。

## 三、生成器返回数据,不要直接改节点

先生成二维数组:

```gdscript
# 0 = 空, 1 = 墙
func generate(seed: int, config: GenerationConfig) -> Array[PackedInt32Array]:
    var grid: Array[PackedInt32Array] = []
    var rng := RandomNumberGenerator.new()
    rng.seed = seed

    for y in config.height:
        var row := PackedInt32Array()
        row.resize(config.width)
        for x in config.width:
            row[x] = 1 if rng.randf() < config.fill_chance else 0
        grid.append(row)

    for i in config.smooth_steps:
        grid = _smooth(grid)

    return grid
```

再由渲染层写进 `TileMapLayer`:

```gdscript
func paint_grid(layer: TileMapLayer, grid: Array[PackedInt32Array]) -> void:
    layer.clear()
    for y in grid.size():
        for x in grid[y].size():
            if grid[y][x] == 1:
                layer.set_cell(Vector2i(x, y), 0, Vector2i(0, 0))
```

生成逻辑不要依赖场景树,这样以后才能放到后台线程。

## 四、元胞自动机洞穴

平滑规则:

```gdscript
func _smooth(grid: Array[PackedInt32Array]) -> Array[PackedInt32Array]:
    var next: Array[PackedInt32Array] = []

    for y in grid.size():
        var row := PackedInt32Array()
        row.resize(grid[y].size())

        for x in grid[y].size():
            var walls := _count_wall_neighbors(grid, x, y)
            row[x] = 1 if walls >= 5 else 0

        next.append(row)

    return next

func _count_wall_neighbors(grid: Array[PackedInt32Array], x: int, y: int) -> int:
    var count := 0
    for oy in range(-1, 2):
        for ox in range(-1, 2):
            if ox == 0 and oy == 0:
                continue
            var nx := x + ox
            var ny := y + oy
            if ny < 0 or ny >= grid.size() or nx < 0 or nx >= grid[0].size():
                count += 1
            elif grid[ny][nx] == 1:
                count += 1
    return count
```

这能生成自然洞穴,但可能出现不连通区域。下一步要做连通性检查。

## 五、房间走廊法

更适合地牢:

```text
随机放房间
丢掉重叠房间
按中心点排序
相邻房间挖走廊
放出生点、出口、宝箱、敌人
```

这种方法更可控,适合设计战斗节奏。

最小房间数据:

```gdscript
class_name Room
extends RefCounted

var rect: Rect2i

func _init(p_rect: Rect2i) -> void:
    rect = p_rect

func center() -> Vector2i:
    return rect.position + rect.size / 2
```

## 六、FastNoiseLite 适合连续地形

噪声适合:

```text
高度图
温度图
湿度图
矿物分布
草地密度
```

```gdscript
var noise := FastNoiseLite.new()
noise.seed = seed
noise.frequency = 0.04

var value := noise.get_noise_2d(x, y)
```

RNG 是“下一个随机数”。噪声是“某个坐标的稳定值”。大地图更适合噪声。

## 七、保存种子

存档里保存:

```json
{
  "level_seed": 123456,
  "generator_version": 2
}
```

如果算法改了,同一个 seed 可能生成不同地图。所以要带 `generator_version`。版本变了,要么迁移,要么明确旧档继续用旧生成器。

---

## 验收

- 同一个 seed 连续生成两次,结果完全一样。
- 改 UI、音效、AI 随机不会影响地图。
- 生成器返回纯数据,不直接 `add_child`。
- 参数在 `GenerationConfig.tres` 里。
- 存档保存 seed 和 generator version。

---

## 常见坑

**坑 1:生成过程读场景树。**

这样结果会被外部状态影响。把需要的数据作为参数传进去。

**坑 2:不检查连通性。**

洞穴可能生成孤岛。至少保证出生点能到出口。

**坑 3:只保存生成结果,不保存种子。**

Bug 无法复现。种子和版本都要存。

**坑 4:把 PCG 当省设计。**

生成器只负责产出候选,好不好玩仍然要靠规则和调参。
