Spanner / Calvin / NewSQL вҖӢ

20-23 иө°иҝҮдәҶдёҖйҒҚеҲҶеёғејҸдәӢеҠЎзҡ„иҝӣеҢ–:2PC жҳҜй”Ғз©ҝдёҖеҲҮзҡ„ејәдёҖиҮҙ,Saga/TCC жҳҜж”ҫејғ I жҚўжҖ§иғҪ,Percolator жҳҜ KV дёҠзіҠдёҖеұӮй”ҒеҲ—гҖӮдҪҶиҝҷдәӣж–№жЎҲйғҪжңүдёҖдёӘе…ұеҗҢзҹӯжқҝвҖ”вҖ”е®ғд»¬еҒҡдёҚдәҶ"е…ЁзҗғеҗҢжӯҘзҡ„ ACID + й«ҳеҗһеҗҗ + йҖҸжҳҺ SQL"гҖӮPercolator TSO жҳҜеҚ•зӮ№,и·ЁеӨ§йҷҶ 100ms е»¶иҝҹ;2PC еңЁйҮ‘иһҚеңәжҷҜиғҪз”ЁдҪҶ TPS ж’‘дёҚдҪҸдә’иҒ”зҪ‘йҮҸзә§гҖӮGoogle еңЁ 2012 е№ҙз”ЁдёҖзҜҮ Spanner и®әж–Үз»ҷеҮәдәҶз»ҲжһҒзӯ”жЎҲвҖ”вҖ”з”ЁеҺҹеӯҗй’ҹ + GPS иЈ…еҮә"йқ и°ұзҡ„зү©зҗҶж—¶й—ҙ",и®©еҲҶеёғејҸж•°жҚ®еә“иғҪеғҸеҚ•жңәж•°жҚ®еә“дёҖж ·и®І"ж—¶й—ҙеҒҸеәҸ"гҖӮеҗҢе№ҙ Yale зҡ„ Thomson & Abadi иө°дәҶдёҖжқЎе®Ңе…ЁзӣёеҸҚзҡ„и·ҜвҖ”вҖ”Calvin:е…Ҳе…ЁеұҖжҺ’еәҸжүҖжңүдәӢеҠЎ,еҶҚе№¶иЎҢзЎ®е®ҡжҖ§жү§иЎҢгҖӮиҝҷдёӨзҜҮи®әж–Ү + Percolator,е®ҡд№үдәҶд»ҠеӨ©жүҖжңү NewSQL зҡ„ж јеұҖгҖӮ

дёҖеҸҘиҜқе…Ҳи®°дҪҸ:Spanner жӢҝ TrueTime жҠҠ"еҲҶеёғејҸзі»з»ҹйҮҢжІЎжңүзңҹж—¶й—ҙ"иҝҷдёӘиҖҒй—®йўҳзЎ¬жҖјеӣһеҺ»вҖ”вҖ”иҠұй’ұиЈ…еҺҹеӯҗй’ҹ + GPS,и®©е…Ёзҗғж•°жҚ®дёӯеҝғйғҪжңү Оө вүҲ 7ms зҡ„зү©зҗҶж—¶й—ҙдёҚзЎ®е®ҡеҢәй—ҙ,з„¶еҗҺз”Ё commit wait зӯүиҝҮиҝҷдёӘеҢәй—ҙ,еҒҡеҮәдәҶ External Consistency(еҸҜзәҝжҖ§еҢ–зҡ„дәӢеҠЎзүҲ)гҖӮCalvin иө°еҸҰдёҖжһҒз«ҜвҖ”вҖ”еҲ«з”Ёй”ҒдәҶ,жүҖжңүдәӢеҠЎе…Ҳ deterministic жҺ’дёӘе…ЁеәҸ,жүҖжңүеүҜжң¬жҢүиҝҷдёӘеәҸе№¶иЎҢи·‘,з»“жһңеӨ©з„¶дёҖиҮҙгҖӮNewSQL дёүеӨ§жөҒжҙҫ:Spanner жҙҫ(CockroachDB / YugabyteDB,жЁЎжӢҹ TrueTime з”Ё HLC)гҖҒTSO жҙҫ(TiDB,з»§жүҝ Percolator)гҖҒCalvin жҙҫ(FaunaDB)гҖӮ

2.1 ж ёеҝғ API вҖӢ

TrueTime жҡҙйңІз»ҷдёҠеұӮзҡ„ API жһҒз®Җ:

struct TTinterval {

    Timestamp latest;    // зңҹе®һж—¶й—ҙдёҚжҷҡдәҺиҝҷдёӘ

TTinterval TT.now();      // еҪ“еүҚж—¶й—ҙзҡ„дёҚзЎ®е®ҡеҢәй—ҙ

2.3 Commit Wait:дёәд»Җд№ҲиҰҒзӯүиҝҷдёӘеҢәй—ҙиҝҮеҺ» вҖӢ

дёәд»Җд№Ҳеҝ…йЎ»зӯү?

External Consistency иҰҒжұӮ:еҰӮжһңдәӢеҠЎ T1 еңЁ T2 ејҖе§Ӣд№ӢеүҚе®ҢжҲҗ(зү©зҗҶдёҠ),йӮЈд№Ҳ T2 еҝ…йЎ»иғҪзңӢеҲ° T1гҖӮ

  • дёҚзӯүе°ұ ack:T1 commit_ts = s,дҪҶзңҹе®һж—¶й—ҙеҸҜиғҪиҝҳжІЎеҲ° s
  • T2 еңЁ s д№ӢеүҚејҖе§Ӣ(зү©зҗҶдёҠ)дҪҶ timestamp = s' > s
  • T2 еә”иҜҘзңӢдёҚеҲ° T1 жүҚеҜ№,дҪҶеӣ дёә s' > s,T2 дјҡзңӢеҲ° T1 вҶ’ иҝқеҸҚ External Consistency

4.1 е®ҡд№ү вҖӢ

External Consistency(еӨ–йғЁдёҖиҮҙжҖ§) = Strict Serializability:

дәӢеҠЎ T1, T2 ж»Ўи¶і:
   еҰӮжһң T1 е®ҢжҲҗ(client ж”¶еҲ° ack)еңЁ T2 ејҖе§Ӣ(client еҸ‘иҜ·жұӮ)д№ӢеүҚ,
   йӮЈд№Ҳд»»дҪ•зңӢеҲ°зҡ„"дәӢеҠЎдёІиЎҢеәҸ"еҝ…йЎ»жңү T1 < T2

дёӨдёӘз»„еҗҲ:

  • Serializability:жңүдёҖдёӘдёІиЎҢеәҸиғҪи§ЈйҮҠи§ӮеҜҹеҲ°зҡ„з»“жһң
  • Linearizability:еҚ• key зҡ„е®һж—¶йЎәеәҸиў«е°ҠйҮҚ
  • Strict Serializability = Serializability + Linearizability(дәӢеҠЎзә§)

4.2 дёәд»Җд№Ҳ Snapshot Isolation дёҚеӨҹ вҖӢ

T1: write x = 1, commit вҶҗ physical time 100ms
T2: read x         вҶҗ physical time 200ms

SI дёҚдҝқиҜҒ T2 дёҖе®ҡиҜ»еҲ° x=1!
еӣ дёә T2 зҡ„ start_ts еҸҜиғҪ вүӨ T1 зҡ„ commit_ts(ж—¶й’ҹдёҚеҮҶ)

Spanner з”Ё TrueTime + Commit Wait дҝқиҜҒ:

  • T1 commit е®ҢжҲҗж—¶,зңҹе®һж—¶й—ҙе·Із»Ҹи¶…иҝҮ commit_ts
  • T2 start ж—¶еҸ–зҡ„ ts > зңҹе®һж—¶й—ҙ > T1.commit_ts
  • жүҖд»Ҙ T2 дёҖе®ҡзңӢеҲ° T1

External Consistency жҳҜеҲҶеёғејҸдәӢеҠЎзҡ„еңЈжқҜгҖӮ99% дёҡеҠЎз”ЁдёҚеҲ°(SI еӨҹдәҶ),дҪҶйҮ‘иһҚзӣ‘з®ЎгҖҒи·Ёең°еҹҹе®Ўи®ЎгҖҒи·ЁжңҚеҠЎдёҘж јеӣ жһңиҝҷдәӣеңәжҷҜйҮҢ,е®ғжҳҜеҲҡйңҖгҖӮ

def spanner_commit(txn):
    # 1. еңЁжҜҸдёӘ participant Tablet дёҠиҺ·еҸ–еҶҷй”Ғ
    for tablet in txn.tablets:
        tablet.acquire_locks(txn.writes)
    
    # 2. Prepare йҳ¶ж®ө(е№¶иЎҢ)
    prepare_timestamps = []
    for tablet in txn.tablets:
        # жҜҸдёӘ Tablet йҖү prepare_ts вүҘ TT.now().latest
        prepare_ts = max(tablet.last_assigned_ts + 1, TT.now().latest)
        tablet.paxos_replicate(prepare_log, prepare_ts)
        prepare_timestamps.append(prepare_ts)
    
    # 3. Coordinator йҖү commit_ts
    commit_ts = max(
        max(prepare_timestamps),    # д»»дҪ• prepare_ts
        TT.now().latest,             # еҪ“еүҚ TrueTime еҢәй—ҙеҸіз«Ҝ
    )
    
    # 4. Commit Wait!е…ій”®жӯҘйӘӨ
    while not TT.after(commit_ts):
        sleep_short()                # йҖҡеёё sleep 4-7ms
    
    # 5. йҖҡзҹҘжүҖжңү Tablet commit
    for tablet in txn.tablets:
        tablet.paxos_replicate(commit_log, commit_ts)
        tablet.release_locks(txn.writes)
    
    # 6. зҺ°еңЁ ack client
    return OK
