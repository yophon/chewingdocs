# 移动端通用知识系列写作计划

这个系列不讲某一个框架怎么写页面,也不替代 Flutter、Android 原生、iOS 原生系列。它只讲所有移动端项目都会遇到的共性问题:包、签名、权限、推送、深链、存储、网络、发布、监控、合规和事故处理。

一句话:**移动端通用知识 = 不管你用 Flutter、React Native、Kotlin 还是 Swift,上线后都绕不开的那套平台规则和工程常识。**

---

## 定位

已有系列边界:

| 系列 | 讲什么 |
| --- | --- |
| Flutter | 跨端 UI、状态、插件、Flutter 工程 |
| Android 原生开发 | Kotlin、Compose、Jetpack、Android 工程 |
| iOS 原生开发 | Swift、SwiftUI、Apple 平台框架 |
| Android 平台基础 | Android 系统组件和平台机制 |
| 移动端通用知识 | 横跨 Android / iOS / Flutter / RN 的发布、签名、系统能力和生产治理 |

这个系列优先服务两类人:

1. 会写 App,但第一次负责上线、渠道、签名、推送、CI/CD。
2. 已经有线上 App,需要补齐发布资产、事故处理、合规和长期维护。

## 写作风格

1. 不写平台宣传,只写工程判断。
2. 每篇都要有"什么时候会出事故"。
3. 涉及政策、商店规则、target SDK、Xcode 要标注复查日期。
4. 涉及密钥、证书、签名材料时,默认按生产事故级别处理。
5. 不把 Android / iOS API 细节展开成平台教程,需要深入时链接到对应系列。
