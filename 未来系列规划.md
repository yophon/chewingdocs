# 未来系列规划

当前目录已经不是"待补基础清单",而是一张成型的工程知识地图。后续重点不再是继续横向开很多新系列,而是做三件事:维护准确性、把阅读路径产品化、用真实项目消化已有知识。

---

## 当前状态

已成型的主系列覆盖这些层:

| 层级 | 系列 |
| --- | --- |
| 工程基础 | 云服务与互联网基础常识、网络、操作系统、算法、程序员的数学、Git、终端工程 |
| 业务系统 | 后端学习、系统设计、分布式系统、DevOps / SRE、数据工程、音视频工程、安全 |
| AI 工程 | AI 学习、AI Infra、Claude Code、数据工程 |
| 客户端 | 前端学习、移动端通用知识、Flutter、Android 原生开发、Android 平台基础、iOS 原生开发 |
| 语言与代码设计 | Go、Rust、设计模式、解释器 |
| 兴趣与产品方向 | Godot 游戏开发、杂项 |

结论:

1. 基础知识图已经补齐,当前没有必须新增的主系列。
2. 下一阶段最值得投入的是"目录与导读质量"和"项目化实战",不是继续扩数量。
3. 顶层入口已补充 [学习路径](学习路径.md) 和 [项目实战](项目实战.md),后续新增内容优先挂到这两页。

---

## 维护优先级

### 1. 统一入口与篇数口径

- README、首页、系列页都以正式章节为准,不把 `00-写作计划.md` 算进篇数。
- 系列表由 `scripts/sync-series.mjs` 生成,新增系列后只改 `data/series.mjs` 和章节文件。
- 各系列总览里的"完整目录 / 学习路径 / 预计时间"要和真实章节同步。

### 2. 做角色化学习路径

已新增 [学习路径](学习路径.md),按目标组织阅读路线:

| 目标 | 推荐路线 |
| --- | --- |
| 后端工程师进阶 | 后端学习 -> 系统设计 -> 分布式系统 -> DevOps / SRE -> 数据工程 |
| AI 应用工程师 | AI 学习 -> Claude Code -> 数据工程 -> AI Infra |
| AI Infra / 推理工程师 | AI 学习主线 -> AI Infra -> 操作系统 -> 网络 -> DevOps / SRE |
| 全栈工程师 | 前端学习 -> 后端学习 -> 云服务与互联网基础常识 -> 系统设计 |
| 移动端工程师 | 移动端通用知识 -> Flutter -> Android 原生开发 / iOS 原生开发 -> Android 平台基础 |
| 工程内功补强 | 操作系统 -> 网络 -> 算法 -> 程序员的数学 -> Git -> 终端工程 |

## 项目化方向

下一步最有价值的不是再写一个孤立系列,而是做 2-3 个贯穿项目。

已新增 [项目实战](项目实战.md),先把项目范围、里程碑、关联系列和优先级固定下来。后续如果展开,仍然优先作为项目专题维护,不急着注册成新系列。

### 项目 A:AI 知识库 / Agent 平台

目标:把 AI、数据工程、后端、AI Infra、DevOps 串起来。

包含:

- 文档摄取、清洗、切分、Embedding、向量检索
- RAG 问答、工具调用、会话记忆、评测集
- 后端 API、权限、限流、日志、监控
- vLLM / SGLang 推理服务与成本估算
- CI/CD、部署、告警和事故复盘

### 项目 B:高并发业务系统

目标:把后端、系统设计、分布式、缓存、消息队列和 SRE 串起来。

候选题目:

- 秒杀 / 库存系统
- 短链与访问统计系统
- Feed / 通知系统
- 订单与支付状态机

### 项目 C:跨端产品闭环

目标:把前端、Flutter、Android / iOS 原生能力、后端和发布流程串起来。

包含:

- Web 管理端
- 移动端 App
- 登录、推送、文件上传、离线缓存
- 灰度发布、崩溃收集、性能指标
