# 音视频质量评估与排障

音视频线上问题最怕一句"用户说卡"。你必须把卡拆成可观测指标。

> 一句话先记住:**质量问题要按链路定位:采集、编码、上行、服务端、分发、播放、设备**。没有指标,只能猜。

---

## 一、核心指标

| 指标 | 说明 |
| --- | --- |
| 首帧时间 | 点击到出画面 |
| 卡顿次数 | 播放中断次数 |
| 卡顿时长 | 中断总时长 |
| RTT | 往返延迟 |
| packet loss | 丢包 |
| jitter | 抖动 |
| bitrate | 实际码率 |
| fps | 实际帧率 |
| dropped frames | 丢帧 |
| audio level | 音量 |

会议和播放器都要打点。

---

## 二、WebRTC 排障顺序

```text
1. 信令通了吗?
2. SDP setLocal / setRemote 成功了吗?
3. ICE connected 了吗?
4. candidate 类型是什么? host / srflx / relay?
5. getStats 有 bytesSent / bytesReceived 吗?
6. 有包但没画面? 查解码 / track / autoplay
7. 有画面但卡? 查 RTT / loss / jitter / bitrate
```

不要一上来怀疑编码器。

---

## 三、播放器排障顺序

```text
1. manifest 能下载吗?
2. segment 能下载吗?
3. MIME 对吗?
4. CORS 对吗?
5. 编码支持吗?
6. 时间戳正常吗?
7. 缓冲增长吗?
8. 解码掉帧吗?
```

浏览器 DevTools Network 面板是第一现场。

---

## 四、FFmpeg 排障

```bash
ffprobe -hide_banner input.mp4
ffprobe -show_streams -show_format input.mp4
```

看:

- codec
- pix_fmt
- time_base
- duration
- bit_rate
- frame rate
- start_time

转码失败一定保存完整 stderr。

---

## 五、常见故障表

| 现象 | 常见原因 |
| --- | --- |
| 本地通线上不通 | NAT / TURN / 防火墙 |
| 有声音没画面 | 视频编码不支持 / track 没绑定 |
| 画面糊 | 码率低 / 分辨率降级 / 丢包 |
| 声音断 | 丢包 / jitter / 音频设备问题 |
| HLS 拖动卡 | 关键帧 / 分片 / 时间戳 |
| 首帧慢 | 清单慢 / 分片慢 / 缓冲策略 |

---

## 六、踩坑提醒

1. **只收业务日志**——音视频必须收媒体指标。
2. **不区分上行和下行**。
3. **不记录设备和浏览器版本**。
4. **服务端无房间级指标**。
5. **不保存失败样本**——无法复现。
