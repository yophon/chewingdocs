# 转码、转封装、裁剪、抽帧、水印

这一篇把常见处理任务按代价分清楚。音视频处理最怕不知道自己触发了重编码。

> 一句话先记住:**转封装便宜,转码昂贵,滤镜一定要重新编码**。先判断任务属于哪类,再写命令。

---

## 一、任务分类

| 任务 | 是否解码 | 是否编码 | 典型代价 |
| --- | --- | --- | --- |
| 转封装 | 否 | 否 | 低 |
| 截取 copy | 否 | 否 | 低,但不精确 |
| 抽帧 | 是 | 图片编码 | 中 |
| 缩放 / 裁剪 | 是 | 是 | 高 |
| 水印 | 是 | 是 | 高 |
| 转码 | 是 | 是 | 高 |

判断标准很简单:只要改画面内容,就要解码成原始帧再编码回去。

---

## 二、转封装

```bash
ffmpeg -i input.mkv -map 0 -c copy output.mp4
```

`-map 0` 表示尽量保留所有流。注意目标容器不一定能装所有流。失败时不要硬转,先看流:

```bash
ffprobe -hide_banner input.mkv
```

---

## 三、标准 Web 输出

```bash
ffmpeg -i input.mov \
  -c:v libx264 -profile:v high -level 4.1 \
  -pix_fmt yuv420p \
  -preset medium -crf 23 \
  -c:a aac -b:a 128k \
  -movflags +faststart \
  output.mp4
```

关键点:

```text
yuv420p           兼容性
H.264 + AAC       Web / 移动端基线
faststart         MP4 可边下边播
```

---

## 四、缩放多档清晰度

```bash
ffmpeg -i input.mp4 \
  -vf scale=1280:-2 \
  -c:v libx264 -crf 23 -preset medium \
  -c:a aac -b:a 128k \
  output_720p.mp4
```

多档转码不要在一个请求线程里干。放队列,记录任务状态,失败可重试。

---

## 五、水印

```bash
ffmpeg -i input.mp4 -i logo.png \
  -filter_complex "overlay=W-w-20:H-h-20" \
  -c:v libx264 -crf 23 \
  -c:a copy \
  output.mp4
```

`W/H` 是主视频宽高,`w/h` 是水印宽高。右下角留 20 像素。

---

## 六、抽帧和封面

```bash
ffmpeg -ss 3 -i input.mp4 -frames:v 1 cover.jpg
```

批量抽帧:

```bash
ffmpeg -i input.mp4 -vf fps=1 frames/%06d.jpg
```

抽帧常用于封面、审核、预览缩略图、AI 分析。

---

## 七、生产任务表

最小任务表:

| 字段 | 作用 |
| --- | --- |
| input_url | 原始文件 |
| output_url | 输出文件 |
| status | pending / running / success / failed |
| profile | 720p / 1080p / audio-only |
| error | 失败原因 |
| duration_ms | 耗时 |

转码是异步任务,不是普通 HTTP 请求。

---

## 八、踩坑提醒

1. **转码阻塞 API 请求**——大文件会拖死服务。
2. **不限制输入大小和时长**——容易被上传打爆。
3. **水印后还 `-c:v copy`**——不可能。
4. **多档清晰度重复解码**——可以考虑复杂滤镜一次解码多路输出。
5. **不保存 ffmpeg stderr**——失败时没法查。
