# 一对一通话实战

一对一通话最小闭环不是两个浏览器互相 `addTrack`。还需要信令、状态机、ICE、错误处理和资源释放。

> 一句话先记住:**一对一 WebRTC = 两端 PeerConnection + 一个信令通道 + 一套协商状态机**。本地 demo 成功只是第一步。

---

## 一、最小架构

```text
Browser A  <--- WebSocket signaling --->  Server  <--- WebSocket ---> Browser B
   |                                                                  |
   |--------------------- WebRTC media path --------------------------|
```

信令服务器只转发:

- join / leave
- offer
- answer
- ice candidate

---

## 二、发起方流程

```js
const pc = new RTCPeerConnection({ iceServers })
const stream = await navigator.mediaDevices.getUserMedia({ video: true, audio: true })

stream.getTracks().forEach((track) => pc.addTrack(track, stream))

pc.onicecandidate = (event) => {
  if (event.candidate) signal.send({ type: "candidate", candidate: event.candidate })
}

pc.ontrack = (event) => {
  remoteVideo.srcObject = event.streams[0]
}

const offer = await pc.createOffer()
await pc.setLocalDescription(offer)
signal.send({ type: "offer", sdp: offer.sdp })
```

---

## 三、接收方流程

```js
await pc.setRemoteDescription({ type: "offer", sdp })

const stream = await navigator.mediaDevices.getUserMedia({ video: true, audio: true })
stream.getTracks().forEach((track) => pc.addTrack(track, stream))

const answer = await pc.createAnswer()
await pc.setLocalDescription(answer)
signal.send({ type: "answer", sdp: answer.sdp })
```

发起方收到 answer:

```js
await pc.setRemoteDescription({ type: "answer", sdp })
```

---

## 四、Candidate 乱序处理

candidate 可能早于 remote description 到达。简单做法:先缓存。

```js
const pendingCandidates = []

async function handleCandidate(candidate) {
  if (!pc.remoteDescription) {
    pendingCandidates.push(candidate)
    return
  }
  await pc.addIceCandidate(candidate)
}

async function flushCandidates() {
  for (const c of pendingCandidates) await pc.addIceCandidate(c)
  pendingCandidates.length = 0
}
```

设置 remote description 后 flush。

---

## 五、挂断清理

```js
pc.getSenders().forEach((sender) => sender.track?.stop())
pc.getReceivers().forEach((receiver) => receiver.track?.stop())
pc.close()
localVideo.srcObject = null
remoteVideo.srcObject = null
```

离开房间、刷新页面、对方断开都要走清理逻辑。

---

## 六、必须打点

最低限度:

- signalingState
- iceConnectionState
- connectionState
- offer / answer 时间
- candidate 数量
- 首帧时间
- getStats 丢包 / RTT / jitter

没有这些,一对一问题也会变成玄学。

---

## 七、踩坑提醒

1. **candidate 乱序不处理**。
2. **挂断不 stop track**。
3. **没有 TURN**。
4. **双方同时发 offer**。
5. **信令断了不等于媒体断了**——状态要分开。
