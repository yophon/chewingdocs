# 训练技巧:过拟合、归一化与初始化

到这一篇,你已经能跑通一个 MLP 训练循环了。但工程实战里,**让模型训得动 ≠ 训得好**。这一篇是把"能跑"变成"能用"的所有现代技巧。

> 一句话先记住:**深度学习的工程秘密不在结构,而在归一化 + 初始化 + 优化器 + 学习率调度**。这四件套撑起了 90% 的实际效果。

---

## 一、过拟合现象与诊断

**过拟合**:模型把训练集"背"下来了,但在新数据上表现差。

诊断最直接的方法:**画 train loss 和 val loss 曲线**。

| 现象 | train loss | val loss | 诊断 |
| --- | --- | --- | --- |
| 欠拟合 | 高 | 高 | 模型太弱 / 训得太短 / 数据没归一化 |
| 正常 | 低且稳定 | 低且稳定 | 完美 |
| 过拟合 | 持续降到很低 | 先降后升 | 经典过拟合,要正则化 |
| 训练崩了 | NaN / 突增 | NaN / 突增 | 学习率太大 / 数据 bug |

```
loss
 ↑
 |  \
 |   \  train ─────────────___
 |    \________            
 |             \____val____
 |                          \____   ← val 反而升,过拟合开始
 |
 +─────────────────────────────→ epoch
                  ↑
                early stop 应该在这附近
```

> **经验法则**:**早停**(early stopping)。监控 val loss,n 个 epoch 不下降就停,保存当时最好的模型。这是最便宜也最有效的"正则化"。

---

## 二、L1 / L2 正则化

往 loss 里加一个"参数大小的惩罚项":

```
L_total = L_data + λ · R(W)

L1:  R(W) = Σ |w|
L2:  R(W) = Σ w²
```

| 类型 | 公式 | 效果 | 用在哪 |
| --- | --- | --- | --- |
| L1 | Σ\|w\| | 让权重**稀疏**(很多 w → 0) | 特征选择、轻量化 |
| L2(weight decay) | Σ w² | 让权重**普遍变小**但不为 0 | 通用正则化 |

> 在 PyTorch 里,L2 通常通过 optimizer 的 `weight_decay` 参数实现:
>
> ```python
> optimizer = torch.optim.AdamW(model.parameters(), lr=1e-3, weight_decay=0.01)
> ```
>
> 这是 LLM 训练的标配。**注意:Adam 和 AdamW 对 weight decay 的处理不同**——Adam 把它混进梯度,AdamW 是"解耦"的纯权重衰减。**生产环境一律用 AdamW**。

## 三、Dropout

训练时**随机把一部分神经元置 0**,逼模型不要依赖任何单个神经元。

```python
# 训练时
mask = (torch.rand_like(x) > 0.5).float()
x = x * mask / 0.5         # 1/keep_prob 缩放,保证期望不变
```

| 参数 | 典型值 | 说明 |
| --- | --- | --- |
| dropout p | 0.1~0.5 | 丢弃比例 |
| 用在哪 | 全连接层之间 / Transformer block 内部 | 卷积层一般不用 |
| eval 模式 | 不丢弃 | `model.eval()` 自动关闭 |

> **关键**:Dropout **只在训练时丢**。`model.eval()` 会自动关闭。**忘了切 eval 是新手 bug 第一名**——推理时 dropout 还在,结果飘忽不定。

```python
class Block(nn.Module):
    def __init__(self):
        super().__init__()
        self.fc1 = nn.Linear(128, 128)
        self.act = nn.GELU()
        self.drop = nn.Dropout(0.1)
        self.fc2 = nn.Linear(128, 128)

    def forward(self, x):
        return self.fc2(self.drop(self.act(self.fc1(x))))
```

## 四、BatchNorm vs LayerNorm

归一化是把激活的分布拉到 "均值 0、方差 1" 附近,**让训练稳得多**。

### 4.1 公式对比

```
BatchNorm:    在 batch 维度上算均值方差
LayerNorm:    在特征维度上算均值方差(对每个样本独立)
```

举个例子,输入 `(b, d) = (32, 128)`:

| 归一化 | 沿哪一维算均值方差 | 输出还要做的 |
| --- | --- | --- |
| BN | 沿 b 维(每个特征独立,跨 32 个样本) | 学一对 (γ, β) 缩放偏移 |
| LN | 沿 d 维(每个样本独立,跨 128 个特征) | 学一对 (γ, β) 缩放偏移 |

### 4.2 关键区别

| 维度 | BatchNorm | LayerNorm |
| --- | --- | --- |
| 依赖 batch size | 强(batch 太小不稳) | 不依赖 |
| 训练 vs 推理行为 | 不同(推理用 running stats) | 相同 |
| 对序列长度敏感 | 是(不同长度统计量不同) | 否 |
| 适合 | CNN / 图像 | RNN / Transformer / NLP |

> **伏笔**:**Transformer 用 LayerNorm 不是 BatchNorm**。原因:
> 1. NLP 的序列长度变化大,BN 的 batch 统计量不稳
> 2. LN 不依赖 batch,推理一致性好
> 3. LN 是逐样本归一化,适合自回归生成

13 篇会再展开 LN 在 Transformer 里的具体位置(Pre-LN vs Post-LN)。

### 4.3 代码

```python
import torch
import torch.nn as nn

# BatchNorm
bn = nn.BatchNorm1d(128)   # 输入 shape (b, 128)
x = torch.randn(32, 128)
y = bn(x)
print(y.mean(0).abs().mean(), y.std(0).mean())   # ≈ 0, ≈ 1

# LayerNorm
ln = nn.LayerNorm(128)
y = ln(x)
print(y.mean(-1).abs().mean(), y.std(-1).mean()) # ≈ 0, ≈ 1
```

> 还有 GroupNorm、RMSNorm(LLaMA 用的简化版 LN)等。LLM 时代 RMSNorm 比 LN 还流行,因为更省一点(不算均值,只缩放)。

## 五、权重初始化

为什么需要好的初始化?

- 全部初始化为 0 → 所有神经元学到一样,等价于 1 个神经元
- 太大 → 前向爆炸、激活饱和、梯度消失
- 太小 → 信号传不下去,梯度消失

**目标**:让每一层的输出方差**保持不变**,这样信号能在深网络里传得远。

| 初始化 | 公式(简) | 适合的激活 |
| --- | --- | --- |
| Xavier(Glorot) | std = √(1/n_in) 或 √(2/(n_in+n_out)) | sigmoid、tanh |
| Kaiming(He) | std = √(2/n_in) | ReLU 系列 |
| 截断正态 | trunc_normal,std=0.02 | Transformer / BERT 默认 |

```python
# PyTorch 里
linear = nn.Linear(128, 64)

# Xavier
nn.init.xavier_uniform_(linear.weight)

# Kaiming(ReLU 默认)
nn.init.kaiming_normal_(linear.weight, nonlinearity='relu')

# 截断正态(BERT/GPT 标配)
nn.init.trunc_normal_(linear.weight, std=0.02)
```

> **2026 现实**:99% 情况下你不需要手动初始化。`nn.Linear` 默认是 Kaiming uniform,够用。**只在自己写非标准 layer 或者复现某个论文时才管这个**。

## 六、Optimizer 选型

| 优化器 | 公式核心 | 何时用 |
| --- | --- | --- |
| SGD | θ ← θ - η·∇L | 简单任务、CNN、研究表明的稳定基线 |
| SGD + Momentum | 加了"动量",像滚雪球 | CNN 图像分类 |
| Adam | 自适应每个参数的学习率 | 通用,默认首选 |
| AdamW | Adam + 解耦 weight decay | **LLM / Transformer 标配** |
| Adafactor / Lion | 省显存 / 大模型预训练 | 训练巨型模型 |

### 6.1 Momentum

```
v ← β·v + ∇L
θ ← θ - η·v
```

直觉:**梯度是速度,动量是惯性**。能跨过小坑,加速直道。

### 6.2 Adam

```
m ← β1·m + (1-β1)·∇L                # 一阶矩(梯度的滑动平均)
v ← β2·v + (1-β2)·(∇L)²             # 二阶矩(梯度平方的滑动平均)
θ ← θ - η · m / (√v + ε)            # 自适应步长
```

直觉:**梯度大的方向走小步,梯度小的方向走大步**(因为除了 √v)。无脑收敛。

### 6.3 AdamW

```
θ ← θ - η · (m / (√v + ε) + λ·θ)    # weight decay 不再混进梯度
```

> **经验法则**(2026):
> - **不知道选啥**:`AdamW`,lr=1e-3,weight_decay=0.01
> - **CNN 图像分类**:SGD + Momentum,lr=0.1,cosine schedule
> - **LLM 微调**:AdamW,lr=2e-5 ~ 5e-4
> - **从头预训练 LLM**:AdamW + 余弦调度 + warmup,lr=1e-4 ~ 6e-4

```python
# 最常见的训练配置
optimizer = torch.optim.AdamW(
    model.parameters(),
    lr=1e-3,
    betas=(0.9, 0.999),
    weight_decay=0.01,
)
```

## 七、Learning Rate Schedule

**学习率不应该恒定**。开始大、结束小,效果普遍更好。

### 7.1 主流策略

| 策略 | 曲线 | 用在哪 |
| --- | --- | --- |
| Step decay | 阶梯下降(每 N epoch 除 10) | 老式 CV |
| Cosine | 像半个余弦波,平滑下降 | 现代 LLM / Transformer |
| Warmup + Cosine | 先线性升到峰值,再余弦下降 | **LLM 标配** |
| OneCycle | 先升后降的三角形 | 快速收敛实验 |
| ReduceLROnPlateau | val loss 不降就 lr 减半 | 监督学习里方便 |

### 7.2 为什么需要 Warmup?

刚开始模型权重随机,梯度方向不靠谱,大 lr 直接爆。**前几百步用小 lr 让模型先稳定**,再放开学习。

```
lr
 ↑
 |    ╭──────╮
 |   ╱        ╲___          ← cosine 下降
 |  ╱             ╲___
 | ╱                  ╲__
 |/                      ╲_
 +─────────────────────────→ step
   ↑warmup
```

```python
from torch.optim.lr_scheduler import CosineAnnealingLR, LambdaLR

# Warmup + Cosine 的标准实现
def warmup_cosine_schedule(optimizer, warmup_steps, total_steps):
    def lr_lambda(step):
        if step < warmup_steps:
            return step / warmup_steps
        progress = (step - warmup_steps) / (total_steps - warmup_steps)
        return 0.5 * (1 + math.cos(math.pi * progress))
    return LambdaLR(optimizer, lr_lambda)
```

> **经验值**:warmup 一般是总步数的 **1%~10%**。微调任务 100~500 步,预训练几千步。

## 八、踩坑

1. **不归一化输入是新手 bug No.1**。MNIST 的 `transforms.Normalize`、文本 embedding 前后的 LayerNorm,都不能省。**输入分布奇怪 → 训练崩**。
2. **`model.train()` / `model.eval()` 一定要切换**。eval 模式下 BN 用 running stats、Dropout 关闭,推理时忘切等于结果飘。
3. **batch size 太小 BN 不稳**。BN 在 batch < 8 时统计量极不稳,会让 train/val 差距大。这种情况换 GroupNorm 或 LayerNorm。
4. **weight decay 别加在 bias 和 LayerNorm 上**。一般做法是分组:权重应用 weight decay,bias 和 norm 参数不加。HuggingFace 的训练脚本默认这么做。
5. **lr 跟 batch size 成正比**。batch 翻倍,lr 大致也要翻倍("线性缩放规则")。这条是大模型分布式训练的常识。
6. **过拟合不是越多正则化越好**。先看 val 曲线,真过拟合再加 Dropout / weight decay,**正则化加多了变欠拟合**。
7. **Adam 的 weight decay 不是真 weight decay**。它把 wd 混进了梯度,等价于在 loss 里加 L2,但和"真正的权重衰减"在数学上不等价。**只用 AdamW**。
8. **LR schedule 写错比没写还糟**。比如 warmup 步数写成 epoch 数(1 个 epoch 几千步,等到 warmup 完模型也练完了)。**调度器的 step 单位是 batch 还是 epoch,看一眼文档别想当然**。
9. **梯度裁剪一行能救命**。LLM/RNN 训练里 `torch.nn.utils.clip_grad_norm_(model.parameters(), 1.0)` 是标配,防止偶发的梯度爆炸把训练搞炸。
10. **保存最好的 checkpoint,不是最后的**。早停 + 保存 val 最优,这一行习惯能让你少哭几次。

> **总结**:这一篇的所有技巧合起来就是"现代深度学习的工程范式"。LLM 预训练、微调、Diffusion 训练……底下都是这一套。掌握了它,你看任何训练脚本都能秒懂在做啥。
