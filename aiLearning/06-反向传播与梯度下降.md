# 反向传播与梯度下降

这一篇是整个深度学习的引擎。读完你会理解:为什么 PyTorch 一行 `loss.backward()` 能搞定几十亿参数的更新,以及为什么学习率太大模型会炸、太小会龟。

> 一句话先记住:**梯度下降是"沿着 loss 下降最快的方向走一小步";反向传播是"用链式法则一次性算出所有参数的梯度"**。前者是策略,后者是手段。

---

## 一、梯度下降直觉:下山

想象你蒙着眼站在山上,要走到最低点。你能感受到"哪边是下坡",于是每次往下坡方向迈一小步,迭代很多次,最终走到谷底。

**这就是梯度下降的全部直觉**。

| 物理 | 数学 |
| --- | --- |
| 你的位置 | 当前参数 θ |
| 山的高度 | loss 值 |
| 每个方向的坡度 | 梯度 ∇L |
| 下坡方向 | 负梯度 -∇L |
| 一步走多远 | 学习率 η |
| 走一步 | θ ← θ - η · ∇L |

更新公式就一行:

```
θ ← θ - η · ∇L(θ)
```

> **核心**:梯度方向是 loss **上升最快**的方向,所以我们走它的**反方向**。

### 几个变种

| 名字 | 一步用多少数据 | 特点 |
| --- | --- | --- |
| Full-batch GD | 全部训练集 | 准但慢,一个 epoch 才更新一次 |
| Stochastic GD(SGD) | 1 个样本 | 快但震荡剧烈 |
| Mini-batch SGD | 一个 batch(16~512) | 现代默认,平衡速度和稳定 |

> **2026 年的"SGD" 通常都指 Mini-batch SGD**。没人真的一次跑一个样本。

## 二、链式法则在神经网络里怎么用

回顾 02 篇的链式法则:

```
y = f(g(x))
dy/dx = f'(g(x)) · g'(x)
```

神经网络就是一个**深度嵌套的复合函数**:

```
loss = L( f_n( f_(n-1)( ... f_1(x) ... )))
```

要算 `∂loss/∂W_1`,就要把整条链上每一层的"局部导数"乘起来。

```
∂L/∂W_1 = ∂L/∂a_n · ∂a_n/∂a_(n-1) · ... · ∂a_2/∂a_1 · ∂a_1/∂W_1
```

> 直接看公式头大,**关键是理解"每层只需要算自己的局部导数,后面的事交给下一层"**。这就是反向传播能算的核心。

---

## 三、反向传播:手推一个两层网络

为了印象深刻,我们手推一遍。模型超级简单:

```
input  x        (1 维标量,简化用)
hidden h = ReLU(w1·x + b1)
output y_hat = w2·h + b2
loss   L = (y_hat - y)²
```

参数:`w1, b1, w2, b2`。求 `∂L/∂w1`。

### 前向

```
z1 = w1·x + b1
h  = ReLU(z1) = max(0, z1)
y_hat = w2·h + b2
L  = (y_hat - y)²
```

### 反向(从 loss 往前)

```
∂L/∂y_hat = 2(y_hat - y)                # loss 对输出
∂y_hat/∂w2 = h                          # 输出对 w2
∂y_hat/∂h = w2                          # 输出对 h
∂h/∂z1 = 1 if z1 > 0 else 0             # 激活函数局部导数
∂z1/∂w1 = x                             # z1 对 w1
```

链式起来:

```
∂L/∂w1 = (∂L/∂y_hat) · (∂y_hat/∂h) · (∂h/∂z1) · (∂z1/∂w1)
       = 2(y_hat - y) · w2 · 1[z1 > 0] · x
```

```
∂L/∂w2 = (∂L/∂y_hat) · (∂y_hat/∂w2)
       = 2(y_hat - y) · h
```

> **观察 1**:`w2` 出现在 `∂L/∂w1` 的式子里——也就是说**前面层的梯度依赖后面层的权重**。所以 `w2` 太小会导致前面梯度也小(梯度消失),`w2` 太大会爆炸。
>
> **观察 2**:ReLU 的导数是 0 或 1。当 `z1 ≤ 0` 时,`∂L/∂w1 = 0`,**这就是 04 篇说的"死神经元"**——一旦输入恒小于 0,梯度永远是 0,这个权重就再也不更新了。

### 一般规律

对每一层:

```
设 a_l 是第 l 层的输出,δ_l = ∂L/∂a_l(从下游传回来的梯度)

本层参数 W_l 的梯度:∂L/∂W_l = δ_l · (∂a_l/∂W_l)
往前传给上一层的梯度:δ_(l-1) = δ_l · (∂a_l/∂a_(l-1))
```

**前向用一遍输入算输出,反向用一遍输出算梯度**。计算量约为前向的 2 倍。

## 四、SGD vs Mini-batch vs Full-batch

| 维度 | Full-batch | Mini-batch | SGD(单样本) |
| --- | --- | --- | --- |
| 一步用多少数据 | 全部 | 32~512 | 1 |
| 一步耗时 | 长 | 中 | 短 |
| 一步梯度方差 | 0(确定) | 小 | 大 |
| 利用 GPU | 一般(可能爆显存) | 高 | 低 |
| 是否容易跳出局部最优 | 不容易 | 中等 | 容易 |
| 现代用法 | 不用 | **默认** | 不用 |

> **经验法则**:batch size 选**显卡能装下的最大 2 的幂**(32 / 64 / 128 / 256...)。太小不利用 GPU 并行,太大显存爆 + 收敛差。LLM 训练 batch 上千,但用了梯度累积技巧。

### 梯度累积(Gradient Accumulation)

显存不够装大 batch?**多次小 batch 的梯度累加,再一起更新**:

```python
optimizer.zero_grad()
for i, (x, y) in enumerate(loader):
    loss = criterion(model(x), y) / accumulation_steps
    loss.backward()                      # 梯度累积
    if (i + 1) % accumulation_steps == 0:
        optimizer.step()
        optimizer.zero_grad()
```

效果上等价于大 batch,代价是慢一点。**LLM 微调几乎都靠这个**。

## 五、学习率的影响

学习率(learning rate, lr)就是"每步走多远"。

| 学习率 | 现象 | 看 loss 曲线 |
| --- | --- | --- |
| 太大(e.g. 1.0) | loss 爆炸 / 振荡 / NaN | 上下乱跳或飙升 |
| 偏大(e.g. 0.1) | 不能精细收敛,在最优附近震荡 | 下降快但稳不住 |
| 合适 | 平稳下降 | 像光滑曲线 |
| 偏小 | 收敛慢 | 缓慢下降 |
| 太小(e.g. 1e-6) | 几乎不动 | 接近水平线 |

> **经验法则**(2026):
> - **Adam / AdamW**:从 `1e-3` 起步(微调 LLM 是 `1e-5` ~ `5e-4`)
> - **SGD**:从 `1e-2` 起步,大模型 `1e-1` 也常见
> - **从大往小试**:`1e-2 → 1e-3 → 1e-4`,看哪个 loss 下得最稳
> - **Warmup + Cosine** 是现代 LLM 标配,07 篇会讲

### 学习率太大的视觉比喻

> 在山谷里,跨步太大,直接跨到对面山坡上,反而比之前还高。loss 不降反升,甚至 NaN。

```python
# 一个能让你"看到"学习率影响的小实验
import torch

x = torch.tensor(5.0, requires_grad=True)
for lr in [0.01, 0.1, 0.5, 1.1]:
    xx = x.clone().detach().requires_grad_(True)
    history = []
    for _ in range(50):
        y = xx ** 2                  # 目标:让 x 到 0
        y.backward()
        with torch.no_grad():
            xx -= lr * xx.grad
            xx.grad = None
        history.append(xx.item())
    print(f'lr={lr}: x final = {history[-1]:.4f}')
# lr=0.01: x final = 1.6406       慢
# lr=0.1:  x final = 0.0057       好
# lr=0.5:  x final = 0.0000       极好(刚好二阶最优)
# lr=1.1:  x final = -52.0...     爆炸
```

## 六、PyTorch autograd 演示

PyTorch 把链式法则全自动化了。每个 tensor 操作都被记录成一张"计算图",`backward()` 沿着图反向传播。

### 6.1 最小例子

```python
import torch

x = torch.tensor(3.0, requires_grad=True)
y = x ** 2 + 2 * x + 1               # y = (x+1)²
y.backward()                         # 自动求导
print(x.grad)                        # 2x + 2 = 8
```

### 6.2 看到中间梯度

```python
x = torch.tensor(2.0, requires_grad=True)
w = torch.tensor(3.0, requires_grad=True)
b = torch.tensor(1.0, requires_grad=True)

y = w * x + b                        # 7
loss = (y - 5) ** 2                  # 4
loss.backward()

print(x.grad)   # tensor(12.)  ∂L/∂x = 2(y-5)·w = 4·3
print(w.grad)   # tensor(8.)   ∂L/∂w = 2(y-5)·x = 4·2
print(b.grad)   # tensor(4.)   ∂L/∂b = 2(y-5)·1 = 4
```

### 6.3 一个 mini 训练循环

```python
torch.manual_seed(0)
x = torch.linspace(-1, 1, 100).unsqueeze(1)
y = 3 * x + 0.5 + 0.1 * torch.randn_like(x)

w = torch.zeros(1, 1, requires_grad=True)
b = torch.zeros(1, requires_grad=True)
lr = 0.1

for step in range(100):
    y_hat = x @ w + b
    loss = ((y_hat - y) ** 2).mean()

    loss.backward()                  # 算梯度
    with torch.no_grad():
        w -= lr * w.grad             # 手动更新
        b -= lr * b.grad
        w.grad.zero_()               # 清零!不然下次会累加
        b.grad.zero_()

    if step % 10 == 0:
        print(f'step {step}: loss={loss.item():.4f} w={w.item():.4f} b={b.item():.4f}')

# 最终 w ≈ 3.0,b ≈ 0.5
```

> 这就是反向传播 + 梯度下降最直白的样子。**真实工程里用 `optimizer.step()` 替代手动减法**,但底层就是这么干的。

## 七、避坑

1. **`backward()` 默认释放计算图**。同一个 loss 不能 backward 两次,会报 `Trying to backward through the graph a second time`。需要的话加 `loss.backward(retain_graph=True)`。
2. **梯度默认会累加,要手动清零**。`optimizer.zero_grad()`(或上文的 `w.grad.zero_()`)必须在每次 backward 前调用。**不调的话梯度会一直加,模型直接发疯**。
3. **梯度累积是利用上面这个特性**。故意不清零,多次 backward 之后再 step,等价于大 batch。
4. **`with torch.no_grad():` 包住更新**。直接 `w -= lr * w.grad` 会被记录到计算图里,导致内存泄漏 + 梯度计算错误。
5. **梯度爆炸先看初始化和 lr**。loss 突然变 NaN,99% 是 lr 太大或者初始化不合理。可以加梯度裁剪 `torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)`。
6. **梯度消失先看激活和层数**。深网络用 sigmoid 必死。换 ReLU/GELU + 残差连接 + LayerNorm 是 Transformer 时代的解法。
7. **`detach()` 切断梯度**。强化学习里 target value 通常 `.detach()`,防止梯度从 target 流回去。LLM 训练的 reward model 也常用。
8. **看到 `grad is None` 大概率是 detach 了或者没设 requires_grad**。一些操作(如 `.numpy()`、`.item()`)会切断梯度链。
9. **`.eval()` 只关闭 BN/Dropout,不影响梯度**。如果想完全不算梯度(节省显存),还得包 `with torch.no_grad():`。
10. **混合精度训练用 `torch.cuda.amp`**。半精度(fp16)算梯度可能下溢,要配合 `GradScaler`。LLM 训练标配。

> **总结**:`loss.backward() + optimizer.step()` 这两行就是整个深度学习的"心脏"。其他都是配套基础设施。理解了这一篇,你已经知道 PyTorch 训练时每秒在干什么。
