## 二、向量与矩阵

### 2.1 标量、向量、张量

| 概念 | 维度 | 直观例子 | shape 写法 |
| --- | --- | --- | --- |
| 标量(scalar) | 0 维 | 温度 25.3 | `()` |
| 向量(vector) | 1 维 | 一个词的 embedding | `(d,)` |
| 矩阵(matrix) | 2 维 | 一张灰度图、一个 batch 的 embedding | `(n, d)` |
| 张量(tensor) | n 维 | 一个 batch 的彩色图 | `(b, c, h, w)` |

> 在 PyTorch 里,**标量、向量、矩阵全都叫 tensor**,只是维度不同。这是 PyTorch 第一个让人迷惑的点:你看到 `tensor` 不要以为是高维的,可能就是个数。

### 2.2 点积(dot product)

两个等长向量,逐位相乘再加和。结果是一个数。

```
a = [1, 2, 3]
b = [4, 5, 6]
a · b = 1×4 + 2×5 + 3×6 = 32
```

**几何意义**:点积 = |a| × |b| × cos(θ)。所以两个向量越"同向",点积越大。这就是为什么 Attention 用点积衡量相似度。

### 2.3 矩阵乘法

`(m, k) @ (k, n) = (m, n)`。**中间那个 k 必须对得上**,这是 90% 的"shape mismatch" 报错的原因。

```
A: (2, 3)        B: (3, 4)        A @ B: (2, 4)
[a b c]          [p q r s]
[d e f]          [t u v w]
                 [x y z 0]
```

> **避坑**:`A @ B ≠ B @ A`。矩阵乘法**不交换**。Transformer 里你看到 `Q @ K^T` 不能写成 `K^T @ Q`,shape 都对不上。

### 2.4 转置(transpose)

行变列,列变行。`(m, n)` → `(n, m)`。在代码里通常是 `.T` 或 `.transpose(-1, -2)`。

### 2.5 广播(broadcasting)

不同 shape 的 tensor 做运算时,"小的"自动扩展成"大的"。这是 numpy / PyTorch 最香的特性,也是最容易踩坑的特性。

| 场景 | shape A | shape B | 结果 shape | 行为 |
| --- | --- | --- | --- | --- |
| 矩阵 + 标量 | (3, 4) | () | (3, 4) | 每个元素都加 |
| 矩阵 + 行向量 | (3, 4) | (4,) | (3, 4) | 每行都加 |
| 矩阵 + 列向量 | (3, 4) | (3, 1) | (3, 4) | 每列都加 |
| 不兼容 | (3, 4) | (3,) | 报错 | 末尾不对齐 |

> **经验法则**:广播看从**最右一维**开始对齐,要么相等,要么其中一个是 1,要么其中一个不存在。其他全报错。

## 三、求导:深度学习的引擎

### 3.1 导数:斜率的别名

`f'(x)` 表示 `f` 在 x 这个点的斜率。直觉理解:**x 微微动一点,f 跟着动多少**。

```
f(x) = x²
f'(x) = 2x

x = 3 时,f'(3) = 6
意思:x 从 3 动到 3.01,f 大约增加 0.06
```

### 3.2 偏导(partial derivative)

函数有多个输入时,**只看其中一个变量动、其他固定时的斜率**。

```
f(x, y) = x² + 3y
∂f/∂x = 2x        (把 y 当常数)
∂f/∂y = 3         (把 x 当常数)
```

### 3.3 梯度(gradient)

把所有偏导组成一个向量,就是梯度。**梯度方向 = 函数上升最快的方向**。

```
∇f = [∂f/∂x, ∂f/∂y]
```

> **核心直觉**:训练神经网络 = 找让 loss 最小的参数。我们计算 loss 对每个参数的梯度,然后**沿梯度反方向**走一小步,loss 就降一点。这就是梯度下降。

### 3.4 链式法则(chain rule)

复合函数怎么求导?**像剥洋葱**,一层一层乘。

```
y = f(g(x))
dy/dx = f'(g(x)) · g'(x)
```

举个具体的:

```
loss = (W·x - y)²

把它拆成:
  z = W·x          → dz/dW = x
  e = z - y        → de/dz = 1
  loss = e²        → d(loss)/de = 2e

链式起来:
  d(loss)/dW = 2e · 1 · x = 2(W·x - y) · x
```

> 这就是反向传播的核心。每层的梯度 = 后一层传回来的梯度 × 本层的局部导数。看不懂没关系,06 篇会再过一遍。

## 四、概率:LLM 输出每个 token 的底层

### 4.1 期望与方差

| 概念 | 含义 | 公式直觉 |
| --- | --- | --- |
| 期望 E[X] | 平均会得到什么值 | 加权平均,权是概率 |
| 方差 Var(X) | 波动有多大 | E[(X - E[X])²] |
| 标准差 σ | 方差开根 | 跟 X 同量纲,直观 |

### 4.2 条件概率与贝叶斯

```
P(A|B) = "在 B 已经发生的前提下,A 发生的概率"

贝叶斯公式:
P(A|B) = P(B|A) · P(A) / P(B)
```

直观例子:模型已经看到了 `"今天天气真"`,问下一个 token 是 `"好"` 的概率,这就是条件概率 `P("好" | "今天天气真")`。**LLM 的本质就是在算这个条件概率**。

### 4.3 最大似然(MLE)

> 给定一堆数据,找一组参数,使得"模型生成这些数据的概率"最大。

这是大部分模型训练目标的数学起点。**最小化负对数似然 = 最大化似然**,所以你看到 loss 经常长这样:

```
loss = -Σ log P(y_i | x_i; θ)
```

这就是 CrossEntropy 的本质,05 篇会展开。

## 五、softmax 与 logits

这两个词在 LLM 文章里出现频率第一。

### 5.1 logits 是什么

**模型最后一层输出的、还没归一化的"原始分数"**。可以是任意实数,可正可负。

```
logits = [2.0, 1.0, 0.1]   # 三个候选词的分数
```

### 5.2 softmax 把 logits 变成概率

```
softmax(z_i) = exp(z_i) / Σ exp(z_j)
```

性质:
- 每一项都在 [0, 1]
- 加起来等于 1
- 大的更大、小的更小(指数放大差异)

```
logits = [2.0, 1.0, 0.1]
exp     = [7.39, 2.72, 1.11]
sum     = 11.22
softmax = [0.659, 0.242, 0.099]
```

> **为什么这么常用?**因为它把任意实数向量平滑地映射成概率分布,且**处处可导**(一会儿要求梯度的)。神经网络爱可导,scalar 转成概率分布最简单的就是它。

### 5.3 temperature 与 softmax 的关系

LLM 推理里调的 temperature,本质就是给 logits 除一个 T:

```
softmax(z / T)

T → 0 :最大值占绝对优势,接近确定性
T = 1 :原版 softmax
T → ∞ :接近均匀分布,完全随机
```

| temperature | 行为 | 适合 |
| --- | --- | --- |
| 0.0 | 总选最大值(贪心) | 代码生成、确定性任务 |
| 0.7 | 略带随机 | 通用对话 |
| 1.0 | 原始分布 | 默认 |
| 1.5+ | 大量随机 | 创意写作 |

> 16 篇讲采样时会再展开。这里先认识一下这个公式。

## 六、numpy 与 torch 代码片段

把上面的概念全用代码过一遍,感觉会清晰很多。

### 6.1 创建 tensor

```python
import torch
import numpy as np

# 标量
a = torch.tensor(3.14)
print(a.shape)              # torch.Size([])

# 向量
v = torch.tensor([1, 2, 3])
print(v.shape)              # torch.Size([3])

# 矩阵
M = torch.tensor([[1, 2], [3, 4]])
print(M.shape)              # torch.Size([2, 2])

# 随机张量(深度学习里最常见)
x = torch.randn(2, 3, 4)    # 标准正态分布,shape (2, 3, 4)
y = torch.zeros(3, 3)
z = torch.ones_like(M)
```

### 6.2 点积、矩阵乘、转置

```python
a = torch.tensor([1.0, 2.0, 3.0])
b = torch.tensor([4.0, 5.0, 6.0])

# 点积
print(torch.dot(a, b))      # tensor(32.)

# 矩阵乘
A = torch.randn(2, 3)
B = torch.randn(3, 4)
C = A @ B                   # 推荐写法
print(C.shape)              # torch.Size([2, 4])

# 转置
print(A.T.shape)            # torch.Size([3, 2])

# 高维转置(只换最后两维)
X = torch.randn(8, 16, 32)
print(X.transpose(-1, -2).shape)   # torch.Size([8, 32, 16])
```

### 6.3 广播

```python
M = torch.randn(3, 4)
row = torch.randn(4)        # (4,)
col = torch.randn(3, 1)     # (3, 1)

print((M + row).shape)      # (3, 4),每行加同一个 row
print((M + col).shape)      # (3, 4),每列加同一个 col
```

### 6.4 求导:autograd 三行版

```python
x = torch.tensor(3.0, requires_grad=True)
y = x ** 2 + 2 * x + 1      # y = x² + 2x + 1
y.backward()                # 自动求导
print(x.grad)               # 2x + 2 = 8
```

是的,PyTorch 的 autograd 把链式法则全帮你算好了。08 篇会展开,这里先有印象。

### 6.5 softmax

```python
import torch.nn.functional as F

logits = torch.tensor([2.0, 1.0, 0.1])
probs = F.softmax(logits, dim=-1)
print(probs)                # tensor([0.6590, 0.2424, 0.0986])
print(probs.sum())          # tensor(1.0000)

# 调 temperature
T = 0.5
hot_probs = F.softmax(logits / T, dim=-1)
print(hot_probs)            # 差异被放大
```

### 6.6 期望与方差

```python
x = torch.randn(10000)      # 标准正态
print(x.mean())             # ≈ 0
print(x.var())              # ≈ 1
print(x.std())              # ≈ 1
```

## 七、给新手的建议

1. **不要硬啃**。这一篇看一遍能搞清"每个名词是干啥的"就够了,不必背公式。后面遇到具体问题再回来翻。
2. **shape 思维 > 公式思维**。读代码时永远先问自己:**这个 tensor 是 (b, n, d) 还是 (b, d, n)?**90% 的 bug 都是 shape 对不上。
3. **能用 torch 不用 numpy**。numpy 不能 GPU,不能自动求导。深度学习几乎所有场景都用 torch tensor。
4. **梯度概念第一优先**。"loss 对参数求梯度,然后沿反方向更新一小步"——这一句话理解了,你已经看懂深度学习的核心。
5. **softmax 和 cross entropy 死磕一下**。这俩是 LLM 输出层和 loss 的全部基础,值得多花十分钟。
6. **可视化你的直觉**。手画一下 `y = x²` 的图,感受一下"x = 3 时斜率是 6"。比看十遍公式都强。
7. **数学卡住时,跳过先跑代码**。看代码运行结果反推公式含义,比死磕公式更高效。

> **一句话记住**:数学是工具,不是目的。**你不需要变成数学家,你需要的是"能看懂工程师写的论文和代码"**。
