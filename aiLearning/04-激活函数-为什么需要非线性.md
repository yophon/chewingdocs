# 激活函数:为什么需要非线性

这一篇专治一个问题:**没有激活函数,神经网络就是一个高级的线性回归**。读完你会明白 ReLU 为什么能横扫深度学习,以及现在 Transformer 为啥都用 GELU。

> 一句话先记住:**激活函数是非线性的来源**。没它,叠 100 层和 1 层是一样的。

---

## 一、没有激活函数 = 多层等价于一层

简单数学证明,看一遍受用一辈子。

假设有两层全连接,中间不加激活:

```
h = W1·x + b1
y = W2·h + b2
  = W2·(W1·x + b1) + b2
  = (W2·W1)·x + (W2·b1 + b2)
  = W'·x + b'
```

也就是说:**两层线性 = 一层线性**。叠多少层都不会改变这点。再多层都只是换了一组等价的 W' 和 b',你能拟合的还是一条直线/超平面,**异或问题永远解不了**。

> **核心结论**:神经网络要有意义,**每两层之间必须插一个非线性函数**。这个函数就叫激活函数(activation function)。

---

## 二、Sigmoid 和 tanh:经典但梯度消失

### 2.1 Sigmoid

```
σ(x) = 1 / (1 + e^(-x))
```

| 性质 | 值 |
| --- | --- |
| 输出范围 | (0, 1) |
| 导数 | σ(x)·(1 - σ(x)) |
| 导数最大值 | 0.25(在 x = 0) |
| 形状 | S 型 |

90 年代深度学习的标配。直觉:把任意实数压到 (0, 1),像个"软开关"。

> 但它有个**致命问题**:**梯度消失**。导数最大才 0.25,叠 10 层后梯度就剩下 0.25^10 ≈ 1e-6,接近 0。深网络根本训不动。

### 2.2 tanh

```
tanh(x) = (e^x - e^(-x)) / (e^x + e^(-x))
```

| 性质 | 值 |
| --- | --- |
| 输出范围 | (-1, 1) |
| 导数 | 1 - tanh²(x) |
| 导数最大值 | 1.0(在 x = 0) |
| 形状 | S 型,但中心化 |

比 sigmoid 好一点(0 中心化、梯度大一点),但**两端还是会饱和**,深网络一样会消失。

> Sigmoid 现在主要用于:**二分类的输出层**(把 logit 压成概率)、**门控**(LSTM/GRU 里那些 0~1 的权重门)。**隐藏层基本不用了**。

---

## 三、ReLU:现代主流

```
ReLU(x) = max(0, x)
```

| 性质 | 值 |
| --- | --- |
| 输出范围 | [0, +∞) |
| 导数 | 1 if x > 0 else 0 |
| 计算成本 | 一个比较,极便宜 |

2012 年 AlexNet 用它把 ImageNet 准确率拔了一截,从此成为 default。

**为什么这么香**:
- 正区间梯度恒为 1,**完全没有梯度消失**(只要 x > 0)
- 计算极快(没指数运算)
- 自带稀疏性(负值都为 0)

### 3.1 死神经元(Dead ReLU)

ReLU 唯一的坑:如果某个神经元的输入永远 < 0(初始化不好或学习率太大),它的梯度永远是 0,这个神经元就**永远死掉**。

为了解决这个,出了一堆变体:

| 变体 | 公式 | 特点 |
| --- | --- | --- |
| LeakyReLU | x if x>0 else 0.01x | 负区间留一个小斜率,解决死神经元 |
| PReLU | x if x>0 else α·x | α 是可学习参数 |
| ELU | x if x>0 else α(e^x - 1) | 负区间平滑,均值更接近 0 |
| ReLU6 | min(max(0, x), 6) | 截断在 6,移动端常用 |

> **经验法则**:90% 的场景上 ReLU 就行。**死神经元真的很少触发**(用合理的初始化 + 不太离谱的学习率),不要把这些变体当默认。

---

## 四、GELU、Swish/SiLU:Transformer 偏爱

### 4.1 GELU(Gaussian Error Linear Unit)

```
GELU(x) ≈ 0.5 · x · (1 + tanh(√(2/π) · (x + 0.044715·x³)))
```

直觉理解:它是 ReLU 的"平滑版"。负区间不再是硬截断,而是渐变地"压低"。

| 模型 | 用的激活 |
| --- | --- |
| BERT | GELU |
| GPT-2/3/4 | GELU |
| Llama 系列 | SiLU |
| Claude / GPT-5 | 没公开,大概率类似 |

### 4.2 Swish / SiLU

```
SiLU(x) = x · σ(x) = x / (1 + e^(-x))
```

(Swish 和 SiLU 是同一个东西,Google 叫 Swish,PyTorch 叫 SiLU。)

性质:平滑、非单调(在 x ≈ -1.28 附近会先降一点再升),负区间不是 0 也不会饱和到 -∞。Llama / Mistral 都是 SiLU。

### 4.3 为什么 Transformer 偏爱平滑激活?

> **猜想**:Transformer 用残差连接 + LayerNorm,梯度流已经很好了,这时候**激活的"平滑性"对优化更友好**。ReLU 的"硬拐点"对深层 Transformer 反而不太好。

实际效果:GELU/SiLU 比 ReLU 在 Transformer 上**普遍好 0.3~1 个点**,所以现在大模型默认这俩。

## 五、对比表与选型

| 激活 | 输出范围 | 梯度消失 | 死神经元 | 计算成本 | 平滑 | 适用场景 |
| --- | --- | --- | --- | --- | --- | --- |
| Sigmoid | (0, 1) | 严重 | 否 | 中 | 是 | 二分类输出、门控 |
| tanh | (-1, 1) | 中 | 否 | 中 | 是 | RNN 隐藏层 |
| ReLU | [0, ∞) | 无 | 有 | 极低 | 否 | CNN、MLP 默认 |
| LeakyReLU | (-∞, ∞) | 无 | 几乎无 | 极低 | 否 | ReLU 死神经元严重时 |
| GELU | (-∞, ∞) | 无 | 几乎无 | 高 | 是 | Transformer / BERT / GPT |
| SiLU | (-∞, ∞) | 无 | 几乎无 | 中 | 是 | Llama / Mistral / 现代 LLM |

> **2026 选型经验法则**:
> - **CNN / 简单 MLP**:ReLU
> - **Transformer / LLM**:GELU 或 SiLU(看你抄哪家代码)
> - **二分类输出**:Sigmoid
> - **多分类输出**:Softmax(05 篇)
> - **RNN / LSTM**:tanh + sigmoid 门(经典配方)
> - 不知道选啥,**先 ReLU,跑通了再换**

## 六、用 PyTorch 切换激活函数对比

实战代码:用同一个模型在 MNIST 上比 4 种激活,看 loss 曲线。

```python
import torch
import torch.nn as nn
import torch.optim as optim
from torch.utils.data import DataLoader
from torchvision import datasets, transforms

# 数据
transform = transforms.Compose([transforms.ToTensor(),
                                transforms.Normalize((0.1307,), (0.3081,))])
train_set = datasets.MNIST('./data', train=True, download=True, transform=transform)
loader = DataLoader(train_set, batch_size=128, shuffle=True)

def make_model(act):
    return nn.Sequential(
        nn.Flatten(),
        nn.Linear(784, 128),
        act,
        nn.Linear(128, 64),
        act,
        nn.Linear(64, 10),
    )

acts = {
    'Sigmoid': nn.Sigmoid(),
    'Tanh':    nn.Tanh(),
    'ReLU':    nn.ReLU(),
    'GELU':    nn.GELU(),
}

device = 'cuda' if torch.cuda.is_available() else 'cpu'
results = {}
for name, act in acts.items():
    model = make_model(act).to(device)
    opt = optim.SGD(model.parameters(), lr=0.05)   # 故意用 SGD,放大差异
    crit = nn.CrossEntropyLoss()
    losses = []
    for epoch in range(3):
        for x, y in loader:
            x, y = x.to(device), y.to(device)
            loss = crit(model(x), y)
            opt.zero_grad(); loss.backward(); opt.step()
            losses.append(loss.item())
    results[name] = losses
    print(f'{name:8s} final loss = {losses[-1]:.4f}')

# 典型输出(每次跑会有点波动):
# Sigmoid  final loss = 0.7834   ← 慢,可能没学好
# Tanh     final loss = 0.2415
# ReLU     final loss = 0.1342
# GELU     final loss = 0.1287
```

> **观察**:Sigmoid 在用 SGD 时收敛明显慢,ReLU/GELU 几乎一样快。如果换 Adam,差距会变小但 ReLU 系仍然占优。

如果想看图,加几行 matplotlib:

```python
import matplotlib.pyplot as plt
for name, losses in results.items():
    plt.plot(losses, label=name, alpha=0.7)
plt.legend(); plt.xlabel('step'); plt.ylabel('loss')
plt.show()
```

## 七、避坑

1. **隐藏层用 sigmoid 是 2026 年的考古行为**。除非你在复现 1998 年的论文,否则别上。
2. **死神经元先看初始化和学习率**。学习率 1e-1 + Kaiming 初始化大概率没事,1e-1 + 默认初始化容易死一片。看到训练后期 loss 平了,可以打印 `(model.layer.weight.grad == 0).float().mean()` 看死了多少。
3. **不要在最后一层加激活**。无论分类回归。分类的 softmax 通常被 loss 函数自带、回归的最后一层就是 Linear。
4. **激活函数和归一化一起调**。BN 在 ReLU 之后还是之前,GELU 在 LayerNorm 之后,都有"规矩",07 篇会讲。新手按主流框架的 default 来就行。
5. **两个激活函数串联通常没意义**。`ReLU(GELU(x))` 不会"更非线性",反而可能让梯度更小。一般每层一个激活就够。
6. **GELU 计算贵但 PyTorch 实现做了优化**。直接用 `nn.GELU()` 即可,不要手写公式,慢且数值不稳。
7. **用 `nn.functional.X` 还是 `nn.X` 没本质区别**。前者是函数,后者是 module。在 `nn.Sequential` 里要用 `nn.X`,在自定义 forward 里两种都行。

> **总结**:激活函数是神经网络的"非线性来源",没它一切白搭。但具体选哪个,**主流模型用啥你用啥**就行,不是性能瓶颈。真正决定模型质量的是数据、loss、优化器、架构,这些后面陆续会讲。
