# 损失函数:网络怎么知道自己错了

这一篇讲一件最朴素的事:**模型怎么知道自己预测得不对?**答案是 loss 函数。它把"对错"翻译成一个数,这个数才能被梯度下降优化。

> 一句话先记住:**loss 是模型唯一的"错觉来源"**。loss 选错了,后面调啥都白调。

---

## 一、什么是损失函数

训练神经网络的标准流程:

```
1. 输入 x → 模型 → 预测 y_hat
2. 拿真实标签 y,算 loss = L(y_hat, y)
3. loss 对参数求梯度,反向传播
4. 沿梯度反方向更新参数,loss 降一点
```

**loss 函数 = 优化目标**。模型最终会学到"让 loss 最小的参数",所以 loss 怎么定义,模型就往哪个方向跑。

> **核心**:loss 必须满足两点
> 1. **可导**(才能算梯度)
> 2. **越小越好**(模型预测得越准,loss 越小)

| 任务类型 | 主流 loss | 输出层激活 |
| --- | --- | --- |
| 回归 | MSE / MAE / Huber | 无(直接输出实数) |
| 二分类 | BCE | Sigmoid |
| 多分类 | CrossEntropy | Softmax(loss 内置) |
| 序列生成 | CrossEntropy(每个 token) | Softmax(同上) |
| 度量学习 | InfoNCE / Triplet | 无(输出 embedding) |
| 知识蒸馏 | KL 散度 | 教师/学生都 softmax |

---

## 二、回归任务:MSE / MAE / Huber

### 2.1 MSE(Mean Squared Error)

```
L = mean((y - y_hat)²)
```

最常见的回归 loss。误差平方后求平均。

特点:
- 误差越大,惩罚的"二次方"越严重
- 对异常值敏感(一个离群点可能主导整个 loss)
- 处处可导,梯度光滑

### 2.2 MAE(Mean Absolute Error)

```
L = mean(|y - y_hat|)
```

绝对值,不平方。对异常值更鲁棒。但在 0 点不可导(虽然实际框架里都做了次梯度处理,影响不大)。

### 2.3 Huber Loss(平衡两者)

```
L = 0.5·(y - y_hat)²        if |y - y_hat| ≤ δ
  = δ·(|y - y_hat| - 0.5δ)  otherwise
```

**小误差用 MSE(光滑),大误差用 MAE(抗异常)**。强化学习里很常用。

| Loss | 异常值敏感 | 梯度 | 何时用 |
| --- | --- | --- | --- |
| MSE | 高 | 0 点附近梯度小 | 数据干净时,默认 |
| MAE | 低 | 始终为 ±1 | 数据有离群点 |
| Huber | 中 | 平滑且不饱和 | 折中,RL 里常用 |

```python
import torch
import torch.nn as nn

y      = torch.tensor([3.0, -2.0, 5.0])
y_hat  = torch.tensor([2.5, -1.0, 4.5])

mse   = nn.MSELoss()(y_hat, y)              # 0.5
mae   = nn.L1Loss()(y_hat, y)               # 0.667
huber = nn.SmoothL1Loss(beta=1.0)(y_hat, y) # ≈ 0.292
```

## 三、分类任务:CrossEntropy

### 3.1 从信息论到 CrossEntropy

熵(entropy)衡量"不确定性":

```
H(p) = -Σ p(i)·log p(i)
```

CrossEntropy 衡量"用预测分布 q 来描述真实分布 p"的代价:

```
H(p, q) = -Σ p(i)·log q(i)
```

直觉:**真实分布说哪个类是 1,模型给那个类的概率越大,loss 越小**。

### 3.2 从 softmax 推一遍

多分类场景:

```
模型最后一层:logits = W·h + b           # shape (b, C)
softmax:    p_i = exp(z_i) / Σ exp(z_j)  # 概率分布
真实 label:  y(one-hot)                  # 比如 [0,0,1,0]
loss:       L = -Σ y_i · log p_i = -log p_true
```

因为 y 是 one-hot,只有一个位置是 1。所以 loss 化简成:

```
L = -log(模型给真实类别的概率)
```

例子:3 类问题,真实 label 是 1(第 2 类),logits = [1, 3, 0.5]。

```
softmax([1, 3, 0.5]) ≈ [0.105, 0.776, 0.119]
loss = -log(0.776) ≈ 0.254
```

如果模型预测得很准(p_true 接近 1),`-log(1) = 0`。预测得很差(p_true 接近 0),loss 趋近 ∞。**loss 是对"对真实类别预测概率"的负对数**。

### 3.3 二分类的 BCE

只有两类时,可以简化成:

```
L = -[y·log(p) + (1-y)·log(1-p)]
```

`p` 是 sigmoid 输出。本质就是 CrossEntropy 的 2 类特例。

## 四、KL 散度与 JS 散度

### 4.1 KL(Kullback-Leibler Divergence)

```
KL(p || q) = Σ p(i) · log(p(i) / q(i))
           = -Σ p(i)·log q(i) + Σ p(i)·log p(i)
           = CrossEntropy(p, q) - Entropy(p)
```

直觉:**用 q 替代 p 时多付出的额外不确定性**。

| 性质 | 说明 |
| --- | --- |
| 非负 | KL ≥ 0,只有 p = q 时为 0 |
| 不对称 | KL(p||q) ≠ KL(q||p) |
| 不是距离 | 没有三角不等式 |

**应用场景**:
- **知识蒸馏**:让小模型 q 逼近大模型 p 的输出分布
- **VAE**:让编码后的分布逼近标准正态
- **RLHF**:奖励模型 + KL 惩罚防止策略跑偏(17 篇会讲)

### 4.2 JS(Jensen-Shannon Divergence)

```
JS(p, q) = 0.5·KL(p || m) + 0.5·KL(q || m),  m = (p+q)/2
```

KL 的对称版,有界(0 ~ log 2)。GAN 早期分析用得多,现代 LLM 用得少。

```python
import torch
import torch.nn.functional as F

# KL 散度(注意 PyTorch 的 KLDivLoss 接受 log_q 和 p)
log_q = F.log_softmax(torch.tensor([1.0, 2.0, 0.5]), dim=-1)
p     = F.softmax(torch.tensor([0.5, 2.5, 0.0]), dim=-1)
kl = F.kl_div(log_q, p, reduction='batchmean')
print(kl)
```

## 五、对比学习损失(InfoNCE / Triplet)

不预测类别,而是学**"哪些样本相似、哪些不相似"**的 embedding。Embedding 模型(23 篇)、CLIP、对比预训练都靠它。

### 5.1 Triplet Loss

```
L = max(0, d(a, p) - d(a, n) + margin)
```

| 符号 | 含义 |
| --- | --- |
| a | anchor(锚点) |
| p | positive(正样本,跟 a 同类) |
| n | negative(负样本,跟 a 不同类) |
| d | 距离函数 |
| margin | 期望的 anchor-positive 比 anchor-negative 近至少多少 |

直觉:**让正样本靠近 a,负样本远离 a,且至少差 margin**。

### 5.2 InfoNCE(现代主流)

CLIP / SimCLR / 大部分对比学习用的就是这个:

```
L = -log(  exp(sim(a, p) / τ)  /  Σ exp(sim(a, x_i) / τ)  )
```

把对比学习包装成"在一堆负样本里把正样本认出来"的多分类问题。τ(temperature)控制锐度。

```python
# 简版 InfoNCE 实现
import torch.nn.functional as F

def info_nce(anchors, positives, temperature=0.1):
    # anchors, positives: (b, d) 已 L2 归一化
    logits = anchors @ positives.T / temperature   # (b, b)
    labels = torch.arange(len(anchors), device=anchors.device)
    return F.cross_entropy(logits, labels)
```

> 别小看这个 loss,**CLIP 就是靠它把图文对齐到一个 embedding 空间**,后续多模态全是它的徒孙。

## 六、PyTorch 中的损失函数(注意点)

| nn 类 | 自带 softmax/sigmoid? | 输入格式 |
| --- | --- | --- |
| `nn.CrossEntropyLoss` | **是,自带 softmax** | logits + 整数 label |
| `nn.NLLLoss` | 否,需要先 log_softmax | log_prob + 整数 label |
| `nn.BCELoss` | 否,需要先 sigmoid | prob + 0/1 label |
| `nn.BCEWithLogitsLoss` | **是,自带 sigmoid** | logits + 0/1 label |
| `nn.MSELoss` | N/A | 实数 + 实数 |

### 6.1 CrossEntropyLoss 最常踩的坑

**坑 1:在模型最后又加了 softmax**。

```python
# ❌ 错误写法
class Model(nn.Module):
    def forward(self, x):
        x = self.layers(x)
        return F.softmax(x, dim=-1)   # 多余!

loss = nn.CrossEntropyLoss()(model(x), y)
# CrossEntropyLoss 内部又做了一次 log_softmax,变成 log_softmax(softmax(...))
# loss 会变形,梯度也奇怪
```

```python
# ✅ 正确写法
class Model(nn.Module):
    def forward(self, x):
        return self.layers(x)         # 直接输出 logits

loss = nn.CrossEntropyLoss()(model(x), y)
```

> **记住口诀**:**CrossEntropyLoss 的输入是 logits**。模型最后一层只输出 logits,**不要加 softmax**。

**坑 2:label 类型错误**。`CrossEntropyLoss` 要的 target 是**整数类别索引**,不是 one-hot。

```python
# ❌ 错误
y = torch.tensor([[0,1,0], [1,0,0]])    # one-hot
nn.CrossEntropyLoss()(logits, y)        # 报错或行为奇怪

# ✅ 正确
y = torch.tensor([1, 0])                # 类别索引
nn.CrossEntropyLoss()(logits, y)
```

> 注意:从 PyTorch 1.10 开始也支持 soft target(浮点数 one-hot/分布),但默认场景还是用整数 label 最稳。

**坑 3:label 越界**。10 类分类的 label 必须在 `[0, 9]`。看到 `IndexError: Target X is out of bounds` 就是这个。

### 6.2 完整示例

```python
import torch
import torch.nn as nn
import torch.nn.functional as F

logits = torch.tensor([[2.0, 1.0, 0.1],
                       [0.5, 2.0, 0.3]])
labels = torch.tensor([0, 1])

# 方式 1:用 nn.CrossEntropyLoss(推荐)
loss1 = nn.CrossEntropyLoss()(logits, labels)
print(loss1)              # tensor(0.5715)

# 方式 2:手写,验证一致
log_probs = F.log_softmax(logits, dim=-1)
loss2 = -log_probs[range(2), labels].mean()
print(loss2)              # tensor(0.5715),一致
```

## 七、选型与避坑

1. **回归默认 MSE,有离群点上 Huber**。MAE 在 0 点不可导,深度学习里反而少用。
2. **多分类一律 `nn.CrossEntropyLoss`**。模型不要加 softmax,label 用整数。**这一条记住能避一半的初学者 bug**。
3. **二分类用 `BCEWithLogitsLoss`,不要 `BCELoss`**。前者数值稳定(用 log-sum-exp 技巧),后者你得自己 sigmoid,容易出现 NaN。
4. **类别不均衡时调 `weight` 参数**。`nn.CrossEntropyLoss(weight=...)` 给少数类更大权重。比 oversampling 简单。
5. **loss 不下降先排查 loss 函数本身**。常见错误:label 错位(比如 0/1 颠倒)、label 范围错(类别越界)、模型最后多加了 softmax/sigmoid。
6. **loss 是 NaN 大概率是 log(0)**。在 sigmoid 后再 log,或者梯度爆炸把 logits 拉到 ±inf 触发 softmax 数值溢出。**用带 logits 的 loss(BCEWithLogitsLoss / CrossEntropyLoss)能避大部分**。
7. **不要追求 loss 等于 0**。训练 loss 接近 0 但 val loss 上升,八成是过拟合。07 篇专门讲。
8. **多任务 loss 加权**。`L = α·L1 + β·L2`,系数怎么调?**先各自归一到同量级**(看一开始两个 loss 大小,算个比例),然后小幅扫一下。无脑相加大概率某个 loss 主导。

> **一句话总结**:loss 是模型的"指南针",指向哪儿模型跑哪儿。先把 loss 选对、写对,再聊优化器、超参数。
