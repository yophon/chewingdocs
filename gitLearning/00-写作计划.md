# Git 系列写作计划

> 来源:`未来系列规划.md` 没明写,但写到第六个系列时自然出现的"基础设施"——所有其他系列都默认你会 git,但**真正把 git 用熟的人不到 20%**。
> 这个系列不是"git 命令大全",是**用工程师视角讲清楚 git 的心智模型**——为什么是 DAG、为什么有 staging、为什么 rebase 会改历史、为什么 reflog 能救命。

---

## 设计思路

四层结构,跟其他系列一样从底往上:

```
心智 + 基础(三层模型 / 提交 / 历史)
   → 协作(分支 / 合并 / 远程)
   → 应急 + 整理(stash / cherry-pick / 撤销 / reflog)
   → 工程化(工作流 / hooks / 大仓库 / 安全 / 高级技巧)
```

不重复 GitHub 教程已经讲烂的"clone 一个仓库",而是聚焦**真实工程场景下会让你卡 30 分钟的那些细节**:rebase 把别人的提交搞丢、submodule 拉不下来、reflog 救回 deleted branch、bisect 找 regression、worktree 同时开两个分支……

## 节奏

```
01-05 是"会用 git"的全部内容,优先级最高,看完能做日常工作
06-10 是"和团队协作"的核心,做开源 / 写多人项目必看
11-15 是"出事了怎么办",每条都对应一个真实事故场景
16-22 是"团队 / 大仓库"层面,不是个人开发者的必修
```

**一个硬指标**:看完 14 篇,你应该能在不查文档的情况下,从一个误删的分支里把代码救回来。能做到的话这个系列就值了。
