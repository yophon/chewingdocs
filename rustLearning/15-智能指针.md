# 15-智能指针

> "Rust 的所有权和借用规则有时太严了。智能指针就像合法的'后门'，让你能在保证安全的前提下，玩转复杂的内存结构。"

普通的指针（引用 `&`）只借用数据，没有所有权。**智能指针（Smart Pointers）**不仅拥有它指向的数据，还能执行额外的元数据管理（比如记录有多少人在用这块内存）。
Rust 中常见的智能指针有 `Box<T>`、`Rc<T>` 和 `RefCell<T>`。

## 一、Box<T>：最简单的堆分配

我们知道，Rust 默认把数据放在栈上。如果你想强制把数据放**堆**上，用 `Box`。
场景：你想创建一个递归结构（比如链表、树）。

```rust
// ❌ 报错：递归类型的大小在编译时无法确定
// enum List {
//     Cons(i32, List), 
//     Nil,
// }

// ✅ 正确：Box 是一个指针，指针的大小（8字节）是固定的。
enum List {
    Cons(i32, Box<List>), 
    Nil,
}

use List::{Cons, Nil};
let list = Cons(1, Box::new(Cons(2, Box::new(Cons(3, Box::new(Nil))))));
```

## 二、Rc<T>：引用计数智能指针

Rust 规定一个值只能有一个所有者。但比如在图结构里，一个节点可能被多条边连接，它需要**多个所有者**。
`Rc` (Reference Counting) 就是干这个的。它会在内部维护一个计数器。当计数降为 0 时，清理内存。

```rust
use std::rc::Rc;

let a = Rc::new(String::from("hello"));
// Rc::clone 只是增加引用计数，极其轻量，不会深度拷贝堆数据
let b = Rc::clone(&a); 
let c = Rc::clone(&a);

println!("目前有 {} 个所有者", Rc::strong_count(&a)); // 输出 3
```
**注意：`Rc<T>` 只适用于单线程场景，多线程请看下一篇。且它只能提供不可变引用！**

## 三、RefCell<T>：内部可变性模式

借用规则：不能同时存在多个可变引用，也不能在有不可变引用时存在可变引用。且这些规则是**编译期**检查的。
有时你确信你的代码逻辑没问题，但编译器太傻看不出来。`RefCell` 允许你在有不可变引用的情况下，修改内部的值（这叫内部可变性）。它把借用检查**从编译期推迟到了运行期**。

```rust
use std::cell::RefCell;

let value = RefCell::new(5);

// 获取一个可变的内部引用
*value.borrow_mut() += 10;

println!("{:?}", value.borrow()); // 15
```
如果你在运行时同时调用了两次 `.borrow_mut()`，程序不会在编译时报错，而是会**在运行时直接 Panic 崩溃**！所以这是最后的妥协手段。

## 四、最强缝合怪：Rc<RefCell<T>>

如果你想要一个数据**既有多个所有者，又能被他们修改**，把两者结合：
```rust
use std::rc::Rc;
use std::cell::RefCell;

// value 现在有多个所有者，且可以被改变！
let value = Rc::new(RefCell::new(String::from("hello")));

let clone1 = Rc::clone(&value);
let clone2 = Rc::clone(&value);

clone1.borrow_mut().push_str(" world");
println!("{}", clone2.borrow()); // "hello world"
```
这就是很多 Rust 开发者实现复杂数据结构（如 DOM 树、双向图）时的万能模板。
