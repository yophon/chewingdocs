# 03-基础语法速通

> 一句话导读：Rust 的基础语法并不怪，真正要适应的是“默认不可变、表达式优先、类型明确、编译器严格”的写法。

这篇不追求覆盖所有细节，而是快速建立 Rust 代码的阅读能力。你需要先能看懂变量、函数、控制流、数据类型和模块入口，后面学习所有权时才不会被语法本身拖住。

## 核心心智模型

Rust 的基础语法有三条主线：

1. 默认保守：变量默认不可变，类型默认静态检查，越界默认拒绝。
2. 表达式驱动：`if`、`loop`、代码块都可以产生值。
3. 显式边界：函数参数和返回值类型要写清楚，整数类型、引用类型、所有权边界都由编译器检查。

Rust 不是让你少写代码，而是让更多约束留在编译期。

## 变量绑定与可变性

Rust 使用 `let` 绑定变量，默认不可变：

```rust
fn main() {
    let x = 5;
    println!("x = {x}");

    let mut y = 10;
    y += 1;
    println!("y = {y}");
}
```

如果你尝试修改不可变变量：

```rust
fn main() {
    let x = 5;
    x = 6;
    println!("{x}");
}
```

会得到类似错误：

```text
error[E0384]: cannot assign twice to immutable variable `x`
```

修正：

```rust
fn main() {
    let mut x = 5;
    x = 6;
    println!("{x}");
}
```

这不是形式主义。默认不可变能减少状态变化，让借用检查更容易推理，也能避免很多“谁在什么时候改了这个值”的问题。

## shadowing 不是 mut

Rust 允许用同名变量重新绑定，这叫 shadowing：

```rust
fn main() {
    let spaces = "   ";
    let spaces = spaces.len();

    println!("space count = {spaces}");
}
```

`mut` 是修改同一个变量；shadowing 是创建一个新的绑定。它可以改变类型，所以常用于解析、清洗、转换数据。

```rust
fn main() {
    let input = "42";
    let input: i32 = input.parse().expect("must be a number");

    println!("{}", input + 1);
}
```

## 常量与静态类型

常量使用 `const`，必须标注类型，且值必须能在编译期确定：

```rust
const MAX_RETRY: u32 = 3;

fn main() {
    println!("max retry = {MAX_RETRY}");
}
```

常量适合真正全局固定的配置，不适合运行时计算结果。

## 标量类型

常见标量类型：

- 整数：`i32`、`u32`、`usize`、`u8`
- 浮点数：`f32`、`f64`
- 布尔：`bool`
- 字符：`char`

```rust
fn main() {
    let count: usize = 10;
    let price: f64 = 19.9;
    let enabled: bool = true;
    let letter: char = '中';

    println!("{count}, {price}, {enabled}, {letter}");
}
```

`usize` 常用于索引和长度，因为它和目标平台指针宽度一致。`u8` 常用于字节。普通整数不写类型时，Rust 常推断为 `i32`。

## 元组与数组

元组长度固定，元素类型可以不同：

```rust
fn main() {
    let user: (&str, u32, bool) = ("alice", 18, true);
    let (name, age, active) = user;

    println!("{name} {age} {active}");
    println!("name again = {}", user.0);
}
```

数组长度固定，元素类型必须相同：

```rust
fn main() {
    let scores = [80, 90, 100];
    let zeros = [0; 5];

    println!("first = {}", scores[0]);
    println!("zeros = {:?}", zeros);
}
```

可变长度集合用 `Vec<T>`，后续单独讲。

## 函数与尾表达式

Rust 函数参数必须写类型，返回值用 `->` 标注：

```rust
fn add(left: i32, right: i32) -> i32 {
    left + right
}

fn main() {
    let result = add(20, 22);
    println!("result = {result}");
}
```

注意 `left + right` 后面没有分号。Rust 中：

- 语句执行动作，不返回可用值。
- 表达式计算出值。
- 函数最后一个表达式就是返回值。

如果写成：

```rust
fn add(left: i32, right: i32) -> i32 {
    left + right;
}
```

会得到类似：

```text
error[E0308]: mismatched types
expected `i32`, found `()`
```

修正就是去掉分号，或显式写 `return left + right;`。Rust 社区更常用尾表达式。

## if 是表达式

```rust
fn main() {
    let score = 87;
    let level = if score >= 90 {
        "A"
    } else if score >= 80 {
        "B"
    } else {
        "C"
    };

    println!("level = {level}");
}
```

因为 `if` 是表达式，所以每个分支必须返回同一种类型。下面代码会报错：

```rust
fn main() {
    let condition = true;
    let value = if condition { 1 } else { "one" };
    println!("{value}");
}
```

错误核心是：

```text
error[E0308]: `if` and `else` have incompatible types
```

修正：让两个分支类型一致。

```rust
fn main() {
    let condition = true;
    let value = if condition { "1" } else { "one" };
    println!("{value}");
}
```

## loop、while、for

`loop` 可以返回值：

```rust
fn main() {
    let mut counter = 0;

    let answer = loop {
        counter += 1;

        if counter == 5 {
            break counter * 10;
        }
    };

    println!("answer = {answer}");
}
```

`while` 适合条件循环：

```rust
fn main() {
    let mut n = 3;

    while n > 0 {
        println!("{n}");
        n -= 1;
    }
}
```

`for` 是遍历集合的首选：

```rust
fn main() {
    let nums = [10, 20, 30];

    for n in nums {
        println!("{n}");
    }

    for i in 1..=3 {
        println!("round {i}");
    }
}
```

## 可编译综合示例

```rust
const PASS_SCORE: u32 = 60;

fn grade(score: u32) -> &'static str {
    if score >= 90 {
        "excellent"
    } else if score >= PASS_SCORE {
        "pass"
    } else {
        "fail"
    }
}

fn average(scores: [u32; 3]) -> u32 {
    let mut sum = 0;

    for score in scores {
        sum += score;
    }

    sum / scores.len() as u32
}

fn main() {
    let scores = [80, 92, 70];
    let avg = average(scores);

    println!("average = {avg}, grade = {}", grade(avg));
}
```

这段代码同时使用了常量、数组、函数、循环、表达式返回和格式化输出，可以直接放进 `src/main.rs` 运行。

## 常见编译器错误与修正

### 错误 1：变量未声明为可变

```text
error[E0384]: cannot assign twice to immutable variable
```

修正：确实需要修改时加 `mut`；如果只是转换含义，优先考虑 shadowing。

### 错误 2：函数返回值分号

```text
expected `i32`, found `()`
```

修正：返回表达式末尾不要加分号。

### 错误 3：整数类型不匹配

```rust
fn main() {
    let index: usize = 1;
    let offset: i32 = 2;
    println!("{}", index + offset);
}
```

Rust 不会自动把 `usize` 和 `i32` 混加。修正：

```rust
fn main() {
    let index: usize = 1;
    let offset: usize = 2;
    println!("{}", index + offset);
}
```

类型转换可以用 `as`，但工程上要确认不会截断或改变符号。

## 工程判断

写 Rust 基础代码时，优先让类型表达意图：

- 索引用 `usize`。
- 字节用 `u8`。
- 业务数量如果不会为负，用无符号整数要谨慎，因为减法下溢也是真问题。
- 外部输入先解析成强类型，再进入业务逻辑。

不要滥用 `mut`。可变性越少，借用冲突越少，代码越容易重构。Rust 的基础语法看起来严格，本质上是在鼓励你把数据流和状态变化写清楚。

## 结尾总结

Rust 基础语法的重点不是记住每个关键字，而是适应它的默认立场：少变、强类型、表达式优先、边界清晰。掌握这些之后，下一篇的所有权机制才会变得合理。
