# 10-集合类型 (Vector, String, HashMap)

> "数组定长存栈上，集合动态长在堆。它们是每天搬砖打交道最多的数据结构。"

除了内置的基础数组和元组，Rust 标准库提供了一系列动态分配大小的集合。最常用的就是三剑客：`Vector`、`String` (之前已讲过它也是集合) 和 `HashMap`。

## 一、Vector：动态数组

`Vec<T>` 允许你在单个数据结构中储存多个同类型的值，它们在内存中彼此相邻地排列在**堆**上。

### 1. 创建与更新
```rust
// 使用宏创建一个带有初始值的 Vec
let mut v = vec![1, 2, 3]; 

// 或者通过 new() 创建一个空的，需要显示标注类型
let mut v2: Vec<i32> = Vec::new();

v.push(4);
v.push(5);
```

### 2. 读取元素
有两种办法读取：`[]` 和 `.get()`。
```rust
let v = vec![10, 20, 30, 40, 50];

// 1. 索引语法：如果越界，会直接 panic 崩溃！
let third: &i32 = &v[2]; 
println!("The third element is {}", third);

// 2. get 方法：返回 Option<&T>，如果越界返回 None，非常安全
match v.get(100) {
    Some(third) => println!("The third element is {}", third),
    None => println!("There is no third element."),
}
```

### 3. 遍历 Vector
```rust
let mut v = vec![100, 32, 57];
for i in &mut v {
    *i += 50; // * 解引用运算符，用于获取引用的底层数据以修改它
}
```

## 二、HashMap：键值对映射

`HashMap<K, V>` 存储了键类型 `K` 到值类型 `V` 的映射。相当于 JS 的 `Map` 或 Go 的 `map`。

### 1. 创建与插入
```rust
use std::collections::HashMap; // 需要手动引入

let mut scores = HashMap::new();

scores.insert(String::from("Blue"), 10);
scores.insert(String::from("Yellow"), 50);

// 会覆盖已有的键
scores.insert(String::from("Blue"), 25); 
```

### 2. 只有键没有值时才插入 (神级 API: entry)
如果你想检查某个键是否存在，如果不存在才插入，存在就啥也不干，Rust 提供了一个极其优雅的写法：
```rust
scores.entry(String::from("Yellow")).or_insert(50);
scores.entry(String::from("Blue")).or_insert(50);
// 第一行啥也不干，因为 Yellow 已经存在。第二行会把 Blue: 50 插进去。
```

### 3. 所有权问题 (非常重要！)
对于实现了 `Copy` trait 的类型（如 `i32`），其值会被拷贝进 `HashMap`。对于像 `String` 这样拥有所有权的值，其**所有权将被转移 (Move)** 给哈希 map，之后你将无法使用原变量！
```rust
let field_name = String::from("Favorite color");
let field_value = String::from("Blue");

let mut map = HashMap::new();
map.insert(field_name, field_value);
// println!("{}", field_name); // ❌ 报错！所有权已经被 map 拿走了！
```
