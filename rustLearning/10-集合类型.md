# 10-集合类型：Vector、String、HashMap

> 一句话导读：Rust 的集合不是“能装东西”这么简单，它们同时体现了所有权、借用、堆分配和错误处理的日常用法。

Rust 标准库提供了很多集合类型。入门阶段最常用的是三类：

- `Vec<T>`：动态数组，连续存储，适合按顺序保存同类型数据。
- `String`：可增长、拥有所有权、UTF-8 编码的字符串。
- `HashMap<K, V>`：键值映射，适合按键快速查找。

它们都把数据放在堆上，变量本身通常只保存指针、长度、容量等元数据。

## 一、机制心智：长度、容量和所有权

集合的核心心智有三点：

1. 长度 `len` 是当前已有多少元素。
2. 容量 `capacity` 是当前分配的空间能容纳多少元素。
3. 插入拥有所有权的值时，值通常会被移动进集合。

```rust
fn main() {
    let mut numbers = Vec::with_capacity(2);

    println!("len={}, cap={}", numbers.len(), numbers.capacity());
    numbers.push(10);
    numbers.push(20);
    numbers.push(30);
    println!("len={}, cap={}", numbers.len(), numbers.capacity());

    let name = String::from("Ada");
    let mut names = Vec::new();
    names.push(name);

    // println!("{name}"); // name 已经被移动进 names
    println!("{:?}", names);
}
```

容量不够时，集合会重新分配更大的内存并搬迁元素。因此，如果你大致知道元素数量，`with_capacity` 可以减少扩容次数。

## 二、Vec：最常用的动态数组

`Vec<T>` 要求元素类型一致，元素在内存中连续排列。它适合日志列表、任务队列、批量数据、排序、过滤等场景。

```rust
fn main() {
    let mut scores = vec![80, 92, 75];

    scores.push(100);
    scores.sort();

    for score in &scores {
        println!("score={score}");
    }

    let average: f64 = scores.iter().sum::<i32>() as f64 / scores.len() as f64;
    println!("average={average:.1}");
}
```

### 读取元素：索引和 get

索引语法越界会 `panic`，`get` 返回 `Option<&T>`。

```rust
fn main() {
    let items = vec!["red", "green", "blue"];

    println!("second item: {}", items[1]);

    match items.get(10) {
        Some(value) => println!("found {value}"),
        None => println!("index out of range"),
    }
}
```

工程代码里，如果下标来自外部输入，优先使用 `get`。如果下标由代码逻辑保证有效，索引语法更直接。

### 修改元素：注意可变借用

```rust
fn main() {
    let mut prices = vec![100, 200, 300];

    for price in &mut prices {
        *price += 10;
    }

    println!("{prices:?}");
}
```

`price` 的类型是 `&mut i32`，要通过 `*price` 修改它指向的值。

### 常见错误：遍历时修改同一个 Vec

```rust
fn main() {
    let mut numbers = vec![1, 2, 3];

    // for n in &numbers {
    //     if *n < 3 {
    //         numbers.push(*n + 10); // 同时存在不可变借用和可变借用
    //     }
    // }

    let additions: Vec<i32> = numbers
        .iter()
        .filter(|n| **n < 3)
        .map(|n| *n + 10)
        .collect();

    numbers.extend(additions);
    println!("{numbers:?}");
}
```

修正思路：先基于不可变借用计算新数据，等借用结束后再修改原集合。

## 三、String：UTF-8 字节序列，不是字符数组

`String` 本质是 `Vec<u8>` 加上 UTF-8 约束。它可以增长，拥有内容所有权。字符串字面量 `&str` 则通常是借用。

```rust
fn main() {
    let mut message = String::from("hello");
    message.push(',');
    message.push_str(" Rust");

    let borrowed: &str = &message;
    println!("{borrowed}");
}
```

Rust 不允许用整数下标直接访问 `String`，因为 UTF-8 字符长度不固定。

```rust
fn main() {
    let text = String::from("你好 Rust");

    // let first = text[0]; // Rust 不支持这样索引 String

    for ch in text.chars() {
        println!("char={ch}");
    }

    for byte in text.bytes() {
        println!("byte={byte}");
    }
}
```

如果需要切片，必须切在合法的 UTF-8 边界上。

```rust
fn main() {
    let s = "你好";
    let first = &s[0..3];
    println!("{first}");

    // let broken = &s[0..1]; // 运行时 panic，不是合法字符边界
}
```

工程上处理自然语言文本时，`chars()` 也不等于“用户看到的字符”。复杂场景需要 Unicode 分词或 grapheme cluster 库。

## 四、HashMap：键值映射

`HashMap<K, V>` 用哈希表保存键值对。键必须实现 `Eq` 和 `Hash`。

```rust
use std::collections::HashMap;

fn main() {
    let mut scores = HashMap::new();

    scores.insert(String::from("blue"), 10);
    scores.insert(String::from("yellow"), 50);
    scores.insert(String::from("blue"), 25);

    let team = String::from("blue");
    match scores.get(&team) {
        Some(score) => println!("{team}: {score}"),
        None => println!("{team} has no score"),
    }
}
```

`insert` 会覆盖旧值。如果只想在键不存在时插入，用 `entry`。

```rust
use std::collections::HashMap;

fn main() {
    let mut counts = HashMap::new();
    let words = ["rust", "go", "rust", "java", "rust"];

    for word in words {
        let count = counts.entry(word).or_insert(0);
        *count += 1;
    }

    println!("{counts:?}");
}
```

`or_insert` 返回的是 `&mut V`，所以可以直接修改值。

## 五、常见错误与修正

### 1. 插入 String 后继续使用原变量

```rust
use std::collections::HashMap;

fn main() {
    let key = String::from("language");
    let value = String::from("Rust");

    let mut map = HashMap::new();
    map.insert(key.clone(), value);

    println!("inserted key: {key}");
    println!("{map:?}");
}
```

如果插入后还要使用原变量，可以 `clone`，或者改变数据结构让 `HashMap` 存借用。但存借用会引入生命周期约束，通常业务数据更适合让集合拥有数据。

### 2. 修改 HashMap 时同时借用同一个 map

```rust
use std::collections::HashMap;

fn main() {
    let mut map = HashMap::from([
        ("a", 1),
        ("b", 2),
    ]);

    // let a = map.get("a");
    // map.insert("c", 3);
    // println!("{a:?}");

    let a_value = map.get("a").copied();
    map.insert("c", 3);
    println!("{a_value:?}");
}
```

拿到 `&V` 后，不能在这个引用仍被使用时修改 `HashMap`。原因是插入可能触发扩容，导致内部地址变化。

### 3. collect 类型推不出来

```rust
fn main() {
    let doubled: Vec<i32> = [1, 2, 3].into_iter().map(|n| n * 2).collect();
    println!("{doubled:?}");
}
```

`collect` 可以收集成很多集合类型，编译器需要你通过变量类型或 `collect::<Vec<_>>()` 明确目标类型。

## 六、工程使用边界

`Vec` 适合：

- 顺序访问和批处理。
- 元素数量变化但类型一致。
- 需要排序、去重、过滤、分页的场景。

`String` 适合：

- 需要拥有和修改文本内容。
- 从文件、网络、用户输入读取文本。
- 作为结构体字段长期保存。

`&str` 适合：

- 函数参数只读文本，避免不必要分配。
- 字符串字面量和临时借用。

`HashMap` 适合：

- 按键查找优先于顺序。
- 计数、索引、缓存、分组。

不建议：

- 小规模固定字段数据滥用 `HashMap`，结构体更清晰。
- 对 `String` 做字节下标假设。
- 在性能敏感路径频繁无脑 `clone` 大集合。

## 七、结尾总结

集合类型是 Rust 所有权模型最常见的练习场。`Vec` 让你理解连续堆内存和借用冲突，`String` 让你理解 UTF-8 与拥有型文本，`HashMap` 让你理解键值所有权和可变访问。写集合代码时，优先问三个问题：谁拥有数据，借用持续到哪里，失败或缺失怎么表达。
