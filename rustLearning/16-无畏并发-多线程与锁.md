# 16-无畏并发：多线程与锁

> "并发编程在其他语言是高级专家干的活，但在 Rust 里，新手写多线程也不会导致系统崩溃（因为如果有 Bug 根本编译不过）。"

多线程最怕什么？数据竞争（两个线程同时写一个变量导致数据错乱）。Rust 的所有权和类型系统直接在**编译期**杀死了数据竞争。这被称为无畏并发（Fearless Concurrency）。

## 一、创建线程与 Move 闭包

使用 `thread::spawn` 创建一个新线程。新线程执行闭包里的代码。

```rust
use std::thread;
use std::time::Duration;

fn main() {
    let handle = thread::spawn(|| {
        for i in 1..10 {
            println!("hi number {} from the spawned thread!", i);
            thread::sleep(Duration::from_millis(1));
        }
    });

    // 主线程干自己的事
    for i in 1..5 {
        println!("hi number {} from the main thread!", i);
        thread::sleep(Duration::from_millis(1));
    }

    // 等待子线程结束，否则主线程一结束，子线程会被强行终止
    handle.join().unwrap();
}
```

如果线程里的闭包想要使用主线程的变量，必须强制**转移所有权（Move）**：
```rust
let v = vec![1, 2, 3];

let handle = thread::spawn(move || { // 👈 必须加 move
    println!("Here's a vector: {:?}", v);
});
// 这里以后再也不能用 v 了，因为已经被子线程拿走了
```
*为什么必须 move？* 因为主线程如果突然结束被销毁，或者修改了 `v`，子线程就会面临悬垂指针或数据竞争。编译器预判了你的失误！

## 二、互斥锁 (Mutex) 与多线程共享状态

如果想要多个线程**修改**同一个变量怎么办？使用互斥锁 `Mutex<T>`（Mutual Exclusion）。
规则很简单：想读写数据，先获取锁。写完，锁自动释放。

```rust
use std::sync::Mutex;

let m = Mutex::new(5);

{
    // lock() 会阻塞直到获取锁
    let mut num = m.lock().unwrap(); 
    *num = 6;
} // 离开作用域，锁自动被 drop 释放，不需要手动 unlock
```

## 三、线程安全的引用计数：Arc<T>

假设我们要把上面的 `Mutex` 传给 10 个线程。
还记得上一篇的 `Rc` 吗？它能让多个变量拥有同一个所有权。但是，`Rc` 的计数器不是线程安全的。
在多线程环境下，我们必须使用它的兄弟：**`Arc<T>` (Atomic Reference Counting)**。

**多线程状态共享黄金组合：`Arc<Mutex<T>>`**

```rust
use std::sync::{Arc, Mutex};
use std::thread;

fn main() {
    // 将计数器包在 Mutex 里，再包在 Arc 里
    let counter = Arc::new(Mutex::new(0));
    let mut handles = vec![];

    for _ in 0..10 {
        // 每个线程拿到一个 Arc 的克隆（增加强引用计数）
        let counter = Arc::clone(&counter);
        let handle = thread::spawn(move || {
            let mut num = counter.lock().unwrap();
            *num += 1;
        });
        handles.push(handle);
    }

    for handle in handles {
        handle.join().unwrap();
    }

    println!("Result: {}", *counter.lock().unwrap()); // 10
}
```

这就是 Rust 的伟大之处：如果你在这里不小心用错了 `Rc` 替代 `Arc`，或者忘记加 `Mutex`，**根本无法通过编译**。Rust 拒绝了带有隐患的并发代码进入生产环境。
