# 05-引用与借用

> 一句话导读：引用让你在不拿走所有权的情况下使用数据，而借用规则保证“读的时候没人乱改，改的时候没人偷读旧视图”。

所有权解决了谁释放内存的问题，但如果每次函数调用都移动所有权，代码会很笨重。引用和借用提供了更实用的方式：函数可以临时访问数据，却不负责释放数据。

## 核心心智模型

把所有权看成房产证，把引用看成钥匙：

- `T`：拿走值，获得所有权。
- `&T`：只读借用，可以有多个。
- `&mut T`：可写借用，同一时间只能有一个。

借用的核心不是语法上的 `&`，而是访问权限管理。Rust 编译器确保引用指向的数据始终有效，并且读写不会互相踩踏。

## 不可变引用

不可变引用使用 `&`：

```rust
fn main() {
    let text = String::from("hello");

    let len = calculate_length(&text);

    println!("'{text}' length = {len}");
}

fn calculate_length(value: &String) -> usize {
    value.len()
}
```

`calculate_length` 只借用 `text`，不拥有它。函数结束时，参数 `value` 离开作用域，但它只是引用，不会释放原始字符串。

不可变引用不能修改数据：

```rust
fn change(value: &String) {
    value.push_str(", world");
}
```

会报类似：

```text
error[E0596]: cannot borrow `*value` as mutable, as it is behind a `&` reference
```

修正：如果函数确实需要修改，使用可变引用。

## 可变引用

```rust
fn main() {
    let mut text = String::from("hello");

    append_world(&mut text);

    println!("{text}");
}

fn append_world(value: &mut String) {
    value.push_str(", world");
}
```

这里有三个条件：

1. 原变量必须是 `mut`。
2. 调用时传 `&mut text`。
3. 函数参数声明为 `&mut String`。

这让“谁可能修改数据”在函数签名里一眼可见。

## 借用规则

Rust 的核心借用规则：

在同一时间，你可以有多个不可变引用，或者一个可变引用，但不能两者同时存在。

多个不可变引用是安全的：

```rust
fn main() {
    let text = String::from("hello");

    let r1 = &text;
    let r2 = &text;

    println!("{r1}, {r2}");
}
```

两个可变引用不允许：

```rust
fn main() {
    let mut text = String::from("hello");

    let r1 = &mut text;
    let r2 = &mut text;

    println!("{r1}, {r2}");
}
```

错误类似：

```text
error[E0499]: cannot borrow `text` as mutable more than once at a time
```

修正：缩短第一个可变借用的使用范围。

```rust
fn main() {
    let mut text = String::from("hello");

    {
        let r1 = &mut text;
        r1.push_str(", Rust");
    }

    let r2 = &mut text;
    r2.push_str("!");

    println!("{text}");
}
```

不可变引用和可变引用也不能重叠：

```rust
fn main() {
    let mut text = String::from("hello");

    let read = &text;
    let write = &mut text;

    println!("{read}");
    write.push_str("!");
}
```

错误类似：

```text
error[E0502]: cannot borrow `text` as mutable because it is also borrowed as immutable
```

修正：让不可变引用最后一次使用发生在可变借用之前。

```rust
fn main() {
    let mut text = String::from("hello");

    let read = &text;
    println!("{read}");

    let write = &mut text;
    write.push_str("!");

    println!("{text}");
}
```

Rust 使用 non-lexical lifetimes，引用的有效期通常到最后一次使用为止，而不是机械地到代码块结尾。

## 为什么编译器这么限制

这些规则防的是引用失效和数据竞争。

假设存在一个不可变引用 `r1` 指向 `String` 的内部缓冲区，同时另一个可变引用 `r2` 调用了 `push_str`。如果字符串容量不够，`push_str` 可能重新分配堆内存，原来的指针失效。此时 `r1` 如果继续使用，就会读到无效内存。

再看多线程场景：两个可变引用同时修改同一块数据，会产生数据竞争。Rust 的借用规则在单线程代码里也严格执行，是因为同一套模型可以自然扩展到并发安全。

编译器不是不让你写复杂代码，而是要求你明确访问关系：读共享，写独占。

## 悬垂引用

Rust 不允许返回指向局部变量的引用：

```rust
fn main() {
    // let value = dangle();
}

// fn dangle() -> &String {
//     let text = String::from("hello");
//     &text
// }
```

如果允许这段代码，`text` 在函数结束时已经释放，返回的引用会指向无效内存。修正方式是返回拥有所有权的值：

```rust
fn no_dangle() -> String {
    let text = String::from("hello");
    text
}

fn main() {
    let value = no_dangle();
    println!("{value}");
}
```

## 可编译综合示例

```rust
fn normalize_name(name: &mut String) {
    let trimmed = name.trim().to_lowercase();
    name.clear();
    name.push_str(&trimmed);
}

fn display(label: &str, value: &String) {
    println!("{label}: {value}");
}

fn main() {
    let mut name = String::from("  Alice  ");

    display("before", &name);
    normalize_name(&mut name);
    display("after", &name);
}
```

这段代码展示了只读借用和可变借用的顺序：先读，再独占修改，再读。每个借用都没有重叠冲突。

## 常见编译器错误与修正

### 错误 1：不可变引用上调用可变方法

```text
error[E0596]: cannot borrow as mutable
```

修正：把参数改成 `&mut T`，调用方也传 `&mut`，并保证原变量是 `mut`。

### 错误 2：同时有两个可变引用

```text
error[E0499]: cannot borrow as mutable more than once at a time
```

修正：缩短第一个可变引用作用范围，或者把两次修改合并到一个函数里。

### 错误 3：读引用还在用时尝试写

```text
error[E0502]: cannot borrow as mutable because it is also borrowed as immutable
```

修正：调整代码顺序，让读引用最后一次使用结束后再创建可变引用。

## 工程判断

函数签名要表达所有权意图：

- 只读取字符串内容，优先用 `&str`。
- 只读取集合，优先用 `&[T]`。
- 需要修改调用方数据，用 `&mut T`。
- 需要保存或跨线程持有数据，通常要拥有 `T`，而不是借用。

不要为了绕过借用检查把所有东西都 `clone`。`clone` 可以用，但应该是业务上需要独立副本，而不是为了逃避访问关系设计。

## 结尾总结

引用让 Rust 代码不必频繁移动所有权；借用规则保证共享读取和独占写入不会冲突。理解“读共享，写独占”之后，很多编译器错误就不再神秘，它们是在阻止悬垂引用、引用失效和数据竞争。
