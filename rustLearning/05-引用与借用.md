# 05-引用与借用

> "你的车你可以借给别人开（借用），但车的户口本（所有权）还是你的。"

在上一篇中，我们发现把变量传给函数就会失去所有权，这太麻烦了。Rust 提供了**引用（References）**功能，它允许你使用值但不获取其所有权。使用引用的行为，在 Rust 中被称为**借用（Borrowing）**。

## 一、不可变引用（Immutable Reference）

获取引用使用 `&` 符号。

```rust
fn main() {
    let s1 = String::from("hello");

    // 我们传的是 &s1，意思是创建一个指向 s1 的引用，但不拿走所有权
    let len = calculate_length(&s1);

    // s1 依然可用！
    println!("The length of '{}' is {}.", s1, len);
}

// 参数声明也必须是引用类型 &String
fn calculate_length(s: &String) -> usize {
    s.len()
} // s 离开作用域，但因为它并不拥有它指向的数据，所以什么都不会发生，数据不会被 drop。
```

默认情况下，变量是不可变的，引用也是一样。**你不能通过不可变引用去修改数据！**

## 二、可变引用（Mutable Reference）

如果你想借别人的车，顺便给车换个颜色（修改数据），你需要**可变借用**。

```rust
fn main() {
    let mut s = String::from("hello"); // 1. 变量本身必须是 mut 的

    change(&mut s); // 2. 借出时使用 &mut

    println!("{}", s); // 输出 "hello, world"
}

fn change(some_string: &mut String) { // 3. 接收时声明类型为 &mut
    some_string.push_str(", world");
}
```

## 三、数据竞争与借用的两大铁律（最容易被编译器骂的地方）

可变引用有一个巨大的限制，这也是 Rust 保证内存和并发安全的核心：

**借用规则：在任意给定的时间，你要么只能有一个可变引用，要么只能有多个不可变引用。**
换言之：**读写锁原则**。

### 限制 1：同一作用域内，不能同时存在两个可变引用
```rust
let mut s = String::from("hello");

let r1 = &mut s;
// let r2 = &mut s; // ❌ 报错！不能两次作为可变借用借出

// println!("{}, {}", r1, r2); 
```
*为什么？* 如果有两个指针同时指向一块内存，且都尝试修改它，在多线程环境下会引发灾难性的**数据竞争（Data Race）**。Rust 在编译期就把这个隐患掐死了。

### 限制 2：不能在有不可变引用的同时，存在可变引用
```rust
let mut s = String::from("hello");

let r1 = &s; // 没问题，不可变借用
let r2 = &s; // 没问题，多个不可变借用互不影响
// let r3 = &mut s; // ❌ 报错！已经有不可变借用了，不能再搞个可变借用

// println!("{}, {}, and {}", r1, r2, r3);
```
*为什么？* `r1` 和 `r2` 预期数据不会变。如果 `r3` 偷偷把数据改了，甚至是引发了底层堆内存的重新分配导致指针失效，`r1` 和 `r2` 就崩溃了。

> **提示**：引用的作用域从声明的地方开始一直持续到**最后一次使用**的地方结束。所以只要后面不再使用 `r1` 和 `r2`，再去声明可变借用也是可以的（Non-Lexical Lifetimes 机制）。

## 四、悬垂引用 (Dangling References)

在 C++ 里，返回一个指向局部变量的指针会导致灾难（悬垂指针），因为局部变量在函数结束时被销毁了。在 Rust 中，编译器确保引用永远不会变成悬垂状态。

```rust
fn main() {
    // let reference_to_nothing = dangle(); 
}

// ❌ 报错：expected named lifetime parameter
// fn dangle() -> &String { 
//     let s = String::from("hello");
//     &s // 返回指向 s 的引用
// } // 这里 s 离开作用域，被丢弃，内存被释放。引用指向了无效的内存！
```
Rust 编译器直接拒绝编译这段代码。要修复它，必须直接返回值转移所有权，而不是返回引用：
```rust
fn no_dangle() -> String {
    let s = String::from("hello");
    s // 转移所有权给调用者
}
```
