# 14-闭包与迭代器

> 一句话导读：闭包让行为可以被当作值传递，迭代器让集合处理变成惰性的流水线，两者共同构成 Rust 中高效、可组合的数据处理方式。

Rust 并不强迫你写函数式风格。`for` 循环仍然很好用。但闭包和迭代器能让很多“筛选、转换、聚合”的逻辑更紧凑，而且通常不会牺牲性能。

这一篇要抓住两件事：

- 闭包会捕获环境，所以必须遵守所有权和借用规则。
- 迭代器默认惰性，只有被消费时才真正执行。

## 一、机制心智：闭包是带环境的匿名函数

函数只能使用参数和全局可见内容。闭包可以捕获定义处附近的变量。

```rust
fn main() {
    let threshold = 10;
    let is_large = |n: i32| n > threshold;

    println!("{}", is_large(8));
    println!("{}", is_large(12));
}
```

`|n: i32| n > threshold` 是闭包。它接收 `n`，同时借用了外部变量 `threshold`。

闭包参数和返回值类型通常可以推断，但一旦编译器第一次推断出来，后续调用就要一致。

```rust
fn main() {
    let echo = |value| value;

    let a = echo(String::from("hello"));
    // let b = echo(123); // 第一次调用后，value 已被推断为 String

    println!("{a}");
}
```

如果你需要真正的泛型行为，通常写泛型函数，而不是指望一个闭包对所有类型都适配。

## 二、闭包捕获环境的三种方式

闭包会根据使用方式自动选择捕获方式。

### 1. 不可变借用

```rust
fn main() {
    let prefix = String::from("[info]");
    let log = |message: &str| println!("{prefix} {message}");

    log("start");
    log("done");
    println!("prefix is still usable: {prefix}");
}
```

闭包只读取 `prefix`，所以不可变借用即可。

### 2. 可变借用

```rust
fn main() {
    let mut count = 0;

    let mut inc = || {
        count += 1;
        count
    };

    println!("{}", inc());
    println!("{}", inc());

    // println!("{count}"); // inc 仍然持有对 count 的可变借用
    drop(inc);
    println!("final count={count}");
}
```

闭包修改外部变量时，需要可变借用。只要闭包还存在，这个可变借用就可能持续存在。

### 3. move 获取所有权

`move` 会把闭包用到的外部变量移动进闭包。多线程和延迟执行场景很常见。

```rust
use std::thread;

fn main() {
    let message = String::from("hello from thread");

    let handle = thread::spawn(move || {
        println!("{message}");
    });

    handle.join().expect("thread should finish");
}
```

没有 `move` 时，线程闭包可能借用当前栈帧里的变量，而当前函数可能在线程结束前返回，所以编译器不允许。

## 三、Fn、FnMut、FnOnce

闭包会自动实现以下一个或多个 trait：

- `Fn`：只读捕获变量，可多次调用。
- `FnMut`：会修改捕获状态，需要可变调用。
- `FnOnce`：会消费捕获的值，只能调用一次。

```rust
fn call_twice<F>(mut f: F)
where
    F: FnMut(),
{
    f();
    f();
}

fn main() {
    let mut count = 0;

    call_twice(|| {
        count += 1;
        println!("count={count}");
    });
}
```

如果闭包内部把捕获的 `String` 移出去了，它通常只能实现 `FnOnce`。

```rust
fn consume_once<F>(f: F)
where
    F: FnOnce() -> String,
{
    println!("{}", f());
}

fn main() {
    let name = String::from("Ada");

    consume_once(|| name);
}
```

理解这三个 trait，能帮助你读懂很多标准库和异步框架的 API。

## 四、迭代器：惰性流水线

迭代器的核心 trait 是 `Iterator`。

```rust
trait Iterator {
    type Item;
    fn next(&mut self) -> Option<Self::Item>;
}
```

每次调用 `next`，返回一个 `Some(item)`；没有元素时返回 `None`。

```rust
fn main() {
    let numbers = vec![1, 2, 3];
    let mut iter = numbers.iter();

    println!("{:?}", iter.next());
    println!("{:?}", iter.next());
    println!("{:?}", iter.next());
    println!("{:?}", iter.next());
}
```

迭代器是惰性的。只写 `map`、`filter` 不会执行任何处理，必须有消费动作。

```rust
fn main() {
    let numbers = vec![1, 2, 3, 4, 5];

    let result: Vec<i32> = numbers
        .iter()
        .filter(|n| **n % 2 == 1)
        .map(|n| *n * 10)
        .collect();

    println!("{result:?}");
}
```

`collect` 是消费者。其他常见消费者还有 `sum`、`count`、`any`、`all`、`find`、`fold`。

## 五、iter、iter_mut、into_iter

这三个方法的区别是 Rust 集合处理的重点。

```rust
fn main() {
    let mut numbers = vec![1, 2, 3];

    for n in numbers.iter() {
        println!("read {n}");
    }

    for n in numbers.iter_mut() {
        *n *= 10;
    }

    for n in numbers.into_iter() {
        println!("owned {n}");
    }

    // println!("{numbers:?}"); // numbers 已被 into_iter 消费
}
```

心智模型：

- `iter()` 产出 `&T`，只读借用。
- `iter_mut()` 产出 `&mut T`，可变借用。
- `into_iter()` 产出 `T`，拿走所有权。

## 六、工程例子：解析日志行

下面是一个完整可编译例子：从多行日志里筛出错误行，提取消息，限制数量。

```rust
fn error_messages(input: &str) -> Vec<&str> {
    input
        .lines()
        .filter(|line| line.starts_with("ERROR "))
        .filter_map(|line| line.split_once(": ").map(|(_, message)| message))
        .take(3)
        .collect()
}

fn main() {
    let logs = "\
INFO boot
ERROR auth: invalid token
WARN cache slow
ERROR db: timeout
ERROR api: bad request
ERROR queue: full";

    let errors = error_messages(logs);
    println!("{errors:?}");
}
```

`filter_map` 适合“过滤 + 转换”同时发生的场景。这里返回 `Vec<&str>`，每个元素都借用自输入 `logs`，没有分配新的字符串。

## 七、常见错误与修正

### 1. 忘记消费迭代器

```rust
fn main() {
    let numbers = vec![1, 2, 3];

    // numbers.iter().map(|n| println!("{n}")); // 不会执行

    numbers.iter().for_each(|n| println!("{n}"));
}
```

`map` 是适配器，返回新迭代器；`for_each` 是消费者，会真正执行。

### 2. `filter` 闭包里的引用层级看不懂

```rust
fn main() {
    let numbers = vec![1, 2, 3, 4];

    let evens: Vec<i32> = numbers
        .iter()
        .copied()
        .filter(|n| n % 2 == 0)
        .collect();

    println!("{evens:?}");
}
```

`iter()` 产出 `&i32`，`filter` 的闭包参数又会收到引用，所以容易出现 `&&i32`。用 `.copied()` 或 `.cloned()` 可以把小的 Copy 值变成拥有型值，让后续代码更清楚。

### 3. 闭包捕获导致后续不能修改

```rust
fn main() {
    let mut values = vec![1, 2, 3];

    {
        let print_values = || println!("{values:?}");
        print_values();
    }

    values.push(4);
    println!("{values:?}");
}
```

让闭包的作用域尽早结束，后续就可以重新可变借用原变量。

### 4. collect 目标类型不明确

```rust
use std::collections::HashSet;

fn main() {
    let unique: HashSet<i32> = [1, 1, 2, 3].into_iter().collect();
    println!("{unique:?}");
}
```

`collect` 的目标集合类型要明确，否则编译器不知道该收集成 `Vec`、`HashSet` 还是别的类型。

## 八、性能：零成本但不是零思考

Rust 迭代器通常会被 LLVM 优化成和手写循环相近的机器码。链式调用本身不是性能问题。

但工程上仍然要注意：

- `collect` 会分配新集合，不需要中间集合时不要过早 `collect`。
- `clone` 大对象可能很贵，别为了绕开借用随意克隆。
- 闭包捕获大对象并 `move` 可能转移所有权，确认这就是你想要的。
- 对热路径可以用基准测试验证，而不是凭感觉改成手写循环。

## 九、工程使用边界

适合用迭代器链：

- 数据处理是线性的：过滤、映射、聚合。
- 每一步都短小清晰。
- 想利用惰性避免中间集合。
- 需要表达“从输入到输出”的转换管道。

适合用普通 `for` 循环：

- 控制流复杂，有多处分支、提前退出、状态机。
- 需要详细错误处理和日志。
- 迭代器链已经长到难以阅读。
- 可变状态很多，强行函数式会更绕。

闭包适合：

- 回调、排序规则、过滤条件、延迟计算。
- 把少量行为传给通用函数。
- 捕获上下文能明显减少样板代码。

## 十、结尾总结

闭包和迭代器是 Rust 日常代码里非常实用的组合。闭包把行为变成值，迭代器把集合处理变成惰性流水线。真正要掌握的是所有权差异：`iter` 借用，`iter_mut` 可变借用，`into_iter` 消费所有权；闭包则根据读取、修改、移动自动落到 `Fn`、`FnMut`、`FnOnce`。理解这些规则后，链式代码既可以清楚，也可以高效。
