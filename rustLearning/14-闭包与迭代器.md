# 14-闭包与迭代器

> "不要写 for 循环了，用迭代器把数据处理变成流水线作业吧！"

闭包（Closures）和迭代器（Iterators）是 Rust 从函数式编程中借鉴来的两大杀器。它们不仅写起来极其优雅，而且因为 Rust 的"零成本抽象"特性，它们的速度和手写底层 for 循环一样快！

## 一、闭包 (Closures)

闭包是能**捕获其所在环境**的匿名函数。

```rust
fn main() {
    let x = 4;
    // 使用 || 定义参数，大括号里是函数体
    let equal_to_x = |z| z == x; 

    // 如果只有一行，大括号也可以省了
    // let equal_to_x = |z| z == x; 

    let y = 4;
    assert!(equal_to_x(y)); // x 被闭包"捕获"并在内部使用了
}
```

### 闭包捕获环境的三种方式
既然闭包能捕获外部变量，就必须遵守借用规则：
1. **获取所有权（Move）**：如果外部变量没有实现 Copy，且你在闭包里获取了它，闭包会拿走所有权。如果你强制要求闭包拿走所有权，可以在前面加 `move` 关键字（在多线程中极常用）。
2. **可变借用（`&mut`）**：在闭包里修改外部变量。
3. **不可变借用（`&`）**：只读外部变量（如上面的例子）。

## 二、迭代器 (Iterators)

在 Rust 中，迭代器是惰性的（Lazy）。意思是说，除非你调用了消耗迭代器的方法，否则什么都不会发生。

```rust
let v1 = vec![1, 2, 3];
let v1_iter = v1.iter(); // 这里啥也没干，只是创建了迭代器

for val in v1_iter {     // 这里才开始真正遍历
    println!("Got: {}", val);
}
```

### 生产消费流水线：适配器与消耗器

- **迭代器适配器 (Iterator adaptors)**：把一个迭代器变成另一个迭代器（依然是惰性的），比如 `map`，`filter`。
- **消耗器 (Consuming adaptors)**：调用它们会耗尽迭代器，并返回一个最终值，比如 `collect`，`sum`。

来看一段极具美感的函数式代码：
```rust
let v1: Vec<i32> = vec![1, 2, 3, 4, 5];

// 把所有的奇数挑出来，然后乘以 10，最后收集成一个新的 Vector
let result: Vec<i32> = v1.into_iter() // into_iter 会拿走 v1 的所有权
    .filter(|x| x % 2 != 0)           // 过滤：只要奇数
    .map(|x| x * 10)                  // 映射：乘 10
    .collect();                       // 消耗并收集结果！必须告诉编译器类型 Vec<i32>

println!("{:?}", result); // [10, 30, 50]
```

## 三、性能神话：零成本抽象

你可能会觉得上面的 `filter` 加 `map` 会产生多次循环，比手写一个普通的 `for` 循环里面加上 `if` 和乘法要慢。
**错！**

Rust 编译器（借助于 LLVM）会把上面这种迭代器链式调用，**直接展开编译成了与最优手写汇编代码相同的机器码**，没有一点额外的循环或函数调用开销。放心大胆地用迭代器吧！
