# 云服务与互联网基础常识系列写作计划

> 这个系列面向准备做网站、App、SaaS、独立开发、云部署的人。
> 它不追求某个厂商的操作手册,而是把 DNS、CDN、SSL、SLA、Serverless、对象存储、计费、安全、监控这些经常听到但容易混淆的概念讲清楚。
> 目标是:看到一个云服务免费套餐、架构图、账单、故障公告、安全配置时,能知道它在说什么、影响什么、风险在哪里。

---

## 设计思路

五层结构,从访问一个网站开始,一路讲到上线、稳定、安全和成本:

```text
互联网入口(DNS / 域名 / HTTPS / CDN)
   -> 云资源抽象(服务器 / 存储 / 数据库 / Serverless)
   -> 稳定性与运维(SLA / 监控 / 日志 / 备份 / 灾备)
   -> 安全与权限(WAF / DDoS / IAM / 密钥 / 合规)
   -> 计费与选型(免费额度 / 带宽 / 请求数 / 成本估算 / 厂商比较)
```

本系列不写成厂商文档翻译,而是用真实问题驱动:

- Cloudflare 免费套餐到底免费在哪里?
- 99.9% SLA 是不是很高?
- CDN 为什么能加速,什么时候反而会出问题?
- 对象存储为什么便宜,和网盘、服务器硬盘有什么区别?
- Serverless 是不是完全不用管服务器?
- 免费额度用完会发生什么?
- 怎么看懂云服务账单?

每篇都要回答四个问题:

1. 这个概念解决什么问题
2. 它在真实系统里放在哪一层
3. 免费套餐或小团队使用时最容易踩什么坑
4. 什么时候需要升级到更专业的方案

## 写作模板

每篇使用固定结构:

```text
1. 一句话解释
   - 这个词到底是什么意思
   - 它最容易和什么概念混淆

2. 放在系统哪里
   - 用户请求链路中的位置
   - 和相邻组件的关系

3. 常见套餐限制
   - 免费额度通常限制什么
   - 超额后可能发生什么

4. 小团队建议
   - 什么时候够用
   - 什么时候该升级
   - 最低限度的安全和成本护栏
```

写作原则:

- 多用真实场景,少堆术语定义。
- 每篇尽量给出一个小例子,例如一次域名解析、一次 CDN 回源、一次账单超额。
- 不绑定单一厂商,但可以用 Cloudflare、Vercel、AWS、阿里云、腾讯云等做对照。
- 不写过时价格表,价格和额度只写查看方法；需要具体数字时标注查询日期。
