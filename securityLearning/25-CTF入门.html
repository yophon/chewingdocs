CTF 入门:Web / Pwn / Reverse / Crypto / Misc 五大类心智 ​

到这一篇,前面 04-23 把密码学、Web、二进制、渗透方法论都过了一遍——但读者多半会有同一个困惑:这些东西看着都懂,但放到一道题里我还是不会做。安全知识点的特点是碎、深、且彼此咬合——XSS 单讲十分钟讲得清楚,但拿到一道实际题"找出 flag",你得同时判断这是 XSS、CSP 怎么绕、能不能 SSRF 接 oauth、token 在哪儿——这种"把碎知识点串成解题路径"的能力,只能靠刷 CTF 练出来。这一篇不教任何新漏洞,只回答一个问题:作为想长真功夫的工程师,CTF 是什么样的训练场,我从零开始怎么用好它。

一句话先记住:CTF = 解谜版的安全学习——把"教科书漏洞"变成"考场上要在 48 小时内组合出 exploit 的实操题"。CTF 不是"打黑客"——所有题目都跑在出题方搭的靶机里,flag 是一串特定格式的字符串(flag{...} / CTF{...}),拿到就赢,和真实黑产、未授权渗透是两个世界。它的真实价值有三个:让你的零散知识被迫串起来、给你一个能炫耀的硬通货成绩单、把你接到一个全球最活跃的技术社区——大厂安全岗、红队、漏洞研究员的招聘里,「打过哪些 CTF / 是哪个战队的」是比简历项目硬十倍的信号。

一、CTF 是什么 ​

1.1 一个最小的题目长什么样 ​

题目:一个网站 http://challenge.ctf.example/
描述:Find the flag.
flag 格式:flag{...}
分值:Web - 200pt

就这么多。没有提示,没有路线图,你要自己侦察、自己找漏洞、自己利用,直到拿到一段以 flag{ 开头的字符串提交到记分板。这就是 CTF 题的标准形态——像高考数学的应用题,但场景全是"模拟一次真实攻击"。

flag 的位置可能在数据库表里(SQL 注入)、服务器文件里(LFI / RCE)、二进制运行后输出(Pwn)、图片隐写层(Misc)、或者得逆向加密算法才能解出(Crypto)。所有题的本质都是"通过解决一个安全问题,证明你具备某种能力"——出题人把 flag 藏在"如果你不懂这个漏洞,就拿不到"的地方。

1.2 CTF 不是什么 ​

新人最大的误解是**"CTF 等于黑客"**——CTF 题目永远在授权环境(靶机是出题方提供的,IP/域名赛方圈定,去打靶机外的任何东西会被取消资格甚至追责),flag 也不是真有破坏性的产物(就是一串特定格式的字符串,删的永远是备份过的虚拟机),也不教社会工程 / 钓鱼(所有题目都是"纯技术对纯靶机")。

本篇所有讨论都默认目标是合法的 CTF 平台、自己搭的靶机、或者明确授权的环境——这一点和第 21-24 篇一致,不再重复声明。

1.3 CTF 的训练价值在哪儿 ​

工程师角度看,CTF 提供了四样别处买不到的东西:强制"知识点串联"(一道题往往把 10/11/13 篇的知识全用上)、逼你"动手出活"(不能只会讲原理,必须真写 payload / 绕 WAF / 拿数据)、训练时间压力下的判断(48 小时一场,必须快速判断"这个方向值不值得继续投入"——这恰是真实漏洞挖掘的核心能力)、构建一个 portfolio(平台账号成绩、CTFtime 战队排名、社交媒体的 writeup,都是面试官能直接看的硬通货)。

二、三种赛制 ​

不同 CTF 比赛长得很不一样,先认清自己在打的是哪种。

2.1 Jeopardy(解题赛):新人 99% 打的是这种 ​

形态:一个网页给你一堆题目卡片,按分类(Web / Pwn / Crypto / ...)和难度(100 / 200 / 500 pt)分布,各题独立,先做哪个都行。比赛通常 24-48 小时,做出来一道交 flag 加分,最终累计分高的赢。

┌─────────────────────────────────────────────┐
│  Web 100    Web 200    Web 500              │
│  Crypto 100 Crypto 300 Crypto 500           │
│  Pwn 100    Pwn 400                          │
│  Misc 50    Misc 200    Misc 300            │
└─────────────────────────────────────────────┘
点开题目 → 下载附件 / 连靶机 → 解 → 交 flag

为什么新人都从这里开始:节奏自由(可以单人打),没人来打你(纯防守 0 攻击),做不出的题可以跳。picoCTF / Google CTF / HackTheBox CTF / 强网杯线上初赛全是这种。

2.2 Attack-Defense(攻防赛):资深战队的"地狱模式" ​

形态:每个战队拿到一台完全一样的有漏洞服务器(里面跑了几个故意写漏的服务),比赛期间:

  • 攻:挖漏洞,打别人的服务器,从别人的 flag 文件里抢走 flag
  • 守:自己服务器被打也要给 flag,但你要给服务打 patch 让别人攻不进来
  • 保活:服务还得保持可用(出题方的健康检查机器人会定期访问,挂了扣分)
全队 8 台一样的服务器,每台跑同一个有漏洞的服务
   ↓
你打别人的(攻)+ 守自己的(patch)+ 让自己服务别挂(运维)
   ↓
每 5 分钟一轮"flag 轮换",每轮按战况发分

为什么超难:三件事并行,任何一件做不好都崩——没攻击力分数刷不起来,patch 错了服务挂掉直接负分。DEF CON CTF Finals / iCTF / 强网杯线下决赛是这种。新人不要碰,先在 Jeopardy 把单兵打稳了再说。

2.3 King of the Hill(KoH 山丘之王):折中款 ​

形态:一台/几台所有战队共享的有漏洞机器,谁能持续控制谁拿分——你打进去后留下自己的标识(类似"占山为王"),保持时间越久分越高,但其他队随时可能把你踢下来。

特点:攻防一体但没自己要守的服务器,节奏紧凑,适合中等水平战队。HackTheBox / TryHackMe 偶尔有 KoH 类型房间。

2.4 三种赛制怎么选 ​

你的水平适合的赛制
完全零基础Jeopardy 入门题(picoCTF Easy)
能稳定做 200 pt 题Jeopardy 中难度赛(各种线上赛)
战队有 5+ 个稳定打手Attack-Defense 线上预选
你是大佬 / 国家队选手DEF CON Finals(顶级线下)

本篇剩下的内容都以 Jeopardy 为主——这是 95% 工程师起步的形态。

三、五大类心智 ​

CTF 的题目分类全世界都几乎一样,就五大类。每一类背后是一种独立的"工程师思考模式"——学会"切换心智"是 CTF 的核心技能。

3.1 Web:从浏览器视角找漏洞 ​

心智:这是一个跑在浏览器 + 服务器之间的应用,我要找配置 / 代码 / 协议层的破口。

典型题型:

  • SQL 注入(题目暗示有登录框 → 试 ' OR 1=1--)
  • XSS 拿管理员 cookie(题目里有 bot 会"访问你提交的链接")
  • SSRF 打内网(题目给一个"输入 URL 我帮你抓"的功能)
  • 文件上传 RCE(传 .php,绕黑名单)
  • JWT 弱算法 / 权限绕过

入门动作(对应第 9-15 篇):

1. 看到网页第一步:F12 打开 DevTools,看请求 / cookie / JS
2. 用 dirsearch / ffuf 跑目录,看有没有 /admin /.git /.env
3. 用 Burp Suite 抓所有请求,看参数 / Header / Body
4. 注入点测试:'  "  <  ../  ${jndi:  {{7*7}}  全试一遍

典型 PoC 节奏(伪代码):

http
POST /login HTTP/1.1
Host: challenge.ctf
Content-Type: application/x-www-form-urlencoded

username=admin' OR 1=1-- &password=x

→ 登进去 → 翻管理后台 → 找到"导出报表"接口 → 试 SSRF 打 http://localhost:6379 → redis 未授权 → 写 SSH key → 拿 shell → 找 flag。这就是一道典型的"Web 综合题"利用链。

Web 题的甜区:最容易入门,工具链最成熟,做出来最有成就感——绝大多数人从 Web 开始。

3.2 Pwn:让二进制程序按你的剧本走 ​

心智:这是一个跑在远端的二进制服务,我要找它的内存破坏漏洞,让它执行我塞进去的 shellcode 或 ROP 链。

典型题型(对应第 16-20 篇):

  • 栈溢出 → 改返回地址 → 跳到 system("/bin/sh")(ret2libc)
  • 格式化字符串 → 任意读写
  • 堆 UAF / double free → tcache / fastbin 攻击
  • 沙箱(seccomp)题:/bin/sh 被禁,要 orw(open / read / write)读 flag 文件

入门动作:

1. file ./pwn  → 看 32/64 位、动态/静态
2. checksec ./pwn  → 看开了哪些缓解(NX / PIE / Canary / RELRO)
3. IDA / Ghidra 看反编译,找漏洞函数(gets / read 超长 / strcpy)
4. 本地调试 pwndbg,确认偏移
5. 用 pwntools 写 exploit:
   from pwn import *
   p = remote("ctf.host", 1337)
   p.sendline(b"A" * offset + p64(ret_address))
   p.interactive()

Pwn 题的反直觉点:远程靶机 libc 版本和本地常常不一样 → 题目通常会给 libc.so,你要本地 patchelf 加载它的 libc 调试;ROP gadget 用 ROPgadget --binary ./pwn | grep "pop rdi" 找;堆题 60% 是"如何把 unsorted bin chunk 改成 main_arena 地址泄露 libc base"——堆题的本质是"操控 ptmalloc 内部数据结构"。

Pwn 题的甜区:和真实 0day 挖洞最接近的训练——做过 100 道堆题再去看 V8 / Chromium CVE,会发现思路一模一样。

3.3 Reverse(逆向):静态/动态分析一个不给源码的程序 ​

心智:有人写了一个程序,只给我编译产物,我要看懂它的逻辑,找到"什么输入会让它说 correct"。

典型题型:

  • 一个 .exe / .elf,运行后让你输入,输对就吐 flag
  • 程序里有混淆 / 加密 / 反调试,要绕过
  • Android APK / iOS IPA 逆向
  • 自定义虚拟机(VM)的字节码题(出题人写了一个小 VM 跑加密逻辑,你要先逆 VM 指令集再逆字节码)

入门动作:

1. file / strings 看基本信息
2. IDA Free / Ghidra / radare2 / Cutter 反编译看 main
3. 找 "if (strcmp(input, "xxx") == 0)" 之类的对比点
4. 复杂逻辑 → 上动态 debug(x64dbg / gdb / lldb)
5. 反编译 + 调试 → 还原算法 → 写一个反向脚本算 flag

典型场景:

c
// 反编译看到的伪代码
for (i = 0; i < strlen(input); i++) {
    if ((input[i] ^ 0x42) != target[i]) {
        puts("Wrong");
        exit(0);
    }
}
puts("Correct!");
print_flag();

Python 一行还原:

python
target = b"\x37\x29\x2b..."   # 从二进制里 dump
flag = bytes([b ^ 0x42 for b in target])

Reverse 题的甜区:对编译器 / 汇编 / 二进制格式的理解,反过来让你写代码时知道"编译器把我的高级语言变成了什么"——是写底层 / 性能敏感代码的极好训练。

3.4 Crypto:密码学题的两个流派 ​

心智:这是一道数学题或者实现陷阱题,不是让你硬猜密钥。

两个流派(对应第 4-8 篇):

(a) 经典数学题:RSA(e 太小、n 公约数、低私钥指数 Wiener)、椭圆曲线(不合规参数 / 不安全随机数)、LLL 格 / Coppersmith、Diffie-Hellman 弱素数。

(b) 实现陷阱题:AES-ECB(相同明文块产生相同密文块 → 切块攻击 / Penguin 图)、AES-CBC Padding Oracle(任意解密,详见第 8 篇)、弱随机数(time() 当种子 → 暴破)、时序攻击(逐字节比较 → 响应时间侧信道泄露)。

入门动作:识别用了什么算法 / 模式 → 套已知攻击(RSA + 小 e + 短消息直接开 e 次方根;AES-ECB 看重复块;"服务器告诉你解密成功否"→ padding oracle)→ 工具:SageMath(数论 + 格运算神器)、PyCryptodome(写攻击脚本)、RsaCtfTool(RSA 常见弱点一把梭)。

Crypto 题的甜区:让你真正理解密码学的"工程陷阱比数学难"——你看完不会再"自己写加密",会看 KMS / HSM / libsodium 像看救命药。

3.5 Misc:把"其他一切"装在一起的杂物间 ​

心智:这道题不属于上面四类,得用通用技术功底拼出来。子类:取证(磁盘镜像 / 内存 dump / pcap)、隐写(图片 LSB / 音频频谱 / PDF 流 / 文档元数据)、网络(pcap 重建 SMB / HTTP)、杂七杂八(Excel 宏 / QR 码 / 摩尔斯码 / ELF 异常结构)。

入门动作:file / binwalk / exiftool / strings 全跑一遍 → 图片可疑就 zsteg / stegsolve / Audacity 频谱 → pcap 进 Wireshark 看协议 → 内存 dump 用 Volatility → 磁盘镜像用 Autopsy。Misc 题的甜区:广度训练——学到的工具(Wireshark / 取证)在真实蓝队 / DFIR 工作里日常用得到。

3.6 五类对比 ​

类别入门门槛工程师收益真实工作场景
Web低直接 = 后端 / 全栈安全能力应用安全工程师 / 渗透测试员
Pwn高系统编程 + 漏洞研究底子漏洞研究员 / 内核 / 浏览器
Reverse中编译器 + 二进制理解恶意软件分析 / 反外挂 / 红队
Crypto中(数学背景加分)加密工程 + 协议设计能力密码学库工程师 / 区块链
Misc低工具链广度 + 取证蓝队 / 应急响应 / DFIR

建议:新人从 Web + Misc 起步,跑通"做出题"的正反馈;熟了再选一个深耕。Pwn / Reverse / Crypto 都需要长期投入,别想着五个全精——CTF 圈里专精一类的选手永远比五边形战士多。

四、入门路径:从零到打第一场 ​

CTF 入门最大的坑是**"看了一堆理论但从来不动手"——下面给一条纯实操路径**:

4.1 平台阶梯(由低到高) ​

picoCTF                  美国卡内基梅隆出的免费教学平台,题目按主题分,带提示
  (难度 ⭐)               强烈推荐零基础第一站,英文不难
   ↓
OverTheWire (Bandit)     Linux / Shell / 网络 / 编程基础打怪
  (难度 ⭐)               全是命令行题,练硬功夫
   ↓
HackTheBox / TryHackMe   订阅制大型靶场,有完整 learning path
  (难度 ⭐⭐)              "做完一个 Path 就具备初级渗透能力"
   ↓
CTFtime 上的国际线上赛   各国战队办的真实比赛,周末 24-48 小时
  (难度 ⭐⭐ - ⭐⭐⭐⭐)     最重要的"真打"环节
   ↓
强网杯 / DEFCON CN /     国内顶级赛事,有线上预选
  红明谷 / N1CTF
  (难度 ⭐⭐⭐⭐⭐)         能进前 30 名,简历直接镀金

4.2 学习闭环 ​

听讲解 → 看 writeup → 做题 → 写 writeup → 教别人
   ↑                                          ↓
   └──────────── 周而复始 ────────────────────┘

关键点:没"做题"这一步,所有理论都白学。哪怕一道题做了 6 小时只挪了 10% 进度,也比看 10 篇 writeup 强。

4.3 怎么读 writeup(关键) ​

新人最容易犯的错:做不出来就直接搜 writeup,抄答案,提交 flag,以为自己学会了——这等于零学习。

正确读法:卡至少 1-2 小时确实拿不动 → 找 writeup 只看"思路概述" 回去自己试 → 还不行就看"用了什么工具 / 什么漏洞"再回去 → 实在不行完整读 writeup,然后用自己的话写一遍"关键卡点是什么",自己再重做一次不查 → 最后找一道同类题,看能不能独立做出来 → 这才算"掌握了"。

好的 writeup 平台:CTFtime / GitHub 上各战队的仓库 / 出题方的官方 wp / 知乎与 BiliBili 的国内战队复盘。坏的 writeup:只贴 exploit 不讲思路——这种是写给已经会的人复习用的,新人看了等于没看。

六、CTF 的局限:它不是真实漏洞挖掘 ​

说完优点,必须给 CTF 降温。CTF 和真实工作有几个根本差异,只会刷 CTF 的人到了真实环境会失灵:

  • 题目有"已知答案":CTF 题一定有解、提前测过难度;真实漏洞挖掘 80% 的时间是"在不知道有没有漏洞的代码里搜",这种"目标不明确"的训练 CTF 给不了
  • 题目刻意简化:CTF Web 只几个接口、Pwn 二进制就一个函数有洞;真实 Chrome 几千万行、银行系统几百个微服务——"找到哪里值得看"本身就是大本事
  • 没有业务逻辑漏洞训练:CTF 几乎不出水平越权 / 业务逻辑绕过——这类漏洞要"知道业务该怎么用",而 CTF 业务都是合成的;真实生产环境最高频的漏洞反而是这类
  • 时间尺度反着的:CTF 一道题平均几小时,真实 0day 挖掘平均几周到几个月——很多 CTF 高手转岗到漏洞研究反而坐不住
  • 没有合规 / 协作 / 文档:写报告、对接修复、走 CVE 流程占真实工作 50%+,CTF 完全不教

结论:CTF 是"硬技能引擎",但成不了"全面安全工程师"。把它当训练手段(像运动员的体能训练),不要当目标。

八、CTF 的真实价值 ​

落到工程师身上,打 CTF 能给你的具体回报:

  • 面试加分:腾讯 / 阿里 / 字节 / 360 / 奇安信 / 蚂蚁,几乎所有安全岗都明确把"CTF 战绩"写进招聘。漏洞研究 / 应用安全 / 红队三类岗,简历筛选时 CTF 战绩 >> 普通项目经验
  • 学习硬实力:读 10 篇 XSS 文章 ≈ 自己挣扎着做 1 道带 CSP 的 XSS 题;"压力 + 反馈"学习效率比"看资料"高数量级——这是被神经科学反复证明的"主动学习 > 被动学习"
  • 社区氛围:CTF 社区是全球最活跃、最 nerdy、最不假的技术社区之一——没有 PUA、没有 KPI、没有"做大做强",纯靠技术说话;混进一个好战队,你的成长会被同伴推着走

九、零基础到打第一场 CTF 的 30 天计划 ​

下面是一个完整的 30 天计划——前提是你每天能投入 1-2 小时,有 Linux 基础(会用命令行),会一门编程语言(Python 最佳)。做到这一篇画的最低限,30 天后你应该能在一个真实线上赛打出至少 3-5 道题。

Week 1:工具链 + Linux + 网络复习(Day 1-7) ​

Day 1-2  装环境
         - 一台 Ubuntu 虚拟机 / WSL2

Day 7    Python 复习 + pwntools 基础
         - 自己写一个小 socket 客户端,练 send/recv

Week 2:Web 入门(Day 8-14) ​

Day 8-10  picoCTF Web 类 Easy 全做(约 15-20 题)
          - 重点:DevTools / Cookie / 隐藏接口 / robots.txt
Day 11    SQL 注入专题
          - 复习第 12 篇 + sqlmap 手册
          - 做 5 道 picoCTF / HackTheBox SQLi 题

          - 复习第 10 篇 + 做 5 道 XSS 题

Week 4:Pwn 入门 + 打第一场赛(Day 22-30) ​

  • 至少做过 50 道 CTF 题(任何平台)
  • 至少 2 篇 writeup 公开发出来
  • 打完一场真实比赛,知道"比赛是什么节奏"
  • 对五大类中的一类有方向感

到这一步你已经超过 95% 自学安全的人——绝大多数人停在"看资料阶段",从来没真正做过题。

十、踩坑提醒 ​

  1. 只看不做——把 30 天计划改成"30 天看 30 篇 writeup",纯无效学习
  2. 直接挑战 500 pt 题——挫败感会让你直接放弃,从 50/100 pt 起步
  3. 抄答案换 flag——交了 flag 但没真学到东西,只在记分板上好看
  4. 五类全开摊大饼——前 3 个月专注一类,打稳了再扩展
  5. 沉迷工具不练手工——sqlmap 一把梭做完所有 SQLi 题,真实变种题就抓瞎
  6. 不写 writeup——做完就忘,3 个月后做同类题还要重新挣扎
  7. 不组队 / 不进社区——CTF 是高度协作的活动,单打到 600 分以上几乎不可能,找战队是 Day 60 后的硬刚需
  8. 以为 CTF = 真黑客——见第六节,CTF 是训练,不是真实场景
  9. 去打别人的真实服务器找成就感——这是犯罪,不是 CTF。所有训练在授权环境
  10. 打不动一道题就转方向——卡 6-8 小时是正常的,真的卡 24 小时再切——切得太快永远学不深

十一、本篇要点 + 渗透层(21-25)小结 ​

CTF 心智:
  解谜版安全学习,知识点的强串联训练
  Jeopardy(单兵)/ Attack-Defense(战队)/ KoH(共享)三种赛制
  五大类:Web / Pwn / Reverse / Crypto / Misc 各自一种心智

入门路径:
  picoCTF → OverTheWire → HackTheBox → 真实线上赛
  工具基本盘:Burp / pwntools / IDA / CyberChef / Wireshark
  关键:动手做题 + 写 writeup + 进战队

局限性:
  题目有解 / 刻意简化 / 没业务逻辑漏洞 / 时间尺度反着的
  → CTF 是硬技能引擎,不是全面安全工程师培养体系

真实回报:
  面试加分(国内安全岗几乎刚需)
  学习效率比看资料高一个数量级
  接到全球最活跃的技术社区

渗透层(21-25)的总结——这 5 篇讲清楚了一条主线:

21 信息搜集(攻击的第一步:Recon)
   ↓
22 Web 渗透方法论(Burp 工作流 + 手工 vs 自动)
   ↓
23 内网渗透(从外网到内网 / 域控的纵深)
   ↓
24 红蓝紫队(组织化的攻防演练)
   ↓
25 CTF 入门(把这一切的技能,在解谜场里反复操练)

结论:第 21-25 篇回答的是"攻击者拿到第一个突破点之后接下来 30 步是什么 + 工程师怎么训练这套技能"。到这里整个系列的"攻击者视角"全部讲完——你应该能在白板前讲清楚一次完整的渗透链条、能识别红蓝紫队三种角色、能列出 CTF 五大类的入门动作。剩下的 5 篇(26-30)转向工程化防御视角——攻击讲完了,接下来要回答:如何在公司里把这些"防御常识"工程化下去,让组织而不只是个人具备安全能力。
