# 安全工程系列写作计划

> 来源:`未来系列规划.md` 第三类「纵深主题」第三项,⭐⭐⭐。
> webLearning 用 1 篇浅讲了「认证与安全」(35-认证与安全.md),networkLearning 用 4 篇讲了 TLS / PKI / WAF / 渗透入门——**这个系列把所有这些零散的"安全片段"接上**,从「攻击者视角」一路打到「工程化防御」。
> **看完这层,你写代码时会有"被攻击的本能"**——SSRF / SQL 注入 / 反序列化 / npm 投毒 / Prompt 注入,不再是"听过的名词",而是能拆开讲清楚利用链 + 防御链的硬通货。

---

## 设计思路

六层结构,从「攻击者心智」到「防御工程」:

```
心智 + 威胁建模(攻击者怎么想 / CIA / 信任边界)
   → 密码学(原语 → 协议 → 工程陷阱)
   → Web 安全深入(超越 webLearning 浅讲)
   → 二进制 / 内存安全(栈堆漏洞 / 缓解机制)
   → 渗透与攻防(CTF / 红蓝紫队 / 真实流程)
   → 工程化防御(SDL / 供应链 / 云安全 / AI 安全)
```

**这个系列不是 OWASP 文档翻译**——OWASP 偏列表式,**本系列偏"工程师视角"**:每个漏洞都讲清楚「漏洞原因 → 利用链 → 实战示例 → 修复 → 防御深度」,看完能直接写到代码评审清单里。

不深入到 0day 挖洞或者 APT 研究层面,**而是讲清楚"应用工程师能看懂、需要懂的安全图景"**——你写 Web / 后端 / AI 应用,但上线前必须懂这些。**95% 的真实漏洞都集中在这层。**

## 写作模板

每篇遵守"漏洞原因 → 利用链 → 实战示例 → 修复 → 防御深度"的五段式:

```
1. 这个漏洞 / 风险为什么存在
   - 设计层面的根本缺陷
   - 历史背景(为什么以前没人发现)

2. 攻击者怎么利用它
   - 利用链 step by step
   - 真实 CVE 或者 CTF 题为例

3. 一段最小可复现代码
   - 漏洞代码 + 攻击 payload
   - 不堆完整 exploit,只讲清楚关键一步

4. 怎么修(短期补丁 + 长期重构)
   - 黑名单 vs 白名单
   - 框架级的彻底防御

5. 纵深防御
   - 单点修复不够,加一道 WAF / CSP / 监控
   - 真实公司是怎么把它写进 SDL 的
```

不堆 payload 字典,**只贴关键的 5-10 行 PoC**,让读者「看到漏洞是怎么一步步发生的」就够了。

## 节奏

```
01-03  心智:看完知道"安全是一种工程纪律,不是事后打补丁"
04-08  密码学:看完不会再"自己加盐 MD5",也不会"AES 直接 ECB"
09-15  Web 安全:**最值钱的一层**,工程师 80% 的安全工作都在这
16-20  二进制:**最反直觉**,但理解了你才知道为什么 Rust / Go 火
21-25  渗透:看完知道"攻击者拿到第一台机器之后,接下来 30 步是什么"
26-30  工程:**面试和真实工作的硬通货**,SDL / 供应链 / AI 安全是当下热点
```

**一个硬指标**:看完 09-15 + 26-30 这 12 篇,你应该能在白板前讲清楚:

- "为什么 prepared statement 不是万能的,模板注入怎么绕过 ORM"
- "为什么 JWT 默认配置依然能被攻击,要禁用哪个算法"
- "为什么 npm 安装一个包就能拿到你 token,SLSA 怎么解"
- "如果你给一个 LLM Agent 接了文件系统工具,Prompt 注入能干掉什么"

能讲清楚一题这个系列就值了。

## 反对的写法

这个系列**不写**这些东西,见到了就是写错了:

1. 「下载这个工具,一把梭」——不是黑客教程,是工程师视角
2. 「先讲个 APT 故事 / 黑客大佬传记」——直接进重点
3. 罗列 CVE 编号——只在举例时引用关键 CVE
4. 「这是面试常考」——读者来这里是为了真懂,不是背名词
5. 完整 0day 利用代码 / 实战攻击手法——只到 PoC 层面,不教武器化
6. 不画图就讲 OAuth 流程 / ROP 链 / TLS 握手——这些东西**不画图永远讲不清楚**
7. 抄 OWASP Cheat Sheet——那个文档在那儿,这里只补它没讲透的"为什么"

## 立场声明

本系列**只服务于防御方视角**——理解攻击是为了写出更安全的代码、做更准的代码评审、设计更可靠的系统。

所有示例代码限制在最小可复现 PoC,**不提供任何用于未授权访问的完整利用工具或自动化武器**。CTF / 渗透篇章默认目标是**自己搭的靶机或授权环境**。
