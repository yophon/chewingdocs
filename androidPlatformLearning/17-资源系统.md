# 资源系统:res / values-* 限定符与主题继承

> 一句话:**Android 资源系统的核心是"按设备配置自动选最佳资源"——`values-zh/` 中文环境用、`values-night/` 深色模式用、`drawable-xxxhdpi/` 大密度屏用,你的代码只写 `R.string.app_name`,系统在运行时自动挑出对应资源**。

---

## 一、`res/` 目录结构

```
res/
├── values/                # 基础资源(默认 fallback)
│   ├── strings.xml
│   ├── colors.xml
│   ├── dimens.xml
│   ├── themes.xml
│   ├── styles.xml
│   └── attrs.xml
├── values-zh/             # 中文资源
│   └── strings.xml
├── values-zh-rTW/         # 繁体中文(Taiwan region)
│   └── strings.xml
├── values-night/          # 深色模式资源
│   ├── colors.xml
│   └── themes.xml
├── values-w600dp/         # 宽 ≥ 600dp 屏幕(平板)
│   └── dimens.xml
├── drawable/              # 默认 drawable
├── drawable-mdpi/         # 中密度(160dpi)
├── drawable-hdpi/         # 高密度(240dpi)
├── drawable-xhdpi/        # 超高(320dpi)
├── drawable-xxhdpi/       # 超超高(480dpi)
├── drawable-xxxhdpi/      # 超超超高(640dpi)
├── drawable-night/        # 深色模式专用
├── layout/                # XML 布局
├── layout-land/           # 横屏布局
├── layout-w600dp/         # 宽屏(平板)布局
├── mipmap-*/              # Launcher 图标(按密度)
├── raw/                   # 原始资源(音频 / 视频)
├── xml/                   # 自定义 XML
└── font/                  # 字体
```

**资源限定符**是目录名后的 `-xxx`——`values-zh-rTW-night` 表示"中文 + 台湾地区 + 深色模式"。

---

## 二、限定符优先级

资源限定符按**官方文档顺序**应用,常用的优先级:

```
mcc/mnc          ← 国家 / 运营商代码(罕用)
locale           ← 语言 + 地区(zh / zh-rCN / en-rUS)
layoutDirection  ← LTR / RTL(ldltr / ldrtl)
smallestWidth    ← 屏幕最短边(sw600dp)
screenWidth      ← 当前宽(w600dp)
screenHeight     ← 当前高(h720dp)
screenSize       ← small / normal / large / xlarge
orientation      ← port / land
uiMode           ← night / car / desk / television
density          ← mdpi / hdpi / xhdpi / xxhdpi / ...
```

**多限定符组合**:`values-zh-w600dp-night` = 中文 + 宽 600dp+ + 深色。

**回退规则**:找不到完全匹配 → 找匹配少一个限定符的 → 直到 `values/`。

## 三、`values-night/` 与深色模式

```xml
<!-- res/values/colors.xml -->
<resources>
    <color name="background">#FFFFFF</color>
    <color name="text_primary">#212121</color>
</resources>

<!-- res/values-night/colors.xml -->
<resources>
    <color name="background">#121212</color>
    <color name="text_primary">#EEEEEE</color>
</resources>
```

```xml
<TextView
    android:background="@color/background"
    android:textColor="@color/text_primary" />
```

**系统切深色 → 自动选 night 资源 → Activity 重建(默认行为)→ UI 自动深色**。

**Activity 重建机制**:深色模式切换会调 `onConfigurationChanged` 或重建 Activity,自动重新读资源——你的代码不用做任何事。

`AppCompatDelegate.setDefaultNightMode` 控制 App 内 主题:

```kotlin
AppCompatDelegate.setDefaultNightMode(AppCompatDelegate.MODE_NIGHT_FOLLOW_SYSTEM)
// 或 MODE_NIGHT_NO / MODE_NIGHT_YES
```

## 四、`values-zh-rCN/`:本地化

```
res/
├── values/strings.xml           # 默认(英文)
├── values-zh/strings.xml        # 简中
├── values-zh-rTW/strings.xml    # 繁中(台湾)
├── values-zh-rHK/strings.xml    # 繁中(香港)
├── values-ja/strings.xml        # 日文
├── values-ko/strings.xml        # 韩文
```

`r` 前缀代表 region(地区码),`zh-rCN` = 中文 + 中国大陆。

**Locale 匹配规则**:用户系统语言 zh-CN → 找 `values-zh-rCN/` → 没有就 `values-zh/` → 还没有就 `values/`(默认)。

`strings.xml` 翻译:

```xml
<!-- values/strings.xml -->
<resources>
    <string name="welcome">Welcome %1$s</string>
    <string name="note_count">
        <plurals>
            <item quantity="one">1 note</item>
            <item quantity="other">%d notes</item>
        </plurals>
    </string>
</resources>

<!-- values-zh/strings.xml -->
<resources>
    <string name="welcome">欢迎,%1$s</string>
    <string name="note_count">
        <plurals>
            <item quantity="other">%d 条笔记</item>
        </plurals>
    </string>
</resources>
```

**`<plurals>`** 处理复数——中文没有单复数变化,只用 `other`;英文要区分 one / other。

代码取:

```kotlin
getString(R.string.welcome, userName)
resources.getQuantityString(R.plurals.note_count, count, count)
```

## 五、`values-w600dp/`:大屏适配

```
res/
├── values/dimens.xml         # 手机默认
└── values-w600dp/dimens.xml  # 宽 ≥ 600dp(平板)
```

```xml
<!-- values/dimens.xml -->
<resources>
    <dimen name="content_padding">16dp</dimen>
    <dimen name="grid_columns">2</dimen>
</resources>

<!-- values-w600dp/dimens.xml -->
<resources>
    <dimen name="content_padding">32dp</dimen>
    <dimen name="grid_columns">4</dimen>
</resources>
```

代码 / XML 用 `@dimen/content_padding` / `R.integer.grid_columns`——平板上自动取大值。

**Compose 项目用 `WindowSizeClass` API 更现代**(现代版 8 篇)——不依赖资源限定符,程序里直接判断。

## 六、密度限定符:`drawable-xxhdpi/` 等

| 限定符 | dpi | 缩放系数 |
| --- | --- | --- |
| `mdpi` | 160 | 1× |
| `hdpi` | 240 | 1.5× |
| `xhdpi` | 320 | 2× |
| `xxhdpi` | 480 | 3× |
| `xxxhdpi` | 640 | 4× |

`R.drawable.icon` 在每种密度目录放一份对应大小的 PNG,系统选最匹配的:

```
drawable-mdpi/icon.png       48×48
drawable-xxhdpi/icon.png     144×144
```

设备 dpi 480 → 选 xxhdpi 的 144×144。

**但 PNG 多版本占空间**——所以 **VectorDrawable**(矢量,XML)更现代,一份适配所有密度:

```xml
<!-- res/drawable/icon.xml -->
<vector xmlns:android="http://schemas.android.com/apk/res/android"
    android:width="48dp" android:height="48dp"
    android:viewportWidth="48" android:viewportHeight="48">
    <path android:fillColor="#000000" android:pathData="..." />
</vector>
```

VectorDrawable 是 SVG 的 Android 子集——一份 XML 任意大小渲染,且 dark mode 颜色用 tint 自动适配。

## 七、`dp` / `sp` / `px`:尺寸单位

- **`dp`**(density-independent pixel)——和密度无关:`48dp` 在任何密度屏幕上物理大小一样
- **`sp`**(scale-independent pixel)——同 dp,但**跟系统字体大小设置缩放**。**字号专用**
- **`px`** —— 物理像素。**几乎从不该用**(不同密度屏幕显示大小不一)

转换:`px = dp × density`,density 是 `getResources().displayMetrics.density`。

Compose 用 `Dp` 类型 + `.dp` 扩展:`16.dp` / `24.sp`。底层一样。

## 八、`?attr/` 引用 theme 属性

```xml
<TextView
    android:textColor="?attr/colorPrimary"          <!-- 取当前主题的 primary 色 -->
    android:background="?android:attr/selectableItemBackground" />
```

`?attr/` 是**"当前 theme 里这个 attribute 的值"**——值由 theme 决定。

```xml
<style name="Theme.NotedX" parent="Theme.Material3.DayNight">
    <item name="colorPrimary">@color/notedx_blue</item>
</style>

<style name="Theme.NotedXAlt" parent="Theme.Material3.DayNight">
    <item name="colorPrimary">@color/red</item>
</style>
```

`?attr/colorPrimary` 在 `Theme.NotedX` 下是 blue,在 `Theme.NotedXAlt` 下是 red——**同一份 XML 在不同 theme 下颜色不同**,这是主题切换的根本机制。

**新代码永远用 `?attr/`,不写死颜色**——否则深色模式 / 主题切换全部失效。

---

## 九、`@style` / `<style>` 继承

```xml
<style name="Button.Primary" parent="Widget.Material3.Button">
    <item name="android:textColor">@color/white</item>
    <item name="backgroundTint">?attr/colorPrimary</item>
</style>

<style name="Button.Primary.Large">
    <item name="android:textSize">18sp</item>
</style>
```

**两种继承**:

- `parent="..."` 显式继承
- 名字以 `Parent.Suffix` 隐式继承(`Button.Primary` 自动继承 `Button` 如果它存在)

**Material 3 主题继承**:

```xml
Theme.NotedX
  ← Theme.Material3.DayNight.NoActionBar
      ← Theme.Material3.DayNight
          ← Base.Theme.Material3.DayNight
              ← Theme.Material3.Light / Dark
```

层层覆盖,你只要改你 App 关心的属性,其他自动继承 Material。

## 十、`getResources()` vs `getResources(configuration)`

```kotlin
val context = context.createConfigurationContext(Configuration().apply {
    setLocale(Locale.JAPANESE)
})
val japaneseString = context.getString(R.string.welcome)
```

**`createConfigurationContext`** 让你**临时按指定 config 取资源**——比如"始终用英文显示某段文字"(国际化日志、错误码描述)。

通常不用——99% 场景用当前 config。

## 十一、`AssetManager`:原样资源

`assets/` 目录下的文件**不经编译**,直接 zip 进 APK:

```
assets/
├── fonts/Inter-Regular.ttf
├── data/preset_notes.json
└── audio/click.wav
```

代码读取:

```kotlin
val inputStream = ctx.assets.open("data/preset_notes.json")
val text = inputStream.bufferedReader().use { it.readText() }
```

**与 `res/raw/` 的区别**:`raw/` 也是原样文件,但**通过 `R.raw.x` ID 访问**;`assets/` 通过路径字符串。

**用例**:`assets/` 适合大文件 / 子目录结构 / 字体;`raw/` 适合代码里用 ID 引用的资源。

## 十二、`R` 类:资源 ID 生成

AAPT 编译 res 时生成 `R.java`(在 build 目录):

```java
public final class R {
    public static final class string {
        public static final int app_name = 0x7f110001;
    }
    public static final class drawable {
        public static final int icon = 0x7f0a0001;
    }
}
```

`R.string.app_name` 编译期就是常量 `0x7f110001`——运行时 `getString(0x7f110001)` 查 resources.arsc。

**多模块下** `R` 类按模块分隔——`com.notedx.feature.home.R` 与 `com.notedx.R` 各管各的。**`android.nonTransitiveRClass=true`**(`gradle.properties`)让 R 类不传递,每个模块只看到自己的 R——大幅减少耦合 + 编译时间。

## 十三、`Resources.getIdentifier`:动态查 ID

```kotlin
val id = resources.getIdentifier("note_$index", "string", packageName)
val text = getString(id)
```

**反射式按字符串查 R ID**——慢且不安全,**几乎不该用**。如果你需要"动态选 string",改用 Map 或 enum:

```kotlin
val map = mapOf("a" to R.string.x_a, "b" to R.string.x_b)
val text = getString(map[key] ?: return)
```

`getIdentifier` 在 R8 混淆下还会出问题(R 类被改),避开。

## 十四、主题切换是怎么生效的

```
用户系统切深色
   ↓
系统广播 Configuration changed
   ↓
你的 Activity 收到 onConfigurationChanged 或者被销毁重建
   ↓
默认行为:重建 Activity(把 isNightMode 视为配置变化)
   ↓
ActivityThread 重新 inflate XML
   ↓
读 themes-night/ → night/ → 默认 → 取对应资源
   ↓
新颜色 / 新主题应用
```

**App 内主动切**:

```kotlin
AppCompatDelegate.setDefaultNightMode(AppCompatDelegate.MODE_NIGHT_YES)
// 触发 Activity 重建,深色生效
```

## 十五、Locale 切换:Android 13+ Per-App Language

```xml
<application
    android:localeConfig="@xml/locales_config" />
```

```xml
<!-- res/xml/locales_config.xml -->
<locale-config xmlns:android="http://schemas.android.com/apk/res/android">
    <locale android:name="en" />
    <locale android:name="zh-CN" />
    <locale android:name="zh-TW" />
    <locale android:name="ja" />
</locale-config>
```

用户在**系统设置 → 应用 → NotedX → 语言**单独给你的 App 切语言——Activity 重建,新 locale 自动应用。**你的代码不写一行**。

API 33 以下需要兼容库(`androidx.appcompat`):

```kotlin
AppCompatDelegate.setApplicationLocales(LocaleListCompat.forLanguageTags("ja"))
```

## 十六、AAB 拆分:按设备需要才下载

App Bundle(02 篇)按 ABI / 屏幕密度 / 语言拆分。用户设备:

- arm64-v8a → 只下 arm64 的 .so
- xxhdpi → 只下 xxhdpi 的 drawable
- 系统语言中文 → 只下 zh 的 strings

**APK 体积省 30-50%**。开发者只要正确组织 `res/values-zh/` 之类,不用做任何额外配置——AAB 自动处理。

## 十八、踩坑

**坑 1:`values-zh-CN/`(没有 r 前缀)**。Android 资源限定符规则:地区码必须 `rXX`,写 `zh-CN` 是错的,系统找不到——`values-zh-rCN/` 才对。

**坑 2:`drawable-night/` 与 `drawable/` 同名 PNG 冲突**。系统选 night 时,如果 night 目录的 PNG 比默认有差异,要确保两份都是有效图。不要只在 night/ 放一半图,默认拿不到。

**坑 3:写死颜色到布局 XML**。`android:background="#FFFFFF"` 在深色模式仍然白色——丑出血。**永远用 `?attr/colorSurface` 或 `@color/x` + values-night 覆盖**。

**坑 4:VectorDrawable 不在主线程 inflate**。大型 vector 解码 100ms+,启动时主线程 inflate 直接拖慢。**复杂 vector 用 AppCompat 兼容版,且懒加载**。

**坑 5:strings.xml 写中文不转义**。`<string name="x">Hello, "World"</string>`——双引号要 `\"` 转义;`&` 要 `&amp;`;`'` 要 `\'`。否则 inflate 报 XML 错。

**坑 6:Plurals 漏 `other`**。`getQuantityString` 找不到对应 quantity → 走 fallback / 崩。**`other` 是必备 quantity**(它的语义是"上面所有都不匹配时的兜底")。

**坑 7:layout-land 与 layout 不同步**。两份 XML 字段名字应保持一致(同 ID),否则横竖屏切换后 findViewById 失败。

**坑 8:resources 字段在 R8 后失效**。`Resources.getIdentifier("x", "drawable", packageName)` —— x 被 R8 改名了。**避免动态查 R ID**。

**坑 9:每个 mipmap 密度都放图,APK 暴增**。Launcher 图标按密度多版本是合理的(显示在桌面);但**普通业务图标用 vector 一份就够**。

**坑 10:`getString(R.string.x, args...)` 参数个数不匹配**。`getString` 内部用 `String.format`,参数数量与 placeholder 不对 IllegalFormatException。**翻译时各语言 placeholder 个数必须一致**(`%1$s` / `%2$d` 这种带索引的写法可以让翻译者调顺序)。
