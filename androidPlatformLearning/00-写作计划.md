# Android 平台基础写作计划

> 本系列与 **androidNativeLearning(现代 Android 原生开发)** 是**互补关系**,不是替代。前者讲"用 Kotlin + Compose 写应用",这一套讲"Android 操作系统本身怎么工作 + 那些任何时代都要懂的平台 API"。

---

## 一句话定位

**让一个会写代码的人,理解 Android 不只是"打开 Android Studio 写 Activity",而是一个完整的操作系统:Linux 内核 + AOSP 中间层 + ART + Binder + 系统服务 + 四大组件 + UI 系统。**

读者画像:
- 已经会编程,但 Android 印象停在"Activity 是个屏幕、XML 是布局、Intent 跳转"
- 想读懂 AOSP 源码 / 第三方 SDK / 老开源项目的代码
- 想能用 ADB / Profiler / Systrace / Perfetto 排查问题
- 想理解 Compose / Hilt / WorkManager 这些现代库**底下**到底在调什么 系统服务
- 不限于"现代应用开发"——既要懂 Binder Looper ART,也要懂 Activity 生命周期 / XML View / SQLiteOpenHelper(读老代码)

---

## 统一心智(贯穿 22 章)

> **Android 是一组运行在自己 UID 沙箱内的进程,通过 Binder IPC 与系统服务对话,由 ActivityManagerService 编排生命周期,在 ART 上跑字节码,通过 Handler/Looper 派发事件,通过 View 树绘制 UI。**

这句话压住整本。每一篇都是它的具体展开:

- **"运行在自己 UID 沙箱"** → 03 进程沙箱、20 安全模型
- **"Binder IPC 与系统服务对话"** → 04 Binder、05 系统服务
- **"AMS 编排生命周期"** → 07 Activity、09 Service
- **"ART 上跑字节码"** → 02 APK、06 ART
- **"Handler/Looper 派发事件"** → 12 Handler/Looper、13 ANR
- **"View 树绘制 UI"** → 14-16 View 三章
- **"老开源代码读得懂"** → 08 Fragment、18 旧存储、13 异步演化

## 写作风格

继承现代版的强约束:

1. 开篇一句话压住核心 claim,不超 40 字
2. 章节标题用「一、二、三」,不用学究腔
3. 段落 ≤ 150 字,长段必拆
4. 强观点直接说,不藏着掖着
5. 每章结尾的"踩坑"对应真实事故场景
6. 看老代码、看 AOSP 源码、用 ADB 实操都给具体命令,不空讲
7. **"为什么这件事在现代 Android 还重要"** 是每一章的必答题——避免讲一堆"已被淘汰的知识"

## 不覆盖的内容

- **Kotlin 语言细节、Compose、协程、Hilt、Retrofit、Room 工程用法**——见现代版
- **AOSP 编译、刷机、ROM 定制**——这是 ROM 开发者的事,不是应用开发者
- **NDK / C++ 深度**——除了在 06 篇短暂提到 .so / ABI,不深入
- **Wear OS / Auto / TV 专属 API**——大屏 / 可穿戴是独立专题
- **Android 12 以前的"细节差异"**——只在重要历史节点(API 26 后台收紧、API 33 通知权限、API 35 edge-to-edge)出现
