# 前端学习路线总览

前端系列不要按"框架百科"读。正确顺序是:

```text
浏览器平台与基础能力
  -> 选一个主力框架
  -> 学工程化闭环
  -> 再补性能、安全、跨端和专题
```

一句话先记住:**前端工程 = 用浏览器平台交付可交互 UI,再用工程化保证它可维护、可测试、可发布、可观测。**

## 三、三条框架线怎么选

### React

适合:

- 招人、生态、组件库、资料都要最大公约数
- 要做复杂 Web App、SaaS、管理台
- 未来可能接 Next.js、React Native

主线:

```text
02 React 心智
03 JSX 与组件
04 Hooks
05 状态管理
06 路由
07 数据获取
08 表单
09 性能
10 Next.js
```

React 的核心心智是:

```text
state 变 -> 组件函数重跑 -> 生成新 UI 描述 -> diff -> 更新 DOM
```

### Vue

适合:

- 喜欢模板和更低上手成本
- 做中后台、内容站、产品迭代
- 团队希望官方方案更多,少做生态拼装

主线:

```text
11 Vue 心智
12 模板与指令
13 组合式 API
14 响应式原理
15 Pinia
16 Vue Router
17 Nuxt
```

Vue 的核心心智是:

```text
响应式数据被模板读取 -> 建立依赖 -> 数据变 -> 用到它的视图更新
```

### SolidJS

适合:

- 想理解细粒度响应式
- 高频更新 UI、实时数据面板、性能敏感界面
- 已懂 React / Vue,想换一个角度看 UI 更新

主线:

```text
18 SolidJS 心智
19 Signals
20 Stores
21 SolidStart
```

SolidJS 的核心心智是:

```text
组件函数只跑一次 -> signal 直接绑定 DOM 节点 -> 数据变时精确更新
```

SolidJS 不建议作为新人第一门主力框架。它更适合做对照组,帮你理解 React 和 Vue 各自的代价。

## 四、基础能力不要跳

### CSS

对应 22-23。

只会组件库不等于会前端。CSS 至少要掌握:

- Flexbox:一维布局
- Grid:二维页面骨架
- 响应式布局
- Tailwind / CSS Modules / scoped style 的取舍
- 组件样式怎么复用,页面样式怎么约束

### TypeScript

对应 24。

前端 TS 不是背类型体操,重点是:

- props、事件、表单、API 返回值怎么建模
- 泛型组件和工具类型怎么读
- 类型边界怎么放在 API 层和 domain 层
- 不要用 `any` 把系统打穿

### JavaScript 异步

对应 25。

你要能解释:

- Promise / async / await
- microtask / macrotask
- fetch、AbortController、超时和取消
- 并发、重试、竞态和 loading 状态

前端大量 bug 本质是异步状态没管好。

## 五、工程化主线

26-31 是把 demo 变成工程:

```text
26 构建工具       Vite、monorepo、环境变量、产物
27 测试体系       Vitest、Testing Library、Playwright
28 认证与安全     Session、JWT、OAuth、XSS、CSRF、CSP
29 Node.js 后端   BFF、REST API、接口边界
30 数据库与 ORM   Prisma / Drizzle,只讲前端闭环需要的部分
31 部署与 CI/CD   Docker、GitHub Actions、Vercel、Cloudflare
```

这几篇不要求你变成后端工程师,但要求你能把前端项目接到真实后端和真实发布流程里。

## 六、生产能力

32-35 是上线后最常见的问题:

```text
32 Web 性能优化
33 WebSocket 与 SSE
34 PWA 与 Service Worker
35 浏览器渲染原理
```

前端生产问题通常不是"组件不会写",而是:

- 首屏慢
- 大列表卡
- 接口竞态
- 状态重复请求
- 缓存失效
- 离线和刷新异常
- 页面看起来能用,但 Lighthouse / Web Vitals 很差

## 七、专题怎么学

36-43 按需学:

| 章节 | 什么时候学 |
| --- | --- |
| 36 WebAssembly | 浏览器里跑重计算、图像处理、音视频、已有 C/Rust 代码 |
| 37 Web Components | 组件跨框架复用、设计系统 |
| 38 微前端 | 多团队、多应用、历史包袱很重 |
| 39 图形与动画 | Canvas、WebGL、Three.js、可视化 |
| 40 无障碍 a11y | 面向公众、政企、国际产品 |
| 41 国际化 i18n | 多语言、多地区、时区和格式化 |
| 42 Electron 与 Tauri | Web 技术做桌面端 |
| 43 React Native 与 Capacitor | Web 团队做移动端 |

专题不是主线。没有业务场景时,先知道边界,不要提前深挖。

## 八、最小闭环

学完以后至少做一个项目:

```text
登录
-> 列表 / 详情 / 表单
-> 服务端数据缓存
-> 权限
-> 文件上传
-> E2E 测试
-> 部署
-> 性能检查
```

能完成这个闭环,才算真的会前端。只刷框架 API,很容易停在"会写页面,不会交付系统"。

## 九、一句话总结

```text
React / Vue 是主力生产框架
SolidJS 是响应式心智对照
CSS / TS / JS 异步是地基
构建 / 测试 / 安全 / 部署决定项目能不能长期维护
浏览器性能和专题能力按业务场景补
```
