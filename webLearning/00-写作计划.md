# Web 开发文档写作计划

已整理为 43 篇。主线不再追求"把所有框架都讲一遍",而是围绕现代前端真正常用的三层能力展开:

```text
基础能力(CSS / TS / JS / 浏览器)
  -> 主力框架(React / Vue)
  -> 响应式对照(SolidJS)
  -> 工程化与生产专题
```

一句话:**前端不是框架 API 合集,而是 UI 状态、浏览器平台、工程化链路和生产交付的组合。**

## 推荐读法

### 新人路线

```text
01 -> 22 -> 23 -> 24 -> 25 -> 02-10 -> 26 -> 27 -> 31 -> 32 -> 35
```

先补 CSS、TypeScript、异步和浏览器心智,再学 React。不要上来就把所有框架都刷一遍。

### React 工程路线

```text
02-10 -> 22-28 -> 31-35 -> 40-41
```

目标是能做真实业务前端:组件、状态、数据、表单、路由、性能、测试、安全、部署。

### Vue 工程路线

```text
11-17 -> 22-28 -> 31-35 -> 40-41
```

Vue 主线更适合偏模板、偏产品交付的团队。Nuxt 负责把 Vue 带到 SSR / 全栈场景。

### 进阶对照路线

```text
18-21 -> 35 -> 36 -> 37 -> 38 -> 39
```

SolidJS 用来理解细粒度响应式。Wasm、Web Components、微前端、图形这些按业务需要学,不是每个前端都必须深挖。

---

## 边界

- 不再维护低频框架主线。遇到团队内部指定技术栈时,以官方文档和项目规范为准。
- 不把框架版本新闻写进主线。React、Vue、Solid、Vite、Next、Nuxt 的快变内容放到 `版本复查清单.md` 维护。
- 不把后端、数据库、DevOps 展开成完整后端系列。这里只讲前端工程师需要完成闭环的部分。

---

## 完结说明

43 篇覆盖现代前端的核心知识图:

- UI 框架:React / Vue / SolidJS
- 基础:CSS / TypeScript / JavaScript 异步
- 工程:构建 / 测试 / 安全 / Node API / 数据库 / 部署
- 浏览器:性能 / 实时通信 / PWA / 渲染原理
- 专题:Wasm / Web Components / 微前端 / 图形 / a11y / i18n / 桌面 / 跨端

下一步最该补的不是新框架,而是项目制教程:用 React 或 Vue 做一个真实 SaaS / 管理台 / 内容产品,把这些章节串起来。
