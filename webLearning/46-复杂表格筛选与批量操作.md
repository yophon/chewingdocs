# 复杂表格、筛选与批量操作

中后台最核心的页面不是首页,是列表页。

一句话先记住:**列表页 = 查询条件 + 表格展示 + 行操作 + 批量操作 + 分页排序 + URL 状态。**

---

## 一、列表页的标准结构

```text
UsersPage
  ├─ UserFilter
  ├─ UserTableToolbar
  ├─ UserTable
  ├─ Pagination
  └─ BatchActionBar
```

不要把这些都塞进一个 `UsersPage.tsx`。

页面只组装:

```tsx
export function UsersPage() {
  const search = useUserSearchParams();
  const users = useUsersQuery(search);

  return (
    <PageShell title="Users">
      <UserFilter value={search} />
      <UserTable data={users.data?.items ?? []} loading={users.isLoading} />
      <Pagination total={users.data?.total ?? 0} />
    </PageShell>
  );
}
```

---

## 二、筛选条件必须进 URL

筛选条件不是组件状态,应该放 URL:

```text
/users?keyword=tom&role=admin&page=2&pageSize=20
```

好处:

- 刷新不丢
- 可复制链接
- 返回按钮可用
- 多标签页互不影响
- 服务端渲染 / 预加载更自然

坏写法:

```tsx
const [keyword, setKeyword] = useState('');
const [role, setRole] = useState('');
const [page, setPage] = useState(1);
```

刷新页面全没。

## 三、URL 参数解析

用 schema 解析,不要到处手写:

```ts
import { z } from 'zod';

export const userSearchSchema = z.object({
  keyword: z.string().trim().optional(),
  role: z.enum(['owner', 'admin', 'member']).optional(),
  page: z.coerce.number().int().positive().default(1),
  pageSize: z.coerce.number().int().min(10).max(100).default(20),
  sort: z.enum(['created-desc', 'created-asc', 'name-asc']).default('created-desc'),
});

export type UserSearch = z.infer<typeof userSearchSchema>;
```

```ts
export function parseUserSearch(searchParams: URLSearchParams): UserSearch {
  return userSearchSchema.parse(Object.fromEntries(searchParams));
}
```

所有 API query、表单初始值、queryKey 都用同一个类型。

## 四、查询和表格解耦

查询 hook:

```ts
export function useUsersQuery(search: UserSearch) {
  return useQuery({
    queryKey: ['users', search],
    queryFn: () => listUsers(search),
    placeholderData: (previous) => previous,
  });
}
```

表格只接数据:

```tsx
type UserTableProps = {
  data: User[];
  loading?: boolean;
  selectedIds: string[];
  onSelectedIdsChange: (ids: string[]) => void;
};

export function UserTable(props: UserTableProps) {
  if (props.loading) return <TableSkeleton />;
  if (props.data.length === 0) return <EmptyState />;

  return (
    <table>
      <thead>{/* columns */}</thead>
      <tbody>
        {props.data.map((user) => (
          <UserRow key={user.id} user={user} />
        ))}
      </tbody>
    </table>
  );
}
```

表格组件不要自己请求数据。否则复用和测试都会变差。

## 五、列定义

复杂表格建议把列定义抽出来:

```tsx
type Column<T> = {
  key: string;
  title: string;
  width?: number;
  render: (row: T) => React.ReactNode;
};

export const userColumns: Column<User>[] = [
  {
    key: 'name',
    title: 'Name',
    render: (user) => <UserNameCell user={user} />,
  },
  {
    key: 'role',
    title: 'Role',
    render: (user) => formatUserRole(user.role),
  },
  {
    key: 'createdAt',
    title: 'Created',
    render: (user) => formatDate(user.createdAt),
  },
];
```

列定义适合处理:

- 显隐列
- 固定列
- 排序列
- 导出字段
- 权限控制

## 六、排序

排序也进 URL:

```tsx
function SortableHeader({ field, children }: Props) {
  const search = useUserSearchParams();
  const setSearch = useSetUserSearchParams();

  const nextSort = search.sort === `${field}-asc` ? `${field}-desc` : `${field}-asc`;

  return (
    <button onClick={() => setSearch({ ...search, sort: nextSort, page: 1 })}>
      {children}
    </button>
  );
}
```

排序变化时重置 `page = 1`。筛选条件变化也一样。

## 七、选择与批量操作

选择状态通常是当前页面 UI 状态:

```tsx
const [selectedIds, setSelectedIds] = useState<string[]>([]);
```

切换筛选条件时清空:

```tsx
useEffect(() => {
  setSelectedIds([]);
}, [search.keyword, search.role, search.page]);
```

批量操作:

```tsx
function BatchActionBar({ selectedIds, onClear }: Props) {
  const disableUsers = useDisableUsersMutation();

  if (selectedIds.length === 0) return null;

  return (
    <div>
      <span>{selectedIds.length} selected</span>
      <Button
        variant="danger"
        loading={disableUsers.isPending}
        onClick={() => disableUsers.mutate(selectedIds, { onSuccess: onClear })}
      >
        Disable
      </Button>
    </div>
  );
}
```

批量操作必须考虑:

- 当前页选中
- 跨页全选
- 操作后刷新数据
- 部分成功 / 部分失败
- 权限不足

## 八、跨页全选

跨页全选不能只存 ID。

```ts
type SelectionState =
  | { mode: 'none'; ids: string[] }
  | { mode: 'page'; ids: string[] }
  | { mode: 'all'; excludedIds: string[]; filter: UserSearch };
```

含义:

```text
page:只选当前加载出的这些 id
all :选中当前筛选条件下的全部结果,再排除 excludedIds
```

提交时:

```ts
await batchDisableUsers({
  mode: selection.mode,
  ids: selection.mode === 'page' ? selection.ids : undefined,
  filter: selection.mode === 'all' ? selection.filter : undefined,
  excludedIds: selection.mode === 'all' ? selection.excludedIds : undefined,
});
```

跨页全选必须让后端配合。前端拿不到全部 id 时不要假装能全选。

## 九、分页

分页类型:

```text
offset pagination: page + pageSize
cursor pagination: cursor + limit
```

中后台常用 offset:

```ts
type PageResult<T> = {
  items: T[];
  total: number;
  page: number;
  pageSize: number;
};
```

信息流更适合 cursor:

```ts
type CursorResult<T> = {
  items: T[];
  nextCursor?: string;
};
```

不要在大数据量场景强行 offset 到第 10000 页。数据库会很痛。

## 十、虚拟列表

超过几百行就考虑虚拟化:

```tsx
import { useVirtualizer } from '@tanstack/react-virtual';

const rowVirtualizer = useVirtualizer({
  count: rows.length,
  getScrollElement: () => parentRef.current,
  estimateSize: () => 44,
});
```

虚拟表格要注意:

- 行高是否固定
- sticky header
- 横向滚动
- 键盘可访问性
- 勾选状态不能依赖 DOM 是否存在

虚拟列表优化的是 DOM 数量,不是接口性能。接口慢要分页 / 索引 / 缓存。

## 十一、空、错、加载

列表页必须有完整状态:

```tsx
if (query.isLoading) return <TableSkeleton />;
if (query.isError) return <ErrorState onRetry={() => query.refetch()} />;
if (query.data.items.length === 0) return <EmptyState />;
return <UserTable data={query.data.items} />;
```

空状态要区分:

```text
没有任何数据        -> 引导创建
筛选后没有结果      -> 清空筛选
接口错误            -> 重试 / 联系支持
无权限              -> 申请权限
```

一个 `No data` 解决不了这些问题。

## 十二、检查清单

- [ ] 筛选、分页、排序是否在 URL
- [ ] 查询参数是否用 schema 解析
- [ ] 表格是否只接数据,不直接请求
- [ ] 列定义是否集中管理
- [ ] 筛选变化是否重置页码
- [ ] 批量操作是否处理部分失败
- [ ] 跨页全选是否和后端协议一致
- [ ] 大列表是否虚拟化
- [ ] loading / empty / error / no permission 是否区分

---

## 十三、心智模型

```text
URL 是列表页状态的来源
queryKey 是服务端缓存的来源
selection 是页面临时 UI 状态
table 只负责展示和交互
batch action 必须和后端协议配合
```

列表页写好了,中后台 60% 的复杂度就被你拿下了。
