# 学习路径

这个站点不适合从第一个系列一路读到最后。正确读法是先定目标,再按路线补知识。

一句话:**按角色读,不要按目录顺序读。**

---

## 一、后端工程师进阶

目标:能独立设计、实现、上线和维护一个中等复杂度业务系统。

推荐路线:

```text
云服务与互联网基础常识
  -> 后端学习
  -> 系统设计
  -> 分布式系统
  -> DevOps / SRE
  -> 数据工程
  -> 安全
```

先读:

- [云服务与互联网基础常识总览](/cloudBasicsLearning/01-云服务与互联网基础常识总览)
- [后端学习路线总览](/backendLearning/01-后端学习路线总览)
- [系统设计总览](/systemDesign/01-系统设计总览)
- [分布式系统总览](/distributedLearning/01-分布式系统总览)
- [DevOps-SRE总览](/devopsLearning/01-DevOps-SRE总览)

不要一开始就钻 Kubernetes。先把 HTTP、数据库、缓存、消息队列、限流、事务、监控和发布链路搞清楚。

## 二、AI 应用工程师

目标:能做 RAG、Agent、工具调用、评测、上线和成本控制。

推荐路线:

```text
AI 学习
  -> Claude Code
  -> 数据工程
  -> 后端学习
  -> AI Infra
  -> 安全
```

先读:

- [AI学习路线总览](/aiLearning/01-AI学习路线总览)
- [RAG检索增强生成](/aiLearning/22-RAG检索增强生成)
- [FunctionCalling与ToolUse](/aiLearning/21-FunctionCalling与ToolUse)
- [Evaluation评测](/aiLearning/32-Evaluation评测)
- [Claude-Code总览与心智](/claudeLearning/01-Claude-Code总览与心智)

关键判断:

```text
Prompt 不是系统
RAG 不是搜索框
Agent 不是自动化万能胶
Eval 和观测决定能不能上线
```

## 三、AI Infra / 推理工程师

目标:理解 GPU、显存、并行、推理引擎、调度和成本。

推荐路线:

```text
AI 学习基础
  -> AI Infra
  -> 操作系统
  -> 网络
  -> DevOps / SRE
  -> 数据工程
```

先读:

- [AI工程链总览](/aiInfraLearning/01-AI工程链总览)
- [GPU与显存心智](/aiInfraLearning/02-GPU与显存心智)
- [推理引擎景观](/aiInfraLearning/06-推理引擎景观)
- [KVCache心智](/aiInfraLearning/07-KVCache心智)
- [推理服务的成本与吞吐](/aiInfraLearning/29-推理服务的成本与吞吐)

这条路线不要只看框架命令。重点是算力账、显存账、吞吐、延迟、调度和故障边界。

## 四、全栈工程师

目标:能把 Web、后端、部署、数据和基础安全串成完整产品。

推荐路线:

```text
前端学习
  -> 后端学习
  -> 云服务与互联网基础常识
  -> 系统设计
  -> DevOps / SRE
  -> 安全
```

先读:

- [前端学习路线总览](/webLearning/01-前端学习路线总览)
- [后端学习路线总览](/backendLearning/01-后端学习路线总览)

推荐路线:

```text

  -> Flutter
  -> Android 原生开发 / iOS 原生开发

  -> 后端学习
```

## 七、独立开发 / 产品化

目标:能从想法做到可上线、可运营、可维护的小产品。

推荐路线:

```text
前端学习
  -> 后端学习
  -> 云服务与互联网基础常识
  -> 移动端通用知识
  -> DevOps / SRE
  -> 安全
```

最小闭环:

```text
登录
支付
内容 / 数据
后台管理
部署
监控
备份
隐私政策
发布和回滚
```

独立开发不是少写代码,而是少引入不必要的复杂度。

---

## 八、怎么用这页

每条路线都按这个节奏读:

```text
总览 -> 核心章节 -> 做一个小项目 -> 回头补细节 -> 用检查清单复盘
```

不要追求一次读完所有系列。更好的方式是带着项目读,遇到问题再回到对应章节。
