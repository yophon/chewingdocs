# K8s 实战部署

前面章节讲的都是概念和单个资源。这一章把它们串起来，完整部署一套 **Spring Boot + MySQL + Redis** 应用到 K8s，涵盖从镜像构建到对外暴露的每一步。

---

## 一、目标架构

```
Internet
   │
   ▼
Ingress (nginx)
   │
   ├── /api/*  →  Service: app-svc  →  Deployment: spring-app (3 副本)
   │                                       │
   │                                       ├── MySQL  StatefulSet
   │                                       └── Redis  Deployment
```

所有组件跑在同一个 Namespace `demo` 里，通过 K8s Service DNS 互相发现。

## 四、MySQL

### Secret（密码）

```yaml
# k8s/mysql/secret.yaml
apiVersion: v1
kind: Secret
metadata:
  name: mysql-secret
  namespace: demo
type: Opaque
stringData:
  MYSQL_ROOT_PASSWORD: "strong-password"
  MYSQL_DATABASE: "appdb"
  MYSQL_USER: "app"
  MYSQL_PASSWORD: "app-password"
```

### PersistentVolumeClaim

```yaml
# k8s/mysql/pvc.yaml
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: mysql-pvc
  namespace: demo
spec:
  accessModes: [ReadWriteOnce]
  resources:
    requests:
      storage: 10Gi
  storageClassName: standard   # 本地用 standard；云上用对应 StorageClass
```

### StatefulSet

```yaml
# k8s/mysql/statefulset.yaml
apiVersion: apps/v1
kind: StatefulSet
metadata:
  name: mysql
  namespace: demo
spec:
  selector:
    matchLabels:
      app: mysql
  serviceName: mysql
  replicas: 1
  template:
    metadata:
      labels:
        app: mysql
    spec:
      containers:
        - name: mysql
          image: mysql:8.0
          ports:
            - containerPort: 3306
          envFrom:
            - secretRef:
                name: mysql-secret
          volumeMounts:
            - name: data
              mountPath: /var/lib/mysql
          resources:
            requests:
              cpu: 250m
              memory: 512Mi
            limits:
              cpu: 1
              memory: 1Gi
          readinessProbe:
            exec:
              command: ["mysqladmin", "ping", "-h", "127.0.0.1"]
            initialDelaySeconds: 20
            periodSeconds: 10
      volumes:
        - name: data
          persistentVolumeClaim:
            claimName: mysql-pvc
---
apiVersion: v1
kind: Service
metadata:
  name: mysql
  namespace: demo
spec:
  clusterIP: None          # Headless，StatefulSet 标配
  selector:
    app: mysql
  ports:
    - port: 3306
```

## 五、Redis

```yaml
# k8s/redis/deployment.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: redis
  namespace: demo
spec:
  replicas: 1
  selector:
    matchLabels:
      app: redis
  template:
    metadata:
      labels:
        app: redis
    spec:
      containers:
        - name: redis
          image: redis:7-alpine
          command: ["redis-server", "--appendonly", "yes"]
          ports:
            - containerPort: 6379
          resources:
            requests:
              cpu: 100m
              memory: 128Mi
            limits:
              cpu: 500m
              memory: 512Mi
          volumeMounts:
            - name: data
              mountPath: /data
      volumes:
        - name: data
          emptyDir: {}        # 生产请换成 PVC
---
apiVersion: v1
kind: Service
metadata:
  name: redis
  namespace: demo
spec:
  selector:
    app: redis
  ports:
    - port: 6379
```

## 六、Spring Boot 应用

### 1. 构建镜像

先写好 Dockerfile（参考第 25 章多阶段构建），然后：

```bash
# 构建并推到镜像仓库
docker build -t your-registry/spring-app:1.0.0 .
docker push your-registry/spring-app:1.0.0
```

本地测试用 minikube 可以直接加载：

```bash
minikube image load your-registry/spring-app:1.0.0
```

### 2. ConfigMap

```yaml
# k8s/app/configmap.yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: app-config
  namespace: demo
data:
  SPRING_DATASOURCE_URL: "jdbc:mysql://mysql.demo.svc.cluster.local:3306/appdb?useSSL=false&serverTimezone=Asia/Shanghai"
  SPRING_DATASOURCE_USERNAME: "app"
  SPRING_REDIS_HOST: "redis.demo.svc.cluster.local"
  SPRING_REDIS_PORT: "6379"
  SERVER_PORT: "8080"
```

> `mysql.demo.svc.cluster.local` 是 K8s DNS 规范：`{service}.{namespace}.svc.cluster.local`。

### 3. Secret（数据库密码）

```yaml
# k8s/app/secret.yaml
apiVersion: v1
kind: Secret
metadata:
  name: app-secret
  namespace: demo
type: Opaque
stringData:
  SPRING_DATASOURCE_PASSWORD: "app-password"
```

### 4. Deployment

```yaml
# k8s/app/deployment.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: spring-app
  namespace: demo
spec:
  replicas: 3
  selector:
    matchLabels:
      app: spring-app
  strategy:
    type: RollingUpdate
    rollingUpdate:
      maxSurge: 1
      maxUnavailable: 0       # 零停机发布
  template:
    metadata:
      labels:
        app: spring-app
    spec:
      containers:
        - name: app
          image: your-registry/spring-app:1.0.0
          ports:
            - containerPort: 8080
          envFrom:
            - configMapRef:
                name: app-config
            - secretRef:
                name: app-secret
          resources:
            requests:
              cpu: 250m
              memory: 512Mi
            limits:
              cpu: 1
              memory: 1Gi
          readinessProbe:
            httpGet:
              path: /actuator/health/readiness
              port: 8080
            initialDelaySeconds: 30
            periodSeconds: 10
            failureThreshold: 3
          livenessProbe:
            httpGet:
              path: /actuator/health/liveness
              port: 8080
            initialDelaySeconds: 60
            periodSeconds: 20
```

### 5. Service

```yaml
# k8s/app/service.yaml
apiVersion: v1
kind: Service
metadata:
  name: app-svc
  namespace: demo
spec:
  selector:
    app: spring-app
  ports:
    - port: 80
      targetPort: 8080
```

### 6. Ingress

```yaml
# k8s/app/ingress.yaml
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: app-ingress
  namespace: demo
  annotations:
    nginx.ingress.kubernetes.io/rewrite-target: /
spec:
  ingressClassName: nginx
  rules:
    - host: api.example.com
      http:
        paths:
          - path: /api
            pathType: Prefix
            backend:
              service:
                name: app-svc
                port:
                  number: 80
```

## 七、一键部署脚本

```bash
#!/bin/bash
set -e

kubectl apply -f k8s/namespace.yaml

# 数据层（顺序有依赖）
kubectl apply -f k8s/mysql/
kubectl apply -f k8s/redis/

# 等 MySQL 就绪
kubectl rollout status statefulset/mysql -n demo --timeout=120s

# 应用层
kubectl apply -f k8s/app/

kubectl rollout status deployment/spring-app -n demo --timeout=180s
echo "Deployment done."
```

## 八、常用运维命令

```bash
# 查看所有资源
kubectl get all -n demo

# 查看 Pod 日志
kubectl logs -n demo -l app=spring-app --tail=100 -f

# 进入容器调试
kubectl exec -it -n demo deploy/spring-app -- sh

# 滚动更新镜像
kubectl set image deployment/spring-app app=your-registry/spring-app:1.1.0 -n demo

# 回滚
kubectl rollout undo deployment/spring-app -n demo

# 手动扩缩容
kubectl scale deployment/spring-app --replicas=5 -n demo

# 查看 ConfigMap
kubectl describe configmap app-config -n demo

# 强制删除卡住的 Pod
kubectl delete pod <name> -n demo --force --grace-period=0
```

## 九、HPA：自动伸缩

```yaml
apiVersion: autoscaling/v2
kind: HorizontalPodAutoscaler
metadata:
  name: app-hpa
  namespace: demo
spec:
  scaleTargetRef:
    apiVersion: apps/v1
    kind: Deployment
    name: spring-app
  minReplicas: 2
  maxReplicas: 10
  metrics:
    - type: Resource
      resource:
        name: cpu
        target:
          type: Utilization
          averageUtilization: 70
```

> 前提：集群安装了 metrics-server（`kubectl apply -f https://github.com/kubernetes-sigs/metrics-server/...`）。

## 十、生产 Checklist

| 项目 | 说明 |
| --- | --- |
| ✅ 资源 requests/limits | 防止 OOM Kill 和节点抢占 |
| ✅ readinessProbe | 保证流量只打健康 Pod |
| ✅ livenessProbe | 保证僵尸进程被重启 |
| ✅ RollingUpdate maxUnavailable=0 | 零停机发布 |
| ✅ Secret 不写明文 | 配合 Vault / SOPS 加密 |
| ✅ 多副本跨 Node | 配置 PodAntiAffinity |
| ✅ PVC 数据库持久化 | 别用 emptyDir 存重要数据 |
| ✅ HPA 自动伸缩 | 应对流量波峰 |
| ✅ Namespace 隔离 | 别把所有东西塞 default |
| ✅ 镜像版本固定 | 别用 `:latest`，生产 tag 用 commit SHA |

## 小结

- **开发阶段**：Docker Compose 一键拉起全套本地环境
- **预发 / 生产**：K8s 管理所有容器，声明式配置，Git 提交即上线
- **数据库**：StatefulSet + PVC，不要用 Deployment
- **配置**：ConfigMap 存非敏感、Secret 存密码，应用通过环境变量读取
- **流量**：Service 内部互通，Ingress 对外暴露
