# 动画与共享元素

UIKit 时代的动画写 `UIView.animate(withDuration: 0.3) { view.alpha = 0 }`——典型的"命令式动画"。SwiftUI 是声明式的:**状态变,UI 跟着变;只要把状态变化包在 `withAnimation { }` 里,SwiftUI 自己算 from / to 与帧间插值**。这一篇讲透:**显式动画与隐式动画、Animation 函数选哪个、`transaction` 上下文、`matchedGeometryEffect` 共享元素、`PhaseAnimator` / `KeyframeAnimator`(iOS 17+)、`TimelineView` 时间驱动动画**。

> 一句话先记住:**SwiftUI 动画的核心是"状态变化 → SwiftUI 在状态变化点插入一个动画事务 → 所有依赖该状态的可动画属性都按 Animation 函数插值"。你不写动画过程,你写的是"以什么节奏完成这次状态变化"。**

---

## 一、隐式动画:`.animation(_:value:)`

```swift
struct LikeButton: View {
    @State private var liked = false
    
    var body: some View {
        Image(systemName: liked ? "heart.fill" : "heart")
            .foregroundStyle(liked ? .red : .gray)
            .scaleEffect(liked ? 1.3 : 1.0)
            .animation(.spring, value: liked)         // 隐式动画
            .onTapGesture { liked.toggle() }
    }
}
```

`.animation(_:value:)` 的语义:**当 `value` 变化时,这个 modifier 及其内层的所有可动画属性按指定 animation 插值变化**。`value: liked` 表示"`liked` 变化时触发动画"。

iOS 17+ 推荐传 `value` 参数(早期 `.animation(.spring)` 不带 value 是 deprecated 的——会触发所有变化的动画,容易意外)。

```swift
// ❌ 旧写法,容易触发意外动画
.animation(.easeInOut)

// ✅ 新写法,只对 liked 变化触发
.animation(.easeInOut, value: liked)
```

## 二、显式动画:`withAnimation { }`

```swift
Button("展开") {
    withAnimation(.spring(response: 0.4, dampingFraction: 0.7)) {
        isExpanded.toggle()
    }
}
```

`withAnimation` 把代码块里所有状态变化包成一个"事务",**所有受这些状态影响的可动画属性都按指定 animation 变化**。这是更显式的方式——你控制在哪个时刻、用什么动画、改哪个状态。

`withAnimation` 的好处:**作用域明确**,不像 `.animation` modifier 那样依赖位置。复杂场景(比如一次操作改 N 个状态字段,要一起动画)都用 `withAnimation`。

```swift
Button("切换") {
    withAnimation(.bouncy) {
        // 多个状态一起改,全部按这个动画
        offset = isOn ? 100 : 0
        opacity = isOn ? 1 : 0.3
        rotation = isOn ? 360 : 0
        isOn.toggle()
    }
}
```

**经验法则**:写 `withAnimation` 而不是 `.animation` modifier,除非:
- 同一个 view 在不同状态变化间应该用不同动画
- 子视图想用与父不同的动画

## 三、Animation 选哪个

```swift
.linear(duration: 0.3)              // 匀速,通常做不出"自然"感觉
.easeIn(duration: 0.3)              // 慢启快收,通常用于"离开"动画
.easeOut(duration: 0.3)             // 快启慢收,通常用于"进入"
.easeInOut(duration: 0.3)           // 两端缓,推荐通用
.spring                              // iOS 17+ 默认弹簧,90% 场景这个就够
.spring(response: 0.4, dampingFraction: 0.7)   // 响应快慢 + 阻尼
.bouncy                              // iOS 17+,弹性大
.snappy                              // iOS 17+,响应快、几乎不弹
.smooth                              // iOS 17+,完全不弹的弹簧
.interpolatingSpring(stiffness:damping:)       // 物理模型
```

**spring 系列在 2026 是首选**。iOS 17 引入的 `.spring` / `.bouncy` / `.snappy` / `.smooth` 是 Apple 调过参数的标准弹簧,直接用就好。`response` 调时长(0.3-0.5 通常),`dampingFraction` 调弹性(0.7-1.0 通常,越小越弹)。

`.easeInOut` 适用于"非弹性"的过渡(opacity、color 等)。`.linear` 几乎只用于"连续动画"(progress bar 跟随实际时间)。

```swift
// 推荐组合
withAnimation(.spring) { ... }                  // 大多数场景
withAnimation(.spring(response: 0.25, dampingFraction: 0.85)) { ... }  // 略快、几乎不弹
withAnimation(.easeInOut(duration: 0.2)) { isVisible.toggle() }  // 简单淡入淡出
```

## 四、transaction:细粒度控制

```swift
withAnimation(.spring) {
    showHeader = true
}

// 子视图想跳过这个动画
SomeChildView()
    .transaction { tx in
        tx.disablesAnimations = true   // 即使外层 withAnimation,这个子视图不动画
    }

// 或者子视图想用不同动画
.transaction { tx in
    tx.animation = .linear(duration: 0.5)
}
```

`transaction` 是 SwiftUI 动画的最底层抽象——`withAnimation` 内部就是设置一个 transaction。`.transaction { ... }` modifier 让你拦截并修改这个事务上下文。

用得不多,通常出现在"系统组件嵌套自定义动画"的场景。

## 五、可动画属性与不可动画属性

并不是所有 modifier 都能动画。可动画的:

- `frame` / `offset` / `padding` / `position`
- `opacity` / `foregroundStyle` / `background`(颜色)
- `scaleEffect` / `rotationEffect` / `rotation3DEffect`
- `cornerRadius` / `clipShape`(部分)
- `blur` / `brightness` / `contrast`

不能动画(或动画跳变):

- View 类型变化(`if-else` 切换两种 View)
- `font` 变化(字体大小可以,字体本身不行)
- `text` 内容变化(iOS 17+ 部分支持)

```swift
// ❌ 类型变化不可动画
if isExpanded {
    Text("展开内容")
} else {
    EmptyView()
}

// ✅ 用 opacity / scale 替代
SomeContent()
    .opacity(isExpanded ? 1 : 0)
    .scaleEffect(isExpanded ? 1 : 0)
```

视图出现 / 消失要动画用 `transition`:

```swift
if isVisible {
    SomeContent()
        .transition(.scale.combined(with: .opacity))
}
.animation(.spring, value: isVisible)
```

`transition` 描述 "view 出现 / 消失时怎么动",和 `animation` 配合使用。常见 transition:`.opacity` / `.scale` / `.slide` / `.move(edge:)` / 自定义 `AnyTransition`。

## 六、matchedGeometryEffect:共享元素过渡

iOS 14+ 提供的"共享元素动画" API,把两个不同位置的视图通过相同 id 标记后,SwiftUI 自动在状态变化时插值它们的位置 / 大小:

```swift
struct GalleryView: View {
    @State private var selectedID: UUID?
    @Namespace private var animation
    
    let photos: [Photo]
    
    var body: some View {
        if let selectedID, let photo = photos.first(where: { $0.id == selectedID }) {
            // 详情态:大图占满
            VStack {
                Image(photo.image)
                    .matchedGeometryEffect(id: photo.id, in: animation)
                    .frame(maxWidth: .infinity)
                Button("关闭") {
                    withAnimation(.spring) { self.selectedID = nil }
                }
            }
        } else {
            // 网格态:小缩略图
            LazyVGrid(columns: [GridItem(.adaptive(minimum: 100))]) {
                ForEach(photos) { photo in
                    Image(photo.image)
                        .resizable()
                        .matchedGeometryEffect(id: photo.id, in: animation)
                        .frame(width: 100, height: 100)
                        .onTapGesture {
                            withAnimation(.spring) { selectedID = photo.id }
                        }
                }
            }
        }
    }
}
```

关键三件事:
1. **`@Namespace`** 在父视图里建立命名空间
2. **两个不同位置的 view 都标 `matchedGeometryEffect(id:in:)`,id 相同**——SwiftUI 认为"这是同一个东西在两个状态下的位置"
3. **状态切换包在 `withAnimation` 里**——SwiftUI 自动插值

`matchedGeometryEffect` 是 SwiftUI 区别于 UIKit 的杀手锏——UIKit 同样动画要写 `UIViewControllerAnimatedTransitioning` + snapshot + 手动 frame 计算,几十行;SwiftUI 几行声明。

## 七、PhaseAnimator(iOS 17+):多阶段动画

不需要数值插值,只需要"按顺序经过几个状态"的动画:

```swift
enum LoadingPhase: CaseIterable {
    case start, middle, end
}

struct LoadingDots: View {
    var body: some View {
        HStack {
            ForEach(0..<3) { i in
                Circle()
                    .fill(.blue)
                    .frame(width: 12, height: 12)
                    .phaseAnimator(LoadingPhase.allCases, trigger: i) { content, phase in
                        content
                            .scaleEffect(phase == .middle ? 1.5 : 1.0)
                            .opacity(phase == .start ? 0.3 : 1.0)
                    } animation: { phase in
                        .easeInOut(duration: 0.3)
                    }
            }
        }
    }
}
```

`.phaseAnimator(_:trigger:content:animation:)` 让一个 view 按 phases 序列循环,每个 phase 你定义视觉样子。Apple 在 SF Symbols 的多阶段动画里大量用这个。

## 八、KeyframeAnimator(iOS 17+):关键帧动画

多个属性沿时间线变化,各自有不同节奏:

```swift
struct BounceLoader: View {
    @State private var trigger = false
    
    var body: some View {
        Image(systemName: "star.fill")
            .keyframeAnimator(initialValue: AnimationValues(), trigger: trigger) { content, values in
                content
                    .scaleEffect(values.scale)
                    .rotationEffect(.degrees(values.rotation))
                    .opacity(values.opacity)
            } keyframes: { _ in
                KeyframeTrack(\.scale) {
                    SpringKeyframe(1.5, duration: 0.3)
                    CubicKeyframe(1.0, duration: 0.2)
                }
                KeyframeTrack(\.rotation) {
                    LinearKeyframe(360, duration: 0.5)
                }
                KeyframeTrack(\.opacity) {
                    CubicKeyframe(0.5, duration: 0.25)
                    CubicKeyframe(1.0, duration: 0.25)
                }
            }
            .onTapGesture { trigger.toggle() }
    }
}

struct AnimationValues {
    var scale: Double = 1.0
    var rotation: Double = 0
    var opacity: Double = 1.0
}
```

`KeyframeAnimator` 适合"复杂的、多属性独立时间线"的动画(loading 动画、入场动画、CTA 强调)。语法复杂,但比 `withAnimation` 嵌套链式 modifier 清晰得多。

## 九、TimelineView:时间驱动渲染

某些动画不是"状态变化 → 动画",而是"按时间持续刷新"(秒表、时钟、波纹效果)。`TimelineView` 提供按时间驱动的重渲染:

```swift
TimelineView(.periodic(from: .now, by: 1.0)) { context in
    Text(context.date, style: .time)        // 每秒更新
}

TimelineView(.animation) { context in
    let angle = context.date.timeIntervalSinceReferenceDate * 50
    Image(systemName: "fan")
        .rotationEffect(.degrees(angle))    // 持续旋转
}
```

`TimelineView` 的 `.animation` 模式让 SwiftUI 按动画帧率(60-120 fps)刷新,适合做持续动画。`.periodic` 按固定间隔刷新,适合时钟。

`TimelineView` 与 `Canvas` 配合可以做 60fps 自定义绘制(粒子、波形、表盘),但要注意性能——每帧都重算意味着 body 频繁调用。

## 十、动画与系统设置

用户在系统设置里能关 "Reduce Motion",App 应该尊重这个设置:

```swift
@Environment(\.accessibilityReduceMotion) var reduceMotion

withAnimation(reduceMotion ? nil : .spring) {
    isExpanded.toggle()
}
```

`nil` 表示不动画,直接跳到目标状态。23 篇 Accessibility 会展开。

iOS 19+ 的 Liquid Glass 视觉系统对动画有新约束(透明背景的运动趋向更柔和),但 iOS 18 主线代码不需要改。

## 十一、踩坑

1. **`.animation` 不带 value**——iOS 15 后 deprecated,会触发所有变化的动画,意外的动画从此满天飞。一律带 value。
2. **`if-else` 切换两种 View 期望平滑过渡**——View 类型变化不能动画,SwiftUI 直接 swap。用 `opacity` / `transition`。
3. **`withAnimation` 包了异步代码**——`withAnimation { Task { await ... } }` 不会让异步结果带动画,因为状态变化发生在 await 之后。正确做法是 await 后再 `withAnimation`。
4. **`matchedGeometryEffect` id 不稳定**——id 每次重建都变(比如用 `UUID()`),SwiftUI 认不出是同一个,动画失败。id 必须基于稳定标识(`note.id`)。
5. **同时多个 `matchedGeometryEffect` 相同 id**——会冲突,SwiftUI 只识别其中一个,其他不动画。同一时刻只能有一个 id 出现两次(分散在两个状态下的两处)。
6. **`.spring` 不带参数效果不理想**——iOS 17+ 的 `.spring` 是新参数,与 iOS 16 同名 API 默认值不同。要兼容老系统手动写参数。
7. **`opacity(0)` 期望隐藏 + 不占空间**——`opacity(0)` 仍占布局空间,不参与点击。要完全隐藏 + 不占位用 `if ... else { EmptyView() }`,但失去动画。或者 `opacity(0).frame(width: 0, height: 0)` hack。
8. **`offset` 后无法动画**——`offset` 是可动画的,但如果父布局本身在动画,可能出现"嵌套动画失同步"。改用 `transition` + `if-else`。
9. **`TimelineView(.animation)` 卡顿**——内部 body 太重,60fps 跑不动。简化 body 内容,或用 `Canvas` 替代 SwiftUI view 树。
10. **`PhaseAnimator` 与 `KeyframeAnimator` 在 iOS 16 上 crash**——iOS 17+ API,要 `if #available(iOS 17, *)` 包起来给降级。
