# 动态规划进阶

入门篇讲了 DP 的四步法。这一篇覆盖 DP 的四大主战场:**背包问题**(0/1 / 完全 / 多重)、**区间 DP**(一维区间上的合并)、**状压 DP**(用位图压缩状态)、**树形 DP**(树上的状态传递)。每种都有死模板——**会模板,80% 的进阶题就是套**。

> 一句话先记住:**DP 进阶题的难点不是 DP 本身,是状态定义和遍历顺序**。同一个问题,状态定义"对不对、紧不紧"决定能不能做出来,**遍历顺序决定能不能做对**(尤其背包,内外循环顺序写反答案就错)。

---

## 一、0/1 背包:每件物品只有"选或不选"

**问题**:有 n 件物品,第 i 件重量 `w[i]`、价值 `v[i]`,背包容量 W。每件**最多选一次**,求最大总价值。

### 二维 DP

```
状态:dp[i][j] = 前 i 件物品、容量 j 时的最大价值
转移:dp[i][j] = max(
  dp[i-1][j],                       // 不选第 i 件
  dp[i-1][j-w[i]] + v[i]            // 选第 i 件(j ≥ w[i])
)
base:dp[0][*] = 0
```

```typescript
function knapsack01(w: number[], v: number[], W: number): number {
  const n = w.length
  const dp: number[][] = Array.from({ length: n + 1 }, () => new Array(W + 1).fill(0))
  for (let i = 1; i <= n; i++) {
    for (let j = 0; j <= W; j++) {
      dp[i][j] = dp[i - 1][j]
      if (j >= w[i - 1]) {
        dp[i][j] = Math.max(dp[i][j], dp[i - 1][j - w[i - 1]] + v[i - 1])
      }
    }
  }
  return dp[n][W]
}
```

### 一维优化(滚动数组)

`dp[i][j]` 只依赖 `dp[i-1][...]`,可以压成一维。**关键:容量必须倒序遍历**:

```typescript
function knapsack01(w: number[], v: number[], W: number): number {
  const dp = new Array(W + 1).fill(0)
  for (let i = 0; i < w.length; i++) {
    for (let j = W; j >= w[i]; j--) {  // 倒序!
      dp[j] = Math.max(dp[j], dp[j - w[i]] + v[i])
    }
  }
  return dp[W]
}
```

**为什么倒序**:正序的话 `dp[j-w[i]]` 已经是当前轮算过的(相当于"选了多次"),变成完全背包了。**倒序保证 `dp[j-w[i]]` 还是上一轮的值**。

### 0/1 背包变体识别

- 子集和:容量是 target,价值就是重量,问能不能装满
- 划分等和子集:0/1 背包 + target = sum/2
- 目标和 ±:转化为子集和

**所有"每个东西选 / 不选,凑某个目标"的题,都是 0/1 背包**。

## 二、完全背包:每件物品无穷个

每件物品**可以选无穷次**。

**只要把内层循环改成正序就行**:

```typescript
function knapsackComplete(w: number[], v: number[], W: number): number {
  const dp = new Array(W + 1).fill(0)
  for (let i = 0; i < w.length; i++) {
    for (let j = w[i]; j <= W; j++) {  // 正序!
      dp[j] = Math.max(dp[j], dp[j - w[i]] + v[i])
    }
  }
  return dp[W]
}
```

**为什么正序**:同一物品在同一轮内可以被"重复使用",正好是完全背包的语义。

### 完全背包变体

- **零钱兑换 II**(组合数):求"用各种面额凑成 amount 的方案数"
- **零钱兑换 I**(最少硬币数):求"最少几枚"
- **完全平方数**:用最少的平方数加起来等于 n

```typescript
function coinChangeII(coins: number[], amount: number): number {
  const dp = new Array(amount + 1).fill(0)
  dp[0] = 1
  for (const c of coins) {
    for (let j = c; j <= amount; j++) {
      dp[j] += dp[j - c]
    }
  }
  return dp[amount]
}
```

**注意外层是 coins 内层是 amount**——求**组合数**(不区分顺序)就这么写;反过来内外换是**排列数**(区分顺序)。**这俩搞混就答错**。

## 三、多重背包:每件物品有数量限制

每件物品**最多选 c[i] 次**。

**朴素**:把 c[i] 个物品当成 c[i] 个独立物品(0/1 背包)。复杂度 O(n·W·max c)。
**二进制分组**:把 c[i] 拆成 1, 2, 4, 8, ... 这样的"组",每组当一件 0/1 背包物品。**复杂度优化为 O(n·W·log c)**。
**单调队列优化**:更进一步 O(n·W),竞赛级,工程基本不需要。

## 四、区间 DP:一维区间合并

**特征**:**操作是把一个区间分成两半合并**。

经典题:**石子合并**——n 堆石子排成一行,每次合并相邻两堆,代价是两堆之和。问最小总代价。

```
状态:dp[i][j] = 把 i..j 合成一堆的最小代价
转移:dp[i][j] = min over k in [i, j-1] {
  dp[i][k] + dp[k+1][j] + sum(i..j)
}
base:dp[i][i] = 0
```

**遍历顺序**:**按区间长度从小到大**——长区间依赖短区间。

```typescript
function mergeStones(stones: number[]): number {
  const n = stones.length
  const prefix = [0]
  for (const s of stones) prefix.push(prefix[prefix.length - 1] + s)

  const dp: number[][] = Array.from({ length: n }, () => new Array(n).fill(0))

  for (let len = 2; len <= n; len++) {           // 区间长度
    for (let i = 0; i + len - 1 < n; i++) {       // 起点
      const j = i + len - 1
      dp[i][j] = Infinity
      for (let k = i; k < j; k++) {
        dp[i][j] = Math.min(dp[i][j],
          dp[i][k] + dp[k + 1][j] + prefix[j + 1] - prefix[i])
      }
    }
  }
  return dp[0][n - 1]
}
```

**复杂度 O(n³)**,n ≤ 500 友好。

### 区间 DP 经典题

- 戳气球(LeetCode 312)——逆向思考,假设 i..j 中 k 是"最后戳的"
- 最长回文子序列——dp[i][j] = i..j 内最长
- 最优 BST 构造

**识别信号**:**问题在区间上有"最后一步是分开两半 / 选一个中点"的特征**。

## 五、状压 DP:用位图压状态

**适用场景**:**集合元素少(≤ 20),但要枚举所有子集**。把"选哪些"用一个二进制数表示。

经典题:**TSP(旅行商)**——访问所有城市的最短路径。

```
状态:dp[mask][u] = 已访问集合 mask、当前在 u 时的最短距离
转移:dp[mask | (1<<v)][v] = min(..., dp[mask][u] + dist[u][v])
base:dp[1<<起点][起点] = 0
```

```typescript
function tsp(dist: number[][]): number {
  const n = dist.length
  const dp: number[][] = Array.from(
    { length: 1 << n }, () => new Array(n).fill(Infinity))
  dp[1][0] = 0

  for (let mask = 1; mask < (1 << n); mask++) {
    for (let u = 0; u < n; u++) {
      if (!(mask & (1 << u))) continue
      if (dp[mask][u] === Infinity) continue
      for (let v = 0; v < n; v++) {
        if (mask & (1 << v)) continue
        const newMask = mask | (1 << v)
        dp[newMask][v] = Math.min(dp[newMask][v], dp[mask][u] + dist[u][v])
      }
    }
  }

  let ans = Infinity
  for (let u = 1; u < n; u++) {
    ans = Math.min(ans, dp[(1 << n) - 1][u] + dist[u][0])
  }
  return ans
}
```

**复杂度 O(2ⁿ · n²)**,n ≤ 20 工业里能跑。

### 状压 DP 套路

- `mask` 用二进制表示集合,1 表示选了
- `mask & (1 << i)` 判断 i 在不在集合
- `mask | (1 << i)` 把 i 加入
- `mask ^ (1 << i)` 把 i 取出

**经典题**:小集合的"恰好选 k 个"、"覆盖所有"、"互相冲突"。

## 六、树形 DP:树上的状态传递

**特征**:**问题在树上,每个节点的答案依赖子树**。

经典题:**树的最大独立集**(打家劫舍 III)——树上偷钱,父子不能同时偷。

```
状态:对每个节点 u 维护两个值
  dp[u][0] = u 不偷,子树最大值
  dp[u][1] = u 偷,子树最大值
转移:
  dp[u][1] = u.val + sum(dp[c][0])  for each child c
  dp[u][0] = sum(max(dp[c][0], dp[c][1]))
```

```typescript
function rob(root: TreeNode | null): number {
  function dfs(node: TreeNode | null): [number, number] {
    if (!node) return [0, 0]
    const [lNo, lYes] = dfs(node.left)
    const [rNo, rYes] = dfs(node.right)
    const no = Math.max(lNo, lYes) + Math.max(rNo, rYes)
    const yes = node.val + lNo + rNo
    return [no, yes]
  }
  return Math.max(...dfs(root))
}
```

**树形 DP 模板就是后序遍历 + 每个节点返回多个状态**。一次 DFS,O(n)。

### 树形 DP 经典题

- 二叉树最大路径和
- 树的直径
- 树的最小覆盖
- 树上数对统计

## 七、DP 优化技巧总览

### 1. 滚动数组(空间)

发现 `dp[i]` 只依赖 `dp[i-1]` 时,**只保留两行甚至一行**。

### 2. 单调队列优化

形如 `dp[i] = min(dp[j])` 在某个范围内时,用单调队列 O(1) 取最值。例:跳跃游戏 IV。

### 3. 斜率优化

`dp[i] = dp[j] + f(i,j)` 形式且能整理成线性,用凸包 / 单调队列优化到 O(n)。**竞赛级**。

### 4. 矩阵快速幂

线性递推 + 巨大 n 时,把递推写成矩阵乘法,**矩阵快速幂 O(log n)**。

## 八、状态定义的"加维度"思路

如果一维状态不够用,**加维度**:

- 题目里有"两个东西在动" → 二维 dp[i][j]
- 题目里有"必须按顺序" → 加一维表示当前阶段
- 题目里有"剩余次数 / 状态机状态" → 加维度

例:**股票买卖含手续费 / 冷冻期**——`dp[i][持有还是没持有]`,二维状态机。

```
dp[i][0] = max(dp[i-1][0], dp[i-1][1] + price[i])    // 今天没持有
dp[i][1] = max(dp[i-1][1], dp[i-1][0] - price[i])    // 今天持有
```

**多状态机型 DP** 是面试热门——能交易 k 次、有冷冻期、含手续费,套路一脉相承。

---

## 九、一句话总结

> **DP 进阶 = 状态定义 × 转移方程 × 遍历顺序**。背包看内外循环和正逆序、区间看长度递增、状压看 bit 操作、树形看后序 DFS——**每种类型背一份模板,变体都是套**。**遍历顺序写错,代码再漂亮也是错答案**——这是 DP 进阶最容易翻车的地方。
