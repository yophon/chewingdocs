# 二分查找

二分查找是**算法书第一章就讲、面试答错率最高**的算法。"找一个数"看起来简单,但**边界条件写错率超过 80%**——`lo <= hi` 还是 `lo < hi`?`hi = mid` 还是 `hi = mid - 1`?`mid = (lo+hi)/2` 还是要防溢出?这一篇用一套**统一模板**把所有变体收拾干净,顺便讲二分的进阶用法:**二分答案**。

> 一句话先记住:**二分查找的本质是"在单调性上做对半减少"**,不仅适用于"在有序数组里找数",更适用于"答案有单调性的最优化问题"。**真正考二分的题里,数组只是表象,单调性才是关键**。

---

## 一、最朴素的二分

在**有序数组**里找目标值,返回下标(没找到返回 -1):

```typescript
function binarySearch(arr: number[], target: number): number {
  let lo = 0, hi = arr.length - 1
  while (lo <= hi) {
    const mid = (lo + hi) >> 1
    if (arr[mid] === target) return mid
    if (arr[mid] < target) lo = mid + 1
    else hi = mid - 1
  }
  return -1
}
```

**关键决策**:
1. `[lo, hi]` 是闭区间还是 `[lo, hi)` 半开区间?**统一用闭区间**,代码更直观
2. 闭区间下:`lo <= hi` 是循环条件
3. **不能写 `lo = mid` / `hi = mid`**——会死循环。必须 `lo = mid + 1` 或 `hi = mid - 1`
4. `(lo + hi) >> 1`(等价于 `Math.floor((lo + hi) / 2)`),用位运算**也防 lo + hi 溢出**(JS 里 number 不溢出但 Java / C++ 会)

**这就是模板的全部,但变体让 80% 的人翻车**。

---

## 二、查找"边界":两种变体

实际场景大多不是"找等于的",而是:
- 找**第一个 ≥ target 的位置**(lower_bound)
- 找**最后一个 ≤ target 的位置**

这两个边界比"找等于"更常用(C++ STL 也是这两个 API)。

### 找第一个 ≥ target

```typescript
function lowerBound(arr: number[], target: number): number {
  let lo = 0, hi = arr.length  // 注意 hi = arr.length
  while (lo < hi) {              // 注意 < 不是 <=
    const mid = (lo + hi) >> 1
    if (arr[mid] >= target) hi = mid  // 注意 hi = mid
    else lo = mid + 1
  }
  return lo  // lo 落在第一个 ≥ target 的位置(或 arr.length 如果都更小)
}
```

**关键**:
- 区间是 `[lo, hi)` **左闭右开**
- 循环条件 `lo < hi`
- `hi = mid`,**不是 `mid - 1`**——因为 `mid` 还可能是答案
- 结束时 `lo == hi`,就是答案

### 找最后一个 ≤ target

```typescript
function upperBound(arr: number[], target: number): number {
  let lo = 0, hi = arr.length
  while (lo < hi) {
    const mid = (lo + hi) >> 1
    if (arr[mid] > target) hi = mid
    else lo = mid + 1
  }
  return lo - 1  // lo 是第一个 > target 的位置,-1 就是最后一个 ≤
}
```

## 三、统一模板心法

死记硬背边界容易出错,**用"红蓝染色"思维**:

把数组想象成两部分:
- **不满足条件**(蓝色,在左边)
- **满足条件**(红色,在右边)

二分要找"**红蓝交界处的第一个红**"。

```
[蓝 蓝 蓝 蓝 | 红 红 红]
              ↑
            想找这个
```

模板:

```typescript
function bisect(check: (i: number) => boolean, n: number): number {
  let lo = 0, hi = n
  while (lo < hi) {
    const mid = (lo + hi) >> 1
    if (check(mid)) hi = mid     // 是红色,答案可能在 mid 或更左
    else lo = mid + 1             // 是蓝色,答案一定在 mid 右边
  }
  return lo
}
```

**所有二分都能套这个模板**,只要你能写出 `check` 函数(给一个位置 / 答案,判断它是不是红色)。

例子,找第一个 ≥ target:

```typescript
const idx = bisect(i => arr[i] >= target, arr.length)
```

例子,找最后一个 ≤ target:

```typescript
const idx = bisect(i => arr[i] > target, arr.length) - 1
```

**这个模板我写了几百次没出过错**。值得熟到一秒写完。

## 四、二分答案:真正的高级用法

二分最强大的用法**不是在数组里找数**,而是:

> **答案在某个范围内,且具有"如果 X 行那么 ≥ X 也行 / 如果 X 不行那么 ≤ X 也不行" 的单调性时,直接二分这个答案**

**核心套路**:
1. 确定答案的取值范围 `[lo, hi]`
2. 写一个判定函数 `check(x)`,问"答案是 x 行不行"
3. 二分这个范围找临界点

### 经典例题:"绳子切段"

**有 n 根绳子,长度各异。要切出 k 段相等长度的绳子,问最长能切多长?**

**朴素**:不知道怎么下手。
**二分答案**:答案 X 是"切出来的段长度"。

- 二分 X 的范围:`[0, max(绳长)]`
- check(X):用 X 去切每根绳子,加起来够不够 k 段
- 单调性:**X 越大,能切的段数越少**——典型的"满足→不满足"分界

```typescript
function maxLen(ropes: number[], k: number): number {
  let lo = 0, hi = Math.max(...ropes)
  while (lo < hi) {
    const mid = (lo + hi + 1) >> 1  // 注意 +1,要找最右
    if (check(ropes, mid) >= k) lo = mid
    else hi = mid - 1
  }
  return lo
}

function check(ropes: number[], len: number): number {
  return ropes.reduce((s, r) => s + Math.floor(r / len), 0)
}
```

**注意 `mid = (lo + hi + 1) >> 1`** ——要"找最右满足"时必须 +1,否则死循环。

### 二分答案的判别信号

**看到"最大化最小值 / 最小化最大值"** 几乎一定是二分答案。

- 木匠切板:让最长板尽量短 → 二分这个长度
- 划船过河:每船能载多人 → 二分这个数
- 派送任务:让最长耗时尽量短 → 二分时间
- 最大间隔最小化:二分间隔

这种"最优化里夹着最优化"的题,几乎都是二分答案 + 贪心 check。

## 五、在旋转数组里二分

**旋转数组**:`[4,5,6,7,0,1,2]`(原本有序,在某个点旋转过)。

它**整体不有序,但局部有序**——分成两段,每段内有序。二分还能用!

```typescript
function searchRotated(arr: number[], target: number): number {
  let lo = 0, hi = arr.length - 1
  while (lo <= hi) {
    const mid = (lo + hi) >> 1
    if (arr[mid] === target) return mid
    if (arr[lo] <= arr[mid]) {
      // 左半边有序
      if (arr[lo] <= target && target < arr[mid]) hi = mid - 1
      else lo = mid + 1
    } else {
      // 右半边有序
      if (arr[mid] < target && target <= arr[hi]) lo = mid + 1
      else hi = mid - 1
    }
  }
  return -1
}
```

**心法**:每次判断 mid 处于哪一段,然后利用那段的有序性决定去哪边。**面试爱出**,因为它考"二分的本质是单调性,不是排序"。

## 六、浮点二分

整数二分要用 mid+1 / mid-1 防止死循环,**浮点数二分用相对精度**:

```typescript
function sqrtBinary(x: number, eps = 1e-9): number {
  let lo = 0, hi = Math.max(1, x)
  while (hi - lo > eps) {
    const mid = (lo + hi) / 2
    if (mid * mid < x) lo = mid
    else hi = mid
  }
  return lo
}
```

**别用 `mid + 1`**(浮点没下一个数),用 **(hi - lo) > eps**。

工业里浮点二分常被牛顿迭代法替代(收敛快)。

## 七、常见坑速查

### 坑 1:溢出

`(lo + hi) / 2` 在 lo + hi > Int32.MAX 时溢出。**用 `lo + ((hi - lo) >> 1)`** 安全。

### 坑 2:死循环

`lo = mid`(没 +1)时,如果 `mid == lo`,下次 `lo` 还是这个值——死循环。

**口诀**:
- 闭区间 `[lo, hi]`:`lo = mid + 1` / `hi = mid - 1`
- 半开 `[lo, hi)`:`lo = mid + 1` / `hi = mid`

### 坑 3:边界没考虑

数组空、target 比所有元素都大、target 比所有都小——这些边界 `lowerBound` 模板会自然返回 `0` 或 `arr.length`。

### 坑 4:错把"找等于"模板用在"找边界"

```typescript
// 这个找的是"任意一个等于 target 的下标",而不是"第一个"
while (lo <= hi) {
  if (arr[mid] === target) return mid
  ...
}
```

**找第一个 / 最后一个一定要用 lower_bound / upper_bound 模板**。

## 八、二分跟其他算法的组合

### 二分 + 双指针:三数之和

固定一个数,剩下俩用双指针 / 二分。

### 二分 + 前缀和:某个范围的和

前缀和 + 二分查找第一个 ≥ X 的位置。

### 二分 + 树:树上二分

线段树 / 平衡树上二分,工程少见,竞赛专用。

### 二分 + 字符串哈希:最长重复子串

二分长度 L,然后用滚动哈希 O(n) 检查"有没有长度 L 的子串重复"。

---

## 九、一句话总结

> **二分的核心是单调性,不是有序数组**。会在数组里找数是入门,**会用"二分答案"识别一类最优化问题**才是进阶。统一模板("红蓝染色")写一次,所有二分变体都能套——**死记每种边界是反人性的,记一个模板永远不忘**。

下一篇讲**双指针与滑动窗口**——线性扫描里的两大利器,把一类 O(n²) 题压到 O(n),"最小覆盖子串" 这种经典模板背下来终身受用。
