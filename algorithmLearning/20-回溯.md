# 回溯

回溯(Backtracking)是**算法版的"穷举但聪明"**——本质是 DFS,核心是"选择 / 撤销选择 / 剪枝"。**所有"求所有方案 / 排列 / 组合"的题,都用回溯**。会写一个标准模板,90% 回溯题都是套。

> 一句话先记住:**回溯 = DFS + 路径记录 + 剪枝 + 撤销**。每一步选一个分支,走不通就退回上一步换一个分支——像走迷宫,走错了就回头再试别的。理解这一点,你就理解了"为什么所有回溯都长得这么像"。

---

## 一、回溯 vs DP 的核心区别

DP 求**最优解 / 方案数**,回溯求**所有方案具体长什么样**。

| 特征 | DP | 回溯 |
| --- | --- | --- |
| 输出 | 数字(最大值 / 计数) | 列表(所有具体方案) |
| 复杂度上限 | 多项式 | 指数 / 阶乘 |
| 典型题 | 最长 / 最少 / 总数 | 全排列 / 所有组合 / 子集 |
| 优化空间 | 状态压缩 | 剪枝 |

**所以输入大小通常不一样**:
- DP 题 n 经常 ≤ 10⁵
- 回溯题 n 通常 ≤ 20(因为是指数)

---

## 二、回溯通用模板

```typescript
function backtrack(path: any[], choices: any[]) {
  if (满足结束条件) {
    result.push([...path])  // 注意要拷贝
    return
  }
  for (const choice of choices) {
    if (不合法) continue   // 剪枝
    path.push(choice)       // 做选择
    backtrack(path, 新的choices)
    path.pop()              // 撤销选择
  }
}
```

**死记这个骨架**——所有回溯题都套这个。

**关键三件事**:
1. **结束条件**(什么时候记录答案)
2. **当前层的选择空间**(本步能选什么)
3. **做选择 → 递归 → 撤销选择**(必须三连,缺撤销就错)

## 三、全排列:回溯入门

"给一个不重复的数组,返回所有全排列。"

```typescript
function permute(nums: number[]): number[][] {
  const result: number[][] = []
  const path: number[] = []
  const used = new Array(nums.length).fill(false)

  function bt() {
    if (path.length === nums.length) {
      result.push([...path])  // 拷贝!path 还会变
      return
    }
    for (let i = 0; i < nums.length; i++) {
      if (used[i]) continue
      used[i] = true
      path.push(nums[i])
      bt()
      path.pop()
      used[i] = false
    }
  }

  bt()
  return result
}
```

**关键细节**:
- **`used` 标记数组**(每个元素只用一次)
- **结束记录时拷贝**`[...path]`(否则后续修改会把已记录的也改了)
- **撤销要彻底**(`pop()` + `used[i] = false`)

## 四、组合 vs 排列:差一个 start

"从 [1..n] 选 k 个数的所有组合"。

**关键区别**:**排列要顺序,组合不要**。所以组合通过"**只往后选**"避免重复:

```typescript
function combine(n: number, k: number): number[][] {
  const result: number[][] = []
  const path: number[] = []

  function bt(start: number) {
    if (path.length === k) {
      result.push([...path])
      return
    }
    for (let i = start; i <= n; i++) {     // 从 start 开始
      path.push(i)
      bt(i + 1)                             // 下次从 i+1 开始
      path.pop()
    }
  }

  bt(1)
  return result
}
```

**`start` 参数是组合题的灵魂**——它把"选过的"和"未选的"分开,**不让重复出现 [1,2] 和 [2,1]**。

### 组合优化:剪枝

如果剩余可选不够凑够 k 个,**直接停**:

```typescript
for (let i = start; i <= n - (k - path.length) + 1; i++)
//                       ^ 剩下还要选的个数
```

**回溯优化的本质就是剪枝**——把"已经不可能产生答案"的分支提前砍掉。

## 五、子集:不需要"凑齐"

"返回数组的所有子集(空集也算)。"

**和组合的区别**:子集**不固定大小**——每经过一个节点都记录一次答案。

```typescript
function subsets(nums: number[]): number[][] {
  const result: number[][] = []
  const path: number[] = []

  function bt(start: number) {
    result.push([...path])  // 每个节点都记录,不等"满"
    for (let i = start; i < nums.length; i++) {
      path.push(nums[i])
      bt(i + 1)
      path.pop()
    }
  }

  bt(0)
  return result
}
```

**子集模板:不需要 `if (条件) return`,直接每次都 push**。

## 六、含重复元素的去重

"给 `[1,1,2]`,返回所有不重复的全排列 / 子集 / 组合。"

如果按朴素回溯,会出 `[1,1,2]` 两次(因为两个 1 被当成不同元素)。

**去重套路**:**先排序,然后跳过同层重复**。

```typescript
function permuteUnique(nums: number[]): number[][] {
  nums.sort((a, b) => a - b)
  const result: number[][] = []
  const path: number[] = []
  const used = new Array(nums.length).fill(false)

  function bt() {
    if (path.length === nums.length) {
      result.push([...path])
      return
    }
    for (let i = 0; i < nums.length; i++) {
      if (used[i]) continue
      // 关键去重:同层里如果前一个相同元素没被用过,跳过
      if (i > 0 && nums[i] === nums[i - 1] && !used[i - 1]) continue
      used[i] = true
      path.push(nums[i])
      bt()
      path.pop()
      used[i] = false
    }
  }

  bt()
  return result
}
```

**理解**:**同一层里**,如果当前元素跟前一个相同,且前一个**没被用过**(说明你跳过它了),那当前元素也跳过——避免在同层选两次相同值。

**这是回溯里最绕的细节**,**画递归树**就能看懂——"同层去重"指的是"在同一个父节点的不同子选择里"。

## 七、N 皇后:经典棋盘类

"在 n×n 棋盘上放 n 个皇后,任意两个不同行 / 不同列 / 不同对角线。返回所有摆法。"

```typescript
function solveNQueens(n: number): string[][] {
  const result: string[][] = []
  const board: number[] = []  // board[r] = col

  function isValid(row: number, col: number): boolean {
    for (let r = 0; r < row; r++) {
      const c = board[r]
      if (c === col) return false
      if (Math.abs(r - row) === Math.abs(c - col)) return false  // 对角线
    }
    return true
  }

  function bt(row: number) {
    if (row === n) {
      const layout = board.map(c => '.'.repeat(c) + 'Q' + '.'.repeat(n - c - 1))
      result.push(layout)
      return
    }
    for (let col = 0; col < n; col++) {
      if (!isValid(row, col)) continue
      board.push(col)
      bt(row + 1)
      board.pop()
    }
  }

  bt(0)
  return result
}
```

**回溯的精华在 `isValid`** ——每一步先判断当前选择是否合法,不合法直接剪枝。

**进阶优化**:用三个 set 分别记"已用列"、"主对角线"、"副对角线",**O(1) 判断**比 O(n) 扫历史快。

## 八、剪枝是回溯的灵魂

回溯的复杂度上限是指数级,**不剪枝就死**。

### 剪枝种类

1. **可行性剪枝**:当前路径已经违反约束,停
2. **最优性剪枝**:当前已经比最优解差,停(求最优解时)
3. **重复剪枝**:走过的状态不再走
4. **顺序剪枝**:让搜索更容易"尽快碰到答案",**比如按值降序选**

### 例:数独求解

只有"可行性 + 顺序剪枝",数独 9×9 能秒出。剪枝写挫的话,有些题指数级跑不完。

## 九、子集和 / 分割:经典回溯应用

### 子集和(给 sum 找方案)

```typescript
function combinationSum(candidates: number[], target: number): number[][] {
  candidates.sort((a, b) => a - b)
  const result: number[][] = []
  const path: number[] = []

  function bt(start: number, remain: number) {
    if (remain === 0) { result.push([...path]); return }
    for (let i = start; i < candidates.length; i++) {
      if (candidates[i] > remain) break  // 剪枝:排序后大于剩余直接 break
      path.push(candidates[i])
      bt(i, remain - candidates[i])     // 同一元素可重复用,所以传 i 不是 i+1
      path.pop()
    }
  }

  bt(0, target)
  return result
}
```

**先排序 + break**剪枝是回溯的常见加速。

### 分割问题

"把字符串分成若干段,每段满足某条件"——回溯模板,在每个位置决定"是否在这切一刀"。

例:回文分割、IP 地址复原、单词拆分(返回方案)。

## 十、回溯 vs 状压枚举

`n ≤ 20` 时,**直接枚举所有子集**(2ⁿ)和回溯复杂度其实差不多。

```typescript
for (let mask = 0; mask < (1 << n); mask++) {
  // mask 二进制表示选了哪些元素
}
```

**优势**:无递归 / 不用栈。**劣势**:无法剪枝。

**选择**:有强剪枝条件 → 回溯;没什么剪枝 → 直接位运算枚举。

## 十一、回溯常见坑

1. **忘记拷贝路径**:`result.push(path)` 全部最后变成空数组(因为 path 是同一个引用)。**必须 `[...path]`**。
2. **撤销不彻底**:`used[i]` 改了没改回来。
3. **去重没排序**:同层去重必须先排序,保证相同元素相邻。
4. **`bt(i + 1)` 还是 `bt(i)`**:同一元素能否重用?能就 `i`,不能就 `i+1`。
5. **start 参数滥用**:全排列不用 start(因为顺序重要),组合 / 子集要 start。

---

## 十二、回溯的复杂度估计

| 题型 | 复杂度 | 上限 n |
| --- | --- | --- |
| 全排列 | O(n!) | 10 |
| 子集 | O(2ⁿ · n) | 20 |
| 组合 C(n,k) | O(C(n,k) · k) | 看 k |
| N 皇后 | O(n!) | 13 |
| 数独 | 不好估,实际很快 | 9×9 标准 |

**看到 n ≤ 10 大概率回溯 / DFS,n ≤ 20 状压 + 回溯**。

## 十三、一句话总结

> **回溯就是 DFS + 撤销 + 剪枝**。一个模板写所有题,差异只在"选择空间"、"结束条件"、"剪枝条件"。**剪枝是它的灵魂**——朴素回溯指数爆炸,加上几条 `if continue` 就能秒出。
