# 双指针与滑动窗口

双指针和滑动窗口是**算法世界里"成本最低、收益最高"的套路**——多数题暴力 O(n²),套上指针就 O(n)。心智简单(两个变量在数组上移动),但变体丰富(快慢、对撞、滑窗、左右扫)。会用模板,这一类题从此变罐头。

> 一句话先记住:**双指针的本质是"利用问题的某种单调性,让两个指针朝同一/相反方向移动,把嵌套循环压成线性"**。所有套路都是这一句话的变体。

---

## 一、双指针的三种模式

### 1. 快慢指针(同向)

两个指针都从左往右,但**步长不同 / 速度不同**。

**典型场景**:
- 链表找中点(slow 步 1,fast 步 2)
- 链表判环(Floyd)
- 删除有序数组中的重复元素
- 移动 0 到末尾

**链表中点**(05 篇讲过)。**这里看个数组的例子**:删除排序数组中的重复元素,要求**原地修改、O(1) 空间**:

```typescript
function removeDuplicates(arr: number[]): number {
  if (arr.length === 0) return 0
  let slow = 0
  for (let fast = 1; fast < arr.length; fast++) {
    if (arr[fast] !== arr[slow]) {
      slow++
      arr[slow] = arr[fast]
    }
  }
  return slow + 1
}
```

**心法**:`slow` 维护"已处理好的边界",`fast` 探索新元素,**slow 永远 ≤ fast**——这是单调性来源。

### 2. 对撞指针(相向)

两个指针**一个左、一个右,向中间靠**。

**典型场景**:
- 有序数组两数之和
- 反转字符串
- 接雨水
- 最大水容量

**例:有序数组两数之和**(O(n)):

```typescript
function twoSum(arr: number[], target: number): number[] {
  let l = 0, r = arr.length - 1
  while (l < r) {
    const sum = arr[l] + arr[r]
    if (sum === target) return [l, r]
    if (sum < target) l++   // 太小,左指针右移
    else r--                 // 太大,右指针左移
  }
  return []
}
```

**关键单调性**:**有序**才能让指针单向移动而不漏解。无序数组得用哈希。

### 3. 分离指针(两个数组)

两个指针**各管一个数组 / 链表**。

**典型场景**:
- 合并两个有序数组 / 链表
- 找两个数组的交集 / 并集

**例:合并两个有序数组**:

```typescript
function merge(a: number[], b: number[]): number[] {
  const res: number[] = []
  let i = 0, j = 0
  while (i < a.length && j < b.length) {
    if (a[i] <= b[j]) res.push(a[i++])
    else res.push(b[j++])
  }
  return res.concat(a.slice(i), b.slice(j))
}
```

**就是归并排序的合并步骤**——双指针是归并的灵魂。

## 二、滑动窗口:双指针的高阶形态

**滑动窗口 = 同向双指针 + 维护一个窗口的状态**。两个指针 `left` / `right` 都向右走,中间夹着的就是当前窗口。

```
left          right
  ↓             ↓
[ a  b  c  d  e  f  g ]
   |_________|
       窗口
```

**操作三步**:
1. **扩大右边界**:right++,把新元素加进窗口
2. **判断窗口是否合法 / 满足条件**
3. **必要时收缩左边界**:left++,直到合法

复杂度 **O(n)**:每个元素最多被 right 加入一次、被 left 移出一次。

---

## 三、滑动窗口的两大模板

### 模板 A:定长窗口(窗口大小固定 k)

```typescript
function fixedWindow(arr: number[], k: number) {
  let result = -Infinity, sum = 0
  for (let i = 0; i < arr.length; i++) {
    sum += arr[i]
    if (i >= k) sum -= arr[i - k]   // 移出窗口最左
    if (i >= k - 1) result = Math.max(result, sum)
  }
  return result
}
```

**典型题**:大小为 k 的子数组最大和、最大平均值。

### 模板 B:变长窗口(找满足条件的最长 / 最短)

**这是 80% 滑窗题用的模板**:

```typescript
function variableWindow(s: string): number {
  let left = 0, ans = 0
  const state = /* 某种数据结构,如 Map */ new Map<string, number>()

  for (let right = 0; right < s.length; right++) {
    // 1. 把 s[right] 加入窗口
    state.set(s[right], (state.get(s[right]) ?? 0) + 1)

    // 2. 当窗口违反条件时,收缩左边界
    while (/* 窗口违反条件 */) {
      state.set(s[left], state.get(s[left])! - 1)
      if (state.get(s[left]) === 0) state.delete(s[left])
      left++
    }

    // 3. 更新答案(此时窗口合法)
    ans = Math.max(ans, right - left + 1)
  }

  return ans
}
```

**死记这个骨架**——所有"满足某条件的最长子串"都套这个。

## 四、经典滑窗题

### 1. 无重复字符的最长子串

```typescript
function lengthOfLongestSubstring(s: string): number {
  const seen = new Map<string, number>()
  let left = 0, ans = 0
  for (let right = 0; right < s.length; right++) {
    if (seen.has(s[right]) && seen.get(s[right])! >= left) {
      left = seen.get(s[right])! + 1
    }
    seen.set(s[right], right)
    ans = Math.max(ans, right - left + 1)
  }
  return ans
}
```

**变体写法**:用 `Map` 存"最近一次出现的位置",直接跳过去。比标准滑窗更短。

### 2. 最小覆盖子串(经典)

"给字符串 s 和 t,找 s 中包含 t 所有字符的最短子串。"

**模板版**:

```typescript
function minWindow(s: string, t: string): string {
  if (t.length > s.length) return ""
  const need = new Map<string, number>()
  for (const c of t) need.set(c, (need.get(c) ?? 0) + 1)

  let left = 0, valid = 0   // valid = 已满足要求的字符种数
  let start = 0, minLen = Infinity
  const window = new Map<string, number>()

  for (let right = 0; right < s.length; right++) {
    const c = s[right]
    if (need.has(c)) {
      window.set(c, (window.get(c) ?? 0) + 1)
      if (window.get(c) === need.get(c)) valid++
    }

    while (valid === need.size) {
      // 收缩前先记录答案
      if (right - left + 1 < minLen) {
        minLen = right - left + 1
        start = left
      }
      const d = s[left++]
      if (need.has(d)) {
        if (window.get(d) === need.get(d)) valid--
        window.set(d, window.get(d)! - 1)
      }
    }
  }

  return minLen === Infinity ? "" : s.substr(start, minLen)
}
```

**面试高频中的高频**,**模板背下来不要每次现想**。

### 3. 找所有异位词

"在 s 中找所有 p 的异位词的起始下标。"

**定长滑窗 + 字符计数**:窗口大小 = `p.length`,每次比较两个计数表是否相等。

## 五、双指针的常见陷阱

### 陷阱 1:无序数组也想对撞

**对撞指针要求有序**(或某种单调性)。无序数组上指针怎么动都漏解。

### 陷阱 2:滑窗收缩逻辑写反

收缩条件是"**窗口违反**"还是"**窗口合法**"取决于求"最长"还是"最短":
- 求最长:违反时收缩,收缩到合法,**收缩前 / 后哪里更新答案?**
- 求最短:满足时收缩,收缩到不满足,**满足时更新答案**

经典错位:在"最长"的内层 while 里更新答案——错!那时窗口已经无效了。

**心法**:
- **求最长**:在外层(窗口刚扩进新元素后)收缩,然后更新答案
- **求最短**:在内层 while 里收缩,**先记录答案再缩**(因为缩了就不满足了)

### 陷阱 3:状态维护出错

窗口元素进出时,**进的时候加、出的时候减**——忘了减计数,后面状态全错。**进入和退出永远成对**。

### 陷阱 4:while 跟 if 选错

收缩左指针通常是 **while**(可能要连缩多次),不是 **if**(只缩一次)。

## 六、什么时候不用双指针 / 滑窗

不是所有"线性扫描两个变量"都叫双指针。

**用不上双指针的特征**:
- **没有单调性**:窗口扩大时,某个状态量并不单调变化
- **元素能随意增删而非按位置进出**:用哈希表 / 频次更直接

**反例**:无序数组找两数之和——哈希 O(n) 优于排序后双指针 O(n log n)。

## 七、双指针解题模板汇总

| 模式 | 形态 | 经典题 |
| --- | --- | --- |
| 快慢同向 | 一快一慢 | 链表中点 / 判环 / 去重 |
| 对撞双向 | 两端往中 | 反转 / 有序两数和 / 接雨水 |
| 分离双指针 | 各管一个数组 | 合并 / 交集 / 找差异 |
| 定长滑窗 | 窗口固定 | 最大子数组和(定长) |
| 变长滑窗 | 窗口变化 | 最长 / 最短满足条件子串 |

---

## 八、一句话总结

> **双指针把嵌套循环的两层减到一层**——只要问题有单调性,你就能用同向 / 反向 / 定长 / 变长四种形态压到 O(n)。**滑动窗口是它最重要的进阶,模板背一次,一类题全 KO**。
