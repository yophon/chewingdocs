# 排序全家桶

排序是算法里"看起来都会、深问都不会"的领域。会写冒泡是 L0,会写快排是 L1,**懂为什么生产代码的 `sort` 是 Timsort、为什么快排在小数组上切到插入排序、为什么稳定性这么重要**——才是 L2。这一篇横扫八种排序,讲清每种的"用在哪、坑在哪"。

> 一句话先记住:**没有最好的排序,只有最适合场景的排序**。快排平均最快但不稳定,归并稳定但费内存,计数 / 基数能 O(n) 但只对特定输入有效。**生产代码的 sort 都是混合算法**,因为没有银弹。

---

## 一、八种排序速查表

| 算法 | 平均 | 最坏 | 空间 | 稳定 | 原地 |
| --- | --- | --- | --- | --- | --- |
| 冒泡 | O(n²) | O(n²) | O(1) | ✅ | ✅ |
| 选择 | O(n²) | O(n²) | O(1) | ❌ | ✅ |
| 插入 | O(n²) | O(n²) | O(1) | ✅ | ✅ |
| 希尔 | O(n^1.3) | O(n²) | O(1) | ❌ | ✅ |
| 归并 | O(n log n) | O(n log n) | O(n) | ✅ | ❌ |
| 快排 | O(n log n) | O(n²) | O(log n) | ❌ | ✅ |
| 堆排 | O(n log n) | O(n log n) | O(1) | ❌ | ✅ |
| 计数 | O(n+k) | O(n+k) | O(k) | ✅ | ❌ |
| 基数 | O(d(n+k)) | O(d(n+k)) | O(n+k) | ✅ | ❌ |

**重点关注**:**O(n log n) 的三个**(归并 / 快排 / 堆)和 **O(n) 的两个非比较排序**(计数 / 基数)。前两个 O(n²) 排序(冒泡 / 选择)实战不用,**只是教学用具**。

## 二、稳定排序是什么、为什么重要

**稳定**:相等元素**排序前后相对顺序不变**。

```
排序前: [{name:"A", score:80}, {name:"B", score:80}]
稳定排序后: [{A, 80}, {B, 80}]   ← 相对顺序保持
不稳定排序后: 可能是 [{B, 80}, {A, 80}]
```

**实战意义**:**多关键字排序**。比如先按部门排,再按工资排——如果第二次排序不稳定,会打乱第一次的部门序。

**只有稳定排序能"先排次要、再排主要"组合出多关键字效果**。

JS / Java / Python 的内置 sort 都保证稳定。

## 三、O(n²) 三剑客:简单但不实用

### 冒泡

```typescript
function bubbleSort(arr: number[]) {
  for (let i = 0; i < arr.length - 1; i++) {
    let swapped = false
    for (let j = 0; j < arr.length - 1 - i; j++) {
      if (arr[j] > arr[j + 1]) {
        [arr[j], arr[j + 1]] = [arr[j + 1], arr[j]]
        swapped = true
      }
    }
    if (!swapped) break  // 一轮没换就是已经有序
  }
}
```

**唯一优点**:实现最简单。生产代码绝不用。

### 选择

```typescript
function selectionSort(arr: number[]) {
  for (let i = 0; i < arr.length - 1; i++) {
    let minIdx = i
    for (let j = i + 1; j < arr.length; j++) {
      if (arr[j] < arr[minIdx]) minIdx = j
    }
    [arr[i], arr[minIdx]] = [arr[minIdx], arr[i]]
  }
}
```

**特点**:交换次数最少(n 次)。**不稳定**(交换打乱相等元素顺序)。

### 插入

```typescript
function insertionSort(arr: number[]) {
  for (let i = 1; i < arr.length; i++) {
    const cur = arr[i]
    let j = i - 1
    while (j >= 0 && arr[j] > cur) {
      arr[j + 1] = arr[j]
      j--
    }
    arr[j + 1] = cur
  }
}
```

**亮点**:
- **小数组(n < 50)上比快排还快**,因为常数小、没递归
- **接近有序数组上是 O(n)**,因为内层 while 几乎不进
- **稳定** + **原地** + **在线**(可以一边接收一边排)

**所以工业级 sort 在递归到小数组时切到插入排序**——综合最快。

## 四、归并排序:稳定的分治标杆

```typescript
function mergeSort(arr: number[]): number[] {
  if (arr.length <= 1) return arr
  const mid = arr.length >> 1
  const left = mergeSort(arr.slice(0, mid))
  const right = mergeSort(arr.slice(mid))
  return merge(left, right)
}

function merge(a: number[], b: number[]): number[] {
  const res: number[] = []
  let i = 0, j = 0
  while (i < a.length && j < b.length) {
    if (a[i] <= b[j]) res.push(a[i++])  // ≤ 保证稳定
    else res.push(b[j++])
  }
  return res.concat(a.slice(i), b.slice(j))
}
```

**特点**:
- **稳定 + 严格 O(n log n)**,最坏情况也不退化
- **空间 O(n)**(merge 要新数组)
- **天然适合外部排序**(归并不需要随机访问,**磁盘 / 流式数据友好**)
- **JS / Java 的 sort 在对象数组上用归并**(稳定)

**链表排序首选归并**——链表没法 O(1) 随机访问,快排难写,归并自然适配(中点用快慢指针)。

## 五、快速排序:工业默认

```typescript
function quickSort(arr: number[], lo = 0, hi = arr.length - 1) {
  if (lo >= hi) return
  const p = partition(arr, lo, hi)
  quickSort(arr, lo, p - 1)
  quickSort(arr, p + 1, hi)
}

function partition(arr: number[], lo: number, hi: number): number {
  const pivot = arr[hi]
  let i = lo - 1
  for (let j = lo; j < hi; j++) {
    if (arr[j] <= pivot) {
      i++
      [arr[i], arr[j]] = [arr[j], arr[i]]
    }
  }
  [arr[i + 1], arr[hi]] = [arr[hi], arr[i + 1]]
  return i + 1
}
```

### 快排为什么快

- **原地**(空间 O(log n) 是栈深度)
- **常数小**(只有比较和交换,没新内存)
- **缓存友好**(连续访问)

### 快排的两大坑

1. **基准选错最坏 O(n²)**:已排好的数组每次选最后一个为基准,直接退化。
2. **不稳定**。

### 工业级快排的优化

**Java Arrays.sort(int[]) 用 Dual-Pivot Quicksort**(双基准),Vladimir Yaroslavskiy 2009 年发明,**实测比传统快排快 10-30%**。

**通用优化套路**:
- **三数取中**或随机选基准 → 避免最坏
- **小数组切插入排序** → n < 47 时插排更快(JDK 数字)
- **三路快排**(< pivot, = pivot, > pivot) → 大量重复元素时性能保持

## 六、堆排序:O(1) 空间但慢

```typescript
function heapSort(arr: number[]) {
  const n = arr.length
  // 建大根堆 O(n)
  for (let i = (n >> 1) - 1; i >= 0; i--) siftDown(arr, i, n)
  // 逐个取最大放到末尾 O(n log n)
  for (let i = n - 1; i > 0; i--) {
    [arr[0], arr[i]] = [arr[i], arr[0]]
    siftDown(arr, 0, i)
  }
}
```

**特点**:
- **原地 + 严格 O(n log n)**(快排做不到的"严格")
- **不稳定**
- **常数比快排大** + **缓存极不友好**(下标跳着访问)

**实战意义有限**——快排平均更快,极端场景用 introsort(开始快排,递归过深时切堆排),保最坏 O(n log n)。

## 七、非比较排序:O(n) 的奥秘

**比较排序的下界是 O(n log n)**(信息论证明,每次比较只能区分 2 种状态)。
**O(n) 排序必须不靠比较**——靠数据本身的特性。

### 计数排序

**前提**:数据范围小(整数,值域 k)。

```typescript
function countingSort(arr: number[], maxVal: number): number[] {
  const count = new Array(maxVal + 1).fill(0)
  for (const x of arr) count[x]++
  const res: number[] = []
  for (let i = 0; i <= maxVal; i++) {
    for (let j = 0; j < count[i]; j++) res.push(i)
  }
  return res
}
```

**O(n + k)**,k 是值域大小。**值域不大时碾压一切**。

经典应用:**前 k 高频元素**(`O(n + k)` 替代堆 `O(n log k)`)。

### 基数排序

**前提**:整数 / 定长字符串。

按位排序(从低位到高位,**用稳定排序**比如计数排序作为子算法)。

复杂度 **O(d·(n+k))**,d 是位数,k 是每位的取值范围(对十进制是 10)。

**实战场景**:大量定长数据(电话号、ID、IP)。

### 桶排序

把元素分到 m 个桶,每桶内排序后串起来。
**前提**:数据均匀分布。

## 八、生产代码的 sort 是什么

| 语言 / 库 | 实现 | 备注 |
| --- | --- | --- |
| C++ `std::sort` | introsort(快排+堆排+插排) | 不稳定,可指定比较器 |
| C++ `std::stable_sort` | 归并 | 稳定 |
| Java `Arrays.sort(int[])` | Dual-Pivot Quicksort | 不稳定但快 |
| Java `Arrays.sort(Object[])` | **Timsort** | 稳定(对象有 equals/hash) |
| Python `list.sort()` | **Timsort** | 稳定 |
| JS `Array.prototype.sort` | 多数引擎用 Timsort(规范要求稳定) | 稳定 |
| Rust `Vec::sort` | Timsort 变体 | 稳定 |

**Timsort 是什么**:Tim Peters 2002 年为 Python 设计,**结合归并 + 插入 + run 检测**——在"已经有序的片段(run)"上飞快,合并 run 时用归并。**实测在真实数据上比纯快排快**,因为真实数据通常已部分有序。

## 九、面试高频题

### Top K(用堆 / 计数)

数据少 → 全排序 O(n log n)。
数据多但值域小 → 计数。
数据多值域大 → 小根堆,O(n log k)。

### 第 K 大(快速选择)

**快排的 partition 思想**:partition 后基准位置就是它的"全排序后位置"。**只递归包含 K 的那一边**,平均 O(n)。

```typescript
function quickSelect(arr: number[], k: number): number {
  let lo = 0, hi = arr.length - 1
  while (true) {
    const p = partition(arr, lo, hi)
    if (p === k) return arr[p]
    if (p < k) lo = p + 1
    else hi = p - 1
  }
}
```

### 链表排序

**用归并**——快慢指针找中点,递归排序两半,合并。

### 大文件排序

外部排序:**归并**。把文件切成能放进内存的块,每块内部排序后写回,然后多路归并。

## 十、选型决策树

```
要稳定吗?
├─ 是 → 归并 / Timsort
└─ 否 →
    数据范围小吗?
    ├─ 是 → 计数 / 基数
    └─ 否 →
        小数组(n < 50)?
        ├─ 是 → 插入
        └─ 否 →
            最坏要 O(n log n)?
            ├─ 是 → 堆排 / 归并
            └─ 否 → 快排
```

但 **99% 场景直接 `arr.sort()`**——语言库已经选好了最优算法。**自己手写排序的唯二理由**:面试 / 教学,或对特定数据特性做定制(如基数排序整数 ID)。

---

## 十一、一句话总结

> **排序的世界没有银弹**:稳定要归并 / Timsort,极致速度要快排,严格上界要堆,值域小要计数。生产代码用混合算法是因为它们各取所长。**理解每种的取舍**,你才能在见到陌生 sort API 时一眼看出"这个稳不稳"、"对小数组优化没"、"最坏会不会爆"。
