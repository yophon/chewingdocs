# 动态规划入门

动态规划(DP)是算法世界**门槛最高、收益最大**的章节。它不是某个具体算法,是一种**思维方式**——把"枚举所有方案"用"重叠子问题"压缩到多项式时间。学不会的人觉得它玄学,学会的人觉得它机械——**因为 DP 题都有固定的"列状态、写转移、定边界、定顺序"四步法**。

> 一句话先记住:**DP = 递归 + 记忆化**。从暴力递归出发,如果发现"同一个子问题被算了多次",把它缓存起来就是 DP。理解到这一层,你就再也不会"想破头都想不到 DP"——**因为 DP 不是想出来的,是从暴力优化出来的**。

---

## 一、DP 的两个必要条件

不是所有问题都能用 DP。要用 DP,必须同时满足:

### 1. 重叠子问题(Overlapping Subproblems)

**算同一个子问题会出现很多次**。

**反例**:归并排序——拆出来的子问题完全不重合(各管各的一半),所以归并不是 DP,是分治。

**正例**:斐波那契——`fib(n-2)` 在 `fib(n)` 和 `fib(n-1)` 里都会被算,**重叠**。

### 2. 最优子结构(Optimal Substructure)

**原问题的最优解可以从子问题的最优解构造出来**。

**反例**:"找最长不重复路径"在一般图上没有最优子结构(因为子路径最长不等于整体最长)。
**正例**:最短路径——A 到 C 的最短路径包含 A 到中间点 B 的最短路径。

**两个条件都满足才能 DP**。其中"重叠"决定了**用 DP 比暴力快多少**,"最优子结构"决定了**能不能用 DP**。

---

## 二、DP 的四步法

任何 DP 题都按这四步走:

### 1. 定义状态

`dp[i]` 或 `dp[i][j]` 代表什么?——**这是 DP 最难的一步**。

**好状态的标准**:
- 描述清晰(一句话能说清)
- 包含足够的信息让后面的状态能用它
- 不冗余(同样信息不重复存)

### 2. 写转移方程

`dp[i]` 怎么由更小的状态算出?

**思考方向**:**最后一步是什么?上一个状态是哪些可能?**

### 3. 定 base case

最小的状态值是什么?(`dp[0]`、`dp[0][0]`)

### 4. 定遍历顺序

**所有需要的子状态必须先于当前状态被计算**。

## 三、入门题:斐波那契

```
状态:dp[i] = 第 i 个斐波那契数
转移:dp[i] = dp[i-1] + dp[i-2]
base:dp[0] = 0, dp[1] = 1
顺序:从小到大
```

```typescript
function fib(n: number): number {
  if (n <= 1) return n
  const dp = new Array(n + 1)
  dp[0] = 0; dp[1] = 1
  for (let i = 2; i <= n; i++) dp[i] = dp[i - 1] + dp[i - 2]
  return dp[n]
}
```

**滚动优化**:只需要前两个,O(1) 空间:

```typescript
function fib(n: number): number {
  if (n <= 1) return n
  let a = 0, b = 1
  for (let i = 2; i <= n; i++) [a, b] = [b, a + b]
  return b
}
```

## 四、爬楼梯:DP 思维启蒙

"楼梯有 n 级,每次上 1 级或 2 级,有多少种走法?"

**朴素**:递归枚举。
**最后一步思考**:走到第 n 级,**最后一步要么是从 n-1 上来,要么是从 n-2 上来**。
**转移**:`dp[n] = dp[n-1] + dp[n-2]`(竟然是斐波那契)。

**这就是 DP 的核心思维**:**不要从头想"怎么走",而是想"最后一步从哪来"**。

## 五、打家劫舍:状态选择类

"沿街房子各有钱,但**相邻不能同时偷**,问最多偷多少。"

**最后一步思考**:第 i 家**偷或不偷**。

```
状态:dp[i] = 考虑前 i 家时能偷的最大值
转移:
  dp[i] = max(
    dp[i-1],          // 不偷第 i 家
    dp[i-2] + nums[i] // 偷第 i 家(那 i-1 不能偷)
  )
base:dp[0] = nums[0], dp[1] = max(nums[0], nums[1])
```

```typescript
function rob(nums: number[]): number {
  if (nums.length === 0) return 0
  if (nums.length === 1) return nums[0]
  let prev2 = nums[0]
  let prev1 = Math.max(nums[0], nums[1])
  for (let i = 2; i < nums.length; i++) {
    [prev2, prev1] = [prev1, Math.max(prev1, prev2 + nums[i])]
  }
  return prev1
}
```

**心法**:**每个状态只对应一个决策(选 / 不选)**。多决策题型把 max 改成枚举所有选项。

## 六、记忆化搜索 vs 自底向上

DP 有两种实现方式,**等价但风格不同**:

### 1. 自顶向下:递归 + 记忆化

```typescript
function rob(nums: number[]): number {
  const memo = new Map<number, number>()
  function dp(i: number): number {
    if (i < 0) return 0
    if (memo.has(i)) return memo.get(i)!
    const result = Math.max(dp(i - 1), dp(i - 2) + nums[i])
    memo.set(i, result)
    return result
  }
  return dp(nums.length - 1)
}
```

**优点**:从问题定义直接来,**不用担心遍历顺序**。
**缺点**:递归栈空间 O(n),JS / Python 上深度大时会爆。

### 2. 自底向上:递推

(上一节代码)

**优点**:常数小、不爆栈。
**缺点**:遍历顺序要自己想清楚。

**实战推荐**:
- 解题时先用记忆化(直觉自然)
- 通过后改成自底向上(性能更稳)
- 二者本质等价,**先递归想通再 DP 才是正确顺序**

## 七、最长递增子序列(LIS):转移要枚举

"给一个数组,找最长**严格递增**的**子序列**(不要求连续)。"

例:`[10,9,2,5,3,7,101,18]` → `[2,3,7,101]`,长度 4。

```
状态:dp[i] = 以 nums[i] 结尾的最长 LIS 长度
转移:dp[i] = max(dp[j] + 1)  其中 j < i 且 nums[j] < nums[i]
       (没有这样的 j 时,dp[i] = 1)
base:dp[i] = 1
答案:max(dp[0..n-1])
```

```typescript
function lengthOfLIS(nums: number[]): number {
  const n = nums.length
  const dp = new Array(n).fill(1)
  for (let i = 1; i < n; i++) {
    for (let j = 0; j < i; j++) {
      if (nums[j] < nums[i]) dp[i] = Math.max(dp[i], dp[j] + 1)
    }
  }
  return Math.max(...dp)
}
```

**O(n²)**。**进阶 O(n log n) 解法用二分**——维护一个递增数组,每次二分找位置替换,这个数组**长度就是 LIS**。

## 八、零钱兑换:无穷物品

"给一组面额 `coins` 和金额 `amount`,凑出 amount 的最少硬币数(每种无穷多)。"

```
状态:dp[i] = 凑成金额 i 的最少硬币数
转移:dp[i] = min(dp[i - coin] + 1)  对每种 coin
base:dp[0] = 0
```

```typescript
function coinChange(coins: number[], amount: number): number {
  const dp = new Array(amount + 1).fill(Infinity)
  dp[0] = 0
  for (let i = 1; i <= amount; i++) {
    for (const c of coins) {
      if (i - c >= 0) dp[i] = Math.min(dp[i], dp[i - c] + 1)
    }
  }
  return dp[amount] === Infinity ? -1 : dp[amount]
}
```

**完全背包的雏形**。下一篇 DP 进阶讲背包系列。

## 九、DP 解题流程实操

新手做 DP 题的正确流程:

1. **写暴力递归**(就枚举所有方案,先把问题表达清楚)
2. **看有没有重叠子问题**(画递归树,能看到重复)
3. **加记忆化**(在递归基础上加 memo)
4. **改自底向上**(把递归翻译成循环)
5. **优化空间**(滚动数组,只保留近几行)

**90% 的 DP 题按这个流程都能做出来**。**直接想"DP 状态怎么定"** 是新手最大的坑——**应该先想暴力,再优化**。

## 十、DP 不是万能的

**用了 DP 但其实没必要 / 不能 DP**:

- 没有重叠子问题 → 用分治 / 贪心 / 直接遍历
- 没有最优子结构 → DP 给出错误答案
- 状态空间爆炸(n!级) → 即使有 DP 也跑不动,要剪枝 / 启发式

**判断信号**:
- 题问"最大 / 最小 / 总数 / 是否能"——可能是 DP
- 题问"所有方案"——回溯,不是 DP(枚举不了重叠子问题)
- 输入很小(n < 25)——优先想暴力,可能是回溯 / 状压

---

## 十一、一句话总结

> **DP = 暴力递归 + 缓存重叠子问题**。先想暴力,再加 memo,再翻成循环——这条流水线下来,80% 的 DP 题都能做。**别试图直接"想出"DP 状态**,那是反人类的;**从问题定义出发自然推**,DP 才有可学习性。
