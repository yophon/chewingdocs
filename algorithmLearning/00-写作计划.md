# 算法与数据结构系列写作计划

> 来源:`未来系列规划.md` → B. algorithms ⭐⭐⭐⭐⭐。
> 这个系列**不是 LeetCode 刷题册**,是"工程师真用得上的算法心智模型"——让你写代码时直觉就知道复杂度、选数据结构时不用猜、读别人代码时一眼看出套路。
>
> **和刷题网站的根本区别**:刷题网站按题号走,讲"这题怎么做";本系列按**心智模型**走,讲"一类问题为什么长这样、套路是怎么抽象出来的"。刷完 100 道题还懵,是因为没把套路抽成可迁移的结构。

---

## 设计思路

四层结构,从"底层武器库"到"实战工程":

```
基础心智 + 复杂度(01-03)
   → 线性结构(数组 / 链表 / 栈 / 队列 / 哈希)(04-08)
   → 树与图(树 / BST / 堆 / Trie / 并查集 / 图)(09-14)
   → 算法范式(排序 / 二分 / 双指针 / 滑窗 / DP / 回溯 / 贪心 / 分治)(15-22)
   → 工程实战(LRU / 限流 / 一致性哈希 / 布隆过滤器 / 调度)(23-28)
   → 工程算法补充(区间 / 随机 / 跳表 / AC 自动机 / Bitmap-HLL)(29-33)
```

**一个判断标准**:看完某一篇后,你再遇到这类问题时,应该**不是回头翻这篇**,而是**条件反射知道怎么拆**。做不到就是讲得不够本质。

## 和其他系列的边界

| 内容 | 已有系列 | 本系列 |
| --- | --- | --- |
| 限流算法(令牌桶 / 漏桶) | backendLearning 讲过用法 | 讲**实现原理 + 代码** |
| 一致性哈希 | systemDesign 未写 / Redis 提过 | 讲**算法本身 + 虚拟节点** |
| LRU 缓存 | Redis / 操作系统提过 | 讲**数据结构实现** |
| 字符串匹配(KMP / Rabin-Karp) | — | 本系列独讲 |
| 图数据库底层 | — | 只讲图算法,不讲 Neo4j |
| 并发数据结构 | — | 只讲单线程数据结构,并发留给 os/并发系列 |

**不做什么**:
- 不按 LeetCode 编号讲题;**按套路分类讲**
- 不堆证明;复杂度推导只讲直觉 + 关键步骤
- 不讲 ACM 竞赛级技巧(后缀自动机 / 李超线段树之类);**只覆盖工程工程师 80% 会用上的**

## 节奏

```
01-03  心智必看,跳过去后面会"不知道自己不知道"
04-08  线性结构,日常 70% 代码用的就这些
09-14  树和图,复杂业务 / 系统设计的基础
15-22  算法范式,面试硬通货
23-28  工程实战,跟 backend / systemDesign 咬合
29-33  工程补充,把区间、随机、索引、匹配、集合统计补齐
```

**硬指标**:看完 22 篇后,面对一道陌生题,你应该能在 1 分钟内**分类**(这是 DP? 滑窗? BFS?)。分类对了,套路就来了;分类错了,刷再多题也是白刷。
