# 区间查询与范围更新

数组篇里已经讲过前缀和和差分,但那只是入门版。真实工程里经常遇到更麻烦的问题:一边不停更新数据,一边不停查某个范围的统计值。比如"最近 1 小时每分钟请求数之和"、"一批商品统一涨价后查某个类目的总价"、"排行榜里某个分段的积分统计"。这类问题的关键词都是两个字:**区间**。

> 一句话先记住:**区间问题先看两个维度:查询是不是区间,更新是不是区间**。静态区间查询用前缀和;批量区间更新用差分;动态点更新 + 区间查询用树状数组;区间更新 + 区间查询用线段树。

---

## 一、区间问题的四象限

把需求拆成"查询"和"更新"两个维度:

| 场景 | 查询 | 更新 | 首选 |
| --- | --- | --- | --- |
| 查 `[l, r]` 的和,数据不变 | 区间 | 无 | 前缀和 |
| 批量给 `[l, r]` 加值,最后统一出结果 | 最后一次性还原 | 区间 | 差分数组 |
| 每次改一个点,频繁查区间和 | 区间 | 单点 | 树状数组 |
| 每次改一段区间,频繁查区间和 / 最大值 | 区间 | 区间 | 线段树 |

**先把象限分清楚**,后面就不是猜算法,而是套结构。

---

## 二、前缀和:静态区间查询的天花板

前缀和解决的是:

> 数组不变,要频繁查询某个区间的和。

核心公式:

```text
prefix[i] = nums[0] + nums[1] + ... + nums[i - 1]

sum(l, r) = prefix[r + 1] - prefix[l]
```

代码:

```ts
function buildPrefix(nums: number[]): number[] {
  const prefix = new Array(nums.length + 1).fill(0)

  for (let i = 0; i < nums.length; i++) {
    prefix[i + 1] = prefix[i] + nums[i]
  }

  return prefix
}

function rangeSum(prefix: number[], l: number, r: number): number {
  return prefix[r + 1] - prefix[l]
}
```

为什么 `prefix` 长度是 `n + 1`?

因为 `prefix[0] = 0` 能把空前缀也表达出来,查询 `[0, r]` 时不用写特殊分支:

```text
sum(0, r) = prefix[r + 1] - prefix[0]
```

### 二维前缀和

二维表格也一样。比如一张热力图,要查任意矩形区域的总访问量。

```text
sum(x1, y1, x2, y2)
= S[x2 + 1][y2 + 1]
- S[x1][y2 + 1]
- S[x2 + 1][y1]
+ S[x1][y1]
```

多减了一次左上角,所以要加回来。

```ts
function build2DPrefix(grid: number[][]): number[][] {
  const rows = grid.length
  const cols = rows === 0 ? 0 : grid[0].length
  const prefix = Array.from({ length: rows + 1 }, () => new Array(cols + 1).fill(0))

  for (let r = 0; r < rows; r++) {
    for (let c = 0; c < cols; c++) {
      prefix[r + 1][c + 1] =
        grid[r][c] +
        prefix[r][c + 1] +
        prefix[r + 1][c] -
        prefix[r][c]
    }
  }

  return prefix
}

function rectSum(prefix: number[][], r1: number, c1: number, r2: number, c2: number): number {
  return (
    prefix[r2 + 1][c2 + 1] -
    prefix[r1][c2 + 1] -
    prefix[r2 + 1][c1] +
    prefix[r1][c1]
  )
}
```

**前缀和的边界**:只能处理静态数据。数据一旦频繁变化,每次更新都要重算后面的 prefix,最坏 O(n)。

## 三、差分数组:把区间更新压成两个点

差分解决的是:

> 有很多次区间加减,但不需要每次更新后立刻查询。

假设要对 `[l, r]` 统一加 `v`,普通写法是遍历整段 O(r - l + 1)。差分的写法只改两个位置:

```text
diff[l] += v
diff[r + 1] -= v
```

最后做一次前缀和,就能还原所有更新后的数组。

```ts
function applyRangeUpdates(n: number, updates: Array<[number, number, number]>): number[] {
  const diff = new Array(n + 1).fill(0)

  for (const [l, r, delta] of updates) {
    diff[l] += delta
    if (r + 1 < diff.length) diff[r + 1] -= delta
  }

  const result = new Array(n).fill(0)
  let current = 0

  for (let i = 0; i < n; i++) {
    current += diff[i]
    result[i] = current
  }

  return result
}
```

直觉是:从 `l` 开始多一个增量,到 `r + 1` 这个增量消失。

差分适合:

- 批量给一批时间段加权重
- 批量给一段座位 / 房间 / 资源打标
- 先收集所有操作,最后统一计算结果

不适合:

- 每次更新完马上查区间和
- 更新和查询交错出现

这时要上树状数组或线段树。

## 四、树状数组:动态前缀和的极简结构

树状数组(Fenwick Tree)解决的是:

> 单点更新 + 区间查询。

它只会两件事:

- `add(i, delta)`:第 i 个位置加 delta
- `prefix(i)`:查询 `[0, i]` 的前缀和

区间和自然就是:

```text
sum(l, r) = prefix(r) - prefix(l - 1)
```

### lowbit 是什么

树状数组的核心是 `lowbit(x) = x & -x`。

它表示一个位置负责的区间长度:

```text
1  -> 1
2  -> 2
3  -> 1
4  -> 4
5  -> 1
6  -> 2
7  -> 1
8  -> 8
```

不用死背。记住一句话就够:

> `i += lowbit(i)` 是往上更新影响范围,`i -= lowbit(i)` 是往前累加前缀。

### 标准实现

```ts
class FenwickTree {
  private tree: number[]

  constructor(size: number) {
    this.tree = new Array(size + 1).fill(0)
  }

  add(index: number, delta: number): void {
    for (let i = index + 1; i < this.tree.length; i += i & -i) {
      this.tree[i] += delta
    }
  }

  prefix(index: number): number {
    let sum = 0

    for (let i = index + 1; i > 0; i -= i & -i) {
      sum += this.tree[i]
    }

    return sum
  }

  rangeSum(l: number, r: number): number {
    if (r < l) return 0
    return this.prefix(r) - (l === 0 ? 0 : this.prefix(l - 1))
  }
}
```

复杂度:

| 操作 | 复杂度 |
| --- | --- |
| 单点更新 | O(log n) |
| 前缀查询 | O(log n) |
| 区间查询 | O(log n) |
| 空间 | O(n) |

树状数组的优势是代码短、常数小、很适合区间和。

劣势是表达能力有限:它天然适合"可加可减"的统计,不适合区间最大值、最小值、复杂状态合并。

### 区间更新 + 区间和查询

树状数组也能做区间更新,但要用两个树状数组。公式不如线段树直观,但代码仍然比线段树短。

```ts
class RangeFenwick {
  private b1: FenwickTree
  private b2: FenwickTree

  constructor(size: number) {
    this.b1 = new FenwickTree(size + 1)
    this.b2 = new FenwickTree(size + 1)
  }

  addRange(l: number, r: number, delta: number): void {
    this.addPoint(l, delta)
    this.addPoint(r + 1, -delta)
  }

  rangeSum(l: number, r: number): number {
    return this.prefix(r) - (l === 0 ? 0 : this.prefix(l - 1))
  }

  private addPoint(index: number, delta: number): void {
    this.b1.add(index, delta)
    this.b2.add(index, delta * index)
  }

  private prefix(index: number): number {
    return (index + 1) * this.b1.prefix(index) - this.b2.prefix(index)
  }
}
```

如果你只是要"区间加 + 区间和",两个 Fenwick 很香。只要需求变成"区间最大值、区间最小值、区间覆盖、区间懒更新",直接上线段树。

## 五、线段树:区间问题的通用解

线段树解决的是:

> 区间查询 + 区间更新,并且查询逻辑不止是求和。

它把数组递归切成区间树:

```text
                 [0, 7]
          [0, 3]       [4, 7]
       [0,1] [2,3]  [4,5] [6,7]
       ...
```

每个节点维护自己区间的统计值。查询 `[l, r]` 时,只访问和目标区间相交的节点。

### 懒更新

如果给 `[l, r]` 整段加 `delta`,线段树不需要把每个叶子都改掉。它可以先把增量挂在上层节点上:

> 这个区间整体已经加过 delta,子节点先别急着更新,等真的访问到子节点再下推。

这就是 lazy propagation。

### 求和线段树实现

```ts
class SegmentTree {
  private tree: number[]
  private lazy: number[]
  private n: number

  constructor(nums: number[]) {
    this.n = nums.length
    this.tree = new Array(this.n * 4).fill(0)
    this.lazy = new Array(this.n * 4).fill(0)

    if (this.n > 0) {
      this.build(nums, 1, 0, this.n - 1)
    }
  }

  rangeAdd(l: number, r: number, delta: number): void {
    if (this.n === 0 || r < l) return
    this.add(1, 0, this.n - 1, l, r, delta)
  }

  rangeSum(l: number, r: number): number {
    if (this.n === 0 || r < l) return 0
    return this.query(1, 0, this.n - 1, l, r)
  }

  private build(nums: number[], node: number, left: number, right: number): void {
    if (left === right) {
      this.tree[node] = nums[left]
      return
    }

    const mid = Math.floor((left + right) / 2)
    this.build(nums, node * 2, left, mid)
    this.build(nums, node * 2 + 1, mid + 1, right)
    this.tree[node] = this.tree[node * 2] + this.tree[node * 2 + 1]
  }

  private add(node: number, left: number, right: number, l: number, r: number, delta: number): void {
    if (l <= left && right <= r) {
      this.tree[node] += delta * (right - left + 1)
      this.lazy[node] += delta
      return
    }

    this.pushDown(node, left, right)

    const mid = Math.floor((left + right) / 2)
    if (l <= mid) this.add(node * 2, left, mid, l, r, delta)
    if (r > mid) this.add(node * 2 + 1, mid + 1, right, l, r, delta)

    this.tree[node] = this.tree[node * 2] + this.tree[node * 2 + 1]
  }

  private query(node: number, left: number, right: number, l: number, r: number): number {
    if (l <= left && right <= r) {
      return this.tree[node]
    }

    this.pushDown(node, left, right)

    const mid = Math.floor((left + right) / 2)
    let sum = 0

    if (l <= mid) sum += this.query(node * 2, left, mid, l, r)
    if (r > mid) sum += this.query(node * 2 + 1, mid + 1, right, l, r)

    return sum
  }

  private pushDown(node: number, left: number, right: number): void {
    const delta = this.lazy[node]
    if (delta === 0 || left === right) return

    const mid = Math.floor((left + right) / 2)
    const leftNode = node * 2
    const rightNode = node * 2 + 1

    this.tree[leftNode] += delta * (mid - left + 1)
    this.tree[rightNode] += delta * (right - mid)
    this.lazy[leftNode] += delta
    this.lazy[rightNode] += delta
    this.lazy[node] = 0
  }
}
```

线段树可以把"求和"换成很多别的聚合:

- 区间最大值
- 区间最小值
- 区间最大连续子段和
- 区间覆盖状态
- 区间是否全部满足某条件

只要一个大区间的答案能由左右两个小区间合并出来,线段树就能做。

## 六、树状数组 vs 线段树

| 维度 | 树状数组 | 线段树 |
| --- | --- | --- |
| 代码复杂度 | 低 | 中高 |
| 常数 | 小 | 较大 |
| 空间 | O(n) | O(4n) |
| 单点更新 + 区间和 | 很适合 | 可以,但偏重 |
| 区间更新 + 区间和 | 可用两个 BIT | 很自然 |
| 区间最大 / 最小 | 不自然 | 很适合 |
| 复杂区间状态 | 不适合 | 适合 |

**经验法则**:

- 只要是"和"、"计数"、"前缀统计" → 优先树状数组
- 只要出现"区间更新 + 多种查询" → 优先线段树
- 数据完全静态 → 前缀和
- 操作全部收集完再统一计算 → 差分

## 七、工程里的区间问题

### 1. 时间桶统计

把一天切成 1440 个分钟桶,每个桶记录请求数。查 10:00 到 11:00 的请求总数,就是区间和。

如果数据只追加、周期性落盘,前缀和够用。

如果每分钟不断更新、查询也频繁,树状数组更合适。

### 2. 批量权益 / 价格调整

"给 1000 到 5000 号商品统一加 3 元优惠"、"给某段座位统一加价",如果最后统一生成结果,差分数组极其简单。

### 3. 排行榜分段统计

查积分 `[1000, 2000]` 的用户数量,可以把积分离散成桶,再用树状数组维护每个分数的人数。

### 4. 稀疏区间

如果下标范围是 `[0, 10^12]`,但真实点只有几万个,不要直接开数组。先做**坐标压缩**:

```text
真实坐标: 100, 50000000000, 999999999999
压缩后:   0,   1,           2
```

线段树也可以做成动态开点,只在访问到的区间创建节点。

## 八、常见坑

**坑一:闭区间和半开区间混用**

前缀和常用 `[l, r]` 查询,但工程 API 常用 `[start, end)`。先统一约定,不要边写边换。

**坑二:差分数组忘记开 `n + 1`**

`diff[r + 1] -= delta` 需要位置承接"增量结束"。不开 `n + 1`,最后一个区间容易越界。

**坑三:树状数组下标从 1 开始**

外部可以用 0-based,内部必须转成 1-based。`index + 1` 少写一次,结果会全错。

**坑四:线段树 lazy 不下推**

查询或继续更新子区间前,必须把父节点的 lazy 下推。否则父节点是对的,子节点是旧的。

**坑五:把区间结构用于数据库级查询**

内存里的线段树不是数据库索引。数据量上亿、需要持久化、并发写入、按多字段过滤时,该用数据库索引、OLAP 引擎或预聚合表。

---

## 九、一句话总结

> **区间算法不是背四个结构,而是按"查询是否区间、更新是否区间、数据是否动态"做选择**。静态查用前缀和,批量改用差分,动态点改用树状数组,复杂区间改用线段树。选对结构,很多 O(n) 扫描会直接消失。
