# 并查集

并查集(Union-Find / Disjoint Set Union, DSU)是**算法界性价比最高的数据结构**——二三十行代码,支持两个操作:**"合并两个集合"** 和 **"查两个元素在不在同一集合"**,复杂度**近乎 O(1)**。任何涉及"动态连通性 / 分组 / 等价关系"的问题,它都是首选。

> 一句话先记住:**并查集用"树"表示集合,根节点是集合的代表**。合并就是让一棵树的根变成另一棵树的子;查询就是沿着父指针找到根。加上两个优化(**路径压缩** + **按秩合并**),单次操作平均 O(α(n)),α 是反阿克曼函数,**实际值永远 ≤ 4**——几乎就是 O(1)。

---

## 一、并查集解决什么问题

给一堆元素,动态地:
1. **union(a, b)**:把 a 和 b 所在的集合合并
2. **find(a)**:查询 a 属于哪个集合(返回代表元素)
3. 推论:**isConnected(a, b)** = `find(a) === find(b)`

**典型场景**:
- 朋友关系网(谁跟谁通过朋友关系相连)
- 社交网络的群组
- Kruskal 最小生成树
- 图的连通分量
- 等价类划分
- 网络拓扑的连通性

---

## 二、朴素实现

### 用数组存"父亲"

```typescript
class UnionFind {
  parent: number[]

  constructor(n: number) {
    this.parent = Array.from({ length: n }, (_, i) => i)
    // 初始每个元素自成一派,父亲是自己
  }

  find(x: number): number {
    while (this.parent[x] !== x) x = this.parent[x]
    return x
  }

  union(a: number, b: number) {
    const ra = this.find(a), rb = this.find(b)
    if (ra === rb) return
    this.parent[ra] = rb
  }
}
```

初始状态,五棵单节点树:

```
0  1  2  3  4
```

`union(0, 1)` 后:

```
1        2  3  4
|
0
```

再 `union(2, 3)`、`union(1, 2)`:

```
        3       4
       / \
      1   2
      |
      0
```

`find(0)` 沿着父指针走:`0 → 1 → 3`,返回 3。

### 问题:退化成链

最坏情况下,union 一直合并"深的那棵",树变成链,`find` 退化 O(n)。

## 三、优化一:按秩合并(Union by Rank)

**总是把"矮的树挂到高的树下"**,避免树长高。

```typescript
class UnionFind {
  parent: number[]
  rank: number[]  // 每棵树的高度上限

  constructor(n: number) {
    this.parent = Array.from({ length: n }, (_, i) => i)
    this.rank = new Array(n).fill(0)
  }

  find(x: number): number {
    while (this.parent[x] !== x) x = this.parent[x]
    return x
  }

  union(a: number, b: number) {
    const ra = this.find(a), rb = this.find(b)
    if (ra === rb) return
    if (this.rank[ra] < this.rank[rb]) {
      this.parent[ra] = rb
    } else if (this.rank[ra] > this.rank[rb]) {
      this.parent[rb] = ra
    } else {
      this.parent[rb] = ra
      this.rank[ra]++  // 两棵一样高,合完高度 +1
    }
  }
}
```

**效果**:树高不会超过 O(log n)。`find` 变 O(log n)。

### 按大小合并(Union by Size)

另一种等价思路:**把小的挂到大的下**。用 `size` 数组存每棵树的节点数。效果等同。两种实现哪种都行,**现代代码常用 size**,信息更直观。

## 四、优化二:路径压缩(Path Compression)

**每次 find 时,顺手把路径上所有节点直接挂到根**。

```typescript
find(x: number): number {
  if (this.parent[x] !== x) {
    this.parent[x] = this.find(this.parent[x])  // 递归压缩
  }
  return this.parent[x]
}
```

过程:

```
查找前:            查找后:
    root               root
    /                 /|  |
   a                 a b c  x
   |
   b
   |
   c
   |
   x
```

**一次 find 把整条链拍平**,下次 find 同一个元素 O(1)。

### 迭代版(避免栈溢出)

```typescript
find(x: number): number {
  let root = x
  while (this.parent[root] !== root) root = this.parent[root]
  // 二次遍历,把路径上节点全挂到 root
  while (this.parent[x] !== root) {
    const next = this.parent[x]
    this.parent[x] = root
    x = next
  }
  return root
}
```

---

## 五、两个优化合起来:α(n) ≈ O(1)

**按秩合并 + 路径压缩**,单次操作的均摊复杂度是 **O(α(n))**,α 是反阿克曼函数。

α 增长**极其极其慢**:
- α(10⁸⁰) < 5(宇宙原子数量级都进不了 5)
- 工程上**完全当 O(1) 看**

这是算法界的一个神奇事实:**两个简单优化合在一起,得到了近乎常数的复杂度**。

## 六、简化版(只路径压缩,不按秩合并)

单用路径压缩,复杂度也有 O(log n) 均摊,**代码更短**:

```typescript
class UnionFind {
  parent: number[]

  constructor(n: number) {
    this.parent = Array.from({ length: n }, (_, i) => i)
  }

  find(x: number): number {
    if (this.parent[x] !== x) {
      this.parent[x] = this.find(this.parent[x])
    }
    return this.parent[x]
  }

  union(a: number, b: number) {
    const ra = this.find(a), rb = this.find(b)
    if (ra !== rb) this.parent[ra] = rb
  }
}
```

**实战里 95% 场景够用**。面试要写完整版加上按秩。

## 七、经典应用

### 1. 朋友圈数量

"一个 n×n 矩阵,`M[i][j]=1` 表示 i 和 j 是朋友。问一共有几个朋友圈?"

```typescript
function friendCircles(M: number[][]): number {
  const n = M.length
  const uf = new UnionFind(n)
  for (let i = 0; i < n; i++) {
    for (let j = i + 1; j < n; j++) {
      if (M[i][j] === 1) uf.union(i, j)
    }
  }
  const roots = new Set<number>()
  for (let i = 0; i < n; i++) roots.add(uf.find(i))
  return roots.size
}
```

**最直观的并查集应用**:合并所有关系,数有多少棵树。

### 2. Kruskal 最小生成树

```typescript
function kruskal(n: number, edges: [number, number, number][]): number {
  edges.sort((a, b) => a[2] - b[2])  // 按权重排序
  const uf = new UnionFind(n)
  let total = 0
  for (const [u, v, w] of edges) {
    if (uf.find(u) !== uf.find(v)) {
      uf.union(u, v)
      total += w
    }
  }
  return total
}
```

**套路**:按权重排序,贪心选边,**用并查集判环**(两端已经连通就跳过)。

### 3. 岛屿数量(二维网格)

"网格里 `1` 是陆地 `0` 是水,上下左右相邻的 `1` 视为同一岛。问几个岛?"

**两种解法**:
- **DFS / BFS**:更直观,O(m·n)
- **并查集**:把每个 `1` 格子和上下左右的 `1` 格子 union,最后数根

**DFS 更常用**,但动态场景(插入 / 删除陆地)只能并查集。

### 4. 带权并查集(进阶)

每条边带"权重"(比如距离 / 比率),查询两点关系时顺路算权重。

**"方程组等价关系"** 类题目(a/b = 2, b/c = 3, 问 a/c = ?)是并查集的进阶应用——**路径压缩时同时更新权重**。

### 5. 动态连通性

**在线算法**:边一条一条加进来,每加一条都要回答"图连通吗"。
并查集是**天然的在线算法**,每 union 完就可以回答。

**离线算法常用 union-find + 反向思维**:比如"从一个图里不断删边,问每步连通分量数"——正着删没法做,但**倒着看就是加边**,变成标准并查集问题。

## 八、并查集不能做什么

⚠️ **并查集是"单向合并"结构——只能合,不能分**。

**反例**:
- "撤销 union 操作" → 朴素 DSU 做不到
- "判断一个节点在哪两个分量之间" → 并查集只知道同不同,不知道"中间"
- "查找路径" → 并查集只保证连通,不存路径

**需要可撤销的场景**,用**可回滚 DSU**(不做路径压缩 + 按秩合并 + 记栈),或者离线处理成"倒序删边 = 正序加边"。

## 九、工程中的变体

### 1. 加权并查集

每个节点存一个"到父节点的权值",用于方程组类问题。

### 2. 种类并查集

用 `2n` 或 `3n` 大小的数组表示"n 个元素 + 每个元素的几种状态"。

例:食物链题(动物 / 天敌 / 食物三种关系),开 3n 的并查集。

### 3. 离线 DSU

先读完所有查询 / 操作,按特定顺序批处理。常见于"离线区间查询"。

## 十、一句话总结

> **并查集就是"一堆不相交集合,支持合并 + 查代表"**。30 行代码,加上路径压缩 + 按秩合并,单次操作 α(n) ≈ 1。遇到"动态分组"、"连通性"、"等价关系"三件事,第一反应就是并查集——它太好写、太快、太省。
