# 贪心

贪心(Greedy)是**算法里最有欺骗性的范式**——代码三五行,但**证明它对**比写它难十倍。一道题"看起来能贪",其实贪了就错;另一道题"看起来不能贪",其实有数学保证。这一篇不堆贪心套路,讲清两件事:**什么场景能贪**,**怎么用直觉判断"我这个贪心策略对不对"**。

> 一句话先记住:**贪心就是"每步选当前最优",不回头看不试别的**。能用贪心的前提是**问题具有"贪心选择性质"**——局部最优能推出全局最优。这个性质的证明通常靠"交换论证":假设最优解和你的贪心解不同,证明可以换一些步骤把最优解变成贪心解,价值不降。

---

## 一、贪心 vs DP vs 回溯

| 范式 | 思路 | 复杂度 | 何时用 |
| --- | --- | --- | --- |
| 贪心 | 每步选最优 | 通常 O(n log n) | 局部最优 → 全局最优 |
| DP | 枚举所有子问题选最优 | 多项式 | 有重叠子问题 + 最优子结构 |
| 回溯 | 枚举所有方案 | 指数 | 求所有方案 / DP 不适用 |

**贪心是 DP 的特例**——当 DP 的"枚举所有上一步"退化成"只看一种最优上一步"时,就是贪心。

**判断信号**:
- 问最优 + 数据特别大(n > 10⁶) → 大概率贪心(DP 都跑不完)
- 问最优 + 中等数据 → DP
- 问所有方案 → 回溯

## 二、能贪心的两类典型场景

### 类型一:排序后扫一遍

**先把数据按某种规则排序,然后线性扫一遍按规则选**。

例:**会议安排**——给一堆会议时间,问最多能安排多少个不冲突的会议。

**贪心策略**:**按结束时间排序,每次选最早结束的**。

```typescript
function maxMeetings(meetings: [number, number][]): number {
  meetings.sort((a, b) => a[1] - b[1])  // 按结束时间排
  let count = 0, lastEnd = -Infinity
  for (const [start, end] of meetings) {
    if (start >= lastEnd) {
      count++
      lastEnd = end
    }
  }
  return count
}
```

**为什么这么贪是对的**:
- 选最早结束的会议,**留给后面的时间最多**
- 不管选哪个,最优方案里的第一个会议**总能被换成最早结束的而不变差**(交换论证)

**反贪心**:按开始时间排——可能选了一个开始最早但结束很晚的,后面少塞好几个。**错!**

### 类型二:用堆 / 排序维护"动态最优"

每步从一堆候选里**取最大 / 最小的**。

例:**合并 k 个有序数组**——用小根堆每次取最小,O(n log k)。

例:**分发饼干**——每个孩子有胃口,每个饼干有大小,饼干 ≥ 胃口才能分给孩子。问最多分给多少孩子。

```typescript
function findContentChildren(g: number[], s: number[]): number {
  g.sort((a, b) => a - b)
  s.sort((a, b) => a - b)
  let i = 0, j = 0
  while (i < g.length && j < s.length) {
    if (s[j] >= g[i]) i++       // 给孩子 i 一个饼干
    j++                          // 试下一个饼干
  }
  return i
}
```

**双指针 + 排序**:贪心常用组合。

## 三、贪心的"反例感"训练

学贪心最重要的不是"知道哪些题能贪",**是听到一个策略时,能立刻想反例反驳**。

### 反例训练 1:找零钱

"用最少的硬币凑 amount,面额是 [1, 5, 10, 25]"——**贪心(每次拿最大的能拿)成立**。

但如果面额是 **[1, 3, 4]**,凑 6:
- 贪心:4 + 1 + 1 = 3 枚
- 最优:3 + 3 = 2 枚 ✗

**反例直接告诉你**:**任意面额硬币的"找零"问题贪心不对,得 DP**。只有特殊设计的货币(如美元 / 人民币)才能贪。

### 反例训练 2:0/1 背包

"物品有重量和价值,容量 W 内最大价值"——**单位价值最大的优先**这种贪心**不对**。

**反例**:容量 50,物品 (10, 60), (20, 100), (30, 120)。
- 单位价值降序:60/10 > 120/30 > 100/20,先拿 (10,60),再拿 (20,100) → 价值 160
- 但 (20,100) + (30,120) = 220 才是最优

**0/1 背包必须 DP**,部分背包(可切分)才能贪。

### 反例训练 3:任务调度

"有任务有截止时间和耗时,问能完成多少个"——按截止时间排序通常对,但**带"优先级"** 可能就不对。

**心法**:**只要能想出反例就别贪,老老实实 DP 或回溯**。

## 四、贪心的三大经典证明套路

### 1. 交换论证(最常用)

假设最优解 OPT 和贪心解 GREEDY 第一次不同,证明 OPT 中那一步**可以换成贪心选的**而不变差,然后归纳。

例:会议安排——证明"最早结束的会议可以放进任何最优解,且不影响后续"。

### 2. 数学归纳

假设前 k 步贪心都对,证明第 k+1 步也对。**很多排序贪心问题适用**。

### 3. Exchange Argument 反证

反证:假设贪心不是最优。从那个反例里推出矛盾。

**面试基本不要求严格证明**,**你能说出"为什么这么贪"** 就够。

## 五、贪心的几个"必背"模板

### 1. 任务调度类:按截止时间排序

例:**Boats To Save People**——每船限重 limit,每个人体重不同,问最少多少船。

```typescript
function numRescueBoats(people: number[], limit: number): number {
  people.sort((a, b) => a - b)
  let l = 0, r = people.length - 1, count = 0
  while (l <= r) {
    if (people[l] + people[r] <= limit) l++
    r--
    count++
  }
  return count
}
```

**贪心 + 双指针**:最重的尽量配最轻的。

### 2. 区间问题:按起点 / 终点排序

例:**合并区间** / **插入区间** / **不重叠区间数**。

经典贪心——按结束时间排序、按起点排序,**两个排序方向各有用武之地**。

### 3. 跳跃游戏

"数组每个位置代表能跳的最大步数,问能不能跳到末尾。"

**贪心**:维护"当前能到达的最远位置",一遍扫:

```typescript
function canJump(nums: number[]): boolean {
  let maxReach = 0
  for (let i = 0; i < nums.length; i++) {
    if (i > maxReach) return false
    maxReach = Math.max(maxReach, i + nums[i])
  }
  return true
}
```

**O(n)**,完爆 DP 的 O(n²)。

### 4. 分发糖果(双向贪心)

"每个孩子有评分,相邻孩子评分高的要拿更多糖果。问最少多少糖果?"

**先从左扫,保证比左边大的拿更多;再从右扫,保证比右边大的拿更多。取两次的 max**。

```typescript
function candy(ratings: number[]): number {
  const n = ratings.length
  const left = new Array(n).fill(1)
  const right = new Array(n).fill(1)
  for (let i = 1; i < n; i++) {
    if (ratings[i] > ratings[i - 1]) left[i] = left[i - 1] + 1
  }
  for (let i = n - 2; i >= 0; i--) {
    if (ratings[i] > ratings[i + 1]) right[i] = right[i + 1] + 1
  }
  return left.reduce((s, v, i) => s + Math.max(v, right[i]), 0)
}
```

**两次扫描 + 取 max**,经典套路。

### 5. 哈夫曼编码

**用最小堆每次合并两个最小**,构造频率最低的编码最长的编码树。**贪心 + 优先队列的标准应用**。

## 六、贪心常见陷阱

### 陷阱 1:排错了

会议安排按起点排错;跳跃游戏按值排没用——**排序键直接决定贪心策略对不对**。

### 陷阱 2:能贪和不能贪混淆

经典:你以为"按价值降序"在 0/1 背包能贪——错。这种"看起来对"的策略要警惕。

### 陷阱 3:忽略边界

贪心代码很短,**正因为短,边界不容易看**——空数组、单元素、所有元素相等等情况要测。

### 陷阱 4:浮点精度

贪心常涉及"按比率排序"(如分数背包),**浮点比较记得加 eps**。

## 七、什么时候选贪心

判断流程:

```
能想到贪心策略吗?
├─ 不能 → 不是贪心
└─ 能 →
    能想到反例吗?
    ├─ 能 → 不是贪心(改 DP / 回溯)
    └─ 不能 →
        能给出非严格的合理性解释吗?
        ├─ 能 → 写贪心提交
        └─ 不能 → 不放心就先 DP,过了再尝试贪心
```

**面试场景**:**先想 DP 通过,再尝试贪心优化**。优化失败也至少有 DP 答案保底。

---

## 八、贪心 + 数据结构

贪心常和**优先队列 / 排序 / 单调栈**组合:

- 哈夫曼:贪心 + 小根堆
- 任务调度:贪心 + 排序
- 区间合并:贪心 + 排序
- 摘水果(滑窗最值):贪心 + 单调队列
- 接雨水(贪心解法):双指针

**记住一句**:**贪心提供策略,数据结构提供高效执行**。

---

## 九、一句话总结

> **贪心是用"局部最优 = 全局最优"的强约束,换 O(n log n) 的速度**。能贪心的问题都很美,但**判断能不能贪比写贪心代码难十倍**。**先想反例**——想得出来就改 DP,想不出来再贪。
