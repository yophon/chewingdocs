# 分治

分治(Divide and Conquer)是**算法世界最古老、最基础的范式**——把大问题切成同类小问题,**递归解决,再合并**。归并排序、快速排序、二分查找、FFT、Strassen 矩阵乘法、Karatsuba 大数乘法 —— 全是分治的产物。

> 一句话先记住:**分治三步:Divide(分) → Conquer(治) → Combine(合)**。`Combine` 是分治的灵魂——**不同分治算法的差异,主要就在合并方式**。归并慢在 merge,快排几乎不用合,Karatsuba 用三次乘法巧妙合并,FFT 用复数旋转合并——**合得越巧,算法越快**。

---

## 一、分治的标准模式

```
function solve(problem):
    if problem 足够小:
        直接解
    分:把 problem 拆成 k 个子问题
    治:递归解每个子问题
    合:把子问题的解合成原问题的解
```

复杂度递推式:`T(n) = a · T(n/b) + f(n)`
- a:子问题个数
- b:每个子问题规模
- f(n):分 + 合的额外开销

**不同的 a / b / f 决定不同的复杂度**:

| 算法 | 递推 | 复杂度 |
| --- | --- | --- |
| 二分查找 | T(n) = T(n/2) + O(1) | O(log n) |
| 归并排序 | T(n) = 2T(n/2) + O(n) | O(n log n) |
| 快速排序 | T(n) = 2T(n/2) + O(n) 平均 | O(n log n) |
| 二叉树遍历 | T(n) = 2T(n/2) + O(1) | O(n) |
| Karatsuba 乘法 | T(n) = 3T(n/2) + O(n) | O(n^log₂3) ≈ O(n^1.585) |
| Strassen 矩阵 | T(n) = 7T(n/2) + O(n²) | O(n^log₂7) ≈ O(n^2.807) |

**主定理**(直觉版):
- f(n) 比 a · (n/b)^c 大 → 复杂度由 f(n) 决定
- f(n) 比 a · (n/b)^c 小 → 复杂度由子问题主导
- 一样 → 多个 log 因子

---

## 二、分治 vs DP 的根本区别

| 特征 | 分治 | DP |
| --- | --- | --- |
| 子问题独立? | **独立** | **重叠** |
| 是否要缓存 | 不需要 | 需要 |
| 经典代表 | 归并 / 快排 | 斐波那契 / 背包 |

**关键区分**:**子问题之间有没有重叠**。
- 归并排序的两半互不相干 → 分治
- 斐波那契 fib(n-1) 和 fib(n-2) 共用 fib(n-3) → DP

**DP = 分治 + 记忆化**(从这个角度看)。

## 三、归并排序:分治的标杆

15 篇讲过实现,这里看**分治视角**:

```
分:把数组对半切
治:递归排序两半
合:merge 两个有序数组(O(n))
```

`T(n) = 2T(n/2) + O(n)` → **O(n log n)**

**应用**:
1. **链表排序**:链表用快慢指针找中点,递归排,合并。
2. **求逆序对数**:在 merge 时统计"右边比左边小"的对数。
3. **外部排序**:超大数据切块磁盘排序后多路归并。
4. **求最近点对**(2D 平面):分治 + 归并思想。

### 求逆序对(归并的妙用)

```typescript
function reversePairs(nums: number[]): number {
  const tmp = new Array(nums.length)
  function mergeSort(lo: number, hi: number): number {
    if (lo >= hi) return 0
    const mid = (lo + hi) >> 1
    let count = mergeSort(lo, mid) + mergeSort(mid + 1, hi)
    let i = lo, j = mid + 1, k = lo
    while (i <= mid && j <= hi) {
      if (nums[i] <= nums[j]) tmp[k++] = nums[i++]
      else {
        count += mid - i + 1   // 关键!nums[i..mid] 都比 nums[j] 大
        tmp[k++] = nums[j++]
      }
    }
    while (i <= mid) tmp[k++] = nums[i++]
    while (j <= hi) tmp[k++] = nums[j++]
    for (let x = lo; x <= hi; x++) nums[x] = tmp[x]
    return count
  }
  return mergeSort(0, nums.length - 1)
}
```

**O(n log n) 解决逆序对** —— 朴素 O(n²) 在 n = 10⁵ 跑不动,这一招立刻通过。

## 四、快速排序:分治 + 原地

```
分:partition 把数组按基准分两半(左 < pivot < 右)
治:递归两半
合:不用合(因为本身原地)
```

**和归并的对比**:
- 归并:**先 divide 再 combine**(combine 是难点)
- 快排:**先 partition 后递归**(partition 是难点,递归后不用做合)

---

## 五、快速选择:分治找第 K

15 篇提过——快排思想找第 K 大,**只递归包含 K 的那一半**。

`T(n) = T(n/2) + O(n)` → **O(n)** 平均(主定理:f 主导)

**比"全排序后取第 K"快一个 log 因子**。Top K 题如果 K 不固定,这是最优解。

## 六、Karatsuba 大数乘法:分治的炫技代表

朴素整数乘法 O(n²)(n 是位数)。Karatsuba 用分治压到 O(n^1.585)。

**核心点子**:把 x = a·B^(n/2) + b,y = c·B^(n/2) + d。

`x · y = ac·B^n + (ad+bc)·B^(n/2) + bd`

朴素需要 4 次小乘:ac、ad、bc、bd。

**Karatsuba 的发现**:`ad + bc = (a+b)(c+d) - ac - bd`,**只需 3 次乘**(ac, bd, (a+b)(c+d))。

```
T(n) = 3T(n/2) + O(n) → O(n^log₂3) ≈ O(n^1.585)
```

**用于实战**:Python `int` 大数乘法、密码学(RSA)、bignum 库。GMP 库用更快的 Schönhage–Strassen / FFT 乘法 O(n log n log log n)。

## 七、平面最近点对:分治经典几何

"给 n 个二维点,找最近的两个点之间的距离。"

**朴素**:O(n²)。
**分治**:**O(n log n)**:

```
分:按 x 坐标排序后切两半
治:递归求两半内的最近距离 d_left, d_right
合:d = min(d_left, d_right);
   只可能跨分界线的点对才能更近,
   且这些点必定 x 距离 ≤ d → 形成一条窄带
   带内按 y 排序,每个点只要比较 y 距离 ≤ d 的几个邻居
```

合并部分巧妙利用"窄带"和"几何性质"——不是无脑两两比对。**这是分治"合并"环节的典范**。

## 八、分治的应用领域

### 1. 排序 / 查找

归并、快排、快速选择、二分查找。

### 2. 几何

最近点对、凸包(分治版)、平面扫描线(分治变体)。

### 3. 数值计算

FFT(快速傅里叶变换):O(n log n) 多项式 / 卷积乘法。
Strassen 矩阵乘法:O(n^2.807)。
Karatsuba:大数乘法。

### 4. 树与图

主算法用 DFS 的本质就是分治(每个子树独立递归)。
点分治 / 边分治:树上路径统计。

### 5. 并行计算

**分治是并行算法的天然结构**——子问题独立,可以分发到多核 / 多机。
MapReduce / Fork-Join 框架本质都是分治。

## 九、分治的写法陷阱

### 陷阱 1:合并比分还重

**写得不好的合并能让 O(n log n) 退化成 O(n²)**——比如 merge 用了 O(n²) 的写法。

### 陷阱 2:递归深度爆栈

输入极不平衡时(比如快排选基准烂),递归 O(n) 深。**改迭代或随机化基准**。

### 陷阱 3:base case 漏

`if (n == 1) return ...`——少了 base case 直接死递归。

### 陷阱 4:子问题不独立硬当分治

会"重复计算"成指数复杂度。**先想清楚子问题独立性**——重叠就改 DP。

---

## 十、分治 vs 减治

**减治**(Decrease and Conquer)是分治的**简化版**——只递归一边,不合并。

例:**二分查找**严格说是减治(只找一半,不合)。

减治 → 复杂度 `T(n) = T(n/2) + f(n)`,通常 O(log n) 或 O(n)。
分治 → `T(n) = a·T(n/b) + f(n)`,通常 O(n log n) 或更高。

**两者本质都是"递归结构问题",只是子问题数不同**。

---

## 十一、分治 vs 动态规划

**两者都用"子问题"思维**,选哪个看子问题独立性:

```
子问题之间独立?
├─ 是 → 分治
└─ 否(有重叠)→
    子问题状态空间多项式吗?
    ├─ 是 → DP
    └─ 否 → 回溯 / 启发式
```

**例外**:即使子问题独立,如果想"自底向上递推",写法也像 DP——但本质还是分治。

---

## 十二、一句话总结

> **分治是把问题"砍成两半递归"的祖传招数**——`Divide → Conquer → Combine`,合并环节决定了上限。归并是分治的入门、快排是分治 + 原地、Karatsuba / FFT 是分治的炫技。**分治和 DP 的边界,就是子问题独不独立**。
