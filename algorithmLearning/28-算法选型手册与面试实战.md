# 算法选型手册与面试实战

最后一篇,把前 27 篇内容浓缩成一张**"看到题怎么想"** 的心智地图。这不是知识点总结,是**面试和工程实战时直接调用的查找表**——遇到陌生题,你的大脑该按什么路径检索答案。

> 一句话先记住:**算法解题 = 识别模式 → 选数据结构 → 套范式模板**。会模式识别是 L2,会模板是 L1,**两个不会就是 L0**。这一篇专门训练"模式识别"——给关键词,直接弹出"这是什么类型的题"。

## 二、按数据范围反推算法

|  n 范围 | 允许的复杂度 | 算法候选 |
| --- | --- | --- |
| n ≤ 10 | O(n!) | 全排列 / 暴力回溯 |
| n ≤ 20 | O(2ⁿ · n) | 状压 DP / 子集枚举 |
| n ≤ 100 | O(n⁴) | 三维 DP / 区间 DP |
| n ≤ 500 | O(n³) | Floyd / 区间 DP |
| n ≤ 5000 | O(n²) | 双层循环 / DP |
| n ≤ 10⁶ | O(n log n) | 排序 / 堆 / 二分 |
| n ≤ 10⁸ | O(n) | 双指针 / 滑窗 / 哈希 |
| n > 10⁸ | O(log n) / O(1) | 二分 / 数学 / 矩阵快速幂 |

**先看 n,再选算法**——n 给了你一个"算法上限"。
**面试拿到题先问输入范围**,不是装内行,**是真的在做工程判断**。

## 三、按"输入特征"选数据结构

### 1. 数据是数字 + 范围小

→ **数组 / 计数排序 / 桶排序**

### 2. 数据是字符串 + 长度长

→ **KMP / Trie / Rabin-Karp / 后缀数组**

### 3. 数据是流式

→ **滑窗 / 蓄水池 / Count-Min Sketch / HyperLogLog**

### 4. 数据是有序的

→ **二分 / 双指针**

### 5. 数据是图 / 树

→ **BFS / DFS / 并查集 / 树 DP**

### 6. 数据有"区间"概念

→ **排序 + 贪心 / 前缀和 / 差分 / 线段树**

### 7. 数据要"实时第 K 大"

→ **堆 / 平衡 BST / 双堆**

## 四、八大经典模板速记

### 模板 1:滑动窗口(变长)

```
left = 0, ans = 0
for right in range:
    扩大窗口(把 right 加进来)
    while 窗口违反条件:
        缩小窗口(把 left 移出去)
        left++
    更新答案
```

### 模板 2:二分(红蓝染色)

```
lo = 0, hi = n
while lo < hi:
    mid = (lo + hi) >> 1
    if check(mid): hi = mid     # 红色
    else: lo = mid + 1           # 蓝色
return lo
```

### 模板 3:回溯

```
function bt(path, start):
    if 满足结束: result.push([...path]); return
    for choice in choices(start):
        if 不合法: continue
        path.push(choice)
        bt(path, next_start)
        path.pop()
```

### 模板 4:DFS(树)

```
function dfs(node):
    if !node: return baseCase
    left = dfs(node.left)
    right = dfs(node.right)
    return combine(node, left, right)
```

### 模板 5:BFS(图)

```
queue = [start]
visited.add(start)
while queue:
    size = queue.length
    for i in size:
        node = queue.shift()
        for next in neighbors(node):
            if next not in visited:
                visited.add(next)
                queue.push(next)
```

### 模板 6:DP(自底向上)

```
dp = array of base values
for i in range:
    for state transition:
        dp[i] = combine(dp[smaller])
return dp[n]
```

### 模板 7:并查集

```
parent = [i for i in range(n)]

function find(x):
    if parent[x] != x: parent[x] = find(parent[x])
    return parent[x]

function union(a, b):
    parent[find(a)] = find(b)
```

### 模板 8:拓扑排序(Kahn)

```
inDeg = count incoming edges for each node
queue = nodes with inDeg == 0
result = []
while queue:
    node = queue.shift()
    result.push(node)
    for next in node's edges:
        inDeg[next]--
        if inDeg[next] == 0: queue.push(next)
return result if length == n else "有环"
```

**这八个模板倒背如流**,你已经能搞定面试 70% 的算法题。

## 五、面试时的标准流程

### 第一步:复述问题(30 秒)

"我理解一下,题目是说……对吗?"
**避免一开始就跑偏**。

### 第二步:问输入范围(15 秒)

"n 大概多大?数据是什么类型?有特殊约束吗?"
**输入范围决定算法选择,这是工程师本能**。

### 第三步:举例(1 分钟)

"我用一个小例子走一遍。比如……"
**自己走例子能发现 50% 的边界 case**。

### 第四步:暴力解(1 分钟)

"我先想一个朴素解法是这样,复杂度 O(...),不过我觉得能优化"
**永远先报朴素解**——给面试官信号"我能解出来,只是想得更优"。

### 第五步:优化(2-3 分钟)

"想优化的话,可以用 ... 把它降到 O(...)"
**展示思路**,而不是直接写代码。

### 第六步:写代码(5-10 分钟)

边写边讲。**注释每个关键步骤**。

### 第七步:测试(2 分钟)

"我跑一下边界 case……空输入,单元素,极大输入。"
**主动找 bug,比让面试官找好**。

### 第八步:复杂度

"时间 O(...),空间 O(...)"。

**这八步流水线下来,即使没做出来,面试官也能给你一个"思路清晰"的评价**。

## 六、常见反模式

### 反模式 1:闷头写

不沟通直接埋头。**面试不是写代码考试,是沟通考试**。

### 反模式 2:先纠结边界

`i <= n` 还是 `i < n` 想 5 分钟。**先写大方向,边界后补**。

### 反模式 3:不画图

链表 / 树 / 图题不画图,纯靠脑子想。**画图能省 50% 思考时间**。

### 反模式 4:不报朴素解

直接憋"最优解",憋不出来就交白卷。**朴素解是保底,先有解再优化**。

### 反模式 5:抗拒提示

面试官给提示了还坚持自己思路。**提示是免费的方向修正**,接受它。

## 七、最值得花时间的 10 个模板

按 ROI 从高到低排:

1. **滑动窗口(变长)** —— 字符串 / 数组高频
2. **二分(红蓝模板)** —— 一次写对终身受用
3. **DFS 模板** —— 树 / 图无脑套
4. **BFS 模板** —— 最短路径必备
5. **回溯模板** —— 所有"列举所有方案"题
6. **DP 状态定义** —— 知道往哪想
7. **单调栈模板** —— "下一个更大"专用
8. **并查集模板** —— 30 行代码超高 ROI
9. **快慢指针** —— 链表题套路
10. **堆 + 优先队列用法** —— Top K 杀手

**优先把这 10 个练到 5 分钟内能写出**,面试通过率会显著提升。

## 八、工程师视角的"不重要的"

LeetCode 上有但工程少见,可以**优先级低**:

- 后缀数组 / 后缀自动机
- 莫队算法 / 分块

**这些是 ACM / 竞赛级**,后端 / 前端 / 全栈面试基本不考。**别在这上面花太多时间**,优先把上面的 10 个模板练熟。

## 九、坚持训练的方法

### 1. 按套路分组刷,不按编号

刷 10 道滑窗连着刷,不要今天滑窗、明天 DP、后天图。**集中刷一个套路才能形成肌肉记忆**。

### 2. 每道题写"复盘笔记"

题号 + 类型 + 套路 + 复杂度 + 一句话核心思路。**积累 100 道题的笔记 = 你的私人查找表**。

### 3. 一周回顾一次

把上周做的题**关电脑默写一遍**。能默写出来才算真会。

### 4. 模拟面试

跟同行 mock。**有人盯着你写代码** 是另一种能力。
