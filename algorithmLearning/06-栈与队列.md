# 栈与队列

栈和队列是"**看起来像玩具,实战里无处不在**"的两种结构。函数调用、撤销/重做、浏览器前进后退、消息队列、BFS、表达式求值、括号匹配——底层全是栈或队列。更厉害的是它们的**变种**:单调栈能把一类 O(n²) 题压到 O(n),单调队列解滑动窗口最大值只用 O(n)。**会用普通栈队列是入门,用单调版才算懂**。

> 一句话先记住:**栈是 LIFO(后进先出),队列是 FIFO(先进先出),双端队列 Deque 两头都能进出**。底层都可以用数组或链表实现,工程上**Deque 是万能选**——当栈用当队列用都行,性能都是 O(1)。

---

## 一、栈:LIFO 模型

### 概念

```
push 1 → [1]
push 2 → [1, 2]
push 3 → [1, 2, 3]
pop    → [1, 2]   (出的是 3)
```

**最后进的最先出**。像一叠盘子。

### 实现

JS 里 `Array` 天生支持栈:`arr.push()` / `arr.pop()` 都是 O(1)。

```typescript
const stack: number[] = []
stack.push(1)
stack.push(2)
const top = stack[stack.length - 1]  // peek
const popped = stack.pop()
```

Java 里**不要用 `java.util.Stack`**(遗留的、带同步锁),用 `Deque<Integer>` / `ArrayDeque`。

### 经典场景

**1. 括号匹配**:

```typescript
function isValid(s: string): boolean {
  const pairs: Record<string, string> = { ')': '(', ']': '[', '}': '{' }
  const stack: string[] = []
  for (const c of s) {
    if (c === '(' || c === '[' || c === '{') stack.push(c)
    else if (stack.pop() !== pairs[c]) return false
  }
  return stack.length === 0
}
```

**套路**:左括号入栈,右括号看栈顶是否匹配。

**2. 逆波兰表达式 / 计算器**:操作数入栈,遇运算符取栈顶两个计算完再压回去。

**3. 函数调用模拟 / DFS 改迭代**:前一篇讲过,递归→手动栈。

**4. 浏览器前进后退**:两个栈,一个后退一个前进。

## 二、队列:FIFO 模型

```
enqueue 1 → [1]
enqueue 2 → [1, 2]
enqueue 3 → [1, 2, 3]
dequeue   → [2, 3]   (出的是 1)
```

**最早进的最早出**。像排队买饭。

### 实现

**别用 `Array.shift()`**——它是 O(n),每次要把整个数组往前挪。

**正确实现(用双栈)**:

```typescript
class Queue<T> {
  private inStack: T[] = []
  private outStack: T[] = []

  enqueue(x: T) { this.inStack.push(x) }

  dequeue(): T | undefined {
    if (this.outStack.length === 0) {
      while (this.inStack.length > 0) {
        this.outStack.push(this.inStack.pop()!)
      }
    }
    return this.outStack.pop()
  }
}
```

**均摊 O(1)**:每个元素最多被搬一次。

**更简单的方案**:用**双端队列**(Deque)或链表。Java 用 `ArrayDeque`,Python 用 `collections.deque`,JS 没有内置需要手写或用 `Array` + 下标推进。

### 经典场景

**1. BFS**:图 / 树的广度优先遍历,队列是标配。

**2. 消息队列**:Kafka / RabbitMQ / Redis list 这些本质都是队列。

**3. 事件循环**:JS 的事件队列、操作系统的任务队列。

**4. 打印任务排队 / 请求缓冲**。

---

## 三、双端队列(Deque):两头都能动

```
addFirst / removeFirst
addLast  / removeLast
```

能当栈(addLast + removeLast)、能当队列(addLast + removeFirst)、能当**滑动窗口专用结构**(两头都要操作)。

**实现**:
- Java: `ArrayDeque`(推荐)/ `LinkedList`
- Python: `collections.deque`
- JS: 没有内置,项目里用得多就自己写一个环形数组

**选型原则**:**不知道选什么就选 Deque**。栈和队列都是它的特例。

## 四、单调栈:把 O(n²) 压到 O(n)

单调栈是**算法里最讨人喜欢的套路之一**——解法简单、套路死、复杂度优美。

### 经典题:下一个更大元素

给一个数组,对每个元素,找**它右边第一个比它大的元素**。

**暴力**:双层循环 O(n²)。

**单调栈**(O(n)):

```typescript
function nextGreater(nums: number[]): number[] {
  const result = new Array(nums.length).fill(-1)
  const stack: number[] = []  // 存下标,栈里对应的值单调递减

  for (let i = 0; i < nums.length; i++) {
    while (stack.length > 0 && nums[stack[stack.length - 1]] < nums[i]) {
      const idx = stack.pop()!
      result[idx] = nums[i]  // 当前值就是栈顶下标的"下一个更大"
    }
    stack.push(i)
  }
  return result
}
```

### 为什么是 O(n)

每个下标**最多入栈一次、出栈一次**,总操作 2n 次。

### 套路通用模板

- **下一个更大**:栈单调递减,来了更大的就弹出并记录
- **下一个更小**:栈单调递增,来了更小的就弹出并记录
- **上一个更大 / 小**:从右往左扫

### 实战题型

- 柱状图最大矩形(经典难题)
- 接雨水(单调栈解法)
- 每日温度(下一个更高气温)
- 去除重复字母使字典序最小

**看到"下一个 X"这种题,脑子里闪第一个念头就该是"单调栈"**。

## 五、单调队列:滑动窗口最大值专用

另一个利器。经典题:**给一个数组和窗口大小 k,返回每个窗口的最大值**。

**暴力**:O(n·k)。

**单调队列**(O(n)):

```typescript
function maxSlidingWindow(nums: number[], k: number): number[] {
  const deque: number[] = []  // 存下标,对应值单调递减
  const result: number[] = []

  for (let i = 0; i < nums.length; i++) {
    // 1. 弹出过期(超出窗口左边界)
    while (deque.length > 0 && deque[0] <= i - k) {
      deque.shift()
    }
    // 2. 弹出队尾比当前小的(它们永远不可能是最大)
    while (deque.length > 0 && nums[deque[deque.length - 1]] < nums[i]) {
      deque.pop()
    }
    // 3. 入队
    deque.push(i)
    // 4. 窗口形成后记录
    if (i >= k - 1) result.push(nums[deque[0]])
  }
  return result
}
```

**核心直觉**:当窗口里出现一个更大的元素,它前面那些比它小的**永远不可能再是窗口最大值了**,直接扔掉。队列里只留"**可能成为未来最大值的候选**"。

**复杂度 O(n)**:同样每个元素最多进出一次。

**注意**:JS 里的 `shift` 是 O(n),严格说要用链表或循环数组。窗口小时影响不大,大窗口得换实现。

## 六、优先队列(预告)

**优先队列不是队列,是堆**——"先出的是最大/最小的,不是先进的"。每次出队都是 O(log n)。
第 11 篇专讲,这里只是提一句:**名字里有"队列"但不是队列**。

## 七、栈 vs 队列的选择信号

| 场景 | 选栈 / 队列 |
| --- | --- |
| 需要"最近的一个" | 栈 |
| 需要"最早的一个" | 队列 |
| DFS | 栈(或递归) |
| BFS | 队列 |
| 撤销 / 回退 | 栈 |
| 消息 / 任务 | 队列 |
| 滑动窗口求最值 | 单调队列 |
| 找"下一个更大/小" | 单调栈 |
| 不确定 | 双端队列(通吃) |

---

## 八、常见坑

1. **JS 别用 `Array.shift` 做队列**——O(n)。改用双栈或链表,或者用下标不真删。
2. **单调栈 / 队列存下标还是值?**——存下标几乎永远更好,能访问到值也能访问到位置。
3. **Java 的 `Stack` 是废弃的**——用 `Deque<Integer> stack = new ArrayDeque<>()`。
4. **Python 的 `list` 做队列慢**——用 `collections.deque`。

---

## 九、一句话总结

> **栈和队列是结构,单调栈和单调队列是招式**。前者给你数据容器,后者让你把一类题从 O(n²) 降到 O(n)。**面试看到"下一个更大"或"滑动窗口最值",心里不冒出这两个词,算法就没学通**。
