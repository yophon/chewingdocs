# 堆与优先队列

堆是**披着二叉树外皮的数组**——结构上是完全二叉树,存储上是数组,操作上是 O(log n)。用途极其广泛:Top K、中位数、Dijkstra、任务调度、合并 k 个有序链表——**只要你需要"总是拿最大/最小"**,堆就是答案。

> 一句话先记住:**堆是"部分有序"——只保证父子关系(父 ≥ 子 或 父 ≤ 子),不保证兄弟有序**。这就是为什么堆建好是 O(n)(比排序 O(n log n) 快),因为它做的事比排序少得多。

---

## 一、堆的物理结构:数组!

虽然概念上是树,但存储上用数组。**完全二叉树的天然编号**:

```
index:  0  1  2  3  4  5  6
value: 10  8  7  3  5  2  6

         10 (0)
        /     \
      8 (1)   7 (2)
      / \     / \
    3(3) 5(4) 2(5) 6(6)
```

**父子关系全靠下标计算**:

```
父(i) = (i - 1) / 2
左子(i) = 2i + 1
右子(i) = 2i + 2
```

**不需要指针、连续内存、缓存友好**——这是堆比红黑树在"排序"类场景更快的关键。

---

## 二、大根堆 vs 小根堆

- **大根堆**:父节点 ≥ 左右子节点 → 堆顶是最大值
- **小根堆**:父节点 ≤ 左右子节点 → 堆顶是最小值

堆**不保证层内有序**——例子里 8 和 7 的大小关系不固定,但都必须 ≤ 父节点 10。

**用哪种看需求**:
- 要"每次拿最大"→ 大根堆
- 要"每次拿最小"→ 小根堆(最常用,Dijkstra / A*)

---

## 三、核心操作:上浮与下沉

堆只有两个核心操作:插入时**上浮**(sift up),弹出时**下沉**(sift down)。

### 插入(上浮)

```typescript
class MinHeap {
  private data: number[] = []

  push(val: number) {
    this.data.push(val)            // 放末尾
    this.siftUp(this.data.length - 1)
  }

  private siftUp(i: number) {
    while (i > 0) {
      const parent = (i - 1) >> 1
      if (this.data[parent] <= this.data[i]) break  // 父更小,满足小根堆
      [this.data[parent], this.data[i]] = [this.data[i], this.data[parent]]
      i = parent
    }
  }
}
```

**过程**:新元素放在末尾(保持完全二叉树),如果比父小就交换,一路上浮,直到父更小或到顶。

**复杂度**:O(log n),最多走树高那么多层。

### 弹出最小(下沉)

```typescript
pop(): number | undefined {
  if (this.data.length === 0) return undefined
  const top = this.data[0]
  const last = this.data.pop()!
  if (this.data.length > 0) {
    this.data[0] = last
    this.siftDown(0)
  }
  return top
}

private siftDown(i: number) {
  const n = this.data.length
  while (true) {
    const l = 2 * i + 1, r = 2 * i + 2
    let smallest = i
    if (l < n && this.data[l] < this.data[smallest]) smallest = l
    if (r < n && this.data[r] < this.data[smallest]) smallest = r
    if (smallest === i) break
    [this.data[i], this.data[smallest]] = [this.data[smallest], this.data[i]]
    i = smallest
  }
}
```

**过程**:弹顶后,把末尾元素放到顶,然后跟两个子的较小者交换,一路下沉。

**复杂度**:O(log n)。

### peek

```typescript
peek(): number | undefined { return this.data[0] }
```

O(1),直接看顶。

## 四、建堆:O(n),不是 O(n log n)

给一个无序数组,把它变成堆,**从最后一个非叶子节点开始,倒着 `siftDown`**:

```typescript
buildHeap(arr: number[]) {
  this.data = arr.slice()
  for (let i = (arr.length >> 1) - 1; i >= 0; i--) {
    this.siftDown(i)
  }
}
```

**为什么是 O(n) 而不是 O(n log n)**:

- 最底层叶子(n/2 个):不用下沉
- 倒数第二层(n/4 个):下沉 1 层
- 倒数第三层(n/8 个):下沉 2 层
- ...
- 根(1 个):下沉 log n 层

总计:n/4·1 + n/8·2 + n/16·3 + ... = **O(n)**(级数收敛)。

**工程意义**:一次性建大堆比"逐个 push"快一个 log n 因子。

## 五、堆排序:原地 O(n log n)

利用堆的思想做排序:

```typescript
function heapSort(arr: number[]) {
  const n = arr.length
  // 1. 建大根堆
  for (let i = (n >> 1) - 1; i >= 0; i--) siftDown(arr, i, n)
  // 2. 逐个取堆顶,放到末尾
  for (let i = n - 1; i > 0; i--) {
    [arr[0], arr[i]] = [arr[i], arr[0]]  // 堆顶放末尾
    siftDown(arr, 0, i)                   // 只对前 i 个维护堆
  }
}
```

**特点**:
- **原地排序**,空间 O(1)
- 时间稳定 O(n log n),不像快排最坏 O(n²)
- **不稳定排序**(相同元素相对顺序可能变)
- **常数比快排大**,工程里不如快排常用

## 六、优先队列

**"优先队列"这个名字具有欺骗性**——它不是队列,就是堆。"出队"的顺序不是按进入先后,而是按优先级(最大或最小)。

各语言实现:

| 语言 | API |
| --- | --- |
| Java | `PriorityQueue<T>`(小根堆默认) |
| C++ | `priority_queue<T>`(大根堆默认) |
| Python | `heapq` 模块(小根堆,没有类只有函数) |
| JS | 没有内置,要手写或用库 |

**Python 小坑**:`heapq` 要大根堆?**把元素取负存进去**。

```python
import heapq
heap = []
heapq.heappush(heap, -5)
heapq.heappush(heap, -3)
max_val = -heapq.heappop(heap)  # 5
```

## 七、经典应用

### 1. Top K 问题

**从 n 个数里找最大的 k 个**。

**朴素**:全排序 O(n log n)。
**小根堆解**:维护 k 大小的小根堆,遍历数组,比堆顶大就替换。O(n log k)。

```typescript
function topK(nums: number[], k: number): number[] {
  const heap = new MinHeap()
  for (const x of nums) {
    if (heap.size() < k) heap.push(x)
    else if (x > heap.peek()!) {
      heap.pop()
      heap.push(x)
    }
  }
  return heap.toArray()
}
```

**当 n 极大、k 小时**,这是最优解——因为堆里只保留 k 个,空间 O(k)。

### 2. 合并 k 个有序链表

每个链表头放进小根堆,弹出最小者加入结果,把该链表下一个头再放进堆。

**时间 O(n log k)**,远优于朴素合并(O(n·k))。

### 3. 中位数(双堆)

**维护一个大根堆存小半边,一个小根堆存大半边**,平衡时中位数在两个堆顶。

```typescript
class MedianFinder {
  private lo = new MaxHeap()  // 小的一半
  private hi = new MinHeap()  // 大的一半

  addNum(num: number) {
    this.lo.push(num)
    this.hi.push(this.lo.pop()!)       // 最大的挪到大堆
    if (this.hi.size() > this.lo.size()) {
      this.lo.push(this.hi.pop()!)      // 平衡
    }
  }

  findMedian(): number {
    return this.lo.size() > this.hi.size()
      ? this.lo.peek()!
      : (this.lo.peek()! + this.hi.peek()!) / 2
  }
}
```

**O(log n) 插入,O(1) 取中位数**。数据流中位数的经典解。

### 4. Dijkstra 最短路径

用小根堆每次取"当前距离最小的节点"扩展,是 Dijkstra 的标准实现,从 O(V²) 优化到 O((V+E) log V)。**图论篇会再讲**。

### 5. 任务调度

按优先级 / 截止时间排序的任务队列。操作系统的进程调度、应用层的任务队列(Bull / BullMQ)都用堆。

## 八、堆的变种

### 索引堆(Indexed Heap)

普通堆没法"修改中间某个元素的优先级"——要先 O(n) 查位置。
**索引堆**额外维护 `key → index` 的映射,能 O(log n) 修改任意元素。

Dijkstra 加"decrease key" 优化时用。

### 斐波那契堆

**理论很牛**:decrease key O(1) 均摊,push O(1)。
**实战少用**:常数超大,只在算法书里活跃。

### 左偏堆 / 二项堆

支持 O(log n) 合并两个堆。合并操作密集场景下考虑。

## 九、堆 vs BST 选型

| 操作 | 堆 | BST |
| --- | --- | --- |
| 最值 | **O(1)** | O(log n)(找最左 / 右) |
| 插入 | O(log n) | O(log n) |
| 删除最值 | O(log n) | O(log n) |
| 查找任意元素 | O(n) | **O(log n)** |
| 范围查询 | ❌ | ✅ O(log n + k) |
| 有序遍历 | ❌ | ✅ 中序 |

**结论**:
- 只关心最值 → **堆**(常数更小、空间更省、缓存友好)
- 要查找、范围、排序 → **BST / 红黑树**

---

## 十、一句话总结

> **堆是"只保证堆顶正确"的数据结构,O(log n) 插 O(log n) 弹 O(1) 看顶**。Top K、中位数、调度、Dijkstra——但凡需要"不断取最大/最小",堆就是默认解。它的妙处在于**只做必要的事,不多做一点**。
