# 版本复查清单

最后复查:2026-06-08

这页只解决一件事:哪些内容会过期,以后按什么口径复查。

不要把这里写成新闻摘要。复查的目标是发现教程里已经不准确、容易误导、或者需要补充版本边界的地方。

---

## 复查原则

1. 只按官方文档、官方 release note、官方迁移指南和官方商店政策改正文档。
2. 不因为一个新版本刚发布就重写整章。只有 API、命令、默认行为、支持窗口、上架政策、推荐栈发生变化时才改。
3. 快变内容不要写死到细小 patch 版本,除非它和安全漏洞、兼容性或迁移步骤有关。
4. 章节里出现"当前"、"最新"、"推荐"、"主流"时,必须能回到本页找到复查源。
5. 每轮复查只做一个批次,不要顺手重构无关章节。

## 复查操作

每轮按这个顺序做:

```bash
npm run docs:audit-review
npm run docs:check
```

如果改了章节:

1. 在对应章节加一句"截至 YYYY-MM-DD"的版本边界,只加在真正需要的地方。
2. 同步改系列总览或写作计划里的目录说明。
3. 跑 `npm run docs:check`。
4. 提交信息用 `docs: refresh <topic> docs`。
